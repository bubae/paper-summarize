TL;DR

  • 문제: 현대 LVLM(large vision-language model)이 long-context(128K+)를 요구하는 task(long-document VQA, video, agentic workflow)를 다루려면 ##long-context continued pre-training (LongPT)##이 필수다. 그러나 ##어떤 데이터 종류, 어떤 length distribution, 어떤 task mixture, 그리고 short-context data를 섞어야 할지##에 대한 ##실용적 recipe가 거의 공개되지 않음##. 기존 LVLM technical report (Qwen3-VL, GLM-4.5V)는 detail이 limited.
  • 발견 (3가지 ablation): (i) ##long-document VQA가 OCR transcription보다 substantially 더 효과적##, (ii) length distribution에서 ##balanced(pool-native) data가 target-length-focused(long-biased)보다 우월##, (iii) ##retrieval-heavy mixture(8:2 extraction-to-reasoning)##가 최고, ##pure long-document VQA가 short-context 능력을 크게 손상시키지 않음##.
  • 해결: 위 finding을 기반으로 한 final recipe = ##MMProLong##. Qwen2.5-VL-7B에서 ##32K → 128K##로 context extension. 단 ##5B token budget##으로 long-document VQA +7.1% 향상, ##256K/512K에서도 추가 학습 없이 generalize##.
  • 비유: 학생에게 ##두꺼운 교과서(긴 문서)에서 ##아무 부분이나 묻는 질문##(VQA)을 시키는 것이 ##교과서 처음부터 끝까지 음독##시키는 것(OCR transcription)보다 ##훨씬 빠르게 'long document를 다루는 능력'##을 만든다. 다양한 위치에서 핵심 정보를 retrieve하는 연습이 ##generalize##되기 때문.

핵심 기여 (Key Contributions)

  1. LongPT recipe 체계적 ablation: LVLM long-context pre-training의 ##5가지 design axis##(data type, sequence-length distribution, task mixture ratio, RoPE base frequency, short-context preservation)를 ##controlled 5B-token budget##으로 ablate. ##이 영역의 ablation이 처음 공개됨##.
  2. Long-Document VQA가 dominant supervision signal: 5가지 candidate task(extract-single, extract-multi, reasoning, OCR-full, OCR-needle)를 비교. VQA tasks는 ##5–6% absolute improvement##, OCR transcription은 ##17.4% degradation##까지. 그 후 SFT를 거쳐도 OCR-trained checkpoint는 ##약간만 회복##(+3.24%, +1.85%).
  3. Pool-native length distribution이 long-biased보다 우월: counter-intuitive — ##128K에서 평가받으면서도 데이터의 83.9%를 100K+ token에 집중##시키는 것보다 ##documents가 자연스럽게 sampling되는 분포##(23.6%만 100K+)가 더 좋다. ##long-context ability는 specific target length가 아니라 다양한 absolute position과 image-text distance에 걸친 retrieval 일반화##.
  4. Information extraction vs Reasoning 8:2 mixture가 최적: extraction:reasoning ratio를 grid search(0:10 ~ 10:0). ##8:2 가 평균 57.70##, ##single-data setting (10:0) 56.94##보다 우월. ##약간의 reasoning data가 task diversity 보존##.
  5. Short-context data 불필요: LLM LongPT(Fu et al.) 관행과 달리, LVLM에서는 ##pure long-document VQA가 short-context 능력을 보존##. 0% short → long avg 57.70 (최고), short avg 65.48 (vs original 66.47, ##-0.99##만 감소). ##instruction-formatted long data가 instruction-following style을 유지##하기 때문.
  6. MMProLong 모델 출시: Qwen2.5-VL-7B 기반, 5B-token LongPT. ##15B 미만 open-source LVLM 중 best avg (57.70)##. InternVL3-38B, Gemma3-27B 같은 ##훨씬 큰 LVLM도 능가##.
  7. Beyond 128K generalization: ##256K에서 55.09 avg, 512K에서 52.52 avg## (additional training 없이). Original Qwen2.5-VL-7B는 ##19.49 (512K)##로 추락. ##53.80 vs 28.80## 평균 — 25점 격차.
  8. Cross-task transfer: long-document VQA training이 ##MM-NIAH(웹페이지 needle-in-haystack) +29.4 avg##, ##VTCBench(vision-text compression) +4.5##, ##long-video(Video-MME +2.7, MLVU +3.3, LongVideoBench +1.6)## 모두 향상. ##도메인-specific training 없이## general long-context capability 학습.

배경 (Background)

왜 long-context LVLM이 필요한가

2025–26년 LVLM(Qwen3-VL, GLM-4.5V, Gemini 3 Pro, GPT-5.4) 모두 ##128K+ context window##를 지원. 이는 다음 capability를 unlock:

  • Multi-hop reasoning over document collections (Wang et al. 2024).
  • Spatiotemporal dependency in hour-long videos (Zhou et al. 2024).
  • Long-horizon agent task context consistency.

두 가지 context extension 접근

접근설명한계
Positional extrapolationRoPE base frequency 조정, attention modification##데이터 학습 없음##, true 능력은 미발달
Continued pre-training (LongPT)긴 시퀀스 데이터로 추가 학습##데이터 design이 critical##, 공개된 recipe 부재

LVLM LongPT recipe의 공개 부재

최근 frontier LVLM technical report들이 모두 long-context 능력을 주장하지만 ##구체적 data recipe는 limited##:

  • Qwen3-VL: high-level data category만 보고.
  • GLM-4.5V: 학습 단계만 list, mixture ratio 미공개.
  • Recent concurrent work(다른 paper)이 long-doc data 구성을 다루지만 ##이미 128K+ supports하는 backbone##(Qwen3-VL, Mistral 3.1) 위에서 — ##true context extension##이 아니라 ##context alignment##에 가까움.

본 paper는 ##native 32K context인 Qwen2.5-VL을 직접 확장##해 ##true LongPT##를 study.

방법: Multimodal Long-Context Data Curation

4.1 Document Pool Construction

##1.5M PDF-formatted documents## (academic papers, books, technical manuals, engineering/medicine/social science/biology 도메인). Data synthesis 시 ##32–50 page document##만 선택 — Qwen2.5-VL의 2×2-pixel unshuffle 후 ##32K–128K multimodal tokens##.

##Evaluation contamination 방지##: SHA-256 hash로 evaluation benchmark와 overlap 필터링. 각 PDF page를 ##DPI=144 image로 렌더##(PyMuPDF). 그리고 ##Seed 2.0##(ByteDance 자체 OCR expert model)으로 ##layout-aware block (title, section, paragraph, table, caption)## 파싱.

4.2 Long-Document VQA Data Synthesis

Segment-Level Synthesis Pipeline (Figure 1)

[Step 1: Segment Sampling] Full document + OCR expert → parsed structure → 8–15 page coherent segment 선택 (title/section 기반) [Step 2: QA Generation] Document segment + task instruction → Seed 2.0 → QA pair (generation model에 8–15 페이지만 제공 — 비용 효율적) [Step 3: Full-Document Instance] Generated question (예: "What percentage of visitors saw a wolf in 2010?") + Full-document context (32–50 pages) → 단일 long-context training instance → 답은 short segment에서 inferable → 모델은 full long context를 process해야 함

핵심 design challenge: locally valid question이 full document에서 ambiguous할 수 있다

예: "What is the reported revenue?"가 sampled section에선 답이 명확하지만 ##financial report 전체##에선 ##여러 부서/년도##에 revenue가 있어 ambiguous. 해결: ##QA 생성 시 explicit segment anchor 추가## ("in the Introduction section", "on pages 20–25"). 이로써 ##global-context false positive를 방지##.

3가지 long-doc VQA task

TaskEvidence typeExample
extract-singleSingle page factual retrieval"According to the Homemade Bitters recipe on Page 39, how long should the herbs soak in vodka?"
extract-multiMulti-page factual aggregation"Based on Pages 6, 13, and 19, list all risk factors mentioned in the report."
reasoningNumerical/logical operations over extracted info"What is the difference between total consumption and total imports for rice production in 2020?"

4.3 OCR Transcription Data Synthesis

비교를 위한 alternative task category:

  • OCR-full: full document의 모든 text element를 transcribe — dense image-text dependency.
  • OCR-needle: 1–3 pages만 transcribe, 나머지는 distractor — retrieval-style.

비판적 분석 (Critical Analysis)

  • Closed-source와의 큰 격차: GPT-5.5는 90.77, MMProLong은 57.70. ##7B vs frontier 격차가 본질적으로 크다##. paper는 "open-source 중 SOTA"라고 framing하지만 ##absolute capability gap##이 여전.
  • 5B token budget의 임의성: 모든 ablation이 5B token으로 fixed. ##더 큰 budget##(20B, 50B)에서 동일 finding이 hold할지 미검증. budget이 클 때 long-biased distribution이 우월해질 가능성 존재.
  • "Pool-native"의 정확한 정의: pool-native는 ##32–50 page document pool에서 자연스럽게 derive##되지만 그 pool 자체가 한 design choice. 다른 page-count distribution을 가진 pool에서 동일 finding이 hold할지 명확하지 않음.
  • OCR transcription에 대한 fair comparison 의문: OCR-full(SFT 후) +3.2 vs VQA +5-6 — 둘 다 동일 prompt template이 아닐 가능성. OCR이 ##본질적으로 instruction-formatted가 아니므로 instruction-following metric에서 불리##할 수 있다. paper는 이를 인정하지만 ##fairer comparison##(OCR에도 instruction wrapper 추가)이 없음.
  • Document domain 편향: training documents가 ##academic + technical + textbook## 위주. ##conversational, web, code, legal 같은 다른 long-context domain##으로 transfer는 미검증. MM-NIAH가 webpage-based여서 transfer를 보였지만 단일 example.
  • Hyperparameter sensitivity: RoPE base를 1×10^6 → 4×10^6으로 scale. 다른 값에서의 ablation은 Appendix에 있다고 명시되지만 ##sensitivity range가 narrow한지##, ##다른 backbone에서 같은 multiplier가 통할지##는 명확하지 않음.
  • Short-context preservation의 LVLM-specific 주장: paper는 "LLM과 다르다"고 주장하지만 ##정확한 메커니즘##(왜 LVLM은 short capability를 잃지 않는가)에 대한 mechanistic analysis가 부재. instruction-formatted라는 explanation은 plausible하지만 정량 검증 부족.
  • OCR-full +SFT의 saturated 분석 부족: SFT가 OCR-trained 모델을 +3.2까지 회복시키지만 여전히 VQA-trained보다 낮음. SFT를 ##더 많이## 하면 동등해질 가능성이 있음. ##SFT budget을 변화##시킨 ablation 부재.
  • Real-world long-context task와 benchmark 차이: MMLongBench-Doc, LD-URL, SLIDE는 모두 ##document-centric##. ##agentic workflow, long-horizon planning##같은 다른 long-context use case에서의 효과는 추후 검증 필요.

Key Takeaways

  • "Long-context는 specific target length가 아니라 generalizable retrieval": pool-native > long-biased finding은 ##LongPT의 본질적 통찰##. 모델은 ##다양한 absolute position과 image-text distance에 걸친 key-information retrieval##을 학습해야 한다 — 단일 sweet-spot length specialization이 아니다.
  • "Instruction-formatted supervision이 OCR transcription보다 우월": 같은 token budget으로 VQA가 OCR보다 ##5-23 절대 점수## 더 좋음. ##supervision의 task structure가 raw scale보다 중요##. 다른 long-context training 영역(audio, video)에도 transferable.
  • "Information retrieval bottleneck": extraction:reasoning = 8:2가 optimal. ##long-context의 진짜 bottleneck은 reasoning이 아니라 key info retrieval##. reasoning data는 task diversity 보존용 minimal.
  • "LVLM은 LLM과 다른 short-context behavior": pure long-context training이 ##short capability를 -0.99만 감소##. LLM 관행과 반대. ##LVLM이 instruction-formatted data로 train되면 instruction-following style이 자연 보존##.
  • "5B token으로 32K → 128K": ##학원 규모에서 LongPT 재현 가능##한 budget. open-source 진영에 ##frontier long-context capability##를 만드는 path를 제공.
  • "Long-doc training이 long-video로 transfer": 비디오 데이터 없이 +1.6~3.3pp. ##temporal sequence가 multi-page structure와 isomorphic##. 다른 multimodal long sequence task(audio, sensor stream)에 응용 가능.
  • "Beyond-training-length generalization": 128K로 학습한 모델이 256K/512K에서 작동. ##pool-native + RoPE scaling##이 ##train length를 넘어선 extrapolation##을 가능하게 한다.
