UniComp: A Unified Evaluation of Large Language Model Compression via Pruning, Quantization, and Distillation

  • 저자: Jonathan von Rad, Yong Cao, Andreas Geiger
  • 소속: University College London, University of Tübingen / Tübingen AI Center
  • 학회/저널: arXiv preprint, February 2026
  • 연도: 2026
  • 링크: arXiv:2602.09130

TL;DR

  • 문제: 기존 LLM 압축 평가는 knowledge-centric multiple-choice 벤치마크에 편중되어 있어, reasoning, 다국어, instruction following, reliability 등 다양한 능력에 대한 압축 영향이 체계적으로 파악되지 않았다. Pruning, quantization, knowledge distillation을 통합 비교한 연구도 부재.
  • 해결: Performance(knowledge, reasoning, multilingual, instruction following), Reliability(truthfulness, safety, fairness, robustness, privacy, ethics), Efficiency(runtime, inference, compute cost) 3차원 13개 메트릭으로 구성된 통합 평가 프레임워크 UniComp를 제안하고, 40개 이상의 데이터셋에서 6가지 압축 기법을 체계적으로 비교.
  • 왜 작동하는가: 다차원 평가를 통해 단일 벤치마크에서는 드러나지 않는 "knowledge bias" — 지식 태스크는 보존되지만 reasoning/다국어/instruction following은 크게 하락하는 현상 — 을 포착하고, 각 압축 방법의 실질적 trade-off를 드러낸다.
  • 비유: 건강검진에서 혈압(knowledge)만 재면 "건강"으로 판정되지만, CT·MRI·혈액검사(reasoning, reliability, efficiency)를 종합하면 숨은 문제가 드러나는 것과 같다. UniComp는 압축 모델의 종합 건강검진이다.

핵심 기여 (Key Contributions)

  1. UniComp 프레임워크: Pruning, quantization, knowledge distillation을 performance·reliability·efficiency 3차원에서 통합 비교하는 최초의 체계적 프레임워크. 13개 메트릭, 40개 이상의 데이터셋 활용
  2. Knowledge Bias 발견: 압축 시 knowledge-intensive 태스크는 잘 보존되지만, reasoning·multilingual·instruction following은 크게 하락하는 일관적 편향을 식별. 기존 평가가 이를 간과했음을 입증
  3. Reasoning-aware Calibration 효과: Pruning 모델에 reasoning-centric calibration 데이터를 사용하면 reasoning 성능이 최대 50% 개선됨을 발견. 단, quantization에서는 효과 없음

1. 배경: 기존 압축 평가의 한계

LLM 압축 기법 개관

LLM 압축은 크게 세 가지 패러다임으로 나뉜다:

  • Pruning: 가중치를 제거하여 모델 크기를 줄이는 방법. Unstructured (SparseGPT, Wanda)와 semi-structured (2:4 sparsity) 방식이 있다
  • Quantization: 가중치의 수치 정밀도를 낮추는 방법. Weight-only PTQ (Post-Training Quantization)인 GPTQ와 AWQ가 대표적
  • Knowledge Distillation (KD): Teacher 모델에서 student 모델로 지식을 전이하는 방법. Prune→distill 파이프라인(Minitron)이나 soft pruning 기반(Low-Rank Clone)이 있다

기존 평가의 문제점

기존 압축 문헌은 MMLU, ARC 등 knowledge-intensive multiple-choice 벤치마크에 집중하여, 이들에서 "거의 무손실"이라는 결론을 내리는 경향이 있다. 그러나:

  • Reasoning, multilingual, instruction following에 대한 영향은 체계적으로 평가되지 않음
  • Reliability (안전성, 공정성, 견고성 등)에 대한 평가가 거의 없음
  • Pruning, quantization, KD를 동일 조건에서 통합 비교한 연구가 부재
  • 최신 reasoning 모델(CoT)이나 MoE (Mixture-of-Experts) 모델이 포함되지 않음

2. UniComp 프레임워크

3차원 평가 구조

모든 점수는 [0, 100]으로 스케일링되며, Performance와 Reliability는 base 모델 대비 retained performance 비율로 계산한다.

I. Performance (4개 메트릭)

Knowledge ($S_K$): 표준 multiple-choice 벤치마크 (MMLU, ARC-E/C, HellaSwag, PIQA, Winogrande)에서의 정확도 유지율. 각 태스크의 base 대비 비율의 비가중 평균:

$$S_K = \frac{100}{N} \sum_{i=1}^{N} \frac{s_{\text{Comp}_i}^K}{s_{\text{Base}_i}^K} \tag{1}$$

Multilingual & Cultural ($S_{\text{Mul}}$): 다국어 이해(Global-MMLU-Lite, 14개 언어)와 편향 감지(BBQ)의 평균 유지율:

$$S_{\text{Mul}} = \frac{100}{2} \left(\frac{s_{\text{Comp}}^{\text{Lan}}}{s_{\text{Base}}^{\text{Lan}}} + \frac{s_{\text{Comp}}^{\text{Bias}}}{s_{\text{Base}}^{\text{Bias}}}\right) \tag{2}$$

Reasoning ($S_R$): GSM8K (4-shot), MATH-500 (4-shot), GPQA-Diamond (5-shot) 등 challenging 벤치마크에서의 유지율. 수식 (1)과 동일 구조.

Instruction Following ($S_{\text{IF}}$): IFBench에서의 지시 수행 능력 유지율.

II. Reliability (6개 메트릭)

TrustLLM(Huang et al., 2024) 프레임워크를 따르며, 30개 이상의 데이터셋에서 GPT-4o-mini를 judge로 사용하여 평가한다:

메트릭평가 내용구성 요소
$S_{\text{Truth}}$Truthfulness내부/외부 misinformation, hallucination, sycophancy
$S_{\text{Safe}}$Safetyjailbreak 저항, misuse 거부, 과도한 안전 인식
$S_{\text{Fair}}$Fairnessstereotype 인식, 비하, 선호 편향
$S_{\text{Rob}}$Robustness자연 노이즈, OOD (Out-of-Distribution) 저항
$S_{\text{Pri}}$Privacy개인정보 인식, 유출 방지
$S_{\text{Eth}}$Ethicsimplicit/explicit 윤리, 역할 인식

III. Efficiency (3개 메트릭)

Efficiency 메트릭은 방법 간 상대 비교로 정규화한다. 최적 방법에 1을 부여하고, geometric mean으로 bottleneck을 페널티:

각 메트릭은 방법 간 상대 비교로 정규화한다. 각 요소를 최적 방법 기준 $[0, 1]$로 스케일링한 뒤, geometric mean으로 집계하여 bottleneck을 페널티한다:

$$\text{Score} = 100 \times \left(\prod_{j=1}^{M} \frac{x_j}{\max_k x_{k,j}}\right)^{1/M} \tag{5}$$

직관적으로, 한 차원이라도 극단적으로 낮으면 기하평균이 크게 감소하므로, 균형 잡힌 효율성을 보상하는 메트릭이다.

  • Runtime Acceleration ($S_{\text{RA}}$): Throughput과 latency의 기하평균 $S_{\text{RA}} = 100 \times \sqrt{\frac{T}{T_{\max}} \cdot \frac{L_{\min}}{L}}$ (6)
  • Inference Efficiency ($S_{\text{IE}}$): GPU 메모리, 모델 크기, FLOPs의 기하평균 $S_{\text{IE}} = 100 \times \left(\frac{M_{\min}}{M} \cdot \frac{P_{\min}}{P} \cdot \frac{F_{\min}}{F}\right)^{1/3}$ (7)
  • Compute Cost ($S_{\text{CC}}$): 압축 시간과 peak GPU 메모리의 기하평균 $S_{\text{CC}} = 100 \times \sqrt{\frac{t_{\min}}{t} \cdot \frac{m_{\min}}{m}}$ (8)

핵심 발견:

  • ##Knowledge Bias: 모든 압축 방법에서 knowledge 태스크($S_K$)는 86~100% 유지되지만, reasoning은 40~87%로 큰 격차.## 특히 MATH-500이 가장 민감 — pruning에서 5.60~8.40으로 대폭 하락
  • Quantization이 전반적으로 최강: AWQ/GPTQ는 $S_K \approx 1.00$, $S_R \approx 81$~99%, $S_{\text{IF}} \approx 89$~100%로 다른 방법 대비 압도적
  • Pruning은 reasoning에 치명적: 50% pruning에서 MATH-500이 baseline 대비 70~80% 하락. Multi-step reasoning이 파라미터 제거에 매우 민감
  • ##Minitron-Depth vs Width: 후반 layer를 제거하는 Depth 방식이 Width보다 reasoning에서 훨씬 열등## (MATH-500: 5.60 vs 12.40). Later layer가 reasoning에 핵심적임을 시사
  • Low-Rank Clone > Minitron: LRC-4B는 10배 적은 토큰(20B vs 200B)으로 증류했음에도 instruction following과 multilingual에서 Minitron을 능가. Soft pruning이 hard pruning보다 효과적
  • Semi-structured pruning (2:4)은 비경쟁적: Knowledge 이외의 모든 영역에서 심각한 하락. Hardware acceleration 이점에도 불구하고 불리한 trade-off
  • Qwen-2.5-7B가 LLaMA-3.1-8B보다 압축에 강건: 동일 트렌드를 보이지만 degradation 폭이 더 작음

Calibration 데이터의 영향 (Figure 3c)

실험 의도: SparseGPT/Wanda의 기본 calibration 데이터(C4)를 reasoning-centric 데이터(MATH, GSM8K, ARC-c)로 교체하면 reasoning 성능이 개선되는가?

결과:

  • ##Pruned LLaMA-3.1-8B에서 GSM8K 정확도가 36.9% → 55.0%로 향상 (+50% 상대 개선)##
  • Qwen-2.5-7B에서도 일관적 개선 확인
  • 단, quantization에서는 reasoning-aware calibration이 유의미한 개선을 보이지 않음 — calibration 데이터의 영향은 pruning에 특화
  • Knowledge 성능은 calibration 변경으로 하락하지 않음 — reasoning 개선이 knowledge 희생 없이 달성

High- vs Low-Resource 언어

Global-MMLU-Lite에서 고자원/저자원 언어 간 체계적 추가 하락은 관찰되지 않음. 압축이 특정 언어를 차별적으로 손상시키기보다, 전반적인 knowledge 능력을 균등하게 감소시킨다.

4. 비판적 분석

강점

  • 포괄적 프레임워크: Performance·Reliability·Efficiency 3차원에 걸친 13개 메트릭은 기존 어떤 압축 벤치마크보다 넓은 범위. 40개 이상 데이터셋 사용
  • Knowledge bias 발견의 실용적 가치: Knowledge 벤치마크만으로 "거의 무손실"이라는 결론이 얼마나 오해를 유발할 수 있는지 정량적으로 보여줌
  • 공정한 압축 비교: KD를 "teacher에서 student를 직접 도출"하는 경우로 한정하여, 별도 pretrained student와의 불공정 비교를 회피
  • Calibration 효과 분석: Task-specific calibration의 효과를 pruning/quantization 양쪽에서 체계적으로 검증
  • 다양한 아키텍처 포함: Dense, reasoning (Qwen-3, DeepSeek-R1), MoE 등 최신 모델 포함

한계 및 질문

  • 주요 실험 2개 모델 한정: 심층 분석은 LLaMA-3.1-8B와 Qwen-2.5-7B에 집중. Scaling 효과는 perplexity로만 측정하여 reasoning/reliability에 대한 규모별 분석 부족
  • 코드 생성, multi-agent 미포함: 저자도 인정하듯 코드 생성, multi-agent 협업 등 중요 도메인이 벤치마크에 누락
  • QAT (Quantization-Aware Training) 미포함: PTQ만 비교. QAT가 reasoning 보존에 더 효과적일 수 있으나 검토하지 않음
  • Reliability 판정의 GPT-4o-mini 의존: Judge 모델의 편향이 reliability 점수에 영향을 줄 수 있음. Human evaluation과의 상관 검증 없음
  • Standard deviation 미보고: 결과의 통계적 유의성 확인 불가. 특히 reliability 점수의 높은 분산을 고려하면 반복 실험이 중요
  • Efficiency 비교의 공정성: Distillation은 ~140시간/256 GPU vs pruning 41초/1 GPU — 본질적으로 다른 자원 요구를 단일 점수로 비교하는 것이 적절한지 의문

5. Key Takeaways

  • Knowledge 벤치마크만으로 압축 모델을 평가하면 안 된다: "Knowledge Bias" — knowledge-intensive 태스크에서의 강건성이 reasoning, multilingual, instruction following의 심각한 하락을 가린다. 압축 모델을 배포하기 전 reasoning·reliability 평가가 필수적이다.
  • 4-bit Quantization이 현재 가장 실용적인 선택: Performance, reliability, efficiency 모든 차원에서 가장 균형 잡힌 trade-off. 특히 AWQ가 전반적으로 우수하며, 배포 시 "safe default"로 권장.
  • Pruning + Reasoning-aware Calibration의 조합이 유망: C4 기본 calibration 대신 reasoning-centric 데이터로 calibration하면 pruned 모델의 reasoning이 최대 50% 개선. Knowledge 손실 없이 달성 가능.
  • Performance 보존 ≠ Reliability 보존: 성능이 잘 유지되는 방법이 안전성·공정성도 유지한다는 가정은 위험. 두 차원을 독립적으로 평가해야 한다.
  • 모델 아키텍처가 압축 강건성을 결정: Reasoning 모델과 MoE는 dense 모델보다 압축에 훨씬 민감. 모델 선택 시 압축 호환성을 고려해야 한다.
