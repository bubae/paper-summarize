TL;DR

  • 문제: 멀티에이전트 시스템(MAS)에서 에이전트들은 텍스트로 통신한다. 매 라운드 디코딩→재인코딩 비용이 누적되고, 텍스트 토큰을 통해 흐르는 그래디언트는 시스템 전체를 한 덩어리로 학습하기 어렵게 만든다.
  • 해결: ##에이전트 간 잠재 상태(last-layer hidden)를 직접 전달하는 RecursiveLink 모듈을 두고, 시스템 전체를 하나의 재귀 언어모델(RLM)처럼 inner/outer loop로 공동 학습한다.##
  • 왜 작동하는가: 텍스트 매개를 제거하면 디코딩 비용이 사라져 런타임 복잡도가 $\Theta(m|V|d_h+\cdots)$ 에서 $\Theta(md_h^2+\cdots)$ 로 줄고, residual로 분포 차이만 학습하므로 그래디언트 vanishing이 완화되어 라운드를 깊게 쌓아도 학습이 안정적이다.
  • 비유: 회의실에서 매번 말로 옮겨 적던 협업을, 각자의 머릿속 메모(잠재 표현)를 USB로 직접 주고받는 협업으로 바꾼 것.

핵심 기여 (Key Contributions)

  1. 시스템 단위 재귀 프레임워크: 기존의 단일 모델 내부 recursion(LoopLM 등)을 ##멀티에이전트 시스템 전체로 확장한 최초 사례##. 이질 모델들을 하나의 looped 컴퓨테이션으로 묶는다.
  2. RecursiveLink 모듈: 2-layer residual projection으로 (i) 같은 에이전트의 last-layer ↔ input embedding을 잇는 inner link, (ii) 서로 다른 에이전트 간 hidden 차원을 정합하는 outer link를 정의한다. residual 덕분에 분포 차이만 학습.
  3. Inner-Outer Loop 학습 알고리즘: inner loop는 각 에이전트의 latent thought 생성을 cosine alignment로 워밍업하고, outer loop는 시스템 최종 CE loss를 풀 trajectory로 backprop해 모든 outer link를 공동 최적화한다.
  4. 이론적 보증: (i) 런타임이 텍스트 기반 대비 vocab 디코딩 비용 $m|V|d_h$ 를 $md_h^2$ 로 대체하며, (ii) 잠재 통신은 그래디언트 norm을 $\geq \Omega(1-\sqrt{(1/d_h)\log(1/\delta)})$ 수준으로 유지해 텍스트 SFT의 vanishing $O(\epsilon)$ 을 회피한다.
  5. 4가지 협업 패턴 × 9개 벤치마크 검증: Sequential / Mixture / Distillation / Deliberation 모든 패턴에서 평균 ##+8.3% 정확도, 1.2–2.4× 추론 속도, 34.6–75.6% 토큰 절감##.

1. Background — Recursive Language Models와 Latent Reasoning

1.1 Recursive Language Model (RLM)

표준 Transformer는 $L$개 레이어를 한 번 통과해 출력을 낸다. 반면 recursive language model은 동일한 $L$-레이어 스택을 $n$번 반복해 적용한다:

$$H^{(0)} = E, \quad H^{(r)} = f_\theta(H^{(r-1)}),\ r=1,\ldots,n. \tag{2}$$

직관적으로 말하면, 모델 파라미터를 늘리는 대신 같은 모델을 여러 번 통과시켜 reasoning depth를 키우는 방식이다 (test-time scaling의 한 축). LoopLM, Mixture-of-Recursions, Hierarchical Reasoning Model 등이 이 계열이다.

1.2 Latent-space generation

표준 auto-regressive 생성은 매 스텝 last-layer hidden $h_t$ 를 vocabulary로 projection 해 token을 sample 한 뒤 다시 embedding 한다. Latent generation은 vocabulary를 거치지 않고 $h_t$ 를 다음 step의 입력 embedding으로 직접 재사용한다:

$$h_{t+1} = f_\theta([E_{\leq t}; h_t]). \tag{1}$$

저자들은 이 $h_{t+1}$ 을 ##latent thought## 라고 부른다. token 단위 결정성(decoding noise, lossy 압축)이 사라지므로 더 풍부한 정보가 다음 forward pass로 흘러간다.

1.3 LLM 기반 멀티에이전트 시스템 (MAS)

$N$개의 에이전트 $\mathcal{A} = \{A_1, \ldots, A_N\}$ 각각이 LLM $f_{\theta_i}$ 로 동작하는 시스템 $\mathcal{S}$. 본 논문은 4가지 대표적 협업 패턴을 다룬다:

  • Sequential: Planner → Critic → Solver처럼 역할을 단계별로 배치 (chain-of-agents).
  • Mixture: 도메인 specialist (Math/Code/Science)들이 병렬로 풀고 Summarizer가 통합.
  • Distillation: 큰 Expert가 작은 Learner에게 지식을 전달.
  • Deliberation: 내부 사고용 Reflector + 외부 도구 호출용 Tool-Caller가 합의에 도달.

1.4 한계: 텍스트 매개 협업의 두 가지 비용

(i) 런타임 비용: 매 라운드마다 vocabulary projection ($m \cdot |V| \cdot d_h$) → token sampling → re-embedding 오버헤드가 누적. (ii) 학습 비용: 텍스트 토큰을 거친 그래디언트는 categorical sampling을 통과하며 정보가 손실되고, confident token (low entropy)에서는 ##gradient norm이 0에 가까워져 sequential 의존이 깊어질수록 vanish##. 결과적으로 시스템 전체를 하나로 co-optimize 하기 어렵다.

2. Recursive Multi-Agent Evolution: 정의

저자들은 $\mathcal{H} = \{H_1, \ldots, H_N\}$ 을 모든 에이전트의 last-layer 표현 집합으로 정의하고, MAS의 recursive evolution을 다음과 같이 정의한다:

Definition 2.1 (Recursive Multi-Agent Evolution): $\mathcal{S}^{(0)} \xrightarrow{H^{(1)}} \mathcal{S}^{(1)} \xrightarrow{H^{(2)}} \cdots \xrightarrow{H^{(n)}} \mathcal{S}^{(n)}$. 각 라운드 $r$ 마다 모든 에이전트의 잠재 표현이 함께 갱신되어 시스템이 점진적으로 정렬(aligned)된다.

핵심은 한 에이전트가 RLM 한 layer 처럼 행동한다는 관점이다. residual stream이 layer 사이에 흐르듯, 잠재 정보가 에이전트 사이를 흐른다.

3. RecursiveLink 아키텍처

3.1 Inner Link (같은 에이전트 안에서)

각 에이전트 $A_i$ 는 inner RecursiveLink $\mathcal{R}_{\text{in}}$ 을 가지며, last-layer embedding $h$ 를 다음 forward pass의 input embedding으로 매핑한다:

$$\mathcal{R}_{\text{in}}(h) = h + W_2 \sigma(W_1 h), \tag{3}$$

$\sigma(\cdot)$ 는 GELU. residual branch가 원래 의미를 보존하므로 모듈은 ##"분포 차이만"## 학습하면 된다.

3.2 Outer Link (이질 에이전트 사이)

서로 다른 hidden 차원을 가질 수 있는 $A_i \to A_j$ 전환을 위해 residual 분기에 추가 linear $W_3$ 을 둔다:

$$\mathcal{R}_{\text{out}}(h) = W_3 h + W_2 \sigma(W_1 h). \tag{4}$$

직관적으로, $W_3 h$ 가 cross-model의 좌표계 변환을 담당하고, MLP branch가 미세한 분포 보정을 담당한다.

3.3 전체 루프

$A_1$ 이 입력 컨텍스트 embeddings $E_{A_1}$ 으로 시작 → last-layer $h_t$ 생성 → $\mathcal{R}_{\text{in}}$ 으로 input space로 되먹임 → $m$ step의 latent thoughts $H_{A_1}=[h_t,\ldots,h_{t+m}]$ 누적. 그 다음 $\mathcal{R}_{\text{out}}$ 으로 $A_2$ 의 입력 공간으로 변환 → $A_2$ 도 같은 과정. 마지막 $A_N$ 의 latent가 첫 에이전트로 되먹임되며 라운드가 닫힌다. 최종 라운드에서만 $A_N$ 이 vocabulary로 디코딩.

3.4 알고리즘 흐름도

[Round 1] E_A1 ─► A1 ─► h_t ─[Inner Link]─► h_{t+1}, ..., h_{t+m} (latent thoughts H_A1) │ [Outer Link A1→A2] ▼ E_A2 ⊕ R_out(H_A1) ─► A2 ─► H_A2 ─► ... ─► A_N ─► H_AN │ loop back to A1 [Round 2,...,n-1] 동일 구조 반복, S^(r-1) → S^(r) [Round n] (마지막) A_N의 latent를 LM head로 디코딩 → 최종 텍스트 출력 y

4. 학습 알고리즘 (Inner-Outer Loop Training)

4.1 Stage 1 — Inner Loop (각 에이전트별, 병렬)

off-the-shelf LLM은 latent thought를 받도록 사전학습되어 있지 않으므로, 먼저 inner link만 워밍업한다. ground-truth $y$ 의 표준 input embedding $\text{Emb}_{\theta_i}(y)$ 을 타겟으로 두고:

$$\mathcal{L}_{\text{in}} = 1 - \cos\bigl(\mathcal{R}_{\text{in}}(H), \text{Emb}_{\theta_i}(y)\bigr), \tag{5}$$

직관적으로 말하면, 각 에이전트가 다음 step에서 "정답을 input으로 받았을 때의 그 embedding"에 가까운 latent를 만들도록 inner link를 학습시킨다. 디코딩→재인코딩 절차 없이도 의미 있는 latent thought를 생산할 수 있게 만드는 단계.

4.2 Stage 2 — Outer Loop (시스템 전체)

모든 에이전트 파라미터는 freeze하고, 시스템을 $n$ rounds 펼쳐 마지막 라운드 출력의 cross-entropy로 모든 outer link를 공동 최적화한다:

$$\mathcal{L}_{\text{out}} = \mathrm{CE}\bigl(\mathcal{S}^{(n)}(\mathcal{S}^{(n-1)}(\cdots \mathcal{S}^{(1)}(x))), y\bigr). \tag{6}$$

전체 recursive trace를 따라 computation graph가 보존되어, 각 outer link는 글로벌 contribution에 따라 gradient를 받는다. 즉, ##시스템을 단일 거대 model로 학습하는 효과##.

5. 이론적 분석

5.1 런타임 복잡도 (Proposition 3.1)

$N$ 에이전트, 각 $m$ tokens 생성, $t$ context length, hidden $d_h$, vocabulary $|V|$ 일 때:

  • 텍스트 기반 Recursive-MAS: $\Theta(N(m|V|d_h + (t+m)d_h^2 + (t+m)^2 d_h))$
  • RecursiveMAS (잠재 기반): $\Theta(N(md_h^2 + (t+m)d_h^2 + (t+m)^2 d_h))$

실무에서 $d_h \ll |V|$ (보통 $d_h \approx 4096$ vs $|V|\approx 150{,}000$)이므로, vocabulary projection 항 $m|V|d_h$ 가 ##더 가벼운 latent transformation $md_h^2$## 로 대체되는 것이 핵심 이득. 라운드를 깊게 쌓을수록 절감 폭이 커진다.

5.2 그래디언트 안정성 (Theorem 4.1)

token entropy $\leq \epsilon \ll 1$ (모델이 confident할 때) 가정 하에:

  • 텍스트 SFT: $\left\| \dfrac{\partial \mathcal{R}_{\text{text}}(h)}{\partial h} \right\|_2 \leq O(\epsilon) \ll 1$ (vanishing).
  • RecursiveMAS: $\left\| \dfrac{\partial \mathcal{R}(h)}{\partial h} \right\|_2 \geq \Omega\left(1 - \sqrt{\dfrac{1}{d_h}\log\dfrac{1}{\delta}}\right)$ with prob $\geq 1-\delta$. (gradient norm $\approx 1$ 유지)

직관적으로, 텍스트 토큰의 categorical sampling은 confident할수록 미분이 거의 0이지만, residual link $h + W_2\sigma(W_1 h)$ 는 항상 identity term을 통해 직선 gradient path를 제공한다. 따라서 라운드를 깊게 쌓아도 학습이 안정적이다.

8. 비판적 분석

  • 이질 모델 jointly trained vs frozen 모호성: 모든 LLM 파라미터를 freeze하고 RecursiveLink만 학습한다고 명시되어 있으나, latent 공간이 모델마다 다른 embedding 분포를 가질 때 outer link 두 layer로 충분한지에 대한 더 깊은 분석은 부족하다. Hidden 차원이 매우 다른 (예: 2B vs 9B) 조합에서 추가 ablation이 필요해 보인다.
  • 비교 베이스라인의 학습 예산 동등성: Table 3에서 LoRA, Full-SFT는 single agent에 적용되었고 RecursiveMAS는 시스템에 적용되어, "동일 데이터/동일 backbone"이라 하지만 trainable parameter 수와 latency 모두 다르다. RecursiveMAS의 13.12M이 LoRA의 15.92M과 비슷하지만, 이게 정말 fair인지 일부 의문.
  • 그래디언트 안정성 정리의 가정: Theorem 4.1은 "token entropy $\leq \epsilon \ll 1$"가정 (confident token)을 사용. 그런데 어려운 reasoning task에서는 token-level uncertainty가 크기도 한다. 가정이 깨지는 영역에서 이론의 보호막이 어떻게 작용하는지가 약점일 수 있다.
  • Latent 표현의 해석 가능성 손실: 텍스트 매개를 제거하면 중간 reasoning 과정을 사람이 읽을 수 없다—debug, 안전성 모니터링, 사용자 신뢰 관점에서 trade-off. Reasoning trace의 interpretability를 어떻게 회복할지에 대한 논의가 거의 없음.
  • 실용적 deploy 비용: Inner-outer loop 학습은 "일회성"이지만, 새로운 에이전트 조합마다 outer link를 다시 학습해야 한다 (heterogeneous outer link이므로). 진정한 modularity와는 거리가 있다.
  • 텍스트 입력/출력 boundary 가정: 마지막 라운드에서만 디코딩하므로, multi-turn dialogue에서 사용자가 중간 단계 텍스트를 보고 개입하는 시나리오에는 부적합. agentic 응용 중 사용자-인-더-루프 케이스 분석이 빠짐.
  • 토큰 절감 75.6%의 측정 방법: "총 토큰"이 무엇을 포함하는지 명확하지 않다. 중간 라운드 토큰까지 모두 합산한 것이라면 자명한 절감이고, 실제 정보량을 비교하려면 latent dim × m steps 와의 대등 변환이 필요.

9. Key Takeaways (이 논문에서 가져갈 것)

  • "시스템을 한 모델처럼 학습"이라는 발상: 별도 fine-tuning이나 textual feedback 대신, residual projection 한 모듈을 끼워 넣어 ##멀티에이전트를 단일 RLM처럼 backprop##할 수 있다는 점이 가장 중요한 포인트.
  • Latent residual 통신 = vanishing 방지 + 디코딩 비용 제거 동시 달성: 두 효과가 같은 메커니즘에서 나온다는 통찰. 추가 인프라 없이 efficiency와 trainability를 함께 얻는 매우 적은 design pattern.
  • 이질 LLM 조합도 OK: $W_3$ 한 줄로 cross-model bridging이 풀린다. 실무에서 작은 specialist 여러 개 + 큰 일반 모델 조합 시 유용.
  • recursion depth = 새로운 scaling axis: 더 큰 모델 / 더 긴 CoT 대신, 같은 모델을 여러 라운드 돌려도 reasoning depth를 늘릴 수 있다. test-time compute 전략의 또 다른 선택지.
  • "왜 이만큼 절감되는가"의 회계: token = vocabulary projection 비용을 latent dim 비용으로 대체. 디코딩 1회 = $O(m|V|d_h)$ → $O(md_h^2)$. $|V|/d_h \approx 30$ ~ $40$ 이라면 단순 산술로도 60–80% 절감이 자연스럽다.
