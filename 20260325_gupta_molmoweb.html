TL;DR

  • 문제: 가장 성능이 좋은 웹 에이전트들은 비공개 모델, 비공개 학습 데이터, 비공개 레시피에 의존하여, 과학적 이해와 재현성, 커뮤니티 발전이 제한된다. 오픈 웹의 에이전트는 오픈으로 구축되어야 한다.
  • 해결: (1) MolmoWebMix — 100K+ synthetic trajectory + 30K+ human demonstration + atomic skill trajectory + GUI perception 데이터를 혼합한 대규모 오픈 학습 데이터, (2) MolmoWeb — Molmo2 backbone 위에 SFT로 학습한 4B/8B 크기의 fully open visual web agent. 스크린샷만으로 브라우저를 조작하며, HTML/AxTree/API에 접근하지 않는다.
  • 왜 작동하는가: (1) 다양한 생성 파이프라인(AxTree single/multi-agent, node traversal, human)의 합성 trajectory가 서로 다른 강점을 커버하고, (2) GUI perception 데이터(grounding + screenshot QA)가 웹 스크린샷 이해 능력의 기초를 제공하며, (3) vision-only 접근이 DOM의 취약성과 토큰 오버헤드를 회피한다.
  • 비유: 사람이 컴퓨터를 사용하듯이 화면만 보고 클릭/타이핑/스크롤하는 에이전트를 만드는데, 교육 자료로 (1) AI 튜터의 시범(synthetic), (2) 사람의 시범(human), (3) 화면 읽기 연습(GUI perception)을 모두 활용하여, 코드를 들여다보지 않고도 어떤 웹사이트든 조작할 수 있게 된다.

핵심 기여 (Key Contributions)

  1. MolmoWebMix — 완전 오픈 학습 데이터: ##278.5K task trajectory (2.2M steps, 2.6K domains) + 10.5M GUI perception 데이터##를 포함하는 대규모 오픈 데이터셋. Proprietary vision agent로부터의 distillation 없이, AxTree agent와 human demonstration으로 구축.
  2. MolmoWeb — 오픈 visual web agent: ##8B 모델이 GPT-4o SoM agent를 능가##하고, o3/GPT-5 SoM agent에 근접하는 성능. WebVoyager 78.2%, DeepShop 42.3%, WebTailBench 49.5%로 동일 크기 오픈 모델 중 SOTA.
  3. Test-time scaling: Parallel rollout + best-of-N selection으로 pass@4에서 WebVoyager 94.7%, Online-Mind2Web 60.5% 달성. 이는 단일 rollout 대비 20pp+ 향상.
  4. Fully open release: 모델 체크포인트, 학습 데이터, 코드, 통합 평가 harness를 모두 공개하여 재현 가능한 오픈 웹 에이전트 연구를 촉진.

1. Introduction — 배경과 동기

웹 에이전트의 두 가지 접근

웹 에이전트는 크게 두 가지 방식으로 웹페이지를 인식한다:

  • Text-based (AxTree/DOM): Accessibility tree나 HTML 구조를 텍스트로 직렬화하여 LLM에 입력. 정확하지만 웹사이트별로 DOM 구조가 다르고, 동적 렌더링 콘텐츠에서 불완전할 수 있으며, 수만 토큰을 소비한다.
  • Vision-based (Screenshot): 스크린샷 이미지를 VLM (Vision-Language Model)에 입력. 사람이 보는 것과 동일한 인터페이스를 사용하므로 일반성이 높지만, pixel-space에서의 정확한 pointing이 어렵다.

MolmoWeb은 ##vision-only 접근##을 채택한다. 세 가지 이유:

  1. 사람이 웹을 사용하는 방식을 그대로 모방하여 해석 가능성이 높다
  2. DOM 기반 표현의 취약성(웹사이트/프레임워크별 차이, 동적 콘텐츠의 불완전성)을 회피한다
  3. AxTree 입력의 수만 토큰 오버헤드를 피하고, 단일 스크린샷으로 compact한 정보 표현이 가능하다

핵심 차별점: Proprietary distillation 없음

기존 오픈 웹 에이전트(Fara 등)는 proprietary vision agent에서 증류한 데이터를 사용하지만, ##MolmoWeb은 vision-based proprietary agent로부터의 distillation을 사용하지 않는다##. 대신 text-only AxTree agent(Gemini-3-Flash)가 생성한 합성 trajectory와 인간 시연 데이터를 활용한다.

2. MolmoWebMix — 학습 데이터

학습 데이터는 3개 broad category로 구성된다: task trajectory, atomic skill trajectory, GUI perception data.

[Figure 2 설명]: MolmoWebMix의 구성. 좌상단: Screenshot QA (OCR, 추론 질문), 우상단: Grounding (element description → click point), 하단: Task trajectory — "birria taco recipe under 3 hours" 찾기 예시. 각 step에서 thought와 action이 생성되며, goto → click → type → keypress → click → send_msg_to_user 흐름으로 태스크를 완수한다.

2.1 Task Trajectories

AxTree Agent에 의한 합성 trajectory

Gemini-3-Flash-Preview를 backbone으로, AxTree(accessibility tree)를 관찰 공간으로 사용하는 LLM agent가 합성 trajectory를 생성한다. 각 step에서 스크린샷도 동시에 캡처하여, AxTree로 행동을 결정하되 학습 데이터에는 스크린샷을 포함시킨다. Bid-space의 action은 pixel-space로 후처리된다. 성공 여부는 WebVoyager LLM judge로 필터링.

Multi-agent trajectory 생성

더 높은 품질의 trajectory를 위해 3-agent 파이프라인을 설계한다:

  • Planner (Gemini-2.5-Flash): 고수준 목표를 순차적 subgoal로 분해
  • Operator (Gemini AxTree agent): 현재 스크린샷와 subgoal에 기반하여 low-level 브라우저 action 실행
  • Verifier (GPT-4o): 최근 5개 스크린샷을 분석하여 subgoal 달성 여부 판정

이 multi-agent 구조가 단일 AxTree agent보다 높은 task completion rate을 달성 (WebVoyager 78.5 vs 74.4).

Human trajectory

Chrome extension으로 crowdworker의 브라우저 상호작용(click, scroll, keystroke)을 캡처한다. 각 task는 subtask 시퀀스로 분해되어 fine-grained annotation이 가능하다. 각 trajectory는 인간 리뷰를 거쳐 품질을 보장한다.

Node traversal trajectory

500+ 인기 웹사이트에서 breadth-first exploration으로 directed graph를 구축하고, root-to-leaf 경로를 LLM-free 결정론적 프로세스로 replay하여 trajectory를 생성한다. URL 매칭으로 성공 여부를 직접 검증 가능하여 신뢰성이 높다.

2.2 Atomic Skill Trajectories

End-to-end task가 아닌 개별 웹 기술을 학습하기 위한 데이터. Human trajectory의 subtask 분할에서 자동 추출하거나, AxTree agent에 fill_form, find_and_open 등 기술별 지시를 주어 생성한다.

Table 1: Atomic web skills taxonomy

2.3 GUI Perception

Grounding

(스크린샷, element description) → click point pair. AxTree에서 모든 clickable element를 열거하고, accessible name/role로 natural language description을 생성. Ground-truth click 좌표는 bounding box 내에서 clipped Gaussian prior로 샘플링하여 정확한 중심만이 아닌 spatially robust clicking을 학습. ##총 7M+ grounding QA pairs## (3.4M template + 3.8M GPT-5 생성). PixmoPoints 데이터에서 1.1M도 포함.

Screenshot QA

(스크린샷, 질문) → 답변 pair. 395개 웹사이트에서 2,237,252 QA pairs. 54% OCR, 26% affordance (페이지에서 가능한 action), 20% summarization. AxTree 특정 정보(element ID 등)를 참조하는 sample은 제거.

3. MolmoWeb — 모델 아키텍처와 학습

3.1 Architecture

Molmo2 아키텍처 기반 — Qwen3 language model + SigLIP2 vision encoder. 이미지와 텍스트의 interleaved sequence를 처리하는 multimodal language model이다.

3.2 Observation and Action Space

Observation: 각 step $t$에서 현재 브라우저 스크린샷 + task instruction + 최근 10개 action history (URL, 페이지 제목 포함).

Action: 모델은 JSON 형식의 thought + action을 출력한다. Thought는 선택 이유에 대한 자연어 설명, action은 브라우저 조작 명령이다.

Table 3: Action space

ActionDescription
goto(url)Navigate to URL
mouse_click(x, y, ...)Click at viewport coordinate
mouse_drag_and_drop(...)Drag from one coordinate to another
scroll(delta_x, delta_y)Scroll by offset
keyboard_type(text)Type a string
keyboard_press(key)Press a key
scroll_at(x, y, dx, dy)Scroll at a specific coordinate
hover_at(x, y)Hover at a specific coordinate
go_back()Navigate to previous page
new_tab()Open a new browser tab
tab_focus(index)Switch focus to a browser tab
noop(wait_ms)Wait (page load, captcha 등)
send_msg_to_user(msg)Display message to user (task 완료 등)

Mouse action의 좌표는 $[0, 100]$으로 정규화하여 학습하고, 실행 시 viewport pixel로 역정규화한다.

3.3 Training

  • Molmo2의 single-image checkpoint에서 시작하여 SFT (supervised fine-tuning)
  • 모든 데이터 타입(task trajectory, atomic skill, GUI perception)을 단일 학습 단계에서 혼합
  • 64 H100 GPUs, global batch size 128, 최대 50K steps (~3.2 epochs)
  • Vision encoder, language model, adapter를 모두 fine-tune

4. Experiments

4.1 Evaluation Setup

  • 벤치마크 (4개): WebVoyager, Online-Mind2Web, DeepShop, WebTailBench
  • Live 웹사이트에서 평가, 시간에 민감한 task는 날짜 수정
  • 벤치마크별 3회 평가, 최대 30 steps/task
  • LLM judge: GPT-4o (WebVoyager, DeepShop), o4-mini (Online-Mind2Web), WebVoyager judge (WebTailBench)
  • Browserbase 환경에서 실행

4.3 Test-Time Scaling

실험 의도: Inference 시 더 많은 compute를 투입하면 성능이 향상되는지, parallel rollout과 step 수 증가 중 어느 것이 효과적인지 비교한다.

Parallel Rollout + Best-of-N Selection

동일 task에 $k$개 독립 agent를 실행하고 LLM judge로 최선을 선택한다. Pass@k 추정:

$$\widehat{\text{pass@}k} = 1 - \frac{\binom{m-c}{k}}{\binom{m}{k}} \quad (1)$$

$m$개 rollout 중 $c$개 성공, $m=5$로 설정.

[Figure 6 설명]: WebVoyager와 Online-Mind2Web에서의 pass@k 성능. k 증가에 따라 일관된 향상. MolmoWeb-8B는 max 100 steps에서 ##pass@4 기준 WebVoyager 94.7%, Online-Mind2Web 60.5%## 달성. 30 steps에서도 pass@4가 pass@1(100 steps)보다 높아, parallel rollout이 step 수 증가보다 효과적이라는 결과. 이는 single incorrect action으로 인한 compounding error를 parallel rollout이 완화하기 때문.

Inference step 증가

30→100 steps로 증가 시 일관된 개선이 있으나, parallel rollout 3회(총 90 steps)가 단일 100-step run보다 더 효과적 (8B WebVoyager: 86.2% vs 78.2%).

6. Capabilities and Limitations

  • Instruction following: 구체적인 지시에서 잘 작동하나, 모호하거나 많은 필터/제약이 있는 지시에서 성능 저하
  • OCR/Reading: 스크린샷에서 암묵적으로 OCR을 수행하지만, 작은 텍스트나 긴 문단에서 실패 가능
  • Latency: 효율적 trajectory를 학습하나, 모델 inference 자체의 latency 최적화는 미적용. Quantization/pruning으로 개선 가능
  • Thoughts as memory: Thought가 trajectory 중 정보 저장 역할을 하지만 (예: 뉴스 헤드라인 추적), 강건하지 않으며 thought-action 불일치 발생
  • Action space 한계: scroll_at, mouse_drag_and_drop, hover 같은 드문 action에서 어려움. web_search(query) 같은 효율적 action 미포함
  • Error correction: go_back 등으로 일부 복구 가능하나, 동일 action 반복(stuck)에서 벗어나지 못하는 경우 존재

비판적 분석 (Critical Analysis)

강점

  • 완전한 오픈 생태계: 모델, 데이터, 코드, 평가 harness를 모두 공개. 이는 오픈 웹 에이전트 연구의 재현성과 발전에 크게 기여한다.
  • Proprietary distillation 없음: Vision-based proprietary agent로부터 증류하지 않고도 경쟁력 있는 성능을 달성. 이는 데이터 파이프라인 설계와 혼합 전략의 가치를 입증한다.
  • 다양한 데이터 소스의 체계적 분석: Synthetic vs human, 데이터 규모, sampling strategy 등 ablation이 풍부하여 실무적 가이드를 제공한다.
  • Test-time scaling의 실용적 가능성: Pass@4에서 WebVoyager 94.7%는 실제 배포 시나리오에서 parallel rollout 전략의 가치를 보여준다.

약점

  • Synthetic trajectory의 teacher 모델 비용: Gemini-3-Flash, GPT-4o, Gemini-2.5-Flash 등 대형 proprietary 모델을 trajectory 생성에 사용. "Proprietary distillation 없음"이라 하지만, 실질적으로 proprietary 모델에 강하게 의존한다.
  • Non-visual teacher와의 성능 격차: Gemini-3-flash AxTree(85.6 WebV) 대비 MolmoWeb-8B(78.2)는 7pp 격차. Vision-only 제약의 한계가 명확히 드러나며, 이 격차를 줄이기 위한 추가 연구가 필요.
  • Human data의 낮은 기여: 30K+ human trajectory 수집 비용 대비 벤치마크 기여가 미미. Synthetic + human이 synthetic only와 거의 동일 (68.5 vs 67.8 WebV). Human annotation의 ROI에 의문.
  • Live web 평가의 재현성: 실제 웹사이트에서 평가하므로 시간에 따른 변화(UI 업데이트, 콘텐츠 변경)로 재현이 어려움. 3회 반복의 표준편차도 미보고.
  • RL/self-improvement 미적용: SFT만 사용하고 RL이나 self-play/self-improvement는 미적용. 저자도 이를 향후 과제로 인정.

Key Takeaways

  1. Vision-only 웹 에이전트가 실용적 수준에 도달: HTML/DOM 접근 없이 스크린샷만으로 8B 모델이 GPT-4o SoM agent를 능가한다. 이는 well-curated 데이터가 model scale과 input richness를 보완할 수 있음을 보여준다.
  2. Synthetic trajectory가 human demo보다 효과적: ##동일 task에서 합성 데이터로 학습한 모델이 인간 데이터보다 일관되게 우수##. AxTree agent의 구조적 이해가 더 직접적이고 일관된 행동 패턴을 생성하며, human annotation의 noise가 모방 학습을 방해한다.
  3. Parallel rollout이 step 증가보다 효과적: 90 steps (3 parallel × 30) > 100 steps (1 sequential). Single incorrect action의 compounding error를 parallel로 완화하는 것이 더 긴 trajectory를 허용하는 것보다 낫다.
  4. Multi-agent trajectory 생성이 single agent보다 우수: Planner-Operator-Verifier 구조가 단일 AxTree agent 대비 더 높은 task completion rate과 thought 품질을 제공한다.
  5. Atomic skill 분해가 compositional web task 학습을 촉진: End-to-end trajectory뿐 아니라 개별 기술(search, fill_form 등)을 따로 학습함으로써, 새로운 조합의 task에 일반화하는 능력이 향상된다.
