TL;DR

  • 문제: "world model"이라는 용어가 RL, 비전, NLP, 로보틱스, AI for science 등 다양한 분야에서 ##서로 다른 의미##로 쓰이고 있어, 같은 시스템을 한 분야의 기준으로는 호평하고 다른 분야의 기준으로는 실패라 평가하는 단절(fragmentation)이 발생.
  • 해결: ##두 축의 통합 분류체계##—(i) 능력 계층 L1 Predictor → L2 Simulator → L3 Evolver 와 (ii) 4가지 지배 법칙 regime (Physical / Digital / Social / Scientific). 두 축이 직교하므로 모든 기존 시스템을 셀에 매핑할 수 있다.
  • 왜 작동하는가: 각 계층 전환에 ##testable boundary conditions##를 부여하여 (예: L1→L2는 long-horizon coherence + intervention sensitivity + constraint consistency) 모호한 "더 나은 world model"을 측정 가능한 기준으로 환원.
  • 비유: 의학에서 진단을 정리하기 위해 "ICD 코드 + 증상 axis"를 도입한 것처럼, 흩어진 world-model 연구를 공통 좌표계로 매핑.

핵심 기여 (Key Contributions)

  1. Capability-based 로드맵 (L1→L2→L3): 단일 계층 분류 대신 ##세 가지 능력 단계##를 testable boundary 와 함께 정의. L1 = one-step prediction, L2 = action-conditioned multi-step rollout under constraints, L3 = evidence-driven model revision.
  2. Cross-domain synthesis via governing laws: ##Physical / Digital / Social / Scientific## 4 regime 도입. 각 regime이 어떤 형식의 transition을 강제하는지(접촉·운동학·보존, API·상태기계, 신념·규범, 인과·증거)를 정리해 단절된 분야의 비교를 가능하게 함.
  3. L3을 별도 capability level로 격상: 기존 서베이가 prediction/simulation 중심이었던 데 비해, ##모델 자체가 deployment evidence로 진화##하는 능력을 고유한 단계로 형식화. closed-loop autonomous discovery (CAMEO, A-Lab 등)가 L3의 가장 성숙한 사례.
  4. 의사결정 중심 평가 + MREP: 단일-step 정확도가 아닌 ##Action Success Rate(ASR)와 Counterfactual Outcome Deviation(COD)##로 능력 검증. Minimal Reproducible Evaluation Package을 제안.
  5. 10개 open problems와 design roadmap: representation/dynamics/control 3축 × L1/L2/L3 × 4 regime의 design space를 정리한 Table 13 매트릭스로 다음 5–10년 연구 방향 제시.

1. 동기와 분류 체계

1.1 왜 통합 분류가 필요한가

"world model"이라는 용어는 분야마다 의미가 다르다:

  • RL (Sutton 1991): 행동 결정을 위해 미래를 상상하는 transition model.
  • 비전 (Sora, Cosmos 등): 시각적·시간적 일관성을 가진 video/3D generator.
  • 언어 (WebDreamer, CodeWM): 웹·툴 환경의 text-grounded simulation.
  • 로보틱스: 안전한 planning과 sim-to-real을 위한 dynamics model.
  • AI for science: 가설 기반 실험을 위한 surrogate.

그 결과 ##"같은 system이 한 커뮤니티에서는 강한 진보로, 다른 커뮤니티에서는 실패로 보고된다"##. 본 서베이는 이 단절을 해소하기 위해 modality·application 중심 분류 대신 capability-based 분류를 제안.

1.2 두 직교 축

축 1 — Capability Level (L1 / L2 / L3): 모델이 무엇을 할 수 있는가.
축 2 — Governing-law Regime (4가지): 모델이 어떤 형식의 transition을 강제 받는가.

RegimeConstraintsCommon Failures
PhysicalGeometry, kinematics, conservationContact instability, sim-to-real gap
DigitalAPI contracts, state machinesGrounding breaks, async races
SocialBeliefs, norms, ToMRole drift, goal forgetting
ScientificCausal mechanisms, evidence chainsHallucinated mechanisms

두 축이 직교한다는 의미는, 같은 capability level 안에서도 regime마다 ##평가 기준이 달라야 한다##는 것. 예를 들어 L2 simulator는 physical에서는 충돌·관통이 없어야 하고, social에서는 약속·규범 위반이 없어야 한다.

2. 형식적 토대 — POMDP 위에서 정의되는 능력

2.1 통합 표기법 (Notation)

환경 $\mathcal{E} = (\mathcal{X}, \mathcal{A}, \Omega, T, O, R, \gamma)$ — POMDP 튜플. learned world-model components:

  • $z_t$: latent / internal state
  • $q_\phi(z_t \mid o_{\leq t}, a_{\leq t-1})$: state inference (encoder)
  • $p_\theta(z_t \mid z_{t-1}, a_t)$: forward dynamics (one-step latent transition)
  • $p_\psi(o_t \mid z_t)$: observation decoder
  • $\pi_\eta(a_t \mid z_{t-1}, z_t)$: inverse dynamics
  • $\hat{p}(\tau \mid z_0, a_{1:H}, c)$: trajectory-level rollout (앵커, 액션 시퀀스, 제약 조건부)
  • $\mathcal{M}_t \to \mathcal{M}_{t+1}$: world-modeling stack의 revision step

2.2 L1 Predictor — 정의 및 boundary

L1 world model은 4개의 local predictive operator로 분해된다:
$$\text{Inference: } q_\phi(z_t \mid o_{\leq t}, a_{\leq t-1}) \tag{1}$$ $$\text{Forward dynamics: } p_\theta(z_t \mid z_{t-1}, a_t) \tag{2}$$ $$\text{Observation decoder: } p_\psi(o_t \mid z_t) \tag{3}$$ $$\text{Inverse dynamics: } \pi_\eta(a_t \mid z_{t-1}, z_t) \tag{4}$$ 이들은 ##one-step (또는 short-horizon) 정확도##를 학습 분포 위에서 보장한다. 다단계 합성에 대한 보장은 없음.

직관적으로 말하면, L1은 "내일 날씨를 잘 맞추는 모델"—한 step 미래 예측에 충실. 하지만 한 달 뒤를 알기는 어렵다.

2.3 L2 Simulator — 3가지 boundary conditions

L1 operator를 합성해 trajectory $\hat{p}(\tau \mid z_0, a_{1:H}, c)$, $\tau = (z_1, \ldots, z_H)$ 를 만들어내는 능력. L2가 되려면 다음 3가지를 동시에 만족해야 한다:

  1. Long-horizon coherence: rollout이 $H$ step 후에도 즉시 무너지지 않고 사용 가능해야 한다 (compounding error 제어).
  2. Intervention sensitivity: counterfactual edit (action 또는 premise 변경)이 ##방향성 있고 안정적인## trajectory 변화를 유도해야 한다.
  3. Constraint consistency: 생성된 미래가 target regime의 지배 법칙을 위반하지 않아야 한다 (충돌, API contract violation, 약속 파기, 보존 법칙 등).

핵심 차이: L1과 L2의 차이는 "한 step 정확도"가 아니라 ##"composition 하 rollout fidelity"##. 한 단계 잘 맞춰도 합성에서 무너지면 L1에 머문다.

2.4 L3 Evolver — 모델 자체의 revision

L2가 ##고정된 scaffold 위 rollout##을 다룬다면, L3는 ##모델 stack 자체를 evidence로 갱신##:

$$(\mathcal{M}_t, d_t) \xrightarrow{\text{diagnose+distill+validate}} \mathcal{M}_{t+1},$$

여기서 $d_t$ 는 deployment evidence (trajectory, error, counterexample, test). 3가지 boundary conditions:

  1. Evidence-grounded diagnosis: 실패가 replayable evidence를 통해 actionable cause로 귀속되어야 한다.
  2. Persistent asset update: 수정이 ##재사용 가능한 자산##(skill, rule, parser, test)으로 승격되어야 한다 — ephemeral in-context patch가 아니라.
  3. Governed validation: 업데이트가 regression/robustness gate (rollback, canary 정책 포함)를 통과해야 default enable.

핵심 차이: L2와의 차이는 ##모델 자체가 revision의 대상이 된다는 것##. 단순한 데이터 buffer 갱신이 아니라 architecture/hypothesis-space까지 변경 가능.

이를 시각화한 design-execute-observe-reflect loop (Figure 9): $\mathcal{M}_t \xrightarrow{\text{design}} a_t \xrightarrow{\text{execute}} o_t \xrightarrow{\text{observe}} d_t \xrightarrow{\text{reflect}} \mathcal{M}_{t+1}$.

3. L1 Predictor 상세 — 4가지 operator

3.1 State Inference

관측 시퀀스를 latent belief로 압축. 주요 계열:

  • Contrastive (CPC, SimCLR, MoCo, CURL, SPR): 시간적 인접성을 positive pair로 사용. CURL이 control-relevant info 보존을 강조.
  • JEPA (I-JEPA, V-JEPA, V-JEPA2, DINOv2): 픽셀 reconstruction 없이 ##masked region embedding##을 직접 예측.
  • RSSM (Dreamer 계열): deterministic + stochastic 분기로 belief state $h_t = f(h_{t-1}, z_{t-1}, a_{t-1})$, $z_t \sim q_\phi(z_t \mid h_t, o_t)$.
  • Object-centric / Programmatic (Thinking with Blueprints): 이미지를 JSON-style 객체 기록으로 변환해 디시전 가능한 scene description으로.

3.2 Forward Dynamics — L1의 핵심

$p_\theta(z_t \mid z_{t-1}, a_t)$ 의 정확도가 L2 elevation의 핵심. 주요 흐름:

  • Continuous latent: PILCO (GP), MuZero (deterministic), RSSM (stochastic), TD-MPC2 (317M unified).
  • Token-based: IRIS (VQ-VAE codebook), TransDreamer (Transformer-XL), Delta-IRIS (delta coding).
  • Diffusion: DIAMOND (pixel-level), Latent Diffusion, DiT.

주요 발견: ##stochastic latent + Transformer 백본## 조합이 long-horizon에서 안정적이지만, 다단계 정확도는 여전히 큰 격차.

3.3 Observation Decoding & Inverse Dynamics

Decoder $p_\psi(o_t\mid z_t)$ 는 학습 신호 + dreaming의 rendering 엔진. VAE/β-VAE/VQ-VAE/Latent Diffusion이 representative. Inverse dynamics $\pi_\eta(a_t\mid z_{t-1}, z_t)$ 는 representation shaping (Pathak curiosity)이나 action labeling (VPT — Minecraft 비디오로부터 action 추론)에 활용. ##Action label quality##가 retrospective inference의 약점.

4. L2 Simulator 상세 — 4 regime별 instantiation

4.1 Physical World

  • Rigid-body simulators: MuJoCo, Brax, Isaac Gym/Lab, Genesis (학습 simulator 아닌 explicit law executor).
  • Video world models: Sora, Lumiere, VideoPoet → appearance-first; Genie/GAIA-1 → action-conditioned; FramePack/Self-Forcing → long-horizon drift 완화.
  • Decision-oriented: SimPLe, DIAMOND, DreamZero (zero-shot policy), DreamDojo, FutureVLA.
  • 3D structured: RoboOccWorld, PointWorld; PIN-WM (differentiable physics + visual).

핵심 question: ##visual plausibility ≠ decision-usability##. PhyWorldBench는 12개 frontier 모델의 conservation/object-permanence success rate가 0.262에 불과함을 보고.

4.2 Digital World

구분 특징: ##transitions are specifiable and verifiable## (formal artifact이므로 mechanically check 가능).

  • Code worlds: CodeWM (LLM + MCTS), WorldCoder (incremental), CWM (32B open-weights, ##65.8% on SWE-bench Verified##).
  • Web Models: WebDreamer, DreamGym (>30% improvement on WebArena), WebSynthesis, WebWorld (1M trajectories).
  • GUI/OS: MobileDreamer, MobileWorldBench (1.4M state-action triplets), gWorld (renderable web code as state).

4.3 Social World

3가지 distinctive properties: ##opacity (mental state 직접 관측 불가), reflexivity (belief가 state를 바꿈), normativity (norm 기반 transitions)##.

  • ToM: BToM (Bayesian inverse planning), ToMnet, LaBToM. FANToM은 ##모든 SOTA LLM에서 illusory ToM##; ExploreToM은 GPT-4o 9% 정확도.
  • Strategic: CICERO (Diplomacy, >2× human score), Werewolf/Avalon (deception testbeds).
  • Sandbox: Generative Agents (25 agents), Sotopia, Project Sid (1,000 agents), OASIS (1M agents).

주요 한계: 2nd-order belief 너머에서 sharp 성능 저하; role drift, goal forgetting; commitment tracking 미흡.

4.4 Scientific World

핵심: ##governing equations이 a priori 알려져 있지 않다## — 데이터로 학습하고 실험으로 검증.

  • Forward simulation: GNS (mesh particles), FNO (1000× speedup), Pangu-Weather/GraphCast (90% target에서 ECMWF 능가), GenCast (97.2%), NeuralGCM, Aurora (Earth foundation model).
  • Decision simulation: ChemBO, P3BO, BAX (active material discovery).

4.5 5가지 공통 failure mode

  1. Compounding error: per-step 작은 편차가 누적.
  2. State aliasing/drift: 비슷해 보이는 다른 state를 같다고 합쳐 잘못된 action.
  3. Controllability failure: action에 둔감한 model은 planning에 무용지물.
  4. Exploitability/simulator escape: planner가 simulator의 loophole을 탐색.
  5. Calibration failure under distribution shift: ##overconfident wrong이 강한 evolution 신호##로 다뤄져야 함.

5. L3 Evolver 상세 — Evidence-Driven Revision

5.1 Modes of growth

  • Parameter update: gradient descent / Bayesian update on new evidence (online learning, continual fine-tuning).
  • Architecture update: 새 module/expert/capacity 추가 (context window 확장, memory slot 할당).
  • Hypothesis-space expansion: 표현 가능한 explanation의 class를 확장 — ##abduction과 진정한 scientific discovery##에 가장 가까운 모드.

5.2 Domain별 maturity

RegimeMaturity대표 시스템Bottleneck

5.3 Revision triggers

  • Anomaly: prediction ≠ observation. 작으면 parameter update; persistent하면 epistemic gap.
  • Epistemic gap: 현재 model class로는 설명 불가. ##hard core (architecture, inductive bias)## 변경 필요 (Lakatos's protective belt vs hard core 비유).

5.4 Governance challenges

  • Benchmark overfitting: regression gate가 학습 분포에 너무 가까우면 "test 통과만" 학습.
  • Knowledge contamination: revision loop가 biased evidence로 OOD에서 silent degradation.
  • Misattribution cascades: 한 fix가 다른 component를 망가뜨림.

완화: held-out probe, canary deployment, causal ablations.

6. Decision-centric Evaluation

6.1 Prediction → Decision-centric 전환

FID/FVD/SSIM 같은 generative metric은 perceptual quality만 측정 — agentic capability와 약한 상관. 진짜 측정해야 할 것은 ##trajectory-level rollout이 planner가 행동에 사용할 정도로 신뢰할 만한가##.

6.2 두 핵심 메트릭

Action Success Rate (ASR):

$$\text{ASR} = \frac{1}{N}\sum_{i=1}^{N}\mathbf{1}[\text{task}_i \text{ succeeds under policy derived from }\hat{p}].$$

직관: world model이 만든 rollout으로 학습된 정책이 실제 환경에서 task를 풀어내는 비율.

Counterfactual Outcome Deviation (COD): intervention sensitivity 측정.

$$\text{COD}(k) = \mathbb{E}\bigl[d(\hat{z}_H^{(1)}, \hat{z}_H^{(2)})\bigr],$$

$d$ 는 task-relevant distance (goal-state distance, edit distance 등). step $k$에서 단일 intervention만 다른 두 정책의 결과를 비교. ##COD가 낮으면 모델이 action에 unresponsive## → counterfactual planning 불가.

6.4 Evaluation gaps

현재 ##대부분의 시스템이 L1-style accuracy만 평가##되고 있다. L2 protocol (counterfactual injection, degradation curves, constraint-violation detection)은 일부 benchmark에서만; L3 infrastructure (regression suite, asset validation, cross-episode improvement)는 ##autonomous science 외에는 거의 부재##. Minimal Reproducible Evaluation Package (MREP)을 제안하여 version-locking + trace logging + boundary-condition mapping 표준화.

7.2 Latency budget by regime (Table 12)

  • Real-time robotics: <100ms — RSSM/neural ODE.
  • Web/GUI agents: ~1–5s — LLM-as-WM.
  • Software engineering: 5–30s — LLM + MCTS.
  • Game AI real-time: <50ms — tree search + value-aware latent.
  • Social/multi-agent: 1–10s — ToM network, $O(n^2)$ pairwise update가 병목.
  • Scientific planning: minutes-to-hours — diffusion ensemble, Bayesian surrogate.

7.3 3 cross-cutting principles

  1. Separate what is learned from what is enforced: hard constraint layer (collision checker, state-machine validator)는 ##inference-time에 적용##해야 한다 — training loss로는 zero-violation을 보장 못함.
  2. Instrument before you iterate: logging, replay, failure attribution을 시스템 시작부터 내장. replay 없으면 L3 revision은 "anecdotal & ungovernable".
  3. Match representation to planner's query: planner가 필요로 하는 변수를 노출하지 못하는 표현은 ##아무리 사실적이어도## 더 단순한 표현보다 못하다.

8. 역사적 발전과 10가지 Open Problems

8.1 4 시대 (Figure 6)

  • Mathematical Principles (–1956): Newton, Laplace, Turing.
  • Symbolic Intelligence (1956–1986): STRIPS, Frame Problem, Lighthill (1차 AI winter).
  • Connectionist Resurgence (1986–2020): Backprop, AlexNet, Transformer, World Models 2018, GraphCast.
  • Generative Revolution (2020–현재): GPT, Diffusion, Sora, DreamerV3, AlphaEvolve. ##neuro-symbolic frontier##로 수렴 중.

8.2 10가지 open problems

Representation & local prediction:

  1. Physical faithfulness beyond visual plausibility: PhyWorldBench 0.262 success — pixel fidelity ≠ physical consistency.
  2. Metric-aware video world modeling: 시간적 일관성 + identity preservation을 동시 달성.
  3. Programmable visual representation: VCode (SVG), Code2Video (Manim script), VIGA (3D Blender) — ##code-based representation##이 pixel보다 editable.

Simulation fidelity & intervention:

  1. Partially observable software as a POMDP: 백엔드 hidden state, async transition을 belief state로 모델링.
  2. Concurrent multi-user state: Dec-POMDP 위 conflict-free replicated data type.
  3. Agent-human behavioral alignment at scale: LLM의 ##mode collapse## + alignment inadequacy in multi-turn.

Evidence-driven revision:

  1. Continual learning of societal transitions: 1만+ agents에서 norm shift detection without catastrophic forgetting.
  2. Closing the surrogate-to-reality gap: 시뮬레이터 데이터로 학습한 surrogate는 real measurement에서 plateau.
  3. Modeling laws that themselves evolve: viral fitness, climate forcing 등 second-order dynamics.
  4. Harness designs for agentic world modeling: ##prompt → context → harness## (tools, memory, feedback loops, inter-agent topology) 진보. harness 자체를 학습 대상으로.

8.3 Beyond L3: Meta-world Modeling

L1→L2→L3는 fixed governing law를 가정. 그 다음은 ##transition function의 space 자체를 추론##하는 meta-world modeling — alternative rule system 탐색. symbolic representation이 자연스러운 interface가 된다는 가설.

9. 비판적 분석

  • 분류의 testability vs over-fitting 위험: L1/L2/L3 boundary가 testable하다고 주장하지만, 실제로 "long-horizon coherence가 실패했다"의 정량 기준 (몇 step? 몇 % degradation?)이 regime마다 달라 ambiguous. 저자도 "schematic, not metric"이라며 인정하지만, taxonomy의 실용성이 약화될 위험.
  • L3의 maturity 단계 비대칭: Scientific을 "established"라고 분류하지만 A-Lab (17일·353 실험)을 제외하면 cross-domain reproduction이 적음. 한두 flagship system을 generalization으로 보는 것은 편향 가능성.
  • Capability-based vs Application 분류의 trade-off: capability 중심은 학술적으로 깔끔하지만, ##실무자가 "내 task에는 어떤 모델을 써야 하는가"##의 답을 찾기 어렵다. application-domain 분류 (autonomous driving, GUI, drug discovery)와의 ##병행이 필요##해 보임.
  • 4 regime의 완전성 의문: physical/digital/social/scientific 4개를 representative라 하지만 economic/financial regime, biological(scientific 안에 포함되지만 별도 dynamics), legal regime 등 명시적 처리 없음. ##"non-exhaustive"이라 명시하지만## taxonomy 일관성 약점.
  • Gov regime의 coupling: 자율주행이 physical+social, Diplomacy가 social+digital — multi-regime system이 오히려 일반적인데, 본 서베이는 cross-regime 분석이 짧다 (4.2.5 한 페이지).
  • 인용 작품의 자가 인용 편향: 저자 그룹이 multi-institution이지만 HKUST 계열 작품 (Zhang, Lin, Chen 등)이 다수 인용되어 selection bias 가능성.
  • L1→L2→L3 hierarchy의 implicit linearity: 실제로 어떤 시스템 (예: closed-loop autonomous lab)은 ##weak L1을 가지고도 강한 L3##일 수 있다. capability 단계가 strict containment라는 묵시적 가정이 약점일 수 있음.
  • Decision-usability와 학습 비용의 trade-off 분석 부족: ASR/COD가 좋은 metric이라 하지만 측정 비용이 매우 높다. proxy metric 추천 또는 cost-aware evaluation이 더 다뤄지면 좋았을 것.

10. Key Takeaways (이 논문에서 가져갈 것)

  • "world model"을 말하기 전에 어느 capability level과 어느 regime인지 명시: 같은 system이 L1/Physical에서는 SOTA지만 L2/Social에서는 부족할 수 있다. 분야 간 비교 시 ##capability × regime 좌표##를 먼저 합의해야 한다.
  • L1→L2 elevation의 진짜 시험은 boundary conditions: 한 step 정확도가 아무리 높아도 long-horizon coherence + intervention sensitivity + constraint consistency 셋 모두 만족해야 simulator. 실험 설계 시 이 3가지를 별도 측정.
  • L3는 "online learning"이 아니라 "model의 hypothesis space evolution": 단순 fine-tuning은 L3 아님. ##persistent reusable asset + governed validation##이 핵심.
  • "separate learned from enforced": 충돌 검사, state machine validator, regression gate를 inference-time에 적용. soft training loss로는 zero-violation 불가.
  • Decision-centric metric (ASR + COD): 자체 시스템 평가 시 perception loss나 FID 대신 "downstream policy success rate"와 "intervention 반응성"을 같이 측정.
  • Code as world model이라는 흐름 (CodeWM, gWorld, Code2World): ##executable한 code가 latent dynamics보다## interpretable·composable·verifiable. 디지털 regime에서 강력한 새 패러다임.
  • Harness가 capability boundary의 일부: tools, memory, feedback loop가 model capability를 본질적으로 변화시킨다. harness 설계 자체가 world modeling research의 한 axis.
