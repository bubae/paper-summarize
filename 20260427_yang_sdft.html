Self-Distillation Bridges Distribution Gap in LLM Fine-Tuning (SDFT)

TL;DR

  • 문제: Llama-2-chat 같은 ##aligned LLM##을 다운스트림 task에 vanilla fine-tune하면 task 정확도는 오르지만 ##general instruction-following, safety alignment, helpfulness가 무너지는 catastrophic forgetting##이 흔히 발생한다.
  • 해결: Catastrophic forgetting의 근본 원인이 ##task dataset과 seed LM의 출력 분포 사이의 distribution gap##이라고 진단. ##Self-Distillation Fine-Tuning (SDFT)##은 학습 전에 seed LM이 ##직접 task의 reference answer를 자기 스타일로 rewriting##하여 distilled dataset을 만들고, 이 위에서 fine-tune한다.
  • 왜 작동하는가: distilled response는 의미는 동일하지만 ##seed LM의 분포에 가까운 표면 형태##라 fine-tune이 만들어내는 parameter shift가 작아진다 → 기존 능력 보존 + 새 task 학습이 양립.
  • 비유: 외국어 시험 답안을 ##한국어 화자가 자기 말투로 다시 적은 모범 답안##으로 공부하는 학생. 의미는 같지만 자기 어법이라 외국어 학습 후에도 한국어가 망가지지 않는다.

핵심 기여 (Key Contributions)

  1. Distribution Gap as Root Cause: catastrophic forgetting을 task dataset과 seed LM 분포 차이로 환원하는 framing. BLEU-4, ROUGE-L, embedding similarity, $\|\Delta\theta\|_2$ 4축으로 distribution shift를 정량화.
  2. SDFT Method: 단순한 distillation template으로 seed LM이 reference answer를 self-rewriting → 한 줄 변경으로 vanilla SFT를 대체. ##LoRA / full fine-tune / Llama-2-7B / 13B / Llama-3-8B-Instruct에 모두 적용 가능##.
  3. 광범위한 검증: 단일 task (GSM8K, OpenFunctions, MagiCoder), 멀티 task (Alpaca, Dolly, LIMA, OpenHermes), safety (Advbench raw/jailbreak), helpfulness (AlpacaEval), general knowledge (MMLU, TruthfulQA, ARC, HellaSwag, Winogrande) 5개 영역에서 일관되게 vanilla FT 대비 우위.
  4. Distribution Shift × Forgetting 인과 검증: sample size 변화·mix ratio (vanilla:SDFT) 조작 실험으로 ##distribution shift 지표(BLEU-4, ROUGE-L, embedding sim 감소 / parameter shift 증가)와 forgetting이 동행##함을 보임.
  5. Plug-and-Play 호환성: 특정 모델·하이퍼파라미터·아키텍처에 종속되지 않는 단순 데이터 전처리 단계로, 어떤 SFT 파이프라인에도 ##즉시 삽입 가능##.

1. 배경 (Background)

1.1 Vanilla Fine-Tuning과 Catastrophic Forgetting

SFT 손실은 task dataset $\{(c^t,x^t,y^t)\}$에서:

$$L_{\mathrm{FT}}(\theta) = -\log f_\theta(y^t \mid c^t, x^t) \quad (2)$$

$L_{\mathrm{FT}}$가 수렴하면 fine-tuned LM의 분포가 task dataset 분포와 정렬된다. 그러나 task dataset의 reference response가 seed LM의 출력 스타일과 ##크게 다른 경우##, 정렬을 위해 parameter가 크게 움직여야 하고, 이 과정에서 ##general instruction-following, safety, helpfulness가 무너진다##. 예: OpenFunctions로 fine-tune하면 HumanEval pass@1이 13.4 → 9.8 (−27%).

1.2 기존 방법의 한계

  • Continual Learning (EWC, PackNet, replay): 원본 학습 데이터·파라미터 importance·distinct neuron 할당 등이 필요하나 LLM은 ##원본 학습 데이터 비공개·태스크 공간 광대##라 적용이 어렵다.
  • Self-Play Fine-Tuning (Chen 2024): 같은 LLM을 generator+discriminator로 사용하지만 결국 task data의 log-likelihood를 최대화하므로 분포 정렬 자체는 그대로 → forgetting 미해소.
  • Self-Refine / Self-Reward: 생성된 응답을 iteratively 개선하나 distribution gap 해소가 목적이 아님.

2. 방법: SDFT

2.1 핵심 아이디어 — Response Rewriting

seed LM $f_\theta$를 사용해 task dataset의 원래 response $y^t$를 다음과 같이 다시 쓴다:

$$\tilde y \sim f_\theta(y \mid c^t, x^t, y^t) \quad (3)$$

여기서 prompt는 ##distillation template## (Figure 3): "Below are an instruction that describes a task along with a reference answer. Using the reference answer as a guide, write your own response." 형식. seed LM은 의미 등가성을 유지하면서 자기 분포에 맞는 표현으로 rewrite한다.

2.2 Quality Filter (Reasoning task)

수학 추론 등 정답 추출이 가능한 경우 conditional selection:

$$\tilde y' = \begin{cases} \tilde y & \text{if Extract}(\tilde y) = y^t \\ y^t & \text{otherwise} \end{cases} \quad (4)$$

최종 답이 reference와 일치할 때만 distilled response 채택, 그렇지 않으면 원본 유지. 이 단순한 휴리스틱이 자기-rewriting의 ##정답 오염 위험을 차단##한다.

2.3 SDFT 손실

$$L_{\mathrm{SDFT}}(\theta) = -\log f_\theta(\tilde y' \mid c_t, x_t) \quad (5)$$

구조적으로 vanilla SFT와 동일 — ##데이터를 distilled으로 교체했을 뿐##. LoRA·full fine-tune 모두 호환.

2.4 알고리즘 흐름도

[Task dataset {(c^t, x^t, y^t)}] ↓ [Seed LM] ──(distillation template + reference answer)→ ỹ ↓ [Quality filter (Eq.4)] — 수학 task에서만 → ỹ' ↓ [Distilled dataset {(c^t, x^t, ỹ')}] ↓ [Standard SFT loss on distilled set] → fine-tuned LM

3. 실험 (Experiments)

3.1 공통 설정

  • Seed LM: Llama-2-7b-chat (메인). 추가: Llama-2-13b-chat, Llama-3-8B-Instruct.
  • Fine-tuning: LoRA (rank=8 on Q,V), lr=$10^{-4}$ cosine→0, batch 8, 2 epoch (대부분).
  • 샘플: 2k(소형 데이터셋), 20k(OpenHermes). LIMA 2 epoch, OpenFunctions 5 epoch.
  • Distillation template: Alpaca 표준 + 수학용 변형 (Figure 11/12).

3.5 General Knowledge 보존 (Figure 4)

OpenLLM Leaderboard 5개 (MMLU, TruthfulQA, ARC, HellaSwag, Winogrande)는 ##fine-tune 어떻게 해도 거의 변동 없음## — vanilla와 SDFT 모두 marginal. 이 결과의 주장: forgetting은 주로 ##general knowledge가 아니라 instruction-following + safety/helpfulness 영역##에서 일어난다.

3.6 Distribution Shift × Forgetting 인과 (Figures 5–9)

  • Sample size ↑ (500→2000): BLEU-4·ROUGE-L·embedding sim ↓, $\|\Delta\theta\|_2$ ↑, MultiArith·Win Rate·Safe Rate 모두 ↓ → 학습 데이터가 많을수록 ##distribution shift 심화 = forgetting 심화##.
  • Mix ratio (vanilla:SDFT 비율 1.0→0.0): SDFT 비율이 높을수록 BLEU-4·ROUGE-L·embedding sim ↑, parameter shift ↓, downstream score ↑.
  • Figure 9: SDFT 모델이 seed model embedding과 ##더 높은 cosine similarity (0.85 vs 0.75)## — distribution shift가 작음을 직접 검증.

주장 매핑: distribution gap이 forgetting의 통제 가능한 원인이며, SDFT가 그 gap을 직접 줄인다.

4. 비판적 분석 (Critical Analysis)

  • Self-rewriting의 정확성 보장 부재: Eq. (4) extract-and-compare는 수학에서만 작동. 코드·tool-calling·자유 형식 task에서는 ##rewrite가 의미를 변형해도 검출되지 않는다##. OpenFunctions에서 distilled response가 추가 explanation을 붙이는 케이스(Figure 17)는 의미 일치 검증 불가.
  • 이론적 frame이 약하다: "distribution gap → forgetting"은 실증적 상관일 뿐 인과 증명이 아니다. Catastrophic forgetting의 이론적 분해(예: Fisher information, NTK 분석)와의 연결 없음.
  • 모델 의존성: rewrite 품질은 seed LM의 instruction-following 능력에 강하게 의존 → ##weak base model에서는 distillation이 오히려 noisy##. Shenfeld et al. 2026과 같은 결론을 (역으로) 뒷받침.
  • LoRA 편중: 메인 결과는 거의 LoRA, full fine-tune은 보조 실험만. fully fine-tuned의 distribution shift 동력이 다를 수 있는데 분석 없음.
  • General knowledge가 안 변하는 이유 미분석: Figure 4에서 MMLU 등이 거의 변하지 않는 점이 "이 영역은 robust"인지 "fine-tune이 영향이 적은 영역"인지 구분 안 됨. 실제 catastrophic forgetting의 표적은 좁다는 시사.
  • Continual learning sequential 비교 부재: 논문의 framing은 single-task fine-tune에 대한 forgetting 방지 — ##Shenfeld et al. 2026이 다루는 sequential multi-task 환경에서의 우월성은 검증되지 않음##.
  • Compute overhead: rewriting은 학습 데이터 전체에 대해 추론 1회 필요. 20k OpenHermes 데이터에 대한 wall-clock 비용·token 길이 변화가 보고되지 않음.
  • Distillation template이 reference answer를 노출: "Using the reference answer as a guide" prompt가 seed LM에게 정답을 보여주므로 ##content-level diversity가 좁을 수 있다## — 다만 이게 forgetting 방지에는 도움.

5. Key Takeaways

  • Distribution alignment first: instruction-tuned LLM을 추가 fine-tune할 때 ##데이터의 표면 형태를 seed LM 분포에 맞추는 것##만으로 forgetting의 큰 부분을 제거할 수 있다는 실용적 인사이트.
  • ##단순한 데이터 전처리##로 SFT pipeline에 즉시 삽입 가능 — 하이퍼파라미터·옵티마이저 변경 없이 vanilla SFT 코드의 데이터 로딩만 교체하면 된다.
  • Catastrophic forgetting의 가장 큰 표적은 ##general knowledge가 아니라 instruction-following + safety + helpfulness##라는 점 (Figure 4 vs Tables 2-3). post-training 평가는 MMLU만으로는 부족.
  • Distribution shift 지표(BLEU-4, ROUGE-L, embedding similarity, parameter shift norm)는 forgetting의 ##조기 경보 metric##으로 사용 가능. 학습 중 monitoring하면 위험한 fine-tune을 사전 진단할 수 있다.
  • Mix ratio 실험은 ##distillation 비율을 hyperparameter로## 사용할 수 있음을 시사. target task 성능 vs 기존 능력 보존의 trade-off curve를 그려 적절한 비율 선택 가능.
  • 한계 인식: rewriting의 의미 보존은 task-specific 검증이 필요하며, weak base model이나 매우 다른 task domain (예: novel tool API)에서는 self-rewriting의 한계가 있다.
