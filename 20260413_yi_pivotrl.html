TL;DR

  • 문제: Long-horizon agentic task의 post-training에서 SFT (Supervised Fine-Tuning)는 compute-efficient하지만 OOD (Out-of-Domain) 일반화가 심각하게 저하되고 (-9.83pp 평균), E2E RL (End-to-End Reinforcement Learning)은 OOD를 보존하지만 매 업데이트마다 full multi-turn rollout이 필요하여 compute 비용이 매우 높다.
  • 해결: PivotRL — 기존 SFT expert trajectory를 재활용하여, (1) pivot이라 불리는 informative한 중간 turn만 선별하고, (2) 정확한 문자열 매칭 대신 functional-equivalent reward를 사용하여 turn-level local RL을 수행. Full trajectory rollout 없이도 E2E RL 수준의 OOD 보존과 SFT 이상의 in-domain 정확도를 달성.
  • 왜 작동하는가: (1) Mixed-outcome turn만 선별하면 group-normalized RL에서 zero가 아닌 advantage signal이 보장되어 rollout budget의 효율적 활용 가능 (Proposition 3.1), (2) Reward variance가 GRPO의 natural gradient signal 크기와 정확히 일치하여 high-variance turn이 strongest learning signal 제공 (Theorem 3.2), (3) Functional reward가 KL-regularized update에서 acceptable action 집합으로 확률을 이동시키면서 reference policy의 ordering을 보존하여 OOD 성능 유지 (Theorem 3.3).
  • 비유: 수학 과외에서 학생이 이미 완벽히 풀거나 전혀 모르는 문제는 건너뛰고, "풀 수도 있고 못 풀 수도 있는" 경계 문제만 집중적으로 연습시키는 것. 또한 채점할 때 "모범답안과 글자 하나하나 일치"가 아니라 "같은 결과를 내는 풀이면 정답"으로 인정한다.

핵심 기여 (Key Contributions)

  1. PivotRL 프레임워크: ##SFT의 compute efficiency와 E2E RL의 OOD accuracy를 동시에 달성##하는 turn-level agentic RL. SFT 대비 +4.17% in-domain, +10.04% OOD accuracy 향상.
  2. Pivot filtering + Functional reward의 이론적 정당화: Mixed-outcome turn만이 nonzero gradient를 생성하고 (Proposition 3.1), reward variance가 natural gradient norm과 동치이며 (Theorem 3.2), functional reward가 probability mass를 acceptable action으로 이동시키면서 reference ordering을 보존함을 증명 (Theorem 3.3).
  3. SWE-Bench에서 E2E RL과 경쟁적 성능: ##E2E RL 대비 ~4× fewer rollout turns, ~5.5× less wall-clock time##으로 동일 정확도 달성.
  4. Production-scale 적용: NVIDIA의 Nemotron-3-Super-120B-A12B의 agentic post-training에서 핵심 방법론으로 채택.

1. Introduction — 배경과 동기

Agentic Task의 Post-Training 딜레마

Long-horizon agentic task는 LLM이 환경과 여러 turn에 걸쳐 상호작용하는 것을 요구한다: conversational tool use (Schick et al., 2023; Qin et al., 2023), agentic coding (Jimenez et al., 2024), terminal interaction (Xie et al., 2024), web search (Yao et al., 2022; Wei et al., 2025) 등. 이러한 능력의 post-training에는 근본적인 효율성-일반화 trade-off가 존재한다:

방법장점단점
SFT (Supervised Fine-Tuning)Compute-efficient, 빠른 학습OOD 일반화 저하 (-9.83pp), catastrophic forgetting
E2E RL (End-to-End RL)OOD 보존, robust 일반화매 업데이트마다 multi-turn on-policy rollout 필요 → 높은 compute 비용

핵심 질문: "Full trajectory rollout 없이도 E2E RL의 일반화 능력과 SFT의 data efficiency를 결합할 수 있는가?"

Naive Local RL의 문제점

자연스러운 접근은 SFT trajectory를 local RL 에피소드로 변환하는 것이다: 중간 state $s_t$에서 시작하여 action을 sample하고, 정답과 정확히 일치하면 reward를 부여한다:

$$r_\text{strict}(s, a) = \mathbf{1}[a = a^*(s)] \quad (2)$$

그러나 이 전략은 경험적으로 미미한 개선만 제공한다 ($\tau^2$-Bench에서 57.34 vs SFT의 58.44). 두 가지 병목이 존재한다:

  1. Uninformative turn: ##$\tau^2$-Bench와 SWE-Bench에서 randomly sampled turn의 71%가 모두 성공 또는 모두 실패##하여 advantage가 0이 되고 gradient에 기여하지 않는다.
  2. Exact-match local credit가 너무 엄격: 생성적 action space에서 functionally equivalent한 action이 무수히 많지만, 단일 문자열 매칭은 이를 오답으로 처리. 예: tool call이나 search query가 정답과 다르게 표현되었지만 동일한 결과를 내는 경우.

2. Preliminaries and Motivating Observations

Turn-Level Agentic Training

Trajectory $\tau = (s_0, a_0^*, \ldots, s_T, a_T^*)$에서 assistant decision boundary마다 turn을 추출한다. 여기서 $s_t$는 시작부터 $t$번째 assistant action 직전까지의 interaction history이고, $a_t^*$는 해당 turn에서의 demonstrated action이다. Action은 개별 토큰이 아니라 model-call boundary의 전체 assistant completion이다.

SFT 목적함수:

$$\mathcal{L}_\text{sft}(\theta) = -\mathbb{E}_{\tau \sim \mathcal{D}_\text{sft}, (s_t, a_t^*) \sim \tau_t}[\log \pi_\theta(a_t^* \mid s_t)]$$

E2E RL (GRPO) 목적함수:

$$\mathcal{J}_\text{GRPO}(\theta) = \mathbb{E}_{s \sim \mathcal{D}_{\pi_\text{old}}} \left[ \frac{1}{G} \sum_{i=1}^{G} \min\left(w_i(\theta)\hat{A}_i, \text{clip}(w_i(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_i\right) - \beta \text{KL}(\pi_\theta(\cdot \mid s) \| \pi_0(\cdot \mid s)) \right] \quad (1)$$

여기서 importance sampling weight $w_i(\theta) = \frac{\pi_\theta(a_i|s)}{\pi_{\theta_\text{old}}(a_i|s)}$이고, $\mathcal{D}_{\pi_\text{old}}$는 on-policy rollout으로부터의 state distribution이다. Group-normalized advantage:

$$\hat{A}_i = \frac{r_i - \frac{1}{G}\sum_{j=1}^{G} r_j}{\text{std}(\{r_j\}_{j=1}^G) + \epsilon_\text{std}} \quad (1)$$

Motivating Observations

관찰 1: Uniformly 성공/실패한 turn은 uninformative: $\{r_i\}_{i=1}^G$이 모두 0이거나 모두 1이면 normalized advantage가 전부 0이 되어 gradient 기여가 없다. 경험적으로 ##random sampling 시 71%의 turn이 이 상태##에 해당.

관찰 2: Exact-match credit가 너무 엄격: Miss rate $= \Pr[r_\text{strict}(s,a)=0 \mid r_\text{func}(s,a)=1]$가 높다. 즉, functionally correct한 action을 정답과 다르다는 이유로 오답 처리하는 비율이 높다.

3. PivotRL

3.1 Method

PivotRL은 naive local RL을 두 가지로 수정한다: (1) pivot filtering으로 informative turn만 선별, (2) functional reward로 exact-match를 대체.

Step 1: Pivot Candidate 추출

Expert trajectory $\tau = (s_0, a_0^*, \ldots, s_T, a_T^*)$에서 모든 assistant turn을 추출하여 pivot candidate dataset을 구성한다:

$$\mathcal{D}_\text{cand} = \{(s_t, a_t^*)\}_{\tau \in \mathcal{D}_\text{SFT}, \, t=0,\ldots,T} \quad (3)$$

Step 2: Offline Turn Selection

Frozen reference policy $\pi_0$에서 각 state $s$에 대해 $K$개의 local rollout을 sampling하고, verifier로 채점하여 reward 통계를 계산한다:

$$\hat{\mu}(s) = \frac{1}{K}\sum_{k=1}^{K} r_\text{func}(s, a^{(k)}), \quad \hat{\sigma}^2(s) = \frac{1}{K}\sum_{k=1}^{K} \left(r_\text{func}(s, a^{(k)}) - \hat{\mu}(s)\right)^2 \quad (4)$$

Nonzero reward variance와 low mean을 가진 turn만 retain한다:

$$\mathcal{D}_\text{adv} = \left\{(s, a^*) \in \mathcal{D}_\text{cand} : \hat{\sigma}^2(s) > 0, \; \hat{\mu}(s) < \lambda_\text{diff}\right\} \quad (5)$$

직관적으로, 첫 번째 필터($\hat{\sigma}^2 > 0$)는 ##이미 다 풀리거나 전혀 못 푸는 turn을 제거##하여 nonzero advantage signal을 보장한다. 두 번째 필터($\hat{\mu} < \lambda_\text{diff}$)는 mixed-outcome이면서도 아직 어려운 turn에 집중한다. 이 필터링된 집합이 pivot이다: $\mathcal{D}_\text{pivot} = \mathcal{D}_\text{adv}$.

Step 3: Verifier-Based Local Reward

State $s$에서 locally acceptable action 집합 $\mathcal{M}(s) \subseteq \mathcal{A}(s)$를 domain-specific verifier로 정의하고:

$$r_\text{func}(s, a) = \mathbf{1}[a \in \mathcal{M}(s)] \quad (6)$$

직관적으로, exact match ($r_\text{strict}$)와 달리, ##정답과 문자열이 다르더라도 기능적으로 동등한 action이면 reward를 부여##한다. Verifier는 도메인에 따라 normalized string/schema validation, task-specific equivalence rule, lightweight LLM judge 등이 될 수 있다.

Algorithm 1: PivotRL 전체 파이프라인

Require: expert trajectories $\mathcal{D}_\text{sft}$, reference policy $\pi_0$, initial policy $\pi_\theta$, verifier $V$, rollout group size $G$, difficulty threshold $\lambda_\text{diff}$ 1. Extract turns: $\mathcal{D}_\text{cand} = \{(s_t, a_t^*)\}$ from $\mathcal{D}_\text{sft}$ 2. Profile each $s \in \mathcal{D}_\text{cand}$ with Eq. (4) [offline, K rollouts per turn] 3. Filter: $\mathcal{D}_\text{pivot} \leftarrow \mathcal{D}_\text{adv}$ [nonzero variance + low mean] 4. for $k = 1, \ldots, K_\text{train}$ do 5. Sample minibatch $\{s_b\}_{b=1}^B \sim \mathcal{D}_\text{pivot}$ 6. for each $s_b$ do 7. Sample $\{a_{b,i}\}_{i=1}^G \sim \pi_{\theta_\text{old}}(\cdot \mid s_b)$ 8. Execute short rollout to score each $a_{b,i}$ 9. $r_{b,i} \leftarrow r_\text{func}(s_b, a_{b,i})$ for $i = 1, \ldots, G$ 10. Compute $\hat{A}_{b,i}$ via group normalization 11. end for 12. Update $\theta$ using GRPO objective (Eq. 7) 13. end for

PivotRL의 GRPO-style 최적화 목적함수:

$$\mathcal{J}_\text{PivotRL}(\theta) = \mathbb{E}_{\substack{s \sim \mathcal{D}_\text{pivot} \\ \{a_i\}_{i=1}^G \sim \pi_{\theta_\text{old}}(\cdot|s)}} \left[ \frac{1}{G} \sum_{i=1}^{G} \min\left(w_i(\theta)\hat{A}_i, \text{clip}(w_i(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_i\right) - D_\text{KL} \right] \quad (7)$$

여기서 $D_\text{KL} = \beta \text{KL}(\pi_\theta(\cdot \mid s) \| \pi_0(\cdot \mid s))$이고, importance weight $w_i(\theta) = \frac{\pi_\theta(a_i \mid s)}{\pi_{\theta_\text{old}}(a_i \mid s)}$이다.

E2E RL 대비 핵심 차이: E2E RL에서는 initial environment state에서 multi-turn trajectory 전체를 rollout해야 하지만, PivotRL에서는 ##expert trajectory의 중간 state에서 시작하여 single-turn action만 sampling##하면 된다. 이것이 compute 절감의 핵심이다.

3.2 Theoretical Analysis

Proposition 3.1: Mixed-outcome turn만이 nonzero gradient를 생성

$s$에서 $G$개의 binary reward $\{r_i\}_{i=1}^G$가 모두 동일하면 (모두 0 또는 모두 1), normalized advantage $\hat{A}_i$가 모든 $i$에 대해 0이 된다. ##따라서 nonzero group-normalized update에 기여할 수 있는 것은 positive reward variance를 가진 rollout group, 즉 mixed-outcome turn뿐##이다.

직관적으로, 모든 학생이 동일한 점수를 받으면 상대 평가에서 아무도 보상/벌칙을 받지 못한다 — PivotRL이 mixed-outcome turn을 필터링하는 것은 바로 이 낭비를 제거하기 위함이다.

Theorem 3.2: Reward variance가 local GRPO signal 크기를 결정

Statewise expected reward objective $J_s(\pi) = \mathbb{E}_{a \sim \pi}[r(s,a)]$와 KL path $\pi_{s,\beta}(a) = \frac{\pi_0(a \mid s) e^{r(s,a)/\beta}}{\sum_b \pi_0(b \mid s) e^{r(s,b)/\beta}}$를 정의하고, population GRPO score를 다음과 같이 정의한다:

$$\gamma_{s,\beta} := -\mathbb{E}_{a \sim \pi_{s,\beta}} \left[ \frac{r(s,a) - \mathbb{E}_{a' \sim \pi_{s,\beta}}[r(s,a')]}{\sqrt{\text{Var}_{a' \sim \pi_{s,\beta}}(r(s,a'))}} \; \partial_\beta \log \pi_{s,\beta}(a) \right] \quad (9)$$

그러면:

$$\gamma_{s,\beta} = \frac{1}{\beta^2} \left\| \nabla^\text{nat} J_s(\pi_{s,\beta}) \right\|_{F, \pi_{s,\beta}} = \frac{\sqrt{\text{Var}_{a \sim \pi}(r(s,a))}}{\beta^2} \quad (10)$$

직관적으로, ##reward variance가 큰 state일수록 natural gradient norm이 크고, 따라서 GRPO 학습 signal이 강하다##. Binary reward에서 더 큰 variance는 성공과 실패가 더 골고루 섞여있다는 의미이므로, PivotRL이 mixed-outcome turn을 선택하는 것은 strongest learning signal을 가진 turn에 rollout budget을 집중하는 것과 동치이다.

Theorem 3.3: Functional reward가 minimal KL 변화로 확률을 이동

$r_\text{func}(s,a) = \mathbf{1}[a \in \mathcal{M}(s)]$인 functional reward와 KL-regularized objective:

$$\mathcal{L}_{\text{func},\beta}(\pi) = \mathbb{E}_{s \sim d}\left[-\mathbb{E}_{a \sim \pi(\cdot|s)}[r_\text{func}(s,a)] + \beta \text{KL}(\pi(\cdot \mid s) \| \pi_0(\cdot \mid s))\right] \quad (11)$$

이 목적함수의 unique minimizer $\pi_\beta^*$에 대해:

  • $\pi_\beta^*(\mathcal{M}(s) \mid s) \ge q_\beta(s) = \frac{\rho(s) e^{1/\beta}}{(1-\rho(s)) + \rho(s) e^{1/\beta}}$ (acceptable action 집합의 총 확률 증가)
  • $\pi_\beta^*$는 $\mathcal{M}(s)$ 내부와 $\mathcal{M}(s)^c$ 내부에서 각각 reference policy $\pi_0$의 ordering을 보존
$$\frac{\pi_\beta^*(a \mid s)}{\pi_\beta^*(b \mid s)} = \frac{\pi_0(a \mid s)}{\pi_0(b \mid s)} \quad \text{for } a, b \in \mathcal{M}(s), \quad \text{and for } a, b \in \mathcal{M}(s)^c \quad (14)$$

직관적으로, functional reward 기반 RL은 ##acceptable action 집합으로 확률 mass를 이동시키되, 그 집합 내부와 외부에서 기존 reference policy의 상대적 순서를 변경하지 않는다##. Action space가 지수적으로 크므로 대부분의 action은 task-unrelated이며, 이들의 상대적 ordering이 보존되면 unrelated task에 대한 성능(즉, OOD 성능)이 유지된다.

4.2 Comparison to End-to-End RL on SWE-Bench

실험 의도: SWE-Bench는 E2E RL이 표준 학습 방법인 도메인이므로, PivotRL과 E2E RL의 direct comparison이 가능하다.

[Figure 1 설명]: (a) SWE-Bench accuracy vs cumulative rollout turns. PivotRL은 ~133K cumulative turns에서 32.67% 도달, E2E RL은 ~542K turns에서 동일 정확도 도달 → ##PivotRL이 ~4× fewer rollout turns##. (b) SWE-Bench accuracy vs cumulative wall-clock time. 동일 compute node에서 PivotRL이 ##~5.5× less wall-clock time##으로 동일 정확도 달성.

PivotRL의 구성: 1024 batch (64 prompts × 16 generations), 1 turn/sample, step 130에서 32.67%. E2E RL: 512 batch (16 prompts × 32 generations), 12-25 turns/trajectory, step ~72에서 32.67%.

  • ##두 구성 요소 모두 필요## — pivot filtering + functional reward (59.68)는 SFT (58.44)를 상회하지만, functional reward를 제거하고 strict reward를 사용하면 (57.34) SFT보다 낮아진다.

5. Related Works

Agentic LLM Training

Agentic 학습의 핵심 방법론은 (1) SFT를 통한 offline trajectory cloning과 (2) RL을 통한 online multi-turn credit assignment이다. SFT는 catastrophic forgetting과 superficial alignment 문제를 겪고, offline imitation의 suboptimality는 task horizon에 따라 quadratically 증가한다. RL은 online data aggregation으로 이 forgetting을 완화하지만 compute 비용이 높다. PivotRL은 SFT trajectory를 turn-level verifier reward로 on-policy 최적화하여 두 접근의 장점을 결합한다.

Adapting Behavior Cloning into RL

SFT의 한계를 RL로 극복하는 기존 접근으로는 partial trace prefix conditioning (Setlur et al., 2026), SFT token rollout reward 사용 (Ming et al., 2026) 등이 있다. PivotRL은 turn-based pivot selection과 functional reward를 통해 multi-turn tool use에 특화된 접근을 제안한다.

6. Conclusion

PivotRL은 long-horizon agentic task를 위한 turn-level RL 알고리즘이다. 4개 agentic 도메인에 걸친 실험에서, 동일 데이터에 대해 SFT 대비 +4.17% in-domain accuracy, +10.04% OOD retention을 달성한다. SWE-Bench에서 E2E RL과 동등한 accuracy를 ~4× fewer rollout turns으로 달성한다. 성능 향상의 원천은 on-policy rollout에 의한 더 나은 action-space coverage와 verifier-derived reward이다. 향후 연구 방향으로 non-programmatic verifier (LLM-as-a-judge), process reward model, dynamic online reward profiling을 계획한다.

비판적 분석

강점

  • 명확한 이론-실험 정합: Proposition 3.1 (mixed-outcome filtering), Theorem 3.2 (variance = gradient), Theorem 3.3 (functional reward = ordering preservation)의 이론적 분석이 ablation 결과와 정확히 일치한다.
  • 공정한 비교: SFT와 PivotRL이 동일 데이터, 동일 base model, 동일 prompt에서 비교되어 결과의 신뢰도가 높다.
  • OOD 보존의 인상적 결과: 8개 OOD 벤치마크에서 평균 +0.21pp라는 near-zero change는 매우 강한 결과.
  • Production-scale 검증: Nemotron-3-Super에서의 실제 적용이 방법론의 scalability를 실증한다.
  • Compute 효율성: E2E RL 대비 4× fewer turns, 5.5× less wall-clock time은 실용적 가치가 높다.

약점 및 한계

  • SWE-Bench In-Domain에서 SFT 대비 열세: PivotRL 32.67 vs SFT 37.40. Software engineering이 가장 중요한 agentic 벤치마크인 점을 고려하면 주의가 필요.
  • Verifier 의존성: 각 도메인마다 domain-specific verifier가 필요. Verifier 설계의 품질이 성능에 직접 영향. Non-programmatic verifier(LLM judge)는 future work으로 남김.
  • 단일 base model: Qwen3-30B-A3B-Thinking-2507에서만 실험. 다른 모델 크기나 아키텍처에서의 일반화는 미검증.
  • Offline profiling 비용: Pivot selection을 위해 각 turn에 $K$개의 rollout이 필요. 이 offline profiling 비용이 명시되지 않음.
  • 표준편차 미보고: 단일 run 결과로 보이며, variance가 보고되지 않음.
  • Pivot selection의 정적 특성: Offline에서 한 번 선택되면 학습 중 policy가 변해도 pivot set이 업데이트되지 않음. Dynamic pivot selection은 future work.

Key Takeaways

  1. SFT의 OOD 저하는 구조적 문제: 4개 도메인 모두에서 SFT가 심각한 OOD regression을 유발하며 (-9.83pp 평균), 이는 이 방법의 근본적 한계이다. Agentic post-training에서 SFT만으로는 insufficient.
  2. "모든 turn이 평등하지 않다": Random sampling 시 71%의 turn이 uninformative (모두 성공 or 모두 실패). Pivot filtering으로 이 낭비를 제거하면 동일 rollout budget에서 훨씬 강한 학습 signal을 얻을 수 있다.
  3. Functional equivalence가 핵심: Exact-match reward는 generative action space에서 과도하게 restrictive하다. "같은 결과를 내면 정답"이라는 functional reward가 더 나은 credit assignment를 제공하며 OOD 보존에도 기여한다.
  4. Turn-level RL이 trajectory-level RL의 viable alternative: Full multi-turn rollout이 필수라는 가정은 틀릴 수 있다. Expert trajectory의 중간 state에서 시작하는 single-turn RL이 4× 적은 비용으로 comparable performance를 달성한다.
  5. Agentic RL의 scalable recipe: PivotRL은 Nemotron-3-Super에서 실증되었듯이 production-scale multi-environment RL 파이프라인에 자연스럽게 통합 가능하다.
