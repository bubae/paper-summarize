TL;DR

  • 문제: 강력한 LLM들이 각자 다른 도메인에 특화되어 있으며 ##어떤 단일 LLM도 모든 task에 universally optimal하지 않다##. 기존 multi-agent coordination은 ##손으로 설계한 scaffold##(MoA, RouterDC, MASRouter, Smoothie 등)에 의존하며, 모델 선택을 단순 routing classifier로 환원하거나 인간이 미리 정의한 topology에서만 선택 가능 — ##expressivity가 본질적으로 제약##.
  • 해결: ##7B Conductor 모델##을 RL(GRPO)로 학습해 ##자연어로 임의의 agentic workflow를 직접 출력##시킨다. 각 step은 (subtask 텍스트, 호출할 worker model id, access list)의 triple. ##Pool 안의 worker는 그대로 두고##, Conductor가 (i) 누가 무엇을 할지 결정, (ii) 그 worker에 줄 prompt를 ##구체적으로 prompt-engineer##, (iii) workflow의 정보 흐름(누가 누구의 답을 보는지) topology를 ##모두 한 번에## 자연어로 설계.
  • 왜 작동하는가: training이 ##end-to-end reward maximization##이므로 prompt engineering, refinement, meta-prompt optimization 같은 collaboration 패턴이 ##암묵적 보상 신호로부터 자연 발현##. 또한 ##자연어가 output medium##이라 routing classifier가 표현할 수 없는 ##임의의 tree/chain/recursive## topology를 자유롭게 표현 가능.
  • 비유: 오케스트라 ##지휘자##와 같다. 지휘자는 베이스 한 음도 안 친다. 그러나 ##누가 언제 들어오고##, ##어떻게 연주해야 하고##, ##누가 누구의 멜로디를 듣고 화답할지## — 그 ##coordination만으로## 단일 명연주자보다 더 큰 음악을 만든다.

핵심 기여 (Key Contributions)

  1. RL Conductor 패러다임 정립: 자연어를 output medium으로 하는 end-to-end RL을 통해 ##powerful coordination strategy가 LLM에서 자연 발현##됨을 실증. workflow delegation을 API call로 묶어 ##tool-using RL의 새 확장##을 정의.
  2. SOTA 달성: 7B 단일 모델이 LiveCodeBench v6에서 ##83.93%로 OpenAI O-series 등 모든 단일 LLM을 능가##하며 official online leaderboard 1위. GPQA-Diamond 87.5%, AIME25 93.3%, MATH500 99.4%, MMLU 94.1%, RLPR 44.75% — 7개 in-domain + out-of-domain task ##평균 77.27%##로 closest baseline GPT-5(74.78)를 3점 능가.
  3. Multi-Agent baseline 압도: MoA(62.13), Smoothie(56.48), MASRouter(56.89), RouterDC(52.41)을 ##평균 72.35##로 전부 능가. 더욱이 ##평균 호출 수 3회##로 MoA(8회), MASRouter(4–5회)보다 적음. 5× consensus / 5× self-reflection 같은 비싼 inference-time scaling도 능가.
  4. Dynamic worker pool extension: random subset에 대해 finetune하면 ##임의의 사용자 정의 agent set##(open-only, closed-only, custom)에 generalize. closed-only는 pretrained 성능 유지, open-only는 Claude Sonnet 4 능가.
  5. Recursive topology = 새로운 test-time scaling 축: Conductor가 ##자기 자신을 worker로 선택## 가능하게 되면, 첫 attempt의 결과를 보고 strategy를 revise하는 ##재귀적 self-referential reasoning##이 발현. BigCodeBench에서 +2.2점 추가 향상 (37.8 → 40.0).
  6. Emergent patterns 분석: 학습 중에 ##targeted instruction, prompt engineering, planner/verifier/refiner 역할 분담, 난이도별 step 수 조정##이 모두 emergent하게 등장. 3B vs 7B Conductor 비교에서 ##agent selection은 같지만 prompt engineering 품질 차이##가 최종 성능 격차를 만듦.
  7. 2× H100 학습: ##200 GRPO iterations, batch 256, no KL regularization##으로 단 ##2 H100 GPU에서 학습 가능##. Code + model weights 공개. ##학원 규모에서 재현 가능한 frontier-level 결과##.

배경 (Background)

RL Reasoning Paradigm: DeepSeek R1 Style

R1 (Guo et al., 2025)이 정립한 RL-for-reasoning recipe는 LLM $\pi_\theta$를 ##custom system prompt 하에서 verifiable problem set $D = \{(q_i, s_i)\}$##에 대해 학습:

  • Model이 <think>/<solution> tag 안에 reasoning + final answer를 생성.
  • Reward: format 실패 시 $r_i = -1$, format OK + 정답 일치 시 $r_i = 1$, format OK + 오답 시 $r_i = -0.5$.

학습은 ##GRPO## (Shao et al., 2024)로 진행 — group $\{o_1, ..., o_G\}$ rollout을 만들고 KL-penalized clipped policy objective:

$$J(\theta) = \mathbb{E}_{q \sim D, \{o\}^G_i \sim \pi_\theta(\cdot|q)}\left[\frac{1}{G}\sum_{i=1}^G \left(\min(r_i A_i, \text{clip}(r_i, 1-\epsilon, 1+\epsilon) A_i) - \beta \mathbb{D}_\text{KL}(\pi_\theta \| \pi_\text{ref})\right)\right] \quad (1)$$

Monte-Carlo advantage:

$$A_i = \frac{r_i - \text{mean}(\{r_1, ..., r_G\})}{\text{std}(\{r_1, ..., r_G\})} \quad (2)$$

이 recipe로 self-emergent thinking이 unprecedented task specialization을 만들었다.

왜 multi-agent coordination이 필요한가

Chang et al. (2024)의 survey가 명확히 한 사실: ##어떤 단일 LLM도 모든 task에서 universally optimal하지 않다##. GPT-5는 math/coding에 강하고, Gemini는 scientific reasoning(GPQA-Diamond)에서, Claude는 code generation with diverse function calling에서 강점. ##Granular sub-task level에서도 다른 모델이 다른 역할에 최적##.

방법 (Method): The RL Conductor

3.1 Framing Agent Coordination in Natural Language

Conductor task 정의

Conductor의 objective: input question $q_i$에 대해 ##indirectly solve## — agent들의 workflow를 설계함으로써. ##Agentic workflow는 workflow steps의 sequence##로 정의되며, 최종 출력이 Conductor의 response $o_i$로 반환.

각 step은 3-tuple:

  • Subtask string: 그 step의 자연어 sub-instruction.
  • Assigned worker agent: 정수 model id (e.g., Model 0, Model 1, ...).
  • Access list: 이전 step들 중 ##어느 sub-task solution을 worker context에 포함##할지 indexing.

구조는 ##chain-of-thought 이후 3개의 Python list##로 parse:

model_id = [2, 0]
subtasks = ["Develop an efficient algorithm to count the number of complete subarrays of an array",
            "Implement the algorithm described by the previous agent in Python"]
access_list = [[], ["all"]]

여기서 worker 2 (planner)가 알고리즘을 설계, worker 0이 ##worker 2의 답을 access##해 Python으로 구현. access_list[i] = "all"은 이전 모든 step 가시, []는 empty.

##이 구조는 simple best-of-N과 sequential chain부터 parallelizable arbitrary tree-structured까지 모든 topology를 표현##. 인간 설계 scaffold 없이 Conductor의 자연어 출력만으로.

Workflow execution & learning dynamics

각 workflow step은 ##sequentially 실행##: 지정된 worker agent에게 (i) subtask string + (ii) access list에 따른 이전 step의 conversational template으로 prompt.

Reward $r_i$ for Conductor response:

  • Format condition: model_id / subtask / access_list 가 parse 불가 → $r_i = 0$.
  • Correctness condition: well-formatted workflow를 실행한 최종 output이 ##정답 일치 시 $r_i = 1$, 오답 시 $r_i = 0.5$##.

주목: format failure는 ##0##이고 valid but wrong은 ##0.5##. 이 ##0.5 baseline reward##가 Conductor가 ##최소한 format을 학습##하도록 인센티브화하고, 그 위에서 correctness를 추구하게 만든다.

훈련은 ##GRPO formulation (Section 2)##을 그대로 사용 — 단지 reward signal이 ##agentic workflow 실행의 결과##라는 점만 다르다. 이 simple recipe로:

  • Problem breakdown 발현,
  • Prompt-engineered subtask가 각 worker의 strength에 매칭,
  • Independent attempt + 최종 debate round의 communication 전략 발현.

3.2 Extending the RL Conductor

Adaptive Worker Selection

Pre-trained Conductor를 ##randomly sampled $k$-model subset## ($k \le n$, $n$ = total pool)에 대해 finetune해 ##사용자 cost/availability constraint에 맞게 generalize##. Training 후 ##desired subset에 대해 strong performance##.

Recursive Topologies & Test-Time Scaling

Recursive call: workflow 중 Conductor가 ##자기 자신을 worker##로 선택 가능. 이때 parent output + 직전 agent response가 추가 input으로 들어가며, ##new agentic workflow를 instantiate하거나 final solution을 직접 반환##할 결정 가능.

Infinite recursion 방지: initial root call 이후 ##최대 specified depth##까지만 허용.

Recursive capability를 unlock하는 방법: pretrained Conductor를 ##half-batch에 1 recursive call을 manually instantiate##하면서 같은 RL recipe로 finetune. After training, inference 시 max recursion calls를 adaptively 증가시켜 ##새로운 형태의 test-time scaling##.

이 scheme의 의미: open-ended chain-of-thought 외에 ##recursion as tunable compute axis##.

실험 (Experiments)

4.1 Training & Evaluation Setup

  • Base model: Qwen2.5-7B (Hui et al., 2024). Max completion 1024.
  • Worker pool: Gemini-2.5-Pro, Claude-Sonnet-4, GPT-5, DeepSeek-R1-Distill-Qwen-32B, Gemma3-27B-instruct, Qwen3-32B (non-thinking + thinking).
  • Training data: ##960 problems## across math/code/general reasoning, from MATH500, MMLU, RLPR, LiveCodeBench V1.
  • Training: 200 GRPO iterations, batch 256, 64 rollouts per sample (4 questions × 64 rollouts), AdamW, lr 1e-6 with cosine, ##no KL regularization, no reference model sync##.
  • Compute: ##2× NVIDIA H100 80GB##.

Evaluation tasks:

  • In-domain: MATH500, MMLU, RLPR, LiveCodeBench V6.
  • Unseen (OOD): AIME25, BigCodeBench, GPQA-Diamond.

4.5 Analyzing Conductor Properties

Conductor scale (Figure 7)

실험 의도: 같은 recipe로 ##3B Conductor##를 학습하면 어떻게 다른가?

  • Agent selection은 동일: 3B도 7B와 같은 ##top-3 strongest model##로 distribution converge.
  • 최종 성능은 격차: LCB에서 7B Conductor가 ##GPT-5 (단일 worker)를 능가##, 3B는 GPT-5에 못 미침.
  • 차이의 원인: ##prompt engineering quality##. 같은 모델을 호출해도 7B가 더 정교한 subtask string을 만든다. ##base model의 natural language capability가 직접 prompt engineering quality로 변환##되어 Conductor가 worker에 대해 ##새로운 agency level##을 얻는다.

Task & difficulty adaptivity (Figure 8)

학습이 진행될수록 task별 step 수 분포가 ##task complexity에 adaptive##:

  • LiveCodeBench (hard coding): 3–4 step workflow가 다수. multi-planner + implementer + verifier 패턴.
  • MMLU (factual recall): 1–2 step workflow가 다수. 1-shot or simple targeted retrieval.

Output trace 분석에 따르면 ##model이 task complexity에 대해 explicitly 추론##한 뒤 workflow를 specify. ##unnecessary computation을 줄이고 hard task에 compute 집중##.

Emergent Behaviors (관찰)

Appendix F.1에 따르면 학습 중 ##자연 발현##한 coordination 패턴:

  1. Sequential with high-level planner: planner → executor → (refiner) → (checker). 가장 일반적 패턴.
  2. Tree coordination: 여러 agent가 independent하게 작업 → aggregator가 통합.
  3. Pure recall: 사실적 recall task에서는 agent-to-agent collaboration 없이 single agent로.
  4. Step-by-step logical decomposition: complex multi-step problem에서 각 agent가 한 단계씩.
  5. Conductor task abdication: Conductor가 ##더 강한 frontier model##(예: Gemini 2.5 Pro)에 ##subtask design 권한 자체를 위임##. — 저자들의 personal favorite.

Figure 3 시각화: 학습 초반(iteration ~25)에는 sound subtask를 issue하지만 ##collaborative strategy를 활용하지 못함##. Iteration ~200 무렵엔 ##planner 활용 + worker간 reasoning share + verification/refinement##까지 발현 → 더 큰 worker agent들의 individual 성능을 능가.

비판적 분석 (Critical Analysis)

  • API 비용 의존: Worker pool이 ##expensive frontier API (Gemini 2.5 Pro, Claude Sonnet 4, GPT-5)##를 포함. Conductor 자체는 7B(저렴)지만 inference 단가는 결국 ##worker 호출 비용에 비례##. open-only subset이 작동함을 보였지만 ##absolute performance gap##이 여전히 존재.
  • Worker capability ceiling: Conductor의 성능 ceiling은 ##worker pool의 best combination##. Conductor가 GPT-5/Claude/Gemini를 능가하지만, 그 ##worker들이 약하면 Conductor 효과도 제한##. 향후 worker가 더 강해지면 Conductor의 marginal value가 어떻게 변할지 불명.
  • Reward 신호의 verifiability 의존: Math/code/MMLU 같은 ##binary correctness가 가능한 task##에만 적용. ##open-ended creative writing이나 multi-modal task##에서는 reward 정의 자체가 challenge.
  • Training data 다양성 제약: 단 960 problems × 4 domain. ##다른 domain (translation, planning, agentic web)## generalization은 추후 검증 필요. AIME25/GPQA-D/BCB OOD 결과는 promising하지만 ##domain transfer## 자체는 아님.
  • Recursive scaling의 모호한 budget: recursive call 수가 inference cost로 직결되나 ##budget 자동 조정 메커니즘이 없음##. Table 2의 +2.2점 gain은 valuable하나 ##2× 호출 비용에 대한 정량 trade-off##가 깊이 분석되지 않음.
  • 3B vs 7B의 prompt engineering 격차 origin이 미해명: 두 모델이 같은 agent를 선택하지만 7B가 더 나은 prompt를 만드는 ##precise mechanism## (어떤 capability가 prompt quality를 결정)에 대한 ablation 부재.
  • Format failure의 0 baseline 설계 정당성: format 실패에 0, wrong에 0.5 — 이 ##0.5 difference##가 cold-start 학습을 가능하게 한다고 paper는 implicit하게 가정. 다른 reward shape(예: -1 / 0 / 1)와의 비교 ablation이 없음.
  • Multi-agent baseline 학습 시 same data: MASRouter/RouterDC도 same 7-agent pool과 same training data로 학습되었지만 ##scaffold 자체가 더 expressive하지 못해서## 성능이 낮은 것인지, ##학습 신호가 본질적으로 다른 것인지## 구별 어려움. 특히 MASRouter는 ##fixed prompt template + scaffold##라 expressivity 차이가 본질적이라는 paper 설명은 타당하나 ##empirical contrastive analysis##가 부족.
  • Reproducibility 부분 우려: 같은 worker pool (특히 GPT-5)이 paper 작성 시점과 다르게 진화하면 결과 재현 어려움. ##API 모델은 시간에 따라 변동##하므로 long-term reproducibility는 challenge.
  • Few-shot OOD의 surprising 발견의 메커니즘: paper의 설명("OOD가 reward-hackable lazy repetition을 막는다")이 합리적이지만 ##정량적 검증##(예: in-distribution few-shot 시 repetition 빈도 측정)이 부족.

Key Takeaways

  • "Coordination strategy는 emergent capability": end-to-end reward maximization에서 prompt engineering, planner/verifier 분담, task adaptivity가 ##모두 자연 발현##. ##explicit scaffold design이 필요 없다##는 강력한 demonstration.
  • "자연어가 ultimate output medium": routing classifier나 graph edge 같은 ##구조화된 output space##는 expressivity가 본질적으로 제약. ##자연어가 임의의 topology를 표현##할 수 있어 더 큰 design space를 열어준다. 다른 ##meta-agent task##(toolchain orchestration, agent composition)에 즉시 transferable.
  • "7B가 frontier를 능가할 수 있다": ##2 H100 GPU와 200 GRPO iteration##으로 학습된 7B model이 GPT-5/Claude/Gemini를 능가. ##collective intelligence가 single-model scale을 보완##할 수 있음을 입증. ##학원 규모 연구의 새 가능성##.
  • "Recursive topology = 새 test-time scaling 축": open-ended CoT 외에 ##self-referential recursion##이 tunable compute axis. 어려운 task에서 추가 compute를 dynamic allocate.
  • "Adaptive worker pool은 distillation의 dual": 비싼 frontier model을 ##직접 distill하는 대신 그들을 더 똑똑하게 조합##. distillation이 capability를 압축한다면 conduction은 capability를 ##활용##. 두 패러다임이 보완적.
  • "Prompt engineering quality가 model scale에 비례": 3B vs 7B 비교. ##base model의 natural language capability가 직접 coordination quality로 변환##. 향후 더 큰 base로 학습하면 추가 gain 기대.
  • "OOD few-shot이 in-domain보다 효과적": counter-intuitive하지만 ##reward-hackable strategy의 lazy repetition을 막는 explicit benefit##. cold-start training의 일반적 lesson: ##exploration을 incentivize하는 prior conditioning##.
