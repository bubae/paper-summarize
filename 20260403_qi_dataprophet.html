DataProphet: Demystifying Supervision Data Generalization in Multimodal LLMs

TL;DR

  • 문제: MLLM (Multimodal Large Language Model) 학습 시, 어떤 supervision 데이터가 특정 target benchmark에 도움이 되는지를 학습 전에 예측할 방법이 없다. "직관적으로 유사한 데이터가 도움이 된다"는 통념은 실험적으로 unreliable함이 밝혀졌다 — 예: OCR 데이터가 chart task보다 spatial reasoning task에 더 도움.
  • 해결: DataProphet — ##training-free metric##으로 multimodal perplexity, cross-dataset similarity (question/answer/image), data diversity (silhouette + entropy)를 곱셈 형태로 결합. 학습 없이 source dataset이 target benchmark에 미치는 영향을 예측한다.
  • 왜 작동하는가: (1) Perplexity가 source 데이터의 난이도와 headroom을 반영하고, (2) cross-modal similarity가 distribution 정렬을 포착하며, (3) diversity가 일반화 가능한 skill coverage를 측정한다. 곱셈 형태는 어느 한 요소가 약해도 전체 점수가 낮아지도록 하여 모든 조건의 동시 충족을 요구.
  • 비유: 시험 준비를 할 때 "이 참고서가 기말고사에 도움이 될까?"를 미리 판단하는 것과 같다. 참고서의 난이도(perplexity), 시험 범위와의 겹침(similarity), 문제 유형의 다양성(diversity)을 종합하면, 실제로 풀어보지 않아도 유용성을 예측할 수 있다.

핵심 기여 (Key Contributions)

  1. Data Influence의 반직관적 특성 규명: 14개 vision-language dataset, 7개 task에 대한 체계적 실험으로 ##data influence가 비대칭적이고, task category가 아닌 individual dataset에 의존##하며, text-rich 데이터가 vision-centric task를 더 잘 도울 수 있음을 실증.
  2. DataProphet — Training-Free Data Influence Predictor: Multimodal perplexity × cross-dataset similarity (Q, A, Image) × data diversity의 product-form metric. ##Kendall's $\tau$ = 86.0%##로 실제 학습 후 influence ranking과 강하게 상관.
  3. Data Selection 응용: Fixed compute budget 하에서 DataProphet 기반 reweighting이 uniform selection 대비 +3.4% (real data), +6.9% (synthetic data) 개선. SoTA training-based selection(ICONS) 대비 +1.4% 우수하며, ##oracle(실제 실험 결과 기반) 대비도 +0.2% 상회##.
  4. RL Post-Training에도 적용 가능: GRPO 학습에서 DataProphet 기반 allocation이 equal allocation 대비 +1.2% (real) / +1.3% (synthetic) 개선.

2. Data Influence Analysis

2.1 실험 설계

Setup: Base model InternVL3-2B에 14개 source dataset을 각각 개별적으로 SFT(20K samples, 1 epoch), 14개 target test set에서 evaluation. Relative performance gain $\Delta_{s \to t} = (A_t^s - A_t) / A_t$로 data influence 정량화.

7개 Task Family (각 2개 dataset):

  • OCR: OCR-VQA (~1002K), ScreenQA (~86K)
  • Chart understanding: ChartQA (~33K), Chart2Text (~44K)
  • Document understanding: DocVQA (~50K), SujetFinance-QA (~107K)
  • General VQA: A-OKVQA (~24.9K), VC-GVQA (~438K)
  • Spatial reasoning: Open-Spatial (~8.7M), CLEVR-Relation (~160K)
  • Counting: CLEVR-Counting (~160K), TallyQA (~287K)
  • Map reasoning: GeomVerse (~23K), MapQA (~800K)

2.2 핵심 발견

[Figure 2 설명]: 14×14 data influence heatmap. 각 cell은 source dataset으로 SFT 후 target test set에서의 relative improvement (%). Diagonal(same-dataset)이 가장 크지만, off-diagonal에서 반직관적 패턴이 다수 관찰됨.

발견 1: Data influence는 비대칭적. $\Delta_{s \to t} \neq \Delta_{t \to s}$. 예: Open-Spatial → DocVQA = 15.92%, DocVQA → Open-Spatial = 26.14%.

발견 2: 같은 task category 내에서도 influence가 다르다. OCR-VQA로 학습 시 ScreenQA(같은 OCR) 개선 17.88%이지만, GeomVerse(map) 개선 21.74%. ##Task category가 아닌 individual dataset 수준에서 influence가 결정됨##.

발견 3: Text-rich 데이터가 vision-centric task를 더 많이 도울 수 있다. ScreenQA(OCR)로 학습 시 chart task 개선(ChartQA 5.61%, Chart2Text 6.13%)보다 spatial task 개선(Open-Spatial 12.86%, CLEVR-R 7.96%)이 더 큼.

3. DataProphet: Demystify Data Influence

3.1 Evaluation Protocol — Two-Way Kendall's $\tau$

Target-centric: 주어진 target $t$에 대해 14개 source를 DataProphet 점수로 ranking → 실제 $\Delta_{s \to t}$ ranking과 Kendall's $\tau_t$ 계산. $\bar{\tau}_{\text{Tgt}} = \text{avg}(\tau_t)$.

Source-centric: 주어진 source $s$에 대해 14개 target을 점수로 ranking → 실제 $\Delta_{t \to s}$ ranking과 $\tau_s$ 비교. $\bar{\tau}_{\text{Src}} = \text{avg}(\tau_s)$.

$$\bar{\tau} = \frac{\bar{\tau}_{\text{Tgt}} + \bar{\tau}_{\text{Src}}}{2}$$

3.2 DataProphet Metric

Component 1: Multimodal Perplexity

$$\text{PPL}(\mathcal{D}) = \exp\left(- \mathbb{E}_{(I,Q,A) \sim \mathcal{D}} \mathbb{E}_{t \sim |T|} \log p_\theta(a_t | I, \tau_Q(Q), a_{Base MLLM $p_\theta$의 answer token에 대한 perplexity. 높은 perplexity = 모델에게 더 어려운 데이터 = 학습 시 더 큰 headroom. 단독으로는 $\bar{\tau}_{\text{Tgt}} = 0.274$로 부족.

Component 2: Cross-Dataset Similarity

Base MLLM의 frozen encoder로 question, answer, image 각각의 embedding을 추출하고, source와 target 간 expected cosine similarity를 계산:

  • QSim: Question embedding 간 cosine similarity
  • ASim: Answer embedding 간 cosine similarity
  • ISim: Image embedding 간 cosine similarity

Perplexity × similarity만으로는 $\bar{\tau}_{\text{Tgt}} = 0.658$. Image similarity(0.627)와 diversity가 추가로 필요.

Component 3: Source Dataset Question Diversity

Source dataset의 question embedding을 K-means ($K=10$)로 clustering한 뒤, Silhouette coefficient + normalized entropy로 diversity 측정:

$$\text{Sil} = \mathbb{E}_{u \sim \mathcal{U}} \left[ \frac{b(u) - a(u)}{\max\{a(u), b(u)\}} \right] \quad (2)$$

$a(u)$: mean intra-cluster distance, $b(u)$: minimum mean inter-cluster distance. Normalized entropy $\text{H} = -(\log K)^{-1} \sum_k \pi_k \log \pi_k$. Diversity = Sil + H (높을수록 well-separated, balanced clusters).

최종 Metric

$$\mathcal{M}(s \to t) = \frac{\text{QSim} \cdot \text{ASim} \cdot \text{ISim} \cdot \text{PPL}(s) \cdot (\text{Sil} + \text{H})}{\text{PPL}(t)} \quad (3)$$

직관적으로 말하면, ##"source가 어렵고(높은 PPL(s)), target이 쉽고(낮은 PPL(t)), 둘이 비슷하고(높은 Sim), source가 다양할수록(높은 Div)" 해당 source의 target에 대한 영향력이 크다##. Product form은 어느 한 요소라도 낮으면 전체가 낮아져 모든 조건의 동시 충족을 강제.

3.3 결과 및 분석

DataProphet의 예측력 (Table 1): $\bar{\tau}_{\text{Tgt}} = 0.863$, $\bar{\tau}_{\text{Src}} = 0.857$, ##$\bar{\tau} = 0.860$##. 모든 task category에서 consistently 높은 상관.

실패한 heuristic들: Question difficulty, model familiarity (image/question), answer length 등은 모두 $\tau$를 오히려 악화시킴 (최대 -0.15).

4. DataProphet for Data Selection

4.1 Real-Data Reweighting

Total budget $N = 14 \times 20\text{K} = 280\text{K}$ samples. DataProphet score에 비례하여 14개 source의 sampling 비율 결정.

4.2 Synthetic-Data Selection

14개 dataset × 10K images × 2 generators (GPT-5, Gemini 2.5 Pro) × 5 QA pairs = ~1.4M synthetic pairs. Instance-level scoring (diversity term 제외):

$$\mathcal{M}(\text{syn\_d} \to \mathcal{T}) = \frac{\text{QSim} \cdot \text{ASim} \cdot \text{ISim} \cdot \text{PPL}(\text{Syn\_D})}{\text{PPL}(\mathcal{T})} \quad (4)$$

5. Related Work

Data Mixtures for LMs: DoReMi(proxy model로 domain weight 학습), DSIR/LESS(gradient 기반 selection), DataComp(dataset filtering 표준화). DataProphet는 proxy run이나 gradient 없이 ##pre-finetuning, training-free## decision rule을 제공하는 점에서 차별화.

Data Mixtures for Multimodal LMs: ICONS(gradient-based influence consensus), MoDoMoDo(RL formulation). DataProphet는 real/synthetic data 모두에 직접 적용 가능한 compute-light 접근.

비판적 분석 (Critical Analysis)

강점

  • 반직관적 발견의 실증적 가치: "같은 task data가 가장 도움"이라는 통념을 14×14 실험으로 체계적으로 반증. Data selection 연구의 fundamentals에 기여.
  • 극도의 단순성과 효과: Training 불필요, frozen encoder embedding + perplexity만으로 $\tau=0.86$ 달성. Oracle조차 상회하는 결과는 인상적.
  • ICLR 2026 채택: Peer review를 통과한 검증된 연구.

약점 및 한계

  • 단일 base model 의존: 모든 실험이 InternVL3-2B 기반. 다른 MLLM(LLaVA, Qwen-VL, GPT-4V 등)에서 동일한 influence pattern과 DataProphet의 예측력이 유지되는지 미검증.
  • 소규모 SFT 실험: 각 source에서 20K samples, 1 epoch만 학습. 대규모 학습(수백K samples, multiple epochs)에서의 influence dynamics는 다를 수 있음.
  • Product form의 이론적 근거 부재: 왜 additive가 아닌 multiplicative combination이 최적인지에 대한 이론적 justification이 없고, heuristic하게 선택됨.
  • Oracle을 상회하는 결과의 해석: DataProphet가 Oracle(실제 실험 기반 reweighting)을 +0.2% 상회하는 것은 Oracle의 reweighting 방식이 최적이 아닐 가능성을 시사하지만, 이 차이가 통계적으로 유의미한지 불확실.
  • VQA task에 국한: 모든 benchmark가 visual QA 형태. Text generation, visual grounding, video understanding 등 다른 multimodal task에서의 적용성 미확인.

Key Takeaways

  1. "비슷한 데이터가 도움이 된다"는 직관은 MLLM에서 unreliable: Data influence는 task category가 아닌 individual dataset 수준에서 결정되며, 비대칭적이고 반직관적이다. Data curation 시 task-level 유사성에만 의존하면 안 된다.
  2. Perplexity가 data influence의 가장 강력한 단일 predictor: 모델에게 "어려운" 데이터가 학습에 더 유용하다는 것은 curriculum learning의 관점과 일치. Image similarity와 diversity가 보완적 역할.
  3. Training-free metric으로 training-based method를 능가 가능: DataProphet의 $\tau=0.86$과 Oracle 초과 결과는 expensive gradient computation 없이도 effective data selection이 가능함을 시사. 특히 synthetic data selection에서 더 큰 이점.
  4. Multimodal data selection에서 vision과 text 모두 중요: Image similarity 제거 시 $\tau$가 0.235 하락으로, vision 정보가 text 못지않게 핵심. 기존 LLM data selection 연구가 text에만 초점을 맞춘 것의 한계를 보여줌.
