TL;DR

  • 문제: Continual learning에서 ##on-policy RL은 catastrophic forgetting을 줄이지만 명시적 reward function이 필요##하다. 반면 expert demonstration만 있을 때 표준 SFT는 ##off-policy##라 sequential fine-tune 시 망각이 심하다.
  • 해결: ##Self-Distillation Fine-Tuning (SDFT)##. 동일 모델 한 본을 두 역할로 사용 — ##student는 query만 조건##(P=π(·|x))으로, ##teacher는 query + demonstration을 조건##(Q=π(·|x,c))으로. student가 생성한 trajectory에서 reverse KL $\mathrm{KL}(P\Vert Q)$를 최소화 → ##reward 없이 demonstration만으로 on-policy 학습##.
  • 왜 작동하는가: 모델의 ##in-context learning (ICL) 능력##이 demonstration-conditioned 분포를 ##trust-region IRL의 optimal policy 근사##로 만든다 — base 분포와 가까이 머물면서도 task 성능을 회복하는 균형점을 자연스럽게 찾는다.
  • 비유: 같은 사람의 ##"두 자아"##를 가르침에 활용 — 답을 보고 푸는 자아(teacher)와 보지 않고 푸는 자아(student) 사이의 차이를 학생이 따라가게 함. 외부 코치(reward function) 없이 ##자기 자신이 자기에게 코치##가 된다.

핵심 기여 (Key Contributions)

  1. SDFT 알고리즘: same-model dual-role distillation에서 reverse KL을 PG로 최적화. demonstration 조건만 다른 두 분포 사이의 차이가 ##implicit reward##로 작동.
  2. 이론적 framing — Self-Distillation as IRL: trust-region regularized RL의 closed-form solution과 ICL 가정 $\pi^*_{k+1}(y|x)\approx \pi(y|x,c)$를 결합해, SDFT의 reverse KL이 ##implicit reward $r(y,x,c)=\log\pi(y|x,c)-\log\pi(y|x)$##를 최대화하는 PG와 동등함을 증명.
  3. Continual Learning 검증: skill learning(Tool Use, Science Q&A, Medical) + knowledge acquisition(2025 자연재해 Wikipedia)에서 SFT, DFT, Re-invoke, CPT, RAG 대비 ##Pareto-superior## (new task 정확도 ↑ + prior task 보존 ↑).
  4. Sequential 3-task 실험: Tool Use → Science → Medical 순차 학습에서 ##single 모델이 모든 skill을 누적 보존## (Figure 3) — SFT는 task 전환 시 이전 skill이 무너짐.
  5. Reasoning models without reasoning data (Table 2): Olmo-3-7B-Think를 ##short answer-only data##로 학습할 때 SFT는 chain-of-thought를 무너뜨리지만 SDFT는 reasoning behavior를 보존하면서 정확도 향상 (31.2 → 43.7%).

1. 배경 (Background)

1.1 Continual Learning의 두 갈래

Foundation model을 배포 후에도 새 skill·지식을 누적시키려면 ##sequential fine-tune##이 필수. 두 종류:

  • Skill Learning: 새 task(예: tool API 호출, 의료 추론)에 적응. demonstration이 풍부.
  • Knowledge Acquisition: 사전학습 cutoff 이후의 ##새 사실##(예: 2025 미얀마 지진)을 통합.

두 경우 모두 ##기존 능력을 망가뜨리지 않으면서## 새 데이터를 흡수하는 것이 핵심.

1.2 Off-policy SFT의 한계

Demonstration $c$로 SFT하면 fine-tuned 모델이 demonstration의 ##짧고 건조한 분포##에 정렬되며, 그 과정에서 reasoning style·alignment·prior knowledge가 무너진다. Ross et al. 2011의 imitation learning 분석: ##off-policy training에서는 inference시 학생이 자기가 만든 trajectory에 들어가면 compounding error##가 누적.

1.3 On-Policy RL의 강점과 한계

RLHF·GRPO 같은 on-policy RL은 ##catastrophic forgetting을 낮춘다##(Shenfeld 2025; Chu 2025; Lai 2025). 그러나:

  • RLHF: preference pair 필요
  • RLVR/GRPO: ##explicit reward function 필요##
  • 대부분의 실세계 demonstration 데이터에는 reward가 없다.

1.4 IRL의 어려움

Inverse RL은 demonstration에서 reward를 ##추정##한 뒤 RL을 돌릴 수 있게 한다. 그러나 reward가 본질적으로 ill-posed라 ##strong structural prior##(Maximum entropy, adversarial, preference-based)가 필요. SDFT는 ##explicit IRL을 우회하고 ICL을 reward의 implicit 표현##으로 사용한다는 새 발상.

1.5 Context Distillation

기존 context distillation (Bai 2022, Snell 2022)는 fixed context(few-shot examples, behavioral guideline)에서 student를 학습. SDFT의 차이:

  • ##Per-query specific demonstration##을 teacher에 조건 — global behavioral prior가 아닌 instance-wise IRL.
  • ##On-policy##: student의 자기 분포 위에서 학습 (compounding error 완화).

2. 방법: SDFT

2.1 Dual-Role 모델

Foundation model $\pi$ 한 본을 다음 두 분포로 사용:

  • Student: $\pi_\theta(\cdot|x)$ — query $x$만 조건. 학습 대상.
  • Teacher: $\pi(\cdot|x, c)$ — query $x$ + demonstration $c$ 조건. ##학습되지 않음## (또는 EMA로 천천히 업데이트).

Teacher 구성용 prompt는 단순 형식:

<Question> This is an example for a response to the question: <Demonstration> Now answer with a response of your own, including the thinking process:

이 prompt가 ##demonstration을 verbatim 복사하지 않고 ICL로 task 의도를 추출##하도록 유도.

2.3 Self-Distillation as Inverse RL (Section 3.1)

Trust-region regularized RL update:

$$\pi_{k+1} = \arg\max_\pi \mathbb{E}_{y\sim \pi}[r(y,x)] - \beta D_{\mathrm{KL}}(\pi(\cdot|x)\Vert \pi_k(\cdot|x)) \quad (3)$$

Closed-form optimal policy (Korbak 2022, Rafailov 2023):

$$\pi^*_{k+1}(y|x) \propto \pi_k(y|x)\exp\left(\frac{1}{\beta}r(y,x)\right)$$

이 식을 reward에 대해 정리하면:

$$r(y,x) = \beta\left[\log\pi^*_{k+1}(y|x) - \log\pi_k(y|x)\right] + C$$

핵심 가정 (##In-Context Assumption##): ICL 능력이 충분한 모델은

$$\pi^*_{k+1}(y|x) \approx \pi(y|x,c) \quad (4)$$

즉 demonstration-conditioned policy가 unknown optimal next policy를 근사. 그러면 implicit reward는

$$r(y,x,c) = \log\pi(y|x,c) - \log\pi(y|x) \quad (5)$$

Token-level reward $r_t(y_t|y_{ $$\nabla_\theta J(\pi_k) = \mathbb{E}_{y\sim \pi_k}\left[\log\frac{\pi(y|x,c)}{\pi_k(y|x)}\nabla_\theta\log\pi_k(y|x)\right] \quad (6)$$

이는 정확히 Eq.(2) reverse KL gradient와 ##expectation 차원에서 동등##. 직관적으로 SDFT는 "##자기 자신의 더 wiser한 demonstration-aware 버전##"을 비교 기준으로 한 implicit IRL.

2.4 ICL 가정의 경험적 검증 (Section 3.2)

  1. Optimality: demonstration-conditioned teacher가 ##100% test reward##(tool call success). 50개 reasoning trace 수동 검토 — chain-of-thought 모두 valid + grounded.

3. 실험 (Experiments)

3.1 Setup

  • Base model: Qwen-2.5-7B-Instruct (대부분 실험), Olmo-3-7B-Think (Section 4.5), Qwen 3B/14B (scaling).
  • Skill Learning task 3종:
    • Science Q&A: SciKnowEval Chemistry L-3 (학부 화학)
    • Tool Use: ToolAlpaca
    • Medical: HuatuoGPT-o1 stage1 train, stage2 eval
  • Knowledge Acquisition: 2025 자연재해 Wikipedia (~200K token), GPT-4가 5× 합성 Q&A pair.
  • Baselines:
    • SFT (표준)
    • DFT (Wu 2025b): importance sampling으로 offline data를 on-policy로 다룸
    • Re-invoke (Lu & Lab 2025): SFT 후 추가 on-policy distillation
    • CPT (continual pre-training, knowledge only)
    • RAG with oracle retriever (knowledge only)
  • 평가: in-distribution accuracy, OOD accuracy(knowledge), prior capabilities(HellaSwag, TruthfulQA, MMLU, IFEval, Winogrande, HumanEval).

3.8 무엇이 성능을 결정하는가? (Figure 6)

두 ablation:

  • SFT from teacher: teacher가 만든 응답을 student가 imitate (offline)
  • Offline distillation from teacher: teacher 응답에 KL loss 적용 (offline)

둘 다 SFT보다 낫지만 SDFT보다 ##일관 underperform##. 결론: ##teacher quality 자체보다 on-policy 학습이 결정적##. 이 실험이 뒷받침하는 주장: SDFT의 gain은 단순히 더 좋은 supervision이 아니라 ##학생 자기 분포 위에서 학습한다는 사실 자체##에서 온다.

3.9 Computational Cost

SDFT는 표준 SFT 대비 ##~2.5× FLOPs / ~4× wall-clock## (on-policy rollout 필요). 그러나 Re-invoke 같은 기존 continual learning은 ##multi-stage##(SFT + on-policy training to restore)이므로 ##총 학습 시간 기준으로는 SDFT가 더 빠르다##.

4. 비판적 분석 (Critical Analysis)

  • ICL 가정의 경계: $\pi^*_{k+1}\approx\pi(\cdot|x,c)$가 핵심 — small model에서 ICL이 약하면 깨진다(Qwen 3B 실패가 직접 증거). ##SDFT의 효과는 base 모델의 ICL 강도와 강하게 결합##.
  • Optimality 검증의 일반성 부족: "100% test reward, 50 trace 수동 검토"는 ToolAlpaca 한 datapoint. open-ended·subjective task(예: 글쓰기)에서 같은 검증이 가능한지 미검토.
  • Trust-region $\beta$의 implicit성: trust-region RL framing에서 $\beta$가 명시적이지만 SDFT에서는 ICL이 그 역할을 ##암묵적##으로 한다 — controllability 손실. 더 강한 deviation을 원할 때(behavioral pivot) 어떻게 조정할지 미해결.
  • EMA hyperparameter sensitivity: teacher를 EMA update — decay rate 영향 분석은 4.6 ablation 수준에서만, sweep 부족.
  • Spurious linguistic patterns (논문 자체 인정): "Based on the text..." 같은 demonstration-prompt artifact를 학생이 흡수. ##첫 몇 토큰 masking##이라는 휴리스틱 해법만 제시 — principled solution은 open.
  • Behavioral pivot 한계: "non-reasoning model을 reasoning 모델로 transform"은 어렵다고 명시. SDFT는 ##기존 capability 보존##엔 강하지만 ##새 generation pattern 강요##엔 약함.
  • Knowledge Acquisition의 RAG 비교 한계: oracle retriever를 사용하므로 RAG 우위가 unfair. real RAG의 noisy retrieval에선 SDFT가 더 강할 가능성.
  • Computational overhead 측정의 실용성: 2.5× FLOPs는 무시할 수 없는 수준. continual learning을 자주 돌려야 하는 setting에서 비용-효과 분석 부족.
  • RL과의 비교 부재: "RL은 reward 필요"라고 motivate하지만, ##reward를 얻기 쉬운 task##에서 SDFT vs RL 직접 비교는 없음. RL이 가능할 때 SDFT가 여전히 가치 있는지 미해결.
  • Tokenizer/architecture mismatch: Yang 2024 SDFT처럼 같은 모델 두 역할이라 ##cross-architecture distillation은 불가능##.

5. Key Takeaways

  • ##ICL을 implicit reward signal로 활용##하는 IRL framing은 demonstration-only continual learning에 강력한 도구. 외부 reward function 없이도 RL의 안정성·일반화 이점을 얻는다.
  • ##Same-model dual-role distillation##(student=query, teacher=query+demonstration)은 추가 모델·데이터 없이 ##self-distillation을 self-supervision으로 변환##하는 elegant 메커니즘. 후속 연구의 framework로 활용 가능.
  • Continual learning의 ##sequential 시나리오##(여러 task 누적)는 single fine-tune보다 forgetting이 훨씬 심한데, SDFT는 ##sequential 환경 자체에서 검증된 첫 demonstration-only 방법##.
  • ##Reasoning behavior 보존##(Olmo-3 결과)은 매우 실용적 함의 — answer-only 데이터로 reasoning 모델을 fine-tune할 때의 표준 함정을 SDFT가 회피.
  • ##Scaling rule##: model size가 커질수록 ICL이 강해지고 SDFT의 격차가 커진다. open-source 7B+ era에서 점점 valuable해지는 방법.
  • 한계 인식: ICL이 약한 small model에선 효과 없음, behavioral pivot 한계, computational overhead 2.5×, EMA hyperparameter 민감도 — design 시 trade-off 인지 필요.
  • Yang 2024 "SDFT (rewriting)"와 ##이름은 같지만 메커니즘이 정반대##. Yang 2024는 데이터 전처리 (offline), Shenfeld 2026은 on-policy distillation. 독립적으로 쓸 수도, 조합할 수도 있을 것이라는 흥미로운 연구 방향.
