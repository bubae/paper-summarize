TL;DR

  • 문제: Zero-shot TTS에서 자연스럽고 표현력 있는 음성을 생성하는 것은 여전히 어려운 과제이다. 기존 hierarchical autoregressive 방식(Moshi 등)은 acoustic token을 depth-wise AR로 생성하여 느리고, speech codec이 semantic과 acoustic 정보를 분리하지 못하면 TTS의 효율성이 저하된다.
  • 해결: Voxtral TTS — (1) Voxtral Codec: hybrid VQ-FSQ 양자화로 semantic (VQ, 1 token) + acoustic (FSQ, 36 tokens) 을 분리 인코딩하는 speech tokenizer, (2) Decoder backbone (Ministral 3B): semantic token만 autoregressive로 생성, (3) Flow-Matching Transformer: acoustic token을 continuous flow-matching으로 생성. 이 hybrid 설계가 AR의 long-range consistency와 flow-matching의 rich acoustic detail을 결합한다.
  • 왜 작동하는가: (1) Semantic token을 Whisper ASR에서 distill하여 텍스트 의미와 강하게 정렬되므로 AR backbone이 intelligible한 시퀀스를 생성하고, (2) Acoustic detail은 per-frame flow-matching으로 독립 생성하여 36-depth AR 대비 8 NFE (Number of Function Evaluations)로 충분하며, (3) DPO를 semantic + acoustic 양쪽에 적용하여 WER, speaker similarity, naturalness를 동시에 개선한다.
  • 비유: 소설을 쓸 때, 줄거리(semantic)는 한 장씩 순서대로 쓰고(AR), 각 장의 문체와 분위기(acoustic)는 별도의 스타일리스트가 한꺼번에 다듬어주는(flow-matching) 것과 같다. 줄거리를 depth-wise로 한 단어씩 다듬는(기존 방식) 것보다 훨씬 효율적이고 표현력이 풍부하다.

핵심 기여 (Key Contributions)

  1. Voxtral Codec — Hybrid VQ-FSQ speech tokenizer: ##1 semantic VQ token + 36 acoustic FSQ tokens##으로 12.5 Hz, 2.14 kbps의 저비트레이트 표현을 달성. ASR distillation으로 semantic token이 텍스트와 강하게 정렬되며, Mimi(Moshi의 codec) 대비 모든 재구성 메트릭에서 우수.
  2. Hybrid AR + Flow-Matching architecture: Semantic token은 decoder-only transformer로 AR 생성하고, acoustic token은 ##lightweight 3-layer flow-matching transformer##로 생성. 기존 36-depth AR (Moshi) 대비 8 NFE로 충분하여 효율적이면서 human eval에서 더 표현력 있음.
  3. DPO for hybrid discrete-continuous TTS: ##Standard DPO (semantic) + flow-based preference objective (acoustic)##를 결합한 최초의 시도. WER, UTMOS를 동시에 개선하고 hallucination과 volume tapering을 감소.
  4. Production-ready serving: vLLM-Omni 기반 2-stage pipeline + CUDA graph acceleration + asynchronous chunked streaming. 단일 H200에서 ##RTF 0.103 (concurrency 1), 1,430 char/s/GPU (concurrency 32)##.
  5. 9개 언어 지원, 3초 reference로 voice cloning: Human evaluation에서 ##ElevenLabs Flash v2.5 대비 68.4% win rate## (voice cloning), flagship voices에서 58.3% win rate. CC BY-NC 라이센스로 weights 공개.

2. Modeling

2.1 Voxtral Codec (~300M params)

24 kHz mono waveform을 12.5 Hz 프레임의 37개 discrete tokens (1 semantic + 36 acoustic)으로 압축하는 convolutional-transformer autoencoder이다.

Waveform Autoencoder

"Patchified" waveform 방식: 24 kHz input을 240 sample 단위로 non-overlapping patch로 분할하여 100 Hz input을 생성.

Encoder: Causal convolution (kernel 7) → 1024-dim projection → 4 encoder blocks. 각 block:

  • 2-layer causal self-attention transformer (sliding window attention, ALiBi positional bias, QK-norm, LayerScale $\epsilon=0.01$). Window sizes: $16 \rightarrow 8 \rightarrow 4 \rightarrow 2$
  • Causal CNN layer (stride 2로 downsample, 첫 3 block). 4번째 block은 stride 1 + 1024→292 dim projection
  • 총 8× downsampling: 100 Hz → 12.5 Hz, 최종 292-dim latent

Decoder: Encoder의 역구조. 292-dim → 1024-dim causal CNN → 4 blocks (transposed CNN 2× upsample + 2-layer causal self-attention transformer) → causal convolution (kernel 7) → 240 samples로 unpatch.

Representation Quantization — Hybrid VQ + FSQ

292-dim latent을 두 component로 분리:

  • Semantic component (256-dim): Learned VQ (Vector Quantization) with codebook size 8192. Training 시 50% 확률로 VQ 적용, 나머지 50%는 unquantized pass-through
  • Acoustic component (36-dim): 각 dimension을 tanh → 21 uniform levels로 FSQ (Finite Scalar Quantization). Training 시 dither-style: 50% FSQ, 25% uniform noise ($1/L$, $L=21$), 25% unquantized

총 bitrate: $12.5 \times (\log_2 8192 + 36 \times \log_2 21) \approx 2.14$ kbps.

Semantic Token Learning — ASR Distillation

기존 방법은 self-supervised representations (HuBERT 등)을 distill하지만 이는 "phonetic"에 가까운 토큰을 생성한다. Voxtral Codec은 ##supervised ASR model (frozen Whisper)에서 distill##하여 더 effective semantic representation을 생성한다.

Whisper의 마지막 decoder layer hidden states $h_l$에 대해, post-VQ semantic embedding $z_f$를 linearly project하여 alignment. Whisper의 cross-attention weights에서 soft alignment matrix $A \in \mathbb{R}^{L \times F}$를 추출 (word-level timestamp와 가장 잘 correlation하는 attention head subset을 DTW로 식별):

$$\mathcal{L}_{\text{ASR}} = 1 - \frac{1}{L} \sum_{l=1}^{L} \frac{\tilde{z}_l \cdot h_l}{\|\tilde{z}_l\| \|h_l\|}, \quad \tilde{z}_l = \sum_{f=1}^{F} A_{l,f} \, z_f \quad (1)$$

$\tilde{z}_l$은 codec frame $f$의 semantic embedding의 attention-weighted sum으로, Whisper의 $l$번째 decoder token에 정렬된 codec representation이다. 이 설계로 ##외부 forced aligner나 paired transcript 없이도 text-aligned semantic token을 학습##할 수 있다.

Adversarial Training

Multi-resolution discriminator: 8 STFT sizes (2296, 1418, 876, 542, 334, 206, 126, 76). 각 discriminator는 real/reconstructed audio를 hinge loss로 분류. $L_1$-based feature-matching loss를 standard GAN generator loss 대신 사용 (evolving discriminator features가 더 discriminative한 signal 제공):

$$\mathcal{L}_{\text{feature}}(\boldsymbol{x}, \hat{\boldsymbol{x}}) = \frac{1}{MN} \sum_{m=1}^{M} \sum_{n=1}^{N} \|D_n^m(\boldsymbol{x}) - D_n^m(\hat{\boldsymbol{x}})\|_1 \quad (2)$$

Training Objective

$$\alpha \mathcal{L}_{\text{feature}} + \beta \mathcal{L}_{\text{ASR}} + \gamma \mathcal{L}_{\text{L1}} + \gamma \mathcal{L}_{\text{STFT}} + \delta \mathcal{L}_{\text{commit}} \quad (3)$$

$\alpha=1.0$, $\beta=1.0$, $\gamma=0.9999^t$ (exponential decay), $\delta=0.1$. Reconstruction losses ($\mathcal{L}_{\text{L1}}$: waveform L1, $\mathcal{L}_{\text{STFT}}$: STFT magnitude L1)는 exponential decay로 학습 초기에 bootstrap하고, 후반에는 adversarial signal이 지배하도록 한다.

[Figure 3 설명]: Voxtral Codec 아키텍처. 2.2 Decoder Backbone (Ministral 3B)

Ministral 3B 아키텍처의 autoregressive decoder-only transformer. 입력 시퀀스: [voice reference audio tokens] + [text prompt tokens] → 출력: [semantic audio tokens] + <EOA>.

  • 각 audio frame은 37개 discrete tokens (1 semantic + 36 acoustic)로 표현
  • 각 codebook은 독립적인 embedding lookup table을 가지며, 프레임별로 합산하여 단일 embedding 생성
  • Linear head가 hidden state $h$를 semantic codebook vocabulary (8192 + <EOA>)에 대한 logits로 투영, standard cross-entropy loss로 학습
  • Acoustic token 예측은 flow-matching transformer에 위임 (Section 2.3)
  • FM transformer의 float-valued output은 21 FSQ levels로 discretize되어 다음 AR step의 discrete token input으로 사용 → ##fully discrete token interface 유지##

2.3 Flow-Matching Transformer

각 AR generation step에서, decoder backbone의 hidden state $h$를 조건으로 acoustic token (36-dim continuous vector)을 생성하는 bidirectional 3-layer transformer.

Flow-matching formulation: Gaussian noise $x_0$에서 acoustic embedding $x_1$으로의 velocity field를 모델링. $0 \leq t \leq 1$의 function evaluation steps에서:

  • Input: $h$ (hidden state from backbone), $t$ (current step, sinusoidal embedding), $x_t$ (current acoustic state, $x_t \in \mathbb{R}^{36}$)
  • 각 입력은 별도 projection layer (activation scale이 다르므로)
  • DiT-style AdaLN보다 separate projection이 우수 (ablation 확인)

Training: Hidden state를 10% dropout하여 "unconditional" modeling → inference 시 CFG (Classifier-Free Guidance) 적용:

$$v_t = \alpha v_\theta(x_t, t, h) + (1-\alpha) v_\theta(x_t, t, \emptyset) \quad (4)$$ $$x_{t-\Delta t} = x_t - v_t \cdot \Delta t \quad (5)$$

$\Delta t = 1/8$, $\alpha = 1.2$. ##CFG가 FM transformer에만 적용되므로, decoder backbone 전체에 CFG를 적용하는 것보다 계산량이 현저히 적다##. 8 NFE (Number of Function Evaluations)가 기본, 16 NFE에서 marginal improvement만 있으므로 8을 채택.

대안 아키텍처와의 비교:

  • MaskGIT: 36 acoustic codebook position + conditioning = per-frame sequence length 38 (vs FM의 3). Human eval에서 expressiveness 열세
  • Depth Transformer (Moshi 방식): 36 autoregressive decoding steps/frame (vs FM의 8 NFE). Quality, compute, latency 모두 열세

3. Training

3.1 Pretraining

Voxtral Mini Transcribe로 pseudo-labeled된 paired audio + transcript 사용. 각 학습 샘플: $(A_1, T_2, A_2)$ — $A_1$은 voice reference, $T_2$는 transcript, $A_2$는 생성 target.

  • $A_1$, $A_2$: 동일 화자의 single-speaker segments, 반드시 temporally adjacent는 아님
  • Maximum duration: $A_1, A_2$ 각각 180초, $A_1$ 최소 1초
  • Voice prompt 최적 길이: 3-25초 (장시간 대화의 long-tail 특성 고려)
  • <next> special token으로 $A_1$과 $T_2$ 사이를, <repeat> token으로 $T_2$와 $A_2$ 사이를 구분

Loss: $A_2$의 tokens에만 loss 계산. Two-part loss:

  1. Semantic loss ($\mathcal{L}_{\text{semantic}}$): Semantic token에 대한 cross-entropy
  2. Acoustic loss ($\mathcal{L}_{\text{acoustic}}$): Conditional flow-matching objective
$$\mathcal{L}_{\text{acoustic}} = \mathbb{E}_{t \sim \mathcal{U}[0,1], x_0 \sim \mathcal{D}, x_1 \sim \mathcal{N}(0,1)} \|v_\theta(x_t, t) - u_t(x_t | x_1, x_0)\|_2^2 \quad (6)$$ $$u_t(x_t | x_1, x_0) = x_1 - x_0 \quad (7)$$

Implementation details:

  • Backbone: Ministral 3B에서 초기화. FM transformer, audio codebook embedding lookup-tables, output projection은 random initialize
  • Text-embedding layers는 freeze (Voxtral Mini Transcribe의 low-frequency text token에 대한 robustness)
  • VAD로 speech 없는 프레임에 lower loss weight, extremely long silence에 loss weight 0
  • LLM-based transcript rewrite로 normalized vs un-normalized text robustness 확보 (예: "5 - 4" vs "five minus four")

3.2 Direct Preference Optimization (DPO)

WER, speaker similarity 개선을 위한 post-training. ##Hybrid discrete-continuous 설정에 DPO를 적용하는 최초의 시도##:

Semantic DPO

Standard DPO objective with uniform weights:

$$\mathcal{L}(\theta) = -\mathbb{E}_{t \sim \mathcal{U}(0,1), x^w, x^l} \log \sigma \left(-\beta \left(\Delta_\theta(x^w, x^l, t) - \Delta_{\theta_{\text{ref}}}(x^w, x^l, t)\right)\right) \quad (8)$$

Flow-based Acoustic DPO

Acoustic token이 flow-matching으로 생성되므로, flow-matching objective를 preference objective로 변환 (Ziv et al., 2025 기반):

$$\Delta_\theta(x^w, x^l, t) = \|v_\theta(x_t^w, t) - u_t(x_t^w | x^w)\|_2^2 - \|v_\theta(x_t^l, t) - u_t(x_t^l | x^l)\|_2^2 \quad (9)$$

Autoregressive 설정에 맞게, 각 토큰이 다른 $\mathbf{t}$를 샘플링하도록 수정:

$$\Delta_\theta(x^w, x^l, \mathbf{t}^l) = \sum_{i=1}^{N_w} \|v_\theta(x_{i,\mathbf{t}_i}^w, \mathbf{t}_i) - u_{i,\mathbf{t}_i}^w\|_2^2 - \sum_{i=1}^{N_l} \|v_\theta(x_{i,\mathbf{t}_i}^l, \mathbf{t}_i) - u_{i,\mathbf{t}_i}^l\|_2^2 \quad (10)$$

Length normalization (winner 길이로 나누기)은 instability를 유발하여 사용하지 않음. $\beta_{\text{semantic}} = 0.1$, $\beta_{\text{acoustic}} = 0.5$, LR $8 \times 10^{-8}$.

DPO 데이터 생성: Rejection-sampling pipeline — single-speaker voice samples + 합성 text prompts (Mistral Small Creative로 생성) → pretrained checkpoint로 multiple samples 생성 → WER, speaker similarity, loudness consistency, UTMOS-v2, LM judge로 winner/loser pair 구성. Pretraining objective와 DPO loss를 결합하여 high-quality speech에서 1 epoch 학습. Synthetic data로 장시간 학습하면 robotic speech가 되므로 1 epoch이 최적.

5.2 Inference Parameters (Figure 4)

NFE (Number of Function Evaluations): 2→4→8→16에서 WER, UTMOS, Speaker Similarity 모두 개선. 8 이후 marginal improvement. 따라서 ##NFE=8##을 기본값으로 채택.

CFG scale $\alpha$: 1.0→1.4에서 WER, UTMOS, Speaker Similarity가 거의 단조 증가하지만, human evaluation에서 높은 $\alpha$는 voice prompt에 과도하게 adherence하고 텍스트에 내재된 감정을 반영하지 못하는 문제 발생. ##$\alpha = 1.2$##을 기본값으로 채택 (고품질 reference에는 낮은 값, in-the-wild recording에는 높은 값이 유리).

6. Inference and Serving in vLLM-Omni

Voxtral TTS는 vLLM-Omni (multi-stage multimodal model을 위한 vLLM 확장)으로 서빙된다. Two-stage pipeline:

  1. Generation stage: Audio token (semantic + acoustic) 예측
  2. Codec decoding stage: Token → waveform 변환

두 stage는 asynchronous chunked streaming protocol로 통신: generation step마다 per-request buffer에 token 저장 → buffer가 pre-defined length에 도달하면 codec decoding stage에 emit. 각 chunk에 이전 프레임의 slice를 포함하여 codec decoder의 causal sliding-window attention이 temporal coherence를 유지.

6.1 CUDA Graph Acceleration for Flow-Matching Transformer

FM transformer가 computational bottleneck: 각 decoding step에 $2 \times N$ forward passes (CFG 포함). Python-level overhead와 kernel-launch latency를 제거하기 위해 ##전체 ODE solver를 CUDA graph로 캡처##. Bucket size별로 warmup에서 eager pass + graph capture → inference 시 bucket 크기로 pad/slice.

비판적 분석 (Critical Analysis)

강점

  • 아키텍처 설계의 우아함: Semantic (AR) + Acoustic (FM) 분리가 직관적이고 효과적. 36-depth AR을 8 NFE FM으로 대체한 것은 quality-efficiency trade-off에서 최적점을 찾은 설계.
  • ASR distillation의 novelty: Self-supervised representation 대신 supervised ASR에서 distill하여 더 semantically rich한 토큰을 생성하고, Whisper의 cross-attention으로 soft alignment를 자동 추출하는 방식이 우아.
  • DPO for hybrid discrete-continuous systems: Flow-matching objective를 DPO로 확장한 것은 TTS 분야에서 새로운 기여이며, WER/UTMOS 동시 개선을 실증.
  • Production 수준의 serving 분석: CUDA graph acceleration, async chunked streaming, concurrency scaling까지 포함하여 학술 논문보다 산업적 가치가 높다.
  • 오픈 공개: CC BY-NC 라이센스로 weights 공개. TTS 분야에서 이 수준의 오픈 모델은 드물다.

약점

  • 학습 데이터 비공개: "Paired audio and transcripts pseudo-labelled with Voxtral Mini Transcribe"로만 기술되어, 데이터 규모, 도메인, 언어별 비율 등이 완전히 비공개. 재현 불가.
  • Hindi에서 DPO regression: WER 3.39→4.99로 악화. Low-resource 언어에서의 DPO 안정성이 보장되지 않으며, 원인 분석이 부족.
  • Human evaluation의 제한된 규모: 77 prompts, 3 annotators per language pair. 통계적 유의성 검정(p-value 등)이 보고되지 않음.
  • Emotion steering 방식의 한계: Voxtral TTS는 explicit emotion tags/instructions를 지원하지 않고, voice prompt의 감정으로만 steering. Explicit control이 필요한 시나리오에서 제한적.
  • Codec 학습 데이터 미기술: Voxtral Codec의 학습 데이터 규모와 도메인이 전혀 기술되지 않음.
  • Competitive baseline 선정: ElevenLabs와만 주로 비교하고, Gemini 2.5 Flash TTS는 flagship evaluation에서만 포함. OpenAI TTS, XTTS, MetaVoice 등 다른 오픈/상업 모델과의 비교 부족.

Key Takeaways

  1. Semantic/Acoustic 분리 + AR/FM 하이브리드가 TTS의 새 패러다임: Semantic token을 AR로, acoustic token을 flow-matching으로 생성하는 설계가 depth-wise AR (Moshi) 대비 quality와 efficiency를 동시에 개선한다. ##핵심 insight: 음성의 "무엇을 말하는가"(semantic)는 AR로, "어떻게 말하는가"(acoustic)는 continuous modeling으로 분리 처리해야 한다##.
  2. ASR distillation이 self-supervised distillation보다 effective: Whisper hidden states에서 distill한 semantic token이 HuBERT/wav2vec2 기반보다 text-aligned하여 intelligibility가 높다. Cross-attention soft alignment가 forced aligner를 대체한다.
  3. VQ-FSQ hybrid quantization이 RVQ보다 efficient: 1 VQ (semantic) + 36 FSQ (acoustic) = 2.14 kbps로 Mimi 16-codebook RVQ (2.2 kbps)를 모든 메트릭에서 상회. FSQ의 단순성(learned codebook 없음)이 학습 안정성과 품질 모두에 기여.
  4. DPO가 TTS에서도 효과적: Flow-matching objective를 preference objective로 변환하는 방법이 실용적이며, hallucination 감소와 UTMOS 개선을 동시에 달성. 다만 low-resource 언어에서의 안정성이 과제.
  5. CUDA graph acceleration이 FM transformer의 병목을 해결: 47% latency 감소, 2.5× RTF 개선. Multi-step ODE solver의 Python overhead를 graph capture로 제거하는 패턴은 다른 diffusion/flow 기반 모델에도 적용 가능.
