TL;DR

  • 문제: LLM post-training에서 RL을 다양한 도메인(수학, 코드, agentic, alignment 등)으로 확장할 때, 도메인 간 간섭(catastrophic forgetting), reward signal 다양성, 학습 불안정성 등이 발생한다. 기존의 단일 multi-domain RL은 이러한 문제를 해결하기 어렵다.
  • 해결: Cascade RL — 도메인별 순차 RL 파이프라인(IF-RL → Multi-domain RL → MOPD → RLHF → Long-context RL → Code RL → SWE RL)을 적용하고, Multi-domain On-Policy Distillation (MOPD)으로 각 단계에서 최강 teacher checkpoint를 선택해 on-policy 증류하여 성능 회복 및 안정화를 달성한다.
  • 왜 작동하는가: Cascade RL은 도메인별 하이퍼파라미터 최적화와 도메인 간 간섭 최소화를 가능하게 하고, MOPD는 dense token-level distillation으로 sparse outcome reward 대비 빠르게 수렴하며 RL 단계 간 regression을 효과적으로 복구한다.
  • 비유: 학생이 여러 과목을 동시에 공부하면 혼란스러우니, 각 과목을 순차적으로 집중 학습(Cascade RL)하되, 중간중간 전 과목 최고 성적의 선생님에게 개인 과외(MOPD)를 받아 종합 성적을 끌어올리는 것과 같다.

핵심 기여 (Key Contributions)

  1. Cascade RL의 확장: Nemotron-Cascade 1 대비 RL 도메인을 크게 확장하여 IF-RL, Multi-domain RL, MOPD, RLHF, Long-context RL, Code RL, SWE RL 7단계 파이프라인을 구축하고, 도메인 간 간섭을 최소화하는 순서 결정 원칙을 제시한다.
  2. Multi-domain On-Policy Distillation (MOPD): ##Cascade RL 파이프라인의 각 단계에서 최강 intermediate teacher checkpoint를 선택하여 on-policy 증류##함으로써 벤치마크 regression을 효과적으로 복구하고, GRPO 대비 훨씬 빠른 수렴을 달성한다.
  3. Gold Medal급 수학/코딩 성능: ##30B MoE (3B activated) 모델로 IMO 2025 Gold (35/42), IOI 2025 Gold (439.28/600), ICPC World Finals 2025 Gold (10/12)##을 동시 달성한 두 번째 오픈 모델이다 (첫 번째는 DeepSeek-V3.2-Speciale-671B).
  4. 완전 오픈소스: 모델 체크포인트, SFT 데이터, RL 데이터를 모두 공개하여 연구 커뮤니티의 재현과 확장을 가능하게 한다.

1. Introduction — 배경과 동기

Cascade RL이란?

Reinforcement Learning (RL)은 LLM post-training의 핵심 기법으로, reasoning, agentic 능력, alignment 등을 향상시킨다. 그러나 RL을 다양한 도메인으로 확장하면 reward signal, 응답 형식, 검증 환경이 모두 달라 학습 복잡도가 급증한다.

Nemotron-Cascade 1 (Wang et al., 2025)에서 처음 도입된 Cascade RL은 도메인별로 순차적 RL 학습을 수행하는 프레임워크이다. 핵심 장점 세 가지:

  1. Catastrophic forgetting에 강건: 도메인별 RL은 이전 도메인 성능을 거의 저하시키지 않으며, 오히려 개선할 수 있다.
  2. 도메인별 최적화 가능: 각 도메인에 맞는 하이퍼파라미터, reward function, training curriculum을 독립적으로 설계할 수 있다.
  3. 계산 효율성: 단일 도메인 내 task 동질성이 높아 응답 길이와 검증 시간이 균일하므로, wall-clock 시간이 크게 절약된다.

Nemotron-Cascade 2의 확장점

Nemotron-Cascade 2는 이전 버전 대비 세 가지 핵심 차별점을 갖는다:

  1. Cascade RL 도메인 확장: reasoning과 agentic 영역을 훨씬 넓게 커버하도록 RL 단계를 확장
  2. Multi-domain On-Policy Distillation (MOPD): Cascade RL 단계에서 발생하는 벤치마크 regression을 복구하기 위해, 각 도메인 최강 teacher로부터 on-policy 증류를 수행하는 새로운 안정화 단계 도입
  3. Multi-domain RL 통합: 유사한 응답 형식과 검증 비용을 가진 도메인은 하나의 RL 단계에서 jointly 학습하여 효율성 향상

3. Supervised Fine-Tuning (SFT)

SFT는 post-training 파이프라인의 첫 번째 단계로, 모델에 reasoning, 대화, instruction following, agentic 능력의 기초를 부여한다.

3.1 Training Framework

  • 시퀀스 길이: 최대 256K tokens로 packing
  • 학습: 약 1.5 epochs에서 최적 성능 도달 (33,000 steps)
  • Optimizer: AdamW ($\beta_1=0.9, \beta_2=0.98$), weight decay 0.1, cosine scheduler
  • LR: max $5 \times 10^{-5}$, min $5 \times 10^{-6}$, 200 warmup steps

Chat Template 변경

Nemotron-Cascade 1과의 두 가지 변경: (1) /think, /no_think 태그 제거, (2) 비추론 모드 활성화를 위해 빈 <think></think> 블록 사용. Tool calling은 시스템 프롬프트 내 <tools> 태그에 도구 정의, <tool_call> 태그로 호출.

전체 학습 파이프라인

Base Model → SFT → IF-RL → Multi-domain RL → MOPD → RLHF → Long-context RL → Code RL → SWE RL → Nemotron-Cascade 2
[Figure 2 설명]: 학습 파이프라인의 전체 흐름. SFT 이후 두 갈래로 나뉘어 진행되는 것처럼 보이지만, 실제로는 순차적이다. 시각적으로 상하 2행에 배치되어 있지만, 실제로는 SFT → IF-RL → Multi-domain RL → MOPD → RLHF → Long-context RL → Code RL → SWE RL의 단일 순차 파이프라인이다.

4.1 Training Framework

4.1.1 Cascade RL 순서 결정 원칙

최적의 RL 단계 순서는 모델의 행동 역학에 따라 달라지며, 세 가지 원칙을 따른다:

  1. Inter-domain interference 최소화: 도메인 간 경쟁/충돌(예: IF-RL의 strict instruction adherence vs RLHF의 human preference alignment)을 관찰하고, foundational prior 역할을 하는 태스크를 먼저, specialized refinement를 나중에 배치한다.
  2. Multi-domain integration으로 확장: 성능 충돌이 없는 도메인은 하나의 RL 단계에서 jointly 학습하여 효율성을 높인다.
  3. MOPD로 안정화: MOPD가 critical stabilization point 역할을 하여 이전 단계의 regression을 복구한다.

4.1.2 RL Training Configuration

전체 Cascade RL에서 GRPO (Group Relative Policy Optimization) 알고리즘을 사용하며, strict on-policy training (importance sampling ratio = 1)을 채택한다. KL divergence term을 제거하여 표준 REINFORCE objective로 단순화한다:

$$\mathcal{J}_{\text{GRPO}}(\theta) = \mathbb{E}_{(q,a)\sim\mathcal{D},\,\{o_i\}_{i=1}^G \sim \pi_\theta(\cdot|q)} \left[ \frac{1}{\sum_{i=1}^G |o_i|} \sum_{i=1}^G \sum_{t=1}^{|o_i|} \hat{A}_{i,t} \right], \quad \text{where } \hat{A}_{i,t} = \frac{r_i - \text{mean}(\{r_i\}_{i=1}^G)}{\text{std}(\{r_i\}_{i=1}^G)} \text{ for all } t \quad (1)$$

직관적으로, 동일 질문에 대한 $G$개 rollout의 reward를 group-normalize하여, 평균 이상 응답은 강화하고 이하는 약화하는 방식이다. 각 iteration마다 현재 policy에서 rollout을 생성하고 single gradient update를 수행한다.

4.2 Instruction-Following RL (IF-RL)

실험 의도: Cascade RL의 첫 단계로 instruction adherence를 확립하여, 후속 단계의 기초를 마련한다.

왜 첫 번째 단계인가: (1) IF-RL이 human alignment(ArenaHard)에 부정적 영향을 미칠 수 있으나, 후속 RLHF가 이를 복구 가능. (2) IF-RL 이후의 모델이 MOPD의 강한 teacher가 됨.

핵심 설계 결정:

  • Thinking mode에서만 학습 (reward model 없이)
  • Dynamic filtering: 모든 rollout이 전부 맞거나 전부 틀린 프롬프트는 제거하여 유효 gradient 보장
  • Overlong penalty: 최대 길이 초과 시 reward = 0
  • 결과: ##IFBench에서 SOTA 83.13%## 달성

하이퍼파라미터: batch 128, 16 rollouts/prompt, LR $2 \times 10^{-6}$, AdamW, ~180 steps

4.3 Multi-domain RL

실험 의도: IF-RL 이후, MCQA (STEM), agentic tool calling, structured output instruction following 세 도메인을 jointly 학습한다.

왜 jointly 학습하는가: (1) 세 도메인 간 성능 충돌이 관찰되지 않으며, 오히려 MMLU-Pro, τ²-Bench, IF-Bench 모두에서 일관된 개선을 보인다. (2) 응답 길이와 검증 시간이 유사하여 학습 효율이 높다.

데이터 믹스: 55% MCQA, 30% agentic tool calling (Workplace Assistant), 15% structured output.

하이퍼파라미터: batch 128, 16 rollouts, LR $3 \times 10^{-6}$, Adam, ~70 steps

4.4 Multi-domain On-Policy Distillation (MOPD)

실험 의도: Cascade RL의 여러 단계를 거치면서 발생하는 벤치마크 fluctuation과 capability drift를 복구하여, 균형 잡힌 최종 모델을 만든다.

왜 MOPD가 필요한가: 잘 설계된 Cascade RL도 도메인 수가 증가하면 완전히 drift를 제거하지 못한다. RLVR 학습은 model entropy를 낮추고 reasoning trace를 짧게 만들어 수학 성능에 악영향을 줄 수 있고, RLHF는 instruction-following behavior와 trade-off가 발생할 수 있다.

MOPD 작동 방식

핵심 아이디어: Cascade RL 파이프라인의 각 도메인별 최강 validation checkpoint를 teacher로 선택하여, student에게 on-policy 증류한다.

Teacher 선택: 기본 설정에서 3개 domain teacher를 사용한다:

  • Math teacher: SFT checkpoint (수학 reasoning trace가 가장 긴 상태)
  • RLHF teacher: RLHF 최적화 checkpoint
  • RLVR teacher: IF-RL + Multi-domain RL 초기 단계 checkpoint

Student policy $\pi^{inf}$에서 응답 $y = (y_1, ..., y_T)$를 샘플링하고, 프롬프트의 도메인에 해당하는 teacher $\pi^{domain_t}$를 선택한다. Token-level distillation advantage를 reverse-KL로 정의한다:

$$a_t^{MOPD} = \log \pi^{domain_t}(y_t \mid s_t) - \log \pi^{train}(y_t \mid s_t) \quad (2)$$

직관적으로, teacher가 현재 student보다 해당 토큰에 더 높은 확률을 부여할 때 양수가 되어, 그 토큰을 강화하라는 dense signal이 된다. 학습이 진행될수록 이 advantage는 0으로 수렴한다.

Train-infer mismatch를 보정하기 위해 truncated importance weighting을 적용한다:

$$r_t = \frac{\pi^{train}(y_t \mid s_t)}{\pi^{inf}(y_t \mid s_t)}, \quad w_t = \text{sg}[r_t] \, \mathbb{1}[\epsilon_{\text{low}} \leq r_t \leq \epsilon_{\text{high}}] \quad (3)$$

최종 surrogate objective:

$$\mathcal{L}_{MOPD} = -\mathbb{E}_{x \sim D, y \sim \pi^{inf}(\cdot|x)} \left[ \frac{1}{|\mathcal{V}(y)|} \sum_{t \in \mathcal{V}(y)} w_t \, \text{sg}[a_t^{MOPD}] \log \pi^{train}(y_t \mid s_t) \right] \quad (4)$$

여기서 $\mathcal{V}(y)$는 token mask에 의해 유효한 응답 토큰의 집합이다.

MOPD의 학습 효율성 이점

##MOPD는 dense token-level signal을 제공하므로 GRPO의 sparse sequence-level reward 대비 훨씬 빠르게 수렴##한다:

  • AIME25: GRPO는 25 steps 후 89.9→91.0, MOPD는 30 steps 만에 92.0 달성 (teacher 수준 회복)
  • ArenaHard v2: MOPD는 52 steps로 Hard Prompt 71.5→85.5 + Creative Writing 40.6→71.0 동시 달성. RLHF는 160 steps로 80.7 + 71.2 (Table 3)
[Figure 3 설명]: (a) Reverse KL: 학습 초기 급감 후 수렴, student가 teacher를 빠르게 모방함을 보여준다. (b) grad_norm: 초기에 매우 크다가 warmup 후 급감하여, warmup이 안정적 학습에 필수적임을 시사한다. (c) AIME25 (avg@64): MOPD(파란 실선)가 GRPO(주황 실선)보다 적은 step으로 teacher 수준(점선)에 도달한다.

하이퍼파라미터: rollout size 4, batch 128 prompts (effective batch 512), max response 98K tokens. 본문에서는 LR $2 \times 10^{-6}$ (warmup $2 \times 10^{-7}$, 30 steps), 40-50 steps 수렴으로 기술하고, Table 8에서는 LR $3 \times 10^{-6}$, 52 steps로 기재. $\epsilon_{\text{low}} = 0.5$, $\epsilon_{\text{high}} = 2.0$.

4.5 RLHF (Reinforcement Learning from Human Feedback)

실험 의도: Creative writing, 비검증 가능 problem-solving의 품질 향상과 human preference alignment.

핵심 설계:

  • GenRM (Generative Reward Model): Qwen3-235B-A22B-Thinking-2507 기반, HelpSteer3 프레임워크
  • 데이터: HelpSteer3, arena-human-preference-140k subset, synthetic safety blend
  • Thinking mode에서만 학습 (non-thinking 포함 시 IF 성능 저하 관찰)
  • Length-normalized reward + quality-gated conciseness bonus로 응답 길이 증가 억제

하이퍼파라미터: batch 128, 16 rollouts, LR $3 \times 10^{-6}$, max response 16K, KL coefficient 0.03, ~25-30 steps

4.6 Long-context RL

실험 의도: Long-context understanding과 reasoning 강화. 다른 도메인을 포함하면 성능 저하가 관찰되어 domain-specific으로 분리.

설정: NVIDIA Nano-v3 RL data blend (long-context만), Nemo-Gym 환경, Qwen3-235B-A22B-Instruct-2507 LLM judge, input 32K / output 49K tokens, ~30 steps

4.7 Code RL

실험 의도: 경쟁 프로그래밍 수준의 코딩 능력 강화.

핵심 설계:

  • Data curation: GPT-OSS-120B가 8/8 rollout 모두 정답인 쉬운 문제를 제거하여 ##최종 3.5K 고난도 프롬프트만 사용##
  • Max response 118K tokens: 극도로 어려운 문제의 긴 reasoning trace 포착
  • Strict binary reward (reward hacking 방지)
  • 비동기 reward verification 서버: $128 \times 16 = 2,048$ code executions/step, 384 CPU cores, batch당 427.2초

하이퍼파라미터: batch 128, 16 rollouts, LR $3 \times 10^{-6}$, top-p 0.95, ~22 steps

4.8 SWE RL (Software Engineering RL)

4.8.1 Agentless RL

Code repair 능력 강화를 위한 agentless 접근. GPT-OSS-120B를 reward model로, golden localization + top-5 retrieved localization으로 프롬프트 구성. reward > 0.5인 rollout이 하나도 없는 어려운 프롬프트는 loss를 mask 처리한다.

4.8.2 Execution-based Agentic SWE RL

RLVR (Reinforcement Learning from Verifiable Rewards)를 agentic SWE scaffold 내에서 직접 적용. OpenHands 프레임워크를 사용하여 agent가 repository interaction → patch 생성 → test execution의 전체 trajectory를 최적화한다.

핵심 설정:

  • Max context 256K tokens, 최대 200 interaction turns
  • 데이터: SWE-Gym + R2E-Subset
  • 100% accuracy (너무 쉬운) 문제와 0% accuracy (너무 어려운, 90% 제거) 문제를 필터링
  • Batch 1024 (16 prompts × 64 rollouts)

비판적 분석 (Critical Analysis)

강점

  • 탁월한 파라미터 효율성: 3B activated parameters로 IMO/IOI/ICPC Gold Medal을 동시 달성한 것은 intelligence density 관점에서 획기적이다. 20배 작은 모델이 671B 모델과 경쟁하는 것은 Cascade RL + MOPD의 효과를 강력히 입증한다.
  • 체계적인 ablation: MOPD vs GRPO 비교 (Table 3, Figure 3c), Agentless RL의 agentic 전이 효과 (Table 4), IF-RL positioning 근거 등 각 설계 결정에 대한 실증적 근거가 풍부하다.
  • 완전 오픈소스: 모델, SFT 데이터, RL 데이터 모두 공개하여 재현성이 높다.
  • 포괄적 평가: Math, Code, Knowledge, Alignment, Long-context, Agentic, Multilingual 7개 범주에 걸쳐 30+ 벤치마크에서 평가.

약점

  • Knowledge-intensive 및 agentic 태스크 약세: MMLU-Pro 79.8 (vs Qwen3.5 85.3), GPQA-Diamond 76.1 (vs 84.2), BFCL v4 52.9 (vs 67.3), SWE Verified 50.2 (vs 69.2)로 상당한 격차. 저자도 "stronger knowledge-intensive pretraining and agentic RL"이 필요하다고 인정하지만, 이 약점이 전체 실용성에 미치는 영향 분석이 부족하다.
  • Cascade RL 순서의 최적성 미증명: 7단계 파이프라인의 순서가 "Rule of thumb"과 실험적 관찰에 기반하지만, 체계적인 순서 탐색(예: 순열 비교)은 보고되지 않았다. 다른 순서가 더 나을 수 있다는 가능성이 열려 있다.
  • 표준편차/반복 실험 미보고: IMO, IOI 등 경시대회 결과는 단일 시도이며, AIME 등 벤치마크도 대부분 단일 점추정치로 보고된다. Seed에 따른 변동성이 불분명하다.
  • Compute cost 미공개: 7단계 Cascade RL 파이프라인의 총 GPU 시간, Code RL의 384 CPU core 검증 서버 비용 등이 보고되지 않아 실용적 재현 가능성 판단이 어렵다.
  • IMO 채점 방법의 한계: P2에서 LLM grader를 사용한 것은 인간 채점과의 차이를 야기할 수 있으며, 전체 35점의 신뢰성에 의문이 남는다.
  • SFT 데이터의 proprietary teacher 의존: GPT-OSS-120B, DeepSeek-V3.2-Speciale 등 대형 모델에서 생성된 합성 데이터에 크게 의존. 이러한 모델 없이 파이프라인을 재현하기 어렵다.

Key Takeaways

  1. Cascade RL은 multi-domain RL의 실용적 해법: 도메인별 순차 학습이 joint 학습 대비 도메인 간 간섭을 크게 줄이고, 하이퍼파라미터 튜닝의 자유도를 높인다. 도메인 수가 증가할수록 이 장점이 더 두드러진다.
  2. MOPD는 Cascade RL의 핵심 안정화 기법: Dense token-level distillation이 sparse outcome reward보다 빠르게 수렴하며, RL 단계 간 regression을 효과적으로 복구한다. ##MOPD 52 steps = RLHF 160 steps## 수준의 효율성.
  3. Intelligence density의 새로운 기준: 3B activated parameters로 수학 올림피아드 Gold Medal을 달성할 수 있음을 보여주며, 이는 post-training의 중요성이 model size를 넘어설 수 있음을 시사한다.
  4. 고난도 데이터의 가치: Code RL에서 "쉬운 문제 제거"(GPT-OSS-120B가 8/8 정답인 문제 필터링)가 3.5K 프롬프트만으로도 강력한 코딩 성능을 이끌어냄. 데이터 양보다 난이도가 중요하다.
  5. Execution-based agentic RL의 가능성: SWE scaffold 내에서 직접 RL을 수행하여 전체 agent trajectory를 최적화하는 접근이 실용적임을 보여준다. 컴파일 결과와 테스트 통과 여부가 자연스러운 verifiable reward가 된다.
