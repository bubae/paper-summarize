TL;DR

  • 문제: 시간 단위(hour-level) 긴 영상을 이해하려면 256K context가 필요한데, 표준 dense attention은 quadratic 비용으로 인해 KV 캐시가 폭발한다. 또 multi-task 학습(긴 영상 이해, agentic tool use, math/STEM, OCR 등)을 한 모델에 동시에 주입하면 catastrophic forgetting이 발생해 base 모델의 reasoning 능력이 무너진다.
  • 해결: 아키텍처 측면은 ##GQA backbone에 DeepSeek Sparse Attention(DSA)을 결합한 멀티모달 first-of-its-kind 적용##으로 256K lossless context 처리. 학습 측면은 ##Cross-Modal Multi-Teacher On-Policy Distillation (MOPD)## — 13개 도메인 specialized teacher가 학생의 on-policy rollout에 token-level feedback을 주어 능력 충돌을 해소. Context-RL과 Video-RL(bucket advantage scaling)도 추가.
  • 왜 작동하는가: (i) DSA는 MQA-style Lightning Indexer로 query-key score를 빠르게 계산하고 Top-k=2048만 선택해 attention 비용을 $O(L^2)$ → $O(Lk)$로 줄임. (ii) GQA backbone에 그대로 얹을 수 있도록 dense warm-up + sparse adaptation 2-stage 학습. (iii) MOPD는 학생이 직접 sample한 응답에 정확히 맞는 teacher의 token-level log-prob 차이를 advantage로 사용해 distillation 안정화.
  • 비유: 256K 토큰을 모두 한 번에 보는 것은 ##2시간 영화의 모든 프레임을 동시에 펼쳐놓고 답하는 것과 같다##. DSA는 "지금 답하는 데 가장 관련 있는 2048개 프레임만 빠르게 골라서 본다"는 식의 똑똑한 선택. MOPD는 13명의 과목별 전문가가 학생이 푼 답을 "여기서 이렇게 더 잘 풀 수 있다"고 마크업하는 과외와 같다.

핵심 기여 (Key Contributions)

  1. Multimodal DeepSeek Sparse Attention 최초 적용: GQA-based MLLM backbone에 DSA를 결합해 256K context를 lossless로 처리. Prefill 비용 0.32×, decode 비용 0.20× 대비 dense attention (128K context).
  2. Cross-Modal Multi-Teacher On-Policy Distillation (MOPD): 13개 RL teacher(safety/math/IF/code/visual STEM/OCR/grounding/counting/video/tool 등)를 dynamic routing으로 sample별로 매칭. Top-k overlap 기반 advantage estimator + Segmented Prompt-Response Re-tokenization (SPRR) + token-category scaling + repetition collapse penalty.
  3. Native-Resolution + 2D RoPE Vision Encoder: Keye-VL-1.5의 SigLIP-400M-384-14 backbone을 변형해 임의 해상도 입력을 지원. NaViT 스타일 Patch n' Pack + FlashAttention으로 variable resolution 학습 가속.
  4. 4-stage Pre-training Curriculum: Stage 0 (projector init, 8K) → Stage 1 (32K, 1T tokens) → Stage 2 (64K, 2T tokens, multi-task injection) → Stage 3 (256K, 500B tokens, long-context). 비디오는 stage별로 max duration이 15초 → 15분 → 2시간으로 확장.
  5. SOTA 결과: ##Video-MME-v2 ACC 35.3 / 42.4## (64/512 frames), ##LongVideoBench 74.1##, ##ActivityNet-TimeLens 58.5, QVHighlights-TimeLens 70.1, Charades-TimeLens 58.4## — 같은 또는 더 큰 규모 open-source 모델(Qwen3.5-35B-A3B, Qwen3-VL-235B-A22B)과 closed-source(Gemini-3-Flash) 대비 leading 또는 경쟁적 성능.

1. Background — 왜 이 문제가 어려운가?

1.1 Long-Video Understanding의 두 가지 벽

(1) Context Scaling 벽: 시간 단위 영상은 수만~수십만 토큰의 시각 입력으로 표현된다. Transformer dense attention은 sequence length $L$에 대해 $O(L^2)$ 시간/메모리. 256K 토큰이면 KV 캐시만 100GB+ 수준. 기존 MLLM은 frame을 aggressive하게 subsample(예: 64 frames) → temporal 연속성을 잃어 이벤트 순서·인과 추론에 취약.

(2) Multi-task Conflict 벽: long-video, agentic tool use, math/STEM reasoning, OCR, grounding 등 여러 capability를 하나의 모델에 주입하려고 end-to-end로 co-optimize 하면, ##task-specific RL이 다른 task의 baseline reasoning capability를 catastrophically forget##. 예: agent task RL 후에 너무 짧은 답변을 생성하거나, math reasoning RL 후에 도구 호출 형식이 깨지는 식.

1.2 DeepSeek Sparse Attention (DSA) 배경

DSA는 DeepSeek에서 제안한 sparse attention 방법으로, 원래 MLA(Multi-head Latent Attention) 기반에 적용되었다. 핵심: 각 query token에 대해 Top-k 가장 관련 있는 KV만 사용해 attention 계산. Quadratic complexity를 $O(Lk)$로 줄인다. 본 논문은 ##DSA를 GQA backbone에 멀티모달 시각 도메인에 최초로 적용##.

1.3 Native-Resolution Vision Encoding 배경

전통적 ViT는 fixed resolution(예: 384×384)을 가정. 큰 이미지는 (a) tiling으로 잘게 나눠 각각 인코딩(InternVL3, MiniCPM-V) 또는 (b) native resolution을 그대로 사용(NaViT, Qwen3.5, K2.5, MiMO). Tiling은 global structure를 깨고 redundant. Keye-VL-2.0은 native-resolution 채택.

2.4 DSA for Long-Context Multimodal Modeling (Section 2.3)

2.4.1 MQA-Style Lightning Indexer (수식 1, 2)

각 query token $h_t$가 모든 preceding token $h_s$에 대해 global index score를 계산:

$$I_{t,s} = \sum_{j=1}^{H^I} w_{t,j}^I \cdot \text{ReLU}(q_{t,j}^I \cdot k_s^I) \tag{1}$$

$H^I$: indexer head 수, $w_{t,j}^I$, $q_{t,j}^I$는 $h_t$에서 derive, $k_s^I$는 $h_s$에서 derive (모든 query head 간 shared). After global score → Top-$k$ sparse index set:

$$\Omega_t = \{s \mid I_{t,s} \in \text{Top-}k(I_{t,:})\} \tag{2}$$

직관적으로: 모든 query head가 한 개의 shared key head를 통해 global score를 빠르게 계산 (MQA 스타일). 이로써 "지금 토큰이 가장 주의해야 할 2048개 토큰"을 효율적으로 선택. FP8 implementation + ReLU scoring으로 indexer가 long context에서도 efficient.

2.4.2 GQA Sparse Aggregation (수식 3)

GQA backbone에서 query head를 G grouping하고, 각 그룹은 shared KV head 사용. 동일한 sparse index set $\Omega_t$를 모든 그룹에 적용. g번째 그룹의 sparse attention output:

$$u_{t,g} = \text{Attn}(h_{t,g}, \{c_{s,g} \mid s \in \Omega_t\}) \tag{3}$$

$k = 2048$로 설정. Attention complexity가 $O(L^2)$ → $O(Lk)$로 감소. GQA backbone의 representation 구조를 그대로 보존하면서 dense attention을 회피.

2.4.3 Dense Warm-up과 Sparse Adaptation (수식 4-7)

Stage A: Dense Warm-up. 메인 모델은 dense GQA 유지, 대부분 파라미터 freeze. Indexer가 모든 GQA group의 dense attention 분포를 cover하도록 학습. Query token $t$, 그룹 $g$에서 dense attention score를 시각·텍스트 토큰에 normalize → target distribution $p_{t,:,g}$.

$$\mathcal{L}_{\text{warmup}}^I = \sum_t \sum_{g=1}^G \mathbb{D}_{KL}(p_{t,:,g} \| \text{Softmax}(I_{t,:})) \tag{4}$$

이 stage는 약 2B multimodal tokens 사용.

Stage B: Sparse Adaptation. 모든 파라미터 unfreeze, sparse mode로 전환. Indexer는 main model의 attention 분포에 align 유지하되, KL loss는 selected Top-$k$ token set에서만 계산:

$$S_t = \{s \mid I_{t,s} \in \text{Top-}k(I_{t,:})\} \tag{5}$$ $$\mathcal{L}_{\text{sparse}}^I = \sum_t \sum_{g=1}^G \mathbb{D}_{KL}(p_{t,S_t,g} \| \text{Softmax}(I_{t,S_t})) \tag{6}$$

Main model은 standard NTP loss. Indexer 입력은 detached(gradient 흐름 방지). 최종 objective:

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{NTP}} + \lambda \mathcal{L}_{\text{sparse}}^I \tag{7}$$

왜 2-stage인가: ##sparse training을 처음부터 하면 indexer가 무엇을 선택해야 할지 모르고, KL signal이 noisy하다##. Dense warm-up에서 dense attention의 행동을 정렬한 후 sparse 전환하면 안정적. Long-context 평가에서 dense model 능력 보존 + inference overhead 대폭 감소.

4. Post-Training (Section 4)

4.1 SFT + Synthetic CoT

SFT corpus: ~500B tokens. ##약 40%가 text-only## (instruction following, knowledge QA, textual reasoning anchor). Multimodal Instruction Mixture는 video, perception, reasoning, agent, long-context를 포괄.

Video data 형식: multiple-choice QA with clue intervals. 모델은 <think> stage에서 candidate temporal segment를 verify하고, 최종 답 + supporting interval `[[mm, mm], ...]`을 출력 — ##temporal evidence localization을 명시적으로 학습##.

Synthetic CoT: high-quality QA pair에서 strong teacher model이 reasoning trace 생성, query/response/process-level 품질 필터링. Math는 Doubt2Clean second-pass review로 doubtful CoT cleaning. 27 dataset에서 사용. Long <think> data는 STEM/counting/video/complex image-text QA용, direct-answer data는 general VQA/caption/OCR/grounding용. 균형 잡힌 mixture로 단순 perception에 불필요한 verbosity 회피.

4.2 Reinforcement Learning

4.2.1 Synthetic-Data RL

Procedurally generated difference-recognition tasks. 두 이미지 $(I_A, I_B)$, $I_B$는 controlled edit set $\mathcal{E} = e_1, \ldots, e_k$로 만들어짐. 모델은 변경 부위를 (a) localization으로 normalized bounding box로 예측, (b) structured reasoning으로 domain-specific operation set DSL 출력.

Reward (예: localization):

$$R_{\text{perc}} = \lambda_F F_{\text{soft}} + \lambda_I \frac{\sum_{m \in \mathcal{M}} u_m}{\max(|\hat{\mathcal{B}}|, |\mathcal{B}|)} - \lambda_D N_{\text{dup}} \tag{8}$$

$\mathcal{M}$: Hungarian matching된 페어 set, $u_m$: m번째 페어의 IoU, $F_{\text{soft}}$: soft matching-based F-score, $N_{\text{dup}}$: duplicate prediction penalty. ##Rule-based로 verifiable, learned reward model 없이 anti-hallucination + structured output 학습##.

4.2.2 General RL — GSPO 사용 (수식 9-11)

SFT 후 multimodal + text-only verifiable QA에서 GRP(General RL). Data: FineVision, MMK12, MM-Eureka, Thyme, mini O3, Open-R1 등. Pipeline: benchmark decontamination, cross-validation by judge, accuracy-based filtering.

Reward: format / outcome / process / ContextRL reward. Algorithm: Group Sequence Policy Optimization (GSPO):

$$\mathcal{J}_{\text{GSPO}}(\theta) = \mathbb{E}_{x \sim \mathcal{D}, \{y_i\}_{i=1}^G \sim \pi_{\theta_{\text{old}}}(\cdot|x)}\left[ \frac{1}{G} \sum_{i=1}^G \min\left(s_i(\theta) \hat{A}_i, \text{clip}(s_i(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_i \right) \right] \tag{9}$$

여기서:

$$s_i(\theta) = \exp\left( \frac{1}{|y_i|} \sum_{t=1}^{|y_i|} \log \frac{\pi_\theta(y_{i,t}|x, y_{i,직관적으로: GSPO는 GRPO와 비슷하지만 sequence-level importance ratio를 token 평균으로 stabilize. Filter groups with zero advantage variance — 모든 rollout이 같은 reward면 학습 신호가 없어 skip → data efficiency 향상.

4.2.6 Cross-Modal Multi-Teacher On-Policy Distillation (MOPD) — Section 4.2.6

왜 MOPD인가: 13개 RL teacher가 각각 specialized된 후, 단순히 mix해서 fine-tune하면 한 teacher의 style이 dominant해진다 (예: agent task RL 후 too short response, math RL 후 verbose reasoning). MOPD는 이를 unified MoE backbone에 안전하게 consolidate.

Dynamic Routing: 각 sample을 modality·task type에 가장 잘 맞는 teacher로 routing. 학생은 on-policy response 생성:

$$y_i = (y_{i,1}, \ldots, y_{i,n}) \sim \pi_\theta(\cdot \mid x_i) \tag{12}$$

State $s_{i,t} = (x_i, y_{i,Segmented Prompt-Response Re-tokenization (SPRR): prompt와 response를 따로 process — teacher와 student의 log-probability가 정확히 같은 response token에 align되도록 보장.

Top-k Overlap Set (수식 13):

$$\Omega_{i,t} = \text{TopK}(\pi_T^{r(i)}(\cdot|s_{i,t})) \cap \text{TopK}(\pi_\theta(\cdot|s_{i,t})) \tag{13}$$

Raw Advantage (수식 14):

$$A_{i,t} = \sum_{v \in \Omega_{i,t}} \pi_\theta(v|s_{i,t}) [\log \pi_T^{r(i)}(v|s_{i,t}) - \log \pi_\theta(v|s_{i,t})] \tag{14}$$

여기서:

$$\pi_\theta(v|s_{i,t}) = \frac{\pi_\theta(v|s_{i,t})}{\sum_{u \in \Omega_{i,t}} \pi_\theta(u|s_{i,t})} \tag{15}$$

$\Omega_{i,t} = \emptyset$이면 $A_{i,t} = 0$. 직관적으로: full vocabulary 대신 ##teacher와 student 모두 plausible한 token만 supervision## — 매우 낮은 확률 token에 대한 unstable 비교 회피, on-policy 유지.

MOPD Loss (수식 16, 17):

$$\mathcal{L}_{\text{MOPD}} = -\mathbb{E}\left[ \frac{1}{|\mathcal{M}_i|} \sum_{t \in \mathcal{M}_i} \hat{A}_{i,t} \log \pi_\theta(y_{i,t} | x_i, y_{i,$\mathcal{M}_i$: valid response-token mask. Token-category-aware advantage scaling을 적용해 formatting token은 down-weight, perception/reasoning token은 up-weight. Long-form generation에서 repetition collapse를 위치 $\tau_i$에서 detect해 그 이후만 penalize:

$$\hat{A}_{i,t} = \tilde{A}_{i,t} - \lambda_{\text{rep}} \cdot \mathbb{1}[t \geq \tau_i] \tag{17}$$

이 디자인의 합으로 ##13개 RL teacher의 expertise가 single MoE로 consolidate되면서, 각 도메인의 response style을 강제로 한 형식으로 통일하지 않음##.

5. Efficient Training and Inference Infrastructure (Section 5)

5.1 ExtraIO와 ViT-LM Heterogeneous Parallelism

ExtraIO: video decoding/frame sampling을 학습 process와 decoupling하는 horizontally scalable I/O service. ViT-LM Heterogeneous Parallelism: ViT와 LM이 같은 GPU group에 colocate되지만 independent sharding strategy. recompute-or-offload로 ViT activation memory를 거의 0으로. Multimodal-token + LM-sample level Load Balancing으로 ViT DP/PP와 LM DP/PP의 compute/memory 균형 → end-to-end throughput +20%.

5.2 DSA Optimization

FlashInfer + TileLang으로 baseline 대비 ##2× speedup##. Top-k Memory Optimization: indexer score storage를 $T \times T$에서 $T \times \text{max\_seq}$로 축소, flashinfer.top_k_ragged_transform 사용. Short-Sequence Optimization: positional index가 top-k보다 작으면 indexer/sparse-attention backward kernel이 causally attendable KV만 iterate → ##1.5× end-to-end speedup##. Indexer Loss: post-top-k indexer score만 sparse-attention backward kernel 안에 store, FlashAttention-style backward recomputation 재사용 → per-layer $T \times T$ intermediates 회피.

5.3 Inference Optimization

Optimization효과

7. 비판적 분석 (Critical Analysis)

강점

  • 아키텍처-알고리즘 co-design: DSA를 GQA backbone에 적용한 첫 사례, dense warm-up + sparse adaptation 2-stage가 sparsity 학습 안정화의 좋은 recipe.
  • Multi-teacher distillation의 implementation 깊이: 단순한 SFT-style distillation이 아니라 Top-k overlap, SPRR, token-category scaling, repetition collapse penalty까지 — 각 디테일이 실용적 문제 해결.
  • 비용 효율성: 30B-A3B (active 3B)로 235B-A22B에 필적 또는 능가. Inference cost는 dense attention 128K에서 prefill 0.32×, decode 0.20×.
  • 실용적 디자인: timestamp prepend로 별도 video encoder 없이 image-as-frame 형식 — code/maintenance 측면에서 우월한 단순함.
  • Adaptive video pixel budget: duration에 따라 token 사용 scaling — 짧은 영상에 over-allocation, 긴 영상에 under-allocation 문제 동시 해결.

약점·의문

  • Technical report style: ablation이 거의 없다. DSA의 dense-only baseline 비교, MOPD vs. simple SFT-distillation 비교, Video-RL의 contribution 분리 등 어느 컴포넌트가 얼마나 기여하는지 분리 평가 부재.
  • Video-MME-v2 Non-Lin에서 235B에 뒤짐: 단순 ACC가 아닌 group-based non-linear score $(N/4)^2$에서는 235B Qwen3-VL이 우위. ##cross-question consistency가 떨어진다는 신호##일 수 있다 — 동일 영상에 대한 4개 관련 질문 중 일관되게 답하지 못함.
  • SWE-bench Verified 62.0 vs Qwen3.5-35B 63.5: 모델이 작아서가 아니라 repository-level 작업 학습이 충분치 않다는 신호.
  • Capability vector / requirement vector 추정: Adaptive video pixel budget threshold (256/512/1024/2048s)가 어떻게 정해졌는지 비공개. Hyperparameter ablation 없음.
  • SOTA 비교 모델의 일관성: 여러 baseline 점수가 "—"로 표시 (직접 비교 불가). 자체 평가의 fair comparison이 어렵다.
  • Hallucination 평가: HallucinationBench 73.5로 Qwen3.5-35B 57.3 대비 좋다고 하지만, Video-MME-v2의 long-range reasoning에서 강한 모델이 더 잘하는 경향과 tension. 어떤 종류 hallucination에서 강한지 분석 부재.
  • 13 teacher 구성: full 13 teacher 중 어떤 게 핵심인지, 일부 dropping 시 loss는 어떨지 분석 부재.
  • Random seed / std 미보고: 모든 결과가 single-run 추정.

제안

  • DSA의 effective $k$ 값 sweep — k=512, 1024, 2048, 4096 별로 정확도/비용 trade-off 측정.
  • MOPD ablation: simple multi-teacher SFT vs. MOPD vs. on-policy distillation without Top-k overlap. 각 컴포넌트 분리.
  • Long-context "needle-in-haystack" 평가: 256K 안에서 specific evidence localization 정확도 측정.
  • 비교 baseline의 inference cost — Keye가 0.20× decode cost라고 주장하면, 같은 256K에서 Qwen3-VL은 어떤 비용인지 직접 비교.

8. Key Takeaways

  • DSA를 GQA에 적용하는 dense warm-up → sparse adaptation 2-stage recipe는 long-context multimodal 모델에 직접 채택 가능한 형태. 다른 sparse attention 방법 (BigBird, Longformer)에도 일반화될 수 있는 패턴.
  • Multi-task RL conflict 해소를 위한 MOPD: ##specialized teacher → student의 on-policy distillation##은 RLHF 시대의 capability conflict를 해결할 표준 도구가 될 수 있다. Top-k overlap advantage는 anywhere applicable한 trick.
  • Native-resolution + timestamp injection: 별도 video encoder를 두지 않고 frame을 high-res image로 처리, timestamp를 자연어로 prepend. ##Code maintenance 단순화 + LLM의 자연어 ability 활용##.
  • Adaptive video pixel budget: duration에 따라 token scaling. Short-form/long-form 양쪽에 효율적 — 다른 video MLLM에 즉시 적용 가능한 lever.
  • Bucket advantage scaling for Video-RL: long-sequence decision tree 안정화 + visual hallucination 감소. 다른 long-form generation RL에도 적용 가능.
  • 30B/3B-active로 235B에 필적: MoE + sparse attention의 합이 production-grade trade-off. 모델 크기를 늘릴 수 없는 환경에서 capability를 끌어올리는 데 핵심 design pattern.
