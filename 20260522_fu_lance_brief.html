Lance — 하나의 AI가 보고·그리고·편집까지 다 한다

이미지와 비디오를 '이해'하고 '생성'하고 '편집'하는 능력을 단 하나의 작은 모델(3B)에 담았다. 비결은 두 능력에 별도의 전문 두뇌를 주되, 같은 작업실에서 함께 훈련시키는 것.

먼저 알아야 할 배경 지식

배경 1 — "Unified(통합) 멀티모달 모델"이란?

지금까지 AI는 보통 한 가지만 잘했다. 이미지를 이해하는 AI(예: "이 사진 속에 뭐가 있어?")와 이미지를 생성하는 AI(예: Midjourney, "고양이를 그려줘")는 서로 다른 모델이었다. Unified 모델은 이 둘을 하나로 합치려는 시도다. 한 모델이 보기도 하고 그리기도 한다.

여러 전용 모델 대신, 하나의 통합 모델이 모든 시각 task를 처리한다.

배경 2 — 왜 합치기가 어려울까? (핵심 딜레마)

이해와 생성을 합치는 게 왜 어려운가? 두 능력이 요구하는 '시각 정보의 종류'가 정반대이기 때문이다.

👁️ 이해(보기)에 필요한 것

고차원 의미(semantic). "이건 고양이고, 슬퍼 보이고, 실내에 있다" — 세부 픽셀은 중요하지 않다.

🎨 생성(그리기)에 필요한 것

저차원 디테일(texture·geometry). 털 한 올, 그림자, 색감 — 픽셀 수준 디테일이 결정적.

🍳 비유 (요리 평론가 vs 요리사): '맛 평론'(이해)에는 "이건 매콤하고 균형 잡힌 요리"라는 고차원 판단이면 충분하다. 하지만 '직접 요리'(생성)에는 "소금 3g, 180도에서 7분"이라는 정밀한 디테일이 필요하다. 한 사람의 머릿속에서 두 모드를 억지로 같은 회로로 처리하면 서로 방해한다.

Lance의 핵심 아이디어 — 2가지 원칙

원칙 1 — 같은 작업실 (Unified Context)

텍스트·이미지·비디오를 모두 하나의 긴 줄(sequence)에 섞어 모델에게 보여준다. 그래서 모든 능력이 같은 맥락을 공유하며 서로 배울 수 있다.

원칙 2 — 별도의 전문 두뇌 (Decoupled Experts)

같은 작업실을 공유하되, '이해 전문가'와 '생성 전문가' 두 개의 별도 두뇌를 둔다. 서로 같은 파라미터를 두고 싸우지 않게.

Lance 전체 구조: 입력 → MaPE로 역할 표시 → 공유 sequence → 두 전문가가 각자 처리.
🧠 비유 (좌뇌·우뇌 같은 작업실): 사람의 뇌도 언어(좌뇌)와 공간 인지(우뇌)에 특화된 영역이 나뉘어 있지만, 같은 머리 안에서 정보를 공유한다. Lance도 이해·생성 전문가를 분리하되 같은 정보(공유 sequence)를 보게 해서, 두 능력이 서로를 강화하게 만든다.

보너스 아이디어 — MaPE (역할 이름표 붙이기)

한 줄에 여러 종류의 시각 토큰(의미용/조건용/생성용)이 섞이면 모델이 헷갈린다. MaPE는 각 토큰 그룹을 '시간축에서 서로 다른 출발점'에 배치해서 "이건 의미용, 이건 생성 타깃"이라고 위치로 알려준다.

🏷️ 비유 (회의실 명찰): 여러 부서 사람이 한 회의실에 모이면 누가 누군지 헷갈린다. 부서별로 다른 색 명찰을 달아주면(=MaPE) 즉시 역할이 구분된다. 특히 '편집'처럼 원본(조건)과 결과물(생성)을 동시에 봐야 하는 작업에서 효과가 컸다.

어떻게 학습했나 — 4단계 훈련

1PT (사전학습)

대량 데이터(이미지 10억·비디오 1.4억)로 기본 보기·그리기 능력 확립.

2CT (지속학습)

편집·subject-driven 등 다양한 task 추가. 여기서 multi-task 시너지가 발생.

3SFT (정밀조정)

고품질 데이터로 지시 따르기·일관성을 다듬음.

4RL (강화학습)

OCR 모델을 채점관으로 써서 이미지 속 글자 정확도를 강화.

가장 흥미로운 발견 — "다재다능이 서로를 키운다"

🎯 Multi-Task Synergy (다중작업 시너지)

여러 task를 함께 학습시키면 보통 "이것저것 다 하느라 각각은 평범"해질 거라 생각한다. 그런데 Lance에서는 정반대였다:

  • 이해 데이터를 추가했더니 → 이미지·비디오 '생성'이 좋아졌다. (보는 연습이 그리기를 도움)
  • 생성 편집 데이터를 추가했더니 → 비디오 '이해'까지 좋아졌다. (그리는 연습이 보기를 도움 — 이건 정말 의외!)

즉, multi-task는 단순히 능력을 '쌓는' 게 아니라 능력들이 서로를 강화(mutual reinforcement)한다.

🎵 비유 (악기와 작곡): 작곡을 배우면 연주 실력이 늘고, 연주를 많이 하면 작곡 감각이 는다. 따로 배우는 것보다 함께 배울 때 두 능력이 시너지를 낸다. Lance의 이해-생성 관계가 딱 이렇다.

한계점 — 균형 잡힌 시각

🔸 "3B"는 약간의 마케팅

'3B 활성 파라미터'를 강조하지만, 두 전문가(MoE) 구조라 전체 파라미터는 더 크다. ViT 인코더·VAE까지 합치면 실제 모델 크기는 "3B"가 주는 인상보다 크고, 128 GPU도 결코 작은 자원이 아니다.

🔸 글자 편집이 약하다

저자도 인정하듯 이미지 속 텍스트 수정(text modification)이 약점이다. 그래서 RL 단계에서 OCR 채점관까지 동원해야 했다. 정밀한 글자 일관성은 여전히 미해결 과제.

🔸 작은 차이로 내린 결론

"다중작업이 비디오 이해를 향상"이라는 흥미로운 주장은 58.1→59.2(약 1점) 같은 작은 차이에 기반한다. 통계적 유의성이나 여러 번 반복한 결과가 보고되지 않아, 이 작은 향상이 노이즈인지 진짜인지는 더 검증이 필요하다.

🔸 "from scratch"의 모호함

초록에서 "처음부터 학습(from scratch)"이라 하지만 실제로는 Qwen2.5-VL 3B 가중치로 초기화한다. 표현과 실제 사이에 약간의 수사적 긴장이 있다.

활용 분야

하나의 모델로 생성·편집·이해를 다 하므로, 여러 도구를 갈아탈 필요 없이 통합 워크플로우가 가능.
"이해와 생성을 한 두뇌에서 싸우게 하지 말고,
별도 전문가로 나누되 같은 작업실을 공유시켜라.
그러면 작은 모델(3B)로도 큰 모델을 이길 수 있고,
능력들이 서로를 키운다."

핵심을 다시 한 번

  • 🎯 Lance = 이미지+비디오의 이해·생성·편집을 하나로 합친 통합 멀티모달 모델.
  • 🧠 비결 = 같은 작업실(공유 맥락) + 별도 전문 두뇌(이해/생성 분리).
  • 🏷️ MaPE = 여러 종류 시각 토큰에 '역할 명찰'을 붙여 헷갈림 방지 (특히 편집에 효과).
  • 🤝 가장 흥미로운 발견 = 다재다능이 서로를 강화한다(보기↔그리기 시너지).
  • 📊 3B 작은 크기로 GenEval 0.90·VBench 85.1 등 7~20B 모델 능가.
  • ⚠️ 단, '3B'는 활성 파라미터 기준이고 글자 편집은 아직 약점.
