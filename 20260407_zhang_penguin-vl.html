TL;DR

  • 문제: 현재 VLM은 massive contrastive pretraining (CLIP/SigLIP) 기반 vision encoder에 의존하지만, contrastive learning의 discriminative objective는 dense captioning과 complex reasoning에 필요한 fine-grained visual cue를 억제한다
  • 해결: Penguin-VL은 text-only LLM (Qwen3-0.6B)을 직접 vision encoder로 초기화하고 (Penguin-Encoder), bidirectional attention + 2D-RoPE로 전환한 뒤, mixed supervision (amplitude + direction + relation loss)으로 pretraining하여 generative objective에 최적화된 vision representation을 학습한다
  • 왜 작동하는가: LLM-initialized encoder는 (1) downstream LLM decoder와 native하게 aligned된 representation space에서 시작하고, (2) rich semantic prior를 상속하며, (3) LLM의 advanced architecture (QK normalization 등)를 활용한다. Relation loss가 inter-patch relationship을 보존하여 attention-based visual reasoning에 핵심적이다
  • 비유: 기존 방법이 "외국어(contrastive representation)를 배운 뒤 번역"하는 것이라면, Penguin-VL은 "모국어(LLM representation)로 처음부터 시각 세계를 이해"하는 것이다

핵심 기여 (Key Contributions)

  1. Penguin-Encoder: ##Text-only LLM에서 직접 vision encoder를 초기화하는 novel approach. Contrastive pretraining 없이, ~400M 파라미터로 SigLIP2를 능가하는 vision representation 학습 (~240M data vs >40B contrastive data)##
  2. Mixed supervision pretraining: Amplitude loss $L_A$, direction loss $L_D$, relation loss $L_R$의 3-component reconstruction objective로, LLM-initialized encoder를 visual feature space에 효과적으로 align
  3. TRA (Temporal Redundancy-Aware) token compression: Key frame과 intermediate frame을 구별하고, 3-stage cascade로 spatial resolution을 content-adaptive하게 압축하여 long video를 효율적으로 처리
  4. Comprehensive training recipe: Encoder training (low-res + high-res) → VLM pretraining (121M) → two-stage SFT (image 39M + video 3.7M)의 체계적 pipeline과 대규모 data curation (Penguin-Recap-I 57.2M, Penguin-Recap-V 3.7M)

1. Introduction

현재 VLM의 지배적 패러다임은 CLIP/SigLIP으로 contrastive pretrained된 ViT를 vision encoder로 사용하는 것이다. 그러나 이 approach에는 근본적인 objective mismatch가 존재한다: contrastive learning은 global [CLS] token 또는 attention pooling에만 supervision을 적용하는 discriminative paradigm으로, LLM의 sequence token prediction과 근본적으로 misaligned되어 있다. 이로 인해 fine-grained visual cue가 억제되고, category-level invariance가 enforce된다.

본 논문은 vision encoder bottleneck에 대한 holistic framework를 제안한다:

  1. Novel vision encoder: LLM architecture 원칙을 활용한 VLM 전용 encoder
  2. Tailored training recipe: Fine-grained perception과 cross-modal reasoning을 progressive하게 harmonize
  3. Comprehensive data curation pipeline: 고품질 training data 구축

2. Methodology

2.1 Model Architecture

Penguin series는 Qwen3 LLM backbone 위에 구축되며, 2B와 8B variant가 있다. Three-module design: (i) Penguin-Encoder (vision), (ii) MLP-based vision-language projector, (iii) LLM.

2.1.1 Penguin-Encoder: From Text-only LLM to Vision Encoder

Text-only LLM (Qwen3-0.6B)에서 직접 vision encoder를 초기화한다. 핵심 수정 두 가지:

  1. Causal → bidirectional full attention: LLM의 causal self-attention mask를 제거하여 symmetric token interaction을 가능하게 함
  2. 1D RoPE → 2D RoPE: Variable-resolution image 입력을 위한 2D rotary positional embedding 적용

이 text-to-vision 초기화의 4가지 장점:

  • Architectural expressivity: QK normalization 등 LLM의 advanced design elements 활용
  • Native alignment: LLM decoder와 호환되는 representation space에서 시작하여 modality gap 최소화
  • Semantic priors: Rich linguistic knowledge를 상속하여 visual concept의 semantic grounding 제공
  • Scalability: LLM scaling 원칙을 활용한 predictable scaling

Mixed supervision pretraining: LLM cross-entropy supervision과 reconstruction-based objectives를 jointly 사용. Teacher encoder의 feature $F_t$에 대해, student의 feature $F_s$로:

(1) Amplitude Loss: Feature 절대값의 직접적 supervision

$$L_A = \frac{1}{N}\sum_N (|F_s - F_t|)$$

(2) Direction Loss: Feature 방향의 cosine similarity (representation collapse 방지)

$$L_D = \frac{1}{N}\sum_N \text{tr}\left(\frac{F_s F_t^\top}{\|F_s\|_2 \|F_t\|_2}\right)$$

(3) Relation Loss: Inter-patch self-correlation similarity (attention mechanism에 특화)

$$L_R = \frac{1}{N}\sum_N \left(\left|\frac{F_s F_s^\top}{\|F_s\|_2^2} - \frac{F_t F_t^\top}{\|F_t\|_2^2}\right|\right)$$

직관적으로, amplitude loss는 "각 패치의 특성 강도"를, direction loss는 "특성의 방향"을, relation loss는 "패치 간의 관계 구조"를 보존한다. ##Relation loss가 특히 중요한데, vision model에서 attention mechanism은 개별 token의 absolute attribute보다 inter-patch relationship에 초점을 맞추기 때문이다 (ablation에서 +1.3 avg score 기여 확인).##

2.1.2 Video Encoding and Projector

TRA (Temporal Redundancy-Aware) token compression: Temporal similarity 기반으로 frame을 key frame (rapid temporal changes)과 intermediate frame (stable context)으로 분류하고, 3-stage cascade로 압축:

  • Stage 1 (Resolution Preservation): 전체 token count가 $T_{\max}$ 이하이면 full resolution 유지: $$\sum_{k \in \mathcal{K}} T_k + \sum_{i \in \mathcal{I}} T_i \leq T_{\max} \tag{1}$$
  • Stage 2 (Synchronous Downscaling): Budget 초과 시, key/intermediate frame을 고정 비율 ($T_k \approx 16T_i$)로 동시 downscale: $$T_k \leftarrow \alpha T_k, \quad T_i \leftarrow \alpha T_i \tag{2}$$
  • Stage 3 (Saturation-Aware Scaling): Intermediate frame이 하한 $T_{\min}$에 도달하면, key frame만 추가 압축

MLP-based projector: Spatial token merge 없이 simple feed-forward transformation으로 visual feature dimension을 LLM hidden size에 맞춤. Simplicity와 efficiency를 우선시하는 design choice.

2.2 Data Construction

2.2.1 Image Data Curation (Penguin-Recap-I)

57.2M image-text pairs from COYO-700M + DataComp-1B. Two-stage pipeline: (1) quality filtering (resolution, aspect ratio, corruption), (2) semantic diversity via hierarchical k-means clustering + greedy selection. Detailed long captioning: 9개 visual aspect (Global Semantics, Subjects, Actions, Spatial Relationships, Scene Context, Objects, OCR, Image Quality, Knowledge-intensive descriptions)에 대한 structured annotation 후 comprehensive long-form caption으로 synthesis.

2.2.2 Video Data Curation (Penguin-Recap-V)

3.7M video-text pairs from 29 public datasets. Multi-granularity annotation: (1) Event-level atomic descriptions (timestamped), (2) Chapter-level narrative syntheses, (3) Holistic video summaries. Temporal reasoning QA (ordering + grounding)도 생성.

3. Training

3-stage training pipeline:

Stage 1: Penguin-Encoder Training ├─ Phase 1 (Low-res): ~100M samples, 2048 tokens, ~600×600 │ caption loss + reconstruction loss (L_A + L_D + L_R) │ teacher: VL3-SigLIP-NaViT └─ Phase 2 (High-res): ~47M samples, 10240 tokens caption loss only (reconstruction branch removed) Stage 2: VLM Pre-training └─ 121M samples, all components trainable 64% general caption + document/grounding/OCR/math/code/science 7.7M grounding + 1.5M region caption QA pairs Stage 3: Supervised Fine-Tuning ├─ Image SFT: 39M samples │ 32.6% General & Caption, 20.9% Document/Chart, 16.6% OCR │ 10.1% Grounding & Counting, 8.9% Math, 3.71% Others └─ Video SFT: 3.7M samples 77.6% General Video, 12.7% Action, 6.9% Temporal, 2.8% Ego

4. Experiments

4.1 Implementation Details

Cosine LR decay, 3% warmup. Max sequence 16,384 tokens (up to 10,240 visual). Vision encoder: Qwen3-0.6B init. LLM backbone: Qwen3-1.7B (2B), Qwen3-8B (8B). Projector: 2-layer MLP with GELU. Video: 1 FPS, max 180 frames, 2× spatial downsampling after encoder. lmms-eval toolkit for evaluation.

4.2 Baselines

2B: Qwen3-VL-2B, InternVL3.5-2B, Gemma3n-E2B-it, SmolVLM2-2.2B. 8B: Qwen3-VL-8B, InternVL-3.5-8B, OpenAI GPT-5-nano.

원문은 encoder comparison에서 3가지 핵심 관찰을 제시한다:

  • Overall performance advantage: Contrastive encoder는 discriminative alignment에 최적화되어 있어 reasoning-centric generation에는 본질적으로 불리한 반면, Penguin-Encoder의 generation-aligned design은 LLM의 high-level semantic space에 직접 mapping하여 autoregressive decoding에 본질적으로 적합하다
  • Scalability: 데이터/task complexity가 증가할수록 contrastive encoder보다 Penguin-Encoder의 성능 향상 폭이 더 크며, 더 높은 performance ceiling을 시사한다
  • Comparison under Matched Data: Pre-trained SigLIP2를 Penguin-Encoder와 동일한 데이터 및 training recipe (Stage 1)로 추가 학습시킨 통제 실험에서도 Penguin-Encoder가 우위를 유지하여, domain overlap이 아닌 architecture 자체의 이점을 확인했다

5. Related Work

Vision-centric Multimodal-LLM: GPT-4o, Gemini, LLaVA-OneVision, Qwen3-VL, InternVL 3.5, Molmo, Gemma 3 등의 발전. Encoder design: CLIP/SigLIP 기반 ViT가 지배적이나, SAM-based encoder, EAGLE Mix-of-Encoder, DeepSeekOCR2의 LLM-based encoder 등 대안이 탐색되고 있다. Video understanding: Fixed frame budget + pooling에서 temporal convolution, dual-branch (SlowFast-LLaVA), dynamic key/motion frame selection (Keye-VL 1.5)으로 발전.

6. Conclusion and Future Work

Penguin-VL은 contrastive pretraining에 대한 대안으로, text-only LLM에서 직접 vision encoder를 초기화하는 approach를 제안했다. 2B와 8B scale에서 document OCR부터 long-context temporal reasoning까지 다양한 image/video 벤치마크에서 competitive 또는 SOTA 성능을 달성했다. 이는 generation-aligned encoder initialization이 massive contrastive pretraining보다 data-efficient하고 우수한 경로임을 입증한다.

Future Work: (1) Real-time inference optimization (adaptive computation, token sparsity), (2) RL-based post-training for VLMs, (3) Agentic visual-language systems (GUI agents, computer-use scenarios).

비판적 분석 (Critical Analysis)

  • Ablation의 제한된 규모: Table 3의 ablation은 10% Stage 1 data + DenseFusion-1M + LLaVA-665k의 경량 설정에서만 수행되었다. Full-scale training에서의 component 기여도가 달라질 수 있다
  • OCRBench와 MathVerse 열위: Qwen3-VL 대비 OCRBench (810 vs 858)과 MathVerse (35.9 vs 52.1)에서의 gap은 상당하며, 특정 도메인에서 contrastive encoder의 이점이 남아있을 수 있음을 시사한다
  • MMMU-Pro 8B gap: 40.2 vs Qwen3-VL 55.9는 매우 큰 차이로, expert-level multi-image reasoning에서의 한계를 보여준다
  • Encoder scaling 미검증: Penguin-Encoder는 ~400M params (Qwen3-0.6B 기반)만 사용. Larger encoder (1.5B+)의 효과는 "future scenarios"로만 언급되고 실험이 없다
  • Teacher 의존성: Stage 1에서 VL3-SigLIP-NaViT를 teacher로 사용하므로, 완전히 contrastive pretraining에서 independent하지는 않다 (teacher의 quality가 encoder 성능에 영향)
  • Inference speed 미보고: TRA의 실제 inference throughput이나 latency 비교가 없어, "efficiency" 주장을 정량적으로 검증하기 어렵다
  • 코드/모델 공개: GitHub + HuggingFace에 2B/8B 모두 공개되어 재현 가능
  • Data curation의 독립적 기여 불명확: Penguin-Recap-I/V 데이터셋의 quality가 encoder architecture의 기여와 confound될 수 있으며, 동일 데이터로 SigLIP2를 학습한 비교 실험이 제한적이다

Key Takeaways

  1. Contrastive pretraining은 필수가 아니다: LLM-initialized encoder가 ~240M 데이터로 >40B contrastive data를 능가한다는 결과는 VLM encoder 학습의 패러다임 전환 가능성을 시사한다
  2. Native alignment의 힘: Encoder와 decoder가 동일한 representation space에서 시작하면, modality alignment이 근본적으로 수월해진다. 이는 "bridge the gap" 대신 "start aligned"라는 설계 철학이다
  3. Relation loss의 중요성: Inter-patch relationship supervision이 individual token representation보다 downstream reasoning에 더 중요하다는 발견은, vision encoder 학습의 objective design에 시사점을 제공한다
  4. Data curation의 가치: 9-aspect structured annotation + hierarchical clustering diversity selection은 raw quality와 data-efficiency를 모두 높이는 체계적 approach이다
  5. Compact model의 경쟁력: 2B model이 대부분의 벤치마크에서 Qwen3-VL-2B를 능가한다는 것은, mobile/edge deployment에서 Penguin-VL의 실용적 가치가 높음을 보여준다
