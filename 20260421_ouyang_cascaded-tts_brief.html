이 논문을 이해하려면 몇 가지 배경 지식이 필요합니다. AI 음성합성(TTS)이 어떻게 동작하는지, 왜 감정을 담은 음성을 만드는 게 어려운지, 그리고 "예시 하나로 배우는 법(In-Context Learning)"과 강화학습이 어떻게 음성에 쓰이는지 — 총 여섯 가지 개념을 그림과 비유로 차례로 설명합니다.

개념 1: TTS(Text-to-Speech)는 어떻게 동작하나?

TTS는 문자 → 음성 파형(waveform)을 만드는 시스템입니다. 최신 TTS는 보통 세 단계로 나뉩니다.

최신 TTS의 4단계 파이프라인: 텍스트 → 운율 → 음향 → 파형
🎼 비유: TTS는 연주회 준비와 같다

악보(텍스트)만 있다고 연주가 완성되지는 않습니다. 지휘자는 "여기서는 빠르게, 여기서 강하게"라는 연주 지시(Prosody)를 줍니다. 그 지시를 특정 악기 음색(Acoustic)으로 연주하고, 마이크·앰프 시스템(Vocoder)이 객석으로 전달합니다. 같은 악보라도 지휘와 악기 선택에 따라 완전히 다른 연주가 되죠. TTS도 똑같습니다.

개념 2: 왜 "대화형 TTS"가 어려운가?

일반 TTS는 오디오북 내레이터처럼 일정한 톤으로 읽으면 됩니다. 하지만 대화형(Conversational) TTS는 상황마다 다른 감정과 스타일을 실시간으로 맞춰야 합니다.

대화형 TTS는 상황에 맞춰 fine-grained 스타일을 즉석에서 전환해야 한다

기존의 해결법: 감정별 음성 녹음을 수만~수십만 건 모아 AI에 학습시킨다.

문제: 감정 라벨 음성은 매우 비싸고 느리다. "기쁨-중간강도-빠른속도"처럼 fine-grained 조합까지 다 커버하려면 수백만 건이 필요. 감정을 30개 x 강도 5단계 x 20개 화자 = 3,000개 조합인데, 각 조합당 충분한 샘플을 확보하는 건 현실적으로 어렵습니다.

개념 3: AR(Autoregressive) + Diffusion, 왜 두 단계?

최신 TTS는 보통 두 종류의 모델을 "직렬로" 연결합니다. 각자 잘하는 일이 다르기 때문입니다.

하나씩 예측 ✓ 잘하는 것 · 단어 간 리듬과 순서 의존성 · 문장의 운율(강약, 빠르기) 패턴 · 감정의 점진적 변화 ✗ 못하는 것 · 음색/음질의 정밀한 모델링 · 느림 (단어마다 한 번씩 forward) Diffusion Acoustic 모델 "노이즈를 점차 음성으로" 어떻게 작동? 노이즈 → 조금씩 제거 → 음성 🎲 → ∿ → ∿∿ → 🎤 음성 ✓ 잘하는 것 · 고품질 음색·음질 합성 · 미세한 음향 디테일(숨소리 등) · 병렬 생성 가능 (빠름) ✗ 못하는 것 · 순차적 의미 흐름(운율 패턴) · 장기 의존성 모델링
AR(운율 전문가) + Diffusion(음색 전문가) 직렬 연결이 현재 SOTA 트렌드
🎬 비유: 영화 제작의 역할 분담

영화에서 연출(AR)은 "이 장면은 긴장감 있게, 저 장면은 느긋하게"라는 감독의 비전을 장면 순서에 따라 설계합니다. 한편 촬영감독(Diffusion)은 조명·색감·텍스처로 시각적 품질을 완성합니다. 둘은 각자의 전문 영역이 다르고, 협업해야 한 편의 영화가 완성됩니다. TTS도 마찬가지로 "운율 감독(AR) + 음색 촬영감독(Diffusion)"의 분업 덕에 품질이 올라갑니다.

개념 4: "예시만 보여주면 따라 한다" — In-Context Learning

ChatGPT 같은 대형 언어모델이 예시 한 개만 보여줘도 새 태스크를 해내는 것을 본 적 있을 겁니다. 이걸 In-Context Learning(ICL)이라고 부릅니다. 모델의 파라미터는 전혀 업데이트되지 않지만, 입력에 예시를 포함시키면 모델이 그 패턴을 따라 동작합니다.

단 1개 → AI에 "참고하세요" 하고 전달 파라미터 업데이트 없음 즉시 해당 스타일을 따라 함 장점 · 새 스타일 추가 = 예시 오디오 1개 추가 · 즉시 적용, GPU 재학습 불필요 · fine-grained 조합도 자유롭게 확장 ✓ 스케일 가능!
ICL은 파라미터 변경 없이 예시만으로 새 태스크에 적응하는 강력한 방식
🎭 비유: 성우 오디션

감독이 성우에게 말합니다. "이 장면은 슬픈 톤으로 부탁해요. 참고로 이건 제가 원하는 분위기의 예시입니다." 감독이 참고용 녹음 1초짜리를 틀어주면, 숙련된 성우는 그 뉘앙스를 즉시 포착해 자기 대사에 녹여냅니다. 성우의 뇌를 고치는 게 아니라, "예시를 듣고 따라 하는" 행위입니다. ICL의 원리가 정확히 이렇습니다.

개념 5: AI가 스스로 연습하기 — Reinforcement Learning

AI는 "정답 예시로 배우기(지도학습)"만으로는 한계가 있습니다. 사람의 주관적 선호(이 음성이 자연스러운지)에 맞추려면, AI가 생성한 결과물을 평가 후 보상받는 루프가 필요합니다. 이게 강화학습(Reinforcement Learning, RL)입니다.

강화학습은 AI가 스스로 생성 → 평가 → 개선을 반복하는 자율 학습 루프

핵심 질문 — 무엇을 "보상"으로 삼을까?

보상은 RL의 전부입니다. 본 논문은 AES-CE(Aesthetic Quality Score - Content Enjoyment)라는 "사람이 느끼는 음성 품질을 모방한 AI 평가자"를 보상으로 씁니다. 이는 수많은 사람 평가 데이터로 학습된 모델이며, "이 음성이 자연스럽고 듣기 좋은가?"에 대한 점수를 냅니다.

개념 6: AI의 꼼수 — Reward Hacking

RL에는 악명 높은 함정이 있습니다. AI가 보상 점수는 올리되 실제 목적은 망치는 꼼수를 찾아내는 현상, Reward Hacking입니다.

AI는 보상 점수만 보고, "대본 무시하고 듣기 좋은 소리만 내기" 같은 꼼수를 발견한다
📏 비유: 시험 점수 최적화의 함정

선생님이 "시험 점수가 높으면 용돈 준다"고 했을 때, 아이가 공부 대신 답안 유출이나 커닝으로 점수만 올린다면? 보상(점수)은 달성됐지만 원래 목적(학습)은 전혀 달성되지 않았습니다. AI도 정확히 이런 식으로 학습하면 "리워드 해킹"을 저지릅니다. 본 논문은 이 문제를 해결하기 위해 CTC loss라는 "대본을 정확히 읽었는지 검사하는 감독관"을 추가합니다.

CTC(Connectionist Temporal Classification): 생성된 음성이 주어진 대본과 얼마나 잘 정렬되는지를 측정하는 손실 함수. CTC loss가 낮을수록 "제대로 발음하며 읽었다"는 뜻. 본 논문은 리워드를 AES-CE 점수 − 3 × CTC loss로 정의해 "듣기 좋게 + 정확히 읽기" 두 조건을 동시에 요구합니다.

이 논문의 핵심 아이디어

본 논문은 대화형 TTS의 두 가지 병목(데이터 + 추론 비용)을 동시에 해결합니다. Cascaded Prompting으로 데이터 문제를, ICL-based Online RL로 추론 품질 문제를 풀어냅니다.

전체 그림: 대화에서 음성까지

User 발화 → ASR → LLM(응답 + 스타일토큰) → AR(운율) → Diffusion(음색) → Vocoder → 음성 출력. RL은 AR만 개선한다.

핵심 1 — Cascaded Prompting: "두 단계에 다른 예시를 준다"

놀라운 실험적 발견: AR 모델과 Diffusion 모델에 서로 다른 granularity의 오디오 예시를 주면 성능이 크게 개선됩니다.

AR은 fine으로 섬세한 운율, Acoustic은 coarse로 안정된 음색. 역할에 맞춘 정보량 조절.
🎨 비유: 수채화와 유화 붓의 분업

세밀한 얼굴 표정을 그릴 때는 가는 세필붓(fine)이 필요합니다. 하지만 배경 하늘의 넓은 면적을 채색할 때 세필붓을 쓰면 ##브러시 자국이 얼룩지고 일관성이 떨어지죠. 대신 넓은 평붓(coarse)으로 한 번에 쓸면 고른 색면이 됩니다. TTS에서도 운율(표정)엔 fine을, 음색(배경)엔 coarse를 쓰는 것이 정답이었습니다.

Speaker Drift — 왜 granularity가 중요한가

Fine-grained prompt를 두 단계에 모두 쓰면 drift 발생. Acoustic에 coarse를 쓰면 일관성 확보.

핵심 2 — 추론 시 재랭킹 대신, 학습 중 자기개선

기존 방식은 "추론 시 여러 개 생성 → 점수 매겨 best 선택" (Posterior Sampling). 본 논문은 그 대신 "학습 중에 AI 스스로 보상을 보며 파라미터를 업데이트"(ICL-based Online RL)합니다.

N개(예:6개) 음성 생성 2. 각각 AES-CE로 점수 매김 3. 최고점 하나만 사용자에게 전달 단점: · 추론 시 N배 계산량 · 모델 자체는 개선되지 않음 · 여전히 6번 중 1번이 좋을 확률 · 실시간 대화엔 너무 느림 ✗ Inference cost ↑ 모델 품질 정체 ICL-based Online RL (제안) 학습 시점에: 1. 훈련 중 N개 생성 → 리워드 계산 2. 리워드로 AR 모델 자체를 업데이트 3. CTC loss로 대본 정렬 강제 추론 시점에: · 단 1번만 생성 (cheap!) · 모델이 이미 "best"를 만들도록 학습됨 · 추론 비용 ↓, 실시간 응답 가능 · 품질은 CMOS +7.1% 개선 ✓ Cheap inference + 고품질 모델 자체가 진화
Posterior는 추론 비용을 N배로 키우고, Online RL은 학습에 그 비용을 몰아 넣고 추론은 1회로 끝낸다.

Reward 설계 — AES-CE - 3 × CTC의 의미

AES는 "좋게 듣기", CTC는 "정확히 읽기". CTC의 3배 가중치로 reward hacking을 강하게 억제.

CTC Regularizer는 정말 필요한가?

CTC를 역전파하지 않으면 AES는 오르지만 대본 정확도는 개선되지 않음 = Reward Hacking의 직접 증거

Speaker Similarity — Granularity 선택의 검증

Coarse-grained prompt가 모든 스타일에서 speaker similarity 최상. Fine-grained이 중간, default가 최하.
🎙 비유: 더빙 스튜디오의 두 계층

애니메이션 더빙에서 감독은 "이 장면은 분노로, 저 장면은 슬픔으로"라는 세부 연기 지시(fine)를 매 컷마다 내립니다. 하지만 음향 엔지니어는 장면마다 마이크와 EQ를 바꾸지 않습니다. 그러면 같은 캐릭터 목소리가 매번 조금씩 달라져 몰입이 깨집니다. 엔지니어는 "이 캐릭터의 톤 그룹" 수준의 개괄 설정(coarse)만 유지합니다. TTS의 Cascaded Prompting은 정확히 이 원칙입니다.

Posterior Sampling은 "대회날 10번 뛰고 그중 가장 빠른 기록만 채택"하는 방식입니다. 순위는 좋아지지만 10배 힘들고 현실적으로 불가능합니다. Online RL은 "훈련 때 매일 10번씩 뛰며 기록을 측정하고 폼을 교정하되, 대회날엔 한 번만 뛰면 되는" 방식입니다. 훈련의 부담을 옮기고 대회(추론)에서는 절약합니다.

어디에 쓰일 수 있나?

실시간 감정 표현이 필요한 거의 모든 음성 인터페이스에 응용 가능

특히 강력한 적용처: 1) Meta의 AI 비서(Meta AI, WhatsApp 음성 기능 등)에 자연스러운 응답 음성 제공. 2) 캐릭터가 여러 감정을 표현해야 하는 게임 및 VR 환경. 3) 다양한 목소리와 톤이 필요한 오디오 드라마/팟캐스트 자동 생성.

"텍스트 지시 + 짧은 예시 음성 + AI 자기평가"의 3종 세트로,
감정 라벨링 없이도 자연스럽고 다채로운 AI 목소리를 만든다.

한계점 — 아직 완벽하지 않은 부분

1. 인하우스 데이터·인하우스 모델

Meta의 voice actor 데이터와 인하우스 TTS 시스템을 사용. 논문은 "공개 데이터로도 재현 가능할 것"이라고 말하지만 실제 오픈소스 코드/모델은 공개되지 않음. 외부 연구자가 같은 품질을 재현하기는 어렵다.

2. Subjective 평가에 크게 의존

AES-CE reward 자체가 사람 판단을 모방한 모델이고, 최종 평가도 CMOS(사람 비교 평가) 기반. 객관적 수치(예: Word Error Rate, F0 RMSE)는 공개되지 않아 일부 축에서의 개선이 숫자로 검증되지 않았다.

3. Human-in-the-Loop 병목

각 style token당 candidate audio 10개+ 생성 후 사람이 청취해 1개를 선별. 스타일 수가 수백 개로 확장되면 큐레이션 비용이 다시 병목이 된다. 완전 자동화된 prompt 선별 방식은 추후 과제.

4. KL 계수가 매우 크다 (β=50)

SFT baseline에서 크게 벗어나지 못하는 보수적 탐색. 더 공격적인 탐색(β 작게)에서 모델이 어떤 행동을 보이는지(과장된 감정 표현이나 품질 저하 등) 분석이 없음.

5. Ethics Statement 부재

Voice cloning 기술은 deepfake 음성, 사칭 사기 등 오남용 위험이 크다. 논문은 인하우스 voice actor data를 썼다고 하나, 실제 배포 시 consent mechanism, 워터마킹, misuse 방지 가이드에 대한 논의가 없음.

6. 평가 규모 제한

CVAD 평가는 스타일당 50샘플, rater 5명. 분산이 적지 않을 수 있으며, 스타일별 차별적 효과(예: "기쁨"에서 효과 크고 "슬픔"에서 효과 작음) 같은 breakdown은 제공되지 않는다.

한눈에 보는 요약

  • 문제: 감정 있는 대화형 TTS는 대규모 감정 라벨링 데이터가 필요 → 현실적으로 scale 안 됨
  • 아이디어 1 (Cascaded Prompting): LLM이 만든 style token + 사람이 큐레이션한 1개 예시 음성 = ICL 기반 single-shot 스타일 전이
  • 아이디어 2 (Prosody-Timbre Decoupling): AR(운율)에는 fine prompt, Diffusion(음색)에는 coarse prompt → speaker drift 방지
  • 아이디어 3 (ICL-based Online RL): 추론 비용을 학습으로 옮기고, AR 모델을 AES-CE 리워드로 직접 학습. 추론은 1회만
  • 아이디어 4 (CTC Regularizer): AES 단독 최적화는 reward hacking 유발. CTC loss를 3배 가중치로 결합해 대본 정렬 강제
  • 성과: ICL이 Zero-shot 대비 CVAD +79.6, GPT-4o 대비 +5.6. RL 추가로 CMOS +7.1% winrate
  • 가치: 실시간 AI 음성 비서·게임 NPC·오디오북 자동 생성 등 expressive TTS가 필요한 제품에 즉시 적용 가능
