TL;DR

  • 문제: 텍스트·이미지·오디오·비디오가 한꺼번에 등장하는 omnimodal 실세계 과제에서 기존 단일 omnimodal LLM(OLLM, 예: Gemini-3-Pro)은 OmniGAIA 벤치마크에서 62.5%에 머물고, 기존 orchestration 프레임워크(AOrchestra 등)는 모달리티 커버리지가 좁고 sub-agent 워크플로우가 선형·휴리스틱이라 비효율적이다.
  • 해결: ##modality-aware task decomposition, online sub-agent specialization, parallel sub-task execution##을 결합한 통일된 orchestration 메커니즘 Orchestra-o1을 제안. main agent가 의존성 그래프를 추론하고, 각 sub-task에 대해 backend(GPT-5/Gemini/Qwen3 등)와 tool(image/audio/video 분석, 웹 검색, page visit, code 실행)을 동적으로 매칭하며, 독립적인 sub-task들을 비동기 병렬로 실행한다. 또한 open-source main agent를 위한 ##decision-aligned GRPO (DA-GRPO)## 학습 레시피를 제안.
  • 왜 작동하는가: (i) 정보이론적으로 modality 별 specialized 처리는 단일 모델이 모든 modality를 압축하는 것보다 정보 손실이 작다 (Proposition 2), (ii) 독립 sub-task의 비동기 병렬 실행은 round-level latency를 max로 줄여 선형 실행 대비 최대 $K_t$배 speedup을 보장한다 (Proposition 1), (iii) DA-GRPO는 final-answer reward만 보는 GRPO와 달리 step-level 결정(delegate, 모델 선택, tool 선택, complete)을 expert trajectory와 rubric reward로 직접 정렬해 데이터 효율을 높인다.
  • 비유: 한 명의 천재가 모든 일을 떠맡는 "독박 작업자"(native OLLM) 또는 정해진 순서로 부하에게 일을 던지는 "관료적 매니저"(linear orchestration)와 달리, Orchestra-o1은 ##업무 의존성을 한눈에 파악하고, 각 부하의 전문성과 가격을 보며 동시에 일을 시키는 똑똑한 PM##에 가깝다.

1. Background — Omnimodal Agent의 두 갈래

1.1 Native Omnimodal Agents (OLLM 직접 사용)

Gemini-3-Pro, Qwen3-Omni, Ming-Omni, Baichuan-Omni 등 omnimodal LLM을 ReAct 스타일로 perception, reasoning, planning, tool-use를 동시에 수행하게 한다. 그러나 OLLM은 long-horizon reasoning, fine-grained cross-modal understanding에서 여전히 한계가 크고, OmniGAIA에서 강력한 proprietary Gemini-3-Pro조차 62.5%에 머문다. 한 모델이 모든 일을 다 하다보니 어느 것도 충분히 잘하지 못하는 문제.

1.2 Orchestration-based Agents (오케스트라식)

텍스트 기반 LLM이 main agent(=orchestrator)로 작동하고, perception/action을 specialized sub-agent + tool로 위임한다. 모듈성과 확장성이 좋지만, 기존 AOrchestra 등은 (a) modality 커버리지가 좁고 (b) sub-agent 워크플로우가 선형·휴리스틱이며 (c) backend/tool 선택이 정적이라는 한계가 있다.

1.3 본 논문의 방향

Orchestration-based agent 진영을 따르되, 세 가지 새로운 메커니즘을 도입: (1) modality-aware decomposition (어떤 modality 증거가 어떤 sub-task에 필요한지 명시), (2) online sub-agent specialization (round 단위로 backend/tool을 동적 매칭), (3) parallel sub-task execution (independent sub-task의 비동기 실행). 추가로 open-source main agent를 위한 DA-GRPO 학습 레시피를 제안한다.

2. Methodology — 문제 정의와 Orchestra-o1 프레임워크

2.1 문제 정의 (Section 3.1)

입력 $x = (q, \mathcal{M})$, 여기서 $q$는 자연어 질문이고 $\mathcal{M} = \{m_i\}_{i=1}^N$은 image/audio/video 등 modality 입력의 집합이다. 목표는 ground-truth $a^*$에 대해 task reward $R(\hat{a}, a^*)$를 최대화하는 답 $\hat{a}$를 생성하는 것.

System은 main agent $\pi_\theta$, sub-agent backend pool $\mathcal{B}$, tool ecosystem $\mathcal{T} = \mathcal{T}^{\text{perc}} \cup \mathcal{T}^{\text{act}}$로 구성. Round $t$에서 main agent는 다음 상태를 본다:

$$s_t = (q, \mathcal{M}, c_t, H_t, \mathcal{B}, \mathcal{T}) \tag{1}$$

여기서 $c_t$는 누적 context, $H_t$는 sub-task history. Main agent는 $y_t \in \{\texttt{delegate}, \texttt{complete}\}$ 중 하나를 출력. $\texttt{delegate}$이면 batch of $K_t$ sub-tasks를 생성한다:

$$\mathcal{U}_t = \{u_{t,j}\}_{j=1}^{K_t}, \quad u_{t,j} = (I_{t,j}, C_{t,j}, b_{t,j}, \mathcal{T}_{t,j}) \tag{2}$$

$I_{t,j}$는 sub-task instruction, $C_{t,j}$는 이전 round에서 넘긴 context, $b_{t,j} \in \mathcal{B}$는 선택된 backend, $\mathcal{T}_{t,j} \subseteq \mathcal{T}$는 할당된 tool subset.

직관적으로: main agent는 매 round마다 (a) 이 task를 어떻게 sub-task로 쪼갤지, (b) 각 sub-task에 어떤 모델·도구를 줄지, (c) 어떤 sub-task가 병렬 실행 가능한지를 모두 결정해야 한다.

2.2 Stochastic Orchestration Policy

$$y_t \sim \pi_\theta(\cdot \mid s_t), \quad y_t = (a_t, \xi_t), \quad a_t \in \{\texttt{delegate}, \texttt{complete}\} \tag{3}$$

최적화 목표는 latency·monetary 비용을 함께 고려:

$$\max_{\pi_\theta} \mathbb{E}_{\tau \sim \pi_\theta}\left[ R(\hat{a}, a^*) - \lambda_c \text{Cost}(\tau) - \lambda_l \text{Latency}(\tau) \right] \tag{4}$$

$\lambda_c, \lambda_l \geq 0$은 각각 비용·지연 페널티 계수. 이전 prompts에 비싸고 느린 모델을 무분별하게 호출하면 보상이 깎인다.

2.3 Flexible Agentic Backends — 비용-성능 매칭 (수식 5)

각 backend $b \in \mathcal{B}$는 capability vector + cost-latency profile로 표현:

$$\phi(b) = (\phi_b^{\text{txt}}, \phi_b^{\text{img}}, \phi_b^{\text{aud}}, \phi_b^{\text{vid}}, \phi_b^{\text{code}}, \kappa_b, \delta_b)$$

처음 5개는 modality별 능력 점수, $\kappa_b$는 단가, $\delta_b$는 예상 latency. Sub-task $u$의 requirement vector $r(u) = (r_u^{\text{txt}}, r_u^{\text{img}}, r_u^{\text{aud}}, r_u^{\text{vid}}, r_u^{\text{code}}) \in [0,1]$이 주어지면 main agent는 cost-aware matching을 수행:

$$b^*(u) = \arg\max_{b \in \mathcal{B}} \underbrace{\langle r(u), \phi_b \rangle}_{\text{capability match}} - \lambda_c \kappa_b \ell(u) - \lambda_l \delta_b \tag{5}$$

$\ell(u)$는 sub-task의 예상 토큰 수. 직관적으로: easy extraction/search는 싼 모델로, 어려운 omnimodal reasoning은 강한 backend로 라우팅한다. Figure 2의 "Model Price Table"에서 GPT-5, Gemini, Qwen3 등이 In/Out 단가별로 등록되어 있는 것이 이 매칭에 활용된다.

2.4 Unified Omnimodal Tool Ecosystem (수식 6, 7)

각 tool $g$는 capability vector $\psi(g) = (\psi_g^{\text{txt}}, \psi_g^{\text{img}}, \psi_g^{\text{aud}}, \psi_g^{\text{vid}}, \psi_g^{\text{web}}, \psi_g^{\text{code}}) \in \{0,1\}$로 표현. Sub-task $u$의 tool subset:

$$\mathcal{T}^*(u) = \{g \in \mathcal{T} : p_\theta(g \mid u, s_t) > \gamma\} \tag{6}$$

또는 sparse coverage 목적함수로:

$$\mathcal{T}^*(u) = \arg\max_{\mathcal{S} \subseteq \mathcal{T}}\left[ \langle r_T(u), \sum_{g \in \mathcal{S}} \psi(g) \rangle - \lambda_s |\mathcal{S}| \right] \tag{7}$$

$\lambda_s$는 sparsity 페널티 — 너무 많은 tool을 부르지 않게 한다. Sub-task $u$의 low-level executor는 $e(u) = (b^*(u), \mathcal{T}^*(u))$.

2.5 Modality-aware Task Decomposition (수식 9-12)

매 round 시작에서 main agent는 latent dependency graph $\mathcal{G}_t = (\mathcal{V}_t, \mathcal{E}_t)$를 induce한다 — sub-goal $v_i$가 $v_j$에 선행되어야 하면 directed edge $(v_i, v_j) \in \mathcal{E}_t$. 각 노드는 modality mask $\mu(v) \in \{0,1\}^4$ (text/image/audio/video 필요 여부)와 tool mask $\alpha(v) \in \{0,1\}^{|\mathcal{T}|}$를 가진다. Round $t$에서 ready set은:

$$\mathcal{R}_t = \{v \in \mathcal{V}_t \setminus C_t : \text{Pred}(v) \subseteq C_t\} \tag{9}$$

$C_t$는 완료된 sub-goal. 그 중 main agent는 budget $K_{\max}, B_t$ 하에서 batch $\mathcal{P}_t$를 선택:

$$\mathcal{P}_t = \arg\max_{\mathcal{P} \subseteq \mathcal{R}_t} \sum_{v \in \mathcal{P}} U_\theta(v \mid s_t) \quad \text{s.t. } |\mathcal{P}| \leq K_{\max}, \sum_{v \in \mathcal{P}} \text{cost}(v) \leq B_t \tag{10}$$

직관적으로: 의존성 그래프에서 "지금 당장 시작 가능한 노드"들을 찾고, 그 중 utility가 큰 것부터 budget이 허용하는 만큼 동시에 던진다.

2.6 Parallel Sub-task Execution과 Latency Advantage

Sub-agent trajectory는 ReAct 스타일:

$$\zeta_{t,j} = \{(\rho_{t,j}^{(\ell)}, a_{t,j}^{(\ell)}, o_{t,j}^{(\ell)})\}_{\ell=1}^{L_{t,j}}, \quad a_{t,j}^{(\ell)} \in \mathcal{T}_{t,j} \cup \{\texttt{Finish}\} \tag{13}$$

$\rho^{(\ell)}$ reasoning state, $a^{(\ell)}$ tool action, $o^{(\ell)}$ observation. 모든 sub-task가 independent하면 결합 분포가 factorize된다:

$$p(Z_t \mid \mathcal{U}_t, s_t) = \prod_{j=1}^{K_t} p(z_{t,j} \mid u_{t,j}, s_t), \quad Z_t = \texttt{AsyncExecute}(u_{t,1}, \ldots, u_{t,K_t}) \tag{15}$$

Proposition 1 (Round-level Latency Advantage)

$K_t \geq 2$ ready sub-task의 실행 시간이 $\delta_{t,1}, \ldots, \delta_{t,K_t} > 0$이고 conditional independent, $\delta_t^{\text{sync}} \geq 0$이 parallel 동기화 오버헤드일 때:

$$\text{Latency}^{\text{linear}}(t) = \sum_{j=1}^{K_t} \delta_{t,j}, \quad \text{Latency}^{\text{parallel}}(t) = \max_{1 \leq j \leq K_t} \delta_{t,j} + \delta_t^{\text{sync}} \tag{16}$$

Parallel이 linear보다 빠를 필요충분조건:

$$\delta_t^{\text{sync}} \leq \sum_{j=1}^{K_t} \delta_{t,j} - \max_{1 \leq j \leq K_t} \delta_{t,j} \tag{17}$$

이때 round-level speedup:

$$1 \leq S_t = \frac{\text{Latency}^{\text{linear}}(t)}{\text{Latency}^{\text{parallel}}(t)} \leq K_t \tag{18}$$

상한 $S_t = K_t$는 $\delta_t^{\text{sync}} = 0$이고 모든 sub-task runtime이 같을 때 달성. 직관적으로: 4개 독립 sub-task가 각 10초 걸리면 linear는 40초, parallel은 max 10초 + sync 약간 = 약 11초. 즉 비슷한 길이의 일이 많을수록 parallel 이득이 극대화된다.

2.7 Context Memory와 Iterative Refinement (수식 19-22)

Memory $H_t = \{h_1, \ldots, h_{m_t}\}$, 각 $h = (I, b, \mathcal{T}, \sigma, \eta, \omega)$. Round 후 update:

$$H_{t+1} \cup \{\text{Summarize}(u_{t,j}, z_{t,j})\}_{j=1}^{K_t} \tag{19}$$

Token budget $L_{\text{ctx}}$ 안에서 compressed context:

$$C_{t+1} = \arg\max_{C : |C| \leq L_{\text{ctx}}} \left[ I(C; q) + \sum_{h \in H_{t+1}} \omega(h) I(C; h) \right] \tag{20}$$

$I(\cdot;\cdot)$는 정보 relevance, $\omega(h)$는 최근/성공 evidence를 가중치로 우대. 중단 조건:

$$p_\theta^{\text{stop}}(s_t) = p_\theta(a_t = \texttt{complete} \mid s_t), \quad p_\theta^{\text{stop}}(s_t) > \tau_{\text{stop}} \tag{22}$$

2.8 Theoretical Advantage over Native Omnimodal Agents

Proposition 2 (Information Gain from Omnimodal Orchestration)

$Y$를 latent task answer, $\mathcal{M} = (M_1, \ldots, M_R)$를 R modality 소스. Native agent는 모든 modality를 단일 evidence variable $E_0 = f_0(q, \mathcal{M})$로 압축. Orchestration agent는 modality-aware sub-task를 dispatch해 $E_{\text{orch}} = (E_1, \ldots, E_R)$를 얻는다 ($E_r = f_r(q, M_r, C_r)$). 다음 가정 하에:

  • (i) main agent는 evidence loss 없이 aggregate
  • (ii) native evidence $E_0$가 modality별 component $(E_1^0, \ldots, E_R^0)$의 joint 정보를 upper bound
  • (iii) specialized 처리가 native 처리보다 every modality step에서 정보적으로 same-or-better, 적어도 한 modality에서 strict gain
$$I(Y; E_r \mid q, E_{그러면:

$$I(Y; E_{\text{orch}} \mid q) > I(Y; E_0 \mid q) \tag{24}$$

Bayes-optimal log-loss prediction에서 expected risk:

$$\mathcal{R}_{\log}(E_{\text{orch}}) < \mathcal{R}_{\log}(E_0) \tag{25}$$

직관적으로: 한 명이 모든 modality를 동시에 처리하면 어딘가에서 정보 손실이 일어나고, 전문가가 modality별로 처리하고 결과를 모으면 손실이 줄어든다 — "분업이 이득"이라는 직관을 정보이론으로 formalize한 것.

4. Training Recipe — DA-GRPO와 데이터 큐레이션 (Section 3.3)

4.1 Training Data Curation Pipeline

FineVideo, LongVideoBench, COCO 2017을 seed로 사용. 시작 seed set: $\mathcal{D}_0 = \{x_i = (q_i, \mathcal{M}_i, a_i, \mathcal{T}_i)\}_{i=1}^N$. 새로운 예제를 만들되 modality 파일은 그대로 유지. 큐레이션은 5단계:

  1. Seed Data 수집: FineVideo, LongVideoBench, COCO 2017 등.
  2. Rollout: Orchestra-o1(GPT-5)로 trajectory 수집. 각 trajectory를 annotated reasoning solution $r_i$ + difficulty level $\ell_i$로 변환.
  3. Anchor Facts 추출: LLM extractor가 $A_i = \{(f_{i,m}, \mu_{i,m}, e_{i,m})\}_{m=1}^{M_i}$ 생성. $f$는 anchor fact, $\mu$는 source modality, $e$는 supporting reasoning step. 이 anchor는 "valid한 rewrite는 반드시 보존해야 하는 사실".
  4. QA Rewrites: 5가지 전략 — pivot swapping, temporal shifting, numerical recombination, entity-sibling querying, multi-hop reordering. 형식적으로:
$$\tilde{x}_{i,k} \sim G_\omega(\cdot \mid x_i, A_i, z_{i,k}), \quad z_{i,k} \in \{A,B,C,D,E\} \tag{27}$$

제약: $\tilde{\mathcal{M}}_{i,k} = \mathcal{M}_i$ (modality 유지), $\tilde{\ell}_{i,k} = \ell_i$ (난이도 유지), $\tilde{\mathcal{T}}_{i,k} \subseteq \mathcal{T}_i$ (tool subset), $|\tilde{s}_{i,k} - s_i| \leq 2$ (reasoning step 차이 ±2). 쉬운 seed는 pivot swap/entity-sibling, 중간은 temporal/numerical, 어려운 seed는 numerical recomb/multi-hop reordering.

  1. Filter & Verify (4-stage cascade):
$$V(\tilde{x}_{i,k}) = \mathbb{1}\{\text{AnchorCov}(\tilde{q}_{i,k}, A_i) \geq 1\} \cdot \mathbb{1}\{\text{Sim}(q_i, \tilde{q}_{i,k}) \leq 0.85\} \cdot \mathbb{1}\{\text{Bypass}(\tilde{q}_{i,k}, \tilde{a}_{i,k}) = 0\} \cdot \mathbb{1}\{\text{NumCheck}(\tilde{r}_{i,k}, \tilde{a}_{i,k}) = 1\} \cdot \mathbb{1}\{\text{Judge}(\tilde{x}_{i,k}) = 1\} \tag{28}$$

각 게이트:

  • AnchorCov: anchor fact를 최소 1개 이상 포함
  • Sim: normalized lexical similarity $\leq 0.85$ (near-duplicate 제거)
  • Bypass: modality 없이도 답이 나오면 reject (modality-bypass test). "이미지·오디오 안 봐도 풀 수 있는 문제"는 omnimodal benchmark로 부적절
  • NumCheck: code execution이나 numeric answer는 Python sandbox로 검증
  • Judge: LLM judge가 factual consistency, difficulty preservation, peer-level duplication 확인

모든 LLM-based processor(extractor, rewriter, verification judge)는 Claude-Opus-4.6 사용. 결과: 300 seed → 1500 raw candidate → 1200 verified examples. 각 trajectory $\tau = \{(s_t, y_t^*, Z_t)\}_{t=1}^N$에서 N개의 decision-level example $(s_t, y_t^*)$를 만들어 dense supervision.

4.2 DA-GRPO (Decision-aligned Group Relative Policy Optimization)

왜 DA-GRPO인가: Standard GRPO는 same prompt에 대해 group of responses를 sampling하고 group 내 final-answer reward를 normalize해 advantage를 계산. 그러나 agent orchestration에서는 final-answer reward가 (a) sparse하고 (b) 전체 multi-agent system 실행 비용이 매우 비싸다.

DA-GRPO의 차별점: 매 main-agent decision을 현재 orchestration state에서 직접 평가. expert trajectory + rubric reward 사용.

각 prompt $s_i$에 대해 G개 candidate decision $\{y_{i,j}\}_{j=1}^G$를 샘플. 각 decision은 4차원 reward로 평가:

$$r_{i,j} = \alpha_1 r_{i,j}^{\text{format}} + \alpha_2 r_{i,j}^{\text{action}} + \alpha_3 r_{i,j}^{\text{tool}} + \alpha_4 r_{i,j}^{\text{decision}} \tag{29}$$
  • $r^{\text{format}}$: valid JSON 출력인가 (binary)
  • $r^{\text{action}}$: action이 valid arguments인가 (binary)
  • $r^{\text{tool}}$: tool/sub-task assignment가 reasonable한가 (graded, $[0,1]$)
  • $r^{\text{decision}}$: overall orchestration decision quality (graded, $[0,1]$)

가중치: $\alpha_1 = \alpha_2 = 0.1, \alpha_3 = 0.2, \alpha_4 = 0.6$. format/action은 초기에도 잘 되므로 작게, tool/decision에 큰 가중치. Reward model은 Claude-Haiku-4.5 한 번 호출로 4 차원을 모두 평가.

Expert decision은 "유일한 정답"이 아니라 reference — 다른 합리적 decomposition도 보상받고, $\texttt{complete}$ decision은 답이 ground-truth와 일치할 때 최고 점수.

Group advantage:

$$\hat{A}_{i,j} = \frac{r_{i,j} - \text{Mean}(\{r_{i,k}\}_{k=1}^G)}{\text{Std}(\{r_{i,k}\}_{k=1}^G) + \epsilon} \tag{30}$$

Clipped policy-gradient + KL regularizer:

$$\mathcal{L}_{\text{DA-GRPO}}(\theta) = -\mathbb{E}_{i,j}\left[ \min(\rho_{i,j}(\theta) \hat{A}_{i,j}, \text{clip}(\rho_{i,j}(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_{i,j}) - \beta D_{\text{KL}}(\pi_\theta(\cdot|s_i) \| \pi_{\text{ref}}(\cdot|s_i)) \right] \tag{31}$$

$\rho_{i,j}(\theta) = \pi_\theta(y_{i,j}|s_i)/\pi_{\text{old}}(y_{i,j}|s_i)$. 두 가지 이점: (1) 비싼 sub-agent trajectory를 매번 실행할 필요 없이 reconstructed state에서 offline scoring, (2) main agent core 책임(decompose/select tool/parallel schedule/stop)에 dense feedback.

6. 비판적 분석 (Critical Analysis)

강점

  • 이론과 실험의 결합: 단순히 "agent system이 잘 된다"가 아니라, parallel speedup(Prop 1)과 information gain(Prop 2)를 정보이론으로 정당화. AOrchestra 같은 휴리스틱 baselines보다 설계가 더 principled.
  • 비용 효율성: 정확도 1.82× 향상하면서 비용 0.60×. 이는 production 환경에서 매우 유의미 — 단순히 SOTA를 달성하는 게 아니라 cost-frontier를 밀어낸다.
  • DA-GRPO의 실용성: 비싼 multi-agent rollout을 매 RL step마다 안 해도 된다는 점이 critical. offline state reconstruction + 4-dim rubric scoring으로 dense supervision 가능.
  • Open-source ablation: Qwen3-8B (12.5 → 30.0) 진척이 명확히 framework + DA-GRPO 양쪽 contribution으로 나뉜다.

약점·의문

  • OmniGAIA 단일 벤치마크: 모든 main result가 OmniGAIA에 의존. 이 벤치마크의 distribution이 Orchestra-o1의 decomposition 패러다임에 우호적일 가능성. 다른 vision-language agent benchmark(GAIA, AgentBench 등)에서의 검증이 부재.
  • Capability vector $\phi(b), \psi(g), r(u)$의 추정: 수식 (5)(7)에서 capability score들이 어떻게 정해지는지 구체적이지 않다. 실제로는 prompt에 "이 모델은 X를 잘한다"고 적어주는 형태로 추정되는데, 이 prompt engineering의 robustness가 평가되지 않음.
  • $\delta_t^{\text{sync}}$의 실제 측정: Proposition 1의 latency advantage는 동기화 오버헤드가 작다는 가정에 의존. 실제로 cloud API 호출의 동시성 한계, rate limiting, 외부 web search latency 변동성 등은 분석되지 않았다.
  • 1200 verified examples: DA-GRPO 학습 데이터가 1200개라는 것은 매우 작다. 더 큰 scale에서 reward saturation이나 over-fitting 양상은 어떨지 분석이 부재.
  • LLM Judge 의존: Reward model이 Claude-Haiku-4.5. Haiku의 평가 편향이 학습된 정책에 그대로 전파될 수 있다. Reward hacking 가능성에 대한 분석이 없음.
  • Standard deviation 미보고: Table 1, 2 어디에도 std나 multiple seed 결과가 없다 — single-run 결과로 보인다.
  • Hard task에서의 절대 성능: open-source Orchestra-o1-8B의 Hard 정확도 26.9%, GPT-5 main agent도 56.4%로 ceiling이 멀다. dependency graph가 깊어지면 어디서 실패하는지 error analysis 부재.
  • Information gain proposition의 가정: Prop 2의 가정 (i) "main agent는 evidence loss 없이 aggregate"는 실제로는 context compression(수식 20)으로 어느 정도 손실 발생. 가정과 알고리즘의 entail이 정확하게 맞지 않는다.

제안

  • OmniGAIA 외에 GAIA-1, BrowseComp, AssistantBench 등에서의 cross-benchmark 검증.
  • Capability score를 prompt 외에 calibration set으로 학습하는 방법.
  • Hard task에서 dependency graph error의 taxonomy (e.g., wrong decomposition, premature stop, wrong backend matching).
  • DA-GRPO scale-up 실험 — 1200 → 12000 → 120000 examples로 scaling curve.

7. Key Takeaways (이 논문에서 가져갈 것)

  • Modality-aware decomposition + parallel execution + dynamic backend matching의 세 가지 디자인 축은 omnimodal agent design의 새로운 design pattern. 단일 OLLM의 한계가 보일 때 직접 채택할 가치가 있다.
  • DA-GRPO의 step-level rubric reward 디자인: format/action/tool/decision 4차원으로 weighted sum. format/action은 작게(0.1), 의미있는 결정에 큰 가중치(0.6)를 — 다른 agent RL 학습에도 응용 가능한 일반 원칙.
  • Cost-aware backend matching (수식 5): $\arg\max_b \langle r(u), \phi_b \rangle - \lambda_c \kappa_b \ell(u) - \lambda_l \delta_b$ — 능력·단가·latency를 한 식으로 — 직접 implementation 가능한 형태.
  • Modal-bypass test로 데이터 정제: omnimodal benchmark/training data를 만들 때 "modality 없이도 풀리는 문제"는 reject 해야 함. 이건 데이터 품질 관리에서 무시되기 쉬운데, 실제로 training signal을 흐린다.
  • Information gain perspective: 분업이 정보이론적으로 이득이라는 framing은, agent system을 hand-wave 하지 않고 분석하는 출발점을 제공한다. 다른 agent paper를 읽을 때 information bottleneck 관점으로 분석하면 inductive bias의 형태가 명확해진다.
  • 비용 frontier 이동: 정확도 ↑ + 비용 ↓ 동시 달성은 흔치 않다. parallelism + backend tier matching이 실무에 직접 적용 가능한 lever.
