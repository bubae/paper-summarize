AI가 세상을 이해하는 방식을 정리하다

"World Model"이라는 같은 단어가 분야마다 다른 의미였던 시대의 끝—능력 단계(L1/L2/L3) × 4가지 영역(물리/디지털/사회/과학)으로 통합 좌표계를 만든 서베이

"World model 연구는 더 큰 모델이 아니라,
학습한 법칙을 내재화하고 스스로 진화하는 시스템에서 미래가 갈린다."

왜 이 서베이가 중요한가

요즘 AI 분야에서는 "world model"이라는 말이 마법처럼 쓰입니다. Sora, DreamerV3, GraphCast, AlphaFold, A-Lab... 모두 "world model"을 만들었다고 하지만 실제로는 서로 다른 능력을 의미합니다.

비전 연구자는 영상 품질로 평가하고, 강화학습 연구자는 의사결정 성능으로, 사회 시뮬레이션 연구자는 약속이 잘 지켜지는지로 평가합니다. 그래서 같은 시스템이 한 분야에서는 SOTA, 다른 분야에서는 실패라는 평가를 받는 일이 흔합니다.

이 서베이는 400편 이상의 논문을 분석해 두 축의 통합 좌표계를 제안합니다: 능력 단계(L1/L2/L3)와 지배 법칙 영역(Physical/Digital/Social/Scientific). 이를 통해 모든 world model 연구를 하나의 격자에 올려놓고 비교할 수 있게 만듭니다.

배경 지식 — 차근차근 5가지 개념

① World Model이란 무엇인가

AI가 환경(세상)을 내적으로 시뮬레이션하는 모델입니다. 마치 사람이 "이렇게 행동하면 이런 결과가 나올 거야"라고 머릿속에서 미리 그려보는 것과 같습니다. 바둑 알파고가 다음 수를 두기 전에 수많은 미래를 시뮬레이션하는 것이 좋은 예시입니다.

내비게이션의 "예상 도착 시간"을 떠올려 보세요. 실제로 운전해보지 않아도 도로 상태, 신호, 우회경로를 시뮬레이션해서 결과를 알려줍니다. 이게 일종의 "교통 world model"입니다.

② 그런데 왜 분야마다 의미가 다를까?

비전 분야의 Sora는 "어떤 영상이 나올지"를 예측합니다. 강화학습 분야의 DreamerV3는 "어떤 행동을 하면 어떤 보상이 나올지"를 예측합니다. AlphaFold는 "이 단백질이 어떤 모양으로 접힐지"를 예측합니다.

이들은 모두 "world model"이라 불리지만, 실제로 모델이 보장하는 능력 수준도, 다루는 법칙도 전혀 다릅니다. 그래서 같은 단어로 묶기 어렵죠.

③ 능력 단계 (L1/L2/L3)란?

이 서베이는 능력을 세 단계로 나눕니다:

  • L1 (Predictor): 한 발자국 앞의 미래를 잘 맞춤. "다음 프레임은 이런 모습"
  • L2 (Simulator): 여러 발자국 앞까지 일관성 있게 시뮬레이션. "이 행동을 30번 반복하면 이런 결과"
  • L3 (Evolver): 예측이 틀렸을 때 모델 자체를 갱신. "현실과 안 맞으니 모델을 다시 만들자"
L1은 "오늘 날씨를 잘 맞추는 기상캐스터", L2는 "한 달 뒤까지 일관성 있게 예보하는 기상 모델", L3는 "예보가 틀린 후 모델을 자가 진단해 다음 시즌에 더 잘 맞추는 시스템"입니다.

④ 4가지 지배 법칙 영역 (Regime)

모델이 따라야 할 규칙의 종류에 따라 4가지로 나눕니다:

  • Physical (물리): 충돌, 중력, 보존 법칙. 두 물체가 서로 통과해선 안 됨.
  • Digital (디지털): API, 코드, UI 상태. 존재하지 않는 함수를 호출해선 안 됨.
  • Social (사회): 신념, 약속, 규범. 약속을 어기면 안 됨.
  • Scientific (과학): 인과 메커니즘, 증거 사슬. 분자가 보존 법칙을 위반하면 안 됨.

⑤ 두 축이 만든 격자

이 두 축을 곱하면 3 × 4 = 12개의 셀이 만들어지고, 모든 world model 연구가 한 셀에 매핑됩니다. 어떤 시스템이 어느 셀에서 강한지 한눈에 비교 가능해지는 거죠.

음식점 평가에 비유하면, "맛있다"라는 한 단어로는 비교가 어렵습니다. 하지만 "장르(한식/일식/중식/양식) × 등급(미슐랭 1성/2성/3성)"으로 정리하면 명확해지죠. 본 서베이는 world model 연구를 그렇게 정리합니다.

핵심 아이디어 — 능력 단계의 boundary 조건

L1에서 L2로 올라가려면 충족해야 할 3가지

조건 1
Long-horizon Coherence

여러 step을 합성해도 즉시 무너지지 않고 유의미한 trajectory가 나와야 한다. 한 step씩 잘 맞춰도 누적 오차로 결국 망가지면 L1.

조건 2
Intervention Sensitivity

"행동 A vs 행동 B"를 가정했을 때 결과가 의미 있게 달라져야 한다. action을 바꿔도 똑같은 trajectory가 나오면 planning에 무용지물.

조건 3
Constraint Consistency

생성된 미래가 해당 영역의 규칙을 위반하지 않아야 한다. 물리에선 보존 법칙, 디지털에선 API 계약, 사회에선 약속.

L2에서 L3로 올라가려면 — 모델 자체가 진화해야 함

  • Evidence-grounded diagnosis: 실패의 원인을 재현 가능한 증거로 추적
  • Persistent asset update: 일회성 패치가 아니라 재사용 가능한 자산 (룰, 테스트, 스킬)으로 적립
  • Governed validation: 새 업데이트가 회귀 테스트를 통과해야 기본값으로 활성화
L2와 L3의 차이는 "Excel로 시뮬레이션하는 회계사"(L2)와 "결산 후 회계 규정 자체를 개정해 다음 분기에 적용하는 회계감독원"(L3)의 차이입니다.

L3 진화 루프 (4단계)

대표 시스템 — 영역별 현황

L3가 가장 성숙한 분야: 자율 과학 (Scientific)

  • A-Lab: 로봇 팔 3개로 자율 화학 합성. 17일 동안 353번 실험, 36개 화합물 발견.
  • CAMEO: 싱크로트론 빔라인에서 베이지안 능동 학습. 새로운 위상-변화 메모리 재료 발견.
  • BacterAI: 사전 생물학 지식 0에서 미생물 대사를 자율 학습.
  • AI Scientist v2: ML 학습 자동 설계 → 실험 → 논문 작성. ICLR 워크숍 피어 리뷰 통과.

L3 부분 성숙: 디지털 (Digital)

  • FunSearch (Romera-Paredes 2024): cap set 문제와 bin-packing의 새 해법 자동 발견.
  • AlphaEvolve (Novikov 2025): Strassen 행렬 곱셈 알고리즘을 56년 만에 개선.
  • SWE-agent: 실제 GitHub 이슈를 자동 수정.

L3 emerging: 물리 (Physical)

  • AdaptSim: 시뮬레이터 파라미터를 실데이터로 반복 갱신.
  • Self-Modeling: 로봇이 자기 morphology가 손상되었을 때 egocentric vision으로 자기 모델을 갱신해 운동 회복.

L3 aspirational: 사회 (Social)

가장 어려운 영역. 사회 실험은 윤리 제약이 크고, 실패 원인 귀속이 본질적으로 모호. Evolving Constitutions (Kumar 2026): 유전 프로그래밍으로 다중 에이전트 거버넌스 규칙을 진화 — 사람이 만든 규칙보다 +123% 향상.

의사결정 중심 평가 — 두 핵심 메트릭

① Action Success Rate (ASR)

"이 world model로 학습한 정책이 실제 환경에서 task를 푸는 비율". 영상 품질, FID 같은 generative metric은 의사결정 능력과 약한 상관 — 진짜 측정해야 할 건 downstream success.

② Counterfactual Outcome Deviation (COD)

"단 한 step에서 다른 행동을 하면 결과가 얼마나 달라지는가". COD가 낮으면 모델이 행동 변화에 반응하지 않음—planning에 쓸모 없다.

예전 평가는 "그림이 얼마나 사실적인가" (FID)였습니다. 새 평가는 "이 그림 보고 행동을 결정한 사람이 실제로 성공하는가" (ASR), "그림 속 인물이 다른 행동을 하면 결과 그림이 정말 달라지는가" (COD)입니다. 사실성이 아니라 의사결정 가능성을 봅니다.

엔지니어링 3원칙

원칙 1
Separate learned from enforced

충돌 검사, 상태 기계 검증, 회귀 테스트는 추론 시점에 강제—training loss로는 zero-violation 못 만듦.

원칙 2
Instrument before you iterate

로깅·재생·실패 귀속 인프라를 시스템 시작부터 내장. 재생 없으면 L3 revision은 "anecdotal & ungovernable".

원칙 3
Match representation to planner's query

플래너가 필요로 하는 변수(자유 공간, 권한 상태, 반응 속도 등)를 노출 못하면 아무리 사실적이어도 더 단순한 표현보다 못함.

한계 및 주의점

① 분류의 모호함

"Long-horizon coherence가 무너졌다"의 정량 기준이 영역마다 다릅니다. 본 서베이도 "schematic, not metric"이라며 인정하지만, 실용성이 약화될 위험.

② L3 maturity 평가의 비대칭

Scientific을 "established"로 분류하지만 실제로는 A-Lab 등 한두 flagship에 의존. cross-domain reproduction이 부족해 일반화에 의문.

③ 4 영역 외의 사각지대

경제·금융·법률 영역은 명시적 처리가 부족합니다. multi-regime 시스템 (자율주행 = physical + social)에 대한 분석도 짧음.

④ 능력 단계의 strict containment 가정

실제로는 weak L1 + 강한 L3 (예: closed-loop autonomous lab)도 가능. L1→L2→L3가 반드시 순차적으로 쌓여야 한다는 묵시적 가정이 약점일 수 있음.

실무자/연구자가 가져갈 인사이트

  1. "World model"이라 부르기 전에 좌표를 명시: 어느 능력 단계, 어느 영역인지 합의해야 비교가 가능.
  2. L1→L2 elevation의 진짜 시험은 3가지 boundary: 한 step 정확도 만으론 부족. coherence + sensitivity + consistency를 동시에.
  3. L3 = "online learning" 아님: 단순 fine-tuning이 아니라 model의 hypothesis space evolution. persistent reusable asset이 핵심.
  4. Decision-centric metric 도입: 자체 평가 시 FID 대신 ASR + COD를 측정.
  5. Code as world model이라는 새 흐름: 디지털 영역에서 executable code가 latent dynamics보다 검증 가능.
  6. Harness가 capability boundary의 일부: 모델만 개선하지 말고 도구·메모리·피드백 루프 같은 harness 설계도 함께.
