TL;DR

  • 문제: 진정한 generalist robot은 ##단일 VLA 모델##만으로 이루어지지 않는다. ##데이터 수집부터 실세계 deployment까지의 전체 robot learning stack##이 함께 설계되어야 한다. 현재 분야의 병목은 (1) teleoperation/UMI 데이터의 한계 (조작자 불편, 정밀도 부족, embodiment gap), (2) generalist VLM backbone이 manipulation에 최적화되지 않음, (3) imitation learning의 last-mile dexterity 부족 + reward/value model 없는 RL의 부재, (4) 고주파 servo 제어와 backbone latency 사이의 mismatch.
  • 해결: ##HyVLA-0.5## -- end-to-end robot learning stack. (a) 데이터: ##fingertip UMI + motion-capture cage##로 sub-millimeter 정밀도의 ##10K-hour ego-centric 데이터셋 (Hy-UMI-10K)## 수집. (b) 모델: ##Hy-Embodied-0.5 4B MoT VLM backbone## + ##flow-matching action expert## + ##compact memory encoder## + ##delta-chunk action representation##. (c) Post-training: ##Track-A (intra-embodiment) / Track-B (UMI-only cross-embodiment)## 두 트랙. (d) RL: ##FlowPRO (RPRO loss)## -- reward-free / critic-free Proximalized Preference Optimization. (e) Deployment: ##async inference + 큐빅 Bézier C¹-continuous chunk stitching##.
  • 왜 작동하는가: ##각 컴포넌트가 서로 독립된 병목을 해결##. (1) 고정밀 UMI 데이터가 last-cm/sub-mm precision의 학습 신호 제공. (2) Embodied-native backbone(Hy-Embodied-0.5-MoT)이 generic VLM 대비 더 빠르게 수렴 + 더 강한 spatial prior. (3) Delta-chunk + rel-EEF representation이 ##embodiment kinematics를 deployment로 분리##해서 cross-embodiment transfer 가능. (4) RPRO의 ##contrastive gradient cancellation + symmetric proximal regularizer##가 plain DPO의 reward-hacking을 막으면서 SFT 데이터와 동일 loss로 안전 co-training. (5) Async + Bézier가 backbone latency를 숨기고 chunk boundary jitter를 제거.
  • 비유: 좋은 셰프 한 명을 만드는 것이 아니라, ##식재료 수급(데이터) → 조리 도구(모델 아키텍처) → 레시피 개발(SFT) → 시식 후 보완(RL) → 서빙 시스템(deployment)##을 모두 갖춘 ##레스토랑 운영 체계## 자체를 만든 것. VLA 모델이 "셰프"라면 HyVLA-0.5는 "레스토랑 전체"다.

1. Introduction

최근 1년간 VLA(Vision-Language-Action) 모델은 폭발적으로 증가하며 차세대 ##embodied foundation model##의 후보로 부상했다. 그러나 진정한 generalist robot이 ##단일 모델 isolation##에서 등장하기는 어렵다. 데이터 수집부터 실세계 deployment까지 robust한 ##full robot learning stack##이 필요하다.

본 보고서가 다루는 ##세 가지 결합된 병목##:

① 데이터 측면

  • Teleoperation: master-slave 인터페이스가 조작자에게 부자연스러운 자세 강요, ##haptic feedback 부재##, delicate manipulation 불가
  • Hand-held UMI (Chi et al., 2024): 인간 데이터의 풍부한 behavioral diversity 제공하지만 ##coarse한 action label##, ##SLAM 의존 localization은 cumbersome handheld rig 필요##, fingertip-level force 미캡처
  • Cross-embodiment gap: human/robot motion space의 embodiment gap, dynamics/actuation의 control gap, egocentric/wrist-mounted camera의 perception gap을 모두 다뤄야 함

② 아키텍처/학습 측면

  • 초기 VLA들은 ##autoregressive discrete action token## 기반 → execution speed와 control precision 양쪽 모두 제한
  • 최근 VLM + flow-matching action expert ##coupling## 방식 (π₀ 류)이 등장했으나 ##foundational visual backbone이 robotic control용으로 명시적 engineering이 안 됨## -- generalist visual representation과 dense spatiotemporal reasoning 사이 gap 존재
  • Imitation learning만으로는 ##last-mile dexterity## 도달 어려움, continuous control RL은 ##brittle reward model이나 value network## 의존

③ Deployment 측면

고주파(50 Hz) closed-loop control을 실제 hardware에서 서빙하는 것은 first-class design target으로 거의 다뤄지지 않았다. 잘 학습된 policy도 servo period 내에 실행되지 않으면 무용지물.

해결책: ##HyVLA-0.5##는 단일 모델이 아니라 ##데이터+모델+RL+deployment의 4-stage 통합 pipeline##이다. 각 단계가 별개의 병목을 다루지만 모두 같은 목표를 공유 -- ##plat-agnostic policy interface를 보존하면서 embodiment-specific 차이를 학습 코어 외부에서 흡수##하는 것.

2. Model Architecture

HyVLA-0.5는 ##VLA paradigm##을 따른다 -- pre-trained VLM이 broad semantic perception을 공급하고, 전용 action 모듈이 multi-modal context를 low-level robot control로 번역. 3 components:

구성역할
BackboneHy-Embodied-0.5 (4B MoT VLM, modality-adaptive 계산 + native-resolution image encoding)
Action ExpertConditional flow matching로 continuous action chunk 생성, dual-tower routing (VLM tower + action expert tower) + shared self-attention
Compact Memory EncoderImage encoder를 video encoder로 확장, interleaved temporal-spatial attention으로 multi-frame history 압축

2.1 Problem Formulation

Manipulation을 ##goal-conditioned, chunk-level control problem##으로 정형화. Step $t$에서 policy가 multi-modal observation $\mathbf{o}_t$를 소비하고 future action chunk $\mathbf{A}_t$ 예측:

$$\mathbf{o}_t = (\mathcal{I}_t, \ell, \mathbf{s}_t), \quad \mathcal{I}_t = \{\mathbf{I}^v_{t-k}\}_{v=1:n}^{k=0:K-1}, \quad \mathbf{A}_t = (\mathbf{a}_t, \mathbf{a}_{t+1}, \ldots, \mathbf{a}_{t+H-1}) \tag{1}$$

여기서 $\mathcal{I}_t$는 multi-view multi-frame visual stream (n cameras × K frames), $\ell$은 language instruction, $\mathbf{s}_t$는 proprioceptive state, $H$는 action chunk horizon. ##Conditional distribution $p(\mathbf{A}_t | \mathbf{o}_t)$를 모델링##.

End-effector-frame (EEF) Representation (핵심 디자인):

  • Proprioceptive state $\mathbf{s}_t$와 action $\mathbf{a}_{t'}$를 모두 ##end-effector frame##에서 표현 -- ##embodiment-agnostic##
  • 각 controlled arm: pose는 3-D Cartesian translation (xyz) + 6-D continuous rotation + 1-D normalized gripper command, ##총 $\mathbb{R}^{10}$ per arm##
  • Proprioceptive state $\mathbf{s}_t$는 embodiment root 기준 EEF, ##각 future action $\mathbf{a}_{t'}$는 $\mathbf{s}_t$를 reference frame으로 한 delta-chunk in relative EEF##
  • 이것이 ##policy가 robot-specific joint kinematics로부터 decouple##되어 cross-embodiment transfer 가능한 이유

Optional Co-Training Tasks: VLM의 foundational vision-language reasoning과 spatial grounding을 보존하기 위해 next-token prediction auxiliary task 통합 -- $\mathcal{D}_{\text{ct}} = \mathcal{D}_{\text{VQA}} \cup \mathcal{D}_{\text{2D}} \cup \mathcal{D}_{\text{3D}}$ (VQA + 2D coords + 3D geometric params).

2.2 Hy-Embodied: Modality-Adaptive Computing Backbone

  • Backbone: Hy-Embodied-0.5-MoT (Tencent 자체 4B parameter compact embodied VLM, edge deployment 최적화)
  • Native-resolution visual encoding: ##Hy-ViT 2.0## -- arbitrary input resolution 허용 native-resolution ViT, 더 큰 internal teacher로부터 distill
  • Modality-adaptive Computation via MoT: 시각 token은 ##duplicated vision-specific parameter set##, 언어 token은 ##original language parameters##로 처리. ##Cross-modal interaction은 shared self-attention layer로만 한정##. 시각/텍스트 parameter independent update.
  • Bidirectional attention strictly among individual image의 visual token, language token에는 standard causal attention

Action을 language-like token으로 discretize하지 않고 ##conditional flow matching## (Lipman et al., 2023)으로 $p(\mathbf{A}_t | \mathbf{o}_t)$ 직접 모델링.

Dual-tower Routing: MoT backbone 위에 ##VLM tower## (understanding) + ##action-expert tower## (generation) 분리. VLM tower는 modality-adaptive params, action expert는 ##projected robot state + noisy action token## $[\mathbf{s}_t, \mathbf{A}^\tau_t]$를 받아 continuous action velocity field 생성. 두 tower는 ##shared self-attention##으로 상호작용.

Block-wise Causal Attention: Token을 3 blocks $[\mathcal{I}_t, \ell], [\mathbf{s}_t], [\mathbf{a}^\tau_{t,0}, \ldots, \mathbf{a}^\tau_{t,H-1}]$로 분할. 각 block 내부는 bidirectional, ##block 간은 strictly causal##. Perception block은 robotics-specific block을 attend 못함 → VLM pre-training 분포 shift 최소화. State block은 keys/values 캐싱. Noisy-action block은 full prefix attend.

Flow-matching Objective:

$$\mathbf{A}^\tau_t = \tau \mathbf{A}_t + (1-\tau)\epsilon, \quad \epsilon \sim \mathcal{N}(0, \mathbf{I})$$ $$\mathcal{L}_{\text{fm}}(\theta) = \mathbb{E}_{p(\mathbf{A}_t|\mathbf{o}_t), q(\mathbf{A}^\tau_t|\mathbf{A}_t)}\left[\|v_\theta(\mathbf{A}^\tau_t, \mathbf{o}_t) - (\epsilon - \mathbf{A}_t)\|_2^2\right] \tag{3}$$

여기서 $v_\theta$는 noise → target action 방향의 velocity field. Flow timestep $\tau \in [0, 1]$은 ##Beta distribution (high-noise regime skewed)##에서 sampling -- denoising 후반의 더 정보적인 단계 강조. 전체 objective: $\mathcal{L}(\theta) = \mathcal{L}_{\text{fm}}(\theta) + \lambda_{\text{ntp}} \mathcal{L}_{\text{ntp}}(\theta)$.

Inference: Forward Euler update $\mathbf{A}^{\tau+\delta}_t = \mathbf{A}^\tau_t + \delta v_\theta(\mathbf{A}^\tau_t, \mathbf{o}_t)$, ##10 integration steps## ($\delta = 0.1$). Conditioning observation prefix $\mathbf{o}_t$의 keys/values는 initial forward pass에서 캐싱되어 후속 steps에서 ##action token만 recompute## → 계산 절감.

2.4 Compact Memory Encoder with Temporal-Spatial Attention

$K$-frame multi-view history $\mathcal{I}_t$를 모두 backbone에 forward하면 ##visual token count가 K배##. 대신 image encoder를 ##video encoder로 확장##해 backbone 도달 전에 temporal dimension 압축.

Factorized Temporal-spatial Attention (Pi-MEM 따름):

  • Standard ViT의 patchify-then-attend 구조 유지, ##매 $L$ layer마다 temporal pass 1회 삽입##
  • 해당 layer에 ##fixed sinusoidal temporal encoding $e(k)$## ($e(0) = \mathbf{0}$) 추가, ViT block의 ##same QKV와 output projection $W_O$ 재사용##
  • 2-pass factorization:
$$\hat{V}_p = \text{CausalAttn}(Q_p, K_p, V_p) \quad \text{over } K \text{ frames at each patch } p \tag{4}$$ $$\tilde{X}_k = W_O \text{Attn}(Q_k, K_k, \hat{V}_k) \quad \text{over } n \text{ patches within frame } k \tag{5}$$

Temporal pass는 ##causal attention## (현재/과거만, on-robot perception streaming에 부합), spatial pass는 frame 내부 standard bidirectional. ##Cost: $\mathcal{O}(n^2 K^2)$ → $\mathcal{O}(Kn^2 + nK^2)$##.

Token-count-preserving Compression: Video encoder upper layer에서 ##past frame patch representation을 discard##, ##current-frame token만 backbone에 forward##. Temporal attention이 historical context를 current-frame representation에 "bake"한 상태.

Parameter-free, Transfer-friendly Design: Single-image Hy-ViT 2.0 대비 ##새 learnable parameter 0개##. Two pass 모두 같은 QKV/$W_O$ projection 재사용, temporal encoding은 fixed sinusoid + learned table 아님. ##K=1일 때 causal temporal attention은 identity, $e(0) = \mathbf{0}$이라 input unchanged## → 각 augmented block이 정확히 pre-trained ViT block으로 reduce. ##Memory-augmented backbone이 Hy-Embodied-0.5 weight로 직접 초기화 가능, single-frame encoder가 special case.##

4. Reinforcement Learning Post-Training (FlowPRO)

SFT 이후 ##failure-driven post-training##으로 deployment 성능 추가 개선. ##FlowPRO recipe## (Wu et al., 2026; arXiv:2606.05468) 따름.

4.1 Design Principles

Real-robot post-training의 3 family와 한계:

Family한계
SFT/DAggerFailure signal을 weak하게만 활용 -- vanilla SFT는 무시, DAgger는 expert 호출 trigger로만 사용
Reward/value-based RLReliable reward/value/advantage model 필요, contact-rich manipulation에서 dense reward 어려움
Preference-based RL (DPO 등)Reward design bypass하지만 plain DPO는 ##reward-hacking failure mode## (정책이 $a^w$와 $a^l$ 둘 다로부터 drift)

FlowPRO 3 design principles:

  1. (P1) Exploit failures directly: Negative trajectory를 discard 또는 re-label하지 않고, ##paired positive correction에 대한 per-state, per-chunk contrastive signal로 action-generation loss에 직접 feed##
  2. (P2) Avoid reward and critic models entirely: Frozen reference policy와 current policy의 ##flow-matching log-likelihood proxy##만으로 closed-form training signal. Dense-reward bottleneck 우회.
  3. (P3) Anchor the implicit reward: ##Symmetric proximal regularizer##가 implicit reward의 absolute magnitude exploding 방지 → plain-DPO의 reward-hacking 구조적으로 금지

4.2 Method: RPRO Loss

SFT-pretrained HyVLA-0.5 base policy 위에서 ##iterative offline-RL loop##. 각 round 3 step:

  1. Teleoperated intervention-and-rollback으로 on-robot preference pair 수집
  2. Sparse trajectory-level correction → ##Smooth Interpolation via cubic Bézier##로 dense per-state preference tuple 생성
  3. RPRO loss를 mixed batch (new pairs + historical pairs + SFT data)에 적용

Previous round의 policy가 다음 round의 reference $\pi_{\text{ref}}$.

RPRO loss 유도:

State $s = (o, l)$, velocity field $v_\theta(a_t, t | s)$가 Gaussian noise $\epsilon$를 action chunk $a$로 transport. Linear interpolant $a_t = (1-t)\epsilon + ta$, conditional velocity $u(a_t | a) := a - \epsilon$. Per-sample flow-matching regression loss:

$$\ell_\theta(s, a) = \mathbb{E}_{t \sim \mathcal{U}[0,1], \epsilon \sim \mathcal{N}(0,I)}\left[\|v_\theta(a_t, t | s) - u(a_t | a)\|^2\right] \tag{6}$$

RPRO의 ##implicit-reward proxy##:

$$r_\theta(s, a) = \frac{\beta}{2}(\ell_{\text{ref}}(s, a) - \ell_\theta(s, a)) \tag{7}$$

이를 PRO pairwise objective에 대입한 ##flow-matching-adapted PRO loss##:

$$\mathcal{L}_{\text{PRO}}(\theta) = -\mathbb{E}_{(s, a^w, a^l) \sim \mathcal{D}}\Big[\underbrace{\log\sigma(r_\theta(s, a^w) - r_\theta(s, a^l))}_{\mathcal{L}_{\text{con}}: \text{ contrastive optimizer}} + \underbrace{\sum_{a \in \{a^w, a^l\}} \tfrac{1}{2}[\log\sigma(r_\theta(s, a)) + \log\sigma(-r_\theta(s, a))]}_{\mathcal{L}_{\text{reg}}: \text{ proximal regularizer}}\Big] \tag{8}$$

$\mathcal{L}_{\text{reg}}$는 ##$r_\theta(s, a) = 0$에서 minimum, $|r_\theta(s, a)|$에 따라 symmetric하게 grow##. Implicit reward 절댓값을 anchor → plain Flow-DPO의 reward-hacking 구조적 봉쇄.

최종 objective (base-policy 성능 보존 + $a^w$ direct regression):

$$\mathcal{L}_{\text{RPRO}}(\theta) = \lambda_{\text{PRO}} \mathcal{L}_{\text{PRO}}(\theta) + \lambda_{\text{SFT}} \mathcal{L}_{\text{SFT}}(\theta), \quad \mathcal{L}_{\text{SFT}}(\theta) = \mathbb{E}_{(s, a^w) \sim \mathcal{D}}[\ell_\theta(s, a^w)] \tag{9}$$

Contrastive Gradient Cancellation (핵심 side-property): ##$a^w = a^l$일 때 $\nabla_\theta \mathcal{L}_{\text{con}} = 0$##, $\nabla_\theta \mathcal{L}_{\text{reg}}$와 $\nabla_\theta \mathcal{L}_{\text{SFT}}$만 active. ##이 덕분에 SFT-style demonstration을 동일 RPRO loss로 안전하게 batch 내 routing 가능##. 직관적으로 말하면, "선호되는 행동과 거부되는 행동이 같은 sample"은 contrastive signal을 만들지 않고 standard SFT처럼 작동.

4.3 Data Collection: Intervention-and-Rollback

Teleoperated intervention-and-rollback pipeline (Fig. 5)으로 paired $(\tau^w, \tau^l)$ 수집:

  1. Current policy rollout 중 erroneous/dangerous action 관찰 시 operator intervene
  2. System이 ##earlier state $s_{t-\Delta}$로 rewind##, executed segment를 ##negative trajectory $\tau^l$##로 기록
  3. $t-\Delta$ observation을 visual reference로 retrieve (환경 변화/scene reset 대비)
  4. Operator의 ##corrective demonstration from $s_{t-\Delta}$를 positive trajectory $\tau^w$##로 기록

Single operator action으로 ##same initial state를 공유하는 naturally paired $(\tau^w, \tau^l)$## 산출. $\Delta$를 다양화하여 per-pair starting state 분산.

Smooth Interpolation and Batch Mixing

$\tau^w$와 $\tau^l$는 $s_{t-\Delta}$ 이후 diverge하므로, dense per-state tuple $(s, a^w, a^l)$을 위해 Smooth Interpolation:

  • State $M$ on $\tau^l$에 대해 weighted distance metric으로 closest point $M'$ on $\tau^w$ 찾기
  • $\tau^w$로의 transition point $J$를 거쳐 $M \to J \to N'$ (= $\tau^w$ 종료) 합성: ##cubic Bézier (positions) + Slerp (orientations) + linear interp (gripper)##
  • $\tau^w$ states 또는 $\mathcal{D}_{\text{SFT}}$ states에 대해서는 $a^w = a^l$ 설정 → contrastive gradient cancellation으로 SFT-like sample로 작용

Batch composition schedule:

  • Round-$k$ pairs $\mathcal{D}^k_{\text{pref}}$, historical pool $\mathcal{D}^{
  • $k=1$: 80/20 ($\mathcal{D}^k_{\text{pref}} / \mathcal{D}_{\text{SFT}}$)
  • $k \geq 2$: ##70/15/15## ($\mathcal{D}^k_{\text{pref}} / \mathcal{D}^{

이 schedule이 newest informative failure에 가중치, 이전 corrected sample을 replay하여 regression 방지, non-trivial SFT share로 base capability anchor.

FlowPRO Hyperparameters (Appendix C):

  • Iterations $k \in \{1, 2, 3\}$, round당 25,000 step, 총 75,000 step
  • Global batch 20 (5 samples/GPU), AdamW LR $1 \times 10^{-5}$ → $2.5 \times 10^{-6}$ cosine, 1K warmup, 15K decay
  • Distance metric: $d(M, M') = \|p_M - p_{M'}\|_2 + 0.5 \cdot d_{\text{geo}}(R_M, R_{M'}) + 0.2 \cdot |g_M - g_{M'}|$ ($p$ = EE position, $d_{\text{geo}}$ = SO(3) geodesic, $g$ = normalized gripper width)
  • Stage-2 checkpoint = $\theta$ 초기화 + frozen reference $\theta_{\text{ref}}$
  • Data scale: $\leq O(10^2)$ preference pair / task, X-Trainer rollout만

5. Deployment

3가지 runtime issue 처리:

  1. End-effector delta chunk를 heterogeneous robot platform에 매핑
  2. VLA prediction을 robot control rate에 서빙 (backbone slower than servo loop)
  3. Independently predicted chunk → smooth motion stitching

5.1 Embodiment-Agnostic Platform Mapping

Policy outputs ##20-dim dual-arm action chunk## (10 dim/EE: 3-D Cartesian + first 2 rows of SO(3) rotation matrix + 1-D gripper). Action은 chunk 시작의 EE pose 기준 ##relative SE(3) prediction##.

Deployment에서 embodiment-specific kinematics 처리:

  • Relative EE prediction × current absolute pose → world-frame target
  • Inverse Kinematics(IK)로 target robot의 joint command 산출
  • ##Track A (intra-embodiment)##: world frame이 deployment와 data collection에서 동일

Track B (cross-embodiment): 데이터 수집과 deployment가 다른 embodiment, 두 embodiment type별 매핑:

Fixed-base arms (예: JAKA K1):

$$^WT_{G_{t+k}} = ^WT_{G_t} \cdot ^{G_t}T_{G_{t+k}} \tag{10}$$

Forward kinematics에서 $^WT_{G_t}$ 가져와 rel-EE chunk를 world frame으로 cast.

Humanoid (예: Astribot S1): deterministic heuristic (Algorithm 1 in Appendix B.1)이 ##fixed chassis frame $^WT_C$## 추론, predicted gripper target에서 ##floating torso frame## 생성:

$$^CT_{G_{t+k}} = (^WT_C)^{-1} \cdot ^WT_{G_t} \cdot ^{G_t}T_{G_{t+k}} \tag{11}$$

Additional 24 head/torso dim (12 each: 3 position + 9 rotation)은 heuristic이 설정. 학습된 action interface는 Track A/B 모두에서 ##unchanged##.

5.2 Asynchronous Execution

High-capacity VLA가 robot servo loop보다 slow → synchronous execution은 robot idle. ##Producer-consumer runtime + thread-safe action buffer $\mathcal{B}$##:

  • Inference thread: latest observation으로 policy query, $\mathcal{B}$를 smoothed action sequence로 ##overwrite##
  • Execution thread: $\mathcal{B}$에서 control frequency로 command pop, recent pose를 tangent estimation용으로 기록
  • 두 loop overlap → backbone latency를 continuous execution 뒤로 숨김

5.3 Latency-Aware Cubic Bézier Chunk Stitching

Async execution에서 delayed chunk를 ##motion discontinuity 없이## 현재 state에 reconnect 필요. ##Cubic Bézier segment##로 ##compact $C^1$-continuous connector##:

Original chunk length $N$, ##stale prefix $K = \lceil N/\alpha \rceil$ ($K \leq N - 3$) 폐기##, 잔존 $\mathcal{F} = \{f_0, \ldots, f_{M-1}\}$, $M = N - K$. Last executed EE position $h_0$. Interior connection point $f_c$ with $c = \text{clip}(\lfloor \gamma M \rfloor, 1, M-2)$.

Cubic Bézier segment $\mathbf{B}(t)$ satisfying:

$$\mathbf{B}(0) = h_0, \quad \mathbf{B}(1) = f_c, \quad \dot{\mathbf{B}}(0) \parallel \hat{d}_{\text{hist}}, \quad \dot{\mathbf{B}}(1) \parallel \hat{d}_{\text{fut}} \tag{13}$$

여기서 tangents:

$$\hat{d}_{\text{hist}} = \frac{h_0 - h_{-1}}{\|h_0 - h_{-1}\|}, \quad \hat{d}_{\text{fut}} = \frac{f_{c+1} - f_{c-1}}{\|f_{c+1} - f_{c-1}\|} \tag{14}$$

Control points:

$$\mathbf{P}_0 = h_0, \quad \mathbf{P}_1 = \mathbf{P}_0 + \lambda \hat{d}_{\text{hist}} \tag{15}$$ $$\mathbf{P}_2 = \mathbf{P}_3 - \lambda \hat{d}_{\text{fut}}, \quad \mathbf{P}_3 = f_c \tag{16}$$

Tangent length $\lambda = \sigma \|\mathbf{P}_3 - \mathbf{P}_0\|$.

$$\mathbf{B}(t) = (1-t)^3 \mathbf{P}_0 + 3(1-t)^2 t \mathbf{P}_1 + 3(1-t) t^2 \mathbf{P}_2 + t^3 \mathbf{P}_3 \tag{17}$$

Position은 $\mathbb{R}^3$ smoothing, orientation은 SLERP, gripper command는 linear interpolation, ##각 arm은 independent## 처리. 결과 transition은 ##$C^1$-continuous, policy-agnostic##, embodiment-dependent params $\alpha, \gamma, \sigma$만 hyperparameter.

8. Discussion

HyVLA-0.5는 ##VLA를 standalone policy가 아니라 data + representation + policy refinement + deployment execution을 co-design한 deployable generalist robot##으로 다룬다. Cross-embodiment deployment는 model scale 너머의 ##complete component set##에 의존:

  • High-fidelity UMI 데이터 → precise manipulation prior
  • Compact memory encoder + rel-EE/delta-chunk representation → temporal context와 platform-specific kinematics 분리
  • FlowPRO → real failure를 compact offline refinement로 변환, large-scale online exploration 불필요
  • Async chunk stitching → real hardware latency 하에서 같은 checkpoint 실행 가능

이들은 ##다른 bottleneck## (data quality, action representation, failure correction, deployment timing)을 다루지만 같은 목표 -- ##stable policy interface 보존하면서 embodiment-specific 차이를 학습 코어 외부에서 absorb##.

Future Work

  • Data side: Motion capture를 넘어 high-precision supervision 보존하는 방향 -- ##exoskeleton-based collection##이 promising route. ##2,000-hour self-collected UMI subset 공개 예정## (precision의 marginal value 연구 + visual augmentation 연구 지원)
  • Real-world execution efficiency: success뿐 아니라 ##practical task cadence## 달성. Deployment-time adaptation + RL 결합이 핵심 다음 단계
  • Embodied intelligence emergence: HyVLA-0.5는 ##zero-shot generalization##을 study하지 않음 (current data scale 부족 가설). π₀.₇ 같은 최근 시스템이 zero-shot 조짐을 보이기 시작 -- evaluation 자체를 어떻게 rigorously 하고, evaluation으로 model/pipeline iteration을 어떻게 drive할지가 open problem

9. 비판적 분석

강점

  • Stack-level co-design의 명료한 그림: 데이터 → 모델 → SFT → RL → deployment까지 ##각 컴포넌트의 역할과 책임을 분리하면서 통합##. 후속 연구의 architecture template로 가치 큼.
  • 고정밀 데이터의 결정적 가치 실증: Mocap-based sub-mm UMI가 SLAM-based UMI 대비 ##precision-critical task에서 결정적##. RoboTwin 2.0 ablation (90.9 → 88.1)과 Track A real-robot에서 일관 입증.
  • Cross-embodiment transfer 입증: ##Target-robot teleoperation 0시간##으로 morphologically 다른 robot에 deploy 가능 -- 산업적으로 매우 큰 의미. 비슷한 형태의 robot마다 별도 데이터 수집 필요라는 통념 도전.
  • FlowPRO의 우아한 설계: Reward/critic 없는 RL은 매력적이고, ##contrastive gradient cancellation##은 SFT 데이터를 동일 loss로 안전 co-train하는 비자명한 시그널. ##Symmetric proximal regularizer##가 plain DPO 반복 패턴 (reward-hacking)을 구조적으로 막음.
  • Async + Bézier deployment의 plug-and-play 성격: 학습 무관, 모든 chunk-based VLA에 retrofit 가능. Cubic Bézier C¹ continuity를 명시적 control point 공식으로 풀어 reproducible.
  • Compact Memory Encoder의 zero-parameter 디자인: ##K=1 special case로 single-image encoder와 정확히 일치##, pre-trained Hy-ViT 2.0 weight 직접 초기화. Backward compatibility의 모범.
  • 실증의 폭: RoboTwin 2.0 50 task 시뮬레이션 + 4개 real platform (Dobot, JAKA, Astribot S1, Unitree G1) + 4 task RPRO. 단일 platform 결과가 아님.
  • Open release: GitHub, HF model, HF dataset, 2K-hour UMI subset 공개 예정. 재현·검증·후속 연구 진입장벽 낮춤.

한계와 의문점

  • Mocap cage의 in-the-wild 부적합성: 저자도 인정 -- ##optical motion-capture system 의존##은 정밀도 우위지만 ##field deployment 환경에서 데이터 수집 불가##. 결과적으로 데이터는 lab-quality에 한정. "10K-hour scale"은 단일 lab의 한계 시사.
  • Pre-training과 fine-tuning의 데이터 distribution gap: Egocentric UMI camera vs robot-mounted deployment camera ##시점/내재 차이##. 저자도 future work로 visual augmentation 언급. 현재 framework가 이 gap을 얼마나 학습 시 보상하는지 ablation 부족.
  • UMI fine-tuning 데이터의 작은 규모: Track B에서 ##task당 200~300 demo / 1.2~1.5h##만으로 cross-embodiment transfer 입증 -- 인상적이지만 ##task 다양성##(JAKA에서 1 task, Astribot에서 1 task)이 좁다. "Put Away Accessory"와 "Clean Up Table" 외에 다른 task에서 같은 패턴이 나오는지 미검증.
  • FlowPRO의 SR 이미 high baseline에서 측정: π₀.₆* 95% → RPRO 99%는 인상적이지만, ##기존이 낮은 task##(예: SFT만으로 50%)에서 RPRO가 어떤 효과인지 불명. "near-ceiling improvement"라는 claim 자체가 편향된 평가.
  • π₀.₆* 비교의 implementation 동등성: π₀.₆*가 advantage-conditioned regression이라는 description은 있지만, 본 논문의 base policy는 HyVLA-0.5이고 π₀.₆*도 HyVLA-0.5 위에 inject한 형태. ##원본 π₀.₆와의 fair comparison##이 아닌 ##recipe 비교##임을 명확히 인식할 필요.
  • Force modality는 후처리적 augment: Hand-held UMI가 force 기록한다는 설계는 매력적이지만, ##current HyVLA-0.5는 RGB만 학습##. Force는 G1 task에서 ##별도 lightweight encoder를 post-hoc bolt-on##. 설계 통합이 아니라 add-on 시연.
  • RoboTwin 2.0 비교의 contemporary 방법 출처 미명시 정도: 표에 ABot-M0, Qwen-VLA, LingBot-VLA, starVLA, Motus, JoyAI-RA 다수 등장. 이들은 모두 같은 시기 출간된 reference robot policy로 보이며, ##모두 자체 보고 success rate 사용##이라 reproducibility 우려. 동일 inference budget/condition인지 불명.
  • Pre-training 외 ablation 부족: MoT vs dense VLM, action-expert dual-tower vs single-tower, dual EEF prediction (rel-EEF + EEF) 등 핵심 설계의 isolated ablation이 없거나 appendix 수준에 머무름. End-to-end 결과만으로는 어느 설계가 결정적인지 판단 어려움.
  • Hy-Embodied-0.5-MoT 자체의 학습 disclosure 부족: 4B MoT backbone이 ##embodied corpora##로 pre-train이라고만 적시. Pre-train data composition, 실제 효과는 unclear. ##"Generic VLM 대비 더 빠른 수렴 + 더 강한 spatial prior"##라는 claim도 직접 비교 ablation 부재.
  • Latency 수치 부재: Async + Bézier가 deployment-friendly라는 강조에 비해, ##실제 backbone forward latency, action smoothing overhead, end-to-end loop time## 등 구체 수치 없음. "high-frequency closed-loop control"이 몇 Hz인지 직접 비교 어려움.
  • Safety / Failure mode 분석 부재: Real-robot manipulation에서 hardware safety는 critical. RPRO 과정 중 사람 개입 (intervention-and-rollback) 외, ##정책이 unsafe behavior로 drift할 risk##나 emergency stop 등에 대한 분석 부재.

10. Key Takeaways (이 논문에서 가져갈 것)

  1. Stack-level Co-design Paradigm: VLA를 ##단일 policy 아닌 production stack##으로 설계. 데이터-모델-RL-deployment 각 단계가 별개 병목을 다루지만 ##stable policy interface 보존이라는 동일 목표##. 후속 연구의 reference architecture.
  2. Rel-EE / Delta-chunk Representation: ##Embodiment-specific kinematics를 deployment 단계로 분리하는 결정적 디자인 선택##. 학습 시점에 robot-agnostic, deployment 시점에 robot-specific. Cross-embodiment transfer의 enabler.
  3. Sub-mm 정밀 데이터의 가치: SLAM-based UMI 대신 ##motion-capture cage + fingertip-attached UMI##. Precision-critical task의 critical sub-step이 sub-cm 정확도에 의존, large-scale 고정밀 pre-train이 그 distribution을 sharpen. ##데이터 quality가 quantity만큼 중요##.
  4. Compact Memory의 Parameter-free 설계: Multi-frame history를 추가 parameter 없이 ##sinusoidal temporal encoding + 기존 QKV/$W_O$ 재사용 + temporal/spatial 2-pass factorization##. ##K=1 special case로 single-frame encoder와 정확히 동일## → pre-trained weight 직접 초기화 가능. ViT 류 변형의 backward-compatible 디자인 패턴.
  5. FlowPRO의 3 Design Principles: (P1) failure를 contrastive signal로 직접 활용, (P2) reward/critic/value 모두 제거, (P3) symmetric proximal regularizer로 implicit reward anchor. ##연락처 풍부 manipulation에서 dense reward design이 어려운 분야의 일반 recipe##.
  6. Contrastive Gradient Cancellation: $a^w = a^l$일 때 contrastive gradient가 0 → ##SFT-style 데이터를 동일 RPRO loss로 batch 내 라우팅 가능##. Multi-objective training에서 batch composition의 우아한 단순화 패턴.
  7. Smooth Interpolation for Sparse → Dense Pair: Trajectory-level intervention-and-rollback이 sparse하지만, cubic Bézier (positions) + Slerp (orientations) + linear (gripper) ##smooth interpolation##으로 per-state tuple 생성. Preference 데이터를 ##인간 효율성##과 ##학습 dense성##에서 모두 만족시키는 설계.
  8. Async Inference + Bézier Stitching: ##학습 무관, plug-and-play, embodiment 무관##, $C^1$-continuous. 모든 chunk-based VLA의 deployment 고질병 (chunk boundary jitter, backbone-servo speed gap) 해결. Production-grade VLA의 표준 component가 될 가능성.
  9. Cross-embodiment Transfer with Zero Target-Teleop: ##UMI 데이터만으로 humanoid (Astribot S1) 와 fixed-base arm (JAKA K1) 두 morphology## 모두에서 성공. 새 robot platform 도입 비용을 ##teleop infra 구축 → UMI demo 200~300개 수집##으로 줄이는 paradigm shift 가능.
  10. Force Modality는 lightweight encoder bolt-on으로: Tactile/F-T sensor를 base VLM에 통합하지 않고 ##50-step F/T window를 lightweight TCN + MLP projector (~2M params)##로 augment. Force-aware manipulation의 minimal-cost baseline.
