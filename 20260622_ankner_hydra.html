Hydra: Sequentially-Dependent Draft Heads for Medusa Decoding

TL;DR

  • 문제: Medusa의 draft head들이 ##sequentially independent##하다. Head $i$는 base model의 hidden state $h_{t-1}$만 보고 $i$번째 미래 토큰을 예측하므로, 같은 candidate continuation 내의 이전 draft token들($\hat{x}_{t+1}, \ldots, \hat{x}_{t+i-1}$)을 ##전혀 인지하지 못한다##. 자연어의 강한 순차 의존성을 고려할 때 이건 분명히 손해.
  • 해결: ##Hydra heads##. 각 draft head $i$가 hidden state $h_{t-1}$ + 이전 candidate token들의 embedding $E_{\hat{x}_{t+1}}, \ldots, E_{\hat{x}_{t+i-1}}$를 ##concatenate##하여 입력 받음. drop-in replacement.
  • 왜 작동하는가: Head 2가 Head 1의 출력을 알게 되면, "If Head 1 says X, then most likely Y" 같은 ##조건부 분포##를 학습할 수 있다. acceptance length가 평균 +0.46 토큰 증가.
  • Hydra++: 추가로 (1) head MLP를 4-layer로 확장, (2) self-distillation training objective, (3) prefix attention layer 추가. Medusa 대비 1.31×, vanilla AR 대비 2.70× throughput.
  • 비유: Medusa heads가 "한 회사 전체의 미래를 각자 따로 예측하는 부서들"이라면, Hydra는 "1팀 결과를 보고 그 위에 2팀이 예측을 쌓는 ##연쇄적 부서##".

핵심 기여 (Key Contributions)

  1. Sequential dependence의 명확한 진단: Medusa head들의 ##sequential independence##가 acceptance rate의 핵심 손실 원인임을 분석. 분석은 단순하지만 통찰적: $p_{\text{draft}}(\hat{x}_{t+i}|x_{\leq t}, \hat{x}_{t+1}, \ldots, \hat{x}_{t+i-1}) = p_{\text{draft}}(\hat{x}_{t+i}|x_{\leq t-1})$ (조건부가 prior token에 의존하지 않음).
  2. Hydra heads (drop-in replacement): $f_{\text{Hydra},i}(h_{t-1}, x_t, \hat{x}_{t+1}, \ldots, \hat{x}_{t+i-1})$. 단순 MLP 입력 확장으로 ##Medusa 대비 acceptance length +0.46##, throughput 1.10×.
  3. Hydra++의 design space 탐색: 3가지 추가 개선. (a) 4-layer MLP scaling, (b) self-distillation, (c) prefix attention (base model에 추가 self-attention decoder layer를 ##draft model 전용 hidden state 생성##). 누적 ##1.31× Medusa, 2.70× AR##.
  4. Batch 환경 검증: batch 1~8까지 throughput 측정. ##batch 8에서도 Hydra++가 Medusa 대비 우위 유지##.
  5. Typical Acceptance에서 quality 보존: non-greedy 환경에서 Hydra++ ##posterior threshold 0.15에서 base model과 동일 quality##(MT-Bench LLM-judge 점수). Medusa나 일반 Hydra는 미달.

1. 배경 (Background)

1.1 Speculative decoding 복습

Base model이 비쌈 → 작은 draft model이 candidate continuation 제안 → base model이 single forward pass로 검증. ##Acceptance length 평균이 클수록 throughput↑##. 검증 기준은 (a) rejection sampling(lossless), (b) greedy acceptance(base의 top-1과 일치하면 채택), (c) typical acceptance(임계값 기반).

1.2 Tree decoding

Single candidate continuation을 ##candidate tree##로 일반화. 각 node에 자식 $m$개. Static tree(설계 시 fixed)가 일반적. Tree 위치별 token이 single sequence로 packing되고 attention mask로 parent만 attend.

1.3 Draft heads as draft model (Stern et al. 2018, Medusa)

독립 학습된 작은 LM 대신 ##base model의 hidden state $h_{t-1}$에 다는 lightweight head들##. Stern et al. 2018의 정의:

$$p_{\text{draft}}(x_{t+i}|x_{\leq t-1}) = f_{\text{draft},i}(h_{t-1}) \quad (1)$$

Medusa는 이 framework의 특정 구현: tree-based candidate, single-layer MLP + residual head.

1.4 Sequential independence의 문제

핵심 관찰: $h_{t-1}$은 ##이미 verify된 sequence##의 함수일 뿐, candidate continuation 내의 draft token에 의존하지 않는다. 따라서:

$$p_{\text{draft}}(\hat{x}_{t+i}|x_{\leq t}, \hat{x}_{t+1}, \ldots, \hat{x}_{t+i-1}) = p_{\text{draft}}(\hat{x}_{t+i}|x_{\leq t-1})$$

직관적으로 말하면, ##Medusa Head 5는 Head 1, 2, 3, 4가 무엇을 예측했는지 전혀 모른다##. 자연어 처리에서 next token 예측은 prior tokens에 강하게 의존하는데, 이 강한 의존성을 무시.

2. Hydra Heads (Section 3)

2.1 핵심 변경 (Figure 1)

[Figure 1 설명]: 왼쪽이 Existing Draft Heads — base model의 hidden state만 input. Head 1 → "Go", Head 2 → "Fast"를 독립적으로 예측. 오른쪽이 Hydra Draft Heads — Head 1이 "Go"를 예측하면 ##그 결과의 embedding이 Head 2의 추가 input##으로 들어감. Head 2는 hidden state + "Go" embedding을 함께 보고 "Fast" 예측. lines going into a head represent inputs to the draft head.

구체적 정의:

$$p_{\text{draft}}(\hat{x}_{t+i}|x_{\leq t}, \hat{x}_{t+1}, \ldots, \hat{x}_{t+i-1}) = f_{\text{Hydra},i}(h_{t-1}, x_t, \hat{x}_{t+1}, \ldots, \hat{x}_{t+i-1})$$

Basic Hydra head architecture: single hidden layer MLP, input은 $h_{t-1}$과 candidate continuation 토큰들의 embedding $E_{x_t}, E_{\hat{x}_{t+1}}, \ldots, E_{\hat{x}_{t+i-1}}$를 feature dimension으로 concat.

2.2 왜 이게 의미 있는가

Head $i$가 head $1, \ldots, i-1$의 출력에 conditional해지므로:

  • "If 'How' then most likely 'are'" 같은 ##2-gram 수준의 conditional distribution##을 학습 가능
  • Tree에서 다른 branches는 서로 다른 conditional이 적용 → ##higher acceptance rate##
  • Hydra head는 여전히 단일 forward pass에서 ##모든 candidate continuation에 대해 병렬 계산## (tree attention처럼)

3. Hydra++: Design Space Exploration (Section 3.1)

저자들이 Hydra 위에 효과적인 변경 3가지를 발견:

3.1 Scaling — MLP를 4-layer로 확장

Medusa의 single-layer MLP를 ##4-layer##로 확장. 5-layer 이상은 추가 이득 없음. ##capacity가 sequential dependence를 충분히 활용하기 위해 필요##함을 시사.

3.2 Distillation — Self-distillation objective

Zhou et al. 2024 따라, ground-truth token 대신 base model의 분포를 target으로 학습. ##Draft heads의 출력 분포가 base model 분포와 align##. 이는 acceptance rate를 직접 끌어올림.

구체적으로, base model의 next-token 분포 $p_{\text{base}}(x|x_{\leq t-1})$를 target으로 cross-entropy:

$$\mathcal{L}_{\text{distill}} = \text{KL}(p_{\text{base}}\|p_{\text{Hydra}})$$

3.3 Prefix Attention — base model에 1개 self-attention layer 추가

핵심 아이디어: draft model이 ##가장 최근의 토큰뿐 아니라 context 전체##에서 정보 가져오게 함. base model에 추가 self-attention decoder layer를 부착하고, ##이 layer는 오직 draft model용 hidden state 생성##에만 쓰임. 매 decoding step마다 1회만 query.

장점: base model의 verified KV cache를 활용한 추가 attention pass. ##draft head MLP에 들어가는 hidden state가 더 정보 풍부##.

4. Decoding Tree Discovery (Section 4)

Hydra도 Medusa처럼 ##static tree topology##(offline 계산) 사용. 그러나 최적 tree는 setup마다 다름(base model, draft model, batch size, hardware).

2-stage 알고리즘:

  1. Proposal tree 시퀀스 생성: $T_1, \ldots, T_N$ (size 1, ..., N). $T_1$은 trivial. Step $i$에서 sample text로 speculative decoding simulate, ##existing node의 child를 추가하면 acceptance length 향상이 가장 큰 것을 선택##해 $T_i$를 만듦.
  2. Throughput 측정: 각 $T_i$를 실제 inference config에서 측정해 throughput max인 tree 선택.

$N=100$, 100-question Alpaca subset 사용.

5. 학습 / 평가 설정

  • Models: Vicuna 7B, 13B, 33B (LLaMA conversation-fine-tuned). Base model frozen.
  • Data: ShareGPT (multi-turn conversations)
  • Training: 8× A100 80GB, HuggingFace Trainer, AdamW (β1=0.9, β2=0.999), cosine LR with warmup, peak LR 1e-3. Hydra/Medusa는 1 epoch (1 epoch 후 saturate), ##Hydra++는 10 epoch##.
  • Hardware (eval): 7B/13B는 A100-40GB, 33B는 A100-80GB.
  • Benchmark: MT-Bench (multi-turn). 기본은 greedy verification이라 quality는 base model과 동일 → throughput만 측정.

7. 종합 평가

강점

  • 진단의 명확성: "Medusa head들이 sequentially independent하다"는 한 문장의 진단. 식 $p(\hat{x}_{t+i}|...) = p(\hat{x}_{t+i}|x_{\leq t-1})$로 명료히 표현. ##핵심 약점을 발견하는 능력##.
  • 최소 침습적 수정: 입력 concatenation 한 줄 변경으로 +0.46 acceptance length. ##drop-in replacement##라 기존 Medusa pipeline에 그대로 통합 가능.
  • Design space exploration의 완성도: scaling, distillation, prefix attention 3개를 ##sequential dependence와 직교##하게 발견. 누적 효과 +27~31%.
  • Batched inference 평가: Medusa 원논문이 미보고한 batched setting에서도 일관 효과. ##production deployment 관련 통찰##.
  • Typical acceptance quality 관점: ##non-distribution-preserving sampling에서 quality를 base model 수준으로 회복##할 수 있는 첫 draft-head 방법. lossy sampling 도입 시 head accuracy의 결정적 역할 입증.
  • Tree discovery 알고리즘: data-driven 방식으로 optimal tree topology 발견. ##반복 가능한 procedure##로 deployment마다 calibrate 가능.

한계 및 의문점

  • 여전히 Medusa의 한계 상속: Hydra++도 ##draft head 패러다임##이라 base model의 layer당 표현력을 그대로 활용하지 못함. 이후 EAGLE이 feature-level autoregression으로 더 큰 가속(5×+) 달성하며 Hydra를 추월.
  • Prefix Attention의 cost 미정량화: base model에 추가 self-attention decoder layer는 ##model parameters를 증가시킴##(forward cost도 +5% 수준일 수). 학습 cost와 추가 메모리는 본문에서 명확히 분해 안 됨.
  • Hydra++의 10 epoch 학습: Hydra (1 epoch) 대비 ##10배 더 긴 학습##. 학습 비용 trade-off가 본문에서 명확히 분석되지 않음.
  • Greedy verification 중심: 본 평가는 대부분 greedy. typical acceptance는 별도 ablation. ##rejection sampling##(strict lossless)에서의 효과는 미보고.
  • 33B 이상 미검증: Vicuna 33B까지. ##70B+ 모델에서의 효과##는 후속 EAGLE 시리즈에서나 확인됨.
  • EAGLE과의 비교 부재 (concurrent work): 저자도 "concurrent work EAGLE"이라 인정하나 본문에선 직접 비교 안 됨. ##sequential dependence vs feature-level autoregression##이라는 같은 직관을 다른 각도에서 시도한 두 연구.
  • Distillation 데이터 출처: self-distillation을 위해 base model의 분포를 사용. ##어떤 prompt set에서 distillation 분포를 추출했는지##(ShareGPT인지 별도인지) 본문에서 모호.
  • Tree calibration cost: 100 question simulate으로 tree 찾기. ##deployment마다 재calibrate##해야 하는 부담.

8. Key Takeaways

  1. "Independence is a bug, not a feature": Medusa의 design choice였던 sequential independence가 사실은 성능 손실. ##간단한 input concatenation으로 해결##되는 fundamental 결함. 새 architecture를 설계할 때 어떤 conditional structure를 만드는지 항상 고려해야 함.
  2. Acceptance length가 throughput의 직접 driver: +0.46 token → +10~11% throughput. ##draft head accuracy의 marginal value를 정량화##.
  3. Self-distillation은 boilerplate 개선 패턴: ground-truth label 대신 base model 분포로 학습 → 어떤 draft model에도 적용 가능한 일반 trick. 후속 EAGLE/EAGLE-3도 비슷한 접근 활용.
  4. Typical acceptance에서 head accuracy가 결정적: lossy sampling 환경에선 head accuracy가 quality 보존의 bottleneck. ##acceptance criterion과 head 디자인의 coupling##을 인지해야 함.
  5. EAGLE에 영향 (concurrent work): 같은 시기 Concurrent하게 발표된 EAGLE이 비슷한 직관(feature-level autoregression)을 다른 각도에서 적용. Hydra는 ##draft head 패러다임의 마지막 milestone##이자 EAGLE의 zeitgeist.
