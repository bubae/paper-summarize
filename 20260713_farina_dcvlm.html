TL;DR

  • 문제: VLM(Vision-Language Model)의 성능은 학습 데이터의 구성에 크게 좌우되지만, 어떤 데이터셋을 얼마의 비율로 섞을지 결정하는 curation 전략에 대한 systematic benchmark가 부재했다. VLM 논문마다 다른 평가 벤치마크를 써서 fair comparison도 어려웠다.
  • 해결: ##DCVLM##(DataComp for VLMs)이라는 controlled benchmark를 제안. 160개 open dataset을 4가지 data type (image-caption, multimodal document, text-only, instruction-tuning)으로 정리한 6T-token pool을 만들고, 1B-8B model + 6.25B-200B token의 4단계 스케일링 사다리에서 curation 전략을 평가할 수 있게 한다. 52개 downstream benchmark로 평가. 1000+ 실험으로 두 가지 핵심 발견: (1) 필터링은 이제 효과가 없다, (2) 데이터 혼합(mixing) 비율이 진짜 lever이며 instruction-heavy mix가 scale better.
  • 왜 작동하는가: DataComp/DCLM 시절에는 raw CommonCrawl에서 filter가 노이즈를 걷어냈지만, 현대 VLM 학습셋은 이미 upstream에서 상당한 curation이 되어 있어 downstream filter의 marginal utility가 감소한다 (다이어그램 4 참조). 반대로 mixing 비율은 이 aggregate 구조에 대응하는 lever. 특히 instruction-tuning 데이터가 많을수록 model size × token budget이 커질수록 gain이 커진다.
  • 비유: 이미 정성껏 손질된 재료들을 모아둔 마트에서, 다시 재료를 골라내는(filter) 것보다 어떤 재료를 얼마나 섞을지(mix)가 훨씬 중요하다. 그리고 요리사 실력이 좋을수록(model scale), 매콤한 양념(instruction data)이 더 살아난다.

핵심 기여 (Key Contributions)

  1. DCVLM 벤치마크: VLM 학습을 위한 첫 data-centric benchmark. 4가지 data type × 160 datasets × 6T token pool. Fixed model architecture + training recipe로 오직 data curation strategy만 비교 가능하게 함.
  2. Principled scaling ladder: small(1B, 6.25B), medium(2B, 25B), large(4B, 100B), x-large(8B, 200B) 4단계. 각 단계 사이에 8× compute increase (2× 모델 크기, 4× 토큰) → 작은 스케일 발견이 큰 스케일로 transfer되는지를 명시적으로 검증.
  3. 52-benchmark evaluation suite: 65 candidate 벤치마크에서 stability(seed variance) + monotonicity(small→medium 개선 여부)로 필터링. 9 domain (General, Knowledge, OCR & Charts, Vision, MTL, Text-only, Safety, Hallucination, Reasoning) 커버. Validation (13) / Core (33) / Extended (52) 3중 tier.
  4. "Filtering ≠ Mixing" 원칙: 60+ filter configuration을 테스트해도 no-filter baseline 대비 어떤 filter도 significant improvement 없음. 대신 image-caption vs instruction-tuning 비율(mixing) 변경만으로 +2~5pp gain.
  5. Instruction-heavy mixture가 scale better: 1B×6.25B에서는 worst였지만 4B×25B에서는 best가 되는 crossover 패턴. 작은 규모에서 얻은 mixture ranking을 큰 규모에 그대로 적용하면 최적을 놓친다.
  6. DCVLM-BASELINE 데이터셋: 이 발견을 반영한 데이터 mix (10% image-caption, 5% multimodal doc, 15% text-only, 70% instruction-tuning). 8B model + 200B token에서 Core-33 63.6% 달성, 이전 최고 open dataset FINEVISION 대비 ##+5.4pp##.
  7. Pretraining→SFT transfer: pretraining-only metric이 post-SFT 성능을 Pearson $r=0.99$로 예측. 그리고 mixture ranking은 Qwen2.5-Base와 Qwen2.5-Instruct 초기화 간에 preserved ($r=0.97$).

1. Background

1.1 VLM(Vision-Language Model)이란?

Modern autoregressive VLM은 세 구성 요소로 이루어진다:

  • Vision encoder (예: InternViT, SigLIP): 이미지를 token으로 인코딩
  • Language model backbone (예: Qwen2.5): 텍스트와 vision token을 함께 처리
  • Connector (2-layer MLP projector 등): vision token을 LM 임베딩 공간에 매핑

초기 "VLM pretraining"은 connector만 image-caption pair로 학습했지만, SoTA VLM(예: InternVL, Qwen-VL)은 모든 파라미터를 학습하고 다양한 data type을 통합한다.

1.2 Data-Centric Benchmark의 계보

  • DataComp (Gadre et al., 2023): contrastive VLM(CLIP)용 data curation benchmark. Raw web crawl에서 필터링이 중요.
  • DCLM (Li et al., 2024): LM pretraining data curation. FastText classifier로 high-quality sample filter → 큰 gain.
  • FineWeb, Nemotron-CC, OlmoASR: 각기 다른 domain의 data curation.
  • DCVLM (본 논문): autoregressive VLM data curation의 첫 systematic benchmark.

1.3 왜 VLM data curation이 어려운가?

  1. Aggregation-based regime: raw web crawl(DCLM)과 달리, VLM은 이미 curation된 datasets을 aggregate한다. 이 aggregation 하에서 무엇이 실제로 성능을 이끄는지 불명확.
  2. Scale mismatch: 기존 open dataset은 millions 규모. SoTA VLM은 trillions of tokens 사용 → 실험 가능한 curation 범위 제한.
  3. Design space 폭발: data type × model scale × training budget = 너무 넓어 exhaustive experiment 불가.
  4. Evaluation의 부재: 논문마다 다른 benchmark suite → fair comparison 어려움.

1.4 데이터 필터링 vs 데이터 혼합(mixing)

  • Filtering: 저품질 sample 제거 (예: CLIP-score threshold, quality classifier).
  • Mixing: 서로 다른 data type/source의 비율 조정.

Filter는 "무엇을 뺄 것인가", mixing은 "무엇을 얼마나 넣을 것인가". 두 접근이 서로 다른 lever임을 명시적으로 대조하는 것이 본 논문의 핵심 주장.

2.2 Model Architecture and Training Recipe (Section 3.2)

Fixed recipe (InternVL3 style):

  • Vision encoder: InternViT-300M
  • Projector: 2-layer MLP
  • Language backbone: Qwen2.5-Base ({0.5B, 1.5B, 3B, 7B})
  • AnyRes tiling: 이미지를 dynamic하게 448×448 tile로 분할, 각 tile은 pixel shuffling 후 256 visual token으로 인코딩
  • Optimizer: AdamW, 3% linear warmup + cosine decay, peak LR $2 \times 10^{-5}$

2.4 Evaluation Protocol (Section 3.4)

65 candidate → 52 benchmark 필터링 프로토콜:

  1. 9 domain 기반 candidate 선정 (majority consensus from InternVL, Qwen-VL, Nemotron, FineVision, InternVL3.5 논문)
  2. Stability filtering: high seed variance benchmark 제거
  3. Monotonicity filtering: small→medium에서 개선되지 않는 benchmark 제거

3-tier nested structure:

  • Validation set (13): rapid iteration용
  • Core set (33): main results tier
  • Extended set (52): 전체 분석. Safety, Hallucination, Reasoning benchmark 포함 (post-training targeted)

9 domain: General Understanding, Knowledge-Centric, OCR & Charts, Vision-Centric, Multilingual, Text-Only, Safety, Hallucination, Reasoning.

3. Towards a Strong Baseline on DCVLM (Section 4)

Base mixture (unless otherwise stated): 75% image-caption, 18% text-only, 4% multimodal document, 3% instruction-tuning (length-proportional sampling).

3.1 Data Filtering (Section 4.1)

실험 의도: 60+ filter configuration을 medium scale에서 테스트하여 quality-based filtering이 VLM 학습에서 유효한지 확인.

테스트된 filter family:

  • CLIP-score filters: OpenAI CLIP ViT-L/14, DFN-CLIP, SigLIP-2-B/16@384
  • Text quality classifiers: DCLM fastText, Nemotron-quality, Mixtral-educational quality
  • Multimodal filters: UniFilter (Qwen2.5-1.5B, Qwen3-0.6B)
  • Perplexity-based: text-only PPL (excluding image tokens), multimodal PPL (including image tokens), Conditional Mutual Information (difference between two)

Local vs Global filtering (두 filtering paradigm)

  • Local filtering: 각 source dataset 내에서 percentile threshold 독립 적용. Global mixture 보존.
  • Global filtering: 전체 pool에서 하나의 threshold 적용. Score distribution이 dataset마다 다르므로 implicit하게 mixture를 reshape.

Small scale에서는 top-10%, medium에서는 top-40% 유지 (smaller model이 more aggressive filter를 benefit한다는 prior evidence).

결과 (Figure 3)

Filter Local Δ Global Δ Conditional MI -3.3 -5.1 DCLM FastText -3.1 - UniFilter (Qwen2.5-1.5B) -0.8 -1.7 UniFilter (Qwen3-0.6B) -0.9 -2.2 Text-Only PPL -0.3 -0.7 Multimodal PPL -0.3 -0.3 NVIDIA Nemotron -0.3 -0.3 NVIDIA Mixtral +0.1 -0.3 OpenAI CLIP 0.0 +0.6 DFN-CLIP 0.0 +0.3 SigLIP-2 +0.2 +0.8 ← 최고

결과 분석:

  1. 어떤 filter도 no-filter baseline을 significant하게 능가하지 못한다. Best case인 SigLIP-2 global filtering도 +0.8pp만 (prior work 예상 대비 훨씬 작음).
  2. Local vs Global filtering 결과가 notably different. Global filtering이 mixture를 reshape하므로 실질적 gain은 filtering이 아닌 implicit mixing change에서 유래 가능성 시사.
  3. 일부 filter (Conditional MI, DCLM FastText, UniFilter)는 오히려 성능 저해.

왜 filter가 도움이 안 되나? (Figure 4)

본 논문의 pool은 이미 upstream curation이 상당히 진행되었기 때문. 검증을 위해 저자들은:

  • Original pool에서 3개 sub-pool 생성 (25%, 65%, 100% "pre-filtered" data)
  • 각 sub-pool에 additional CLIP-score filtering 적용
  • Small-scale gain 측정

결과: 25% pre-filtered일 때 downstream filter gain +2.4pp, 65%에서 +1.3pp, 100%에서 +0.6pp로 diminishing returns. Additional filter on top of already-curated data operates in a regime of diminishing returns.

3.2 Data Mixing (Section 4.2)

실험 의도: Base mixture 위에서 filtering이 무효하다면, mixing 비율이 진짜 lever인지 검증.

세 mixture를 image-caption ↔ instruction-tuning axis에서 비교 (text-only 15%, multimodal doc 5% 고정):

  • Caption-heavy: 65% image-caption + 15% instruction-tuning
  • Balanced: 40% image-caption + 40% instruction-tuning
  • Instruction-heavy: 10% image-caption + 70% instruction-tuning

3 model size (1B, 2B, 4B) × 3 token budget (6.25B, 12.5B, 25B) grid에서 평가.

Instruction-heavy mixtures scale better (Figure 5)

Model Size × Tokens Best mixture 1B × 6.25B Caption-heavy (41.0) > Balanced (39.4) > Inst-heavy (37.1) 1B × 25B Balanced (42.0) ~ Inst-heavy (41.6) > Caption-heavy (41.0) 2B × 6.25B Inst-heavy (48.4) ~ Balanced (48.1) > Caption-heavy (47.9) 2B × 25B Inst-heavy (51.7) > Balanced (50.5) > Caption-heavy (50.0) 4B × 6.25B Balanced (52.5) ~ Caption-heavy (52.2) > Inst-heavy (51.8) 4B × 25B Inst-heavy (55.8) > Balanced (54.8) > Caption-heavy (54.4)

Crossover 패턴: Instruction-heavy가 1B×6.25B에서 worst였다가 4B×25B에서 best가 된다. 가장 가파른 scaling slope를 가진다.

Practical implication: ##Mixture rankings at small scale do not transfer to larger scales##. Small scale (1B×6.25B) 최적화만 하면 Caption-heavy를 선택해서 결국 Instruction-heavy가 우승하는 큰 scale의 최적을 놓친다. Scale-aware data curation이 필요.

3.3 Control Experiments (Section 4.3)

Pretraining results transfer post-SFT (Figure 6 left)

Concern: SFT (Supervised Fine-Tuning)가 instruction-tuning data를 다시 쓰기 때문에, pretraining stage의 instruction-heavy mixture 이점이 사라지는가?

실험: 27개 pretrained checkpoint (3 mix × 3 model size × 3 token budget)을 두 SFT dataset (LLaVA-665K, Mammoth-VL-12M)으로 fine-tuning, 총 54 SFT run.

결과: Pearson $r=0.99$, Spearman $\rho=0.99$. Pretraining ordering이 완벽하게 보존. Pretraining-only metric으로 curation research 진행이 정당화됨.

Robust to LM initialization (Figure 6 right)

Qwen2.5-Base 대신 Qwen2.5-Instruct로 초기화해도 (3 mix × 3 token budget × 2B model), mixture ranking이 preserved. Pearson $r=0.97$. 특히 larger token budget에서 강한 일치.

- Image-Caption data: 10% - Multimodal Documents: 5% - Text-Only data: 15% - Instruction-Tuning data: 70%

Medium/Large 스케일에서 최적. Small 스케일은 Caption-heavy가 실제 최적이지만 simplicity를 위해 통일 mixture 사용. 각 data type 내에서는 length-proportional sampling으로 sub-source 선택.

5. 비판적 분석

강점

  1. Systematic scale-aware benchmark: 4-scale ladder가 curation strategy의 scaling behavior를 명시적으로 검증한다. Data curation research에서 "small scale finding은 large scale로 transfer되지 않는다"는 것이 실증됨.
  2. 1000+ 실험의 규모: Filter family(60+), mixture(3+), scale(4), post-SFT(54 run) — data curation 분야에서 매우 큰 규모의 controlled experiment.
  3. "Filtering ≠ Mixing" 원칙: Data-centric AI 담론을 filter 중심에서 mixture 중심으로 옮긴 명확한 evidence. 이 발견 자체가 향후 연구 방향에 실질적 영향.
  4. Pretraining→SFT transfer의 실증: $r=0.99$의 near-perfect correlation은 pretraining-only metric으로 data research를 진행하는 것을 정당화. 큰 SFT cost 없이 curation experiment 가능.
  5. Open release: 데이터 pool (160 datasets), evaluation suite (52 benchmarks), model checkpoint at 4 scales, experimental infrastructure. 재현 가능성과 후속 연구 확장성.
  6. Repeatability tolerance: Instruction-heavy가 8× repetition까지도 gracefully degrade한다는 실증적 결과는 실무자에게 매우 유용. Instruction dataset은 web crawl보다 작지만 repeat이 가능함을 보임.

약점 및 열린 질문

  1. Vision encoder 고정의 제약: InternViT-300M만 사용. Vision encoder scaling이 data mixing 최적점에 영향을 미치는지에 대한 실험 없음. 다른 encoder (SigLIP, DINOv3 등)로 결과가 유지되는지 불확실.
  2. Data type의 4개 분류의 과단순화: "instruction-tuning"이 8개 capability(chart, OCR, math 등)를 포함한다. Instruction-heavy 이점이 특정 capability에 유래하는지 vs 전반적 diversity에 유래하는지 구분 없음.
  3. InternVL SoTA와의 큰 gap 미해명: InternVL-3-8B (68.5) vs DCVLM-Baseline 8B (63.6)의 4.9pp gap. Data mix 외에 어떤 요소가 이 gap을 만드는가? Vision encoder(InternViT), longer training, post-training이 각각 얼마나 기여하는지 breakdown 부재.
  4. Filter의 완전한 부재는 아님: SigLIP-2 global filter가 +0.8pp라는 통계적으로는 significant한 gain을 준다. "no significant improvement"라는 주장은 다소 강하며, +0.8pp가 large scale에서 얼마인지 확인 없음.
  5. Multimodal document의 낮은 비중 (5%): MINT-1T 같은 대규모 interleaved document가 5%만 사용된 근거 부족. Recent 연구(예: Chameleon)에서 interleaved document의 중요성 강조 → 이 결과와 상반될 가능성.
  6. Language backbone의 종속성: Qwen2.5만 사용. Llama-3, Mistral 등 다른 backbone에서 mixture ranking이 유지되는지 확실치 않음 (Qwen2.5-Instruct도 Qwen family 내 variant).
  7. Reasoning benchmark의 상대적 소외: Reasoning, Safety, Hallucination이 Extended tier에만 포함. Core benchmark는 Reasoning이 없어 modern VLM의 핵심 능력(chain-of-thought, tool use)을 반영하지 못함.
  8. Compute cost의 심각성: x-large 실험 하나가 20,480 H100 hours. 이 규모의 benchmark를 다른 팀이 재현·확장할 수 있는지 실질적으로 의문. Open dataset은 공유했지만 실제 실험 재현은 대형 조직만 가능.

후속 연구 방향

  • Data type의 fine-grained mixing (instruction-tuning sub-category 별 최적화)
  • Multimodal document의 역할 재검증
  • Vision encoder scaling과 data mix 상호작용
  • Multilingual과 low-resource language data의 curation
  • Post-training 시 curation strategy도 함께 최적화하는 unified data recipe
  • DCVLM benchmark를 활용한 다른 참가자의 결과 (competition 성격)

6. Key Takeaways — 이 논문에서 가져갈 것

  • VLM 데이터 curation에서 filter는 죽었다, mix가 왕이다: Aggregation-based regime에서 quality filter는 marginal 이득만 준다. 대신 image-caption vs instruction-tuning 비율이 실질적 lever.
  • Instruction-heavy mixture가 scale better: 4B×25B에서 55.8%, 8B×200B에서 63.6%. 큰 model + 긴 학습일수록 instruction-tuning 비중을 70%까지 올려야 한다.
  • Small scale mixture ranking은 transfer되지 않는다: 1B×6.25B에서 Caption-heavy가 최적이지만 4B×25B에서는 Instruction-heavy가 최적. Curation 결정은 target scale의 sub-scale에서 실증해야 한다.
  • Moderate repetition은 감수 가치 있음: Instruction data 4× repetition도 unique caption-heavy와 대등 (Core 49.8 vs 50.3). Instruction dataset의 크기 부족은 큰 barrier가 아님.
  • Pretraining metric = post-SFT metric: Pearson $r=0.99$. Pretraining-only 실험으로 data research가 가능. SFT run 없이 curation strategy 비교.
  • Backbone-agnostic finding: Qwen2.5-Base와 Qwen2.5-Instruct에서 mixture ranking 유지 ($r=0.97$). 초기화 선택이 data curation 결론에 영향 미미.
  • DCVLM-Baseline은 open VLM 학습의 새 SoTA: 200B token / 8B model에서 Core-33 63.6%. Prior best open dataset FINEVISION 대비 +5.4pp. 후속 open VLM 연구의 default 데이터.
