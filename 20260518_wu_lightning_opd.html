Lightning OPD: Efficient Post-Training for Large Reasoning Models with Offline On-Policy Distillation

TL;DR

  • 문제: On-policy Distillation (OPD)는 효과적이지만 ##학습 내내 live teacher server를 함께 띄워야## 해서 GPU 자원이 크게 분할되고, 학원 연구 규모에서는 실질적으로 적용하기 어렵다 (8B 모델 OPD = 120 GPU hours, 그중 상당이 teacher serving overhead).
  • 해결: ##teacher의 log-probability를 SFT rollout에 대해 한 번만 precompute하고 학습 동안 재사용##하는 offline 방식 Lightning OPD. 단, 단순 offline化는 실패한다 — 그 실패 원인을 분석한 결과 ##teacher consistency## (SFT teacher와 OPD teacher가 동일해야 함)라는 새로운 design principle을 발견.
  • 왜 작동하는가: Teacher consistency가 성립하면 (1) online/offline OPD가 같은 optimum(공유 fixed point)을 가지고, (2) gradient 차이가 $G \sigma_A \sqrt{\chi^2(\pi_\theta\|\pi_\text{ref})}$로 bounded되며, (3) covariance term이 ##KL penalty 없이도 policy drift를 막는 implicit regularizer##로 작동한다.
  • 비유: 매 수업마다 멘토를 옆에 두고 첨삭받지 말고, ##학습 시작 전에 멘토에게 한 번 모든 문제의 모범 풀이를 받아둔 다음## 그것만 보며 혼자 공부하라. 단, 그 모범풀이를 만든 멘토와 평소 가르치는 멘토가 같은 사람일 때만 통한다.

핵심 기여 (Key Contributions)

  1. Teacher Consistency 원칙 발견: SFT 단계 teacher와 OPD 단계 teacher가 ##다르면 online/offline OPD 모두 gradient bias가 생긴다##는 점을 이론적으로 증명. 기존 관행 (예: Thinking Machines Lab이 OpenThoughts-3 = QwQ-32B 생성 데이터로 SFT하고 Qwen3-32B를 OPD teacher로 사용)이 이 원칙을 위반하고 있음을 지적.
  2. Lightning OPD 프레임워크: SFT rollout에 대해 teacher log-prob를 ##사전 1회 precompute##한 뒤 학습 동안 재사용. Live teacher server 제거로 ##모든 GPU를 student 학습에 할당##할 수 있어 대형 MoE도 가능.
  3. 이론 분석 4종: (i) Theorem 3.5 gradient discrepancy bound, (ii) Theorem 3.6 shared fixed point, (iii) Theorem 3.7 implicit regularization via covariance, (iv) Theorem 3.8–3.9 teacher inconsistency bias.
  4. 실증 성과: Qwen3-8B-Base + Qwen3-32B teacher로 ##AIME 2024 69.9%를 30 GPU hours로 달성## (vs OPD 120 GPU hours, ##4.0× 가속##). Qwen3-30B-A3B MoE를 ##single 8×H100 node에서 학습##해 AIME 2024 71.0%, LCB v5 60.8% — 학계 접근성 대폭 향상.
  5. Teacher Inconsistency 실증: 8B 스케일에서 SFT teacher Qwen3-32B → OPD teacher QwQ-32B로 바꾸면 Lightning OPD는 ##6.8점 하락##, 표준 OPD는 3.7점 하락 — offline 방식이 inconsistency에 더 취약함을 정량화.

배경 (Background)

OPD가 RLVR보다 매력적인 이유

LLM 후처리(post-training)는 일반적으로 SFT → RL의 두 단계로 진행된다. RL 단계는 (1) RLVR (verifiable reward, 예: 수학 정답 일치) 혹은 (2) On-Policy Distillation (OPD)로 구성. OPD는 RLVR 대비:

  • Dense supervision: 매 토큰마다 teacher의 log-prob 차이를 advantage로 사용 (vs RLVR의 sparse 종단 reward).
  • 학습 안정성: KL gradient 형태이므로 reward hacking 위험이 낮음.
  • 비용 효율: RL이 수천 step의 rollout/scoring을 요구하는 데 비해 OPD는 ##수십 ~ 수백 step에서 수렴##.

표준 OPD의 정식화

Teacher $\pi_T$ (fixed), student $\pi_\theta$. 응답 $x = (a_1, ..., a_T)$는 autoregressive하게 생성되며 $\pi_\theta(x|q) = \prod_t \pi_\theta(a_t|s_t)$. Per-token OPD advantage:

$$A_t(\theta) = \log \pi_T(a_t | s_t) - \log \pi_\theta(a_t | s_t) \quad (2)$$

직관적으로 말하면, ##teacher가 student보다 이 토큰을 더 확신하면 positive (학생이 따라가야 함), 반대면 negative##. 표준 OPD objective:

$$J_\text{on}(\theta) = \mathbb{E}_{q\sim p,\, x\sim\pi_\theta}\left[\sum_{t=1}^{T} A_t(\theta)\right] \quad (3)$$

여기서 $x \sim \pi_\theta$가 핵심 — student의 ##현재 정책##에서 rollout이 sampling되므로, 매 gradient step마다 teacher가 새로운 rollout에 대한 log-prob을 줘야 한다 → ##live teacher server 필수##.

Live Teacher Server의 비용

구체적 예: Qwen3-8B student + Qwen3-32B teacher → 32B teacher를 추가로 multi-GPU에 띄워야 함. 8×H100 한 노드를 student와 teacher가 share → ##GPU memory fragmentation, scheduling 복잡도, 그리고 학계 접근성 저하##.

Offline 가능성에 대한 선행 관찰

두 가지 empirical observation이 offline化의 motivation:

  • Yue et al. (2025): RL-trained 모델의 reasoning trajectory는 ##SFT 모델 분포의 reweighted subset##일 뿐, 새 trajectory를 거의 발굴하지 않는다.
  • Shenfeld et al. (RL's Razor): On-policy 업데이트는 본질적으로 ##reference policy로부터의 KL을 최소화하는 방향##으로 biased되어 있다.

이 두 사실은 "student 분포가 학습 내내 SFT 분포 근처에 머무른다"는 직관을 뒷받침하고, 따라서 ##rollout을 SFT 모델에서 한 번 뽑아두고 재사용해도 큰 문제가 없을 수 있다##는 가설로 이어진다.

왜 단순 offline化가 실패하는가

저자들은 naive offline OPD (SFT rollout에 teacher log-prob 한 번 precompute → 학습 중 재사용)를 시도하고, 표준 OPD 대비 ##성능이 일관되게 하락##함을 관찰. 근본 원인 분석 결과 offline approximation 자체가 아니라 ##teacher inconsistency##가 진짜 범인임을 밝힌다.

방법 (Method)

Lightning OPD 파이프라인

[Stage 1: SFT] Base model π_base + Teacher π_T + Prompt set Q_SFT ↓ teacher가 생성한 trajectory로 SFT Reference policy π_ref [Stage 2: Offline OPD] Preprocessing phase (한 번만): ∀ q ∈ Q_OPD: sample x ~ π_ref(·|q) precompute and store {log π_T(a_t | s_t)} for all t → Offline dataset D_OPD Training phase: Initialize π_θ ← π_ref Repeat (≈150 steps): mini-batch from D_OPD A_t(θ) = log π_T(a_t|s_t) [read from disk] - log π_θ(a_t|s_t) [computed online] clip A_t to [-τ, τ] θ ← θ + η · ∇J_off(θ) end

핵심 차이: 표준 OPD가 $x \sim \pi_\theta$로 매 step에 rollout을 새로 뽑는 반면, Lightning OPD는 ##$x \sim \pi_\text{ref}$로 고정한 rollout과 사전 계산된 teacher log-prob을 사용##.

$$J_\text{off}(\theta) = \mathbb{E}_{q\sim p,\, x\sim\pi_\text{ref}}\left[\sum_{t=1}^{T} A_t(\theta)\right] \quad (4)$$

Teacher Consistency: 핵심 design principle

Lightning OPD가 작동하려면 두 stage의 teacher가 같아야 한다:

  • SFT-stage teacher $\pi_T^\text{SFT}$: $\mathcal{D}_\text{SFT}$의 trajectory를 만드는 teacher.
  • OPD-stage teacher $\pi_T^\text{OPD}$: log-prob을 precompute하는 teacher.

이 두 teacher가 같지 않으면 (예: $\mathcal{D}_\text{SFT}$가 QwQ-32B 생성 trajectory, OPD teacher는 Qwen3-32B) $\pi_\text{ref}$는 ##$\pi_T^\text{SFT}$ trajectory에 집중##해 있는데 advantage가 $\pi_T^\text{OPD}$를 기준으로 계산되어 ##irreducible bias##가 발생한다.

이론 1: Gradient Discrepancy Bound (Theorem 3.5)

3가지 표준 가정 (bounded advantage, support coverage, bounded score) 하에서:

$$\|\nabla J_\text{on}(\theta) - \nabla J_\text{off}(\theta)\|_2 \le G \cdot \sigma_A \cdot \sqrt{\chi^2(\pi_\theta \| \pi_\text{ref})} \quad (8)$$

여기서 $\chi^2(\pi_\theta\|\pi_\text{ref}) = \mathbb{E}_{x\sim\pi_\text{ref}}[w(x;\theta)^2] - 1$, $w = \pi_\theta/\pi_\text{ref}$. 직관적으로 말하면, ##online과 offline gradient의 차이는 student가 reference로부터 얼마나 멀리 drift했는지에 비례##한다. Initialization에서 $\pi_\theta = \pi_\text{ref}$이므로 $\chi^2 = 0$, gradient가 ##정확히 일치##.

저자들은 Pinsker inequality를 통한 KL bound를 의도적으로 피한다 (이유: KL bound는 $\max |A_t|$를 포함하는데 이 값은 $\pi_\theta(a_t|s_t) \to 0$이면 발산. $\chi^2$ bound는 advantage clipping과 KL regularization을 통해 작게 유지 가능).

이론 2: Shared Fixed Point (Theorem 3.6)

$J_\text{on}(\theta) = -\text{KL}(\pi_\theta \| \pi_T) \le 0$, 최대점은 $\theta^* \in \arg\min_\theta \text{KL}(\pi_\theta\|\pi_T)$. Teacher가 model class 안에 representable ($\pi_T \in \Pi_\Theta$)이라면 $A_t(\theta^*) = 0$ almost surely → ##online과 offline gradient 모두 0##이 되는 ##공유 fixed point##.

이 시점에서 student는 teacher를 정확히 재현. 이 결과의 함의: Lightning OPD는 표준 OPD와 ##같은 곳으로 수렴## (capacity limit 안에서).

이론 3: Implicit Regularization (Theorem 3.7)

Importance-sampling decomposition으로부터:

$$\nabla J_\text{off}(\theta) = \nabla J_\text{on}(\theta) - \text{Cov}_{\pi_\text{ref}}[w(x;\theta), f(x;\theta)] \quad (\text{Thm 3.7})$$

여기서 $f(x;\theta) = \sum_t A_t(\theta) \cdot \nabla \log \pi_\theta(a_t|s_t)$. 직관적으로 말하면, offline gradient는 online gradient에서 ##covariance term을 빼는 형태##.

이 covariance term의 효과:

  • $\pi_\theta = \pi_\text{ref}$에서 $w \equiv 1$이므로 $\text{Cov} = 0$ → 차이 없음.
  • Student가 $\pi_\text{ref}$로부터 drift하면 $w$가 일부 trajectory에서 커지고, 같은 trajectory에서 $f$도 큰 norm을 가지는 경향. ##두 벡터 모두 drift 방향과 정렬##되어 $\text{Cov}[w, f]$가 drift 방향으로 커짐.
  • 이를 빼는 것은 ##drift 방향에서 gradient의 한 component를 제거##하는 것 = ##implicit KL-like regularization##.

중요: ##explicit KL term을 추가하지 않아도## policy drift가 자연스럽게 억제된다. trust-region effect.

이론 4: Teacher Consistency Bias (Theorem 3.8, 3.9)

SFT-stage teacher $\pi_T^\text{SFT}$와 OPD-stage teacher $\pi_T^\text{OPD}$가 다르면, $\Delta_t := \log \pi_T^\text{SFT}(a_t|s_t) - \log \pi_T^\text{OPD}(a_t|s_t)$. Bounded mismatch $\mathbb{E}_{x\sim\pi_\text{ref}}[(\sum_t |\Delta_t|)^2] \le \sigma_\Delta^2$ 하에서:

$$\|\nabla J_\text{on}(\theta) - \nabla J_\text{off}(\theta)\|_2 \le G(\sigma_A + \sigma_\Delta) \sqrt{\chi^2(\pi_\theta\|\pi_\text{ref})} \quad (9)$$

또한 offline gradient가 consistent offline gradient (mismatch가 없을 때) 대비 ##$G \sigma_\Delta$의 persistent bias##를 가진다.

표준 OPD에 대한 영향 (Theorem 3.9): initialization에서 $\theta = \theta_\text{ref}$일 때 $\|\nabla J_\text{on}(\theta_\text{ref}) - \nabla J_\text{on}^{\delta=0}(\theta_\text{ref})\|_2 \le G \sigma_\Delta$. ##online OPD도 inconsistency의 영향을 받지만 offline OPD가 더 크게 받는다## (왜? offline은 rollout 자체도 mismatched teacher의 trajectory를 따르므로 ##bias가 두 번 누적##).

Algorithm 1: Lightning OPD pseudo-code

// Stage 1: SFT 1. Collect D_SFT = {(q, x) | q ∈ Q_SFT, x ~ π_T(·|q)} 2. Fine-tune π_base on D_SFT → π_ref // Stage 2 Phase 1: Preprocessing (한 번) 3. for each prompt q ∈ Q_OPD: 4. sample x ~ π_ref(·|q); store log π_T(a_t|s_t) for all t 5. end for 6. Form D_OPD = {(q, x, {log π_T(a_t|s_t)})} // Stage 2 Phase 2: Training 7. Initialize π_θ ← π_ref 8. for each training step: 9. Sample mini-batch from D_OPD 10. Compute A_t(θ) = log π_T(a_t|s_t) - log π_θ(a_t|s_t), clip to [-τ, τ] 11. Update θ ← θ + η · ∇J_off(θ) 12. end for

패턴 분석:

  • Consistent diagonal이 항상 최강: 두 패러다임 모두에서 SFT teacher = OPD teacher일 때 최고 성능. Theorem 3.8–3.9의 실증적 검증.
  • Lightning OPD가 inconsistency에 더 취약: Qwen3-32B SFT → QwQ-32B OPD로 mismatch 시 Lightning OPD는 ##6.8점## 하락, 표준 OPD는 ##3.7점## 하락. ##차이의 원인##: standard OPD는 매 step student의 fresh rollout을 받아 부분 보정이 가능하지만, Lightning OPD는 fixed $\pi_\text{ref}$ rollout을 사용하므로 ##bias가 reference와 rollout 두 군데에 동시에 누적##.
  • Teacher consistency는 Lightning OPD에 더 critical한 design requirement: 저자들의 핵심 design principle을 정량적으로 입증.

Off-Policy RL과의 관계

저자들은 Appendix D에서 Lightning OPD를 conventional off-policy RL과 구분: off-policy RL은 sparse reward와 fixed dataset 두 가지 문제를 모두 다루어야 하므로 OOD value estimation이 challenge. 반면 Lightning OPD는 ##dense per-token teacher supervision##을 받으므로 OOD value estimation 문제가 없고, ##teacher consistency가 unique한 challenge##.

비판적 분석 (Critical Analysis)

  • 도메인 좁음 (math/code only): 모든 평가가 verifiable reasoning task에 집중. ##open-ended writing이나 instruction following에서의 효과##는 미검증. SFT trajectory가 다양한 도메인을 cover하지 못할 때 $\pi_\text{ref}$ 분포가 좁아져 $\chi^2$이 더 빨리 커질 수 있다.
  • $\chi^2$이 작게 유지된다는 가정의 의존성: Theorem 3.5의 bound는 $\chi^2$이 작을 때만 의미가 있다. 저자들은 advantage clipping과 짧은 학습(150 step)으로 이를 보장한다고 주장하지만, ##더 긴 학습이나 더 어려운 task에서 $\chi^2$이 폭발하면## bound가 vacuous해진다. 학습 step 수 sensitivity에 대한 추가 분석이 필요.
  • Teacher consistency의 실용적 비용: SFT 데이터를 OPD teacher가 직접 생성해야 한다는 요구는 ##OpenThoughts-3 같은 기존 SFT corpus를 그대로 활용 불가##. 좋은 기존 SFT dataset (QwQ-32B 생성)이 있어도 OPD teacher를 그것에 맞춰 바꿔야 하거나, SFT를 새로 해야 함. 이 ##실용 비용 vs 4.0× speedup## trade-off가 어느 쪽이 우세한지는 도메인마다 다를 수 있다.
  • Implicit regularization 주장의 정확한 강도: Theorem 3.7의 covariance term이 KL penalty를 ##정확히 대체##하는지, 아니면 일부만 흉내내는지 명확히 구분되지 않음. Figure 3a에서 empirical하게 trust-region 효과를 보였다고 하나, 명시적 KL과의 직접 비교 실험은 부족.
  • Capacity-limited case의 모호함: Theorem 3.6은 $\pi_T \in \Pi_\Theta$ (teacher가 representable)일 때만 shared fixed point가 정확히 성립. 실제 환경 (Qwen3-32B teacher가 Qwen3-8B student class에 representable이 아닐 가능성 높음)에서 어떤 분포로 수렴하는지는 ##empirical 주장에 의존##.
  • Rollout collection 비용 측정 누락 가능성: 30 GPU hours에 rollout collection 10h가 포함되지만, ##precompute phase의 teacher inference 비용##이 hardware-dependent. 저자들은 4h라 보고하나 다른 환경에서의 일반화는 불명확.
  • 표준편차/seed 미보고: AIME24 0.5점 차이 같은 소량 격차들이 noise level인지 확인 불가. 특히 inconsistency ablation (3.7점 vs 6.8점)도 multi-seed 검증이 필요.

Key Takeaways

  • "Teacher consistency를 디자인 원칙으로": 기존 OPD/RL 파이프라인 (Thinking Machines 등)이 이를 위반하고 있던 점이 가장 인상적. ##SFT data와 OPD teacher가 동일 모델 출신##인지를 ablation으로 반드시 확인할 것.
  • "Live teacher serving은 필수가 아니다": 학원/소규모 lab에서도 30B MoE 학습이 가능한 새 default. 8×H100 한 노드로 SOTA-급 reasoning 모델 학습.
  • "$\chi^2$ vs KL bound 선택의 중요성": 이론 derivation에서 KL bound가 vacuous해지는 상황 ($\pi_\theta \to 0$)을 피하려면 $\chi^2$ bound + advantage clipping 조합이 안전. 다른 RL/distillation 분석에도 응용 가능.
  • "Implicit regularization via covariance": KL penalty 없이 trust-region 효과를 얻는 메커니즘. Theorem 3.7의 covariance decomposition은 ##다른 offline RL/distillation 알고리즘 설계에도 transferable##.
  • "Offline OPD가 inconsistency에 더 취약한 비대칭성": bias가 reference policy와 rollout 두 곳에 동시 누적되기 때문. 실무 적용 시 teacher 변경은 SFT 단계부터 일관되게 적용해야 함.
