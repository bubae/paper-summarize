먼저 알아야 할 것: TIR과 Zero-RL

이 논문을 이해하려면 두 개념을 알아야 합니다.

  1. Tool-Integrated Reasoning (TIR): AI가 추론 중에 ##Python 코드를 짜서 실행##하고, 그 결과를 다시 보고 추론을 이어가는 방식. ReTool 같은 방법이 대표적.
  2. Zero-RL: DeepSeek R1이 입증한 패러다임으로, ##사전 SFT(supervised fine-tuning) 없이 base model에서 직접 강화학습##. 모델이 ##자유롭게 새로운 추론 패턴을 발견##할 수 있는 장점.

자연스러운 시도: "TIR을 Zero-RL로 학습하면 자유롭게 도구 사용 전략을 발견하지 않을까?"

문제: 학습이 폭발한다

결과는 실패였습니다. ##Multi-turn TIR을 base model에서 직접 RL##하면 학습 그래디언트가 폭발(gradient explosion)하고 성능이 무너집니다(performance collapse).

Naive 방식은 시간이 지날수록 더 못 풀게 됨

왜 폭발하는가? 진단

원인: Tool 피드백이 "이상한 입력"이라서

AI는 ##문장 예측##으로 사전 학습됐습니다. 그런데 외부 Python 인터프리터가 만든 출력(예: NameError: 'x' is not defined)은 ##AI가 본 적 없는 종류의 텍스트##입니다. 이런 ##OOD(Out-Of-Distribution) 입력##이 다음 추론을 ##불안정하게## 만듭니다.

Tool 피드백의 OOD 효과가 누적되어 결국 의미 없는 응답으로 붕괴
비유: 말을 잘하던 사람이 외국어를 듣고 헷갈리는 상황

한국어로 잘 대화하던 사람이 외국어(tool output) 한 마디를 듣고 약간 흔들립니다. 또 외국어를 듣고 더 흔들리고... 4번째 대화에선 ##혼잣말 같은 알 수 없는 소리##(void turn)를 합니다. 이 사람의 행동을 보고 학습하면 모든 게 망가집니다.

왜 gradient가 폭발하는가? 수학적 진단

저확률 토큰은 두 가지 방식으로 gradient를 폭발시킵니다.

원인 1 — Importance Ratio 폭발: 토큰을 ##아주 낮은 확률##로 생성한 후 그게 ##틀린 답##이라면, 그 토큰의 확률을 줄이려는 gradient가 ##무한대 가까이 커짐##.

원인 2 — Probability-dependent 항: 토큰 확률이 낮을수록 gradient norm을 키우는 ##수식의 다른 항##도 크게 유지됨.

두 요인이 결합해 ##gradient norm spike##가 빈발 → 모델이 ##안정적으로 학습 불가##.

SimpleTIR의 해결: Void Turn Filtering

핵심 통찰: ##저확률 토큰을 token-level로 찾기 어렵다##. 하지만 그 ##관찰 가능한 결과##인 "void turn"은 쉽게 식별 가능. void turn 한 개라도 포함된 trajectory를 통째로 학습에서 제외하면 문제 해결.

비유: 학생의 시험 답안 채점

학생들이 시험을 봤는데 일부 답안지에 ##판독 불가한 낙서##가 섞여 있습니다. 그런 답안지를 학습 데이터에 포함하면 채점 기준이 망가지죠. SimpleTIR은 "##낙서 있는 답안지는 통째로 빼라##"고 합니다. 깨끗한 답안지로만 학습.

왜 token-level가 아니라 trajectory-level?

"낙서가 있는 단어만 지우고 나머지는 채점"하면 ##나머지가 어떻게 점수를 받아야 할지 헷갈립니다##(credit assignment 문제). 답안지 전체를 빼면 깔끔. 마찬가지로 trajectory 전체를 빼는 게 token-level masking보다 효과적.

활용 분야

multi-turn agent RL이 필요한 모든 영역에 적용 가능한 일반 원리

비유로 다시 정리

비유 1: 외국어 듣고 헷갈리는 한국어

한국어 잘하던 사람이 외국어 한 마디 듣고 약간 헷갈리고, 또 듣고 더 헷갈리고... 결국 ##혼잣말 같은 알 수 없는 소리##. 이 사람의 마지막 대답만 보고 학습하면 모든 게 망가짐.

비유 2: 답안지 통째로 빼기

학생들 답안지 중 ##판독 불가한 낙서##가 섞인 답안지는 통째로 채점에서 제외. 깨끗한 답안지만으로 채점 기준 학습. 부분 마스킹보다 깔끔.

비유 3: 손이 꼬인 신입사원

신입이 첫 일은 잘했는데 그 후 도구 사용에 헷갈려 ##점점 일이 꼬이고 마지막엔 손도 못 댐##. 그 일과의 전체 결과로 신입을 평가하면 부당. SimpleTIR은 "##꼬인 일과 전체는 평가에서 빼고 정상적 일과만 평가##".

비유 4: void turn = 멈춤 신호

야구 경기에서 ##공이 굴러 멈추면 죽은공##(dead ball) 처리. SimpleTIR의 void turn도 ##추론이 죽은 시점##의 명확한 신호. 미시적 토큰 확률을 보지 않고 거시적 멈춤만 보면 됨.

비유 5: 게임 캐릭터의 죽음 처리

RPG에서 캐릭터가 죽으면 그 ##플레이 세션 전체##의 보상을 무효화. 죽기 직전의 행동만 페널티 줘서는 ##정상 플레이까지 페널티##. SimpleTIR이 똑같이 ##trajectory 단위##로 처리.

비유 6: 강의 없는 자유 학습

SFT 사전 학습이 ##커리큘럼 따라 가르치는 학원##이라면, Zero-RL은 ##시험 점수만 보고 학생이 자유롭게 공부##. 후자가 ##더 다양한 학습법##을 발견. SimpleTIR이 후자를 안정적으로 가능케 함.

한계점

⚠️ 1. 수학 task만 평가

AIME, MATH, Olympiad 6개 math benchmark만. ##코딩, agent task (SWE-bench), 웹 브라우징##에서의 효과는 미지수.

⚠️ 2. Void turn 정의가 task 특화

"complete code block OR final answer"는 ##수학 TIR 특화##. 일반 agent (예: web browsing)에서 void turn을 어떻게 정의할지는 ##추가 연구 필요##.

⚠️ 3. ReTool과의 비교 공정성

ReTool은 ##Math-Instruct cold-start##, SimpleTIR은 ##base from scratch##. 둘은 다른 setup. 같은 base에서 비교한 ZeroTIR 대비로는 SimpleTIR이 압도적이지만, ReTool 67% vs SimpleTIR 59.9% (AIME24)는 base의 차이가 큼.

⚠️ 4. Filtered trajectory 비율 미보고

학습 중 ##몇 %의 trajectory가 void로 제외##되는지 미공개. ##sample efficiency 평가 어려움##.

⚠️ 5. Max turn에 따른 task 의존성

Math500은 turn 늘릴수록 향상하지만 ##AIME24는 일관된 향상 없음##. 어떤 task가 multi-turn benefit을 받는지 ##task-level 분석 부재##.

역사적 위치

SimpleTIR은 ##tool-use RL의 instability 문제를 깔끔히 해결##한 milestone입니다.

  • 2023 2월: Toolformer — self-supervised tool learning
  • 2025 1월: DeepSeek R1 — Zero-RL paradigm 입증 (text reasoning)
  • 2025 4월: ReTool — RL + tool 결합 (cold-start SFT 필요)
  • 2025 7월: Effective TIR, ARPO 등 — 다양한 cold-start TIR
  • 2025 8월: ZeroTIR (Mai et al.) — 첫 Zero-RL TIR 시도
  • 2025 9월: SimpleTIR — void turn filtering으로 Zero-RL TIR 안정화 + SOTA

##Zero-RL TIR##의 진정한 첫 SOTA로, ##DeepSeek-R1 패러다임을 tool-use에 확장##.

FAQ

Q: ReTool과 어떤 차이?

A: ReTool은 ##cold-start SFT 후 RL##. SimpleTIR은 ##base에서 직접 RL##. ReTool은 안정성을 SFT로 얻고 패턴 다양성을 잃음. SimpleTIR은 void turn filter로 둘 다 얻음.

Q: void turn은 정확히 어떻게 식별?

A: turn의 응답이 (a) 완전한 <code>...</code> 블록도 없고 (b) final_answer도 없으면 void. 단순 규칙으로 식별.

Q: 왜 token-level masking이 부족했나?

A: ##credit assignment 문제##. 일부 token만 mask해도 trajectory 전체의 advantage는 한 reward에 의존. ##망친 부분만 빼면 정상 부분이 잘못된 점수##를 받음. 통째로 빼야 함.

Q: SimpleTIR이 다른 분야에 적용 가능?

A: 일반 원리(##distributional drift 식별 + trajectory-level filtering##)는 모든 multi-turn agent RL에 적용 가능. 다만 void turn의 정확한 정의는 ##task별로 customize 필요##.
