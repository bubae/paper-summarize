TL;DR

  • 문제: 강력한 speech-to-text (S2T) LLM을 speech-to-speech (S2S) 모델로 확장할 때, backbone을 fine-tuning하면 원래의 S2T 성능이 저하되고, downstream talker를 붙이면 직렬 TTS 병목이 재발생한다.
  • 해결: ##PRIME-Speech는 backbone을 완전히 freeze하고, 중간층 hidden state를 speech-text 동기화 인터페이스로 사용하여 causal audio post-decoder를 concurrent하게 실행한다##. 완성된 텍스트나 chunk를 기다리지 않고 hidden-state trajectory에서 직접 codec token을 생성.
  • 왜 작동하는가: 중간층 hidden state가 이미 semantic + paralinguistic 정보를 풍부하게 담고 있으며(CKA 분석 기반), 이를 concurrent하게 소비하는 audio branch는 backbone의 reasoning trajectory와 자연스럽게 timestamp 수준으로 동기화된다. Multi-turn 시에는 text KV cache만 축적하고 audio KV cache는 turn마다 reset하여 stale acoustic drift를 방지.
  • 비유: 회의 통역사가 발표자의 완결된 문장을 기다리지 않고, 발표자의 사고 흐름(중간 hidden state)을 실시간으로 청취하며 병렬로 통역을 내보내는 방식. 대신 발표자(text)와 통역(audio)의 노트는 분리해서 관리한다.

핵심 기여 (Key Contributions)

  1. Frozen-backbone S2S 프레임워크 공식화: S2S 확장을 "frozen-backbone conversion" 문제로 정의하고, ##중간층 hidden state를 preserved S2T reasoning과 trainable speech generation 사이의 인터페이스##로 제안.
  2. Concurrent audio post-decoder: hidden-state, text, audio-history를 혼합한 conditioning을 통해 forced alignment, 완성된 응답, 고정 text chunk 없이 backbone의 evolving state에서 직접 codec을 생성.
  3. Turn-level cache policy: multi-turn 대화에서 ##text cache는 축적(dialogue memory)하고 audio cache는 turn마다 reset(cross-turn acoustic drift 방지)##하는 비대칭 정책.
  4. MTP as efficiency adapter: Multi-Token Prediction을 semantic alignment가 아닌 순수 efficiency component로 사용. codec 예측 rate를 25Hz → 6.25Hz로 감소시키면서 reasoning path 무변경.

1. Introduction — 문제 설정

1.1 배경: S2S 시스템의 구조적 딜레마

Speech-to-speech (S2S) 상호작용은 language assistant의 자연스러운 목표이다. 현재의 지배적인 해법은 cascade: ASR → text LLM → TTS의 파이프라인. 편리하지만 recognition error, reasoning delay, 그리고 완성된 텍스트가 나올 때까지 speech generation이 미뤄지는 serial bottleneck을 겪는다.

이 병목을 없애기 위해 LLM-centric S2S 시스템들이 등장했지만, 여기에 근본적 긴장이 있다: 강력한 S2T LLM이 이미 존재하는데, 여기에 speech output을 붙이면서 backbone을 훼손시키지 않을 방법이 무엇인가?

1.2 기존 아키텍처의 한계

접근법동작 방식한계
Unified token-interleavingtext token과 audio token을 하나의 autoregressive stream에 배치주 decoder가 heterogeneous text/audio 목적함수를 동시에 balance해야 함 → S2T 저하
Decoupled Thinker–Talkerspeech rendering을 reasoning에서 분리talker가 finalized text, text chunk, text-side handoff state에 의존 → 이미 완성된(materialized) 표현을 따를 뿐, evolving hidden state는 무시
MTP on lightweight talker가벼운 talker에 multi-token prediction 적용thinker path는 그대로여서 end-to-end speedup 제한. 반대로 main backbone에 MTP를 적용하면 S2T 훼손 risk

PRIME-Speech(Preserving Reasoning and Intelligence while enabling More Efficient Speech-to-Speech)는 이 세 축을 동시에 다룬다: complete S2T backbone freeze + hidden-state synchronization + audio-branch MTP.

2. Method

2.1 개요: Frozen Backbone + Concurrent Audio Branch

입력 x = (x^τ, x^a)에서 x^τ는 optional text prompt, x^a는 입력 speech waveform. Frozen backbone은 text response y^τ를 생성하고, 추가된 audio branch는 codec-token response y^a를 생성한다. 변환은:

$$P(y^\tau, y^a \mid x) = P_{\text{bb}}(y^\tau \mid x) \, P_{\text{aud}}(y^a \mid y^\tau, H^{\text{mid}}; \theta_a) \tag{1}$$

여기서 P_bb는 frozen backbone 분포, P_aud는 trainable audio-branch 분포, H^mid는 중간층 hidden state sequence, θ_a는 audio branch 파라미터. 스트리밍 시 y^τ에 대한 의존은 prefix-restricted 되어, decoding update s에서 frozen backbone은 hidden state h^mid_s를 노출하고, text head와 audio branch가 이 state를 parallel로 소비한다.

##직관적으로 말하면, backbone은 여전히 speech perception과 reasoning을 전담하고, audio branch는 preserved backbone trajectory의 "synchronized speech readout"으로 작동한다.## Text token은 audio branch가 다음 update에서 참조하지만, 같은 update의 audio는 아직 미출력 상태의 concurrent text에 조건화되지 않는다.

2.2 Frozen Backbone과 Codec Target

Backbone은 speech encoder + projection module로 acoustic embedding e^sp를 만들고, text embedding e^τ와 concat하여 frozen transformer stack이 처리:

$$h^{(\ell)} = \text{Backbone}^{(\ell)}([e^\tau, e^{\text{sp}}]), \quad \ell = 1, \dots, L. \tag{2}$$

Speech-conditioning interface로 고정된 중간 layer stream H^mid = h^(ℓ_mid)를 사용. ℓ_mid는 backbone의 약 2/3 지점 layer로 설정. 이 선택은 layer-wise centered kernel alignment (CKA) 분석에서 이 depth가 ##paralinguistic 정보가 가장 풍부하면서 semantic grounding도 유지##되는 층임을 발견하여 결정되었다.

Target speech는 CosyVoice2 tokenizer (25Hz)로 semantic codec token으로 표현:

$$y^a = \{y^a_t\}_{t=1}^{T_a}, \quad y^a_t \in \{1, \dots, V_a\} \tag{3}$$

T_a는 codec frame 수, V_a는 codec vocabulary size. Audio post-decoder가 이 token을 예측하고, paired codec decoder가 다시 waveform으로 변환.

2.3 Hidden-State-Synchronized Audio Post-Decoder

Audio post-decoder는 frozen text path와 동일한 streaming update loop에서 실행되는 causal transformer. 이 loop를 s로 인덱싱. Update s에서 frozen backbone state h^mid_s가 두 branch로 fan out: text head는 다음 text token y^τ_s를 예측하고, audio post-decoder는 audio block y^a_s = (y^a_{s,1}, \dots, y^a_{s,B_s})를 예측. Flat codec sequence (Eq. 3)는 이 loop 내 연속 block으로 간주. MTP 이전에는 B_s = 1, MTP 활성화 시 B_s ≤ k.

두 branch는 concurrent지만 audio branch는 causal: 현재 hidden state와 s 이전에 committed된 history에 조건화되며, 같은 update에서 예측 중인 text token에는 의존하지 않는다. Post-decoder 모델:

$$P(\mathbf{y}^a_s \mid \mathbf{y}^a_{두 branch가 output을 emit한 후, y^τ_s와 y^a_s가 commit되어 s+1의 history가 된다. ##Synchronization은 timestamp-level (chunk-level이 아님)##: PRIME-Speech는 완성된 text, 고정 text chunk, word-to-frame alignment를 기다리지 않고, audio branch가 backbone trajectory에서 detach된 free-standing clock으로 실행되지도 않는다.

2.4 Mixed Conditioning Vector

Conditioning state는 세 신호를 결합: synchronized backbone state, 이전 text embedding, recent audio history. e^τ_{s-1}은 직전 committed text token의 embedding, r^a_{s-1}은 직전 audio update가 commit한 codec token embedding의 평균:

$$r^a_{s-1} = \frac{1}{B_{s-1}} \sum_{j=1}^{B_{s-1}} e^a_{s-1,j} \tag{5}$$

s=1일 때는 BOS embedding 사용. Mixed conditioning vector는:

$$h^{\text{mix}}_s = w_h h^{\text{mid}}_s + w_\tau e^\tau_{s-1} + w_a r^a_{s-1} \tag{6}$$

실험에서는 w_h = w_τ = w_a = 1.0의 단순한 fixed-weight setting을 사용 (held-out ablation에서 최적으로 선정). 세 항의 역할:

  • h^mid: semantic state (backbone reasoning의 현재 상태)
  • e^τ: lexical anchoring (직전 결정된 단어에 lexical 정렬)
  • r^a: local acoustic continuity (직전 audio 흐름의 매끄러움)

따라서 text와 audio는 single interleaved token stream이 아니라 parallel branch로 생성되며, 각자의 committed cache를 유지하고, hidden state가 synchronization interface 역할을 한다.

2.5 학습 목적

학습은 동일한 causal graph를 따른다. Teacher forcing에서 frozen backbone은 reference text prefix로 평가되고, audio post-decoder는 s-1까지의 reference history를 받는다. 현재 audio block만 speech-generation loss에 기여. 추론 시에는 generated text token과 audio block이 reference history를 대체.

2.6 Multi-Token Prediction (MTP) for Codec Efficiency

Codec token은 25Hz로 생성되어 autoregressive audio decoding이 response latency를 지배할 수 있다. PRIME-Speech는 single-token 학습으로 stable hidden-state alignment를 확보한 뒤, audio post-decoder에 MTP head를 부착. Update s에서 post-decoder state는 k개의 미래 single-codebook codec token 분포를 예측:

$$p_{s,i} = P(y^a_{s,i} \mid \mathbf{y}^a_{목적함수는 유효 미래 token loss의 가중합:

$$\mathcal{L}_{\text{mtp}} = -\sum_s \sum_{i=1}^{k} \lambda_i \log p_{s,i}(y^a_{s,i}) \tag{8}$$

utterance boundary를 넘어가는 position은 mask 처리. 추론 시 각 synchronized audio update가 최대 k개의 codec token을 commit하여 effective codec prediction rate가 25Hz → 25/k Hz로 감소.

##MTP는 여기서 semantic alignment가 아닌 순수 efficiency adapter로 작동##한다. PRIME audio branch를 frozen reasoning pathway 변경 없이 가속.

2.7 Multi-Turn Packing과 Cache Reset Policy

실제 multi-turn S2S 데이터는 수집 비용이 크고, 긴 cross-turn audio history로 학습하면 불안정. PRIME-Speech는 additional multi-turn S2S supervision을 요구하지 않는다. 대신 frozen S2T backbone의 기존 dialogue-level text context 유지 능력을 재사용하고, audio branch는 turn-level cache policy로 학습.

Training packing: 관련 없는 single-turn 예제를 packed pseudo-dialogue로 concat. Audio branch가 text-side context와 explicit turn boundary에 노출되지만, 이전 턴의 acoustic realization을 다음 턴 context로 취급하지 않도록 방지.

Text KV state는 across turns로 축적, audio KV state는 각 assistant turn마다 reset:

$$P(y^a_t \mid \mathbf{C}^{(Audio position도 동일 boundary 적용. Packed single-turn 예제 학습 시 text position은 packed sequence에서 global 유지, audio positional index는 매 packed segment 시작 시 reset. 추론 시에도 새 assistant response마다 reset. 토큰 index i, modality tag m_i, 현재 audio segment 시작 index s_n일 때 audio branch가 사용하는 position:

$$\mathcal{P}^{(n)}(i) = \begin{cases} i, & m_i \in \text{Text}, \\ i - s_n, & m_i \in \text{Audio}_n. \end{cases} \tag{11}$$

##이 training-time packing과 turn-local audio reset의 조합이 backbone의 text-side multi-turn 능력을 활용하면서 stale audio state가 repetition/drift를 유발하는 것을 방지##한다.

2.8 알고리즘 흐름도

[Input: (x^τ, x^a)] ↓ Speech Encoder + Projection → e^sp Text Embedding → e^τ ↓ [Frozen Backbone (all L layers)] ├─→ Text Head → y^τ_s (concurrent) └─→ h^mid_s (mid-layer, ~2/3 depth) ↓ [Mixed Conditioning] h^mix = w_h·h^mid + w_τ·e^τ_{s-1} + w_a·r^a_{s-1} ↓ [Audio Post-Decoder (2B params, 10 causal layers)] ↓ [MTP Head (100M, k-step)] ↓ y^a_{s,1..k} → codec tokens (25/k Hz) ↓ [CosyVoice2 Codec Decoder → waveform] Multi-turn: Text KV cache: accumulate across turns (dialogue memory) Audio KV cache: reset at each turn (avoid stale drift)

3.2 Model Configuration

PRIME-Speech는 Phi-4-MM-7B (advanced S2T LLM) 기반. Backbone은 2M hours speech data + 5T text tokens로 pre-training. Trainable modules:

  • Audio post-decoder: ~2B params, 10 causal transformer decoder layers. 각 layer는 backbone의 hidden size와 attention config를 따라 H^mid와 speech branch 간 mismatch 감소.
  • MTP module: multi-head MLP, hidden dim 2048, ~100M params.

Frozen modules: speech encoder, projection, transformer backbone, text LM head, text decoding path (학습·평가 전 구간). ##S2S 변경분은 학습된 speech branch에만 귀속되며, S2T 동작은 원본 backbone이 지배##한다는 것이 이 setup의 핵심.

3.3 Training Curriculum (2단계)

Stage 1 — Audio-branch training: audio post-decoder를 native 25Hz codec rate에서 standard next-token codec prediction으로 학습. Full task-balanced mixture 1 epoch, AdamW, learning rate 1×10^-4 linear decay. LibriHeavy가 dense audio-text correspondence를 제공하여 acoustic-token modeling을 stabilize, translation/spoken QA/assistant-style 예제가 post-decoder에게 verbatim transcript만이 아닌 inferred/generated content를 render하도록 가르침.

Stage 2 — MTP training: MTP 활성화, 동일 learning rate schedule로 20k step. Pure alignment 예제는 down-weight, translation과 spoken QA 예제는 active 유지. 이 시점에는 post-decoder가 stable hidden-state conditioning을 학습했으므로, MTP는 codec-token prediction을 압축하는 efficiency component로 학습.

4.1 Baselines & Metrics

비교 대상: GLM-4-Voice, Kimi-Audio, Step-Audio-2-Mini, VocalNet, Qwen2.5-Omni, Qwen3-Omni, GPT-4o, 그리고 frozen Backbone-LLM. Backbone-LLM은 speech-generation branch가 없으므로 S2T mode만 리포트.

  • Spoken QA: UltraEval-Audio (LLaMA-QA, TriviaQA, WebQ)

4.2.1 Reasoning path preservation

Backbone-LLM과의 controlled 비교는 preservation의 가장 직접적 증거. ##Translation, spoken QA, BigBench-Audio 전반에서 PRIME-Speech의 S2T score가 frozen backbone과 거의 unchanged##. Speech-generation branch가 text pathway를 rewrite하지 않으므로, S2S gain은 backbone의 원래 speech understanding/reasoning을 희생시켜 얻은 것이 아니다.

4.2.2 From text correctness to spoken correctness

S2S column은 audio branch가 backbone의 semantic decision을 보존하는지 검사. Translation과 spoken QA에서 PRIME-Speech는 strong text response를 small degradation, consistently low WER로 speech로 변환. Multi-turn과 BigBench-Audio 결과가 특히 유의미 (transcript 재현에 덜 tied되므로): 작은 S2T–S2S gap은 audio branch가 backbone response를 따르고 있음(speech-side reasoning error를 추가하는 것이 아님)을 시사. ##PRIME-Speech는 경쟁적 task accuracy를 유지하면서 리포트된 S2S 태스크 중 가장 낮거나 낮은 수준의 rendering WER를 달성##.

4.3.1 왜 backbone을 freeze하는가

LoRA + Post LM과 LoRA + ESI variant는 같은 side data를 사용하지만 backbone을 update. In-domain S2S metric에서는 viable하지만, ##BigBench-Audio S2T score가 낮아 reasoning model을 adapting하는 비용을 명시적으로 보여줌##. 이는 PRIME-Speech의 decomposition을 지지: reasoning은 frozen backbone에 남고, speech realization은 post-decoder에서 학습된다.

4.3.2 MTP가 기여하는 것

MTP는 semantic alignment가 아닌 efficiency adapter로 해석해야 한다는 것이 중요한 실험 의도. Stage-1 model → k=1 model 비교는 audio curriculum이 이미 rendering quality를 개선함을 보여준다(codec rate 감소 전에도).

k=1에서 k=4로 horizon 증가 시 effective codec prediction rate가 25Hz → 6.25Hz로 감소하면서도 S2S task score는 broadly stable. k=4 setting은 smaller horizon에서 관찰되는 main quality (UltraEval-Audio, multi-turn, BigBench-Audio S2S 포함)를 preserve. 따라서 ##MTP는 의도된 역할을 수행: frozen reasoning path 변경 없이 또는 significant task-quality loss 없이 codec-generation loop을 압축##.

5. Conclusion & 저자 요약

PRIME-Speech는 강력한 S2T LLM에 S2S 능력을 reasoning backbone 변경 없이 추가하는 프레임워크. Speech-to-speech translation, spoken conversation, speech understanding에서 강한 결과를 달성하면서 small S2T-S2S modality gap을 유지. MTP는 high-rate codec token의 latency와 real-time factor를 개선하면서 frozen-backbone system의 전체 S2S behavior를 보존.

비판적 분석 (Critical Analysis)

강점

  • Preservation이 명시적으로 측정된 실험 설계: Backbone-LLM을 direct baseline으로 포함하여 S2T degradation을 controlled 방식으로 검증. 대부분의 S2S 논문이 이 비교를 생략하는 것과 대조적.
  • 3-way decomposition ablation: reasoning backbone freeze / audio branch / codec compression 세 효과를 별도로 분리한 것이 methodology의 강점. LoRA variant와의 비교가 특히 informative.
  • Multi-turn cache policy의 causal diagnostic: turn-wise WER trajectory로 acoustic drift를 명확히 드러냄. Average WER만 봤다면 실용상 큰 문제가 은폐되었을 가능성.
  • Data 효율: 대규모 multi-turn S2S 수집 없이도 packing + turn reset만으로 stable multi-turn 성능 달성. Practical한 recipe.

약점 및 의문점

  • Backbone dependency: 이 접근법의 성능은 근본적으로 backbone S2T LLM의 품질에 의존. Phi-4-MM-7B라는 이미 강한 S2T 모델을 전제하므로, 약한 backbone에서 동일한 gain을 얻을 수 있는지는 unclear.
  • Layer selection의 heuristic 성격: ℓ_mid를 backbone의 ~2/3 depth로 CKA 분석 기반 선택했다고 하지만, layer selection에 대한 systematic ablation (예: 다양한 depth의 성능 비교)이 부재. Backbone마다 optimal layer가 달라질 가능성.
  • Expressive prosody 무평가: 저자들이 명시적으로 evaluation을 transcript-level correctness와 rendering WER에 한정. Speaker consistency, emotional prosody, paralinguistic 요소는 평가 대상에서 제외. GPT-4o, Kimi-Audio 등의 강점 영역과 직접 비교 불가.
  • Target speech가 synthetic: Azure TTS로 생성된 target을 사용하므로 semantic alignment는 clean하나, 실제 human speech distribution과의 gap이 있을 수 있음. 실제 사람 대화 style로 확장 시 성능 저하 가능성.
  • Post-decoder 크기의 정당화 부족: 2B params post-decoder + 100M MTP는 결코 lightweight talker가 아님. 저자들도 인정하듯 raw codec throughput은 VocalNet 대비 뒤처짐. "왜 굳이 2B가 필요한가"에 대한 크기 ablation이 부재.
  • Multi-lingual 대응: 실험은 대부분 EN target에 국한. 다국어 out target에서 hidden-state synchronization이 얼마나 견고한지 미검증.
  • Mixed conditioning weight의 임의성: w_h = w_τ = w_a = 1.0이 "held-out ablation에서 최적"이라 했으나, ablation의 세부 데이터가 부재.

재현성

  • Code/checkpoint 공개 여부 명시 없음. GLM-4-Voice, VocalNet 등과 달리 재현성이 제한적.
  • Phi-4-MM-7B backbone은 공개되어 있으나, 학습된 audio post-decoder weight의 공개 여부가 불투명.
  • In-house X2EN 10k hours, Azure TTS synthesized target data는 재현 시 재구성이 필요.

Key Takeaways — 이 논문에서 가져갈 것

  • "Freeze the reasoner, train the renderer": 강력한 S2T LLM이 이미 존재할 때, 이를 훼손시키지 않는 유일한 견고한 방법은 backbone을 완전히 freeze하는 것. S2S 확장 시 첫 번째 선택지로 고려할 architecture principle.
  • Hidden-state는 chunk보다 fine-grained synchronization interface: 완성된 텍스트, 텍스트 chunk, forced alignment 없이도 중간층 hidden state를 concurrent branch의 conditioning으로 활용할 수 있음. Thinker-Talker의 한계를 넘어서는 실용적 대안.
  • Text/audio KV cache 비대칭 관리: multi-turn에서 두 modality의 cache 정책을 분리하는 것이 stale acoustic drift 방지에 필수. 단순한 아이디어지만 3턴 이상 대화에서 성능이 무너지는 것을 방지하는 핵심 요소.
  • MTP는 alignment tool이 아닌 efficiency adapter: MTP의 semantic 학습 효과를 기대하지 말고, 안정된 single-token 학습 후 pure compression module로 사용하는 것이 clean한 사용법.
  • Turn-wise WER trajectory 리포트: multi-turn S2S 시스템 평가 시 average WER 대신 turn-position별 성능 저하 pattern을 살펴봐야 stale state 문제를 놓치지 않는다.
