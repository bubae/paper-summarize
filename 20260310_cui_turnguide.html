TurnGuide: Enhancing Meaningful Full Duplex Spoken Interactions via Dynamic Turn-Level Text-Speech Interleaving

TL;DR

  • 문제: End-to-end (e2e) Full-Duplex Speech Language Model (FD-SLM)은 실제 이중 채널 대화 데이터로 학습하여 자연스러운 대화가 가능하지만, 텍스트 기반 LLM 대비 의미적 품질(semantic quality)이 크게 떨어진다. 텍스트를 음성 스트림에 끼워 넣어 가이드하려 해도, 삽입 시점(insertion timing)과 삽입 길이(insertion length)가 정밀하지 않으면 오히려 대화 흐름이 깨진다.
  • 해결: TurnGuide는 어시스턴트 음성을 동적으로 "대화 턴(turn)" 단위로 분할하고, 각 턴에 대응하는 텍스트를 turn 수준에서 interleave하여 학습 데이터를 구성한다. Channel-wise interleaving (사용자/어시스턴트 음성 청크 교차)과 text-speech interleaving (텍스트 청크를 각 턴의 첫 음성 청크 위치에 삽입) 두 전략을 결합한다.
  • 왜 작동하는가: 턴 단위 분할 덕분에 텍스트가 의미 단위로 응집되어 모델이 텍스트 정보를 효과적으로 활용할 수 있고, 턴의 시작점에만 삽입하므로 단어별 정밀 타임스탬프에 대한 의존성이 줄어 정렬 오류에 강건하다.
  • 비유: 라디오 진행자가 대본을 한 단어씩 보며 말하는 대신(Moshi TS), 한 문단씩 미리 읽고 나서 자연스럽게 이야기하는 것과 같다. 문단 단위로 의미를 파악했기 때문에 대화가 더 일관되고, 정확한 단어 타이밍에 얽매이지 않아도 된다.

핵심 기여 (Key Contributions)

  1. Turn-level text-speech interleaving 방식 제안: 기존의 token-level 삽입(Moshi TS)이 아닌, ##동적 턴 분할 + 턴 수준 텍스트 삽입##이라는 새로운 접근으로 e2e FD-SLM의 의미적 품질과 turn-taking 자연스러움을 동시에 개선한다.
  2. Dynamic Turn Segmentation and Alignment 프레임워크: VAD → IPU merging → ASR + timestamp extraction → word grouping 파이프라인을 통해 비정렬 이중 채널 음성 데이터에서 텍스트-음성 정렬된 턴 단위 학습 데이터를 자동 생성한다.
  3. Text-Guided Full Duplex Dialogue Modeling 프레임워크: Channel-wise interleaving과 text-speech interleaving 두 전략을 결합하여, 이중 채널 대화 모델링과 텍스트 가이던스를 동시에 수행한다.
  4. 광범위한 실험적 검증: Semantic GPT-score에서 베이스라인 대비 30% 이상 개선, Full-Duplex-Bench에서 거의 모든 turn-taking 메트릭에서 SOTA 달성, 인간 평가(MOS)에서도 최고 점수를 기록한다.

1. Introduction — 배경과 동기

Full-Duplex Speech Language Model이란?

Speech Language Model (SLM)은 음성 기반 사용자 인터랙션을 위한 foundation model이다. Full-Duplex SLM (FD-SLM)은 그 중에서도 사용자와 어시스턴트의 이중 채널 오디오를 동시에 처리하여 실시간 대화를 가능하게 하는 모델로, 끼어들기(interruption), 맞장구(backchannel), 겹침(overlap) 등 복잡한 대화 턴 전환을 자연스럽게 처리하는 것을 목표로 한다.

FD-SLM은 크게 두 유형으로 나뉜다:

  • Cascaded FD-SLM: 별도의 state predictor가 대화 상태(예: "interrupted", "keep speaking")를 예측하고, 이에 따라 SLM의 행동을 제어한다. LSLM, Mini-Omni 2, Freeze-Omni, Minmo 등이 이 방식이다. 규칙 기반 상태 전이에 의존하므로 자연스러운 대화에 한계가 있다.
  • End-to-end (e2e) FD-SLM: 실제 이중 채널 대화 데이터로 직접 학습하여 대화 역학을 모델 자체가 포착한다. dGSLM, SyncLLM, Moshi, OmniFlatten, NTPP 등이 이 방식이다. 더 자연스럽지만, ##음성만으로 학습하면 텍스트 기반 LLM 대비 의미적 품질이 크게 떨어진다##.

Text-speech interleaving의 필요성과 어려움

SLM의 의미적 능력 저하를 완화하는 유망한 방법은 text-speech interleaved generation, 즉 텍스트를 먼저 생성하고 이를 가이드로 활용하여 음성을 생성하는 것이다. 이는 TLM (Text-based Language Model)의 지식을 SLM으로 전이하는 효과가 있다.

그러나 e2e FD-SLM에 텍스트 가이던스를 적용하는 것은 두 가지 핵심 도전 과제를 야기한다:

  1. 삽입 시점 (Insertion timing): 텍스트를 너무 일찍 삽입하면 사용자 맥락 없이 응답하여 hallucination이 발생하고, 너무 늦게 삽입하면 음성이 텍스트보다 먼저 나와 가이던스 효과가 사라진다.
  2. 삽입 길이 (Insertion length): 텍스트가 너무 길면 모델이 과도하게 발화(overspeak)하고 후속 사용자 입력에 적응하지 못하며, 너무 짧으면 의미가 단편화되어 가이던스 효과가 약해진다.
[Figure 1 설명]: 왼쪽은 Cascaded FD-SLM으로, SLM 아래에 State Predictor가 있어 <start_speaking>, <stop_speaking> 같은 상태를 예측하고 이에 따라 음성 생성을 제어한다. 오른쪽은 E2e FD-SLM으로, 별도 상태 예측기 없이 이중 채널 오디오를 직접 모델링하여 Parallel Speaking, Smooth Turn-Taking, Backchanneling 등 다양한 대화 이벤트를 자연스럽게 처리한다. 단, Fragmented Speech (단편적 발화)와 같은 의미적 문제가 발생할 수 있다.

2. Related Works

2.1 Full Duplex Speech Language Models

Cascaded FD-SLMs

Cascaded 방식은 state predictor + backbone SLM 두 컴포넌트로 구성된다. State predictor가 유한 개의 사전 정의 상태를 예측하여 SLM의 행동을 제어한다:

  • LSLM (Ma et al., 2025): "interrupted"와 "keep speaking" 상태, 특정 키워드로 끼어들기 트리거
  • Mini-Omni 2 (Xie & Wu, 2024): "start speaking"과 "stop speaking" 상태, "Stop Omni!" 발화로 생성 중단
  • Freeze-Omni (Wang et al., 2024): 사용자 발화 종료 여부 기반 상태 결정
  • Minmo (Chen et al., 2025): 모델이 발화해야 할지, 계속 청취할지를 결정

이 모든 방법은 합성 대화 데이터에 의존하며 사전 정의 상태로 제한되어, 실제 인간 대화의 복잡한 역학을 포착하기 어렵다.

End-to-end FD-SLMs

E2e 방식은 transformer 기반 아키텍처로 실제 이중 채널 대화 오디오를 직접 학습한다:

  • dGSLM (Nguyen et al., 2023): 각 채널을 별도 Transformer로 처리하고, cross-attention으로 채널 간 상호작용을 모델링하는 dual-Transformer 구조
  • SyncLLM (Veluri et al., 2024): 단일 LLM 내에서 이중 채널 오디오를 통합 시퀀스로 jointly 모델링
  • Moshi (Défossez et al., 2024): RQ-Transformer 아키텍처로 사용자 입력을 기반으로 어시스턴트 출력을 예측. 텍스트 토큰을 해당 음성 토큰과 함께 삽입하지만, 텍스트를 과도하게 단편화하여 성능 향상이 미미
  • OmniFlatten (Zhang et al., 2024): 이중 채널 오디오를 단일 통합 시퀀스로 변환하여 GPT 모델로 처리
  • NTPP (Wang et al., 2025): Next-Token-Pair Prediction으로 양 채널 토큰을 동시에 예측

다만, 이 모델들의 대다수는 아직 완전히 오픈소스되지 않았기 때문에 (Table 1 참조), 본 논문에서는 주로 dGSLM, Moshi, 그리고 동일 backbone/데이터로 학습한 자체 베이스라인(STI, SCI, Moshi TS)과 비교한다.

2.2 Text-Guided Speech Generation in SLMs

SLM은 TLM 대비 성능이 낮으며, 이를 완화하기 위해 텍스트 정보를 통합하는 두 가지 접근이 있다:

  • Representation alignment: ASR/TTS 태스크를 통한 사전학습 단계에서 텍스트-음성 표현을 정렬
  • Text-speech interleaved generation: 텍스트 응답을 먼저 생성하고 이를 음성 생성의 가이드로 활용

현재까지 e2e FD-SLM에 text-guided generation을 적용한 유일한 연구는 Moshi이다. 그러나 Moshi는 각 텍스트 토큰을 해당 음성이 발화되는 정확한 시점에 삽입하는 token-level 방식을 사용하여, 텍스트 의미를 과도하게 단편화시키고 SLM 성능을 유의미하게 개선하지 못한다. 반면 TurnGuide는 텍스트를 턴 단위로 집약하여 삽입함으로써, 의미적 품질을 크게 향상시킨다.

3. Methodology

TurnGuide는 두 핵심 프레임워크로 구성된다: (1) Dynamic Turn Segmentation and Alignment Framework — 이중 채널 음성을 턴 단위로 분할하고 텍스트와 정렬하는 데이터 전처리 파이프라인, (2) Text-Guided Full Duplex Dialogue Modeling Framework — 정렬된 데이터를 활용하여 FD-SLM을 학습시키는 모델링 프레임워크.

3.1 Dynamic Turn Segmentation and Alignment Framework

이 프레임워크의 목표는 비정렬 이중 채널 음성 데이터에서 턴 단위 텍스트-음성 정렬 학습 데이터를 자동 생성하는 것이다.

오디오(A) → [VAD] 음성 구간 검출 → [IPU Merging] 연속 구간 병합 → [ASR + Timestamp Extraction] 단어별 텍스트 + 타임스탬프 → [Word Grouping] 각 IPU에 단어 할당 → {(W_j, S_j)} 턴-정렬 데이터

Step 1: VAD (Voice Activity Detection)

오디오 시퀀스 $A$에 pyannote 라이브러리를 적용하여 음성 구간을 검출한다:

$$V = \text{VAD}(A) = \{(\tau_1^{start}, \tau_1^{end}), (\tau_2^{start}, \tau_2^{end}), ...\} \quad (1)$$

각 튜플 $(\tau_i^{start}, \tau_i^{end})$는 $i$번째 음성 구간의 시작/종료 타임스탬프이다.

Step 2: IPU (Inter-Pausal Unit) Merging

VAD 결과는 너무 세분화되어 있으므로, 짧은 묵음(pause)으로 분리된 연속 구간을 하나의 IPU로 병합한다. IPU는 "동일 화자의 연속적인 발화 단위"로, 짧은 쉼표 정도의 묵음은 하나의 발화로 간주한다:

$$IPU_j = \bigcup_{i \in G_j} (\tau_i^{start}, \tau_i^{end}) \quad (2)$$

여기서 $G_j = \{i : \tau_{i+1}^{start} - \tau_i^{end} < \tau_{IPU}\}$이고, 병합 임계값 $\tau_{IPU} = 0.5$초이다. 직관적으로, 0.5초 미만의 묵음은 동일 발화 내 쉼(pause)으로 간주하여 하나의 턴으로 묶는다.

Step 3: ASR + Word-level Timestamp Extraction

Whisper medium 모델과 whisper-timestamped 패키지를 사용하여 전사(transcription)와 단어별 타임스탬프를 동시에 추출한다:

$$W = \text{ASR}(A) = \{w_1, w_2, ...\} \quad (3)$$ $$\Theta = \text{TE}(A) = \{(\delta_1^{start}, \delta_1^{end}), (\delta_2^{start}, \delta_2^{end}), ...\} \quad (4)$$

$w_i$는 전사의 $i$번째 단어, $(\delta_i^{start}, \delta_i^{end})$는 해당 단어의 시작/종료 타임스탬프이다.

Step 4: Word Grouping

각 단어를 해당 IPU에 할당한다. 타임스탬프의 불일치를 보정하기 위해 허용 오차 $\tau_{tol} = 0.6$초를 도입한다:

$$W_j = \{w_i : \tau_{IPU_j}^{start} - \tau_{tol} \leq \delta_i^{start} \leq \tau_{IPU_{j+1}}^{start} - \tau_{tol}\} \quad (5)$$

최종 출력은 정렬된 텍스트-음성 쌍 $\{(W_j, S_j)\}$으로, 각 텍스트 세그먼트가 대응하는 음성 세그먼트와 시간적으로 정렬된다.

[Figure 2 설명]: TurnGuide의 전체 프레임워크를 2단계로 도식화한다. 상단(Part 1)은 Dynamic Turn Segmentation으로, 원본 오디오에 ASR과 VAD를 병렬 적용하여 단어별 타임스탬프와 IPU를 얻고, Word Grouping으로 각 IPU에 단어를 할당하는 과정이다. 예시로 "OK, the thing is, London is a great, but it rains a lot"이 4개의 턴으로 분할된다. 하단(Part 2)은 Text-Guided Full Duplex Dialogue Modeling으로, 이중 채널 음성을 channel-wise로 교차한 뒤, 어시스턴트 채널에 텍스트 청크를 turn 단위로 삽입하는 text-speech interleaving 과정을 보여준다.

3.2 Text-Guided Full Duplex Dialogue Modeling Framework

정렬된 턴 데이터를 활용하여 FD-SLM을 학습시키는 프레임워크이다. Backbone으로 GLM-4-Voice를 사용한다. GLM-4-Voice는 speech tokenizer (오디오 → 이산 토큰), language model (토큰 처리), vocoder (토큰 → 음성 복원)를 포함하며, 강력한 텍스트-음성 정렬 능력을 갖추어 text-guided speech interaction의 이상적인 backbone이다.

Channel-wise Interleaving

이중 채널 대화 모델링을 위해, 사용자와 어시스턴트의 음성 토큰을 청크 단위로 교차 배치한다. 사용자 채널 토큰을 $s_u$, 어시스턴트 채널 토큰을 $s_a$라 하면, 각 채널을 청크 크기 $\tau_{SC} = 5$로 분할한다. GLM-4-Voice의 프레임 레이트가 $fr = 12.5$Hz이므로, 각 청크는 400ms의 오디오에 해당한다.

교차 배치된 시퀀스는 다음과 같다:

$SC = [SC_u^1, SC_a^1, SC_u^2, SC_a^2, ...]$

각 청크에는 화자 구분을 위한 고유 ID 토큰이 앞에 붙는다: $SC_u^i = [UID, s_u^{i \times \tau_{SC}+1}, ..., s_u^{i \times \tau_{SC}+\tau_{SC}}]$, $SC_a^i = [AID, s_a^{i \times \tau_{SC}+1}, ..., s_a^{i \times \tau_{SC}+\tau_{SC}}]$.

Text-speech Interleaving

##텍스트 가이던스는 어시스턴트 채널에만 적용##된다. 각 턴 $j$의 텍스트 $W_j$를 청크 크기 $\tau_{TC} = 5$로 분할하여 텍스트 청크 $TC_j = [TC_j^1, TC_j^2, ...]$을 생성한다. 마지막 텍스트 청크에는 "end of turn" ($\langle EOT \rangle$) 토큰을, 나머지 청크에는 "end of chunk" ($\langle EOC \rangle$) 토큰을 붙인다. 이를 통해 모델이 턴 경계 정보를 학습할 수 있다.

각 텍스트 청크 $TC_j^k$가 삽입될 음성 청크 위치 $k_j$는 다음으로 결정된다:

$$k_j = \lfloor \frac{\tau_j^{k0} \times fr}{\tau_{SC}} \rfloor \quad (6)$$

여기서 $\tau_j^{k0}$는 $k$번째 텍스트 청크의 첫 텍스트 토큰에 대응하는 타임스탬프이다. 직관적으로, 텍스트 청크가 발화되기 시작하는 시점의 음성 청크 위치에 텍스트를 삽입하여, ##텍스트가 항상 음성보다 먼저 (또는 동시에) 제공되어 가이던스 역할을 수행##하게 한다.

최종 시퀀스는 두 interleaving 전략이 결합된 형태이다:

$FC = [C_u^1, C_a^1, C_u^2, C_a^2, ...]$

여기서 $C_u^i = SC_u^i$이고, $C_a^i$는 텍스트가 있는 위치에서는 $[TC_j^k, SC_a^i]$, 없는 위치에서는 $SC_a^i$이다.

텍스트와 음성의 5:5 비율 설계 의도: 텍스트 생성이 음성 생성보다 항상 먼저 이루어지도록 하여, 효과적인 텍스트 가이던스를 보장한다.

4. Experiments

TurnGuide의 효과를 검증하기 위한 실험은 세 가지 핵심 Research Question을 중심으로 설계되었다:

  • RQ1: TurnGuide가 생성된 full-duplex 대화의 의미적 품질과 일관성을 유의미하게 개선하는가?
  • RQ2: TurnGuide가 텍스트를 추가로 삽입하면서도 자연스러운 대화 흐름(turn-taking)을 유지할 수 있는가?
  • RQ3: 텍스트 삽입 시점과 길이를 변경하면 모델 성능이 어떻게 변하는가?

4.1 Experimental Setups

Training Data

Fisher dataset (Cieri et al., 2004)을 사용한다. 이는 약 2,000시간 규모의 이중 채널 전화 대화 코퍼스이다. 선행 연구를 따라 120초 단위로 전처리하며, train/validation/test로 분할한다 (part ID 000-110/111-113/114-116). 테스트셋에서 약 1,000개 샘플을 무작위 선택하여 평가에 사용한다.

Baselines

  1. dGSLM (Nguyen et al., 2023): 각 채널을 별도 Transformer로 처리하는 dual-Transformer 구조
  2. STI (Speech Token Interleaving): GLM-4-Voice backbone에서 두 채널의 speech token을 하나씩 번갈아 배치하는 token-level interleaving
  3. SCI (Speech Chunk Interleaving): STI와 유사하나 청크 크기 5로 교차 배치하는 chunk-level interleaving
  4. Moshi TS (Moshi Training Strategy): Moshi의 text-speech token alignment 방식을 GLM-4-Voice에 적용한 것. 각 텍스트 토큰을 해당 음성이 발생하는 정확한 시점에 삽입
  5. Moshi: 원본 Moshi 모델 (별도 데이터/아키텍처이므로 직접 비교는 제한적)

4.2 Results

4.2.1 RQ1 — 의미적 품질 (Semantic Quality)

실험 의도: TurnGuide가 생성한 대화의 의미적 의미성(meaningfulness)과 일관성(coherence)이 기존 방법 대비 유의미하게 향상되는지를 검증한다.

GPT-score 평가: 생성된 이중 채널 대화 오디오를 Whisper-large-v3 + stable-ts로 전사하고, 양 채널의 문장을 시간순으로 정렬한 뒤, GPT-4o가 0~5 점 척도로 의미적 품질을 평가한다. Ground truth 텍스트(Fisher dataset)의 평균 GPT-score가 3.3이므로, 0~3.3 범위를 0~10.0으로 rescale하여 보고한다. GPT-4o 평가의 신뢰성은 인간 평가(Appendix A)에서 검증됨.

Moshi TS가 TurnGuide보다 낮은 성능을 보이는 세 가지 이유

  1. 텍스트 의미 단편화: Moshi TS는 텍스트 토큰을 하나씩 분산 삽입하여, 모델이 일관된 텍스트 표현을 학습하기 어렵게 만든다. TurnGuide는 텍스트를 턴 단위로 응집하여 이 문제를 해결한다.
  2. 음성의 본질적 단편성 미보완: 음성은 단어가 발음되는 시점에만 의미 정보를 담고 있어 본질적으로 단편적이다. Moshi TS는 이미 의미가 있는 지점에만 텍스트를 삽입하므로 의미 보강 효과가 제한적이다. TurnGuide는 턴 전체에 걸쳐 밀도 높은 의미 정보를 제공한다.
  3. 정밀한 단어 타임스탬프 의존: Moshi TS는 ASR의 정확한 단어별 타임스탬프에 크게 의존하지만, 실제로는 타임스탬프가 정확하지 않아 text-speech alignment가 깨질 수 있다. ##TurnGuide는 턴 시작 시점만 사용하므로 단어 수준 타임스탬프 정확도에 대한 의존성이 크게 줄어든다##.

4.2.2 RQ2 — Turn-Taking 자연스러움 (Full-Duplex-Bench)

실험 의도: TurnGuide가 텍스트를 시퀀스에 추가하면서도 자연스러운 대화 흐름을 유지할 수 있는지, 특히 Smooth Turn-Taking, User Interruption, Pause Handling, Backchanneling 네 가지 turn-taking 이벤트에서 성능을 평가한다.

Full-Duplex-Bench (Lin et al., 2025)는 이러한 turn-taking 이벤트를 세밀하게 평가하기 위해 설계된 벤치마크이다:

  • Smooth Turn-Taking: 사용자의 발화 턴이 끝난 후 자연스럽게 대화를 이어받는 능력
  • User Interruption: 사용자가 끼어들 때 새로운 내용을 인식하고 적응하는 능력
  • Pause Handling: 사용자가 짧은 묵음 후 다시 말할 때, 모델이 그 동안 침묵을 유지하는 능력
  • Backchanneling: 사용자가 말하는 동안 적절한 시점에 짧은 맞장구를 치는 능력

4.2.3 RQ3 — 삽입 시점/길이 Ablation

실험 의도: TurnGuide의 텍스트 삽입 시점과 길이가 성능에 미치는 영향을 분석하여, 동적 턴 분할의 중요성을 실증적으로 검증한다.

네 가지 시뮬레이션 시나리오를 설정한다:

4.3 학습 전략 비교 (Appendix C)

[Figure 5 설명]: 다양한 학습 전략의 training loss와 validation loss 비교. SCI는 STI 대비 큰 폭의 training loss 감소를 보이며, 이는 speaker ID 토큰 추가와 청크 구성이 학습을 용이하게 하기 때문이다. TurnGuide(Ours)와 L2:1, L3:1은 SCI와 STI보다 더 낮은 training loss를 달성한다. 반면 Moshi TS는 낮은 training loss를 기록하지만, 이는 반복적인 text pad 토큰의 예측 용이성 때문이며 실제 성능 향상으로 이어지지 않는다.

핵심 인사이트: Moshi TS의 낮은 training loss는 ##text pad 토큰의 반복성이 예측을 쉽게 만들어 인위적으로 loss가 낮아진 것##이며, 실제 성능과 괴리가 있다. 반면 TurnGuide의 loss 감소는 text-speech 정렬의 실질적 효과를 반영한다.

4.4 Latency Analysis (Appendix F)

TurnGuide의 e2e first-package latency는 약 1.05초이다:

  • $T_{st}$ (Speech Tokenization, 5 tokens): 0.01s
  • $T_{LLM}$ (LLM processing, pre-filling 5 user speech tokens + decoding 10 assistant tokens (5 text + 5 speech)): 0.3s
  • $T_{SD}$ (Speech Decoding, 10 speech tokens): 0.75s
  • $T_{ALL} = T_{st} + T_{LLM} + T_{SD} = 1.05$s

최적화 시 Speech Decoding을 5 토큰만으로 줄일 수 있으나, GLM-4-Voice vocoder가 최소 10 토큰을 요구하므로 추가 최적화가 필요하다.

4.5 Corpus-Level Turn-Taking (Appendix D)

Corpus-level 평가에서는 30초 프롬프트와 90초 continuation 간의 turn-taking 이벤트 (IPU, pause, gap, overlap)의 Pearson 상관을 측정한다. TurnGuide는 베이스라인과 비슷한 수준의 상관을 보이며, 텍스트 스트림 추가가 corpus-level 대화 패턴에 유의미한 변화를 일으키지 않음을 확인하였다. 단, 저자는 corpus-level 통계가 해석하기 어려우므로 Full-Duplex-Bench 결과를 더 신뢰할 것을 권장한다.

5. Conclusion & Limitations

결론

TurnGuide는 어시스턴트 음성을 턴 단위로 분할하고, channel-wise + turn-level text-speech interleaving으로 학습하여 e2e FD-SLM의 의미적 품질과 turn-taking 자연스러움을 동시에 개선한다.

한계

  • 데이터셋 한정: Fisher dataset (전화 대화)에서만 검증. 다화자, 다국어, 더 자발적인 대화 스타일로의 일반화 검증 필요
  • 평가 방법론: GPT-4o 기반 자동 평가에 크게 의존하며, 이것이 인간의 미세한 대화 품질 인식을 완벽하게 반영하지 못할 수 있음
  • 안전성: Fisher dataset의 실제 전화 대화에서 편향적/민감한 내용을 학습할 수 있으며, RLHF 등 안전성 튜닝이 적용되지 않음. 연구 목적으로만 사용 권장

향후 방향

  • 더 다양한 대화 데이터셋으로의 일반화
  • 노이즈/저자원 환경에서의 robustness 강화 (불완전한 VAD, 저품질 ASR 등)
  • 시각적 단서(표정, 제스처) 등 추가 modality 통합

비판적 분석 (Critical Analysis)

강점

  • 직관적이고 효과적인 접근: "텍스트를 의미 단위(턴)로 묶어서 삽입한다"는 아이디어는 단순하지만 Moshi TS의 token-level 삽입 대비 30%+ 성능 개선으로 매우 효과적이다.
  • 공정한 비교: Moshi와의 직접 비교가 어려움을 인지하고, Moshi TS를 동일 backbone/데이터로 재현하여 공정한 비교를 시도한 점이 좋다.
  • 다면적 평가: Semantic GPT-score, perplexity, Full-Duplex-Bench, Human MOS, Human-AI alignment까지 5가지 평가 축으로 결과를 검증하였다.
  • 재현성: 코드, 데모, Fisher dataset split을 모두 공개하여 재현성을 높였다.

약점

  • 단일 데이터셋 검증: Fisher dataset만 사용하여 일반화 가능성이 불확실. 전화 대화는 실제 AI 어시스턴트 시나리오와 상당히 다르다.
  • Backbone 한정: GLM-4-Voice에서만 검증되어, 다른 SLM backbone (예: Whisper-based, AudioPaLM 등)에서의 효과 미검증.
  • 표준편차/신뢰구간 미보고: 1,000개 샘플에서의 평가이지만, 모든 결과가 단일 점추정치로 보고되어 통계적 유의성 판단이 어렵다. 다수의 학습 seed로의 반복 실험도 보고되지 않았다.
  • Compute cost 미보고: 학습에 소요되는 GPU 시간, 메모리 등 계산 비용이 보고되지 않아 실용성 판단이 어렵다.
  • Conditional 설정에서의 gap: Unconditional (AVG 9.03)과 Conditional-All (AVG 7.18) 간 성능 차이가 크며, 실제 상호작용 시나리오(Conditional)에서의 개선 폭이 Unconditional 대비 상대적으로 작다.
  • IPU 임계값 민감도 미분석: $\tau_{IPU} = 0.5$s, $\tau_{tol} = 0.6$s 등 핵심 하이퍼파라미터에 대한 민감도 분석이 없어, 이 값들의 최적성이 보장되지 않는다.

Key Takeaways

  1. Turn-level이 Token-level보다 낫다: 텍스트-음성 interleaving에서, 의미 단위(턴)로 텍스트를 묶어 삽입하는 것이 토큰별 삽입보다 의미적 품질과 자연스러움 모두에서 우수하다. 이는 텍스트 가이던스의 핵심이 "정확한 시점"이 아니라 "의미의 응집성"에 있음을 시사한다.
  2. 삽입 시점이 삽입 길이보다 중요: Ablation에서 Too Early/Too Late가 Too Long/Too Short보다 성능 저하가 더 크다. 특히 너무 이른 삽입(사용자 맥락 부재)이 가장 치명적이다.
  3. 텍스트 loss 비중 증가의 효과: 텍스트와 음성의 loss 가중치를 2:1 또는 3:1로 높이면 일관되게 성능이 향상된다. 이는 SLM에서 텍스트 모달리티의 학습이 부족할 수 있음을 의미하며, 다른 text-speech 모델에도 적용할 수 있는 범용적 통찰이다.
  4. Low training loss ≠ High performance: Moshi TS의 사례는 pad 토큰 등의 예측 용이성으로 인위적으로 낮아진 loss가 실제 성능과 괴리될 수 있음을 경고한다.
  5. E2e FD-SLM 연구에 유용한 평가 프레임워크: GPT-score + Full-Duplex-Bench + Human MOS의 다면적 평가 프레임워크와 Fisher dataset split 공개는 이 분야의 후속 연구에 좋은 참조점이 된다.
