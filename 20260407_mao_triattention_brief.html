먼저 알아야 할 것: AI의 "기억 장치" 문제

TriAttention을 이해하려면, AI가 긴 답변을 생성할 때 왜 메모리 문제가 생기는지를 먼저 알아야 합니다. 네 가지 핵심 개념을 차례로 설명합니다.

개념 1: AI는 "모든 이전 단어를 참조"하며 다음 단어를 만든다

ChatGPT 같은 AI(Transformer)는 다음 단어를 예측할 때, 이전에 나온 모든 단어를 참조합니다. 이때 각 단어는 두 가지 정보를 남깁니다: Key(열쇠)는 "나는 이런 종류의 정보야"라는 표지판이고, Value(값)는 실제 전달할 내용입니다.

AI는 다음 단어를 예측할 때 이전 모든 단어의 Key-Value 쌍을 참조한다

개념 2: KV Cache -- AI의 "메모 노트"가 넘치는 문제

AI가 100번째 단어를 만들 때, 앞의 99개 단어의 Key와 Value를 다시 계산하면 너무 느립니다. 그래서 한 번 계산한 Key-Value를 GPU 메모리에 저장해 두는데, 이것이 KV Cache입니다. 문제는, AI가 긴 답변을 생성할수록 이 저장 공간이 폭발적으로 커진다는 것입니다.

KV Cache는 생성 토큰 수에 비례해 선형 증가하며, long reasoning에서는 GPU 메모리의 주요 병목이 된다
노트 페이지가 가득 차는 비유

AI가 긴 수학 문제를 푸는 것은 학생이 시험지에 풀이를 쓰는 것과 같습니다. 풀이 과정의 모든 중간 계산을 노트에 적어두어야 나중에 참고할 수 있습니다.


문제가 복잡할수록 노트 페이지가 계속 늘어납니다. 그런데 책상(GPU 메모리)은 크기가 정해져 있습니다. 노트가 책상을 넘치면 더 이상 문제를 풀 수 없습니다.


KV Cache 압축은 이 노트를 더 작은 글씨로 요약하거나, 덜 중요한 페이지를 버리는 것입니다. TriAttention은 "어떤 페이지가 정말 중요한지"를 정확하게 판단하는 방법입니다.

개념 3: KV Cache Pruning -- "덜 중요한 메모를 지우기"

KV Cache 문제를 해결하는 가장 직관적인 방법은 덜 중요한 토큰의 Key-Value를 지우는 것입니다. 이것을 KV Cache Pruning(가지치기)이라 합니다. 하지만 핵심 질문이 있습니다: "어떤 토큰이 미래에도 중요할지 어떻게 알 수 있는가?"

KV Cache Pruning의 성패는 "어떤 토큰이 미래에도 중요할지"를 정확히 예측하는 데 달려 있다

개념 4: 위치 정보(RoPE)가 만드는 "회전하는 관찰 창" 문제

AI는 각 단어의 위치(1번째, 2번째, ...)를 기억하기 위해 RoPE(Rotary Position Embedding)라는 기법을 사용합니다. RoPE는 각 단어의 Key/Query 벡터를 위치에 따라 회전시킵니다. 이 회전 때문에, 기존 중요도 측정 방법에 근본적 한계가 생깁니다.

기존 방법은 RoPE 회전 후의 좁은 관찰 창에서만 판단하지만, TriAttention은 회전 전의 안정적 공간에서 분석한다
회전하는 망원경의 비유

기존 방법은 높은 탑 위에서 계속 회전하는 망원경으로 도시를 관찰하는 것과 같습니다. 망원경이 북쪽을 가리킬 때는 남쪽 건물이 보이지 않으므로 "남쪽에는 중요한 건물이 없다"고 잘못 판단할 수 있습니다.


TriAttention은 망원경을 내려놓고, 도시의 구조적 지도(건물 높이, 위치, 용도)를 분석합니다. 지도는 망원경이 어느 방향을 가리키든 변하지 않으므로, 어느 방향에서 보더라도 중요한 건물을 정확히 파악할 수 있습니다.

핵심 발견: Q/K 벡터의 "나침반" 현상

TriAttention의 출발점은 놀라운 관찰입니다: RoPE 회전을 적용하기 전의 Q/K 벡터들은, 마치 나침반 바늘처럼 특정 방향(고정 중심)을 향해 강하게 모여 있습니다. 이 현상은 입력 내용이나 위치와 무관하게 모델 자체의 고유 특성으로 안정적으로 나타납니다.

Pre-RoPE 공간에서 Q/K 벡터는 고정 중심 주위에 강하게 집중되며, 이는 모델 고유의 안정적 특성이다
나침반 바늘의 비유

나침반 바늘은 어디에서든, 어떤 날씨에서든 항상 북쪽을 가리킵니다. 이것은 나침반의 고유 특성이지, 주변 환경에 의해 달라지지 않습니다.


TriAttention이 발견한 것은 이와 같습니다: AI 모델의 Q/K 벡터들은 입력 텍스트가 수학이든, 코딩이든, 대화든 항상 같은 방향(고정 중심)을 가리킵니다. 이 "자기 북쪽"을 알면, 모든 상황에서 미래의 attention 패턴을 예측할 수 있습니다.

이 발견이 왜 중요한지 한 단계 더 깊이 살펴보겠습니다. Q/K 벡터가 고정 중심에 모여 있다는 것은, attention의 작동 방식을 "거리의 함수"로 예측할 수 있다는 뜻입니다.

삼각함수로 미래를 예측하다

Q/K 벡터가 고정 중심에 모여 있으면, "현재 Query 위치에서 어떤 Key가 높은 attention을 받을지"가 두 위치 사이의 거리로 결정됩니다. TriAttention은 이 거리-attention 관계가 파도(삼각함수)의 합으로 정확히 표현된다는 것을 증명했습니다.

Attention이 거리의 삼각함수 합으로 예측됨: 다양한 주파수의 파도를 합치면 실제 attention 곡선이 재현된다
바다 파도 예측의 비유

기상학자가 바다 파도의 높이를 예측할 때, 큰 파도(조류)와 작은 파도(바람)를 합쳐서 계산합니다. 각 파도는 단순한 사인/코사인 곡선(삼각함수)이고, 이들을 합치면 복잡한 실제 파도 패턴이 재현됩니다.


TriAttention도 마찬가지입니다. AI의 attention 패턴은 복잡해 보이지만, 여러 주파수의 삼각함수를 합치면(trigonometric series) 정확히 예측됩니다. 그리고 이 삼각함수의 "파도 크기와 위상"은 앞서 발견한 Q/K 고정 중심에서 계산됩니다.

여러 주파수의 삼각함수(파도)를 합치면 실제 attention 패턴을 정확히 재현할 수 있다

Step 1: Q/K 고정 중심 측정 (사전 준비)

먼저 소량의 calibration 데이터(아무 텍스트나 가능!)를 모델에 통과시켜, 각 attention head에서 Q/K 벡터들의 평균 방향(고정 중심)을 계산합니다. 놀라운 점은 이 중심이 입력 내용에 무관하게 안정적이라는 것입니다. Google 홈페이지 HTML로 측정해도, 수학 문제로 측정해도 거의 같은 결과가 나옵니다.

Calibration 데이터의 종류에 무관하게 Q/K 중심이 안정적이며, 어떤 데이터로 측정해도 동일한 성능을 보인다

Step 2: 삼각함수로 Key 중요도 점수 계산

고정 중심을 알게 되면, 각 Key에 대해 "미래의 Query가 이 Key를 얼마나 중요하게 볼 것인지" 점수를 매길 수 있습니다. 이 점수는 두 가지 신호의 합입니다:


1) 삼각함수 점수 (Trigonometric Score): 고정 중심과 각 Key의 각도/거리 관계에서 삼각함수를 계산. "미래 어느 위치에서든 이 Key가 높은 attention을 받을 것인지" 예측.

2) 크기 점수 (Norm Score): Key 벡터의 크기가 클수록 attention에서 영향력이 큼. 삼각함수 예측이 불확실한 경우를 보완.


두 점수를 합산하되, Q/K 집중도가 높은 head에서는 삼각함수 점수에 더 큰 가중치를, 낮은 head에서는 크기 점수에 더 큰 가중치를 자동으로 부여합니다.

삼각함수 점수와 크기 점수를 Q/K 집중도에 따라 지능적으로 결합하여 최종 중요도를 계산한다

Step 3: 높은 점수의 Key만 보존 (스마트 가지치기)

최종 점수가 계산되면, 정해진 예산(budget) 내에서 가장 높은 점수의 Key만 남기고 나머지를 제거합니다. 예를 들어 budget이 2,048이면, 30,000개의 Key 중 상위 2,048개만 GPU 메모리에 보존합니다.


추가로 두 가지 실용적 최적화가 적용됩니다:

  • 128 토큰마다 한 번만 가지치기: 매 토큰마다 하면 계산 비용이 크므로, 일정 간격으로 수행
  • 여러 미래 거리에서 점수 평균: 당장의 미래뿐 아니라 먼 미래까지 고려하여 점수를 매김
최종 점수가 높은 Key만 보존하며, 기하급수적 간격으로 여러 미래 시점을 동시에 고려한다

기존 방법과 무엇이 다른가?

KV Cache 압축에는 여러 접근이 있습니다. TriAttention이 어디에 위치하는지 큰 그림을 먼저 보겠습니다.

TriAttention은 기존 가지치기 방법과 달리, Pre-RoPE 공간에서 삼각함수로 미래 중요도를 예측한다

기존 방법 (SnapKV, R-KV)은 "방금 5분간 어떤 책이 대출되었는지"만 보고, 1년 뒤에 필요할 책을 예측하려는 것과 같습니다. 오후에 소설이 인기였다고 다음 날 아침에도 소설이 인기일 거라는 보장은 없습니다.


TriAttention은 도서관의 구조적 배치(장르별 위치, 추천 동선)를 분석합니다. 이 구조는 시간대와 무관하게 일정하므로, 어느 시점에든 어떤 책이 필요해질지 정확히 예측할 수 있습니다.

연구팀은 "재귀적 시뮬레이션(DFS)" 벤치마크를 설계하여, KV cache를 줄여도 중간 계산 결과를 잊지 않는지 테스트했습니다.


결과: 재귀 깊이 16단계까지 TriAttention은 Full Attention과 동일한 성능을 유지합니다. 오히려 깊이 8~12에서는 약간 더 좋은 성능을 보였습니다. 반면 R-KV는 깊이 16에서 정확도가 61%에서 31%로 급락하며, 중간 상태 보존에 심각한 문제를 드러냈습니다.

활용 분야

TriAttention은 긴 추론이 필요한 모든 AI 응용에서 메모리 병목을 해소할 수 있다

실용적 시나리오: 소비자 GPU에서 대형 모델 구동

연구팀은 Qwen3-32B 모델을 RTX 4090 (24GB) 한 장에서 TriAttention으로 구동하는 것을 시연했습니다. Full Attention으로는 불가능한 long-context 추론이 가능해졌습니다. 이는 고가의 A100/H100 없이도 대형 모델을 사용할 수 있다는 것을 의미합니다.

"AI가 깊이 생각할 때 메모리가 넘치는 문제를,
벡터의 고유한 '나침반 방향'을 이용한 삼각함수 예측으로 해결하다"

한계점: 균형 잡힌 시각

모델 규모의 한계

실험은 8B~20B 규모의 모델에서만 수행되었습니다. 70B 이상의 초대형 모델이나 MoE(Mixture of Experts) 아키텍처에서의 효과는 아직 검증되지 않았습니다. 모델이 커질수록 Q/K concentration 특성이 변할 가능성이 있습니다.

Offline Calibration 필요

Q/K 중심 계산을 위해 사전에 calibration 단계가 필요합니다. 모델이 업데이트(fine-tuning)되면 재계산해야 합니다. 다만, calibration 데이터의 종류나 양에 민감하지 않다는 점(Google 홈페이지 HTML로도 충분)은 긍정적입니다.

일부 Attention Head의 불확실성

전체 attention head의 약 10%에서는 Q/K concentration이 0.95 미만으로, 삼각함수 예측의 정확도가 떨어집니다. 이 head들에서는 Norm 기반 점수에 의존하지만, 만약 이 head가 핵심 정보를 담당한다면 성능에 영향을 줄 수 있습니다.

Prefill 단계 미적용

TriAttention은 decoding(생성) 단계의 KV cache만 압축합니다. 긴 프롬프트를 처음 처리하는 prefill 단계의 메모리 병목은 해결하지 못합니다. 완전한 솔루션을 위해서는 prefill 최적화 기법과의 결합이 필요합니다.

통계적 신뢰도

AIME 벤치마크 결과에 표준편차가 보고되지 않았습니다. 문제당 8개 샘플로 평가했지만, 결과의 분산이 얼마나 큰지 불명확합니다. 특히 AIME는 30문제로 구성된 작은 벤치마크이므로, 통계적 유의성에 주의가 필요합니다.

균형 잡힌 시각

TriAttention은 Pre-RoPE Q/K concentration이라는 탄탄한 이론적 기반 위에 세워진 방법이며, 4개 모델 + 다양한 벤치마크에서 일관되게 최고 성능을 보입니다. 다만:

  1. 초대형 모델에서의 효과는 아직 미검증입니다
  2. 실용적 도입 시 calibration 파이프라인 구축과 prefill 최적화와의 결합이 필요합니다
  3. 코드가 공개(GitHub)되어 있어 독립적 검증이 가능합니다
