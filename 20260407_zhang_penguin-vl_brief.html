먼저 알아야 할 것: AI는 어떻게 사진을 이해하나?

Penguin-VL을 이해하려면, AI가 이미지와 텍스트를 동시에 다루는 방법을 먼저 알아야 합니다. 세 가지 핵심 개념을 그림과 함께 설명합니다.

개념 1: Vision-Language Model (VLM) -- AI가 사진을 "읽는" 방법

ChatGPT 같은 AI는 글(텍스트)만 이해할 수 있습니다. 사진이나 영상까지 이해하려면 시각 정보를 언어 정보로 변환하는 추가 장치가 필요합니다. 이렇게 시각+언어를 함께 처리하는 AI를 Vision-Language Model (VLM)이라고 부릅니다.

VLM의 3요소: 이미지를 분석하는 눈(Vision Encoder), 시각 정보를 언어로 변환하는 번역기(Projector), 그리고 사고하는 뇌(LLM)
해외 여행 가이드의 비유

VLM은 해외 여행 중 현지 풍경을 보고 설명해주는 가이드와 같습니다. 가이드의 눈(Vision Encoder)이 풍경을 보고, 머릿속 번역기(Projector)가 시각 정보를 말로 옮기고, 입(LLM)이 관광객에게 설명합니다. 눈이 나쁘면 중요한 디테일을 놓치고, 번역기가 부실하면 뉘앙스가 사라집니다.

개념 2: AI 모델의 크기가 왜 중요한가

AI 모델은 "파라미터"라고 불리는 숫자들의 집합입니다. 파라미터가 많을수록 더 똑똑하지만, 더 많은 메모리와 전력이 필요합니다. 현재 최고 성능 VLM들은 72B(720억 개) 이상의 파라미터를 가져서 고가의 서버급 GPU가 필수입니다.

모델이 작을수록 더 다양한 기기에서 실행할 수 있다. Penguin-VL은 2B와 8B에서 최고 성능을 목표로 한다

문제는, 모델을 작게 만들면 보통 성능도 크게 떨어진다는 것입니다. Penguin-VL의 목표는 "작지만 큰 모델 못지않게 똑똑한" AI를 만드는 것입니다.

개념 3: 기존 "눈"의 문제점 -- Contrastive Learning

현재 대부분의 VLM은 CLIP이나 SigLIP이라는 방식으로 만들어진 Vision Encoder를 사용합니다. 이들은 Contrastive Learning (대조 학습)이라는 방법으로 훈련되었습니다.

대조 학습의 원리는 간단합니다: 이미지와 설명 문장을 짝지어 보여주고, "이 이미지와 이 문장이 짝인지 아닌지" 구별하는 훈련을 합니다. 그런데 이 방법에는 근본적인 한계가 있습니다.

Contrastive Learning은 "분류(이것이 뭔지)"에는 강하지만, VLM이 필요로 하는 "세밀한 설명 능력"에는 부족하다
시험 공부의 비유: "분류 시험 vs 서술형 시험"

기존 방식(Contrastive Learning)은 "이 사진이 펭귄인지 고양이인지" 맞히는 객관식 시험만 열심히 공부한 학생과 같습니다. 분류는 잘 하지만, "이 사진에서 무슨 일이 일어나고 있는지 500자로 설명하라"는 서술형 시험에는 약합니다.


Penguin-VL은 처음부터 서술형 시험을 목표로 훈련한 학생과 같습니다. 사진의 모든 디테일을 세밀하게 관찰하고, 풍부한 언어로 설명하는 능력을 키웁니다.

개념 4: 데이터 효율성 -- 기존 방식의 또 다른 문제

기존 Vision Encoder(CLIP, SigLIP 등)는 400억 개 이상의 이미지-텍스트 쌍으로 훈련됩니다. 이 데이터를 수집하고 처리하는 데 천문학적 비용과 시간이 듭니다. Penguin-VL은 이것의 1/167 수준인 약 2.4억 개 데이터만으로 더 나은 성능을 달성합니다.

기존 대비 167배 적은 데이터로 더 좋은 성능. "더 똑똑하게 배우는" 훈련 방식 덕분이다

핵심 1: 언어 모델의 뇌를 "눈"으로 변환하다 (Penguin-Encoder)

Penguin-VL의 가장 대담한 아이디어: 텍스트만 이해하는 언어 모델(Qwen3-0.6B)을 가져와서 그대로 Vision Encoder로 변환합니다. 기존처럼 대조 학습으로 새로운 눈을 만드는 것이 아니라, 이미 "언어를 아는 뇌"를 "시각도 아는 눈"으로 재훈련하는 것입니다.

Penguin-VL은 언어 모델을 변환하여 Vision Encoder를 만든다. 눈과 뇌가 "같은 DNA"를 공유하므로 소통이 자연스럽다
모국어 화자 vs 통역사의 비유

기존 방식은 영어(시각 정보)와 한국어(언어 정보)를 각각 따로 배운 뒤, 통역사(Projector)를 통해 소통하는 것과 같습니다. 통역 과정에서 뉘앙스가 항상 조금씩 빠집니다.


Penguin-VL은 한국어 원어민(LLM)에게 영어(시각)까지 가르친 것과 같습니다. 같은 사고 체계를 가진 사람이 두 언어를 모두 구사하므로, 번역 손실 없이 자연스러운 소통이 가능합니다.

변환 과정에서 두 가지를 수정합니다:

두 가지 간단한 수정만으로 텍스트 모델이 이미지 모델로 변신한다

핵심 2: "크기 + 방향 + 관계"를 동시에 배우는 훈련법

LLM을 Vision Encoder로 변환한 뒤, 이미지를 제대로 이해하도록 훈련해야 합니다. Penguin-VL은 세 가지 측면을 동시에 학습시키는 Mixed Supervision이라는 방법을 사용합니다.

세 가지 학습 목표를 동시에 달성하면, 각 패치의 개별 특성뿐 아니라 패치 간의 "관계 구조"까지 보존된다
미술 수업의 비유: "모사(따라 그리기)"

Penguin-Encoder를 훈련하는 것은 미술 학생이 선생님의 그림을 따라 그리는 것과 비슷합니다.


크기 학습: "이 부분은 진하게, 저 부분은 연하게" -- 각 영역의 강도를 맞춤

방향 학습: "붓질의 방향을 맞춰라" -- 각 특성의 패턴 방향을 맞춤

관계 학습: "하늘과 바다의 색조가 서로 호응해야 한다" -- 영역 사이의 조화를 맞춤


특히 관계 학습이 가장 중요한데, 각 부분을 개별적으로 잘 그리는 것보다 전체가 조화를 이루는 것이 더 중요하기 때문입니다.

핵심 3: 영상을 똑똑하게 압축하는 기술 (TRA)

영상(비디오)은 이미지보다 처리가 훨씬 어렵습니다. 1초에 30장의 이미지가 연속으로 들어오기 때문입니다. 모든 프레임을 전부 분석하면 메모리와 시간이 폭발합니다. Penguin-VL은 TRA (Temporal Redundancy-Aware)라는 기술로 이 문제를 해결합니다.

TRA는 영상에서 중요한 장면과 덜 중요한 장면을 자동으로 구분하여 자원을 차등 배분한다
영화 하이라이트 릴의 비유

일반적인 영상 처리는 2시간짜리 영화의 모든 프레임을 똑같이 분석하는 것과 같습니다. 지루한 대화 장면과 박진감 넘치는 액션 장면을 동일하게 처리하니 비효율적입니다.


TRA는 편집자가 만드는 "하이라이트 릴"과 같습니다. 장면 전환, 중요한 동작, 새로운 사건이 등장하는 순간(Key Frame)은 고화질로 상세하게 보여주고, 비슷한 장면이 계속되는 구간(Intermediate)은 요약만 보여줍니다. 전체 영화의 핵심은 놓치지 않으면서도 상영 시간(메모리)은 크게 줄어듭니다.

성과 1: 2B 모델 -- 스마트폰급 AI의 놀라운 실력

Penguin-VL-2B는 같은 크기의 경쟁 모델들을 대부분의 벤치마크에서 앞섰습니다. 문서 이해, 차트 분석, 수학 문제 풀이, 영상 이해까지 모두 포함합니다.

Penguin-VL-2B는 문서, 차트, 지식, 수학, 영상 등 7개 주요 벤치마크에서 2B급 모델 중 1위

성과 3: Penguin-Encoder의 놀라운 데이터 효율성

Penguin-Encoder는 Contrastive Learning 없이, 기존 대비 167배 적은 데이터로 훈련했음에도 더 높은 평균 점수(65.0)를 달성했습니다. 이는 기존 최강 SigLIP2(61.6)와 Qwen3VL-32B의 ViT(62.0)를 모두 능가하는 수치입니다.

Penguin-Encoder는 데이터를 167배 적게 사용하면서도 모든 기존 Encoder보다 높은 성능을 달성

성과 4: 각 요소가 얼마나 기여하는가 (Ablation Study)

연구팀은 각 구성 요소를 하나씩 빼보며, 무엇이 얼마나 중요한지 체계적으로 검증했습니다.

모든 구성 요소를 조합했을 때 최고 성능(34.6) 달성. LLM 초기화와 관계 학습이 각각 +1.3점으로 가장 큰 기여

비밀 무기: 고품질 데이터 큐레이션

Penguin-VL이 적은 데이터로 높은 성능을 달성하는 비결 중 하나는 데이터의 양보다 질에 집중한 것입니다. 연구팀은 수억 장의 이미지에서 엄선된 데이터만 골라 쓰는 정교한 큐레이션 파이프라인을 구축했습니다.

단순히 많은 데이터를 모으는 것이 아니라, 다양하고 고품질인 데이터만 엄선하여 사용한다

기존 방법 대비 장점

Penguin-VL은 Encoder와 LLM이 같은 "언어 DNA"를 공유하여, 기존 방식의 번역 손실 문제를 근본적으로 해결한다

활용 분야

Penguin-VL의 작은 크기(2B/8B)는 데이터센터뿐 아니라 스마트폰, 로봇, IoT 등 다양한 환경에서의 배포를 가능하게 한다
"AI의 눈을 처음부터 새로 만드는 대신, 이미 언어를 아는 뇌를 눈으로 변환했더니
167배 적은 데이터로 더 정확하게, 더 작게, 더 빠르게 세상을 본다"

알아두어야 할 한계점

특정 분야에서의 약점

Penguin-VL이 모든 분야에서 1위인 것은 아닙니다. OCR 특화 벤치마크(OCRBench: 810 vs Qwen3-VL 858)와 수학 추론(MathVerse: 35.9 vs Qwen3-VL 52.1)에서는 열위를 보입니다. 또한 8B 모델에서 전문가 수준의 다중 이미지 추론(MMMU-Pro: 40.2 vs Qwen3-VL 55.9)에서 큰 차이를 보였습니다.


이는 특정 도메인(OCR, 고난이도 수학)에서는 여전히 대규모 전용 데이터가 필요할 수 있음을 시사합니다.

완전히 독립적이지 않은 훈련

Penguin-Encoder는 "Contrastive Learning 없이" 훈련되었다고 하지만, 실제로는 Stage 1에서 VL3-SigLIP-NaViT라는 기존 Encoder를 선생님(Teacher)으로 사용합니다. 즉, 기존 방식에 완전히 의존하지 않되, 기존 Encoder의 지식을 참조하여 배우는 구조입니다.

Encoder 크기 확장 미검증

현재 Penguin-Encoder는 약 4억 개 파라미터(Qwen3-0.6B 기반)만 사용합니다. 더 큰 LLM(1.5B+)을 Encoder로 변환했을 때의 효과는 아직 실험되지 않았습니다. "미래 과제"로만 언급되어 있어, 이 접근법의 확장성은 아직 열린 질문입니다.

실제 추론 속도 미보고

논문은 "효율성(Efficiency)"을 강조하지만, 실제 추론 속도(latency)나 처리량(throughput) 비교 데이터가 없습니다. TRA 압축이 실제로 얼마나 빠른지, 경쟁 모델 대비 속도 차이가 어느 정도인지는 확인되지 않았습니다.

데이터 큐레이션 vs 아키텍처의 기여 분리 어려움

Penguin-VL의 높은 성능이 Penguin-Encoder 아키텍처 덕분인지, 아니면 고품질 데이터 큐레이션(Penguin-Recap-I/V) 덕분인지를 명확히 분리하기 어렵습니다. 동일한 데이터로 기존 SigLIP2를 훈련한 대조 실험이 제한적입니다.

균형 잡힌 시각

Penguin-VL은 VLM의 Vision Encoder를 만드는 패러다임 자체를 바꾸려는 의미 있는 시도입니다. "Contrastive Learning이 유일한 답이 아니다"라는 것을 실증적으로 보여준 점에서 학술적 가치가 높으며, 2B/8B 모델과 코드가 모두 공개되어 즉시 활용 가능하다는 것도 큰 장점입니다.


다만 특정 분야(OCR, 고난이도 수학/추론)에서의 약점과, Encoder 확장성에 대한 검증 부재는 후속 연구에서 다뤄져야 할 부분입니다.
