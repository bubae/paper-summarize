1. 먼저 알아둘 배경 지식

1.1 LLM의 Fine-Tuning이란?

대형 언어 모델(LLM)은 수십억 개의 파라미터를 가진 거대한 신경망이다. 일반 용도로 잘 학습된 모델(예: ChatGPT, Llama-2-chat)을 특정 task(코딩, 수학, 함수 호출 등)에 맞게 추가 학습시키는 과정을 fine-tuning이라 한다.

그런데 fine-tuning이 잘못 되면 모델이 가진 다른 능력을 망가뜨린다. 이를 catastrophic forgetting (치명적 망각)이라 부른다.

Vanilla fine-tuning의 zero-sum 문제

1.2 왜 망가지는가? — Distribution Gap

이 논문의 핵심 통찰: fine-tune 데이터의 답변 스타일이 원래 모델의 출력 스타일과 너무 다르면, 모델은 그 격차를 메우기 위해 파라미터를 크게 움직인다. 그 과정에서 원래 가지고 있던 능력이 함께 흔들린다.

예: GSM8K 수학 문제집의 답은 짧고 건조한 형식("The answer is 15.")인데, Llama-2-chat은 친절하고 단계별로 설명하는 스타일이다. 이 두 스타일을 강제로 맞추려다 친절함 자체가 사라지는 것.

📚 비유: 외국어 시험과 모국어

한국어가 모국어인 학생이 영어로 적힌 모범 답안을 그대로 외워 시험을 본다고 하자. 영어 점수는 오를 수 있지만, 본인의 한국어 사고 방식과 너무 달라서 영어를 강제로 익히는 동안 한국어 표현력도 어색해진다.
SDFT의 발상: 영어 모범 답안을 한국어 화자의 어법으로 다시 적은 뒤 그걸로 공부하자. 의미는 같지만 표현이 자기 어법이라, 한국어가 망가지지 않는다.

1.3 LoRA란?

본 논문 실험에서 자주 쓰는 LoRA (Low-Rank Adaptation)는 모델의 모든 파라미터가 아니라 작은 부분만 학습시키는 효율적인 fine-tune 기법. 메모리·시간이 적게 들지만 forgetting 문제 자체는 해결 못함. SDFT는 LoRA 위에서도 작동.

2. SDFT의 핵심 아이디어

2.1 Self-Rewriting (자기-재작성)

SDFT는 새 task의 답변을 모델 자신이 자기 스타일로 다시 적게 한다. 그렇게 만든 distilled dataset으로 fine-tune.

SDFT는 데이터 전처리 단계만 추가한 단순한 변형

2.2 Distillation Template

모델에게 다음과 같은 prompt를 준다:

"Below are an instruction that describes a task along with a reference answer.
Using the reference answer as a guide, write your own response.

### Instruction: {질문}
### Reference Answer: {원본 답}
### Response:"

모델은 이 prompt에 따라 reference answer를 그대로 베끼지 않고 자기 스타일로 풀어 쓴다. 의미는 같지만 표면 형태가 자연스러워진다.

2.3 수학 task의 정확성 안전장치

수학 같은 정답이 있는 task에서는 재작성된 답에서 최종 숫자만 추출해 원본과 비교. 일치하면 distilled 답 채택, 다르면 원본 유지. 이 단순한 휴리스틱이 재작성이 정답을 망가뜨릴 위험을 차단한다.

2.4 왜 작동하는가?

SDFT는 fine-tune의 "이동 거리"를 줄여 기존 능력을 보존

3.4 Distribution Shift = Forgetting의 직접 증거

학습 데이터 양을 늘릴수록 (BLEU·ROUGE·embedding 유사도)는 감소하고 파라미터 변화 ‖Δθ‖는 증가하며, 동시에 downstream 성능이 떨어진다. 즉 distribution shift와 forgetting이 선형적으로 동행한다는 정량 증거.

역으로 vanilla:SDFT mix ratio를 0→1로 올리면 모든 지표가 단조적으로 개선된다. 이 실험은 SDFT가 단순한 우연이 아니라 distribution gap을 직접 줄여 forgetting을 막는다는 인과 검증.

5. 활용 분야

"데이터를 모델 자기 분포로 다시 쓰면, fine-tune이 새 능력은 더하고 기존 능력은 유지하는 단순한 변신을 한다"

6. 한계점 (균형 잡힌 시각)

① Self-rewriting의 정확성 보장 부재: 수학 task에는 정답 추출 검증이 있지만, 코드·툴 호출·자유 형식 task에선 재작성이 의미를 변형해도 발견 못함.

② weak base model에선 효과 감소: rewrite 품질이 seed LM 능력에 의존 — 아주 작은 모델에선 distilled 답이 noisy할 수 있다.

③ Compute overhead: 학습 데이터 전체를 한 번 추론 돌려 rewrite해야 함. 대규모 데이터(20k+)에선 cost가 무시 못함.

④ 이론적 정당화는 약함: distribution gap → forgetting은 실증적 상관이지 인과 증명이 아니다.

⑤ 순차적 multi-task continual learning은 미검증: single fine-tune의 forgetting만 다룸. 여러 task를 순서대로 누적 학습할 때의 효과는 별도 연구(Shenfeld et al. 2026) 영역.
