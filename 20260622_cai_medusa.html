Medusa: Simple LLM Inference Acceleration Framework with Multiple Decoding Heads

TL;DR

  • 문제: Speculative decoding은 별도의 작은 draft model 필요 → ##학습/배포/distribution shift 모두 부담##. 게다가 distributed serving 환경에서 두 모델 동기화는 추가 복잡도. "draft model 없이 그냥 LLM 자체를 빠르게 할 수 없을까?"
  • 해결: Backbone LLM 자체에 ##K개의 추가 decoding head##(Medusa heads)를 붙여서 한 번의 forward pass로 token $t+1, t+2, \ldots, t+K+1$를 ##병렬 예측##. 거기에 ##tree-based attention##으로 head별 top-s 예측을 Cartesian product로 만든 다중 후보를 한 번에 검증.
  • 왜 작동하는가: 단일 forward pass에서 last hidden state로 다중 미래 토큰을 예측하는 게 가능하다는 사실. 추가된 head는 단순 FFN+residual로 매우 가볍고 backbone weight를 활용 → ##low cost로 multi-token prediction 능력##.
  • 비유: 표준 LLM이 "다음 글자만 보는 비서"라면, Medusa는 "다음 5글자를 동시에 추측하고, 그 모든 조합을 한꺼번에 검증하는 비서". 비유 그대로 '메두사' 머리 여러 개.

핵심 기여 (Key Contributions)

  1. Medusa heads: Backbone LLM의 last hidden state에 K개의 lightweight decoding head 부착. 각 head $k$가 position $t+k+1$의 token 분포를 예측. ##draft model 불필요##.
  2. Tree-based attention: K개 head의 top-$s_k$ 예측을 Cartesian product → $\prod s_k$개 candidate continuation을 ##단일 forward pass##에서 attention mask로 분리해 동시 검증.
  3. Medusa-1 (frozen backbone): Backbone 고정, Medusa head만 학습. ##단일 GPU에서 7B를 5시간 학습##(quantized backbone과 결합 가능).
  4. Medusa-2 (joint training): Backbone + heads joint training. 더 높은 acceptance rate. ##Combined loss + differential learning rate + heads warmup## 세 트릭으로 backbone quality 보존.
  5. Typical acceptance: Rejection sampling보다 효율적인 acceptance scheme. Temperature를 deviation budget으로 사용해 typical 후보만 채택. ##higher acceptance + similar quality##.
  6. Self-distillation: Public dataset이 없는 경우(RLHF 모델 등) 모델 자신의 출력으로 학습 데이터 생성. ##데이터 부재 해결##.
  7. 실증: Vicuna 7B/13B/33B + Zephyr 7B에서 ##Medusa-1 2.18×, Medusa-2 2.83× wall-clock speedup## (MT-Bench, batch 1, A100). Coding task에서 3.62× 최대.

1. 배경 (Background)

1.1 LLM inference의 memory-bound 문제

LLM inference는 ##memory-bandwidth-bound##: 매 forward pass마다 모든 모델 parameter를 High-Bandwidth Memory(HBM)에서 accelerator cache로 옮겨야 함. 한 번 옮긴 parameter로 단일 token만 생성 → ##arithmetic intensity##(FLOP/byte 비율)가 매우 낮음.

해결 방향:

  • (a) Arithmetic intensity 증가: 한 번 옮긴 weight로 더 많은 연산 수행
  • (b) Decoding step 수 감소: speculative decoding 방향

1.2 Speculative decoding의 한계

Speculative decoding(Leviathan et al. 2022, Chen et al. 2023)은 별도의 작은 draft model을 사용. 하지만:

  • 적절한 draft model 확보가 어려움 (특히 새로운 모델 family).
  • Draft model이 별도 pre-training 필요 → ##275 A100-hour##(Miao et al. 2023 사례).
  • Distribution shift: draft와 original의 출력 분포 불일치.
  • Distributed serving: 두 모델 관리 복잡도.

1.3 Stern et al. 2018의 blockwise parallel decoding 부활

저자들은 ##Stern et al. (2018)의 blockwise parallel decoding##에서 영감. 원래 NMT/Image SR 분야의 방법인데, LLM serving에 적용 가능한지 재조명. 핵심 아이디어: 모델 위에 multiple prediction head를 부착.

2. 방법 (Methodology)

2.1 전체 구조 (Figure 1 요약)

[Figure 1 설명]: 왼쪽 Original Model — Embedding → Transformer Layers → LM Head ("It is difficult"). 오른쪽 위에 Medusa Heads 3개가 transformer 마지막 hidden state $h_t$에 연결. 각각이 top-$k$ 예측 출력 — Head 1: "It, I, As", Head 2: "is, ', the", Head 3: "difficult, is, '". 이 top-$k$ 예측들이 Cartesian product로 결합되어 candidate continuation 만들어짐. 마지막에 ##tree-based attention##으로 모든 후보를 단일 forward에서 동시 검증. 가장 긴 accepted prefix 채택.

2.2 Medusa Heads (Section 2.1.1)

$h_t \in \mathbb{R}^d$를 backbone의 마지막 hidden state (position $t$)라 할 때, $k$-th head는 position $t+k+1$ token 분포를 예측:

$$p_t^{(k)} = \text{softmax}\left(W_2^{(k)} \cdot \left(\text{SiLU}(W_1^{(k)} \cdot h_t) + h_t\right)\right) \quad (1)$$

여기서 $W_1^{(k)} \in \mathbb{R}^{d \times d}$, $W_2^{(k)} \in \mathbb{R}^{d \times V}$. ##단일 FFN layer + residual connection + SiLU activation##. 즉 LLaMA의 standard FFN과 구조 동일.

초기화 trick: $W_2^{(k)}$는 original LM head와 동일하게 초기화 (모든 head가 처음엔 same predictor), $W_1^{(k)}$는 0으로 초기화. ##residual을 통해 처음엔 base 모델과 동일한 분포##에서 출발 → 안정적 학습.

$p_t^{(0)}$는 backbone의 원래 LM head 예측. Notation 일관성을 위해 0-th head로 정의.

2.3 Tree Attention (Section 2.1.2)

[Figure 2 설명]: $s_1 = 2, s_2 = 3$인 경우 — Head 1의 top-2 ("It", "I")와 Head 2의 top-3 ("is", "'", "the"). Cartesian product = $2 \times 3 = 6$ candidate. Tree 구조: Root → 2 branches (It, I) → 각각 3 branches → 6 leaf. 각 candidate는 root에서 leaf까지 path. ##Attention mask##는 각 token이 자신의 ancestor만 attend하게 설계. Positional encoding도 ##tree depth에 맞춰 조정##. 결과적으로 batch size 늘리지 않고 6개 후보를 1회 forward에서 검증.

Cumulative candidate 수: $\sum_{k=1}^K \prod_{i=1}^k s_i$. ##batch size 확장 없이 추가 token 처리##.

2.4 Sparse Tree Construction (Section 2.3.3)

Cartesian product는 단순하나 ##모든 head의 top 예측이 같은 acceptance rate를 가지지 않는다##. Calibration dataset으로 각 head의 top-$i$ 예측 accuracy $a_i^{(k)}$를 측정. Candidate sequence $[i_1, \ldots, i_k]$의 expected acceptance ≈ $\prod_{j=1}^k a_j^{(i_j)}$ (independence 가정).

Tree construction: ##greedy node 추가##. 빈 tree에서 시작, 매 step마다 expected acceptance contribution이 가장 큰 node를 tree에 추가. Total node 수가 desired 값에 도달할 때까지 반복. ##acceptance length 기댓값을 최대화##하는 tree.

2.5 Training Strategies

Medusa-1: Frozen backbone (Section 2.2.1)

Backbone freeze, Medusa head만 학습.

$$\mathcal{L}_{\text{Medusa-1}} = \sum_{k=1}^K -\lambda_k \log p_t^{(k)}(y_{t+k+1}) \quad (1)$$

$\lambda_k = 0.8^k$ (k-th power of 0.8): ##k가 클수록 예측이 어려우므로 가중치 감소##. backbone freeze이므로 quantization과 호환 → ##단일 consumer GPU에서 학습 가능##.

Medusa-2: Joint training (Section 2.2.2)

Backbone + heads joint training. 3가지 트릭:

  • Combined loss: $\mathcal{L}_{\text{Medusa-2}} = \mathcal{L}_{\text{LM}} + \lambda_0 \mathcal{L}_{\text{Medusa-1}}$, 여기서 $\mathcal{L}_{\text{LM}} = -\log p_t^{(0)}(y_{t+1})$. ##Backbone의 next-token 예측 능력 보존##.
  • Differential learning rates: Backbone은 작은 lr, Medusa head는 큰 lr. ##backbone capability 유지하면서 head 빠른 수렴##.
  • Heads warmup: 2-stage 학습. Stage 1: Medusa-1로 head만 학습. Stage 2: backbone + heads joint training. ##초기 large gradient로 backbone 파라미터 distort 방지##.

2.6 Typical Acceptance (Section 2.3.1)

Rejection sampling의 비효율: temperature를 올리면 acceptance rate가 떨어지고, draft가 original과 분포 다르면 다 reject. ##Truncation sampling (Hewitt et al. 2022)##에서 영감.

Candidate $(x_{n+1}, \ldots, x_{n+K+1})$에 대해 다음 조건이면 accept:

$$p_{\text{original}}(x_{n+k}|x_1, \ldots, x_{n+k-1}) > \min\left(\epsilon, \delta \exp(-H(p_{\text{original}}(\cdot|x_1, \ldots, x_{n+k-1})))\right)$$

여기서 $H$는 entropy. $\epsilon$는 hard threshold, $\delta$는 entropy-dependent threshold. 직관적으로 말하면, ##원본 모델이 그 후보를 합리적이라고 여길 만큼 충분히 높은 확률을 부여하면 accept##. Distribution이 평평할 때(높은 entropy)는 threshold도 작아짐.

Empirical: $\epsilon = \sqrt{\alpha}$로 설정. 첫 token은 unconditional greedy decoding(보장). 가장 긴 accepted prefix 선택.

2.7 Self-Distillation (Section 2.3.2)

Public training data가 없거나 RLHF로 분포가 변한 경우, ##모델 자신의 출력으로 dataset 생성##:

  1. Seed prompt set 선정 (ShareGPT, UltraChat).
  2. 모델이 자체적으로 response 생성. Zephyr 같은 self-talk 가능 모델은 prompt 한 개로도 multi-turn 생성.
  3. Vicuna-33B 같은 single-turn 모델은 prompt를 순차 feed.

Medusa-2 backbone training 시 ##추가 KL distillation loss##:

$$\mathcal{L}_{\text{LM-distill}} = \text{KL}(p_{\text{original}, t}^{(0)} \| p_t^{(0)})$$

그러나 두 모델 동시 메모리 부담. ##LoRA 활용##: backbone에 LoRA adapter, LoRA OFF면 original 분포. ##추가 메모리 없이 self-distillation##.

2.8 알고리즘 흐름

[Inference] 1. Backbone forward → h_t (last hidden state) + p_t^(0) 2. 각 Medusa head k가 h_t로부터 top-s_k 예측 p_t^(k) 3. Cartesian product (or sparse tree)로 candidate 생성 4. Tree attention mask + 조정된 positional encoding으로 모든 candidate를 backbone에 1회 forward 검증 5. Typical acceptance로 가장 긴 valid prefix 선택 6. 그 prefix를 다음 decoding step의 input으로 사용

3. 실험 (Experiments)

3.1 셋업

  • Models: Vicuna 7B/13B (public SFT), Vicuna 33B (private SFT, self-distillation 필요), Zephyr 7B (RLHF, self-distillation 필요)
  • Benchmark: MT-Bench (multi-turn chat)
  • Hardware: NVIDIA A100 PCIE GPU (batch=1 가정)
  • Training data: Vicuna 7B/13B에서 ShareGPT 2 epoch. Vicuna-33B와 Zephyr-7B는 self-distill로 ~100K 샘플.
  • Heads: K=5 (empirically 충분, 4도 OK), $\lambda_k = 0.8^k$.

4. 종합 평가

강점

  • 패러다임의 명확한 단순화: "draft model을 제거하라"는 단일 통찰. ##별도 모델 학습/배포/관리의 부담을 완전히 제거##. 코드베이스에 5 epoch FFN-style head 부착으로 끝.
  • Distributed serving 친화: 두 모델 동기화가 없으므로 ##기존 LLM serving infrastructure에 minimal change##로 통합. TensorRT, HuggingFace TGI 채택 사례 (논문 작성 시점 기준).
  • Memory 효율: Medusa-1 + quantization으로 ##단일 consumer GPU에서 7B 모델 학습 가능##(5시간 A100). QLoRA 수준 democratization.
  • 실용적 design choices: heads warmup, differential lr, self-distillation, sparse tree — 모두 ##production에서 발생할 실제 문제##(quality 손실, 데이터 부재)를 정면 해결.
  • Ablation의 완성도: speedup 분해 표(Table 3)에서 각 contribution의 marginal effect를 보여줌. tree size, typical threshold, two-stage 등 design knob를 모두 검증.
  • 역사적 의의: ##Speculative decoding 연구의 새 방향## 제시. 이후 EAGLE, HASS, Hydra 등이 multiple decoding head 아이디어를 발전시킴. DeepSeek-v3의 Multi-Token Prediction도 이 lineage.

한계 및 의문점

  • Token-level autoregression 제한: Head $k$가 head $k-1$의 출력을 직접 참조하지 못함 (parallel prediction). 따라서 ##먼 token일수록 accuracy 급락##(empirically $\lambda_k = 0.8^k$로 가중치 감소). 후속 연구 EAGLE은 feature-level autoregression으로 이 한계 극복.
  • Batch size = 1 가정: 본문이 단일 사용자 시나리오 중심. ##Higher batch에서의 효과는 미보고##. 후속 EAGLE-2, EAGLE-3 등에서 batch=64까지 효과 측정. (논문 끝 "can be generalized to larger batch-size settings"라고만 언급)
  • Medusa-2의 quality 손실 가능성: Vicuna 13B에서 -0.14, Zephyr에서 -0.07. ##잘 visible하진 않지만 quality drift는 존재##. self-distillation으로 부분 완화하나 완전히 해결되진 않음.
  • Tree size의 hardware-specific tuning: ##최적 tree size가 model size, GPU, batch에 따라 다름##. A100 외의 hardware에서 재calibration 필요. production deployment 시 추가 engineering 부담.
  • Cumulative candidate explosion: K=5, $s_k = 4$이면 candidate ~1024개. ##검증 forward pass의 sequence length 확장##으로 memory/compute 증가. Tree pruning이 필수.
  • Typical acceptance의 quality guarantee 약함: Rejection sampling은 ##lossless 보장##(분포 동일). Typical acceptance는 heuristic으로 ##strict한 lossless 보장 없음##. 단 empirically quality 유사.
  • EAGLE/EAGLE-3에 의해 추월됨: 2024 이후 EAGLE 시리즈가 Medusa의 2.83× 대비 ##5×+ speedup##. feature-level autoregression이 multi-token parallel prediction보다 강력함을 입증. ##Medusa는 historical milestone##.
  • 33B 이상 scaling 부족: 33B까지만 실험. 70B/405B 같은 더 큰 모델에서의 효과 미보고. ##scale에 따른 다른 동작## 가능성.

5. Key Takeaways

  1. "Draft model을 제거할 수 있다": ##단순함이 강력함##. 하나의 모델에 여러 head를 붙이는 minimal change로 2.83× 가속. Production deployment의 friction 0에 가까움.
  2. Tree attention은 candidate diversity의 일반 패턴: ##단일 batch 확장 없이 다중 후보 검증##. Beam search, contrastive decoding 등 다른 multi-candidate 알고리즘에도 적용 가능.
  3. Quantization과의 호환성: Medusa-1은 backbone freeze이므로 ##INT8/INT4 quantized backbone에 그대로 부착##. 메모리 한정 환경에 매력적.
  4. Self-distillation으로 데이터 문제 해결: RLHF/proprietary 모델에 적용하는 일반 패턴. ##public seed prompt + 자체 응답 생성##.
  5. 다음 세대의 출발점: EAGLE, EAGLE-2/3, HASS, Hydra가 모두 Medusa의 multiple-head 아이디어를 기반으로 발전. DeepSeek-v3의 MTP도 Medusa lineage. ##speculative decoding 연구의 분기점##.
