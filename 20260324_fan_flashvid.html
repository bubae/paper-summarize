FlashVID: Efficient Video Large Language Models via Training-Free Tree-Based Spatiotemporal Token Merging

  • 저자: Ziyang Fan, Keyu Chen, Ruilong Xing, Yulin Li, Li Jiang, Zhuotao Tian*
  • 소속: Harbin Institute of Technology (Shenzhen), The Chinese University of Hong Kong (Shenzhen), Shenzhen Loop Area Institute
  • 학회/저널: ICLR 2026
  • 연도: 2026
  • 링크: arXiv:2602.08024 · GitHub

TL;DR

  • 문제: Video Large Language Model (VLLM)은 대량의 visual token을 처리해야 하며, attention의 $O(n^2)$ 복잡도로 인해 계산/메모리 비용이 급증한다. 기존 가속 방법은 spatial redundancy와 temporal redundancy를 독립적으로 압축하여, 비디오의 동적 특성(위치/크기/방향 변화)으로 인한 시공간 상관관계를 놓친다.
  • 해결: FlashVID — (1) ADTS (Attention and Diversity-based Token Selection)로 각 프레임에서 informative하면서도 diverse한 대표 토큰을 선별한 뒤, (2) TSTM (Tree-based Spatiotemporal Token Merging)으로 인접 프레임 간 유사 토큰을 계층적 redundancy tree로 연결하여 시공간 redundancy를 jointly 압축한다.
  • 왜 작동하는가: 비디오에서 의미적으로 유사한 객체는 프레임 간 고정된 위치에 있지 않고 이동/변형하므로, rigid spatial correspondence 대신 feature similarity 기반으로 tree를 구성하면 동적 변화를 포착하면서 redundancy를 정확히 식별할 수 있다.
  • 비유: 영상 편집에서 중복 장면을 제거할 때, 기존 방법은 "같은 화면 위치의 픽셀이 비슷하면 제거"(TTM)하는 반면, FlashVID는 "장면 속 같은 객체가 움직여도 추적하여 중복된 표현을 하나로 합치는"(TSTM) 방식이다.

핵심 기여 (Key Contributions)

  1. 시공간 redundancy의 coupled 특성 규명: 비디오에서 temporal redundancy는 고정된 공간 위치에 국한되지 않으며, spatial/temporal redundancy가 본질적으로 결합되어 있음을 관찰하고, 이를 jointly 모델링해야 함을 주장한다.
  2. TSTM (Tree-based Spatiotemporal Token Merging): ##Feature similarity 기반으로 인접 프레임의 토큰을 계층적 redundancy tree로 연결하고, tree 단위로 aggregation##하여 비디오의 동적 변화를 포착하면서 시공간 redundancy를 제거한다.
  3. ADTS (Attention and Diversity-based Token Selection): [CLS] attention + event relevance로 calibrate된 Max-Min Diversity Problem (MMDP)을 풀어, 각 프레임에서 informative하면서 diverse한 대표 토큰을 선별한다.
  4. Training-free plug-and-play: 추가 학습 없이 기존 VLLM에 적용 가능하며, ##10% visual token만 유지해도 LLaVA-OneVision의 99.1% 성능 보존##. Qwen2.5-VL에 적용 시 10배 더 많은 프레임 처리로 8.6% 성능 향상.

2. Background and Motivation

2.1 VLLM Inference Pipeline

VLLM의 inference는 세 단계로 구성된다:

  1. Encoding: Vision encoder (CLIP, SigLIP 등)가 각 프레임을 독립 처리하여 $N_v$개 visual embedding 생성 → modality connector로 text space에 투영하여 $H_v \in \mathbb{R}^{F \times N_v \times d}$ 생성
  2. Prefilling: LLM이 전체 입력 $H$에 대해 self-attention 수행. 각 layer $l$에서 $\mathbf{Q}^l = H^l \mathbf{W}_Q^l$, $\mathbf{K}^l = H^l \mathbf{W}_K^l$, $\mathbf{V}^l = H^l \mathbf{W}_V^l$ 계산하고 KV Cache에 저장
  3. Decoding: Auto-regressive 생성, 새 토큰의 KV만 cache에 추가

Efficiency bottleneck: FLOPs $= L \times (4nd^2 + 2n^2d + 2ndm)$으로, 시퀀스 길이 $n$에 quadratic하게 증가. 비디오에서는 visual token 수 $n_v$가 text token $n_t$보다 수십 배 많으므로, visual token 압축이 핵심이다.

2.2 Key Observations

비디오의 시공간 redundancy에 대한 두 가지 핵심 관찰:

  1. Temporal redundancy는 고정된 공간 위치에 국한되지 않는다: 의미적으로 유사한 시각 요소가 프레임 간 위치/크기/방향을 변화시키므로, rigid spatial correspondence에 기반한 temporal merging (Temporal Token Merging, TTM)은 덜 상관된 토큰을 병합할 위험이 있다.
  2. Spatial과 temporal redundancy는 본질적으로 결합되어 있다: 한 프레임 내의 redundant 영역이 여러 프레임에 걸쳐 반복되므로, 분리 압축은 이 구조를 놓친다.
[Figure 3 설명]: (a) 동일 threshold에서 TSTM(주황색)이 TTM(파란색)보다 프레임당 더 많은 토큰을 병합하면서, 평균 병합 유사도(녹색)도 더 높다. 이는 TSTM이 실제로 더 유사한 토큰 쌍을 찾아 병합함을 의미한다. (b) TTM은 같은 공간 위치의 토큰끼리만 비교하므로 객체 이동 시 다른 토큰을 병합할 수 있다. (c) TSTM은 feature similarity 기반 tree 구조로 이동/변형된 객체도 올바르게 추적하여 병합한다.

3. Methodology

3.1 Overview

FlashVID는 두 개의 시너지 모듈을 순차적으로 적용한다:

Video Frames → Vision Encoder → Visual Features E_v ∈ R^{F×N_v×d} → [Stage 1: ADTS] Frame-wise token selection (MMDP + calibration) → Representative tokens I^(f), Remaining tokens R^(f) → [Stage 2: TSTM] Cross-frame redundancy tree construction + aggregation → Compressed tokens C ∪ (⋃ I^(f)) → LLM → Response
[Figure 4 설명]: FlashVID의 전체 구조. 왼쪽: ADTS가 intra-frame distance matrix를 calibrate하여 diverse + informative 토큰을 선별. TSTM이 인접 프레임 간 similarity로 spatiotemporal redundancy tree를 구성하고 aggregation. 오른쪽: Vision Encoder → Compression → LLM의 before-LLM compression 파이프라인.

3.2 TSTM (Tree-based Spatiotemporal Token Merging)

Step 1: Spatiotemporal redundancy tree 구성

Video features $E_v \in \mathbb{R}^{F \times N_v \times d}$에서, 인접 프레임 간 cosine similarity matrix를 계산한다:

$$S^{(f)} = \cos(E_v^{(f)}, E_v^{(f+1)}) \in \mathbb{R}^{N_v \times N_v} \quad (4)$$

$S^{(f)}(j, k)$는 프레임 $f$의 $j$번째 토큰과 프레임 $(f+1)$의 $k$번째 토큰 간 유사도이다. 각 토큰은 이전 프레임에서 가장 유사한 토큰에 연결되며, 유사도가 merging threshold $T_\tau$를 초과할 때만 연결한다. 이렇게 점진적으로 spatiotemporal redundancy tree가 형성되며, 비유사한 토큰의 병합을 방지한다.

Step 2: Tree aggregation으로 시공간 redundancy 압축

$$c^{(i)} = \text{Agg}(\mathcal{T}^{(i)}) \quad (5)$$

$\mathcal{T}^{(i)}$는 $i$번째 redundancy tree, $\text{Agg}(\cdot)$는 aggregation function (mean pooling)이다. 각 tree의 모든 토큰이 하나의 compact representation으로 합쳐진다.

핵심: TTM (Temporal Token Merging)은 같은 공간 위치($j = k$)의 토큰만 비교하지만, ##TSTM은 모든 위치 간 비교##($N_v \times N_v$ matrix)를 수행하여, 움직이거나 변형된 객체도 올바르게 추적하여 병합한다.

3.3 ADTS (Attention and Diversity-based Token Selection)

TSTM만으로는 noisy/redundant 입력에서 중요한 토큰을 놓칠 수 있으므로, ADTS가 각 프레임에서 대표 토큰을 먼저 선별한다.

기본 원리: Frame-wise Max-Min Diversity Problem (MMDP)을 풀어, 최대 최소 거리를 가진 diverse token subset을 선택한다.

Frame-wise cosine distance matrix:

$$D^{(f)} = 1 - \cos(E_v^{(f)}, E_v^{(f)}) \quad (6)$$

Diversity만으로는 가장 informative한 토큰을 놓칠 수 있으므로, 두 가지 calibration을 도입한다:

[CLS] Attention Calibration

Vision encoder의 attention matrix에서 각 토큰이 받는 평균 attention weight를 추출한다:

$$A = \text{Softmax}(QK^T / \sqrt{d}) \in \mathbb{R}^{F \times N_v \times N_v} \quad (7)$$

$A_{\text{[CLS]}} \in \mathbb{R}^{F \times N_v}$로 각 토큰의 informative 정도를 나타낸다.

Event Relevance Calibration

각 토큰이 전체 비디오 이벤트와 얼마나 관련되는지를 측정한다. Global average pooling으로 프레임 임베딩 $f_v = \text{GAP}(E_v)$를 얻고:

$$\mathbf{S}_e = \frac{1}{F} \sum_{i=1}^{F} (E_v \cdot f_v^\top)[:, :, i] \in \mathbb{R}^{F \times N_v} \quad (8)$$

이를 통해 비디오의 전체 맥락과 관련된 토큰을 강조한다.

최종 calibrated MMDP:

$$\mathcal{I} = \text{MMDP}(D, A_{\text{[CLS]}}, \mathbf{S}_e) \quad (9)$$

Algorithm 1: FlashVID Compression

Stage 1: ADTS for f = 1 to F: Compute D^(f), A_[CLS]^(f), S_e^(f) I^(f) ← MMDP(D^(f), A_[CLS]^(f), S_e^(f)) // selected tokens R^(f) ← E_v^(f) \ I^(f) // remaining tokens Stage 2: TSTM Initialize R^(1) tokens as root nodes, C = ∅ for f = 2 to F: for each r_i^f ∈ R^(f): p* ← argmax_{p ∈ R^(f-1)} sim(r_i^f, p) if sim(r_i^f, p*) ≥ T_τ: connect r_i^f to p* for each tree T: C ← C ∪ Agg(T) // mean pooling return X̃ = C ∪ (⋃ I^(f)) // compressed output

4. Experiments

FlashVID의 효과를 검증하기 위한 핵심 질문:

  • RQ1: FlashVID가 기존 SOTA training-free 방법 대비 더 나은 성능-효율 trade-off를 보이는가?
  • RQ2: 고정 계산 예산에서 더 많은 프레임을 처리할 수 있는가?
  • RQ3: ADTS와 TSTM 각각의 기여는?

5. Conclusion

FlashVID는 ADTS + TSTM 두 모듈로 비디오의 시공간 redundancy를 jointly 압축하는 training-free acceleration framework이다. 3개 VLLM × 5개 벤치마크에서 기존 SOTA를 일관되게 능가하며, plug-and-play 모듈로 더 긴 비디오 처리를 가능하게 한다.

비판적 분석 (Critical Analysis)

강점

  • 명확한 문제 정의와 직관적 해법: "시공간 redundancy는 coupled되어 있다"는 관찰이 설득력 있고, tree-based merging이라는 해법이 자연스럽다. Figure 3의 TSTM vs TTM 비교가 매우 효과적이다.
  • 포괄적 실험: 3개 VLLM (LLaVA-OneVision, LLaVA-Video, Qwen2.5-VL) × 5개 벤치마크 × 4개 retention ratio로 광범위하게 검증. 아키텍처 일반성을 잘 보여준다.
  • 실용적 가치: Training-free, plug-and-play로 즉시 적용 가능. 10% retention에서 99.1% 성능 보존은 매우 실용적이다.
  • ICLR 2026 accepted: 검증된 venue에서 accept됨.

약점

  • TSTM의 $O(F \times N_v^2)$ 복잡도: 인접 프레임 간 $N_v \times N_v$ similarity matrix 계산이 필요. 프레임당 토큰 수가 많으면 compression 자체의 오버헤드가 커진다 (Table 6에서 60.2ms). 실시간 비디오 처리에서는 병목이 될 수 있다.
  • Merging threshold $T_\tau$의 민감도 미분석: TSTM의 핵심 하이퍼파라미터인 $T_\tau$에 대한 ablation이 본문에 없다. Appendix A.3에서 DPC-kNN threshold ablation은 있으나 $T_\tau$ 자체는 불명확.
  • Audio/subtitle 등 다른 modality 미고려: Long video understanding에서 audio 정보가 중요할 수 있으나, visual token만 압축한다.
  • Tree depth/breadth 제약의 효과 미검증: 저자가 "tree depth와 breadth를 제약해봤으나 무시할 수 있는 차이"라고 언급하지만, 이에 대한 수치가 본문에 없다.
  • 표준편차 미보고: 모든 결과가 단일 점추정치로 보고되어 통계적 유의성 불명.

Key Takeaways

  1. 시공간 redundancy는 jointly 압축해야 한다: 비디오에서 spatial/temporal redundancy를 독립 처리하면 객체의 동적 변화를 놓치고 무관한 토큰을 병합할 위험이 있다. Feature similarity 기반의 tree 구조가 이를 해결하는 효과적인 방법이다.
  2. "Less is more"가 VLLM에서도 성립: 90% 토큰을 제거해도 99.1% 성능 보존, 심지어 일부 설정에서는 vanilla을 능가. 과도한 visual token이 오히려 노이즈로 작용할 수 있다.
  3. Training-free compression으로 프레임 수를 10배 확장: 토큰 압축으로 절약한 예산을 프레임 수 증가에 투자하면, 동일 비용으로 더 긴 시간적 맥락을 활용하여 ##long video understanding에서 8.6% 성능 향상##을 달성할 수 있다.
  4. Diversity + Relevance의 조합이 핵심: MMDP(diversity)만으로는 informative 토큰을 놓치고, attention(relevance)만으로는 다양성이 부족. [CLS] attention + event relevance로 calibrate된 MMDP가 최적의 token selection을 제공한다.
