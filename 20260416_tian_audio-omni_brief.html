먼저 알아야 할 것: AI와 소리의 세계

Audio-Omni를 이해하려면 5가지 배경 지식이 필요합니다. 하나씩 그림과 함께 설명합니다.

개념 1: 컴퓨터가 "소리"를 다루는 방법

우리에게 소리는 귀로 듣는 것이지만, 컴퓨터에게 소리란 숫자의 나열입니다. 소리를 처리하려면 먼저 소리를 컴퓨터가 이해할 수 있는 형태로 변환해야 합니다. 이 과정은 세 단계를 거칩니다.

마이크 → 파형 → 스펙트로그램 → Mel-스펙트로그램: AI가 소리를 "보는" 과정
사진과 소리의 비유

카메라가 빛을 포착하여 이미지(2D 픽셀 격자)를 만들듯, 마이크가 공기 진동을 포착하여 스펙트로그램(2D 주파수-시간 격자)을 만듭니다. AI에게 소리를 이해시키는 것은 사실상 "소리 사진"을 보여주는 것입니다. Mel-스펙트로그램은 이 "소리 사진"의 해상도를 사람 귀에 맞춰 최적화한 버전입니다.

개념 2: 소리의 세 가지 세계 — 왜 통합이 어려운가?

우리가 듣는 소리는 크게 세 종류로 나뉩니다. 중요한 것은 이 세 종류가 파형 구조, 생성 원리, AI 처리 방법이 모두 다르다는 점입니다.

세 도메인은 파형 특성, AI 과제, 평가 방법이 모두 다르다 — 하나의 모델로 통합하기 어려운 이유

기존 AI 모델들은 이 중 한두 가지 도메인에만 특화되어 있었습니다. 음성 인식(ASR) 모델은 사운드 효과를 만들지 못하고, 음악 생성 모델은 "안녕하세요"를 말하지 못합니다. Audio-Omni는 이 세 가지 세계를 모두 다루는 최초의 통합 모델입니다.

개념 3: 소리를 다루는 세 가지 능력

AI가 소리를 다루는 방법은 세 가지로 나뉩니다. 각각 완전히 다른 기술이 필요하고, 기존에는 각 능력마다 별도의 모델이 필요했습니다.

이해는 "분석", 생성은 "창작", 편집은 "수정" — 기존에는 각각 별도의 AI 모델이 필요했다
비유: 사진 전문가의 세 가지 능력

이해는 사진 평론가 — 사진을 보고 "이것은 일몰의 해변 풍경이다"라고 설명합니다.
생성은 사진작가 — 아무것도 없는 상태에서 "일몰의 해변"을 촬영합니다.
편집은 포토샵 디자이너 — 기존 사진에서 "하늘을 석양빛으로 바꿔라"를 수행합니다.

기존에는 이 세 사람을 따로 고용해야 했습니다. Audio-Omni는 이 세 가지 능력을 모두 가진 한 명의 오디오 전문가입니다.

개념 4: MLLM — 여러 감각을 가진 AI 두뇌

ChatGPT 같은 대형 언어 모델(LLM)은 텍스트만 이해합니다. 여기에 "눈"(이미지/비디오 이해)과 "귀"(오디오 이해)를 추가한 것이 MLLM (Multimodal Large Language Model)입니다.

MLLM은 텍스트·이미지·오디오를 모두 이해하는 "멀티감각 AI 두뇌"

Audio-Omni는 이미 학습 완료된 MLLM(Qwen2.5-Omni-3B, 약 49억 파라미터)을 그대로 가져와서 "두뇌"로 사용합니다. 이 두뇌가 소리를 이해하고, 그 이해를 바탕으로 "소리를 만드는 손"에게 지시를 내립니다.

개념 5: Diffusion Model — 잡음에서 소리를 만드는 마법

오디오를 "만들어내는" 부분은 Diffusion Model(확산 모델)이라는 기술을 사용합니다. 핵심 원리를 단계별로 설명합니다.

Diffusion: 학습 시 소리→잡음 과정을 관찰하고, 생성 시 잡음→소리 역과정을 100단계에 걸쳐 수행
비유: 조각가의 작업 과정

조각가는 대리석 덩어리(순수 잡음)에서 시작하여, 조금씩 불필요한 부분을 깎아내어 작품(깨끗한 소리)을 완성합니다. 100번의 정교한 끌질(step)을 거치며, "어떤 조각상을 만들지"는 설계도(conditioning signal)가 알려줍니다.

Audio-Omni가 사용하는 Rectified Flow는 기존 diffusion의 "구불구불한 경로"를 "직선 고속도로"로 바꾼 것입니다. 같은 100단계여도 더 빠르고 정확하게 목적지에 도달합니다.

Audio-Omni의 핵심 아이디어

핵심 아이디어는 놀라울 정도로 간결합니다: "이해하는 두뇌(MLLM)"와 "소리를 만드는 손(DiT)"을 분리하되, 정교한 이중 소통 채널로 연결하자."

전체 구조: 두뇌 + 두 갈래 소통 + 생성기

MLLM 두뇌(49억 파라미터)는 얼리고, DiT 생성기(30.5억)만 학습. 두 갈래 채널이 핵심 혁신.
비유: 영화 촬영 현장

MLLM = 수십 년 경력의 감독. 대본(텍스트), 참고 영상(비디오), 참고 음악(오디오)을 종합적으로 이해합니다. 새 프로젝트마다 감독을 재교육할 필요가 없습니다(Frozen).

DiT = 새로 모집한 오케스트라. 감독의 지시를 받아 실제 연주(소리 생성)를 담당합니다. 이 오케스트라만 훈련합니다(Trainable).

두 채널 = 감독이 오케스트라에게 지시하는 두 가지 방법:
  (1) "분위기 지시": "이 장면은 슬프게, 느리게" → 자유롭게 해석 가능 (Cross-Attention)
  (2) "정확한 박자/타이밍": "3초 뒤 시작, 이 비트에 맞춰" → 정밀하게 따라야 함 (Concatenation)

두 갈래 채널의 원리: Cross-Attention vs Concatenation

이 논문의 핵심 기술적 혁신은 두 종류의 조건부 신호를 서로 다른 메커니즘으로 주입하는 것입니다. 왜 하나의 방식으로 통일하지 않고 굳이 두 가지를 쓸까요?

의미적 지시(무엇을) → Cross-Attention, 시간적 신호(언제/어떻게) → Concatenation

실험 결과, 이 두 방식을 분리하는 것이 모든 태스크에서 최적의 성능을 보였습니다. 모든 정보를 Cross-Attention으로만 전달하면 T2A 성능이 40.91에서 28.90으로 크게 떨어졌고, 반대로 모든 것을 Concatenation으로 전달해도 60.58로 나빠졌습니다.

비밀 무기: AudioEdit 데이터셋 (100만+ 건)

AI를 "편집"까지 가르치려면 "원본 소리 + 편집 지시 + 결과물"이 묶인 대량의 데이터가 필요합니다. 하지만 이런 데이터는 거의 없었습니다. Audio-Omni 팀은 독창적인 2갈래 파이프라인으로 100만 건 이상의 편집 데이터를 직접 만들었습니다.

4가지 편집 태스크

4가지 편집 태스크를 합쳐 총 110만 학습 샘플 + 2,000 테스트 샘플

Real Data Branch: 실제 녹음에서 편집 쌍 채굴

VGGSound(실제 영상 오디오 데이터셋)에서 시작하여, AI(Gemini 2.5 Pro)가 소리 속 물체를 식별하고, 소스 분리(SAM-Audio)로 각 물체별 트랙을 분리합니다. 원본과 분리된 트랙이 자연스럽게 "편집 전/후" 쌍이 됩니다.

다만 품질이 좋은 쌍만 사용해야 하므로, 다단계 필터링(VAD + CLAP semantic alignment)을 거칩니다. 최종 잔존율은 약 9.2%로 엄격한 품질 관리를 수행합니다.

Synthesis Data Branch: 프로그래밍으로 합성

Scaper 툴킷으로 ESC-50의 전경 이벤트를 AudioCaps의 배경에 프로그래밍 방식으로 믹싱합니다. SNR(0~3dB), 피치(±3 semitones), 속도(0.8~1.2×) 등을 랜덤으로 변화시켜 다양한 시나리오를 자동 생성합니다. "편집 전/후"가 프로그램으로 완벽히 정의되므로 annotation이 자동입니다.

비유: 요리 학원의 커리큘럼

Real Data Branch = 미슈랭 셰프의 실제 레시피를 분석하여 "소스를 추가/제거하면 맛이 어떻게 변하는지" 학습. 높은 품질이지만 수집이 어렵고 비쌈.

Synthesis Data Branch = 수십만 가지 자동 생성 레시피 변형으로 다양한 상황을 경험. 품질은 약간 낮지만 대량 생산이 가능.

두 가지를 합치면 품질(Real) × 다양성(Synthesis)의 시너지가 발생합니다. 실험으로 확인: 합성만 사용하면 FD 37.96, 실제만 사용하면 20.69, 혼합하면 20.48로 최적.

왜 이렇게 설계했나? — 핵심 실험 근거

논문은 세 가지 핵심 설계 선택이 정말 최적인지 실험으로 검증합니다.

실험 1: Feature는 어디에서 추출할까?

MLLM에서 DiT에 전달할 feature를 어느 레이어에서 추출할지가 중요합니다. 결론: 마지막에서 두 번째 레이어(Penultimate Layer)가 최적입니다.

마지막 레이어는 텍스트 예측에 과도하게 특화되어 있어, 소리 합성에 필요한 풍부한 의미·음향 정보가 압축되어 있습니다. 바로 그 직전 레이어가 가장 풍부한 정보를 유지하고 있습니다.

Penultimate Layer가 IS 11.26 / FD 2.75로 압도적 최적 — 고품질 합성은 dense feature에 직접 접근이 핵심

실험 2: 통합 MLLM vs 전문 인코더

MLLM(Qwen) 대신 각 모달리티별 전문 인코더(T5, CLAP, CLIP, VideoMAE)를 쓰면 어떨까? 결론: 통합 MLLM이 모든 전문 인코더보다 우수합니다.

이유: MLLM은 텍스트·오디오·비디오를 함께 학습하면서 modality 간의 관계를 깊이 이해하게 됩니다. 전문 인코더는 자기 모달리티만 잘 알지, 다른 모달리티와의 연결고리가 약합니다.

놀라운 보너스: 학습하지 않은 능력의 발현

Audio-Omni는 직접 가르치지 않은 능력까지 보여줍니다. MLLM의 방대한 지식이 자연스럽게 DiT로 전이되기 때문입니다.

지식 기반 생성 (Knowledge-Augmented Generation)

"지미 헨드릭스가 유명하게 연주한 악기 소리를 생성하라"는 프롬프트에 대해, MLLM이 해당 악기가 일렉트릭 기타임을 스스로 추론하여 기타 멜로디를 합성합니다. 일반 사운드 생성 AI는 이런 세계 지식을 갖고 있지 않아 불가능합니다.

맥락 학습 생성 (In-Context Generation)

피아노 녹음을 들려주고 "이 분위기로 긴장감 있는 chord progression을 만들어라"고 하면, 피아노의 음색을 추출하여 새로운 작품에 적용합니다. "참고 음원의 스타일을 이해하고 확장하는" 능력입니다.

Zero-Shot 다국어 소리 생성

거의 영어만으로 학습했음에도, 중국어(FAD 2.26) · 독일어(2.92) · 일본어(3.13) · 스페인어(4.32) · 프랑스어(4.21)로 된 텍스트 설명을 이해하고 대응하는 소리를 생성합니다.

Zero-Shot Voice Cloning / Speech Editing

Speech prompt masking 학습 덕분에, 별도 지도 없이 화자의 목소리를 복제하거나, 기존 발화의 일부 내용을 수정할 수 있습니다. 20~75%의 mel-spectrogram을 마스킹하여 학습하면, 모델이 부분적 음향 정보만으로 화자를 추론하고 전체를 재구성하는 능력을 자연스럽게 획득합니다.

비유: 다국어를 구사하는 통역 작곡가

여러 언어를 유창하게 하는 음악 감독이 있습니다. 일본어로 "슬픈 비 오는 날의 피아노 곡"이라고 주문하면, 감독이 일본어를 이해하고 오케스트라에 영어로 지시하여 곡을 만듭니다. 또한 "지미 헨드릭스 스타일로"라고 하면 자신의 음악 지식으로 일렉트릭 기타 연주를 지시합니다. Audio-Omni의 MLLM이 바로 이 "다국어·박학다식 감독" 역할입니다.

한계점

전문 모델 대비 일부 격차: 만능형 모델이므로, Text-to-Music(FAD 1.94 vs AudioX 1.53)이나 Video-to-Audio(FAD 1.71 vs AudioX 1.19)에서는 해당 분야만 집중 학습한 전문 모델에 약간 뒤집니다. 이는 "만능형"과 "전문형"의 본질적 trade-off입니다.

이해 능력이 기저 모델에 종속: MLLM을 얼려두었기 때문에, 이해 성능은 Qwen2.5-Omni-3B의 한계에 묶입니다. 더 강력한 MLLM이 나오면 성능이 올라가겠지만, 현재 상태에서는 이해 능력의 개선이 불가능합니다.

생성 속도: 100단계의 diffusion 과정을 거치므로, 실시간 음성 대화 같은 초저지연 응용에는 부담됩니다. Few-step distillation(소수 단계 증류) 등의 가속 기법이 향후 과제입니다.

편집 데이터의 Real Data 다양성: Real Data Branch의 최종 잔존율이 9.2%로 매우 낮아, 실제로 확보된 고품질 real data의 다양성이 합성 데이터만큼 풍부한지 의문이 있습니다.

윤리적 우려: Voice conversion과 speech synthesis 기능은 딥페이크 음성 생성에 악용될 수 있습니다. 연구진은 responsible-use terms과 오디오 워터마킹/탐지 기법 개발을 촉구합니다.
