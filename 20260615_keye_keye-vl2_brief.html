📄 전문가용 상세 요약 보기 →
긴 영상의 모든 프레임을 다 보지 않고, "지금 답하는 데 가장 중요한 2048개 토큰만 골라서 보는" 똑똑한 멀티모달 모델

📚 배경 지식 — 왜 영상 이해가 어려운가?

1. 영상 = 어마어마한 토큰 더미

AI가 이미지를 처리할 때 그림은 작은 조각(token)으로 쪼개져 모델에 입력됩니다. 1장의 이미지가 약 1000~2000 토큰이라면, 1초당 1프레임씩 추출한 1시간 영상은 약 360만 토큰이 됩니다.

💡 비유: 1장의 사진이 짧은 메모라면, 2시간 영상은 책 100권을 한꺼번에 던져주고 "이 영상에 대해 답해줘"라고 하는 것과 같습니다. 모델은 모든 페이지를 다 읽지 않고도 답할 수 있어야 합니다.

2. Attention의 비용 폭발 — Quadratic 문제

Transformer 모델의 핵심 메커니즘인 "attention"은 입력 토큰 수에 대해 제곱(quadratic)으로 비용이 늘어납니다. 토큰 수가 2배가 되면 계산량이 4배, 10배가 되면 100배가 됩니다.

💡 Dense vs Sparse 비유: Dense attention은 "회의에 100명이 모이면 각자가 다른 모든 사람과 일대일로 이야기하는" 방식입니다 (대화 횟수 = 100×100 = 10000). Sparse attention은 "각자가 가장 관련 있는 5명과만 이야기하는" 방식입니다 (대화 횟수 = 100×5 = 500). 결과는 비슷하면서 효율은 압도적으로 좋습니다.

3. Multi-task 충돌 — Catastrophic Forgetting

한 모델이 여러 능력을 한꺼번에 익히려고 하면, 새로운 것을 배울 때 옛 것을 잊는 현상이 일어납니다. 영상 이해 → 도구 사용 → 수학 추론 → OCR 순서로 학습하면, 마지막 단계에서 OCR은 잘하지만 영상 이해 능력은 떨어집니다.

4. Mixture-of-Experts (MoE)란?

모델 안에 여러 "전문가(expert)"를 두고, 각 입력에 가장 적합한 전문가만 활성화하는 구조입니다. Keye-VL-2.0은 총 30B 파라미터를 가지지만, 실제로 매번 활성화되는 건 3B만 — 작은 모델 비용으로 큰 모델 성능을 냅니다.

💡 MoE 비유: 종합병원에는 다양한 과(내과·외과·정형외과...)가 있지만, 환자가 오면 증상에 맞는 1-2개 과만 진료합니다. 모든 의사가 다 진료하는 게 아닙니다. MoE도 마찬가지로, 입력 종류에 맞는 전문가만 일하고 나머지는 잠들어 있습니다.

🎯 핵심 아이디어 — 4가지 혁신

1

DSA로 256K context

DeepSeek Sparse Attention을 GQA 백본에 적용. 매 토큰이 가장 관련 있는 2048개만 골라서 attention 계산. Dense 대비 prefill 0.32×, decode 0.20× 비용.

2

Native Resolution + 2D RoPE

이미지를 정해진 크기로 자르거나 늘리지 않고 원래 해상도 그대로 인코딩. NaViT의 Patch n' Pack으로 다른 크기 샘플을 함께 batch.

3

4단계 Pre-training

Stage 0(8K, projector init) → Stage 1(32K, 일반) → Stage 2(64K, 다중 task 주입) → Stage 3(256K, 긴 영상). 단계마다 영상 길이도 15초 → 15분 → 2시간으로 확장.

4

MOPD — 13명의 전문가

Cross-Modal Multi-Teacher On-Policy Distillation. 13개 도메인 전문가(safety/math/code/video/tool 등)가 학생의 답에 token-level 피드백. 각 전문가의 능력을 단일 모델로 합치되 충돌 없이.

🔍 DSA가 어떻게 작동하나

Sparse Attention의 핵심 단계

💡 DSA 비유: 도서관에서 논문을 쓸 때 모든 책을 처음부터 끝까지 읽지 않습니다. 먼저 검색(Lightning Indexer)으로 가장 관련 있는 책 20권을 빠르게 찾고, 그 책들만 자세히 읽습니다(Sparse Aggregation). DSA는 매 토큰마다 이 과정을 자동으로 합니다.

Dense Warm-up + Sparse Adaptation

Sparse 학습을 처음부터 하면 indexer가 무엇을 골라야 할지 모릅니다. 그래서:

  1. 1단계 (Dense Warm-up): 일반 dense attention을 하면서, indexer가 dense의 attention 분포를 따라하도록 학습 (KL divergence로 align)
  2. 2단계 (Sparse Adaptation): 모든 파라미터 unfreeze. Indexer는 dense 분포에 align 유지하되, KL loss는 selected Top-k token에서만 계산
💡 비유: 학생에게 처음부터 "중요한 부분만 골라서 시험 봐라"라고 하면 무엇이 중요한지 모릅니다. 먼저 모든 부분을 다 본 결과를 보여준 후 (warm-up), "이 패턴을 보고 핵심을 골라라"라고 가르치는 것 (adaptation)이 효과적입니다.

🎓 MOPD — 13명의 과외 선생님

일반 Distillation의 문제와 MOPD의 해결

💡 MOPD 비유: 한 학생이 수학·물리·영어를 잘하려면, 한 명의 만능 선생보다 각 과목 전문 과외 선생이 좋습니다. 다만 학생이 풀어본 실제 답안을 보고 "여기는 이렇게 고쳐"라고 하는 게(on-policy), 그냥 모범답안을 베끼게 하는 것보다 효과적입니다. MOPD가 정확히 이 방식입니다.

⚙️ 영상 처리의 핵심 트릭

1. Adaptive Pixel Budget — 영상 길이별 압축률 조정

짧은 영상은 redundant한 frame이 많고, 긴 영상은 더 많은 frame이 필요합니다. Keye-VL-2.0은 영상 길이에 따라 픽셀 budget을 다르게 할당:

💡 비유: 1분짜리 광고는 1초당 1프레임이면 충분하지만, 2시간 영화는 더 많은 프레임이 필요합니다. 짧은 영상의 고해상도 처리는 낭비, 긴 영상의 저해상도는 정보 손실. 이 trade-off를 자동으로 조정합니다.

2. Timestamp 자연어 삽입

각 프레임 앞에 "00:15", "00:32" 같은 타임스탬프를 자연어로 미리 적어줍니다. 별도의 video encoder를 두지 않고 image-as-frame으로 처리. 모델은 이미 자연어를 이해할 수 있으므로, 시간 순서·인과 관계·절대 시각을 자연스럽게 학습합니다.

3. Scene-wise Dense Caption

영상을 scene(장면) 단위로 분해하고, 각 scene에 timestamp + 자세한 caption + global overview를 부여. 이로써 scene 경계 인식 + temporal alignment가 강화됩니다.

🌟 활용 분야

🎯 한 줄 결론

"긴 영상의 모든 토큰을 다 보지 않고, 매 시점마다 가장 중요한 0.8%만 골라보는 Sparse Attention과, 13명 전문가의 능력을 충돌 없이 합치는 MOPD로, 30B/3B-active 모델이 235B 모델을 능가"

⚠️ 한계점

1. Ablation 부족

Technical report 스타일이라 각 컴포넌트의 기여도를 분리한 ablation이 거의 없습니다. DSA의 dense baseline 비교, MOPD vs simple SFT-distillation 비교 등 어느 게 얼마나 효과적인지 불분명합니다.

2. Long-range 일관성에서 235B에 뒤짐

Video-MME-v2의 group-based non-linear score (멀리 떨어진 4개 질문에 일관되게 답하는 능력)에서는 더 큰 235B 모델에 뒤집니다. 모델 크기와 long-range reasoning 사이의 trade-off가 여전히 존재.

3. Repository-level Code 약함

SWE-bench Verified에서 62.0으로 Qwen3.5-35B(63.5)에 약간 뒤집니다. Algorithmic problem은 잘하지만 large repository 환경에서의 코딩은 추가 학습 필요.

4. 13개 Teacher의 구성 분석 부재

13개 전문가 중 어떤 것이 핵심이고, 일부를 제거하면 어떻게 되는지 분석이 부재합니다. 어떤 task에 어떤 teacher가 가장 기여했는지 알기 어렵습니다.

5. Hyperparameter Tuning 비공개

Adaptive video pixel budget의 threshold (256s/512s/1024s/2048s)나 Top-k=2048이 어떻게 정해졌는지, 다른 값에서는 어떨지 sweep 결과가 없습니다.
