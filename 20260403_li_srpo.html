Unifying Group-Relative and Self-Distillation Policy Optimization via Sample Routing (SRPO)

TL;DR

  • 문제: RLVR(Reinforcement Learning with Verifiable Rewards)에서 GRPO는 안정적이지만 coarse credit assignment(sequence-level advantage, 모든 토큰에 동일하게 적용)로 sample efficiency가 낮다. SDPO(Self-Distillation PO)는 dense logit-level guidance로 초기 수렴이 빠르지만, (1) correct sample에 대한 distillation이 optimization ambiguity를 초래하고 (2) self-teacher의 signal quality가 학습 후반에 degradation하여 ##late-stage collapse##가 빈발.
  • 해결: SRPO (Sample-Routed Policy Optimization) — ##correct rollout은 GRPO branch로, incorrect rollout(with teacher info)은 SDPO branch로 routing##하는 통합 on-policy framework. SDPO branch에는 entropy-aware dynamic weighting으로 unreliable(high-entropy) distillation target을 suppress.
  • 왜 작동하는가: (1) Correct sample에는 reward-aligned GRPO가 적합 (sequence-level advantage가 already-correct trajectory를 안정적으로 reinforce). (2) Incorrect sample에는 SDPO의 dense logit correction이 token-level deviations를 효과적으로 수정. (3) Routing이 training 진행에 따라 자연스럽게 적응 — 초기에는 failure 많아 SDPO 비중 ↑ (빠른 수렴), 후기에는 success 많아 GRPO 비중 ↑ (안정적 plateau).
  • 비유: 수학 교실에서, 맞은 학생에게는 "잘했어, 이 방식으로 계속해"(GRPO: reward reinforcement)라고 하고, 틀린 학생에게는 "이 부분이 틀렸으니 이렇게 고쳐봐"(SDPO: dense correction)라고 하는 것이다. 둘 다에게 같은 피드백을 주면 비효율적이다.

핵심 기여 (Key Contributions)

  1. SDPO의 Late-Stage Instability 원인 규명: (1) Correct sample에 대한 self-distillation이 ##optimization ambiguity## 유발 — reward-equivalent한 다른 sibling의 logit distribution을 강제하는 것이 counterproductive. (2) Self-teacher의 token-level entropy가 학습 진행에 따라 증가하여 distillation signal의 reliability가 degradation.
  2. SRPO — Sample-Level Routing Framework: Binary correctness flag + teacher availability로 sample routing. 두 branch가 동일 policy의 on-policy trajectory에서 동작하므로 ##policy-gradient 구조 보존##. 별도 mixing hyperparameter 없이 sample 비율로 자연스러운 adaptive weighting.
  3. Entropy-Aware Dynamic Weighting (DW-SDPO): SDPO branch에서 token-level self-teacher entropy $H_{i,t}$에 따라 $\tilde{w}_{i,t} = \exp(-\beta H_{i,t})$로 weight. High-confidence(low entropy) correction 강조, uncertain target suppress.
  4. 5 Benchmark, 2 Model Scale에서 검증: Qwen3-8B에서 5-benchmark avg 77.4 (GRPO 74.0, SDPO 71.1 대비 ##+3.4, +6.3##). Qwen3-4B에서도 74.2 (GRPO 69.7 대비 +4.5). Per-step compute cost 최대 17.2% 감소.

2. Preliminaries

2.1 GRPO (Group Relative Policy Optimization)

Prompt $x$에 대해 $G$개 on-policy rollout $\{y_i\}_{i=1}^G$. Group-normalized advantage:

$$A_i^{\text{GRPO}} = \frac{r_i - \bar{r}}{\sigma_r + \epsilon}$$

Clipped surrogate objective:

$$\mathcal{L}_{\text{GRPO}}(\theta) = \mathbb{E}\left[\min\left(\rho_{i,t}(\theta) A_i^{\text{GRPO}}, \; \text{clip}(\rho_{i,t}(\theta), 1-\varepsilon, 1+\varepsilon) A_i^{\text{GRPO}}\right)\right]$$

$A_i^{\text{GRPO}}$는 sequence-level scalar → 모든 token에 동일하게 적용. Correct rollout에는 적절하지만, incorrect rollout에서 ##어떤 개별 토큰이 오류인지## 식별하지 못한다.

2.2 SDPO (Self-Distillation Policy Optimization)

Self-teacher $\pi_\theta(\cdot | x, f)$ (feedback-conditioned)에서 dense logit-level supervision. 학생과 교사 간 KL divergence 최소화:

$$\mathcal{L}_{\text{SDPO}}(\theta) = \sum_t \text{KL}\left(\pi_\theta(\cdot | x, y_{i,Self-teacher는 successful sibling rollout이나 environment feedback $f$를 conditioning하여, 원래 trajectory를 re-score. Dense하지만, correct sample에 적용하면 ##reward-equivalent한 다른 reasoning path를 임의로 강제##하는 ambiguity 발생.

3. The SRPO

3.1 Sample-Level Routing

각 rollout $y_i$에 대해:

  • $c_i = \mathbf{1}[y_i \text{ is correct}]$: Correctness flag
  • $m_i = \mathbf{1}[\text{teacher information available for } y_i]$: Teacher availability

Routing mask:

$$z_i^{\text{SDPO}} = (1 - c_i) \cdot m_i, \quad z_i^{\text{GRPO}} = 1 - z_i^{\text{SDPO}}$$

##Incorrect + teacher available → SDPO branch. 나머지 전부 → GRPO branch##. 두 branch 모두 동일 policy $\pi_\theta$의 on-policy trajectory에서 동작하므로 policy-gradient 구조 보존. Gradient의 형태만 다름:

$$\nabla_\theta \mathcal{L}_{\text{GRPO}} = -\mathbb{E}\left[\sum_t \nabla_\theta \log \pi_\theta(y_t | x, y_{GRPO의 advantage는 reward-derived + sequence-level, SDPO의 advantage는 teacher-derived + logit-level. ##Sample routing은 각 sample에 가장 적합한 advantage estimator를 선택##하는 것.

3.2 Dynamic-Weighted SDPO (DW-SDPO)

Self-teacher의 token-level entropy:

$$H_{i,t} = -\sum_{v \in \mathcal{V}} q_{i,t}(v) \log q_{i,t}(v)$$

Entropy-aware weight:

$$\tilde{w}_{i,t} = \exp(-\beta H_{i,t})$$

Normalized weight (total SDPO loss scale 보존):

$$w_{i,t} = \frac{\tilde{w}_{i,t}}{\frac{1}{|\Omega_{\text{sdpo}}|} \sum_{(j,s) \in \Omega_{\text{sdpo}}} \tilde{w}_{j,s}}$$

$\ell_{i,t}^{\text{DW-SDPO}} = w_{i,t} \cdot \ell_{i,t}^{\text{SDPO}}$. ##Low-entropy(high confidence) token에 더 큰 weight → 교사가 확신하는 correction만 강조##. $\beta=1$ (default).

3.3 Training Objective

$$\mathcal{L}_{\text{final}} = \frac{\sum_{i,t} z_i^{\text{GRPO}} \ell_{i,t}^{\text{GRPO}} + \sum_{i,t} z_i^{\text{SDPO}} \ell_{i,t}^{\text{DW-SDPO}}}{\sum_{i,t} z_i^{\text{GRPO}} + \sum_{i,t} z_i^{\text{SDPO}}}$$

Denominator가 routed token 수로 normalize하여 ##각 branch가 cover하는 token 비율에 비례하여 기여##. 별도 mixing $\lambda$ 불필요 — sample 비율이 자연스럽게 결정. 초기(failure 많음): SDPO dominant. 후기(success 많음): GRPO dominant.

Construct teacher info {f_i} 5. for i = 1 to G: 6. if y_i is incorrect AND teacher info available: 7. Compute DW-SDPO loss (with entropy weighting) 8. else: 9. Compute GRPO loss 10. Aggregate routed losses, update θ

4. Experiments

4.1 Setup

  • Models: Qwen3-4B, Qwen3-8B (instruct-tuned base)
  • Benchmarks: Chemistry, Physics, Biology, Materials (from SciKnowEval), Tool Use (ToolAlpaca). Train-test split per benchmark.
  • Baselines: (1) GRPO — strengthened with asymmetric clipping, unbiased advantage normalization, off-policy correction. (2) SDPO — self-distillation from feedback-conditioned self-teacher (successful sibling as $f$).
  • Training: Batch 32, 8 rollouts/prompt. GRPO: mini-batch 8, lr $1 \times 10^{-6}$. SDPO: mini-batch 32, lr $1 \times 10^{-5}$. SRPO: mini-batch 32, lr $5 \times 10^{-6}$, $\beta=1$. 8× NVIDIA H20 GPU. Metric: avg@16 accuracy.

Pattern 1: Self-distillation이 효과적인 경우 (Chemistry, Biology): SDPO가 초기에 빠르게 상승하지만 이후 collapse. SRPO는 초기 상승을 track하면서 collapse를 회피하여 최종 성능 최고.

비판적 분석 (Critical Analysis)

강점

  • SDPO instability의 근본 원인 진단: Correct-sample ambiguity와 signal degradation이라는 두 가지 독립적 원인을 식별하고, 각각 routing과 DW로 해결한 설계가 논리적으로 명쾌.
  • 자연스러운 adaptive behavior: Mixing hyperparameter 없이 sample 비율로 자동 적응하는 설계가 우아. 초기 SDPO 비중 높음 → 후기 GRPO 비중 높음이 training dynamics와 자연스럽게 align.
  • Later training에서 compute 절감: 성능 개선과 비용 절감을 동시에 달성하는 드문 사례.

약점 및 한계

  • Working in Progress 상태: 아직 peer review를 거치지 않은 초기 단계 논문.
  • 과학 QA + Tool Use에 국한: Math, code 등 RLVR의 주요 도메인에서의 검증 부재. SDPO의 effectiveness가 domain-dependent이므로 SRPO의 이점도 domain에 따라 다를 수 있음.
  • Teacher information 가용성 가정: SDPO branch가 동작하려면 successful sibling rollout이 존재해야 함. 모든 rollout이 실패하면(매우 어려운 문제) SDPO branch를 활용할 수 없어 순수 GRPO로 fallback.
  • Qwen3 모델에만 검증: 다른 model family(Llama, Mistral 등)에서의 일반화 미확인.
  • Hyperparameter $\beta$의 robustness 미분석: $\beta=1$만 사용했으며 sensitivity analysis 없음.

Key Takeaways

  1. GRPO와 SDPO는 complementary하다: Correct sample에는 coarse reward-aligned reinforcement, incorrect sample에는 dense logit-level correction이 각각 최적. 둘을 ##sample 수준에서 routing##하는 것이 advantage-level mixing보다 robust.
  2. SDPO의 late-stage collapse는 해결 가능한 문제다: Correct-sample ambiguity 제거(routing)와 signal quality modulation(entropy weighting)으로 SDPO의 빠른 초기 수렴을 보존하면서 long-horizon stability 확보 가능.
  3. Entropy-aware dynamic weighting이 late-stage를 살린다: 1h에서 +0.4 → 10h에서 +1.8로 widening. Self-teacher가 noisier해지는 시점에서 high-confidence correction만 남기는 것이 핵심.
  4. 성능 개선과 compute 절감이 동시에 가능: SRPO는 10h에서 GRPO 대비 +3.4% 성능 + 17.2% per-step compute 절감. 학습이 진행될수록 SDPO overhead가 자연스럽게 감소하는 구조적 이점.
