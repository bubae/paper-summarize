VibeThinker-3B

단 30억 파라미터 모델이 1조 파라미터 모델과 수학·코딩에서 어깨를 나란히 한다 ―
"작은 모델은 큰 모델의 대체재가 아니라, 새로운 길이다"

전문가용 상세 요약 보기 →
대형 모델이 모든 것을 외워야 한다는 통념을 깨고, "추론 능력은 작은 모델로도 압축 가능하다"는 새 가설을 실증한 3B 모델

한 컷 요약

VibeThinker-3B는 자기보다 200~333배 큰 flagship 모델들과 수학 reasoning에서 동급 성능을 보인다.

먼저 알아둘 개념들

이 논문이 무슨 의미인지 이해하려면 다음 다섯 가지 배경 지식을 알아야 한다. 각 개념을 하나씩 짚어본다.

① 모델 "파라미터" = AI의 뇌세포 수

AI 언어 모델의 "크기"는 보통 파라미터 수로 표현된다. 파라미터는 모델 내부의 학습 가능한 숫자들의 총 개수로, 인간 뇌의 시냅스 수에 비유할 수 있다. B는 Billion(10억), T는 Trillion(1조)이다.

일반적으로 "파라미터가 많을수록 똑똑하다"는 scaling law가 통용되었다. 그래서 ChatGPT 같은 강력한 reasoning 모델은 보통 100B 이상이며, 최근 모델은 1T(1조)에 달한다.

② "추론(reasoning)"과 "지식(knowledge)"은 다르다

AI가 할 일은 크게 두 가지로 나눌 수 있다:

저자들의 핵심 통찰: "추론"은 작은 모델로 압축 가능하지만, "지식"은 모델 크기와 직결된다.

③ "검증 가능한(verifiable)" 보상이란?

AI를 학습시킬 때, 답이 맞았는지 자동으로 확인할 수 있으면 큰 이점이 있다. 이를 verifiable reward라 한다.

수학과 코드처럼 정답을 명확히 판단할 수 있는 task는 학습 신호가 깨끗해서 RL이 잘 작동한다. VibeThinker-3B는 이 영역에 집중한다.

④ AI는 어떻게 학습하는가: SFT → RL

현대 LLM의 학습은 보통 다음 단계를 거친다:

SFT는 "다양한 풀이 방법을 알려주는 단계"이고, RL은 "직접 시도해보고 정답에 보상받는 단계"이다. VibeThinker-3B는 이 두 단계를 모두 세심하게 설계한다.

⑤ Curriculum Learning: 쉬운 것부터 어려운 것으로

사람이 산수→방정식→미적분 순으로 배우듯, AI도 단계적으로 가르치면 더 잘 배운다.

비유: 의대생 교육
의대생은 1년차에 해부학·생리학(broad coverage)을 배우고, 4년차에 가서야 희귀 질환과 복잡 수술 케이스(hard reasoning)에 집중한다. 1학년부터 희귀 질환만 배우면 기초가 흔들리고, 평생 일반 의학만 배우면 전문성이 안 생긴다. VibeThinker-3B의 2-stage SFT는 이와 같은 발상이다.

VibeThinker-3B의 4단계 학습 파이프라인

저자들은 3B 모델을 거대 모델 수준으로 끌어올리기 위해, 정교한 4단계 후처리(post-training) 파이프라인을 설계했다.

VibeThinker-3B의 전체 학습 파이프라인 — 각 단계가 명확한 역할을 갖는다.

🧩 Stage 1: Supervised Fine-Tuning (모방 학습 단계)

여기서는 모델에게 "이런 식으로 풀어보자"라는 다양한 예시를 충분히 보여준다. 핵심은 단순히 정답을 외우게 하는 게 아니라, "다양한 풀이 방법의 공간"을 넓혀주는 것이다.

비유: 요리학원
좋은 요리학원은 한 가지 김치찌개 레시피만 가르치지 않는다. 신김치 버전, 돼지고기 버전, 참치 버전, 두부 추가 버전 등 같은 문제의 여러 풀이를 알려준다. 그래야 학생이 나중에 재료가 달라도 응용할 수 있다. VibeThinker의 "Diversity-Exploring Distillation"이 바로 이 발상 — 한 문제에 여러 풀이 경로를 모두 학습시킨다.

🎯 Stage 2: Reinforcement Learning (능동 학습 단계)

SFT가 "교과서 따라하기"였다면, RL은 "실제 시험 보면서 점수 받기"다. 모델이 직접 답을 만들어보고, 자동으로 채점받은 결과(reward)로 더 잘 만들도록 학습한다.

비유: 우수 학생 답안의 "간결성 추가 점수"
두 학생이 모두 수학 문제를 맞혔다. 한 명은 풀이가 5장, 다른 한 명은 1장. 정답인 학생들 사이에서만 "더 간결한 풀이"에 추가 점수, "긴 풀이"에 약간 감점을 준다. 오답에는 손대지 않는다. 합산하면 정답률은 그대로지만 모델은 점차 짧고 명료한 풀이를 선호하게 된다.

🔄 Stage 3: Offline Self-Distillation (자기 증류)

Math/Code/STEM 단계가 끝나면, 각 단계가 만들어낸 "좋은 풀이들"을 다시 모아 학생 모델에 재학습시킨다. 마치 한 학기 동안 가장 좋은 노트를 모아 시험 직전에 한꺼번에 복습하는 것과 같다.

📋 Stage 4: Instruct RL (사용자 지시 따르기)

모델이 수학·코드는 잘 풀어도 "200자 이내로, 한국어로, 마크다운 형식으로 써줘" 같은 복잡한 사용자 지시를 어기면 실제 사용성이 떨어진다. 이 단계는 그 부분을 보강한다.

CLR: 시험 보는 새로운 방식

학습이 끝난 모델에게 더 어려운 문제를 풀게 할 때, 그냥 한 번 답하는 게 아니라 여러 번 답하고 검증해서 신뢰성을 높이는 방법이 있다. 이것이 CLR (Claim-Level Reliability Assessment)이다.

비유: 변호사의 "단계별 증거 검증"
변호사가 법정에서 어떤 주장을 할 때, 5단계 논거를 든다고 하자. 단순 평균은 "5단계 중 4단계가 맞으면 80% 신뢰"라고 하지만, CLR은 "한 단계라도 거짓이면 전체가 무너진다"라는 보수적 기준이다. 거듭제곱 5는 그 보수성을 수학적으로 표현한 것이다 — 0.8⁵ = 0.33이라 한 군데라도 어긋나면 신뢰도가 1/3로 떨어진다.

이 방식 덕분에 AIME25 91.4 → 96.7, BruMO25 93.8 → 99.2 같은 추가 향상이 가능하다 (모델 파라미터 변경 없이).

OOD(미공개 문제) 일반화: LeetCode 콘테스트

학습 데이터에 절대 들어갈 수 없는 "방금 출제된 새 문제" 환경에서의 성능. 이는 모델이 단순 암기를 넘어 진짜로 알고리즘을 이해했는지를 보여준다.

핵심 가설: "압축 vs 보관"

저자들은 결과를 단순한 "작은 모델도 잘한다"가 아니라 새 이론 프레임으로 정리한다.

비유: 체스 엔진 vs 위키피디아
Stockfish 같은 체스 엔진은 단지 수십 MB의 코드와 가중치만으로 세계 챔피언을 이긴다. 그러나 위키피디아 전체를 외우려면 수십 GB가 필요하다. 같은 "지능"이라도 종류가 다르다 — VibeThinker-3B는 "수학·코딩 엔진" 쪽에 모든 자원을 투입한 모델이고, "백과사전" 일은 큰 모델에게 맡긴다.

이 가설을 받아들이면 두 가지 결론이 따라온다:

  • 1. 작은 모델은 "compute-saving fallback"이 아니라 독립적인 길이다.
  • 2. 큰 모델은 "지식의 자연스러운 그릇", 작은 모델은 "추론 깊이의 압축기" — 보완 관계.

한 줄 결론

VibeThinker-3B는 "작은 모델은 큰 모델의 대체재"가 아니라, "추론은 압축할 수 있고, 지식은 압축할 수 없다"는 새 가설의 실증적 증거다.

이 모델의 한계

완벽한 모델은 아니다. 솔직하게 한계를 짚어보자.

⚠️ 일반 지식이 약하다

GPQA-Diamond 70.2는 Gemini 3 Pro의 91.9에 21점 뒤진다. 즉 대학원 수준의 일반 과학 지식, 사실 매핑, 광범위한 상식 질의에서는 큰 모델이 여전히 유리하다. "이 약의 부작용은?", "이 역사적 사건의 배경은?" 같은 질문은 VibeThinker-3B에게 어려울 수 있다.

⚠️ Base 모델 의존성

Qwen2.5-Coder-3B라는 이미 코드·수학 특화 base에서 시작했다. 같은 파이프라인이 Llama-3나 일반 base 모델에서도 작동할지는 미검증. 즉 "3B면 다 된다"가 아니라 "좋은 base + 정교한 파이프라인"이 필요.

⚠️ Ablation 부재

2-stage SFT, Long2Short, CLR, Self-Distillation, Instruct RL 중 어느 단계가 결정적인지 분리 실험이 없다. 특히 CLR은 +2~5점의 큰 boost를 주는데, 추론 시 비용(추정 1280배 추가 inference)에 대한 분석도 부족.

⚠️ Multi-turn / Agent 미평가

모든 벤치마크가 단일 응답 기반. 다중 대화, 도구 사용, 장기 agent 작업에서의 성능은 평가되지 않아 실제 챗봇/agent 응용 범위는 불명.

⚠️ CLR 비교의 공정성

CLR은 VibeThinker에만 적용된 test-time scaling. 다른 모델에도 같은 기법을 적용하면 얼마나 더 향상될지는 미평가. "+CLR이 적용된 VibeThinker가 적용 안 된 GLM-5와 동급"이라는 비교는 약간의 불균형이 있다.

왜 중요한가

🌍 AI 접근성 확대

지금까지 reasoning 능력은 OpenAI, Google, Anthropic 같은 대형 사업자의 API를 통해서만 접근 가능했다. VibeThinker-3B는 중소기업·대학·개인 개발자도 자기 GPU에서 직접 frontier reasoning을 실행할 수 있게 만든다.

💸 비용 구조의 근본적 변화

1T 모델 한 번 호출 ≈ 3B 모델 수천 번 호출. 수학·코드처럼 verifiable한 영역에서 "내장 처리(on-device)"로 처리 가능해진다. 모바일 디바이스의 수학 튜터, 로컬 코드 보조 등이 현실적이 된다.

🔬 연구 방향성 전환

"무조건 더 크게(scaling law)"라는 일방향에서, "reasoning은 작게, knowledge는 크게"의 이원적 연구 방향을 제시. 작은 reasoning 코어 + 큰 knowledge retriever (RAG)의 결합 아키텍처가 차세대 시스템의 표준이 될 가능성.
