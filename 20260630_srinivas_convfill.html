TL;DR

  • 문제: 음성 conversational agent는 ##millisecond-level 응답성##이 필요하지만, frontier model의 reasoning/tool-call/retrieval은 ##수 초 단위 latency##를 동반한다. 작은 on-device model로 latency는 잡되 ##복잡 task의 정확도가 떨어지는 deadlock##.
  • 해결: ##Conversational Infill## task — 작은 Talker(SLM)가 즉시 ##contextually grounded filler response##를 생성해 외부 Reasoner(frontier LLM)의 latency를 hide하고, Reasoner의 streamed knowledge가 도착하면 ##fluently incorporate##. 두 모델이 ##concurrently## 작동.
  • 왜 작동하는가: Talker는 user query 위에서 conversation을 ##유지##하기만 하면 되므로 ##135M-1.7B의 작은 모델##로 실시간 동작 가능. Reasoner는 ##비동기##로 답을 만들어 knowledge chunk로 streaming. Talker는 ##special token <sil>로 silence 구간을 인지##하고 적절히 filler 또는 knowledge-grounded response를 생성. ConvFill dataset (290,571 examples)으로 학습 가능.
  • 비유: ##토크쇼 진행자(Talker)와 백스테이지 자료 조사 팀(Reasoner)##. 진행자는 시청자와의 대화 흐름을 유지하면서, 조사팀이 노트를 건네면 자연스럽게 답변에 녹여낸다. 시청자는 진행자만 보지만 답변의 정확성은 조사팀의 능력에 가깝다.

핵심 기여 (Key Contributions)

  1. Conversational Infill task 정식화: lightweight Talker SLM + powerful Reasoner의 ##두 모델 collaboration##을 ##inference-time## 작업으로 formalize. Christakopoulou et al. (2024)의 Talker-Reasoner 개념을 ##구체적 system + dataset + training pipeline##으로 발전.
  2. ConvFill Dataset (290,571 training examples): 6개 도메인 (general advice, assistant query, event planning, customer service, education, medicine + DSTC8 task-oriented dialogues). ##2-part 4-stage validator##로 hallucination 검출. 111,552 silence-based filler + 179,019 knowledge-grounded examples.
  3. 7개 SLM 학습 가능성 입증: Qwen3 0.6B, Llama 3.2 1B, Gemma 3 270M/1B, SmolLM2 135M/360M/1.7B (135M-1.7B 범위). 모든 family에서 ConvFill task가 learnable.
  4. Latency-capability Pareto frontier 이동: ##Talker model이 frontier accuracy의 6.3% 이내 (single-turn)##까지 회복. millisecond-level TTFR(Time-to-First-Response)을 유지하면서 Reasoner의 high-latency 능력을 leverage.
  5. Live user study (n=18, Apple M2 SoC): ConvFill이 frontier model과 ##overall on-par 평가## + ##significantly better perceived latency##. ##retrieval-heavy task에서 ConvFill 선호도가 가장 높음## (RAG: 12/18 rank-1 vs Frontier 5/18).

배경 지식

음성 Conversational Voice Agent란?

Siri, Alexa, ChatGPT Voice 같은 음성 비서. text-based task-oriented system (customer service, virtual assistant, tutoring 등)이 점점 더 capable해지지만, ##음성 기반 모델은 text-only에 비해 일관되게 뒤처짐## (Lin et al. 2025).

왜 voice agent가 어려운가

본 논문은 fundamental tension을 강조:

  • capability를 키우는 기법 — multi-step reasoning, tool call, retrieval — 은 ##inherently iterative and time-consuming##
  • 음성 상호작용 — seamless user experience를 위해 ##milliseconds## 단위 응답이 필요 (Jacoby et al. 2024)

"advanced capability는 답의 질을 높이지만 음성 interactive에 필요한 responsiveness를 erode한다" (Lin et al. 2026).

기존 voice 시스템의 분류

  • Cascaded: ASR → text LLM → TTS. latency가 누적되고 ##conversational flow를 break##.
  • Full-duplex speech models (Défossez et al. 2024 — Moshi 류, Iribe et al. 2024 — Gao et al. 2025): audio를 직접 생성해 ASR/TTS 우회. latency 감소하지만 ##single LLM backbone에 의존##해 flexibility 희생, real-time 제약이 모델 size를 제한 (Wang et al. 2025a).

Filler 처리 방식 비교

침묵을 다루는 기존 접근:

  1. Fixed filler phrase: "I'm thinking..." 같은 generic phrase 반복.
  2. Heuristic selection: context 기반 다른 filler set 선택 (Mahmood et al. 2025, Gao et al. 2025).
  3. Dynamic infill with multiple models: lower-latency model이 incorrect info를 주면 higher-capability가 correction. 그러나 ##repeated revision은 user trust/confidence/engagement에 harm## (Lin et al. 2025).

ConvFill의 차별점: ##user는 Talker하고만 interact하며, Talker가 외부 knowledge를 verbatim relay하지 않고 internally incorporate##. → multi-turn 전체에 걸친 ##일관된 voice##.

방법론 1 — Task Formulation

3.1.1 Talker Interface

$\tau$를 turn index, $u_\tau$를 turn $\tau$의 user utterance라 하자. Talker의 full response는 phrase sequence $T_\tau = (t_1, t_2, \ldots, t_p)_\tau$. lookback $\ell$로 conversation history:

$$h_\tau(\ell) = (u_{\tau-\ell} \oplus T_{\tau-\ell}, \ldots, u_{\tau-1} \oplus T_{\tau-1})$$

$\oplus$는 concatenation. turn $\tau$ 내에서 phrase index $i \in [1, \ldots, p]$.

3.1.2 Reasoner Interface

Reasoner는 asynchronous하게 external knowledge stream을 생성:

$$E_\tau = (e_1, e_2, \ldots, e_p)_\tau$$

$e_i$는 ##knowledge chunk $r_i$ 또는 silence token $s_i$##. $r_i$는 reasoning/retrieval/inference/tool-call의 결과를 담은 semantically complete text phrase. ##Reasoner가 predetermined time 안에 chunk를 보내지 않으면 silence token $s_i$가 동적으로 삽입##.

3.1.3 Inference-Time Collaboration (식)

Talker와 Reasoner는 ##concurrently## 작동. position $i$에서 Talker가 next element $e_i$를 observe:

$$\begin{cases} h_\tau(\ell) \oplus [u_\tau \oplus (t_1, \ldots, t_{i-1})_\tau] \oplus (r_i)_\tau \to t_i & \text{if } e_i = r_i \\ h_\tau(\ell) \oplus [u_\tau \oplus (t_1, \ldots, t_{i-1})_\tau] \to t_i & \text{if } e_i = s_i \end{cases}$$

$e_i = s_i$와 $e_i = r_i$는 ##동시에 발생할 수 없음##. constraint: $|T_\tau| = |E_\tau|$.

직관적으로 말하면, Reasoner가 ##knowledge $r_i$를 push하면## Talker는 그것을 conditioning 정보로 받아 ##grounded response $t_i$를 생성##하고, Reasoner가 ##침묵 $s_i$를 emit하면## Talker는 ##contextually contingent filler phrase##만 생성. 이렇게 Reasoner의 variable latency를 Talker가 stall 없이 absorb.

방법론 2 — ConvFill Dataset

Dataset 구성 (Composition)

  • ##8,443 conversation##
  • ##290,571 training examples##: 111,552 silence-based filler + 179,019 knowledge-grounded
  • JSONL format, 전체 영어
  • API cost: Claude Opus 4.6으로 ~$2,400 (validation retry 포함)

Generation Pipeline

Conversation은 JSON object로 구성된 turn array. 각 turn은:

  • user utterance
  • thoughts array: 0-3 <sil> silence placeholder + Reasoner phrase에 해당하는 statement
  • response array: Talker phrase. <sil>에 해당하는 위치는 conversationally contingent filler, Reasoner phrase 위치는 그 phrase를 prior conversation과 일관된 style/form으로 incorporate한 entry

Generation Source 두 부류

  1. DSTC8 task-oriented dialogues (Rastogi et al. 2020): labeled user utterance 유지, labeled system response를 Reasoner thoughts로 사용. <sil> placeholder를 turn 시작에 padding.
  2. Random-seeded topic corpus: 6 domain (general advice, assistant queries, event planning, customer service, education, medicine) × 1,000 unique conversation premise seed. 7-10 turn template with 0-3 <sil> placeholders/turn. 모든 세 stream (user, thoughts, response)을 LLM이 동시에 생성.

4-Stage Validator

Claude Opus 4.6 같은 frontier model조차 ##response phrase가 thoughts array contents를 subtly reference하는 hallucination##을 만든다. 4-stage validator로 iterative re-generation:

  1. Structural parsing: turn 수, matching Talker-Reasoner phrase pair
  2. BERTScore ordering check: thoughts와 response의 ##temporally aligned pair##가 가장 높은 BERTScore를 가져야 함 (ordering error 검출)
  3. DeBERTaV3 NLI entailment: MNLI fine-tuned, 각 grounded phrase pair가 entailment 관계인지
  4. Proper noun verification: response field의 proper noun이 user/thoughts에 ##이미 등장했는지## (entity hallucination 방지)

방법론 3 — Infill Model Training (Section 5)

Training Format (Figure 3)

대부분 pretrained SLM은 user role ([USER]), assistant role ([ASST]), end-of-turn ([END]) control sequence를 가진다. ConvFill은 ##new control sequence [KNOWLEDGE]##를 추가하고 각 모델의 existing control-token template으로 mapping. 또한 ##new special token <sil>##을 추가해 Reasoner knowledge가 아직 available하지 않은 case를 표시.

training example 구조 (lookback $\ell = 1$ 예시):

[USER] (history) user utterance τ-1 [END]
[ASST] (history) assistant utterance τ-1 [END]
[USER] user utterance τ [END]
[KNOWLEDGE] external knowledge chunk (e_i) [END]
[ASST] t_1, ..., t_{i-1}, t_i [END]

모델은 ##red text $t_i$를 predict##하도록 fine-tune. 모든 학습은 ##text only## (no audio), 따라서 full-duplex 접근의 audio modeling resource를 회피.

Inference Pipeline (Figure 2)

두 개의 queue:

  • Streamed Knowledge Queue: Reasoner의 knowledge chunk buffer
  • TTS Queue: Talker의 response phrase buffer (TTS로 voice 출력)

Talker는 greedy inference로 available knowledge chunk를 ingest하면서 TTS queue로 push. ##knowledge queue가 empty이고 TTS queue도 empty##면 ConvFill이 ##<sil> token을 동적으로 emit##하여 Talker가 conversational filler phrase를 생성. ##<sil>은 turn 시작에서 Reasoner를 기다릴 때 가장 빈번##.

비판적 분석

강점

  • Architectural elegance: Talker-Reasoner decoupling은 ##system 설계상 modularity##. Reasoner를 freely swap 가능하면서 Talker는 retained → frontier model 변경 cost를 줄임. token cost가 변하는 시대에 큰 장점.
  • 실증적 사용자 평가: 18 participant live study + 144 interaction + 635 turn. ##actual hardware (Apple M2) + actual STT/TTS pipeline##에서 측정 — synthetic 평가의 한계를 극복.
  • Generalizability across model families: 7개 모델 4개 family에서 모두 task가 learnable. ##model family-specific hack이 아닌 일반 framework##.
  • Affordable training: 7 모델 학습에 ##총 $133.70 + 데이터 생성 $2,400##. 단일 RTX 6000으로 2.8-48.9 GPU 시간. ##연구 진입 장벽이 낮음##.
  • 4-stage validator의 정직성: Claude Opus 4.6조차 hallucination을 만든다는 발견을 정직히 보고. dataset 품질에 대한 transparency.
  • 완전 오픈소스: dataset, generation pipeline, training/inference code, 7 model weights, demo software 모두 공개.

약점·의문

  • 영어 단일: 저자도 limitation으로 인정. ##multilingual 확장은 future work##. 한국어/중국어 등에서 동일 방법이 통할지 불명.
  • Safety coupling 미흡 (저자 인정): Talker는 independently 작동하므로 ##filler가 Reasoner의 safety check 전에 emit될 수 있음##. 만약 Reasoner가 unsafe query에 refuse한다 해도 Talker가 이미 user에 "I'll look that up..."류 응답을 emit한 상태. ##safety-critical 배포에서 위험##.
  • Reasoner timeout / excessive latency case 미평가: 평가는 Reasoner가 ##eventually responds##하는 경우만. timeout이나 5초 이상 지연되면 ##prolonged filler가 disruptive##해질 가능성. real-world deployment에서 edge case.
  • Naturalness 약점: ConvFill의 Naturalness rating (3.83)이 Frontier (4.41)보다 낮음. ##filler phrase에 익숙하지 않은 user들에게는 어색함##. cultural acceptability 차이도 있을 수 있음 (예: 한국어 사용자는 filler "음...", "그러게요"에 다르게 반응).
  • Conversational Infill을 단순 "filler text 생성"으로 trivialize할 위험: 더 깊이 들어가면 ##context-aware filler 선택## (예: medical question vs casual chat의 filler 차이)이 필요한데, 일반화된 filler 생성으로는 ##domain-appropriate## tone을 맞출 수 없음.
  • $\tau^2$-Bench-level evaluation 부재: SimpleQA, LLAMA1, MultiWOZ는 standard benchmark지만 ##agent-style task ($\tau^2$, VitaBench 등)##나 ##long-horizon scenario##에서의 평가가 없음. real-world voice agent의 복잡도를 fully capture하지 못함.
  • Reasoner의 output format dependency: Gemini 3.1 Pro가 SimpleQA에서 9.8% degradation을 보인 것은 ##ConvFill의 prompting format에 sensitive##하기 때문. 일반화된 framework로 자리잡으려면 ##diverse Reasoner output format에 robust##해야 함.
  • Single-turn LLAMA1 saturation: 대부분 Talker가 95%+ accuracy → ##scaling trend가 잘 안 보임##. 더 challenging benchmark에서의 검증이 더 필요.
  • Filler-Reasoner 연결의 비가시성: 학습 데이터가 ##LLM-generated##라 실제 사용자 traffic의 filler-knowledge 패턴과 다를 수 있음. ##Real user filler usage##를 capture한 corpus가 있다면 더 자연스러울 가능성.

한계의 자체 인정 (Section "Limitations")

  • Talker가 Reasoner 응답 전에 emit하는 filler는 ##safety constraint를 즉시 따르지 않을 수 있음##. Future work: tighter safety coupling.
  • Reasoner timeout / excessive latency case 평가 부재. prolonged filler가 disruptive할 수 있음. ##inter-filler pause/fallback strategy 필요##.
  • User study sample 작음 (n=18). larger eval로 robust 검증 필요.
  • English-only 평가.

Key Takeaways (이 논문에서 가져갈 것)

  1. Voice agent의 fundamental tension은 architectural decoupling으로 해결 가능: 한 모델이 ##모든 일을 다 하는 대신##, 작은 Talker가 conversational flow를 유지하고 큰 Reasoner가 capability를 제공. ##frontier 6.3% 이내##까지 follow하면서 ##millisecond TTFR## 달성.
  2. Conversational Infill은 단순 filler 생성이 아니다: <sil>-conditioned filler와 ##knowledge-grounded incorporation##을 한 모델이 모두 학습해야 함. 4-stage validator로 hallucination 검출이 ##학습 데이터 품질의 핵심##.
  3. Reasoner-agnostic framework: Claude, GPT, Gemini 등 어떤 frontier model이 와도 same Talker re-use 가능. token cost·model performance가 빠르게 변하는 시대에 ##interaction consistency를 유지##하는 실용적 가치.
  4. 4 model family × 7 size 학습 가능성: Gemma, Qwen, Llama, SmolLM 모두 가능. ##135M~1.7B##의 wide range로 ##smartphone deployment##까지 가능. INT8 quantization으로 136 MB-1.69 GB.
  5. RAG/tool 시나리오에서 user preference가 가장 강함: ConvFill이 RAG에서 12/18 rank-1 (Frontier 5/18). ##long-latency task일수록 ConvFill의 가치가 커진다##. SLM 단독은 RAG에서 3.8초 TTFR — 즉, ##Talker offloading의 본질적 효과##.
  6. Naturalness vs Responsiveness tradeoff: filler를 모든 silence에 적용하면 over-fillery 보일 수 있음. ##latency-adaptive behavior## (Reasoner의 예상 지연에 따라 filler 양 조절)이 future direction.
  7. Decoupled Architecture의 일반화 가능성: Voice 외에도 ##low-latency interactive UI##(코딩 보조, 실시간 chat UI 등)에 적용 가능. "fast-response surface" + "slow-think backend"는 일반 패턴.
  8. $133.70 + $2,400으로 7개 모델 학습: ##연구 진입 장벽 매우 낮음##. 다른 도메인 적용에 부담 없음. 오픈소스 dataset/code가 reproduction 용이.
