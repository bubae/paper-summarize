EAGLE-3: Scaling up Inference Acceleration of LLMs via Training-Time Test

TL;DR

  • 문제: EAGLE/EAGLE-2는 ##draft model이 target의 top-layer feature를 재현(regress)하도록 강제##되어 (1) feature-level autoregression의 표현력이 token loss 외 feature loss에 의해 묶이고, (2) training data를 늘려도 성능이 saturate된다. 즉 LLM 학습 데이터 scaling law의 혜택을 받지 못한다.
  • 해결: ##두 가지 변화##. (1) Feature regression loss($l_{\text{fea}}$) 제거 → draft model이 token 예측에만 집중. (2) top-layer feature 대신 ##low/mid/high level feature를 fusion##해 풍부한 multi-layer context로 conditioning. 추가로 (3) ##training-time test##: 학습 중에 draft model의 예측을 다음 step의 input으로 feeding하여 multi-step inference 분포 mismatch를 학습.
  • 왜 작동하는가: feature constraint가 사라지면 model capacity가 모두 token 예측에 쓰이고, multi-layer fusion은 단일 layer 대비 더 풍부한 정보 제공. training-time test는 inference 시의 ##누적 오차## 분포를 학습 시 직접 노출시켜 robust해진다.
  • 비유: EAGLE이 "모범 답안을 똑같이 베끼며 푸는 학생"이라면, EAGLE-3는 "##답안 형식 무시하고 정답만 맞히는 학생##". 거기에 ##모의고사를 실제 시험처럼 풀게 만들어 시험불안을 미리 학습##시킨 학생.

핵심 기여 (Key Contributions)

  1. Training-time test 아키텍처: 학습 시 draft model의 multi-step 예측을 simulate하여, inference에서 발생하는 distribution shift를 학습 단계에 통합. ##feature regression loss를 제거##하고 token prediction에만 집중.
  2. Multi-layer feature fusion: top-layer feature 대신 low/mid/high feature를 concat → FC layer로 fuse한 $g$ vector를 draft input으로 사용. ##단일 layer 정보의 한계 극복##.
  3. Inference acceleration scaling law 발견: ##새 아키텍처는 training data가 8배 증가할 때 speedup이 비례 증가##(MT-bench LLaMA-3.1-8B 기준 3.6×→4.4×). EAGLE은 ~3.2×에서 정체. LLM 학습 데이터 scaling law가 speculative decoding에도 적용됨을 입증.
  4. 실증: 5개 task × 5개 모델에서 최대 ##6.5× speedup, EAGLE-2 대비 1.4× 개선##. Vicuna-13B HumanEval 6.47×. SGLang batch 64에서 38% throughput 증가.

1. 배경 (Preliminaries)

1.1 Speculative sampling 복습

Speculative sampling (Leviathan et al. 2023, Chen et al. 2023): draft model이 $k$ token $\hat{T}_{j+1:j+k}$를 autoregressive하게 생성하고 probability $\hat{p}$ 기록. Target model이 한 번의 forward pass로 검증, token $\hat{t}_{j+i}$의 accept prob = $\min(1, p_{j+i}(\hat{t}_{j+i})/\hat{p}_{j+i}(\hat{t}_{j+i}))$. Reject 시 $\text{norm}(\max(0, p - \hat{p}))$ 분포에서 resample하고 그 뒤 draft는 폐기.

핵심 metric:

  • Speedup ratio: vanilla AR 대비 wall-clock 속도 비율
  • Acceptance length $\tau$: drafting-verification cycle당 평균 accept된 token 수
  • Acceptance rate $n$-$\alpha$: ##앞 $n$개 token이 모두 accept된 조건에서 다음 token의 accept 확률##. EAGLE처럼 multi-step에서 input에 자기 예측을 쓰는 모델은 이 metric이 핵심.

1.2 EAGLE / EAGLE-2 복습 (이 논문의 출발점)

EAGLE의 아이디어: draft model이 vanilla speculative sampling처럼 작은 LLM을 새로 학습하는 게 아니라, ##target model의 top-layer feature를 autoregressive하게 예측##. 그 후 target의 LM head로 token으로 변환.

EAGLE의 loss:

$$\mathcal{L} = l_{\text{fea}} + l_{\text{token}}$$

여기서 $l_{\text{fea}}$는 draft feature와 target top-layer feature의 regression loss, $l_{\text{token}}$은 token CE loss. EAGLE-2는 dynamic draft tree (acceptance rate 기반 pruning) 추가.

1.3 EAGLE의 두 가지 한계

(a) Feature regression constraint의 한계

저자들의 분석: $l_{\text{fea}}$는 draft가 target top-layer feature와 일치하도록 강제한다. 이는 ##draft model의 표현력을 제한##한다. Token prediction을 ultimate goal로 보면, feature regression은 부가 제약(over-constraint)일 뿐.

실험적 증거 (Figure 3, 4): $l_{\text{fea}}$를 제거하면 1-$\alpha$ (step 1)가 향상되지만, draft output $\hat{a}_{t+1}$이 ground-truth $f_{t+1}$에서 멀어져 step 2부터 input이 training distribution을 벗어남 → 2-$\alpha$가 폭락. 단순 제거만으론 안 됨.

(b) Top-layer feature의 한계

LM head는 full-rank이므로 top-layer feature의 정보는 ##next token logit에 unique하게 매핑##된다. 이걸 그대로 쓰면 "다다음 토큰"을 예측하는 데 정보가 부족하다. ##Intermediate (low/mid) layer는 더 일반적 표현##을 담고 있어 multi-step prediction에 유리.

2. 방법 (Method): EAGLE-3

2.1 Training-time test (핵심 아키텍처)

[Figure 3 설명]: 위쪽은 EAGLE — Training 시 feature loss $l_{\text{fea}}$를 사용. Test 시 draft가 자신의 예측 $\hat{f}$를 다음 step input으로 사용. Training/Test 분포 mismatch. 가운데는 EAGLE + $l_{\text{fea}}$ removal — feature loss는 제거했지만 training은 여전히 ground-truth feature만 사용. Test에서 draft 예측 $\hat{a}$이 ground-truth와 멀어져 step 2에서 입력이 distribution 밖. 아래쪽은 EAGLE-3 — ##Training 시 draft model의 예측을 다음 step의 input으로 직접 사용##해서 inference 분포를 학습 단계에 simulate. Token prediction loss만 사용.

구체적으로, training-time test에서 ##unconstrained vector $a$##(EAGLE의 $f$ 대체)를 draft model의 출력으로 정의하고, 이를 다음 step input에 feed back. 이 과정을 학습 중에 simulate.

이렇게 하면 EAGLE-3의 input이 training 시에도 test 시처럼 ##draft 자신의 출력##이 되므로 distribution mismatch가 사라진다. Feature regression loss 없이 token CE loss만 사용.

2.2 Multi-layer Feature Fusion

[Figure 5 설명]: Target model이 prefix "How can"을 처리한 뒤 next token "I"를 생성. 이때 ##low feature $l$, mid feature $m$, high feature $h$##를 모두 추출. 각각이 $k$-dim. Concat → FC layer로 $3k \to k$ projection → fused feature $g$. 그 다음 draft model에서 token embedding $e$와 $g$를 concat하여 input.

구체적 inference pipeline:

  • Step ①: Prefill에서 "I"가 생성되면, $g_{\text{how}}, g_{\text{can}}$ + embedding $e_{\text{I}}$로 draft에 입력 → output $a_{\text{I}}$ → LM head → sample "do".
  • Step ②: $g$를 재사용하려면 "I"에 대한 target의 $g_{\text{I}}$가 필요하지만, "I"는 아직 target에서 verify되지 않음. 대신 draft의 자기 출력 $a_{\text{I}}$를 사용. 이를 다음 token embedding $e_{\text{do}}$와 concat → draft → $a_{\text{do}}$ → "are".
  • Step ③: 마찬가지로 $a_{\text{do}}$를 사용.

즉 draft의 multi-step prediction에서 $g$를 새로 못 받는 상황을 ##$a$로 대체##해야 하고, 이 대체된 $a$의 분포를 학습 시 노출하는 것이 training-time test.

2.3 Attention Mask 설계 (Figure 6)

[Figure 6 설명]: 학습 시 prefix가 "How can I"이면 (1) Step 1: 표준 causal triangular mask로 $a_{\text{how}}, a_{\text{can}}, a_{\text{I}}$ 출력. (2) Step 2 (simulated): input이 "How can I are we do"로 확장. 여기서 "are"의 query는 "How", "can", "I"의 key만 보고, "we"는 "How", "can", "I", "are"의 key, "do"는 "How", "can", "I", "we"... 형태로 ##diagonal-only한 sparse attention##. (3) Step 3: 또 한 step 확장. 모든 attention mask가 diagonal-only이므로 matrix mult 대신 ##vector dot product로만 계산##해 efficient.

2.4 Training algorithm 흐름

[Training Step] 1. Target model이 prefix "How can I" 전체 forward → l, m, h (low/mid/high) extracted → g = FC([l; m; h]) 2. Step 1 (native): draft input = [g_t ⊕ e_{t+1}] → predict a_{t+1}, loss_token_1 3. Step 2 (simulated): draft input = [a_{t+1} ⊕ e_{t+2}] → predict a_{t+2}, loss_token_2 4. Step 3 (simulated): draft input = [a_{t+2} ⊕ e_{t+3}] → predict a_{t+3}, loss_token_3 5. Total loss = sum of token CE losses (no l_fea) 6. Attention mask: diagonal-only for steps 2, 3

3. 실험 (Experiments)

3.1 셋업

  • Target models: Vicuna-13B, LLaMA-3.1-Instruct 8B, LLaMA-3.3-Instruct 70B, DeepSeek-R1-Distill-LLaMA 8B
  • Tasks: MT-bench (multi-turn chat), HumanEval (code), GSM8K (math), Alpaca (instruction following), CNN/DM (summarization)
  • Training data: ShareGPT + UltraChat-200K (~68K + 464K). DeepSeek-R1는 추가로 OpenThoughts-114k-math.
  • Training: AdamW, (β1, β2)=(0.9, 0.95), grad clip 0.5, lr=5e-5. Target model이 직접 응답 생성한 데이터 사용.
  • Baselines: SpS, PLD, Medusa, Lookahead, Hydra, EAGLE, EAGLE-2, HASS
  • Metrics: speedup, $\tau$, $n$-$\alpha$

3.3 Scaling Law 발견 (Figure 1)

[Figure 1 설명]: x-축이 ShareGPT 대비 데이터 양 비율(1, 2, 4, 8). y-축이 (a) Speedup ratio, (b) Acceptance length τ. EAGLE-2는 데이터 8배에도 거의 평평한 곡선 (speedup ~3.2×, τ ~4.1). ##EAGLE-3는 단조 증가하는 곡선##(speedup 3.6×→4.4×, τ 5.2→6.1).

이 실험의 의도: training data 증가가 speculative decoding 가속에 미치는 영향. 즉 ##LLM 학습 scaling law가 draft 모델에도 적용되는가##?

결과: EAGLE은 feature regression constraint 때문에 학습 데이터를 더 많이 줘도 새로 배울 게 제한적. EAGLE-3는 제약이 풀려 ##데이터 양에 비례한 가속##. 이는 ##추론 가속의 새 scaling law##.

3.4 Acceptance rate 분포 (Figure 7)

[Figure 7 설명]: MT-bench LLaMA-3.1-8B에서 EAGLE은 step이 진행될수록 acceptance rate가 0.72→0.50로 ##대폭 감소##. EAGLE-3는 ~0.78 수준으로 ##거의 일정##. training-time test가 누적 오차를 완화한 직접 증거.

4. 종합 평가

강점

  • 문제 정의의 깊이: EAGLE의 두 한계(feature constraint, top-layer 한계)를 명확히 분리하고, ##feature constraint 단순 제거는 distribution mismatch로 실패##함을 실증 후 training-time test로 해결. logical step이 깔끔.
  • Scaling law 발견: ##추론 가속에 scaling law가 적용된다##는 사실은 매우 fundamental한 발견. 단일 시점 SOTA 달성을 넘어 future scaling의 방향성 제시.
  • Multi-model 검증: 13B (chat), 8B (instruct), 70B (large), R1 (reasoning) 네 가지 모델에서 일관 향상. ##모델 카테고리 cross-cut##.
  • Production framework 통합: SGLang/vLLM에서 batch 64까지 가속 유지. EAGLE이 high-batch에서 실패하는 양상을 정직히 보고하고, EAGLE-3가 그를 해결.
  • Ablation의 명료성: Table 2가 단 3행으로 두 contribution의 marginal 효과를 깔끔히 분리.
  • Implementation 효율: training-time test의 attention mask가 diagonal-only이므로 ##matrix multiplication 대신 vector dot product로 최적화##. 학습 비용 control.

한계 및 의문점

  • Multi-step의 수: 본문에서 "Step 2, 3"까지만 표시. ##training-time에서 simulate하는 step 수가 inference 시 draft 트리 깊이와 어떻게 관련##되는지 디테일 부족.
  • Lossless 보장의 검증 부재: speculative sampling은 이론적으로 lossless이나, EAGLE-3가 unconstrained $a$를 쓰므로 ##numerical drift가 sampling에 미치는 영향## 검증 unclear.
  • $l_{\text{fea}}$ 제거의 generality: HASS, Falcon 등 다른 feature-prediction 기반 방법에도 같은 변경이 효과적일까? 본문은 EAGLE-3 specific하게만 보고.
  • Training cost 보고 없음: training-time test는 다중 step simulate으로 학습 시간 증가 가능. ##GPU hour나 wall-clock training time##이 명시 안 됨.
  • R1 (reasoning) 결과의 차이: DSL 8B GSM8K 5.01×로 다른 모델 대비 유난히 높음. 이게 OpenThoughts-114k-math 추가 데이터 때문인지 R1 architectural 특성인지 불확실.
  • EAGLE-3 attention mask와 KV cache 호환성: production framework integration 시 attention mask 변경이 KV cache 관리에 미치는 영향을 SGLang팀이 별도 검증했다고만 언급. ##구현 세부의 black box##.
  • Feature fusion FC layer의 input dim 폭증: target model의 $k$가 LLaMA-3 8B면 4096. low/mid/high concat 시 12288 → 4096 FC. parameter는 5M+. 이 부분이 draft model 크기에 미치는 영향 정량화 부재.
  • 새 scaling law의 saturation point 미탐색: 8× data까지만 보고. 더 늘리면 어디서 정체될까?

5. Key Takeaways

  1. "Feature regression이 token prediction의 적이다": 표면적으로 feature loss가 도움 될 것 같지만, ##over-constraint로 token capacity를 갉아먹는다##. 다른 distillation 방식에도 적용 가능한 일반 원리.
  2. Training-time test = "exposure bias 해결의 일반 패턴": NMT의 scheduled sampling, RNN의 teacher forcing 문제와 유사. ##학습-추론 mismatch 해결의 보편 원리##를 speculative decoding에 적용한 사례.
  3. Multi-layer feature는 더 풍부하다: top-layer만 쓰는 다양한 distillation/probe 연구에 시사. ##low/mid/high concat + FC##는 minimal cost로 정보 풍부화.
  4. Inference acceleration scaling law의 존재: data를 늘리면 가속이 늘어난다는 사실은 ##LLM 학습 scaling law를 speedup에 그대로 transfer## 가능함을 시사. EAGLE-3 후속 연구(DFlash, DFLARE)의 토대.
  5. Production 채택의 강력한 후보 (이미 채택됨): ##SGLang/vLLM 통합 완료##, EAGLE-2 대체. 모든 후속 speculative decoding 연구의 baseline.
