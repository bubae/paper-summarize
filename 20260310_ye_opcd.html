TL;DR

  • 문제: LLM의 in-context knowledge (경험적 지식, system prompt 등)는 일시적이어서 세션이 끝나면 사라지고, 매번 context로 다시 제공해야 한다. 기존 context distillation은 off-policy + forward KL을 사용하여 exposure bias와 mode-covering 문제를 겪는다
  • 해결: On-Policy Context Distillation (OPCD) — student가 context 없이 자신의 trajectory를 생성하고, context-conditioned teacher와의 reverse KL divergence를 최소화하여 context 지식을 파라미터에 내재화
  • 왜 작동하는가: On-policy sampling이 학습/추론 분포 간 gap (exposure bias)을 해소하고, reverse KL이 teacher의 high-probability 영역에 집중하는 mode-seeking behavior를 유도하여 student가 teacher의 핵심 행동을 더 정확히 모방
  • 비유: 기존 방법이 "교과서(teacher 생성 데이터)를 달달 외우는 것"이라면, OPCD는 "자기가 직접 문제를 풀어보고(on-policy), 선생님의 피드백(reverse KL)을 받아 교정하는 것"과 유사

핵심 기여 (Key Contributions)

  1. OPCD 프레임워크: On-policy sampling + reverse KL divergence를 context distillation에 적용하여, student가 자신의 generation trajectory 위에서 context-aware teacher의 분포를 학습하는 방법론 제안
  2. Experiential knowledge distillation: 모델이 과거 문제 풀이에서 추출한 경험적 지식을 누적하고, OPCD로 파라미터에 내재화하는 새로운 응용 시나리오를 도입. ##OPCD가 In-Context보다 높은 정확도를 달성하면서 OOD 성능도 더 잘 보존##
  3. System prompt distillation: 의료/안전 도메인의 최적화된 system prompt를 모델에 내재화하여, 추론 시 prompt 없이도 동일한 행동을 재현
  4. Cross-size distillation: 큰 teacher의 경험적 지식을 작은 student에 OPCD로 전달할 수 있지만, 동일 지식을 작은 모델의 context에 직접 주입하면 오히려 성능이 하락한다는 흥미로운 발견

1. Introduction — 일시적 지식의 영구화 문제

배경: In-context learning의 한계

LLM은 프롬프트에 정보를 추가하는 것만으로 행동을 조정할 수 있는 in-context learning 능력을 보여준다. 그러나 이 지식은 일시적(transient)이다 — 세션이 리셋되면 사라지고, 매번 프롬프트에 다시 제공해야 한다. 자연스러운 질문이 떠오른다: 일시적 in-context knowledge를 모델의 영구적 파라미터에 내재화할 수 있는가?

기존 Context Distillation의 문제

Context distillation은 context-conditioned teacher의 행동을 student가 모방하도록 학습하여 context를 weight에 압축하는 방법이다. 그러나 기존 방법은 off-policy + forward KL을 사용한다:

  • Exposure bias: Student가 teacher 생성 데이터로 학습하지만, 추론 시에는 자신의 autoregressive 출력을 소비해야 한다. 학습과 추론의 분포 불일치가 발생
  • Mode-covering (forward KL의 문제): Forward KL은 teacher의 모든 토큰에 확률을 배분하도록 강제하여, student가 teacher 분포를 정확히 모델링할 capacity가 부족할 때 과도하게 넓은(hallucination-prone) 분포를 학습

OPCD의 핵심 아이디어

OPCD는 두 가지 설계 선택으로 이 문제를 해결한다:

  1. On-policy sampling: Student가 context 없이 자신의 응답을 생성 → 학습/추론 분포 일치
  2. Reverse KL divergence: Student의 생성에서 teacher가 높은 확률을 부여하는 토큰에 집중 (mode-seeking) → teacher의 핵심 행동을 정밀하게 모방

2. Related Work

Context Distillation

In-context knowledge를 모델 파라미터로 압축하여 추론 시 context 처리 오버헤드를 제거하는 기법이다. 기존 방법은 off-policy + forward KL을 사용하여 teacher-guided training과 autoregressive inference 간의 불일치로 인한 exposure bias를 겪는다.

On-Policy Distillation

On-policy distillation은 student의 자체 생성 trajectory에서 학습하여 exposure bias를 완화한다. MiniLLM은 reverse KL divergence를 최소화하여 student가 teacher의 high-likelihood 영역에 집중하게 하고, mode-averaging 문제를 회피한다. 본 연구는 이 on-policy 패러다임을 context internalization 문제에 적용한다.

Self-Distillation

모델이 자기 자신의 출력이나 conditioned 버전에서 학습하는 메커니즘이다. STaR은 self-generated solutions를 반복 학습하여 추론 능력을 부트스트랩한다. OPCD는 self-distillation의 한 형태로 볼 수 있지만, teacher가 별도 모델이거나 동시 업데이트되거나 frozen될 수 있어 더 유연하다.

3. Method — On-Policy Context Distillation

핵심 설계

목표: context $c$를 student model $\pi_\theta$의 파라미터 $\theta$에 압축하여, student가 context 없이도 context-aware teacher $\pi_{\text{teacher}}$의 행동을 재현하게 한다.

[Figure 1 설명]: OPCD 개요. Context $c$와 입력 $x$가 주어지면, Student는 $x$만 보고 응답 $y$를 생성(on-policy). Teacher는 $[c; x]$를 보고 토큰별 확률을 계산. Student와 Teacher의 분포 간 reverse KL divergence를 최소화하며, gradient가 Student로 역전파되어 context 지식을 파라미터에 내재화한다.
입력(x) → [Student π_θ] → 응답 y 생성 (on-policy, context 없이) ↓ [Context c; x] → [Teacher π_teacher] → 토큰별 확률 계산 ↓ Reverse KL: D_KL(π_θ || π_teacher) 최소화 → θ 업데이트

손실 함수

Sequence-level divergence를 token-level divergence의 합으로 분해한다:

Token-level reverse KL divergence의 계산:

$$D_{\text{KL}}(\pi_\theta(\cdot \mid x, y_{여기서 $\mathcal{V}$는 전체 vocabulary이다. 실제 구현에서는 student의 top-$k$ 토큰으로 합산을 제한하여 계산 효율을 높인다.

Reverse KL의 mode-seeking 효과: Reverse KL을 최소화하면 student는 teacher가 높은 확률을 부여하는 토큰의 확률을 높이고, teacher가 낮은 확률을 부여하는 토큰은 무시한다. 직관적으로, student가 토큰을 생성했을 때 teacher(context를 본 상태)가 "그 토큰이 좋다"고 평가하면 해당 확률을 높이고, "그 토큰은 별로다"라고 평가하면 억제한다.

Algorithm 1: OPCD Training

Input: 학습 데이터 D = {(x, c)}, Student π_θ, Teacher π_teacher Output: 학습된 Student π_θ for each batch (x, c) ~ D: 1. On-policy rollout: y ~ π_θ(· | x) [Student가 context 없이 응답 생성] 2. Token-level reverse KL 계산 (Eq. 2) 3. Sequence-level loss 계산 (Eq. 1) 4. θ 업데이트: minimize L(θ) return π_θ

3.1 Teacher Model 설정

OPCD는 두 가지 teacher 설정을 지원한다:

  • Teacher-Student Distillation ($\pi_{\text{teacher}} \neq \pi_\theta$): Teacher가 student보다 크거나 같은 별도 모델. Teacher는 frozen 또는 주기적 업데이트. 기본 설정
  • Self-Distillation ($\pi_{\text{teacher}} = \pi_\theta$): Teacher와 student가 동일 가중치를 공유. 입력만 다름 (teacher: $[c;x]$, student: $x$). 동시 업데이트

4.1 평가 과제

4.1.1 Experiential Knowledge Distillation

모델이 과거 문제 풀이 경험에서 전이 가능한 지식을 추출하여 context로 축적한 뒤, OPCD로 파라미터에 내재화하는 3단계 과정이다:

  1. Experiential Knowledge Extraction: 모델이 문제를 풀고 solution trace를 생성한 뒤, 이로부터 "- EXPERIENCE ITEM:" 형식의 일반적 통찰을 추출 (ground-truth label 불필요)
  2. Experiential Knowledge Accumulation: 다른 문제에서 추출한 경험을 결합하여 미래 문제를 위한 context $c$를 구축 (30회 축적 × 10 step)
  3. Experiential Knowledge Consolidation: OPCD로 축적된 경험적 지식을 student의 weight에 전이

데이터셋: DAPO-Math-17K (영어 수학 문제 ~14K), Frozen Lake (격자 탐색 게임), Sokoban (박스 푸시 퍼즐) — TextArena 환경에서 텍스트 기반으로 구현.

4.1.2 System Prompt Distillation

MetaSPO에서 최적화된 의료/안전 system prompt를 context $c$로 사용하여 모델에 내재화한다.

데이터셋: 의료 — MedMCQA (500 test samples). 안전 — TweetEval + Hatecheck + Ethos (500 test samples).

4.4 Cross-Size Distillation (Q3)

[Figure 2 설명]: Qwen3-8B teacher로부터 Qwen3-1.7B, 4B, 8B student로 경험적 지식을 distillation한 결과. OPCD(녹색)는 모든 student 크기에서 initial student(파란색)를 일관되게 능가한다. 반면, 같은 경험적 지식을 작은 모델의 context에 직접 주입(주황색 In-Context)하면 1.7B와 4B에서 오히려 initial student보다 성능이 하락한다.

##teacher가 생성한 경험적 지식을 작은 모델의 context에 직접 주입하면 성능이 오히려 하락##한다. 이는 경험적 지식과 그것을 소비하는 모델 사이의 on-policy alignment가 중요함을 시사한다. Teacher의 context에서 유효한 지식이 다른 모델의 context에서는 유효하지 않을 수 있다. 반면, OPCD를 통해 지식을 파라미터에 내재화하면 student 크기와 무관하게 개선된다.

4.5 On-Policy Distillation이 Forgetting을 완화 (Q2)

[Figure 3 설명]: Qwen2.5-3B-Instruct (student)를 frozen Qwen2.5-7B-Instruct (teacher)로 safety system prompt distillation 시 OPCD vs off-policy의 학습 곡선. 좌측(ID: safety accuracy)에서 OPCD는 off-policy보다 높은 정확도를 안정적으로 달성. 우측(OOD: medical accuracy)에서 off-policy는 initial student 대비 ~8 포인트 하락하지만, OPCD는 ~4 포인트만 하락하여 ##OOD forgetting을 약 절반으로 줄임##.

On-policy sampling이 student의 실제 분포에서 학습하므로, student의 기존 능력을 덜 교란한다. Off-policy 학습은 teacher 분포를 강제로 모방하게 하여 student의 고유 능력을 더 많이 덮어쓴다.

5. Conclusion

OPCD는 on-policy sampling과 reverse KL divergence를 통해 in-context knowledge를 모델 파라미터에 내재화하는 프레임워크이다. 수학, 텍스트 기반 게임, 도메인 특화 과제에서 기존 방법을 일관되게 능가하며 OOD 일반화도 더 잘 보존한다. 향후 연구로 경험적 지식의 지속적 축적, 적응형 context 선택 전략, 더 넓은 도메인으로의 확장을 계획한다.

강점

  • 깔끔한 이론적 동기부여: Off-policy + forward KL의 두 가지 문제(exposure bias, mode-covering)를 on-policy + reverse KL로 정확히 해결하는 설계가 명확하고 설득력 있음
  • 다양한 과제에서의 일관된 개선: 수학 추론, 텍스트 게임(Frozen Lake, Sokoban), 의료, 안전 도메인에서 모두 개선을 보여 방법론의 일반성을 입증
  • Cross-size distillation의 흥미로운 발견: 큰 모델의 경험을 작은 모델에 직접 주입하면 오히려 해가 된다는 관찰은 LLM deployment에 실용적 시사점을 제공
  • 표준편차 보고: Test-time 설정에서 여러 경험 샘플에 대한 표준편차를 보고하여 결과의 변동성을 투명하게 제시
  • Ablation이 풍부: Teacher-student vs self-distillation, raw trace vs knowledge, model size 효과 등 다양한 차원에서 분석

약점 및 한계

  • 계산 비용 미보고: On-policy sampling은 off-policy 대비 추가 inference cost가 발생한다. Student의 rollout 생성 + teacher의 평가가 필요하므로, 학습 시간과 GPU 비용에 대한 비교가 없다
  • 학습 규모가 작음: 50 steps, batch 128은 매우 짧은 학습이다. 더 긴 학습에서 off-policy와의 gap이 유지되는지 불확실
  • 모델 규모 제한: 최대 8B 모델까지만 실험. 70B+ 모델에서의 효과와 비용은 미검증
  • 경험 지식의 질에 대한 분석 부족: 어떤 종류의 경험이 가장 유용한지, 경험 축적의 수렴 특성, 잘못된 경험의 영향 등이 분석되지 않음
  • 제한된 OOD 평가: IF-Eval 단일 벤치마크로만 OOD 평가. 다양한 OOD 벤치마크에서의 검증이 필요
  • Reverse KL의 잠재적 단점 미논의: Reverse KL은 teacher 분포의 일부 mode를 무시(mode-dropping)할 수 있어, teacher의 다양한 행동이 필요한 과제에서는 부적합할 수 있음

Key Takeaways

  • On-policy + reverse KL은 context distillation의 강력한 조합: Exposure bias 해소 + mode-seeking behavior가 context internalization에 핵심적이며, 이 두 설계를 결합한 것이 OPCD의 성공 요인
  • 경험적 지식은 "추출 → 내재화" 파이프라인이 필요: Raw solution trace를 직접 사용하면 오히려 해로우며, 일반화된 통찰을 추출한 뒤 distillation으로 내재화해야 효과적. 이 패턴은 RAG 파이프라인에도 시사점을 줌
  • Cross-model knowledge transfer에는 alignment가 필수: 큰 모델의 context 지식을 작은 모델에 직접 주입하면 해가 되므로, distillation을 거쳐야 한다. 모델 크기에 맞게 지식을 "번역"하는 과정이 필요
  • Frozen teacher가 self-distillation보다 안정적: Self-distillation은 학습 신호의 variance가 높아 발산 위험이 있으므로, 실용적으로는 frozen teacher 설정을 권장
  • 50 step만으로도 유의미한 context internalization 가능: 매우 적은 학습으로도 in-context knowledge를 파라미터화할 수 있다는 점은 실용성 측면에서 주목할 만함
