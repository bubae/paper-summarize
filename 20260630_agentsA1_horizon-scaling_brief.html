먼저 알아야 할 것 1: AI 에이전트란?

요즘 AI는 단순히 질문에 답하는 챗봇을 넘어, 스스로 도구를 사용하고 결과를 검증하며 다음 단계를 결정하는 "에이전트"로 진화하고 있습니다. 코드를 작성·실행하고, 웹 검색으로 증거를 찾고, 실패하면 전략을 바꿉니다.

에이전트는 단일 응답이 아니라, 여러 단계의 의사결정 chain을 이룬다
🎯 비유: "주문 받는 직원" vs "프로젝트 매니저"
일반 챗봇은 "커피 한 잔 주세요"에 즉시 답하는 매장 직원과 같습니다.
AI 에이전트는 "신제품 출시 보고서 작성"을 받아서 → 자료 조사 → 데이터 수집 → 분석 → 초안 작성 → 검토 → 수정의 며칠짜리 프로젝트를 끌고 가는 매니저입니다. 도구도 쓰고, 실패하면 다시 시도하고, 중간 결과를 검증합니다.

먼저 알아야 할 것 2: "Long-Horizon"의 의미

Horizon이란 "AI가 한 번의 작업에서 다루는 결정/단계의 길이"입니다. 챗봇 답변은 보통 1~2 turn이지만, 진짜 어려운 작업(연구 보고서, 코드 디버깅, 머신러닝 실험)은 수십~수백 turn에 걸친 trajectory가 필요합니다.

Agents-A1의 학습 데이터는 평균 45,000 토큰의 긴 trajectory다 — 단순 QA의 45배 분량

먼저 알아야 할 것 3: AI 스케일링의 두 갈래

지금까지 AI를 더 똑똑하게 만드는 정석은 "모델을 더 키우는 것"이었습니다. GPT-3 (175B) → GPT-4 → 현재 1조(1T) 파라미터급. 이 논문은 다른 길을 제안합니다.

파라미터를 키우는 대신, 학습 trajectory의 길이와 품질을 키운다
🎓 비유: "수재 늘리기" vs "현장 경험 쌓기"
Parameter Scaling은 IQ가 높은 천재를 더 많이 모으는 것입니다. 효과적이지만 비쌉니다.
Horizon Scaling은 보통 사람에게 실패와 성공 사례, 도구 사용법, 검증 방법을 풍부하게 가르치는 것입니다. 머리는 같은데, 경험과 절차가 풍부해져서 어려운 프로젝트도 끝까지 끌고 갑니다.

먼저 알아야 할 것 4: KAG — "작업 매뉴얼" 인프라

긴 trajectory를 학습시키려면 단순 텍스트 corpus로는 부족합니다. 증거(Evidence), 행동(Action), 관찰(Observation), 검증(Verifier)이 모두 연결된 구조가 필요합니다. 이를 Knowledge-Action Graph (KAG)라 부릅니다.

KAG는 4-tuple (C_d, A_d, O_d, V_d) — corpus, action, observation, verifier로 정의된다
📓 비유: "백과사전" vs "현장 실습 일지"
일반 텍스트 corpus는 백과사전입니다 — "에펠탑은 324m"라는 사실은 있지만, 누가 어떻게 그 높이를 측정했는지, 잘못된 측정은 어떻게 걸러냈는지의 과정은 없습니다.
KAG는 현장 실습 일지입니다 — 측정 도구, 측정 행위, 관측 결과, 검증 절차가 모두 timestamp와 함께 기록됩니다. 같은 측정을 다른 학생에게 가르칠 때, "어떻게"를 가르칠 수 있습니다.

먼저 알아야 할 것 5: Distillation — "전문가 → 일반화" 전달

각 도메인(검색, 과학, 코딩, 도구 사용 등)별로 전문가 모델(Teacher)을 학습한 뒤, 그들의 능력을 하나의 학생 모델(Student)에 전달합니다. 이를 Distillation(증류)이라 부릅니다. Agents-A1의 핵심은 On-Policy Distillation — 학생이 자기 출력으로 답을 만들고, 그 답에 teacher가 직접 점수를 줍니다.

6개 도메인 전문가 모델이 하나의 35B 학생에게 token-level 신호로 전달
🏫 비유: "전공 교수 6명 → 학생 1명"
검색 전문 교수, 과학 전문 교수, 코딩 전문 교수가 따로 있습니다. 학생이 검색 문제를 풀 때는 검색 교수가 채점하고, 과학 문제를 풀 때는 과학 교수가 채점합니다. 이 채점이 누적되어 학생은 6개 과목 전부에 능통한 한 명의 종합 인재로 자랍니다.
핵심은 학생이 직접 답을 쓰고(on-policy), 교수가 그 답에 직접 첨삭한다는 점입니다. 다른 학생의 모범답안을 베끼는 게 아닙니다.

Stage 1: Full-Domain Supervised Fine-Tuning

Stage 2: Domain-Level Teacher Training

각 도메인마다 specialize된 전문가 모델을 만듭니다. SFT만으로는 부족한 경우 강화학습(RL)도 추가합니다.

Stage 3: Multi-Teacher On-Policy Distillation

6명의 전문가의 능력을 1명의 학생에게 통합. 핵심 혁신 두 가지:

  • Salient Vocabulary Alignment (SVA): 단순히 한 토큰의 확률을 맞추는 게 아니라, teacher가 "가장 가능성 있는 top-k 후보" 전체의 분포 모양을 학생에게 정렬.
  • Domain-Normalized Loss: 도메인마다 token 길이가 달라도 균등하게 학습 (instruction-following 짧다고 무시되지 않음).
🎯 비유: "정답 한 글자 채점" vs "답안 전체 첨삭"
기존 distillation은 학생 답안의 한 글자를 보고 "이 자리에 'A'를 썼는지" 채점합니다. 만약 'B'를 썼더라도, 'B'와 비슷한 'D'도 잘못된 답이라는 걸 가르치지 못합니다.
SVA는 "이 자리에 들어갈 수 있는 정답 후보 top-5(A, C, F, M, T)"를 teacher가 알려주고, 학생의 5개 후보 확률을 정답 후보 5개에 정확히 맞추도록 학습합니다. 잘못된 옵션들은 동시에 억제됩니다.

두 번째 사례 — 지구과학 데이터 분석

2008년 사이클론 Nargis(미얀마에 큰 피해를 준 열대 폭풍)의 진행 경로를 분석하라는 과제를 받자, Agents-A1은 완전히 스스로:

  • IBTrACS(국제 폭풍 트랙 데이터셋)를 데이터 소스로 자동 식별
  • 데이터 추출 → 정제 → 파생 지표 계산 (속도, 진행 방향 변화)
  • WMO/IMD와 JTWC/USA 두 기관의 풍속 추정치를 각자 보존 (서로 다른 운영 관례를 섞지 않음)
  • 5가지 시각화 (트랙 지도, 풍속 진화, 위경도, 진행 속도, 진행 방향) 생성
  • 벵골만에서 발생 → 북서진 → NE 재회전 → 미얀마 상륙 의 진화를 정확히 재구성

이는 단순 도구 호출이 아닌, 계획·코딩·실행·결과 검증·과학적 해석·보고서 작성이 모두 통합된 closed loop입니다.

한계와 주의점

⚠️ 1T 모델을 모든 면에서 이기진 않음

BrowseComp(웹 검색 광범위 지식), SciCode(연구 코딩), MLE-Bench-Lite(end-to-end ML 엔지니어링)에서는 여전히 1T 모델이 우세합니다. 특히 안정된 long-process 메모리가 필요한 작업에서 모델 capacity의 한계가 드러납니다.

⚠️ 평가 공정성 이슈

Agents-A1은 도구 사용 환경에서 평가됐고, 비교 1T 모델은 도구 없는 환경에서 평가됐습니다. 또한 $\tau^2$-Bench의 Qwen baseline 결과가 공식(81.2)과 재현(32.5) 사이에 차이가 있어, 일부 개선폭은 과대평가 가능성이 있습니다.

⚠️ 기본 능력의 부족 (저자 인정)

저자도 future work에서 인정했듯이, long-horizon에서 필요한 몇 가지 atomic ability가 아직 부족합니다:

  • "reasoning 전에 계획 세우기"
  • "act 전에 reflection 하기"
  • "긴 context의 핵심 정보 요약"
  • "과거 중요 정보 식별"

⚠️ 재현성과 generalization

학습에 사용된 GPU 시간·총 토큰 수가 명시되지 않았고, 동일 레시피가 Qwen 이외의 다른 base 모델(Llama, DeepSeek 등 35B)에 적용 가능한지는 검증되지 않았습니다. 즉, "35B + horizon scaling = 1T급"이 보편적인 공식인지, Qwen 특화 현상인지 단정하기 어렵습니다.
