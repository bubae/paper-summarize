먼저 알아야 할 것: AI가 음성으로 대화한다는 것

ChatGPT의 음성 대화, GPT-4o의 실시간 음성 응답 — 이런 기능이 어떻게 작동하는지 이해하려면 몇 가지 기본 개념이 필요합니다. 하나씩 살펴봅시다.

개념 1: 옛날 방식 — "직렬 파이프라인"

전통적인 음성 AI는 세 단계 파이프라인이었습니다: 귀 → 뇌 → 입. 사용자가 말하면 (1) 음성인식으로 텍스트를 만들고 → (2) 텍스트 AI가 답변을 생성하고 → (3) TTS(Text-to-Speech)가 음성으로 바꿉니다.

전통적 파이프라인은 편리하지만 지연(latency)이 크다
🍜 비유: 배달 앱의 3단계 처리

고객이 주문하면 (1) 상담원이 주문을 받아 종이에 적고 → (2) 셰프가 그것을 읽고 요리하고 → (3) 배달원이 완성된 요리를 가져갑니다. 각 단계가 완전히 끝나야 다음 단계 시작. 확실하지만 느립니다. 셰프가 조리하는 동안 배달원이 미리 출발할 수 없어요.

개념 2: 새로운 방식 — "End-to-End 음성-음성 (S2S)"

최근에는 GPT-4o처럼 음성이 들어가면 음성이 바로 나오는 통합 모델이 유행합니다. 세 단계를 하나의 큰 AI가 모두 처리하죠. 반응이 빠르지만... 새로운 문제가 생깁니다.

통합 모델은 빠르지만, 하나의 AI에 두 가지 다른 능력을 넣는 것이 어렵다
🎨 비유: 화가에게 성악을 가르치면?

훌륭한 화가에게 오페라 성악을 추가로 훈련시킨다고 상상해봅시다. 몇 달간 열심히 노래 연습을 시키면... 노래는 그럭저럭 부르게 되지만 그림 실력이 오히려 떨어집니다. 뇌의 자원이 새로운 능력에 배분되면서 원래의 전문성이 희석되죠. 이것이 catastrophic forgetting 문제입니다.

개념 3: 왜 "중간 뇌 상태"가 핵심인가?

AI(Transformer)는 여러 층(layer)이 쌓여 있는 구조입니다. 마치 다층 건물처럼요. 사람이 문제를 풀 때 뇌에서 정보가 여러 처리 단계를 거치듯, AI의 각 층도 다른 종류의 정보를 담고 있습니다.

CKA 분석으로 발견: 중간층이 음성 생성에 가장 적합한 "의미 + 음운" 정보를 담고 있다
🎤 비유: 통역사의 두 가지 방식

회의 통역사가 두 가지 방식으로 일할 수 있습니다:

방식 A (전통적): 발표자의 한 문장이 완전히 끝날 때까지 조용히 기다렸다가 통역 시작. 정확하지만 느립니다.

방식 B (PRIME-Speech): 발표자의 사고 흐름을 청취하면서 병렬로 통역을 이어감. 발표자의 두뇌가 다음 단어를 결정하는 순간, 통역사도 이미 그에 맞춰 통역을 진행. 매우 빠르고 자연스럽죠. 하지만 이건 발표자와 통역사가 서로 다른 사람이어야만 가능. 한 사람이 두 역할을 다 하려면 뇌가 분열됩니다.

이 논문이 풀려는 문제

Microsoft에는 이미 Phi-4-MM이라는 훌륭한 음성 이해 AI가 있습니다. 이 AI는 음성을 듣고 텍스트로 답변을 잘 만듭니다. 그런데 음성 응답까지 만들려면 다음 중 하나를 해야 합니다:

  • 선택 1: Phi-4-MM에 TTS를 뒤에 붙인다 → 느려지고, 결국 옛날 파이프라인이 됨
  • 선택 2: Phi-4-MM 전체를 다시 학습해서 음성 출력을 만든다 → 원래의 이해/추론 능력이 저하됨
  • 선택 3: 답변 텍스트가 나오면 그것을 talker에게 넘겨서 음성으로 → 여전히 텍스트가 완성돼야 함

PRIME-Speech는 완전히 새로운 방식: Phi-4-MM을 그대로 얼려두고(freeze), 중간층 뇌 상태를 실시간으로 감시하는 별도의 "음성 생성 부서"를 추가.

여러 번 대화할 때: 왜 오디오 메모리는 지워야 하는가?

실제 대화는 여러 턴(turn)에 걸쳐 이어집니다. "안녕" → "안녕하세요" → "오늘 뭐 할까?" → "..." 이런 식이죠. AI는 이전 대화를 기억(KV cache)하고 있어야 문맥을 유지할 수 있습니다.

그런데 PRIME-Speech는 여기서 비대칭 정책을 씁니다: 텍스트 기억은 계속 쌓고, 오디오 기억은 매 턴마다 지웁니다.

이 정책 없이는 3턴 이후 WER이 100%+ 폭증 (완전한 붕괴)
📓 비유: 회의록과 녹음기의 차이

연속 미팅에서 회의록(텍스트)은 계속 쌓아둡니다 — 지난 안건을 잊으면 안 되니까요. 하지만 녹음기 파일(오디오)은 각 미팅마다 새로 시작합니다. 지난 미팅의 잡음이나 잔향이 이번 미팅 녹음에 섞이면 안 되기 때문입니다. PRIME-Speech도 마찬가지: 텍스트 기억은 대화의 맥락, 오디오 기억은 현재 응답의 소리에 국한.

속도 트릭: MTP (한번에 여러 소리 예측)

음성 codec은 초당 25번(25Hz) 예측이 필요합니다. 답변이 3초짜리면 75번 예측 = 75번의 forward pass. 느립니다.

Multi-Token Prediction (MTP)은 한 번의 예측으로 미래의 k개 codec token을 동시에 뽑아냅니다. k=4면 25Hz가 실질적으로 6.25Hz가 되어 4배 빠름.

MTP는 오디오 branch에만 적용. 뇌는 여전히 얼려진 상태.
⌨️ 비유: 타자기 vs 음성 인식 워드프로세서

타자기(k=1)는 한 글자씩 순서대로 칩니다. 음성 인식 워드프로세서(k=4)는 발화된 단어 뭉치를 한 번에 받아서 여러 글자를 동시에 씁니다. 결과물은 같지만 속도가 크게 다르죠. 중요한 것은: 문장을 생각하는 뇌는 여전히 사람 그대로, 손가락(출력) 부분만 바꿈.

어디에 쓸 수 있나?

기존 S2T LLM이 이미 잘하는 분야에 음성 응답을 추가하고 싶은 모든 곳에 유용

한 줄 결론

"AI에게 새 능력을 가르치는 대신,
기존 능력을 '엿듣는' 별도의 부서를 만들어라"

PRIME-Speech는 모델 변경 없이도 새 modality를 추가할 수 있다는 것을 보여준 논문입니다. 앞으로 이미 강력한 텍스트 LLM(GPT-4, Claude 등)에 음성 뿐 아니라 이미지 출력, 비디오 출력을 추가할 때도 이 원리가 적용될 수 있습니다.

한계와 남은 숙제

1. Backbone 의존성: 이 방법의 성능은 근본적으로 기존 Backbone LLM의 품질에 의존합니다. Phi-4-MM 같은 이미 강력한 모델이 있어야 효과적. 약한 backbone에서 시작하면 결과도 약할 것.

2. 감정과 개성이 없다: PRIME-Speech는 내용 전달에 최적화됨. 화자의 목소리 유지, 감정 표현, 자연스러운 억양은 평가 대상에서 제외되었습니다. GPT-4o의 감정적 음성과는 아직 다른 방향.

3. 학습 데이터의 인공성: Target 음성이 Microsoft Azure TTS로 합성된 데이터. 실제 사람 발화의 자연스러운 변화(호흡, 발음 변주, 감정)는 학습되지 않음.

4. Layer 선택은 발견적(heuristic): "2/3 지점 layer가 최적"이라고 하지만, backbone마다 이 지점은 다를 수 있음. 새로운 모델에 적용할 때마다 layer selection이 필요.

5. Post-decoder가 결코 가볍지 않음: 2B params + 100M MTP = 여전히 큰 부담. VocalNet의 lightweight talker(~8B 대비 매우 작음)에 raw throughput은 뒤처짐. Frozen backbone의 이득으로 상쇄.
