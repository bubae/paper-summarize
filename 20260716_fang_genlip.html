TL;DR

  • 문제: 기존 Vision-Language Pre-training(VLP)은 (i) CLIP/SigLIP류 contrastive는 MLLM(Multimodal Large Language Model)의 next-token-prediction 목적과 objective mismatch가 있고, (ii) CapPa/AIMv2/OpenVision2 같은 generative는 별도 text decoder를 두어 아키텍처가 복잡하고 vision encoder가 gradient를 간접적으로만 받는다.
  • 해결: ##ViT(Vision Transformer) 하나만으로 image와 text token을 concat한 시퀀스에 표준 autoregressive language modeling loss를 적용##. Text decoder도, contrastive loss도, CLS token도 없다.
  • 왜 작동하는가: Prefix-LM attention(이미지 부분은 bidirectional, 텍스트 부분은 causal) + MRoPE(Multimodal Rotary Position Encoding)로 단일 Transformer가 vision과 language를 자연스럽게 처리. 발생하는 attention sink는 gated attention으로 억제하여 vision token의 정보를 살린다.
  • 비유: 기존 방법이 "번역기(vision encoder) + 통역사(text decoder)" 2인 1조라면, GenLIP은 ##ViT 자기가 직접 말을 배워서 그림 보고 설명하게 시키는## 방식이다.

핵심 기여 (Key Contributions)

  1. 단일 Transformer + 단일 objective: 별도 text encoder/decoder 없이 하나의 ViT가 image token과 text token을 concat한 시퀀스를 처리하고, text 부분에만 standard LM loss를 적용. Contrastive batch 구성 불필요.
  2. Prefix-LM attention + Multimodal RoPE: 이미지 토큰은 서로 bidirectional, 텍스트 토큰은 causal로 attend. Absolute positional embedding을 버리고 MRoPE(2D image position + 1D text position)를 도입.
  3. Gated Attention으로 attention sink 완화: 혼합 모달리티 환경에서 첫 토큰이 attention을 독점하는 sink 현상을 관찰하고, per-head input-dependent gate로 억제. Loss spike 감소, ImageNet linear probing 76.2 → 84.3 (So/16).
  4. 데이터 효율성 실증: ##8B samples로 SigLIP2 (40B samples)를 앞서거나 대등한 성능## 달성. 특히 Doc&OCR 벤치마크에서 큰 폭 개선(SigLIP2 대비 +3.3 avg on So/16).
  5. Native aspect ratio 2단계 학습: Stage 1은 224×224 고정, Stage 2는 native aspect ratio + longer captions로 detail-sensitive task 성능 추가 개선.

Background: 왜 Generative VLP인가

기존 VLP의 세 계열과 각각의 한계

Vision-Language Pre-training(VLP)은 크게 세 가지 계열로 나뉜다:

(1) Dual-Encoder Contrastive Pretraining — CLIP, SigLIP, MetaCLIP 등. 이미지와 텍스트를 각각 별도의 인코더로 처리한 뒤 InfoNCE 계열 contrastive loss로 임베딩 공간을 정렬. Loss는 대략 다음 형태:

$$\mathcal{L}_{\text{CLIP}} = -\log \frac{\exp(\langle f_I(I), f_T(T) \rangle / \tau)}{\sum_{T'} \exp(\langle f_I(I), f_T(T') \rangle / \tau)}$$

직관적으로: batch 안의 정답 (I, T) 쌍은 임베딩을 가깝게, 나머지 negative 쌍은 멀게 만든다. 이 방식은 image classification, retrieval 같은 discriminative task에 강하지만, 두 가지 문제가 있다:

  • Objective mismatch: MLLM은 궁극적으로 next-token prediction으로 최적화되는데, contrastive는 discriminative alignment를 배운다. 두 목적이 근본적으로 다르다.
  • Global alignment only: image-text pair 전체 수준의 alignment만 학습하므로, cross-modal deep interaction (예: 이미지 내 텍스트 인식, 지역 세부사항) 학습이 부족.

(2) Encoder-Decoder Generative Pretraining — CapPa, AIMv2, GIT, OpenVision2. Vision encoder 위에 text decoder를 얹고, decoder가 caption을 autoregressively 생성하도록 학습. 여기서 vision encoder는 decoder를 통과한 gradient로 간접적으로만 업데이트된다. Hybrid(CoCa, SigLIP2)는 contrastive + captioning을 결합해 gap을 좁히지만 오히려 아키텍처와 loss가 복잡해진다.

(3) Single-Transformer Pretraining — SuperClass, VL-BEiT, OneR, SAIL, Chameleon 등. 단일 트랜스포머로 통합하려는 시도. GenLIP은 이 계열에 속하지만, ##"scalable vision encoder를 처음부터 학습"##하는 것이 목표라는 점에서 native MLLM을 목표로 하는 SAIL/NEO와 구분된다.

Objective Mismatch를 실증한 diagnostic (Figure 1 Right)

저자들이 이 논문 첫 페이지에서 던지는 진단 실험이 인상적이다: 서로 다른 방식으로 학습된 vision encoder를 동일한 LLM에 붙이고, projector만 튜닝했을 때(PPL1)와 projector+LLM을 튜닝했을 때(PPL2)의 perplexity를 비교. Generative objective로 학습된 encoder(CapPa, AIMv2, OpenVision2, GenLIP)가 contrastive encoder(SigLIP, SigLIP2)보다 낮은 PPL을 보였고, 특히 GenLIP이 가장 낮았다.

[Figure 1 설명]: 왼쪽 다이어그램은 CLIP/SigLIP(contrastive dual-encoder), CapPa/AIMv2/OpenVision2(encoder-decoder generative), GenLIP(단일 Transformer + LM head)의 세 가지 패러다임을 대조. 오른쪽 그래프는 LLM에 붙였을 때 PPL1(projector만 튜닝)과 PPL2(projector+LLM 튜닝) 모두에서 generative 계열, 특히 GenLIP이 낮은 perplexity를 달성함을 보인다. 낮은 PPL은 LLM의 생성 목적과의 궁합이 좋다는 뜻.

Approach: GenLIP

3.1 Framework 개요

Data Format: 학습 데이터는 image-text pair $\{(I_i, T_i)\}_{i=1}^N$. 이미지 $I_i$는 patch embedding layer로 $\{v_0, v_1, \dots, v_M\}$ non-overlapping patch로 분할. 텍스트 $T_i$는 off-the-shelf Qwen3 tokenizer로 $\{t_0, t_1, \dots, t_L\}$ subword로 tokenize. 최종 입력 시퀀스:

$$S = [v_0, \dots, v_M, t_0, \dots, t_L] \quad (1)$$

즉, ##이미지 토큰이 prefix, 텍스트 토큰이 suffix##.

Architecture: 세 부분으로 구성된 단일 Transformer.

  • Modality-specific embedding layer (image patch용 conv patch embed, text용 token embed)
  • Prefix-LM attention을 쓰는 unified Transformer
  • Layer Normalization + LM head

표준 ViT와의 차이는 두 가지 minimal modification:

  1. Multimodal RoPE (MRoPE): absolute position embedding을 버리고 MRoPE를 도입. 이미지 patch는 2D 좌표 $(h, w)$로, 텍스트 토큰은 1D 시퀀스 인덱스로 rotation을 적용. Figure 2(c)에서 image patch의 MRoPE index가 $(1,1), (1,2), (2,1), (2,2), \dots$ 형태로 부여됨을 볼 수 있다.
  2. Prefix-LM Attention: 이미지 토큰끼리는 full(bidirectional) attention, 텍스트 토큰은 causal attention. 텍스트는 앞의 이미지 토큰 전체 + 자기 앞의 텍스트만 볼 수 있다.
[Figure 2 설명]: (a) GenLIP 전체 구조 — patchify → concat with text token → 단일 Transformer → LM head가 text token만 예측. 이미지 부분은 loss에 기여하지 않음. (b) Gated Attention Layer — standard attention output $A$에 element-wise gate $G$를 곱해 $\tilde{A} = G \odot A$를 만들고, SwiGLU FFN을 거침. Gate $G$는 입력 $X$에서 sigmoid로 계산. (c) Prefix-LM attention의 attention mask — 상단 왼쪽 블록(image→image)은 full, 우하단 텍스트 블록은 하삼각(causal), 텍스트→이미지는 full.

3.2 Objective

Standard autoregressive LM loss, ##단 텍스트 부분에만 적용##:

$$\mathcal{L}_{\text{LM}} = -\sum_{k=0}^{L} \log P(t_k | \{v_j\}_{j=0}^{M}, \{t_i\}_{i=0}^{k-1}; \theta) \quad (2)$$

직관적으로: 모델은 앞의 모든 이미지 토큰과 지금까지의 텍스트 토큰을 조건으로, 다음 텍스트 토큰의 확률을 예측한다. 이미지 토큰 자체를 pixel-space에서 재구성하는 부분은 없다(VL-BEiT/CapPa와의 차이). 결과적으로 vision encoder는 "이 이미지가 어떤 텍스트로 서술될 만한 정보를 담고 있는가"라는 정보량 목적으로 직접 gradient를 받는다.

Vision Encoder로 사용할 때

MLLM에 GenLIP을 vision encoder로 붙일 때는:

  • 마지막 Transformer block 다음의 LN layer output을 feature로 사용
  • 2-layer MLP projector를 LLM 입력 공간에 정렬
  • Text 관련 모듈(tokenizer, LM head) 제거
  • Prefix-LM attention은 이미지 토큰뿐이므로 자동으로 full attention이 됨

3.3 Gated Attention: attention sink 문제 해결

문제 관찰: 위 unified 구조를 그대로 학습하면 attention sink가 심하게 발생한다. 시퀀스의 첫 토큰이 attention mass를 과도하게 흡수하는 현상이다. Figure 3에서 27-layer So/16 모델의 layer-wise attention 분석 결과를 보면:

[Figure 3 설명]: X축은 layer index(1~27). (a) vision token → 첫 sequence 토큰으로 가는 attention: w/o GA 평균 34.4, w/ GA 평균 0.8. (b) generated text token → 첫 토큰: w/o GA 17.1, w/ GA 0.1. (c) generated text → vision tokens: w/o GA 8.9, w/ GA 13.4. ##Gated attention이 sink 흡수를 극단적으로 줄이고, 대신 텍스트→비전 attention을 증가시켰다##.

왜 발생하나: softmax attention의 sum-to-one 제약 때문에, 모델은 "안전한 dumping ground"를 필요로 한다. Prefix-LM에서는 이미지 토큰이 bidirectional로 정보를 aggregate할 수 있어서 앞쪽 몇 개의 vision token이 "이미지 요약"이 되어 sink 역할을 하게 되고, 뒤의 텍스트 토큰들은 이 소수 sink token만 참조하는 shortcut을 학습한다. 결과적으로:

  • 학습 시 loss spike 발생
  • Spatial diversity 감소 → 개별 patch feature의 discriminative 능력 저하 (ImageNet linear probing 저하)
  • Scaling behavior 불안정

Gated Attention 정의: Qiu et al. [54]의 아이디어를 차용. Transformer block의 hidden state $X \in \mathbb{R}^{n \times d}$에 대해 standard attention $A = \text{Attn}(X)$를 구하고, per-head input-dependent gate를 적용:

$$G = \sigma(X W_g + b_g), \quad \tilde{A} = G \odot A \quad (3)$$

직관적으로: 각 head, 각 token position마다 sigmoid gate가 "이 attention output을 얼마나 통과시킬지"를 학습. Text token이 sink token 하나에 attention을 몰아넣더라도, gate가 이를 억제할 수 있다. Figure 2(b)에서 gate signal(빨간 선)이 attention output과 element-wise 곱해지고 SwiGLU를 거치는 흐름을 볼 수 있다.

효과: (i) Loss spike 완화, (ii) 수렴 가속, (iii) scaling 안정화, (iv) ImageNet linear probe 76.2 → 84.3 (So/16). Table 10에서 register token(1개, 4개)과 비교했을 때도 gated attention이 대부분 벤치마크에서 우수.

알고리즘 흐름도

[학습 시 forward pass] 이미지 I ─→ Patch Embed ─→ {v₀, ..., v_M} │ │─→ concat: S = [v₀,...,v_M, t₀,...,t_L] 텍스트 T ─→ Qwen3 Tokenizer ─→ {t₀, ..., t_L} │ ↓ [Transformer Block × N] (Prefix-LM attn + MRoPE + Gated Attn) ↓ LN → LM head → P(t_k | ...) ↓ LM loss (text token만) [MLLM inference 시 vision encoder로 사용] 이미지 I ─→ Patch Embed ─→ {v₀, ..., v_M} ↓ [Transformer Block × N] (Full attn, gated, MRoPE) ← LM head/tokenizer 제거 ↓ LN output ↓ 2-layer MLP projector ↓ LLM 입력

3.3 Pretraining Details

두 단계 학습:

Stage 1: Fixed Resolution Pretraining

  • Dataset-S1: curated 1B image-text pair
  • Resolution: 224×224 고정, 196 patch
  • Total samples: ##8B (Dataset-S1을 8 epoch)##
  • 목적: 강한 foundational visual representation 학습

Stage 2: Diverse Resolution Adaptation

  • Dataset-S2: 37M image-text pair, longer caption + higher resolution
  • Native aspect ratio 유지, vision token 수 [16, 1024] 범위
  • 1 epoch만 학습
  • 목적: variable resolution + fine-grained detail (OCR, chart) 학습

4. Experiments

4.1 Setup

실험은 논문이 답하려는 세 가지 핵심 질문으로 정리된다:

  1. GenLIP이 기존 contrastive/generative baseline보다 실제로 강한가?
  2. Data/Model 양쪽에서 scale하는가?
  3. 어떤 컴포넌트(gated attention, native aspect ratio, single-transformer 등)가 실제로 성능을 만드는가?

Baselines: CLIP (L/14, 12.8B), AIMv2 (L/14, 12.0B), OpenVision2 (L/16, 12.8B), SigLIP (L/16, 40B), SigLIP2 (L/So/g/16, 40B). 참고로 GenLIP은 8.0B로 학습.

Protocol: 주로 frozen visual representation evaluation. Cambrian 논문의 프로토콜을 따라, vision encoder는 얼리고 LLM만 fine-tune. LLM은 Qwen2.5-1.5B-Instruct와 Qwen2.5-7B-Instruct 두 크기, framework는 LLaVA-NeXT, SFT 데이터는 LLaVA-OneVision (3M+). 이 세팅은 ##"vision encoder 자체의 순수 quality"##를 측정하는 데 목적.

Benchmark 14종 (ALL AVG):

  • Doc&OCR (7): ChartQA, OCRBench, DocVQA, TextVQA, AI2D, InfoVQA, SEED-Bench-2-Plus
  • General VQA (4): MME-P, VQAv2, GQA, ScienceQA
  • Caption (3): NoCaps, COCO, TextCaps (CIDEr)

4.3 Scalability Analysis

Data Scaling (Figure 4)

실험 의도: 1B → 8B로 데이터를 늘렸을 때 성능이 지속 개선되는가, 그리고 gated attention의 이득이 data scale에 따라 어떻게 변하는가.

결과: 세 task군(OCR, VQA, Caption) 모두에서 1B → 4B 구간의 gain이 가파르고 4B → 8B에서 완만해짐. Caption/VQA는 후반에 saturation 조짐, OCR은 여전히 상승. ##Gated attention은 특히 low-data regime에서 이득이 크며, data scale에 걸쳐 일관되게 유효##.

Model Scaling (Table 7)

L/16 → So/16 → g/16으로 갈수록 모든 벤치마크에서 성능 개선. ##L/16이 So/16보다 저조한 것은 fixed-resolution stage에서만 나타나고, native-aspect adaptation 이후 gap이 좁혀짐##. So/16과 g/16의 격차는 상대적으로 작다. 저자는 "적절한 model size가 중요하며, 무작정 키우는 이득은 감소한다"고 서술.

4.4 Ablations

4.4.1 Controlled Comparison with Other VLPs (Table 8)

실험 의도: 앞의 main result는 pretraining data가 다르다는 반박이 가능. 같은 2.0B budget으로 SigLIP, OpenVision2, GenLIP을 직접 비교하면 GenLIP의 원천적 우위가 확인되는가.

세팅: 모두 Dataset-S1의 2.0B sample로 학습. GenLIP은 first stage만, SigLIP/OpenVision2는 pretrain + 0.2B의 high-res adaptation.

Appendix 핵심 논의

D. Attention Sink 이론적 배경

Attention sink는 softmax의 sum-to-one 제약에서 발생. Vision Transformer의 bidirectional self-attention에서는 low-semantic region의 소수 token이 "register" 역할을 하며 sink가 되고, autoregressive LM에서는 구조적으로 첫 토큰이 subsequent 모든 position에서 접근 가능하므로 자연스러운 sink 후보가 된다. Prefix-LM은 이 두 성질이 결합된 형태로, 첫 vision token $v_0$가 특히 favorable sink candidate가 된다.

Encoder-decoder generative VLP(CapPa 등)에서는 vision encoder와 text decoder가 분리되어 있어 sink 문제가 decoder에 국한되지만, GenLIP처럼 단일 트랜스포머로 통합하면 sink가 vision representation 자체를 열화시킨다. 이것이 gated attention이 GenLIP에서 특히 결정적인 이유.

E. Language Prior의 함정 (SAIL 대비 분석)

Figure 10, 11의 attention pattern 분석에 따르면, LLM 초기화된 SAIL 변종은 초반 레이어에서 generated text token이 attention의 대부분을 prompt/sink token에 몰아넣고 vision token에는 거의 attend하지 않는다. Gated attention을 추가해도 이 편향이 완화될 뿐 완전히 사라지지 않는다. 반면 scratch에서 학습된 SAIL-g-Scratch는 vision token에 더 많이 attend하고, attention pattern이 GenLIP-So16과 유사해진다.

##"강한 LLM prior + 약한 vision conditioning"의 조합이 captioning objective를 만족시키는 shortcut을 만들어, vision encoder의 grounded representation 학습을 방해##한다는 통찰. 이는 modular MLLM용 vision encoder를 pretrain할 때 중요한 함의를 갖는다.

비판적 분석

강점

  • 아키텍처적 미니멀리즘: 별도 text decoder 없이 단일 ViT + LM head라는 극단적 단순화. 이는 코드베이스, 학습 파이프라인, downstream 통합 모두에 이롭다.
  • 실증적으로 강력한 data efficiency: 8B samples로 40B의 SigLIP2를 앞서는 것은 objective mismatch 가설의 강력한 empirical support.
  • Attention sink의 modality-specific 분석: gated attention이 vision encoder pretraining 성능에 직접 영향을 준다는 것은 새로운 관찰. Figure 3의 layer-wise 분석은 매우 설득력 있다.
  • SAIL과의 controlled comparison: "LLM init이 항상 좋다"는 통념을 정면으로 반박. Vision encoder라는 target을 위해서는 오히려 scratch가 낫다는 반직관적 결과.
  • Discriminative ability 유지: caption objective만 썼는데도 ImageNet 85.2 (g/16), ADE20K 44.5로 SigLIP/CLIP과 대등. Emergent patch semantics(Figure 7)까지 확인.

약점 및 우려

  • Dataset-S1/S2의 정체 불명: "1B curated"와 "37M long-caption"이지만 정확히 어떤 소스에서 어떤 필터링/re-captioning을 거쳤는지는 밝히지 않았다. Caption quality가 결정적일 수 있는데, 재현이 어렵다.
  • Baseline과의 data 이질성: Table 3/4는 pretraining data를 통제하지 않았다. Controlled Table 8이 있지만 2B budget뿐이며, high-res adaptation을 GenLIP은 생략하는 등 세팅이 완전히 대칭적이지 않다.
  • ImageNet에서 SigLIP2에 밀림: 순수 discriminative 관점에서는 SigLIP2 (dense supervision 사용)가 여전히 우세. GenLIP의 표현이 모든 downstream을 dominate하는 것은 아님.
  • Scaling ceiling: 데이터를 4B → 8B로 늘렸을 때 이득이 감소. 진짜 SigLIP2 규모(40B)로 확장했을 때의 거동은 미확인.
  • Native MLLM에는 부적합: GenLIP은 modular MLLM의 vision encoder 목적이라 명시. Chameleon/NEO/SAIL 스타일의 fully-native MLLM으로는 활용할 수 없다.
  • Compute cost 분석 부재: 40B samples로 학습된 SigLIP2 대비 8B가 이득이라는 주장은 강력하지만, 절대적 GPU-hour나 wall-clock 비교는 없다. 두 배 넓은 모델(g/16이 1.1B)이 encoder-only baseline(SigLIP2 g/16 대비 파라미터 수) 대비 정확히 얼마나 무거운지도 애매.
  • Cross-modal alignment 심층 분석 부재: patch semantics readout은 몇 가지 예시만 정성적으로 보여줄 뿐, 정량 평가(예: patch classification accuracy, grounding metric)로 검증되지 않았다.
  • Prefix-LM attention의 실효성: Full attention vs Prefix-LM vs full causal의 비교 ablation이 명시적으로 제시되지 않아, MRoPE와 prefix-LM 각각의 기여가 분리되지 않는다.

후속 연구 방향

  • ##Dataset-S2 scale up##: 37M을 훨씬 늘려 native aspect ratio 데이터의 scaling curve를 그리는 실험.
  • ##Instruction-following pretraining##: caption 외에 VQA, region description 같은 richer supervision signal 추가.
  • ##Gated attention의 일반화##: 다른 VLP나 pure LM에서도 attention sink 완화 효과를 정량화.
  • ##Dense supervision hybrid##: SigLIP2가 도입한 dense supervision을 GenLIP에 결합해 discriminative gap을 좁힐 수 있는지.
  • ##Video로 확장##: 시간 축을 MRoPE에 추가해 video-language pretraining으로 확장.

Key Takeaways

  1. Vision encoder pretraining의 objective를 MLLM downstream(next-token prediction)에 맞춘다는 발상 자체가 지금 시점에서 매우 실용적. Contrastive는 discriminative task에 유리하지만 MLLM용 vision encoder로는 objective mismatch가 있고, 이를 실증한 PPL diagnostic(Figure 1)은 강력한 근거.
  2. 단순화가 오히려 data efficiency를 높인다: 별도 text decoder를 없애고 gradient가 vision backbone에 직접 흐르게 하는 것이 8B → 40B baseline 대비 gain의 원천. 아키텍처 단순화가 항상 옳지는 않지만, 이 논문은 매우 명확한 예.
  3. Attention sink는 vision encoder pretraining의 실제 병목: register token 대신 gated attention을 쓰면 ImageNet linear probe가 76.2 → 84.3까지 회복. 다른 vision pretraining 프로젝트에서도 참고할 만한 실전 팁.
  4. LLM initialization은 만능이 아니다: vision encoder라는 target을 위해서는 scratch가 낫다는 SAIL 대비 실험. "modular"와 "native"의 목적 차이를 인지하고 pretraining 결정을 내려야 함.
  5. Doc&OCR에 큰 gain: chart, invoice, document 분석 같은 실무 응용에 즉시 유용. Long/native-aspect adaptation Stage 2가 detail-sensitive task에 결정적임을 두 단계 pretraining schedule이 잘 보여준다.
