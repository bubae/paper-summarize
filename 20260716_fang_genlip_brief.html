"눈(ViT)이 스스로 말하게 하라"
— 하나의 트랜스포머로 그림을 보고 설명까지 배우는 새로운 사전학습

전문가용 상세 요약 보기 →
"복잡한 2인 1조 대신, ViT 혼자서 그림을 보고 말을 배운다.
1/5의 데이터로 SigLIP2를 뛰어넘다."

먼저 알아야 할 것들

GenLIP을 이해하려면 크게 다섯 가지를 알아야 합니다. AI가 "그림을 이해한다"는 것이 무슨 의미인지(Vision Encoder), 왜 챗GPT 같은 MLLM이 그림을 처리하려면 이 vision encoder가 필요한지, 그리고 지금까지 시도된 CLIP과 CapPa 두 가지 학습 방식의 한계, 마지막으로 이 논문이 새롭게 지적한 Attention Sink라는 문제를 순서대로 살펴봅시다.

개념 1: Vision Encoder — AI의 "눈"

사람이 그림을 볼 때 뇌는 픽셀 하나하나를 처리하지 않습니다. "고양이", "털이 갈색", "귀가 뾰족" 같은 의미 있는 특징으로 변환하죠. AI에서도 마찬가지입니다. Vision Encoder는 이미지를 받아 "이 그림이 담고 있는 개념"을 숫자 벡터로 변환하는 신경망입니다.

Vision Encoder는 이미지를 "의미의 숫자"로 압축하는 변환기

대표적인 Vision Encoder는 ViT (Vision Transformer)입니다. 이미지를 작은 patch(예: 16×16 픽셀 조각) 여러 개로 나누고, 각 patch를 하나의 "단어"처럼 취급해 Transformer로 처리합니다. GenLIP도 이 ViT를 기반으로 합니다.

개념 2: MLLM (Multimodal LLM) — 그림도 이해하는 챗봇

GPT-4o, Claude, Gemini 같은 최신 AI 챗봇은 텍스트뿐 아니라 이미지도 이해합니다. 이런 모델을 MLLM(Multimodal Large Language Model)이라고 합니다. MLLM은 대체로 세 부품으로 구성됩니다:

MLLM = Vision Encoder + Connector + LLM

여기서 Vision Encoder의 품질이 MLLM 성능의 상한선을 결정합니다. 눈이 흐릿하면 아무리 뇌가 똑똑해도 그림을 제대로 설명할 수 없죠. 그래서 어떤 Vision Encoder를 어떻게 학습(pretrain)시킬지가 매우 중요한 연구 주제입니다.

개념 3: CLIP — 지금까지의 "표준" 학습 방식

CLIP(2021년, OpenAI)은 Vision Encoder 학습에 혁명을 일으킨 방법입니다. 인터넷에서 수억 장의 (이미지, 설명 텍스트) 쌍을 모아 다음처럼 학습합니다: "같은 쌍은 서로 가깝게, 다른 쌍은 서로 멀게" 만든다.

CLIP: 이미지와 텍스트를 각각 별도 인코더로 인코딩 후 정렬

CLIP의 문제는 무엇일까요? 이미지 검색이나 분류에는 잘 작동하지만, MLLM은 다르게 훈련됩니다. MLLM은 "다음에 어떤 단어가 나올까?"를 예측하는 모델(Autoregressive Language Model)이거든요. 그래서 저자들은 이렇게 말합니다:

🎯 비유: 목적이 다른 훈련

축구를 잘하려는 선수에게 야구 훈련을 시킨다고 상상해보세요. 둘 다 "공을 다루는 운동"이지만, 필요한 근육과 감각이 다릅니다. CLIP은 "물건 분류"에 최적화된 야구 훈련이고, MLLM은 "문장 생성"이라는 축구를 하는데, 우리는 축구 선수를 만들고 싶은 상황이죠. 훈련 목적과 실전 목적이 어긋난 겁니다. 이걸 논문에서는 Objective Mismatch라고 부릅니다.

개념 4: CapPa/AIMv2 — Generative 방식의 시도

CLIP의 문제를 인식한 연구자들이 만든 대안이 generative pretraining입니다. Vision Encoder 뒤에 별도의 Text Decoder를 붙이고, 그림을 보고 caption을 생성하도록 훈련시킵니다. CapPa, AIMv2, OpenVision2 등이 이 계열입니다.

통해서만 흐름 → 간접적
Generative 방식의 한계: text decoder라는 "중간 다리"가 필요

이 방식은 CLIP보다 낫지만 여전히 문제가 있습니다:

  • Text decoder라는 추가 모듈을 학습해야 함 (아키텍처 복잡화)
  • Vision Encoder는 gradient를 text decoder를 통해서 간접적으로만 받음
  • 학습 후엔 text decoder를 버려야 함 (낭비)

개념 5: Attention Sink — 숨겨진 함정

이 논문이 새롭게 조명한 문제입니다. Transformer는 "각 위치가 다른 모든 위치를 얼마나 참고할지"를 결정하는 Attention이라는 메커니즘을 씁니다. 그런데 이 attention 값은 합이 1이 되도록 정규화됩니다.

🚰 비유: 무엇을 어디에 흘려보낼지 정하는 수도관

Attention을 100L짜리 물통에 물을 나눠 담는다고 생각해보세요. 총 100L를 어떻게든 나눠 담아야 합니다. 문제는, 정말로 참고할 게 없을 때조차 "0L"이 아니라 어딘가엔 부어야 한다는 겁니다. 그래서 모델은 "쓰레기통 역할을 할 특정 토큰 하나"를 학습합니다. 여기에 남는 물을 다 부어버리는 거죠. 이 쓰레기통 토큰을 Attention Sink라고 합니다.

보통 트랜스포머에서는 시퀀스의 첫 번째 토큰이 이 sink 역할을 합니다. Sink 자체는 흔한 현상이지만, GenLIP처럼 이미지와 텍스트를 한 모델에서 함께 처리할 때는 문제가 커집니다. 이미지 정보가 소수의 sink 토큰에 압축되어버리고, 결과적으로 개별 patch feature의 품질이 떨어집니다.

GenLIP의 핵심 아이디어: "ViT가 직접 말하게 하자"

GenLIP의 발상은 매우 단순합니다. 텍스트 디코더도, 대조학습도, 특수 모듈도 없이, 하나의 ViT에게 그림과 텍스트를 이어 붙여 넣어주고 "다음 단어를 맞춰봐"라고 시키는 겁니다. LLM이 문장을 학습하는 그 방식 그대로요.

🎨 비유: 화가에게 말하는 법을 가르치기

지금까지의 접근이 "화가(vision encoder)에게 그림을 그리게 하고, 그것을 옆의 통역사(text decoder)가 설명하게 한다"였다면, GenLIP은 "화가가 직접 말하는 법을 배운다"는 접근입니다. 통역사가 필요 없죠. 화가는 그림을 그리는 사고 회로 안에 언어를 곧바로 통합시킵니다. 그림을 그리는 감각과 설명하는 감각이 한 사람 안에서 조율됩니다.

핵심 트릭 1: Prefix-LM Attention

이미지 부분과 텍스트 부분은 서로 다르게 attention을 걸어야 합니다. 그림은 좌우가 서로를 볼 필요가 있고(위치는 순서가 없으니까), 텍스트는 앞의 단어만 봐야 합니다(다음 단어를 예측할 때 정답을 미리 보면 안 되니까).

이미지끼리는 자유롭게, 텍스트는 앞만, 그리고 텍스트가 이미지 전체를 볼 수 있게

핵심 트릭 2: Multimodal RoPE (MRoPE)

이미지는 2D 좌표(가로, 세로)로 위치를 표현하고, 텍스트는 1D 순서(1번째, 2번째, ...)로 위치를 표현합니다. GenLIP은 이 두 위치 정보를 Rotary Position Encoding이라는 방식으로 자연스럽게 통합합니다. 이미지 patch는 (h, w) 2D 좌표로, 텍스트 토큰은 1D 시퀀스로 각각 인코딩됩니다.

핵심 트릭 3: Gated Attention (Attention Sink 완화)

앞서 설명한 Attention Sink 문제를 해결하는 장치입니다. 각 Transformer 층에서, attention 결과에 sigmoid gate를 곱해서 "얼마나 통과시킬지"를 학습합니다. 텍스트가 sink 토큰 하나에 몰아 넣은 attention을 gate가 차단해버릴 수 있죠.

GenLIP-So/16 모델에서 gated attention 유무에 따른 discriminative 성능

2단계 학습 과정

GenLIP은 두 단계로 나눠 학습합니다:

기초 → 세밀함 순으로 커리큘럼 학습
📚 비유: 초등학교 → 전공심화

Stage 1은 초등학교 교과과정과 비슷합니다. 큰 그림에서 "이건 개, 이건 자동차, 이건 하늘"을 대량으로 훈련시킵니다. Stage 2는 대학 전공심화입니다. 이제 영수증에 적힌 작은 숫자, 차트의 축 레이블, 표의 미세한 정렬 같은 디테일에 집중해서 정밀도를 끌어올립니다. 두 단계 다 필요합니다 — 처음부터 세밀한 것만 훈련하면 큰 그림을 놓치고, 반대면 디테일을 못 봅니다.

흥미로운 발견들

1) LLM 초기화가 오히려 해롭다

비슷한 접근인 SAIL은 이미 훈련된 LLM(Qwen3-0.6B)의 가중치로 시작합니다. "이미 언어를 아는 상태에서 시작하면 더 낫지 않을까?"라는 자연스러운 발상이죠. 하지만 GenLIP 저자들의 통제 실험 결과는 반대였습니다:

방법 LLM 초기화 ALL AVG
SAIL (원본) ✅ Qwen3-0.6B 53.6
SAIL-g (gated attn 추가) ✅ Qwen3-0.6B 54.8
SAIL-g-Scratch (처음부터) ❌ 랜덤 56.0

🎓 비유: 언어 능통자의 "꾀"

이미 말을 잘하는 사람에게 "그림 설명"을 시키면, 그림을 제대로 안 보고 말솜씨로 얼렁뚱땅 넘어가려는 경향이 생깁니다. "이 그림은 자연 풍경 같네요, 어딘가 산과 나무가 있고..." 같은 general한 말로 얼버무리는 거죠. 백지에서 시작한 사람은 그림을 진지하게 관찰해야만 답을 낼 수 있어서, 오히려 더 좋은 visual feature를 배웁니다. Attention pattern 분석에서 SAIL의 텍스트 토큰이 이미지가 아닌 prompt 토큰만 참조하는 shortcut을 학습함이 실증되었습니다.

2) Patch 하나하나가 "단어"와 정렬된다

GenLIP은 명시적으로 patch에 레이블을 붙여준 적이 없습니다. 그런데 학습이 끝난 뒤 개별 patch feature를 LM head에 넣어보면, 놀랍게도 그 patch가 표현하는 개념의 단어가 top 예측으로 나옵니다.

Patch feature를 LM head로 unembedding하면 해당 지역의 개념이 나온다 (emergent 성질)

3) 데이터 스케일링 곡선

학습 데이터를 1B → 8B로 늘렸을 때 성능이 어떻게 변하는지 살펴봤습니다:

데이터를 늘릴수록 좋아지지만 4B 이후 완만해짐. GA는 데이터 적을 때 효과 큼

🎯 비유: 요리 학교의 세 가지 커리큘럼

CLIP 방식은 "요리 사진 보고 이름 맞추기" 훈련. 사진 인식에는 강하지만 요리를 설명하는 법은 배우지 않음.
CapPa 방식은 "요리사(vision encoder)가 만든 걸 옆의 손님(text decoder)이 설명하는" 학습. 요리사가 직접 말하는 훈련은 아님.
GenLIP은 "요리사가 직접 요리하면서 재료·조리법·맛을 설명하는" 통합 훈련. 손 감각과 말 감각이 하나로 통합되어, MLLM이라는 "TV 방송에서 요리하는" 실전에 곧바로 최적화됨.

한계와 앞으로의 과제

📚 학술 세팅 한정 검증

실험은 LLaVA-NeXT라는 학술적 MLLM 프레임워크에서만 이루어졌습니다. GPT-4o나 Gemini 같은 최첨단 상용 MLLM에 GenLIP을 붙였을 때 얼마나 잘 작동할지는 아직 미지수입니다.

📦 데이터 규모 제한

학습 데이터 Dataset-S1은 1B 규모입니다. SigLIP2가 사용한 40B에는 아직 미치지 못하며, 이보다 훨씬 큰 규모(예: 100B, 1T)로 확장했을 때 여전히 gain이 유지될지, 아니면 어느 지점에서 saturation이 올지는 검증되지 않았습니다.

💰 고품질 캡션의 비용

GenLIP은 고품질 image-caption 쌍에 의존합니다. 이런 데이터를 인터넷에서 그냥 긁어오는 것이 아니라, LLM으로 re-captioning하거나 curation을 거쳐 만들어야 합니다. 이 데이터 확보 비용은 학습 데이터 규모를 늘릴 때 무시할 수 없습니다.

🎯 순수 분류·인식은 SigLIP2 아래

ImageNet 이미지 분류에서는 SigLIP2(dense supervision 활용)가 GenLIP보다 앞섭니다(88.9 vs 85.2). GenLIP은 MLLM용 vision encoder에 특화되어 있고, 순수한 vision-only downstream(예: 세밀한 분류, 검색)에는 SigLIP2가 여전히 강력할 수 있습니다.

🧩 Native MLLM에는 부적합

GenLIP은 modular MLLM(Vision Encoder + Connector + LLM 조합)의 vision encoder용으로 설계되었습니다. Chameleon이나 NEO 같은 native MLLM(처음부터 하나의 트랜스포머로 vision과 language를 함께 다루는 방식)에는 그대로 적용되지 않습니다.

한 줄 결론

"복잡한 걸 걷어내고, ViT가 스스로 그림을 보고 말하게 하라.
5분의 1 데이터로도 SOTA를 넘을 수 있다."

기술적 요약: GenLIP은 vision encoder pretraining을 단일 트랜스포머 + 표준 language modeling loss로 극단적으로 단순화합니다. 별도 text decoder나 contrastive learning이 없고, image patch와 text token을 concat한 시퀀스에 다음 단어 예측만 시킵니다. Prefix-LM attention, MRoPE, 그리고 attention sink를 억제하는 gated attention이 이 minimal 설계를 뒷받침합니다.

실용적 의미: MLLM에서 vision 파트를 개선하려는 팀에게 매력적인 선택지. 특히 문서/차트/OCR 응용에서 gain이 크고, 학습 데이터 확보 비용을 5배 절감할 수 있는 잠재력. 단, 순수 이미지 분류나 native MLLM에는 다른 접근이 필요.

연구적 통찰: (1) Objective mismatch의 실증 — pretraining objective가 downstream objective와 얼마나 정렬되는지가 데이터 효율을 결정한다. (2) Attention sink가 vision representation 자체를 열화시킬 수 있음을 발견. (3) LLM initialization이 vision encoder 학습에는 오히려 역효과라는 반직관적 결과.
