SenseNova-U1 — "보는 AI"와 "만드는 AI"를 처음부터 하나로 키운다

Vision encoder도 VAE도 없이 raw pixel을 직접 다루는 native unified multimodal model. GenEval 0.91 (open-source SOTA), CVTG-2K 0.940 (closed-source까지 능가), 텍스트가 들어간 인포그래픽 생성에서 새 기준.

한 줄 요약

"이미지를 보는 능력과 이미지를 만드는 능력을 별개 시스템으로 두면 둘 사이에 항상 trade-off가 생긴다. 두 능력을 하나의 native multimodal process로 키우면 그 trade-off 자체가 사라진다."

왜 이 문제가 중요한가

현재 멀티모달 AI의 dichotomy

현재의 vision-language model은 두 가지 capability를 분리해서 학습한다:

▲ 기존 unified model의 근본 문제: VE와 VAE가 반대 방향으로 학습된 module이라 합쳐도 한 쪽이 약해진다.

SenseNova-U1의 해결 방향

두 module을 다 빼버리고 raw pixel과 word를 직접 다루는 하나의 native architecture로 학습한다.

▲ "보는 능력"과 "만드는 능력"이 처음부터 하나의 모델 안에서 같이 자란다.

배경 지식: 알아두면 좋은 6가지 개념

1) VLM (Vision-Language Model)

이미지와 텍스트를 동시에 받는 large model. GPT-4V, Gemini, Qwen-VL이 대표. 보통 이미지를 텍스트로 답할 수만 있다 (understanding만).

2) Unified Multimodal Model (UMM)

understanding (이미지를 보고 답)과 generation (이미지를 만든다) 두 능력 모두를 한 모델이 한다. BAGEL, Show-o, OmniGen, NEO-unify 등. 보통 두 능력 사이에 trade-off가 있다.

3) Vision Encoder (VE) / CLIP / SigLIP

이미지를 semantic feature vector로 변환하는 모듈. discriminative 학습되어 semantic은 잘 잡지만 fine pixel detail은 손실. 거의 모든 VLM이 의존.

4) VAE (Variational Autoencoder)

이미지를 저차원 latent로 압축, generation 시 그 latent space에서 작업. Stable Diffusion 같은 모델의 표준. 압축 과정에서 semantic 구조가 약해진다.

5) Flow Matching

Diffusion의 진화형. noise → clean의 trajectory velocity를 학습. 더 안정적이고 fewer step으로 생성 가능. FLUX, Stable Diffusion 3가 사용.

6) MoE (Mixture-of-Experts)

전체 파라미터 중 일부 expert만 활성화. Qwen3-30B-A3B = 30B 중 3B 활성. capacity는 늘리되 inference cost는 통제.

핵심 아이디어: 3-Layer Architecture

1Near-Lossless Visual Interface

2-layer convolution (16x16 stride)으로 image를 token으로. VE도 VAE도 없음. 32× compression이지만 pixel detail 보존.

2Native Mixture-of-Transformers

Understanding stream + Generation stream이 모든 layer에서 shared self-attention. parameter는 fully decoupled (token type에 따라 dynamic routing).

3Joint Training Objective

$L = \lambda_1 L_\text{Und} + \lambda_2 L_\text{Gen}$. Autoregressive cross-entropy (text) + pixel-space flow matching (image)을 동시 최적화.

왜 32× compression이 작동하는가

16×16 conv stride = 32×32 pixel patch가 하나의 token으로 압축. 이는 매우 공격적인 압축이지만, deeply correlated learning이 token-level pixel-word correspondence를 효과적으로 보상한다.

📷 비유: 1000만 화소 사진을 1000개 정도의 의미 있는 단어로 요약하는 것과 같다. 각 단어는 픽셀 한 점을 묘사하는 게 아니라 해당 영역의 semantic + texture + structure를 종합한다. 압축률은 32×인데, 그 작은 토큰 안에 pixel detail까지 reconstructable한 정보가 들어있다.

두 stream이 한 sequence에서 interact

Understanding stream은 clean image + text를 처리하고, generation stream은 noise-conditioned input을 처리한다. 둘은 같은 sequence에 함께 있고, 매 layer에서 shared self-attention을 통해 정보를 교환한다.

▲ 같은 attention layer 안에 두 stream이 공존, 각자 다른 attention 패턴.

5단계 학습 레시피

처음부터 한 번에 학습하는 게 아니라, 점진적으로 capability를 쌓는다.

1Understanding Warmup

NEO에서 초기화. 0.75T tokens로 vision-language understanding backbone 구축. CE:MSE = 1:0.

2Generation Pre-Training

3 phases. Understanding branch freeze, generation을 pixel-space flow matching으로 학습. Phase III에서 editing + interleaved data 추가. 1.38T tokens.

3Unified Mid-Training

두 branch 동시 학습. 1.19T tokens로 joint convergence. CE:MSE = 0.1:1 (generation이 dominate).

4Unified SFT

High-quality instruction data로 9K steps. instruction alignment + task-specific sharpening.

5Post-Training

Flow-GRPO RL + Distribution Matching Distillation (DMD2). NFE 100→8로 효율화.

누적 3T+ training tokens. 학원 규모에서는 거의 재현 불가능한 scale.

🌱 비유: 아이를 키우듯이 단계별로 능력을 쌓는다. 먼저 보는 법 (Stage 1), 그 다음 그리는 법 (Stage 2), 그 다음 둘을 함께 (Stage 3), 그리고 섬세한 지시 따르기 (Stage 4), 마지막으로 효율적으로 빨리 그리기 (Stage 5).

가장 흥미로운 발견 3가지

① 32× compression에서도 near-lossless reconstruction

NEO-unify (2B) reconstruction이 MS-COCO에서 PSNR 31.56, SSIM 0.85. FLUX VAE의 PSNR 31.56, SSIM 0.93에 근접. VAE 없이도 fine pixel detail이 보존된다는 놀라운 결과.

② CoT가 generation에도 큰 효과

WISE benchmark (science-oriented generation)에서 U1-SFT 8B w/o CoT 0.69 → w/ CoT 0.78. RISEBench Logical dimension이 7.1 → 20.0. explicit reasoning이 complex edit goal decomposition을 돕는다.

③ Data scaling이 단조 향상

Training token 0→2.5T로 갈수록 T2I-Reasoning(WISE)이 40 → 67로 단조 증가. Editing-Reasoning(RISEBench)도 11 → 20. native MoT 아키텍처가 data scale에 잘 반응한다는 검증.

▲ Training token 증가에 따른 단조 향상. native 아키텍처가 scale-friendly임을 증명.

Disaggregated Inference: 두 engine으로 서빙

Unified model이지만 실제 inference 시에는 두 engine으로 분리해 서빙한다.

🧠 LightLLM

Understanding + text streaming + control flow. LLM-oriented tensor parallelism.

🎨 LightX2V

Image generation. Diffusion-oriented (CFG parallelism, sequence parallelism).

🔗 Pinned Shared Memory

두 engine 사이 generation state 교환. unified API abstraction을 유지하면서도 두 path를 독립 최적화.

🍽 비유: 한 셰프(unified model)가 앞 주방(LightLLM)과 뒷 주방(LightX2V)을 모두 운영하지만, 실제 조리 시에는 두 주방을 다른 인프라(스토브 vs 오븐)로 분리하고 패스 데스크(shared memory)로 연결한다. 손님(API)은 한 명의 셰프를 부르지만, 내부는 specialized.

Step Distillation: 100 → 8 NFE

Post-training에서 Distribution Matching Distillation (DMD2)로 생성 step 수를 100에서 8로 줄인다. production-ready efficiency. Generator G가 fake flow F의 5 update마다 1번 update되는 distillation scheme.

한계점

① 학습 cost가 어마어마하다

총 3T+ training tokens. GPU hour 비용과 사용 자원이 명시되지 않음. 학원 규모에서는 거의 재현 불가능.

② 일부 OCR task에서는 VE 기반 모델이 우세

InfoVQA에서 Qwen3VL-8B 86.00 vs U1-8B 82.46. fine-grained text-in-image OCR에서는 VE의 강점이 남아있다.

③ 100+ contributor 단체 저작

individual ablation에 대한 깊이 있는 분석이 부족. peer review의 책임 추적이 어렵다.

④ Complex commercial scenario에서 closed-source와 큰 격차

BizGenEval에서 U1 39.7 / 61.7 vs GPT-Image-2 82.1 / 92.5. complex layout + text + brand identity가 결합된 영역에서는 still gap.

⑤ Image editing이 specialized model 대비 약함

Qwen-Image-Edit-2511 (20B) 4.51 vs U1-8B 3.90. unified model의 본질적 trade-off일 수 있음.

⑥ VLA / World Modeling 결과가 preliminary

Vision-language-action과 world modeling 능력은 qualitative 시범 수준. quantitative benchmark 없음.

한 줄 결론

Vision encoder와 VAE의 trade-off는 엔지니어링 artifact가 아니라 design choice였다.
둘 다 빼버리고 raw pixel-word에서 시작하면, 보는 능력과 만드는 능력이 서로를 강화한다.
SenseNova-U1은 그 가설을 8B/30B 스케일에서 처음 입증했다.

가져갈 인사이트 3가지

  • "Pretrained module의 prior는 양날의 검": CLIP, VAE 같은 잘 작동하는 module은 편한 출발점이지만 그 inductive bias가 unified model의 진짜 잠재력을 막는다. 처음부터 native하게 학습할 용기가 next paradigm을 연다.
  • "Compression과 fidelity는 zero-sum이 아니다": 32× compression에서도 PSNR 31.56 보존. 압축률이 높을수록 detail이 잃는다는 직관이 deep learning에서는 틀린 경우가 있다 — 모델이 충분히 크고 데이터가 충분하면.
  • "Bidirectional synergy는 측정 가능하다": Uni-MMMU GaU 같은 benchmark가 generation이 understanding을 돕는지를 정량 측정. unified model은 단순히 두 능력을 같이 둔 게 아니라 진짜 synergize시켜야 한다.
