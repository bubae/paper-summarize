TL;DR

  • 문제: 대규모 LLM은 높은 정확도를 달성하지만, inference 비용과 throughput이 실용적 배포의 병목이다. 기존 MoE 설계는 FLOP 효율에 초점을 맞추지만, memory bandwidth, 통신 오버헤드, 실제 하드웨어 제약을 간과하여 실질적 비효율이 발생한다.
  • 해결: Nemotron 3 Super — 120B total / 12B active parameter의 hybrid Mamba-2 + LatentMoE + Attention 아키텍처. LatentMoE로 expert 연산을 latent space에서 수행하여 accuracy-per-byte를 최적화하고, shared-weight MTP (Multi-Token Prediction)로 native speculative decoding을 지원하며, NVFP4로 25T 토큰을 안정적으로 사전학습한다.
  • 왜 작동하는가: (1) LatentMoE가 hidden dimension을 줄이는 대신 expert 수와 top-K를 비례 증가시켜 동일 inference cost에서 더 높은 accuracy를 달성, (2) Mamba-2 블록의 linear-time 처리 + 소수의 attention anchor로 1M context를 효율적으로 지원, (3) 21개 도메인에 걸친 multi-environment RL + PivotRL로 agentic 능력을 체계적으로 강화.
  • 비유: 고급 레스토랑 주방에서, 기존 MoE가 모든 셰프에게 큰 도마(hidden dim)를 주는 대신, LatentMoE는 작은 도마를 주되 셰프 수를 대폭 늘린다. 각 셰프가 처리하는 양은 줄지만 더 다양한 전문 셰프가 요리에 참여하여 전체 품질이 향상된다.

핵심 기여 (Key Contributions)

  1. LatentMoE — 하드웨어 효율적 MoE 아키텍처: ##Latent space에서 expert 연산을 수행하여 memory bandwidth와 통신 비용을 줄이면서, 절약분으로 expert 수(512)와 top-K(22)를 확대##하여 동일 inference cost에서 더 높은 accuracy를 달성한다.
  2. Shared-weight MTP로 robust speculative decoding: 독립 offset head 대신 ##공유 파라미터 MTP head##를 사용하여 draft 길이 증가 시에도 안정적인 acceptance rate을 유지 (SPEED-Bench 평균 3.45, DeepSeek-R1 2.70 대비).
  3. NVFP4 대규모 사전학습: ##25T 토큰을 NVFP4 정밀도로 안정적으로 사전학습##한 최초의 120B급 모델. Zero-valued weight gradient 현상을 분석하고 BF16 대비 성능 저하 없음을 확인.
  4. Agentic 능력 중심 post-training: 21개 환경 × 37개 RL dataset으로 multi-environment RLVR + SWE-RL + RLHF + MTP healing 4단계 파이프라인. PivotRL로 agentic RL의 효율성을 크게 개선.
  5. GPT-OSS-120B 대비 2.2배, Qwen3.5-122B 대비 7.5배 throughput: 동등 이상의 accuracy에서 inference throughput을 크게 상회 (8K input / 64K output 기준, B200 GPU).

2.1.1 LatentMoE — Hardware-Aware Expert Design

기존 MoE 설계의 5가지 원칙을 도출하여 LatentMoE를 설계한다:

  1. Low-latency serving: Memory bandwidth가 expert weight 읽기에 지배 → expert FFN의 hidden dimension $d$를 줄여야 한다
  2. Throughput serving: All-to-all routing이 $d \times K$에 비례 → $d$ 또는 $K$를 줄여야 한다
  3. Quality 보존: 유효 비선형 예산 $K \cdot m$은 유지해야 한다 → $K$와 $m$을 고정
  4. Feature rank: 과제별 effective feature rank $r_{\text{eff}}$가 $d$의 하한 → $d$를 과도하게 줄이면 성능 붕괴
  5. Expert 조합 확장: $N$과 $K$를 동시에 늘리면 expert 조합 공간이 지수적으로 확장되어 품질 향상

##핵심 아이디어##: 원칙 (1)-(3)에서 hidden dimension $d$가 가장 유망한 축소 대상이고, 원칙 (5)에서 절약분으로 $N$과 $K$를 $d/\ell$만큼 증가시키면 동일 inference cost에서 더 높은 accuracy를 달성할 수 있다.

LatentMoE 작동 방식: 입력 토큰 $x \in \mathbb{R}^d$를 learnable down-projection $W_\downarrow \in \mathbb{R}^{\ell \times d}$로 latent space $\mathbb{R}^\ell$에 투영 → latent space에서 routing 및 expert 연산 수행 → up-projection $W_\uparrow \in \mathbb{R}^{d \times \ell}$로 원래 dimension $d$로 복원. Routing gate, shared expert, non-expert layer는 원래 dimension $d$에서 작동하여 품질을 보존한다.

Nemotron 3 Super에서: $d = 4096$, $\ell = 1024$ (4배 축소). Expert 수를 128에서 ##512로 4배##, top-K를 5에서 ##22로 ~4배## 확대하여 동일 비용에서 더 풍부한 expert 조합을 제공한다.

2.1.2 Multi-Token Prediction (MTP)

MTP는 각 position에서 다음 여러 토큰을 동시에 예측하는 auxiliary objective이다. (1) Multi-step dependency를 포착하여 representation 품질 향상, (2) 예측 head가 native speculative decoding의 draft model 역할.

Shared-weight MTP head 설계: 기존 MTP는 각 offset별 독립 head를 학습하지만, inference 시 self-generated hidden state에 조건화되면 train-inference mismatch가 발생하여 acceptance rate이 draft 길이 증가 시 급격히 저하된다. Nemotron 3 Super는 ##여러 MTP head가 파라미터를 공유##하여, 각 head가 multiple offset에 노출되어 regularization 효과를 얻고, autoregressive drafting 시 self-generated state에 대한 robustness가 향상된다.

2.1.3 Hybrid Interleaved MoE Architecture

88-layer stack에서 Mamba-2 블록이 주를 이루고, 소수의 self-attention layer가 "global anchor"로 삽입된다. Mamba-2는 constant-size state로 linear-time sequence modeling을 제공하여 memory overhead와 latency를 대폭 줄이고, attention anchor가 full-token interaction과 long-range information routing을 가능하게 한다. 이 조합으로 1M context length를 지원한다.

2.2 NVFP4 Pretraining

대부분의 linear layer를 NVFP4 (E2M1 format, 16-element micro-block, E4M3 micro-block scaling, FP32 global scale)로 학습한다. 안정성을 위해 일부 layer는 BF16 또는 MXFP8로 유지:

Layer TypeFormatRationale
All Linear (default)NVFP4–
Final 15% of NetworkBF16학습 안정성
Latent ProjectionsBF16Step-time 영향 무시할 수준
MTP LayersBF16Multi-token prediction 보존
QKV & Attention ProjectionsBF16소수 attention layer의 fidelity
Mamba Output ProjectionMXFP8NVFP4에서 underflow 관찰
EmbeddingsBF16–

Zero-valued gradient 분석: NVFP4 학습 시 weight gradient의 7%가 zero-valued로 수렴하는 현상 관찰. 이는 expert layer의 FC1 output channel과 FC2 input channel에서 magnitude pattern이 형성되어 발생하며, NVFP4의 low precision이 small-magnitude gradient를 underflow시키기 때문이다. 그러나 BF16으로 전환해도 이미 유사한 magnitude pattern이 존재하므로, ##NVFP4가 BF16의 자연스러운 학습 동역학을 가속할 뿐 새로운 문제를 도입하지 않는다##.

2.3 Pretraining Data

25T 토큰, 16개 high-level category, 2-phase curriculum. Nemotron-CC taxonomy (Su et al., 2025)에 따라 web crawl을 5단계 품질로 분류하고, 합성 대응물과 함께 배합한다.

  • Phase 1 (80%, 20T): 다양성과 broad coverage 중심. crawl-medium, crawl-medium-high, crawl-high + syn-crawl variants, nemotron-cc-code, code-sft, general-sft, multilingual, crawl++, academic, code, math, finepdfs 등
  • Phase 2 (20%, 5T): 고품질 데이터와 benchmark accuracy 중심. Wikipedia, code-sft, stem-sft, crawl-high 비중 대폭 증가. Crawl++ = OpenWebText, BigScience, Reddit datasets
[Figure 10 설명]: Phase 1과 Phase 2의 데이터 혼합 비율 pie chart. Phase 1에서는 syn-crawl-high (22.4%)가 최대 비중이고 crawl-medium, nemotron-cc-code, general-sft 등이 고르게 분포. Phase 2에서는 syn-crawl-high (22.4%)와 crawl-high (8.0%), code (14.2%), math (7.4%)가 강화된다.

2.4 Hyperparameters

  • WSD (Warmup-Stable-Decay) LR schedule: peak $4.5 \times 10^{-4}$, 200B tokens warmup
  • Stable plateau phase 후, 최종 5T tokens에서 minus-sqrt decay로 $4.5 \times 10^{-6}$까지 annealing
  • AdamW optimizer: $\beta_1=0.9$, $\beta_2=0.95$, weight decay 0.1
  • Batch: 3,072 sequences × 8,192 tokens = 25.17M tokens/batch
  • MoE: 512 experts, top-22 routing, sigmoid router score + expert biasing
  • Load balancing: auxiliary-loss-free strategy (update rate $10^{-3}$) + standard load balancing loss ($10^{-4}$)
  • MTP objective: loss scaling factor 0.3
  • Hybrid precision: BF16 + NVFP4 혼합

2.5 Tracking Merge Evaluation

WSD schedule의 stable phase에서는 LR이 일정하여 개별 checkpoint의 벤치마크 성능이 noisy하다. Checkpoint merging (sliding window weighted average)을 적용하여, 별도 LR decay run 없이도 안정적인 모델 품질 readout을 얻는다.

  • Merge window: {125B, 250B, 500B} tokens, 1,000 iteration마다 checkpoint 저장 (≈25B tokens/checkpoint)
  • 12개 벤치마크 (MMLU-Pro, MMLU, HumanEval, HumanEval+, MBPP, MBPP+, GSM8K, MATH-500, RACE, ARC-Challenge, HellaSwag, WinoGrande) 평균에서 merged checkpoint가 trained checkpoint 대비 ##일관되게 2-4pp 향상##
  • LR decay phase (20T→25T)에서는 gap이 좁아져 수렴. 짧은 annealing horizon에서 merge가 가장 효과적
  • ##~16% pretraining FLOP 절약## (별도 decay run 불필요: ~4T tokens 분량의 compute 절감)
[Figure 11 설명]: 25T 학습 전체에 걸친 trained checkpoint (파란색) vs best offline merge (주황색)의 12-benchmark 평균. Stable phase에서 merge가 2-4pp 일관적 우위. LR decay phase (음영 영역)에서 gap이 좁아진다.

2.6 Long-Context Extension

사전학습 말미에 long-context phase (LC-Phase)를 추가하여 1M context를 지원한다:

  • Stage 1: Continuous pretraining (CPT)으로 1,048,576 (1M) context length에서 34B tokens 학습. LR $4.5 \times 10^{-6}$, global batch 16, 64-way context parallelism + 2-way TP + 64-way EP, GB200 GPU
  • Stage 2: 1M과 4K 시퀀스를 교대로 학습 (17B tokens). Stage 1에서 관찰된 수학 벤치마크 소폭 하락을 완화하기 위함
  • LC data blend: 20% document QA (Nemotron 2 & 3 Nano에서 재사용) + 80% downscaled Phase 2 data

3. Post-Training

Base (1M context) → SFT (7M samples, 8GB tokens) → RLVR Round 1 (25 env types) → RLVR Round 2 (30 env types, + Low Effort) → RLVR Round 3 (26 env types, + Agentic Focused) [37 environments total, up to 4,000 instances/batch] → MTP Healing (18B tokens) → RLHF (18B tokens) → SWE-RL (20B tokens) → Nemotron 3 Super

3.1 Supervised Fine-Tuning

Two-stage SFT Loss

Single-stage SFT가 long-input-short-output 시나리오에서 성능 저하를 유발하므로 2-stage 접근을 채택:

  • Stage 1: Token-level global average: $\mathcal{L}_{\text{tok}} = \frac{\sum_{c \in \mathcal{B}} \sum_{t \in \mathcal{O}_c} \ell_t}{\sum_{c \in \mathcal{B}} |\mathcal{O}_c|}$ — 모든 output token에 대한 평균 loss (Eq. 1). 강한 reasoning 행동을 유도.
  • Stage 2: Sample-level average: $\mathcal{L}_{\text{samp}} = \frac{1}{|\mathcal{B}|} \sum_{c \in \mathcal{B}} \left(\frac{1}{|\mathcal{O}_c|} \sum_{t \in \mathcal{O}_c} \ell_t \right)$ — 각 conversation을 output-token count로 정규화 후 평균 (Eq. 2). Long output이 loss를 지배하는 것을 방지.

SFT Training Configuration

  • Stage 1: 256K sequence length packing, global batch 64, constant LR $1 \times 10^{-5}$, 30K warmup samples
  • Stage 2: 512K sequence length packing (256K/512K long context data 포함), global batch 32, constant LR $1 \times 10^{-5}$
  • MTP during SFT: 사전학습에서 사용한 shared-weight MTP head를 유지하여 SFT 중에도 2개 MTP layer를 학습. Auxiliary loss = per-token loss × 0.3 scaling factor

3.1.3 Reasoning Control

세 가지 reasoning mode를 지원: reasoning-on (full thinking), reasoning-off (no thinking), low-effort (짧은 thinking). Low-effort mode는 GPT-OSS-120B의 low-effort 출력으로 SFT data의 2%를 구성. Reasoning-off는 3% 샘플에서 thinking trace를 strip. Inference-time budget control을 위해 main SFT 후 350 steps의 semi-on-policy SFT 추가 (모델 rollout 수집 후 12% reasoning trace를 random 길이로 truncate).

3.2 Reinforcement Learning

Stage 1: Multi-environment RLVR

##21개 환경, 37개 RL dataset##에 걸쳐 통합 학습. Single-environment RL이 다른 벤치마크에서 심각한 regression을 유발하므로, 모든 환경을 동시에 학습하여 안정적 개선을 달성한다. 3 rounds로 진행:

  • Round 1: 25개 environment types
  • Round 2: 30개 environment types + Low Effort reasoning mode 추가
  • Round 3: 26개 environment types + Agentic Focused 강화

RLVR 도메인 상세:

  • Math: 경쟁 수학 문제, Python execution tool 유무 두 설정. 새로운 formal proof verification 환경 추가
  • Code: Nemotron 3 Nano + Nemotron-Cascade code data (Wang et al., 2025a)
  • STEM: Nano의 STEM dataset + 더 어려운 scientific problems 추가 큐레이션
  • Instruction Following: Nano data + new multi-challenge dataset (복잡한 user instruction을 predetermined rubric으로 평가)
  • Safety: (1) over-refusal 감소 환경, (2) jailbreak robustness 환경. PAIR (Chao et al., 2024) attack pipeline으로 초기 SFT checkpoint을 공격하여 high attack success rate 프롬프트를 수집하고 RL seed로 사용
  • Long Context: Nemotron 3 Nano와 동일한 long-context 환경
  • Agentic Tool Use: Nano의 tool-use environments + conversational tool use 및 terminal use 전용 환경 추가
  • Reasoning Gym (Stojanovski et al., 2025): 다양한 reasoning task를 포괄하는 환경

Low-effort reasoning: RL prompt 중 초기 2% → 후기 1%를 low-effort mode로 변환. Math, STEM QA, competitive coding의 subset을 대상으로 하며, reward는 correctness와 generated token 수의 함수로 조정. Competitive coding에서는 SFT에서 withheld된 별도 문제 set을 사용.

3.2.4 Asynchronous GRPO Algorithm

Training과 inference가 분리된 비동기 설정:

  • Inference workers가 지속적으로 trajectory를 생성하여 rollout buffer에 저장
  • 충분한 trajectory 수집 시 training engine에 batch 전송 → single gradient update
  • Weight update 즉시 inference workers에 push (in-flight weight update). 진행 중인 rollout 완료를 기다리지 않음
  • 단, 과도한 policy lag 방지를 위해 inference workers는 최대 1-step 이전 모델까지만 허용
  • Importance sampling ratio masking으로 train-inference mismatch와 policy lag의 off-policy 효과를 안정화
  • Multi-environment RLVR: 256 prompts/step × 16 rollouts = batch 4,096. Max generation 49K→64K tokens

PivotRL — Agentic RL 효율화

Long-horizon agentic task(conversational tool use, terminal, SWE)에서 end-to-end RL은 매 update마다 online interactive rollout이 필요하여 비용이 매우 높다. SFT는 저렴하지만 OOD generalization이 약하다. ##PivotRL은 이 trade-off를 해결하는 assistant-turn-level RL 방법##이다:

  1. Offline SFT expert trajectory에서 "pivots" — 현재 policy가 불확실한 informative assistant turn — 을 식별
  2. 이 pivot turn에서만 RL을 수행하여, expert의 전체 trajectory를 재실행하지 않고도 학습
  3. Domain-appropriate reward를 사용하여 expert action과 정확히 일치하지 않아도 유사한 action에 credit을 부여

PivotRL을 Agentic Programming, Search, Terminal Use, Conversational Tool Use 전 agentic 도메인에 적용. End-to-end agentic RL의 OOD 문제 없이 효율적으로 agentic 능력 강화.

Stage 2: SWE-RL

Software engineering 능력 전용 RL. SWE rollout은 생성이 느리고 context가 길어 (최대 256K) 다른 환경과 co-training 시 throughput bottleneck이 되므로 별도 stage로 분리.

  • 환경: Apptainer (formerly Singularity) container에서 target repo + OpenHands agent loop 실행. .sif pre-built image로 filesystem isolation
  • Agent loop: Modified OpenHands — runtime 초기화 → problem statement 제시 → configurable turn limit → git patch 추출 → cleanup
  • Reward: Ground-truth test에 대한 binary pass/fail
  • Harness diversity: OpenCode, Codex agent class를 OpenHands 내에 구현하여 Claude Code/Codex CLI의 tool I/O format을 매칭. 학습 중 harness를 변경하면서 단일 환경에서 multi-harness 학습
  • Infrastructure: Memory watchdog (RSS 모니터링 + OOM 방지), command blocklist (kill/pkill 차단), orjson serialization (대용량 trajectory payload 최적화)
  • Scaling: Ray + SPREAD scheduling으로 수백 개 concurrent agent-codebase interaction을 분산 실행

Stage 3: RLHF

GenRM (Generative Reward Model)을 학습하여 instruction-following, robustness, interaction quality를 개선:

  • GenRM 학습: Qwen3-235B-A22B-Thinking-2507에서 초기화. Principled GenRM (Wang et al., 2025b) 접근 — identity/safety 등 중요 도메인에서 모델 행동을 guide
  • 데이터: HelpSteer 3 (Wang et al., 2025c), lmarena-140k commercially friendly subset, 최근 수집 human preference data
  • 적용: Multi-environment RL 전체에서 GenRM을 함께 사용 (RLVR rounds에서도) + 마지막에 별도 RLHF-only stage 수행

Stage 4: MTP Healing

RLVR 이후 MTP head의 정확도가 저하된다 — RL이 main model weight만 최적화하고 MTP head는 frozen이 아니지만 RL objective에 직접 포함되지 않기 때문. 이를 복구하기 위해:

  • ##모델 본체(backbone)를 frozen##하고 MTP head만 학습 가능하게 설정
  • RLVR에서 사용된 prompts에서 현재 모델이 생성한 응답을 수집
  • 이 (prompt, generated response) pairs에 대해 negative log-likelihood loss로 MTP head를 SFT
  • 18B tokens 학습, MTP accuracy를 유의미하게 회복

3.2.5 Infrastructure — Large-Scale Async RL

NeMo Gym + NeMo RL 통합 인프라:

  • NeMo Gym: Agent servers (rollout kernel), Model servers (vLLM inference engine), Resource servers (reward verification API)의 3가지 서버 추상화
  • NeMo RL: Megatron-Core 기반 training loop controller. 모든 rollout은 NeMo Gym + vLLM을 통해 routing
  • Deployment: Ray cluster에서 training workers, vLLM generation workers, Gym environments, judge models를 모두 orchestration
  • Resiliency (1K GPU scale): (1) Port conflict 해결 — TOCTOU race condition으로 인한 port binding failure → 해결, (2) Startup 최적화 — 병렬 초기화, 가상환경/바이너리 프리페칭, upstream repo 캐싱, (3) Checkpoint recovery — intermittent hardware/software failure에서의 자동 복구

4.2 NVFP4 (W4A4) Checkpoint

NVFP4는 prefill-heavy workload (coding-agent 등)에서 linear/MoE GEMM이 주요 bottleneck이므로 특히 매력적. Blackwell GPU에서 native NVFP4 가속.

NVFP4 inference format: Signed E2M1 values, 1D blocks (size 16), per-block FP8 E4M3 scale + per-tensor FP32 global scale.

Hybrid FP4 PTQ recipe (baseline 대비 개선):

  • Weight scaling: Max-based가 아닌 MSE-minimizing per-block scale (calibrated offline → 비용 무시 가능)
  • Activation scaling: Dynamic per-block max-based (runtime에서 효율적 계산)
  • Selective promotion: AutoQuantize (NAS-inspired)로 per-operator sensitivity를 추정하고, NVFP4/FP8/BF16 중 최적 precision을 knapsack-style optimization으로 할당. Effective precision budget 4.75 bits

AutoQuantize 결과 (Table 7):

  • Sparse Expert (Routed) GEMM: NVFP4 (전체)
  • Shared Expert GEMM: NVFP4 / FP8 / BF16 혼합
  • MoE Latent Projection GEMM: FP8 / BF16 혼합
  • Attention/Mamba Projection GEMM: FP8 / BF16 혼합
  • Router: FP32 (고정)

Full mixed-precision PTQ: 단일 B200 node (8 GPU)에서 512 SFT samples, seq_len 4096으로 2시간 미만 완료. ##결과 모델은 BF16 대비 99.8% median accuracy##.

4.3 Mamba State Quantization — Stochastic Rounding

Memory-bound decoding에서 Mamba SSM cache (state cache)의 DRAM read가 주요 bottleneck이 된다. SSM cache는 기본 FP32로 저장되며, FP16으로 cast하면 메모리를 절반으로 줄일 수 있다.

문제: Mamba의 recurrent state update $h_t = A_t h_{t-1} + B_t x_t$에서, 양자화 error $e_t$가 이전 step의 error를 전파하여 누적된다:

$$h_{q,t} = h_t + \sum_{i=0}^{t} \left(\prod_{j=i+1}^{t} A_j\right) e_i \quad (3)$$

이 누적은 단일 step의 local error와 달리 시간에 따라 증가하며, 특히 ##FP32→FP16 cast 시 RTNE (Round to Nearest, Ties to Even) rounding이 deterministic bias를 도입##한다. 동일 input에 대해 항상 같은 방향으로 rounding되므로, recurrent 구조에서 이 bias가 coherently 누적되어 systematic drift를 유발한다.

증상: SSM cache를 FP16으로 cast하면 ##코드 생성에서 최대 40% verbosity 증가## (Table 8). Weight/activation이 BF16인 경우에도 SSM cache FP16만으로 37% verbosity 증가.

해결: Stochastic rounding: RTNE 대신 stochastic rounding을 적용하면, 각 rounding이 확률적으로 위/아래로 이루어져 ##expectation에서 unbiased## — systematic drift가 zero-mean noise로 대체된다.

비판적 분석 (Critical Analysis)

강점

  • 완전한 기술 공개: 아키텍처 세부사항, NVFP4 학습 분석, zero-gradient 현상, Mamba state quantization의 stochastic rounding까지 매우 상세한 기술 보고서.
  • LatentMoE의 원리적 설계: 5가지 MoE 설계 원칙을 명시하고, 이를 기반으로 LatentMoE를 도출하는 과정이 체계적. Hardware-software co-design 관점이 인상적.
  • Throughput 이점이 실질적: 정확도만이 아니라 실제 B200 GPU에서의 throughput 비교를 제시하여, 실용적 가치를 명확히 보여준다.
  • Mamba state quantization 분석: Recurrent state에서의 error 누적을 수식으로 분석하고, stochastic rounding으로 해결한 것은 hybrid Mamba-Transformer 양자화의 중요한 기여.

약점

  • LatentMoE의 standalone ablation 부재: Standard MoE vs LatentMoE의 controlled comparison (동일 active param budget)이 본 보고서에 없다. Elango et al. (2026) 별도 논문에 위임.
  • Agentic 벤치마크에서 Qwen3.5 대비 열세: SWE-Bench 60.47 vs 66.40, TauBench 61.15 vs 74.53. "Agentic reasoning"을 강조하지만, 가장 중요한 agentic 벤치마크에서 경쟁 모델에 뒤진다.
  • Post-training 복잡도: SFT (2-stage) + RLVR (3 rounds, 21 env) + MTP healing + RLHF + SWE-RL의 6+ 단계 파이프라인. 재현 난이도가 높고, 각 단계의 기여를 분리하기 어렵다.
  • PivotRL 상세 미공개: "We will have a manuscript with more details soon"으로 핵심 기법의 상세가 미공개.
  • NVFP4 pretraining의 명확한 이점 미입증: NVFP4로 학습한 것이 BF16 대비 어떤 구체적 이점(학습 속도, 비용)을 제공하는지 정량적으로 비교되지 않음.

Key Takeaways

  1. LatentMoE는 MoE 효율성의 새로운 방향: Hidden dimension을 축소하고 expert 수/top-K를 확대하는 전략이 동일 inference cost에서 더 높은 accuracy를 달성한다. 이는 ##accuracy per FLOP가 아닌 accuracy per byte/per-parameter##를 최적화하는 것이 실제 deployment에서 더 중요하다는 통찰을 제공한다.
  2. Shared-weight MTP는 speculative decoding의 실용적 해법: 별도 draft model 없이 3.45 average acceptance length를 달성하여, 추가 메모리/계산 없이 inference를 가속한다.
  3. Multi-environment RL이 single-environment보다 안정적: 21개 환경을 동시에 학습하면 개별 벤치마크의 regression을 방지하면서 전반적 성능을 향상시킨다. Cascade RL (Nemotron-Cascade 2)과는 다른 접근이지만 유사한 목표를 추구한다.
  4. Mamba의 recurrent state quantization은 특별한 주의가 필요: Standard rounding(RTNE)의 bias가 recurrent 구조에서 누적되어 verbosity 증가를 유발하므로, ##stochastic rounding이 hybrid Mamba-Transformer의 양자화에서 필수적##이다.
  5. Two-stage SFT loss가 reasoning과 long-context를 동시에 지원: Token-level average로 reasoning을 강화한 뒤, sample-level average로 long-input-short-output 성능을 복구하는 전략은 다른 모델에도 적용 가능하다.
