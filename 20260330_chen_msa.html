TL;DR

  • 문제: LLM의 long-term memory를 수억 토큰 규모로 확장하는 것이 인간 수준의 평생 기억을 위해 필요하지만, full attention은 $O(L^2)$로 불가능하고, linear attention(RWKV, DeltaNet)은 fixed-size state에 의한 catastrophic forgetting, RAG는 end-to-end 학습 불가와 shallow semantic matching, KV-centric 방법(DSA, MemGen)은 100M 규모에서 비용 폭발이 문제이다.
  • 해결: MSA (Memory Sparse Attention) — 문서별 독립 KV cache를 chunk-wise mean pooling으로 압축하고, Router Projector로 top-k 문서를 선별하여 sparse attention을 수행하는 end-to-end trainable latent memory framework. Document-wise RoPE로 문서 수에 대한 position 독립성을 확보하여 학습 시 소수 문서에서 inference 시 수백만 문서로 외삽 가능하다.
  • 왜 작동하는가: (1) Sparse routing이 retrieval과 generation을 하나의 differentiable pipeline으로 통합하여 end-to-end 학습이 가능하고, (2) document-wise RoPE가 문서 수로부터 positional semantics를 분리하여 length extrapolation을 보장하며, (3) KV cache compression이 memory footprint를 대폭 줄여 100M 토큰을 2×A800 GPU에서 처리 가능하게 한다.
  • 비유: 도서관에서 책을 찾을 때, RAG는 사서(retriever)에게 부탁하여 책을 가져오지만 사서가 이해하지 못하는 맥락이 있을 수 있다. MSA는 직접 모든 책의 목차(압축된 routing key)를 훑어보고 관련 책을 골라 펼쳐 읽는(sparse attention) 것과 같다. 목차를 보는 것은 빠르고($O(L)$), 실제로 읽는 분량은 일정하므로 도서관 크기에 관계없이 일정 시간이 걸린다.

핵심 기여 (Key Contributions)

  1. Scalable sparse attention architecture with document-wise RoPE: ##End-to-end trainable하면서 16K에서 100M 토큰까지 9% 미만의 성능 저하##로 확장 가능한 memory model. 64K context에서 학습하고 100M에서 inference할 수 있는 length extrapolation 능력.
  2. KV cache compression + Memory Parallel: Chunk-wise mean pooling으로 KV cache를 압축하고, routing key만 GPU에, content KV는 CPU에 저장하는 tiered storage + distributed scoring으로 ##2×A800 GPU 단일 노드에서 100M 토큰 inference 실현##.
  3. Memory Interleave: Multi-hop reasoning을 위한 adaptive iterative retrieval-generation 메커니즘. 모델이 document ID를 생성 → 해당 문서를 context에 추가 → 다시 ID 생성을 반복하여, 분산된 증거를 점진적으로 통합한다.
  4. SOTA 결과: 9개 QA 벤치마크에서 same-backbone RAG 대비 평균 16.0% 향상, SOTA RAG 대비 7.2% 향상. RULER NIAH에서 1M 토큰에서 94.84% accuracy, frontier LLM과 external memory system 모두를 상회.

1. Introduction — 배경과 동기

인간 수준의 평생 기억이 필요한 이유

인지과학 연구에 따르면 인간의 기능적 정보 용량은 $10^9$ bits 수준이며, 토큰당 3-5 bits의 semantic density를 가정하면 이는 ##약 200-300M 토큰##에 해당한다. 그러나 현재 LLM의 effective context length는 128K-1M 토큰에 불과하며, 명시적으로 long context를 위해 설계된 아키텍처(Qwen2.5-14B-1M, Qwen3-30B/80B-A3B)도 1M 토큰을 거의 넘지 못한다. Digital Twins, 대규모 소설 이해, multi-agent 장기 히스토리 관리 등의 시나리오에서 이 격차가 치명적이다.

기존 방법의 두 가지 근본 한계:

  1. Limited scalability of high-fidelity memory: 높은 정밀도를 제공하는 방법(parameter-based, KV-centric sparse attention)은 고정된 context/state 크기에 의해 제한되고, 확장 가능한 방법(RAG)은 정밀도가 중간 수준에 머무른다.
  2. Lack of end-to-end trainability: RAG의 retrieval pipeline은 generation objective와 분리되어 있어, retrieval metric과 downstream task 간 optimization gap이 존재한다.

2. Related Work

Parameter-Based Memory

LoRA/CPT로 knowledge를 parameter에 직접 주입하는 방법. MLP-Memory (parametric retriever → differentiable memory store), FLEXOLMO (MoE로 targeted knowledge integration), Engram (N-gram embedding으로 sparse memory), Titans/Nested Learning (test-time training으로 inference 중 weight 업데이트). 그러나 catastrophic forgetting, 복잡한 memory management, 용량 제한이 근본 한계.

External Storage-Based Memory

RAG (vector similarity 기반 retrieval)가 대표적. 한계: discrete semantic representation(raw text/embeddings)에 의존하여 end-to-end 학습 불가, shallow semantic matching. 개선: Memory³ (KV pairs를 attention layer에 직접 주입하지만 model-agnostic embedding에 의존), MemAgent (RL로 memory read/write/overwrite policy 학습), MemGAS (hierarchical multi-granularity retrieval). 그러나 모두 retrieval과 generation 사이의 optimization gap을 해결하지 못한다.

Latent State-Based Memory

모델 내부의 latent representation (hidden states, KV cache)으로 memory를 구성. KV-centric 방법(DSA, MemGen)은 높은 정밀도이지만 100M 규모에서 비용 폭발. Linear attention (RWKV, DeltaNet)은 $\mathcal{O}(L)$이지만 fixed-size state에 정보를 압축하므로 extreme-length에서 catastrophic forgetting 발생. ParallelComp (KV eviction policy)도 유한 cache 내에서의 trade-off. ##MSA는 latent state-based이면서 sparse attention + KV compression으로 $\mathcal{O}(L)$ complexity와 high precision을 동시에 달성##.

3. Memory Sparse Attention

3.1 Overall Design

MSA는 end-to-end trainable한 unified latent memory framework으로, memory sparse retrieval과 answer generation을 하나의 jointly-optimized differentiable pipeline에 통합한다. "Retrieve-then-read" 같은 decoupled 파이프라인의 한계를 넘어, retrieval과 generation이 동일한 loss로 동시에 최적화된다.

3.2 Architecture

3.2.1 Sparse Attention Mechanism

Memory bank: 문서 집합 $\mathcal{D} = \{d_1, d_2, \ldots, d_N\}$. 각 문서 $d_i$의 hidden state $H_i$에서 backbone의 projection weights를 사용하여 standard Key/Value 행렬을 생성하고, 별도의 Router K Projector ($W_{K^R}^h$)로 routing용 key 행렬을 생성:

$$K_{i,h} = H_i W_K^h, \quad V_{i,h} = H_i W_V^h, \quad K_{i,h}^R = H_i W_{K^R}^h \quad (1)$$

이 세 행렬을 고정 길이 chunk로 분할하고 chunk-wise mean pooling $\phi(\cdot)$으로 압축:

$\bar{K}_{i,h} = \phi(K_{i,h})$, $\bar{V}_{i,h} = \phi(V_{i,h})$, $\bar{K}_{i,h}^R = \phi(K_{i,h}^R)$

Inference 시 routing: Query $H_q$에서 backbone의 projection으로 $Q_{q,h}$를 생성하고, Router Q Projector ($W_{Q^R}^h$)로 routing query $Q_{q,h}^R = H_q W_{Q^R}^h$를 생성한다. Chunk $j$의 문서 $i$에 대한 relevance score:

$$S_{ij} = \max_{\text{token } t} (\text{mean}_{\text{head } h} (\cos((Q_{q,h}^R)_t, \bar{K}_{ij,h}^R))) \quad (2)$$

직관적으로, query의 각 token과 각 memory chunk의 routing key 간 cosine similarity를 head 평균 → token max → chunk max 순서로 집계하여 document-level relevance score $s_i = \max_j S_{ij}$를 얻는다. Top-$k$ 문서를 선택하여 해당 문서의 압축된 KV만 로드:

$$K_{\text{ctx}} = [\{\bar{K}_i\}_{i \in \mathcal{I}}; K_q], \quad V_{\text{ctx}} = [\{\bar{V}_i\}_{i \in \mathcal{I}}; V_q] \quad (3)$$ $$\text{Output} = \text{Attention}(Q_q, K_{\text{ctx}}, V_{\text{ctx}}) \quad (4)$$

Layer-selective MSA routing: 실험적으로 초기 layer의 hidden state는 high-level semantic abstraction이 부족하여 routing이 비효율적임을 발견. 따라서 ##MSA routing은 모델의 후반 절반(latter half) layer에만 적용##하고, 전반 layer에서는 Independent Document Processing (문서별 독립 self-attention, MSA routing 없음)을 유지한다. 전반 layer는 문서 상태를 업데이트하면서 hierarchical representation을 구축하고, 후반 layer에서 sparse retrieval + memory integration이 수행된다.

Query에 대해 Global RoPE를 적용. 중앙: Documents 측 — $Q, K, V, K^R$을 생성하고 mean pooling. Positive/Negative document에 대한 contrastive $\mathcal{L}_{\text{aux}}$ 계산. 오른쪽: Query 측 — $Q^R$로 routing하여 top-k 문서의 $\bar{K}, \bar{V}$를 선택, query의 local $K, V$와 concat하여 attention.

3.2.2 Parallel and Global RoPE

MSA의 핵심 확장 전략: ##문서별 독립 RoPE (Document-wise RoPE)##와 Global RoPE의 병행 사용.

Document-wise RoPE (Memory documents): 각 문서에 독립적으로 position ID를 0부터 할당한다. 이를 통해 positional semantics가 memory bank 내 총 문서 수로부터 완전히 분리된다. 모델이 소수 문서(예: 학습 시 수십 개)로 학습해도, inference 시 수백만 문서로 외삽할 수 있는 이유: 각 문서 내부의 position encoding이 동일하므로, 문서 수 증가가 position semantics에 영향을 주지 않는다.

Global RoPE (Active context): User query와 autoregressive generation에는 global RoPE를 적용하되, position ID가 retrieved 문서 수 $k$만큼 offset되어 시작한다. 즉 query의 position ID는 $k$부터 시작하여, 모델이 active context를 retrieved background information의 논리적 continuation으로 인식하게 한다. 이 전략적 offset이 causal dependency를 보존하여 coherent generation을 가능하게 한다.

3.3 Training

3.3.1 Continuous Pre-training

##158.95B 토큰##의 deduplicated corpus에서 continuous pre-training 수행. 목표: Generative Retrieval — 모델이 관련 문서의 고유 document ID를 autoregressively 생성하도록 학습.

Auxiliary contrastive loss ($\mathcal{L}_{\text{aux}}$): Standard generation loss $\mathcal{L}_{\text{LLM}}$에 더해, Router Projector의 routing 결정을 지도하는 contrastive loss를 도입. Query $q$에 대해 positive documents $\mathcal{P}$와 negative documents $\mathcal{N} = \mathcal{D} \setminus \mathcal{P}$를 정의하고:

$$\mathcal{L}_{\text{aux}} = -\frac{1}{|\mathcal{P}|} \sum_{i=1}^{|\mathcal{P}|} \log \frac{\exp(s_i^+ / \tau)}{\exp(s_i^+ / \tau) + \sum_{j=1}^{|\mathcal{N}|} \exp(s_{i,j}^- / \tau)} \quad (5)$$

$\tau$는 temperature. 이 supervised contrastive objective가 latent routing space에서 relevant/irrelevant document chunk를 명시적으로 분리한다.

Two-phase optimization schedule:

  • Warmup phase: $\mathcal{L} = 0.1\mathcal{L}_{\text{LLM}} + \mathcal{L}_{\text{aux}}$, LR $1 \times 10^{-4}$. Router head가 빠르게 effective selection policy를 학습하도록 auxiliary loss에 집중
  • Main pre-training phase: $\mathcal{L} = \mathcal{L}_{\text{LLM}} + 0.1\mathcal{L}_{\text{aux}}$, LR $6 \times 10^{-6}$. Generation task를 우선하면서 routing의 discriminative power를 유지

3.3.2 Post-Training (SFT)

2-stage curriculum learning:

  • Stage 1 (MSA-S1): 8K context에서 대규모 SFT. Instruction-following과 reasoning 기초 확립
  • Stage 2 (MSA-S2): 데이터 품질 강화 + memory context를 8K→64K로 확장. 이 curriculum transition이 massive memory bank으로의 extrapolation robustness를 크게 향상시킴

3.4 Inference

3.4.1 Three-Stage Inference Process

Stage 1: Global Memory Encoding (Offline, one-time) Corpus → 각 문서별 forward pass → K, V, K^R 생성 → chunk-wise mean pooling → 압축된 (K̄, V̄, K̄^R)을 Memory Bank에 cache Stage 2: Routing and Context Assembly (Online) User Query → Router Q Projector → Q^R 생성 Q^R × K̄^R → relevance scoring → Top-k 문서 선택 선택된 문서의 (K̄, V̄)만 GPU로 로드 → query의 local K, V와 concat → sparse context 구성 Stage 3: Sparse Generation (Online) Assembled sparse context에서 autoregressive generation Attention(Q_q, [{K̄_topk}; K_q], [{V̄_topk}; V_q]) → Answer
[Figure 3 설명]: Memory Interleave를 포함한 3-stage inference. 첫 iteration에서 query로 document ID를 생성하고, 해당 문서의 원문을 context에 추가. 두 번째 iteration에서 추가된 context를 기반으로 새 document ID를 생성하고 해당 원문을 추가. 모델이 충분한 증거를 수집했다고 판단하면 "End-of-Retrieve" 토큰을 생성하고 최종 답변을 생성. Multi-hop reasoning에서 각 hop이 이전 hop의 결과를 기반으로 다음 검색을 수행하는 iterative 과정.

3.4.2 Memory Parallel — 100M 토큰 inference를 2×A800에서

100M 토큰의 압축된 KV 및 $K^R$ cache는 pooling kernel $P=64$, 8 heads, head dim 128, 18 layers 기준으로 약 169GB를 요구하며, 2×A800 (총 160GB VRAM)으로는 monolithic 저장이 불가능하다. 이를 해결하는 tiered storage + distributed scoring:

  • GPU-Resident Routing Keys: Routing key $\bar{K}^R$만 다중 GPU VRAM에 분산 저장. 100M context에서 $\bar{K}^R$만으로도 ~56GB이므로 분산 필수
  • CPU-Offloaded Content KVs: Content KV ($\bar{K}, \bar{V}$)는 host DRAM (CPU memory)에 저장. Top-$k$ routing 완료 후 선택된 문서의 content KV만 GPU로 async 전송

Memory-Parallel Retrieval and Distributed Scoring: 4B backbone의 compact size를 활용하여 full model weights를 각 GPU에 복제. Query hidden state를 모든 GPU에 broadcast → 각 GPU가 local shard의 routing key에 대해 독립적으로 similarity scoring → global reduce로 top-$k$ 선택. Tiling mechanism으로 대규모 scoring matrix의 peak memory 관리.

3.5 Memory Interleave

Complex multi-hop queries를 위한 adaptive iterative retrieval-generation 메커니즘. Stage 2 (routing) + Stage 3 (generation)을 반복적으로 교대:

  1. Memory bank의 KV cache 로드 후, 모델이 주어진 query에 대해 document ID 시퀀스를 autoregressive로 생성 (special delimiter로 종료)
  2. 생성된 document ID에 해당하는 원문 텍스트를 시스템이 검색하여 query context에 추가
  3. 확장된 context로 다시 Generative Retrieval → Context Expansion 반복
  4. 모델이 충분한 증거를 수집했다고 판단하면 "End-of-Retrieve" 토큰을 생성하고 최종 답변으로 전환

##각 round에서 생성되는 document ID 수는 고정이 아니라 모델이 adaptive하게 결정##. 학습 시 multi-hop 데이터의 각 retrieval chain을 개별 학습 샘플로 분할하여, 각 샘플이 single-query 또는 기존 document context 기반의 single retrieval step을 담당.

4. Experiments

4.1 Experimental Setup

Implementation:

  • Backbone: Qwen3-4B-Instruct-2507
  • Router Projectors: Backbone weights로 초기화, randomly initialized
  • Continuous pre-training: 158.95B tokens
  • Compression chunk size: $P = 64$ tokens
  • Top-$k$ documents for attention: $k = 16$
  • Model variants: MSA-S1 (8K context SFT only), MSA-S2 (+ 64K context curriculum)

Datasets & Metrics: 9개 QA 벤치마크 (memory bank 277K-10M tokens):

  • Single-hop: MS MARCO v1 (7.34M), Natural Questions (1.47M), DuReader (277K), TriviaQA 10M (10M)
  • Multi-hop: NarrativeQA (538K), PopQA (1.18M), 2WikiMultiHopQA (722K), HotpotQA (1.35M), MuSiQue (1.41M)
  • LLM judge (Qwen3-4B) 평가, 0-5 scale
  • NIAH: RULER benchmark (SA1-3, MK1-3, MV, MQ 8가지 sub-task), 32K-1M tokens
  • MSA는 @adaptive (모델이 자율적으로 retrieval 수를 결정), RAG는 @1, @5, @10 (고정 retrieval 수)

Baselines:

  • Same-backbone RAG (Qwen3-4B): Standard RAG (Qwen3-4B-Embedding + Qwen3-4B), RAG with reranking (+ Qwen3-4B-Rerank), HippoRAG2 (knowledge graph-augmented)
  • Best-of-breed RAG: KaLMv2-Embedding-Gemma3-12B-2511 retriever + Qwen3-235B/Llama-3.3-70B generator ± Qwen3-8B-Rerank. UltraRAG v2.0 pipeline
  • NIAH baselines: MemoryAgent-14B, Qwen3-Next-80B-A3B, Qwen3-30B-A3B, Qwen2.5-14B-1M, Qwen3-4B-Instruct backbone

5. Analysis

5.1 Efficiency Analysis

$M$: query length, $G$: average document length, $k$: top-$k$ selected, $P$: pooling chunk size (64), $L$: total memory size.

5.1.1 Training Complexity

  1. Independent Document Processing: 각 문서 intra-document self-attention $\mathcal{O}(G^2)$ × $L/G$ 문서 = $\mathcal{O}(LG)$
  2. Sparse Routing: Query와 $L/P$ pooled chunks 간 scoring = $\mathcal{O}(ML/P)$
  3. Sparse Generation: Query + $k$개 selected 문서 ($kG/P$ chunks)에 대한 attention = $\mathcal{O}((M + kG/P)^2)$, $L$에 독립
$$\mathcal{O}_{\text{train}} = \mathcal{O}(LG) + \mathcal{O}(ML/P) + \mathcal{O}((M + kG/P)^2) = \mathcal{O}(LG) \quad (6)$$

$L \gg G$일 때 첫째 항이 지배적. ##Training은 $\mathcal{O}(LG)$로 memory size에 선형##.

5.1.2 Inference Complexity

  1. Offline Pre-processing: 전체 corpus의 KV cache 생성 = $\mathcal{O}(LG)$. One-time cost, 모든 query에 대해 재사용
  2. Online Routing: Query를 $L/P$ entries에 대해 scoring = $\mathcal{O}(ML/P)$
  3. Online Generation: $M + kG/P$ 크기의 sparse context에서 autoregressive = $\mathcal{O}(T \cdot (M + kG/P)^2)$, $L$에 독립
$$\mathcal{O}_{\text{inference}} = \mathcal{O}(ML/P) + \mathcal{O}(T \cdot (M + kG/P)^2) = \mathcal{O}(L) \quad (7)$$

Routing이 $\mathcal{O}(L)$로 지배적이며, 비싼 offline pre-processing $\mathcal{O}(LG)$는 query 간 amortize된다. ##Per-query inference는 memory size에 선형## — conventional full attention의 $\mathcal{O}(L^2)$ per-query와 대비된다.

5.2 Context Degradation

MS MARCO에서 16K→100M 확장 시 MSA는 4.023→3.669 (##8.8% 하락##). GPT-4.1, DeepSeek-V3.2 같은 frontier 모델도 competitive 초기 점수에서 context 증가 시 하락. Qwen3-4B backbone은 128K에서 catastrophic collapse (score < 1.5 at 512K). Document-wise RoPE와 sparse routing이 ##memory capacity를 reasoning capability로부터 효과적으로 분리##하여, massive-scale knowledge base에서도 안정적으로 작동.

6-7. Conclusion & Limitations

MSA는 sparse attention + document-wise RoPE + KV compression + Memory Parallel + Memory Interleave를 결합하여 100M 토큰까지 end-to-end 확장 가능한 memory model을 제시한다. Long-context QA와 NIAH에서 frontier LLM, SOTA RAG, external memory system을 모두 상회한다.

Limitations:

  • 다수 문서 간 강한 structural dependency가 있는 경우(tightly coupled cross-document relationships), intrinsic memory만으로 정확한 structural alignment을 유지하기 어렵다.
  • Memory Interleave가 이를 부분적으로 완화하지만, inter-document relationship을 더 체계적으로 보존하는 설계가 필요하다.

비판적 분석 (Critical Analysis)

강점

  • 명확한 포지셔닝: Table 1의 체계적 비교가 기존 방법의 한계를 정확히 짚고, MSA가 모든 desiderata를 동시에 충족하는 유일한 방법임을 설득력 있게 보여준다.
  • 100M 토큰에서 9% 미만 하락은 매우 인상적: 4차 orders of magnitude 확장에서의 안정성은 기존 어떤 방법으로도 달성하지 못한 수준이다. 이는 document-wise RoPE의 설계가 핵심적으로 기여한다.
  • End-to-end trainability의 실증적 가치: 4B 모델이 235B generator + SOTA retriever를 가진 RAG를 평균적으로 상회하는 것은 joint optimization의 위력을 강력히 입증한다.
  • Memory Parallel의 실용성: 2×A800에서 100M 토큰 inference는 즉시 적용 가능한 실용적 기여.
  • 체계적 ablation: 4개 구성 요소(curriculum, interleave, CPT, original text)의 기여를 명확히 분리.

약점

  • 4B 모델만 검증: Qwen3-4B backbone 하나에서만 실험. 더 큰 backbone (14B, 70B)에서의 효과 미검증. Model scale에 따른 MSA의 이점 변화가 불확실.
  • QA task에 제한된 평가: Question Answering과 NIAH만 평가. Summarization, 소설 이해, agent long-term history 등 논문에서 언급한 motivation 시나리오에서의 검증이 없음.
  • Top-$k$ document selection의 hard selection: Top-$k$ routing은 미분 가능하지 않은 hard selection이므로, 엄밀히 말하면 routing 결정 자체는 $\mathcal{L}_{\text{aux}}$로 간접 학습될 뿐 generation loss에서 직접 backprop되지 않는다. "End-to-end trainable"의 정확한 범위에 대한 명확화가 필요.
  • Offline encoding의 비용: 100M 토큰의 offline encoding ($\mathcal{O}(LG)$)은 one-time이라 해도 상당한 비용. Memory bank이 자주 업데이트되는 시나리오에서의 비용 분석이 없다.
  • Chunk-wise mean pooling의 정보 손실: 64 token chunk를 단일 벡터로 압축하므로, 세밀한 토큰 수준 정보가 손실된다. 압축 비율과 정확도의 trade-off 분석(chunk size ablation)이 없다.
  • 표준편차/반복 실험 미보고: 모든 결과가 단일 점추정치로 보고.

Key Takeaways

  1. Memory capacity를 reasoning으로부터 분리하는 것이 핵심: Document-wise RoPE로 position semantics를 문서 수에서 독립시키면, 학습 시 소수 문서에서도 inference 시 수백만 문서로 외삽할 수 있다. ##이는 train-on-short, infer-on-long 패러다임의 가장 효과적인 구현##이다.
  2. End-to-end joint optimization이 decoupled pipeline을 이긴다: 4B 모델이 235B generator를 가진 SOTA RAG를 상회하는 것은, retrieval과 generation 사이의 optimization gap이 model scale보다 더 큰 bottleneck임을 시사한다. 이는 RAG 시스템 설계에 중요한 시사점을 제공한다.
  3. KV cache compression + tiered storage로 extreme-scale inference 실현: Routing key만 GPU에, content KV는 CPU에 저장하는 분리 전략이 메모리 제약을 극복한다. 이 패턴은 다른 long-context 시스템에도 적용 가능하다.
  4. Memory Interleave가 multi-hop reasoning의 핵심: Single-shot retrieval로는 불충분한 complex query에서, 모델이 adaptive하게 retrieval 횟수를 결정하면서 점진적으로 증거를 수집하는 전략이 효과적이다.
  5. Contrastive auxiliary loss가 sparse routing의 정밀도를 보장: Generation loss만으로는 Router Projector가 정확한 selection을 학습하기 어렵고, 별도의 supervised contrastive signal이 latent routing space에서의 분리를 보장한다. 2-phase schedule (warmup에서 routing 집중 → main에서 generation 집중)도 실용적 가이드라인.
