TL;DR

  • 문제: 기존 멀티모달 모델은 ##비전 능력을 LLM에 부착한 형태##—agentic task에서 perception이 reasoning, planning, tool use, execution과 분리되어 있어 실세계 환경(이미지·비디오·웹·문서·GUI)에서 자율적 행동에 한계.
  • 해결: ##멀티모달 perception을 reasoning의 보조 인터페이스가 아니라 핵심 컴포넌트##로 통합. CogViT(403M) 새 인코더 + Multimodal Multi-Token Prediction(MMTP) 새 학습법 + 30+ task category에 걸친 joint RL + 전용 toolchain.
  • 왜 작동하는가: ##visual understanding이 perception → reasoning → action 전체 loop의 첫 단추##라는 통찰을 architecture·training·toolchain 세 층에 일관되게 반영. 약점 perception이 하류 reasoning 실패를 만들기 때문.
  • 비유: 사람이 일을 할 때 "보고 → 생각하고 → 행동"하듯, 모델이 ##스크린·문서를 직접 보고 결정##하는 능력을 자연스럽게 갖춘 것.

핵심 기여 (Key Contributions)

  1. CogViT 비전 인코더 (403M): dual-teacher distillation (SigLIP2 + DINOv3) + Muon optimizer + QK-Norm으로 ##ImageNet-1K Zero-shot 83.5%, 38 CLIP Bench mean 70.4, 14 General Object Bench 45.1## 달성. SigLIP2-SO(427M)와 DFN-H(632M)를 능가.
  2. Multimodal Multi-Token Prediction (MMTP): 비전 입력에서도 multi-token prediction을 가능하게 함. ##<|image|> placeholder## 방식이 직접 visual embedding 전달 또는 vision token masking 대비 더 낮은 학습 손실과 안정적 수렴.
  3. 30+ task category joint RL: perceptual (RefCOCO +4.8%, PointBench +3.2%), reasoning (MMMU +1.8%), agentic (OSWorld +4.9%) 등 multi-task 동시 RL이 single-task보다 안정적이며 ##cross-task transfer## (예: UI-to-code RL이 multi-turn coding 향상).
  4. 외부 에이전트 프레임워크 통합: Claude Code, AutoClaw 통합으로 GLM-5V-Turbo가 ##Vision-Language controller##, Claude Code가 logic, AutoClaw가 GUI 자동화 역할로 정착.
  5. ImageMining 벤치마크 + 5개 design lens: ##217개 vision-centric multi-step tool-use 테스트 케이스##. 7개 도메인×5개 reasoning 카테고리. "Visual Jump"로 모델이 텍스트 대신 이미지를 직접 파싱하도록 강제. 그리고 4개의 design lens (perception 우선, 계층 최적화, 명세된 task, 협업 RL).

1. Background — 왜 native multimodal 모델인가

1.1 단일 모달 LLM의 한계

최근 foundation model은 텍스트 이해에서 ##agentic real-world interaction##으로 이동: 지식 작업, 소프트웨어 엔지니어링, GUI와 상호작용. 하지만 일반 목적의 agentic 모델은 단순한 지능을 넘어 ##이미지·비디오·웹·문서·GUI 같은 복합 멀티모달 컨텍스트##를 perceive, interpret, act할 수 있어야 한다.

1.2 비전을 "부착"하는 기존 접근의 문제

대표적 VLM (Vision-Language Model)은 LLM에 비전 인코더를 후착(post-attached) 형태로 붙임. 이 경우:

  • perception이 reasoning과 분리되어 ##세부 디테일 손실##.
  • fine-grained perception 오류가 high-level planning까지 propagate.
  • multimodal coding, visual tool use 같은 ##복합 task에서 성능 ceiling##.

1.3 GLM-5V-Turbo의 설계 철학

"Multimodal perception is integrated as a ##core component of reasoning, planning, tool use, and execution##, rather than an auxiliary interface to a language model." 즉 보고 이해하는 능력을 모델 design의 가장 깊은 곳부터 박는다.

2. 아키텍처 — 모델·학습·인프라

2.1 CogViT 비전 인코더

두 단계 학습 레시피:

Stage 1 — Distillation-based masked image modeling:

  • Student ViT (403M)가 ##35% mask ratio + 224×224## resolution로 마스킹된 영역 reconstruction.
  • Dual teacher: ##SigLIP2 (semantic) + DINOv3 (texture)## — 두 보완적 표현을 동시 흡수.
  • Quality-aware mixture: ##80% high-quality natural images + 10% instruction-following data + 10% scientific imagery##.
  • Optimizer: ##Muon## (cosine decay schedule), ##QK-Norm##으로 attention computation 전 query/key normalization → logit explosion 완화, 대규모 학습 안정성.

Stage 2 — Contrastive image-text pretraining:

  • 고정 224×224 → ##NaFlex (variable-size, aspect ratio 보존)##.
  • Global batch size 64K, ##sigmoid-based SigLIP loss + bidirectional distributed##.
  • ##8B bilingual (Chinese-English) image-text corpus## — cross-lingual understanding 강화.
  • Muon이 모듈별 LR과 decay schedule을 차별화.

2.2 Multimodal Multi-Token Prediction (MMTP)

Multi-Token Prediction (MTP, [11])은 텍스트에서 효율과 품질을 동시 향상시키는 기법. 본 논문의 도전: ##multimodal 입력에서도 MTP 헤드가 작동해야 한다##.

저자들은 비전 token을 MTP head에 어떻게 전달할지 ##3가지 옵션##을 비교 (Figure 2):

  1. Option 1: LLM backbone의 visual embedding을 MTP head에 직접 전달.
  2. Option 2: 모든 visual token을 MTP head 입력에서 mask out (텍스트-only MTP).
  3. Option 3 (채택): visual positional 정보는 보존, ##모든 visual token을 학습 가능한 <|image|> placeholder##로 대체.

왜 Option 3이 이기는가:

  • 0.5B ablation에서 ##더 낮은 training loss + 더 안정적 수렴##.
  • MTP head는 lightweight라 visual embedding을 직접 받기에 capacity 부족 가능. <|image|> placeholder는 ##더 균일한 입력 형태##.
  • sequence/context parallelism과 자연스럽게 호환—visual embedding을 pipeline-parallel stage 간 전송할 필요 없음.

요약: MMTP는 ##멀티모달 입력에서도 MTP의 효율을 살리면서, 인프라 복잡도를 줄이는 디자인 절충##.

2.3 알고리즘 흐름도

[입력] Visual Input ──► CogViT + MLP Adapter ──┐ ├─► [|image|] placeholder ─┐ Text Input ────► Embedding Layer ──┐ │ └────► Transformer Block × L ────► MTP Module 1, 2, 3 │ (multi-token prediction) │ ▼ [출력 텍스트/액션]

3. 학습 — Pretraining → SFT → Joint RL

3.1 Pretraining (Section 2.3)

Vision과 Language를 ##사전학습 단계부터 깊게 통합##:

  • 혼합 데이터: plain text + multimodal data.
  • 커버리지: world knowledge, interleaved image-text, OCR, coding, GUI, video, multimodal tool-use, spatial perception, grounding, academic problem-solving.
  • 특히 ##multimodal coding data##에 emphasis—visual understanding과 code generation의 alignment 강화.

3.2 SFT (Supervised Fine-Tuning)

본문에서 자세히 다루지 않지만 Joint RL stage 이전 단계로 묘사. perception → reasoning → action loop 데이터로 학습.

3.3 Joint RL across 30+ task categories

이것이 GLM-5V-Turbo의 ##가장 차별적인 학습 단계##. 30+ task category로 동시 RL.

실험 의도: 단일 task RL이 안정적이지 않을 때 multi-task collaborative training이 ##더 풍부한 strategy 분포##를 노출시켜 robust solution으로 steering할 수 있는지.

중요한 관찰:

  • SFT의 cross-domain trade-off가 RL에서 약화 — 여러 도메인이 ##동시에## 안정 향상.
  • narrow distribution (single-task RL)에서 자주 보이는 oscillation이 multi-task collaborative training에서 ##안정화##됨.
  • Cross-task transfer 발견: ##한 도메인의 thinking pattern이 다른 도메인으로 전이##—ex. UI-to-code RL이 multi-turn coding 능력 향상.

이는 multi-task RL이 단순한 coverage 향상 도구가 아니라 ##strategy pattern level에서 unification##을 유도하는 path임을 시사한다.

그러나 한계도 명시: RL이 sampled task distribution에 ##집중되면서 under-represented domain에서 capability decline##이 발생할 수 있음. 그래서 proxy task design이 미묘하게 중요해진다 (예: single-turn UI-to-code RL이 더 복잡한 multi-turn coding 능력을 키운다).

3.4 Multimodal RL at scale (Section 2.4)

VLM RL Gym이라는 통합 환경 인터페이스를 구축. 4가지 dimension의 인프라 재설계:

  1. Unified task and reward abstraction: rule-based verifier (local/sync) + model-based judges (async via API). data-source tag로 reward와 pass@k를 parallel group별로 보고.
  2. Full-pipeline decoupling, asynchrony, stage overlap: rollout inference, reward eval, batch construction, weight transfer를 decouple. 각 inference request에 ##completion callback##을 등록해 reward 연산을 trigger—long-tail batch 대기 비용 제거. CPU↔GPU async pre-fetch.
  3. Fine-grained runtime memory management: vision-side ViT/projector 모듈에 ##targeted recomputation + CPU offloading##—여러 이미지로 activation memory가 linear scaling하는 것을 방지.
  4. Topology-aware partitioning + dynamic load balancing: data-loading 단계에서 CP/TP partitioning을 align—rank별로 필요한 partition만 receive. visual input partitioning을 GPU communication path에서 ##CPU path로 이동##해 communication buffer overhead ##~7GB 감소##. variable-length sequence는 ##sequence length × ViT token count##로 joint bin-packing.

7. Design Lenses (Section 4)

저자들이 개발 과정에서 도출한 4가지 design 관점:

Lens 1: Perception remains foundational

최근 연구가 planning, reasoning, reflection 같은 ##higher-level abilities에 강조##를 두고 있지만, 저자들의 관찰은 fine-grained perception 오류가 downstream 전체에 propagate한다는 것. ##multimodal coding과 grounding이 perceptual learning의 좋은 proxy task##임을 발견 (frontend/SVG coding이 layout/structure/local detail capture를 강제). Critic 데이터로 ##모델이 자기 perception을 비판##하도록 학습—GUI-agent의 instruction tuning에 효과적.

Lens 2: Hierarchical optimization

Agent 능력은 ##capability hierarchy의 multiple level에 분산##되어 학습할 때 더 효율적. 즉 high-level long-horizon task에만 집중하지 말고:

  • Element perception
  • GUI grounding
  • Single-step action prediction
  • Trajectory-level action prediction

각 level이 더 안정적으로 학습되며, low-level이 underdeveloped일 때 high-level training은 ##unstable##해진다.

Lens 3: Clear specification + reliable verification

End-to-end long-horizon task의 진짜 어려움은 horizon이 아니라 ##stable enough specification, verification, evaluation##. Vision2Web을 예시로 — task를 단순 텍스트 instruction이 아니라 ##PRD + mockup + reference page + resource asset##의 풍부한 specification으로 정의. workflow-based verification으로 controlled comparison.

Lens 4: Critic + self-evaluation

(implicit, GUI-agent 경험에서) 모델이 reasoning 중 자기 perception 오류를 비판하면 hallucination 감소.

8. Remaining Challenges

  • Better agentic strategy emergence: cold-start trajectory가 narrow → ##trajectory diversity 증가##가 RL이 nearby variant를 발견하도록 도움. 그러나 단일 starting pattern의 variation에 머무름. 진짜 도전은 ##sub-agent decomposition, multi-agent collaboration##을 모델 스스로 발견하게 하는 것.
  • Multimodal context management: 이미지·비디오는 텍스트보다 ##context budget을 훨씬 더 aggressively 소비##. 많은 시스템이 long trajectory 시 earlier visual observation을 drop. 텍스트는 compaction이 효과적이지만 ##multimodal 압축은 layout, spatial relations, temporal change를 보존해야 해 어렵다##.
  • Model & harness가 capability boundary를 co-shape: agentic system에서 effective capability boundary는 ##model alone이 아니라 task decomposition, tool-use, memory, verification loop가 함께## 결정. 같은 모델이 다른 harness 하에서 매우 다르게 행동. 따라서 model improvement만으로는 progress 측정 불가능.

9. 비판적 분석

  • 30+ task category RL의 정확한 구성 미공개: "30+"라고만 명시. 어떤 task가 weight를 얼마나 받았는지, sampling strategy가 무엇인지 부족. 재현이 어렵고, 결과의 "어느 부분이 RL에서 왔는지"의 attribution도 불분명.
  • 벤치마크 cherry-picking 가능성: 저자가 만든 ImageMining에서 +30.7%로 1위지만, OSWorld(Claude 우위)·CC-Backend(Claude 우위)에서는 밀린다. ##GLM-5V-Turbo가 잘하는 영역에서 더 많은 벤치마크 표시##되어 있음. fair comparison을 위해 weighted-average score가 필요해 보임.
  • 비교 모델의 동시대성: Kimi K2.5와 Claude Opus 4.6 vs GLM-5V-Turbo의 정확한 release date 정렬이 필요. 모델 세대가 다르면 단순 점수 비교는 misleading.
  • MMTP의 ablation 부족: 0.5B 모델에서만 비교되었고 full-scale에서 옵션 1/2/3의 차이가 어떻게 변하는지 알 수 없음. scale-dependent behavior가 반전될 가능성.
  • Pre-training data composition 80/10/10의 근거 부재: high-quality natural / instruction / scientific 비율이 quality-aware라 하지만 ablation은 없음. 다른 비율(예: 70/20/10)에서의 성능 검증이 누락.
  • "Native multimodal"의 정의 모호: 본 논문이 native라 자칭하지만 결국 vision encoder + LLM 결합. 진짜 native (단일 transformer가 multimodal 토큰을 모두 처리)와의 비교가 필요.
  • Tool ecosystem의 lock-in: zai_ 도구들은 proprietary—재현 불가. 외부 사용자가 Claude Code/AutoClaw 통합이라는 vendor 조합에 lock-in됨.
  • Reasoning gain (+1.8% MMMU)이 perceptual gain (+4.8% RefCOCO)에 비해 작음: 본 모델의 강점은 perception/grounding이지 reasoning이 아님. "agent reasoning"을 강조하지만 실제 수치는 less convincing.
  • Vision2Web에서 Claude에 패배 (31.0 vs 43.5): full-stack web replication은 진짜 challenging task인데, 여기서 Claude에 12.5pt 차이로 밀린다. 이는 multimodal coding의 ##highest-level integration##에서 GLM-5V-Turbo가 약점이 있음을 시사.

10. Key Takeaways (이 논문에서 가져갈 것)

  • "Perception이 가장 underlying한 capability": high-level RL이나 fancy reasoning보다 ##fine-grained visual perception##이 agentic 성능의 ceiling을 결정한다는 관찰이 중요. perceptual learning을 위해 SVG/frontend coding을 proxy로 사용하는 idea가 영리하다.
  • Hierarchical optimization across capability levels: long-horizon task에만 RL을 적용하면 unstable. ##element perception → grounding → single-step → trajectory## 계층 분해가 더 안정적.
  • Multi-task collaborative RL의 cross-domain transfer: 한 도메인의 thinking pattern이 다른 도메인으로 전이된다는 발견. Single-task RL보다 multi-task가 ##oscillation 방지 + strategy unification## 효과.
  • <|image|> placeholder for MMTP: 직접 visual embedding 전달보다 균일한 placeholder가 ##학습 안정성 + 인프라 호환성## 양쪽에서 우월. multi-token prediction을 multimodal로 확장하는 실용적 방법.
  • Agent harness가 capability boundary의 일부: model 만 개선하지 말고 ##도구 ecosystem, memory, verification##도 함께 설계. Claude Code + AutoClaw 통합 패턴이 좋은 예.
  • Visual context management의 새로운 도전: 텍스트 compaction과 달리 multimodal 압축은 layout/spatial/temporal을 보존해야 해 본질적으로 어렵다 — 향후 long-horizon multimodal agent의 큰 병목.
