Why Does Self-Distillation (Sometimes) Degrade the Reasoning Capability of LLMs?

TL;DR

  • 문제: Self-distillation은 Chemistry, 코드 등의 도메인에서는 성능을 향상시키지만, 수학적 추론 (mathematical reasoning) 도메인에서는 오히려 성능을 저하시키는 현상이 관찰된다. 왜 "올바른 답을 학습하는데도" 성능이 떨어지는가?
  • 해결: 이 현상의 근본 원인이 epistemic verbalization (인식론적 언어화)의 억제임을 규명한다. Self-distillation에서 teacher가 풍부한 context를 제공할수록 student는 "wait", "hmm", "maybe" 같은 불확실성 표현을 줄이게 되며, 이것이 추론 능력 자체를 저해한다.
  • 왜 작동하는가: Epistemic verbalization은 단순한 스타일적 장식이 아니라 self-Bayesian reasoning의 핵심 메커니즘이다 — 중간 가설을 재검토하고, 불확실성을 표현하여 잘못된 경로에서 복구할 기회를 만든다. 이를 억제하면 모델이 자신감 있지만 수정 불가능한 추론을 생성하게 된다.
  • 비유: 시험 중 "잠깐, 이게 맞나?" 하고 다시 확인하는 습관이 있는 학생이, 풀이가 담긴 교과서를 보고 공부하면 그 습관을 잃게 된다. 평소 문제에서는 더 빠르고 정확해지지만, 처음 보는 어려운 문제에서는 "다시 확인하는 능력"이 사라져서 틀린 답에 그대로 돌진한다.

핵심 기여 (Key Contributions)

  1. Self-distillation의 성능 저하 원인 규명: 수학적 추론에서 self-distillation이 실패하는 근본 원인이 ##epistemic verbalization의 억제##임을 정보이론적 분석과 제어 실험으로 밝혀냄.
  2. Information richness와 task coverage의 상호작용 분석: Conditioning context의 정보량이 높을수록 epistemic verbalization이 억제되고, task coverage가 넓을수록 이 억제가 OOD (Out-of-Distribution) 성능에 해로워짐을 체계적으로 분석.
  3. 3개 base model에 걸친 일관된 검증: Qwen3-8B, DeepSeek-R1-Distill-Qwen-7B, OLMo-3-7B-Instruct에서 최대 40%의 OOD 성능 저하를 관찰하며, 현상의 모델 독립적 robustness를 입증.

1. Introduction — 배경과 동기

Self-Distillation이란?

Self-distillation (Snell et al., 2022)은 LLM post-training의 효과적인 방법으로, 같은 모델의 두 인스턴스를 활용한다: (1) student — ground-truth solution에 접근하지 못하는 상태에서 응답을 생성하고, (2) teacher — ground-truth solution과 같은 더 풍부한 context $c$에 접근하여 student에게 informative reward signal을 제공한다.

RLVR (Reinforcement Learning from Verifiable Rewards)과 결합한 대표적 알고리즘이 SDPO (Self-Distillation via Reinforcement Learning)로, GRPO와 비교하여 다양한 도메인에서 효과적인 것으로 알려져 있다.

문제의 발견: 수학에서의 역설적 성능 저하

[Figure 1 설명]: (a) Chemistry 도메인에서 GRPO와 SDPO의 training score 및 response length 비교. SDPO가 GRPO보다 빠르게 높은 점수에 도달하면서 response length도 크게 줄인다 — 더 간결하고 효과적인 학습. (b) Math 도메인 (DAPO-Math-17k)에서의 동일 비교. SDPO는 response length를 줄이지만, training score가 점차 하락한다 — response가 짧아지면서 성능도 함께 떨어지는 역설적 패턴.

이는 직관적으로 이해하기 어려운 결과이다: "올바른 답을 향해 학습하는데도 왜 성능이 떨어지는가?"

수학적 추론의 특수성: Self-Bayesian Reasoning

LLM에서의 수학적 추론은 self-Bayesian reasoning으로 이해할 수 있다: 모델은 문제와 이전에 생성한 토큰에만 조건부로 각 단계를 생성하며, 중간 가설에 대한 belief를 반복적으로 업데이트한다. 이 과정에서 수학적 추론은 산술, 대수, 기하, 논리적 패턴 인식 등 매우 다양한 하위 기술을 요구하여 평가 벤치마크가 학습 데이터와 자주 OOD가 된다.

이 과정에서 핵심 역할을 하는 것이 epistemic verbalization — "wait", "hmm", "perhaps", "maybe", "actually", "alternatively" 등의 불확실성 표현이다. DeepSeek-R1 (Guo et al., 2025b) 같은 강력한 추론 모델은 이러한 표현을 빈번히 사용하며, Kim et al. (2026)은 이를 제거하면 유의미한 성능 하락이 발생함을 보였다.

[Figure 2 설명]: (2a) Epistemic verbalization이 있는 추론 — "I think the answer is likely 12, but I'm not entirely sure. Let's explore this further..." 이렇게 불확실성을 인정하면서 가설을 재검토한다. Epistemic verbalization 없는 추론은 "Step 1... Step 2... Therefore, the answer is 15."처럼 직선적으로 진행하여, 잘못된 경로에서 복구할 기회가 없다. (2b) Unguided vs. Solution-guided generation — teacher에게 정답이 제공되면 더 짧고 자신감 있는 응답을 생성한다.

2. Preliminaries

Self-Distillation의 형식적 정의

입력 $x \in \mathcal{X}$와 언어 모델 $\pi_\theta$가 생성하는 응답 $y = (y_1, \ldots, y_T)$에 대해, self-distillation에서 같은 모델이 student와 teacher 역할을 모두 수행한다. Teacher는 더 풍부한 context $c$ (예: 정답 solution, 환경 피드백 등)에 접근하여:

$$\pi_\theta^{\mathcal{T}}(\cdot \mid x, c) = \pi_\theta(\cdot \mid x, c)$$

으로 정의된다. 학습은 student와 teacher의 next-token distribution 간 divergence를 최소화한다:

$$\mathcal{L}_\text{SD}(\theta) = \sum_t \text{KL}\left(\pi_\theta(\cdot \mid x, y_{직관적으로, student는 inference 시에는 접근할 수 없는 teacher의 풍부한 정보를 활용해 학습 시 더 나은 방향을 안내받는다.

Epistemic Verbalization의 정의와 측정

Kim et al. (2026)을 따라, 10개의 epistemic marker를 정의한다:

$\mathcal{T} = \{\text{wait, hmm, perhaps, maybe, actually, alternatively, seems, might, likely, check}\}$

응답 $y$의 epistemic token count를 $E(y) = \sum_{t \in \mathcal{T}} \text{count}(t, y)$로 측정한다. 이 토큰들은 모델이 추론 중 불확실성을 외부화(externalize)하는 실용적 지표이다.

3. LLM Reasoning Behavior Under Richer Information

실험 의도: Self-distillation을 분석하기 전에, 먼저 "더 풍부한 정보가 주어지면 LLM의 추론 행동이 어떻게 변하는가?"를 정보이론적으로 분석한다.

Conditional Mutual Information

Context $c$가 target sequence $y$에 대해 제공하는 정보량을 조건부 상호정보로 정의한다:

$$I(y; c \mid x) = H(y \mid x) - H(y \mid x, c) \quad (2)$$

직관적으로, $I(y; c \mid x)$는 context $c$가 추가될 때 $y$에 대한 불확실성이 얼마나 감소하는지를 측정한다.

4가지 생성 설정의 비교

DAPO-Math-17k dataset에서 base model이 0.125~0.5 정확도를 보이는 100개 문제를 선택하여 4가지 설정을 비교한다:

  1. Unguided generation: $c = \emptyset$, $I(y; c \mid x) = 0$ (정보 없음)
  2. Solution-guided generation: $c = s$ (전체 solution 제공), 최대 정보량
  3. Solution-guided (without think): $c = s_{\text{think}}$ (<think> 내용 제거된 solution), $I(y; s_\text{think} \mid x) \le I(y; s \mid x)$
  4. Regeneration-conditioned: $c = \hat{y}$ (이전에 생성된 응답), $I(y; \hat{y} \mid x) \le I(y; s \mid x)$

이 설정들은 conditioning context의 정보량에 따른 ordering을 형성한다:

$$I(y; c \mid x)\Big|_{(1)} = 0 < I(y; s_\text{think} \mid x)\Big|_{(3)} \le I(y; \hat{y} \mid x)\Big|_{(4)} \le I(y; s \mid x)\Big|_{(2)} \quad (3)$$

핵심 실험 결과

설정Avg. ScoreAvg. LengthEpistemic Token Count
(1) Unguided0.3013,054182.5
(2) Solution-Guided ($c=s$)0.981,8738.8
(3) Solution-Guided ($c=s_\text{think}$)0.7812,036159.8

패턴 분석:

  • ##Conditioning context가 더 informative할수록, 모델은 더 짧고 자신감 있는 응답을 생성하며, epistemic token 사용이 극적으로 감소##한다 — Unguided 182.5개 → Solution-Guided 8.8개 (97% 감소).
  • Response length와 epistemic token count는 $I(y; c \mid x)$의 증가에 따라 단조 감소: $\mathbb{E}[L(y)]\big|_{(1)} > \mathbb{E}[L(y)]\big|_{(3)} > \mathbb{E}[L(y)]\big|_{(4)} > \mathbb{E}[L(y)]\big|_{(2)}$.
  • Setting (3)에서 <think> 태그 제거 시 epistemic token이 159.8로 거의 unguided 수준을 회복하면서도 길이도 12,036으로 크게 증가 — ##정보 손실이 불확실성 표현을 다시 촉발##한다.

Takeaway 1: Conditioning context $c$가 더 informative하고 직접적으로 유용할수록, LLM은 더 자신감 있게 답하며 epistemic uncertainty의 표현을 줄인다.

4. Supervised Finetuning with Self-Distillation

실험 의도: Section 3의 관찰이 단순히 스타일적인 것인지, 실제 추론 능력에 영향을 미치는지 확인하기 위해 off-policy self-distillation (SFT)을 수행한다.

실험 설계

DeepSeek-R1-Distill-Qwen-7B를 base model로, 각 800개의 올바른 trajectory로 구성된 두 데이터셋으로 SFT를 수행한다:

  • $\mathcal{D}_\text{ug}$: Unguided responses ($c = \emptyset$), 높은 $\mathbb{E}[E(y)]$와 $\mathbb{E}[L(y)] \approx 12k$ tokens
  • $\mathcal{D}_\text{sg}$: Solution-guided responses ($c = s$), 낮은 $\mathbb{E}[E(y)]$와 $\mathbb{E}[L(y)] \approx 2k$ tokens

핵심 차이: 두 데이터셋 모두 올바른 정답으로만 구성되며, 유일한 차이는 epistemic density이다.

결과

DeepSeek-R1-Distill-Qwen-7BAIME24AIME25AMC23MATH500
Base54.7937.9289.0692.19
SFT on $\mathcal{D}_\text{ug}$51.0440.0087.6690.93
SFT on $\mathcal{D}_\text{sg}$20.2112.7157.0365.52

패턴 분석:

  • ##$\mathcal{D}_\text{sg}$로 학습 시 모든 벤치마크에서 20~30+pp의 극적인 성능 저하## — AIME24: 54.79 → 20.21, AMC23: 89.06 → 57.03.
  • $\mathcal{D}_\text{ug}$로 학습 시에는 유의미한 성능 변화 없음 — 정답 trajectory임에도 epistemic density가 높으면 추론 능력이 보존된다.
  • 핵심: 두 데이터셋 모두 정확한 답을 포함하지만, solution-guided 응답은 teacher의 풍부한 context 덕에 자신감 있지만 epistemic token이 부족하다. 이것을 학습하면 ##inference 시 접근할 수 없는 정보를 전제로 한 추론 스타일을 모방##하게 된다.

Takeaway 2: 올바른 trajectory로 학습하더라도, 과도하게 epistemic verbalization을 억제하면 추론 성능이 실질적으로 저하될 수 있다.

5. On-Policy Self-Distillation

실험 의도: Off-policy (SFT) 결과가 on-policy 설정 (GRPO vs SDPO)에서도 재현되는지 확인한다.

실험 설계

DAPO-Math-17k dataset에서 GRPO와 SDPO를 비교한다. SDPO에서는 두 가지 conditioning을 테스트:

  • $c = s$: 전체 solution (최대 정보량)
  • $c = s_\text{think}$: <think> 제거된 solution (정보 감소)

Teacher policy는 initial policy로 고정 (moving target 대비 성능이 우수함을 Section 5.4에서 확인).

Base model: Qwen3-8B (Thinking Mode ON/OFF), DeepSeek-R1-Distill-Qwen-7B

5.1 DeepSeek-R1-Distill-Qwen-7B

[Figure 3 설명]: (a) Training score와 response length: GRPO는 training score를 약간 증가시키며 $\mathbb{E}[L(y)]$도 소폭 증가. SDPO ($c=s$)는 training score와 response length 모두 급격히 하락 후 GRPO 대비 낮은 수준 유지. (b-c) OOD 평가 (AMC23, AIME24): GRPO는 modest gain (AIME24: 54.7 → 56.0, AMC23: 89.3 → 91.1). ##SDPO ($c=s$)는 OOD 성능을 대폭 저하## (~40% on AIME24, ~15% on AMC23). $c = s_\text{think}$으로 conditioning을 줄이면 OOD 저하가 완화되지만 여전히 base model 이하. (d) Epistemic token 변화: GRPO는 epistemic token을 증가시키지만, SDPO는 더 공격적으로 억제.

5.2 Qwen3-8B (Thinking Mode: ON)

Thinking mode가 활성화된 Qwen3-8B는 초기에 매우 긴 응답과 많은 epistemic token을 생성한다.

[Figure 4 설명]: GRPO와 SDPO 모두 response length와 epistemic verbalization을 줄이지만, SDPO의 더 공격적인 억제가 AIME24에서 더 큰 OOD 성능 저하로 이어진다. 특히 SDPO ($c = s_\text{think}$)에서도 AIME24의 progressive degradation이 관찰된다.

5.3 Qwen3-8B (Thinking Mode: OFF)

<think> 태그 없이 사용할 경우, 모델은 초기에 짧은 응답과 낮은 성능을 보인다.

[Figure 5 설명]: GRPO는 $\mathbb{E}[L(y)]$를 빠르게 증가시키며 epistemic verbalization을 촉진하여 높은 training score를 달성. SDPO는 $\mathbb{E}[L(y)]$를 줄이고 더 느리게 개선되며, AIME24에서 training score가 약간 증가해도 성능은 0.25 → 0.23으로 소폭 감소.

Takeaway 3: Teacher context가 information-rich할수록, on-policy self-distillation은 epistemic verbalization을 줄이고 응답을 짧게 만든다. 이 효과는 base model이 원래 표현하던 불확실성 수준에 따라 달라진다.

5.4 Ablation: Fixed vs. Moving Target Teacher

Naive on-policy self-distillation에서 teacher와 student가 같은 policy를 공유하면 teacher가 moving target이 된다. SDPO는 EMA-smoothed teacher (rate 0.05)를 사용하여 이를 완화한다.

[Figure 6 설명]: EMA rate 0.05의 moving target teacher도 fixed teacher 대비 더 심한 epistemic 억제와 성능 저하를 보인다. 이는 ##self-distillation에서 epistemic 억제가 feedback loop##으로 작동함을 시사한다 — 학습이 진행될수록 모델이 더 자신감 있는 응답을 생성하고, 이 자신감 있는 응답이 다시 teacher로 사용되어 더 강한 억제를 유발한다.

6. Relationship Between Task Coverage, Epistemic Verbalization and Generalization Ability

실험 의도: Self-distillation이 Chemistry, 코드 등에서는 성능을 향상시키는데, 왜 수학에서만 저하되는가? 저자들은 이것이 task coverage의 차이로 설명된다고 가설을 세운다.

6.1 Task Coverage 비교

DomainDataset 특성
ScienceQ&A (Chemistry)2,400문제이지만 6개 주요 유형에서만 출제. Train/Eval split: 90% training, 10% evaluation.
LiveCodeBench v6131문제, 동일 문제가 train/eval 모두에 사용 (90% train에 노출)
DAPO-Math-17k14,000문제, 반복 샘플링으로 25,600 샘플 중 78%가 distinct. 다양한 문제 유형, 평가는 학습에 포함되지 않은 표준 벤치마크 (AIME 2024/2025, AMC 2023, MATH500)

직관적으로, Chemistry와 Code는 학습 데이터가 평가 데이터를 높은 비율로 커버하여 in-domain 최적화가 유효하지만, Math는 학습 데이터와 평가 데이터 간의 gap이 크다.

6.2 Task Coverage와 학습 성능의 관계

$|\mathcal{D}| \in \{1, 8, 64, 128, 512\}$로 학습 문제 수를 변화시키며 GRPO와 SDPO의 행동을 비교한다 (Qwen3-8B, Thinking Mode OFF).

[Figure 7-8 설명]: (1) $|\mathcal{D}| \le 128$에서 SDPO는 빠르게 높은 점수에 도달하며 $\mathbb{E}[L(y)]$를 최대 8배까지 감소 — 작은 task set에서의 효율적 최적화. 그러나 $|\mathcal{D}| = 512$에서는 SDPO의 training score가 GRPO 대비 하락하기 시작. (2) OOD 평가: GRPO는 $|\mathcal{D}| = 1$에서 빠르게 수렴 후 정체하고, 더 큰 $|\mathcal{D}|$에서 점진적으로 더 높은 final score를 달성하면서 $\mathbb{E}[L(y)]$도 증가. SDPO는 패턴이 역전 — ##작은 $|\mathcal{D}|$에서는 더 심각한 OOD 저하##, 가장 큰 $|\mathcal{D}|$에서도 base model 이하.

Takeaway 4: Epistemic verbalization의 가치는 일반화 요구에 비례한다: 반복적이고 익숙한 task (작은 $|\mathcal{D}|$)에서는 중복되어 제거 가능하지만, task 다양성이 증가하면 점점 더 중요해진다.

7. Conclusion

저자들은 self-distillation에 대한 정보이론적 관점을 제공한다. 핵심 발견은 self-distillation의 효과가 정보 제공 방식과 모델이 불확실성을 추론 과정에 통합하는 방식에 의존한다는 것이다. Self-distillation은 모델의 추론 행동을 재형성(reshape)하여 더 높은 자신감으로 답하게 만든다. Task coverage가 제한적이면 이 효과가 빠른 in-domain 최적화를 가능하게 하지만, task coverage가 넓으면 일반화를 저해할 수 있다.

Epistemic verbalization의 억제는 표준 목적함수에 의해 penalize되지 않지만, OOD 성능에 부정적 영향을 미친다. 이는 ##post-training 목적함수가 정답 correctness만이 아니라 uncertainty-aware reasoning behavior도 고려해야 함##을 시사한다.

강점

  • 명확한 인과 메커니즘 제시: "왜 self-distillation이 때로 실패하는가?"라는 질문에 대해 epistemic verbalization 억제라는 구체적이고 검증 가능한 메커니즘을 제시한다.
  • 정보이론적 프레임워크: Conditional mutual information을 활용한 4가지 생성 설정의 ordering이 직관적이면서도 이론적으로 정당화된다.
  • 다중 모델 검증: DeepSeek-R1-Distill-Qwen-7B, Qwen3-8B (ON/OFF), OLMo-3-7B-Instruct의 3개 모델 패밀리에서 일관된 패턴을 확인하여 모델 독립적 현상임을 보인다.
  • Task coverage 분석: Self-distillation이 성공하는 도메인(Chemistry, Code)과 실패하는 도메인(Math)의 차이를 task coverage로 설명한 것이 설득력 있다.
  • 실용적 시사점: "정답으로 학습해도 추론이 저하될 수 있다"는 발견은 LLM post-training 전반에 중요한 경고이다.

약점 및 한계

  • 해결책 미제시: 문제를 진단했지만 구체적인 해결 방법(예: epistemic verbalization을 보존하면서 self-distillation하는 방법)은 제시하지 않는다.
  • Epistemic marker의 단순성: 10개의 사전 정의된 토큰만으로 epistemic verbalization을 측정한다. 더 미묘한 형태의 불확실성 표현(예: conditional statements, hedging)은 포착하지 못한다.
  • 인과관계 vs 상관관계: Epistemic token 감소와 성능 저하의 상관관계는 강하지만, epistemic token을 직접 주입하여 성능이 회복되는지 실험하지 않아 인과관계의 완전한 입증이 부족하다.
  • 수학 도메인 편중: 대부분의 실험이 수학적 추론에 집중되어 있어, 다른 추론 도메인(법률 추론, 의학 진단 등)에서의 일반화 가능성은 미검증.
  • 표준편차 미보고: 다수의 seed에 걸친 변동성이 보고되지 않았다.
  • SDPO가 유일한 self-distillation 대표: 다른 self-distillation 알고리즘(예: OPSD)에서도 동일한 패턴이 나타나는지 제한적으로만 확인 (Appendix C).

추가 관찰

  • $|\mathcal{D}| = 512$에서도 SDPO가 base model을 능가하지 못하는 것은 DAPO-Math-17k의 높은 diversity 때문일 수 있으며, 더 중간 규모의 task coverage에서는 다른 결과가 나올 수 있다.
  • Thinking Mode ON/OFF에 따른 차이는 Qwen3 아키텍처에 특화된 현상일 가능성이 있다.
  • Appendix E에서 learning rate와 top-k distillation 파라미터 조정이 degradation을 근본적으로 해결하지 못함을 보여, 이것이 hyperparameter 문제가 아닌 구조적 문제임을 강화한다.

Key Takeaways

  1. Epistemic verbalization은 추론의 핵심 메커니즘: "wait", "hmm" 같은 불확실성 표현은 단순한 장식이 아니라, LLM이 잘못된 추론 경로에서 복구하는 self-correction 메커니즘의 일부이다.
  2. "정답으로 학습해도 해로울 수 있다": Post-training에서 올바른 답을 학습 신호로 사용하더라도, 그 과정에서 추론 스타일이 바뀌면 일반화 능력이 저하될 수 있다. 정답의 correctness만이 아닌 reasoning process의 quality도 중요하다.
  3. Task coverage가 self-distillation 효과를 결정: In-domain coverage가 높은 task에서는 self-distillation이 효과적이지만, OOD 일반화가 중요한 task에서는 해로울 수 있다. 이는 self-distillation 적용 시 domain 특성을 고려해야 함을 시사한다.
  4. Post-training 목적함수 설계의 새로운 차원: 현재의 post-training 목적함수(GRPO, SDPO 등)는 correctness만 최적화하지만, uncertainty-aware reasoning behavior를 명시적으로 보존하거나 촉진하는 목적함수가 필요할 수 있다.
  5. Teacher 설계의 중요성: Self-distillation에서 teacher에게 제공되는 context의 informativeness를 적절히 조절해야 한다. 과도한 정보가 오히려 학습을 해칠 수 있다.
