TL;DR

  • 문제: long-horizon agent 능력을 키우려면 통상 1T급 파라미터로 frontier 모델을 키워야 한다. 그러나 35B급 작은 모델이 ##knowledge-action 인프라와 multi-teacher 증류##만으로 1T 모델과 어깨를 나란히 할 수 있는가?
  • 해결: ##Agent horizon scaling##이라는 새로운 축을 제시. (1) Knowledge-Action Graph (KAG) 인프라로 평균 45K 토큰짜리 trajectory를 대량 생성하고, (2) 3단계 학습(full-domain SFT → domain teacher → multi-teacher OPD)으로 6개 도메인을 하나의 35B 모델에 통합.
  • 왜 작동하는가: trajectory가 단순 텍스트 supervision을 넘어 evidence·action·observation·verifier outcome을 모두 노드로 가진 process-level 그래프이기 때문. 이로부터 도메인별 teacher들이 specialized policy를 학습한 뒤, Salient Vocabulary Alignment (SVA) 기반 reverse KL 증류로 token-level signal까지 transfer된다.
  • 비유: 1T 모델이 "백과사전 전체를 외운 사람"이라면, Agents-A1은 "##도구 사용법, 검색 노트, 실패 경험까지 모두 정리된 작업 매뉴얼##을 가진 35B 전문가 팀"이다. SEAL-0, HiPhO, FS-O, MolBench-Bind에서 1T 모델을 능가.

핵심 기여 (Key Contributions)

  1. Agent Horizon Scaling의 정식화: 파라미터 스케일링과 분리된 별도의 scaling 축으로서 agent horizon (trajectory 길이·복잡도)을 제시. 35B MoE 모델이 1T급 모델(Kimi-K2.6, DeepSeek-V4)과 long-horizon 태스크에서 경쟁 가능함을 실증.
  2. Long-Horizon Knowledge-Action Infrastructure (KAG): long-horizon agentic competence를 5개 atomic ability(information acquisition, tool calling, executable iteration, evidence verification, constraint tracking)로 분해하고, 각 도메인에 대해 (corpus, action, observation, verifier)의 4-tuple로 정의된 typed knowledge-action graph를 구축. self-play proposer-solver-verifier 게임으로 그래프를 자동 확장.
  3. Domain-Routed On-Policy Distillation with Salient Vocabulary Alignment (SVA): sampled-token OPD의 한계(단일 토큰 surrogate가 high-prob alternative를 제약하지 못함)를 보완. teacher-supported top-k vocabulary에 truncated reverse KL을 적용해 token-level signal을 강화하고, domain-normalized loss로 high-frequency 도메인의 dominance를 제거.
  4. 3단계 학습 레시피: ①Full-domain SFT로 broad long-horizon coverage 확보 → ②6개 domain teacher (Search·MLE·Science·Instruction·Tool·Agentic)를 SFT/RL로 specialize → ③Multi-teacher OPD로 도메인 간 reasoning pattern conflict를 해소한 단일 deployable student를 도출.
  5. 17개 벤치마크 전반의 SOTA-class 결과: SEAL-0 56.4 / IFBench 80.6 / HiPhO 46.4 / FS-O 79.0 / MolBench-Bind 56.8에서 1T 모델 포함 전체 best. SciCode (44.3), HLE (47.6), BrowseComp (75.5)에서도 35B 동급 베이스라인을 압도.

배경 지식

Long-Horizon Agentic Model이란?

최근 LLM은 단순 응답에서 ##agent로 진화##하고 있다 — 도구 호출, 외부 환경과의 상호작용, 다단계 계획, 검증을 통한 자가 수정. 이런 long-horizon 시나리오(software engineering, scientific research, 복잡한 의사결정)에서는 모델이 다음을 수행해야 한다:

  • 정보 수집 → 작업 분해 → 도구 호출 → 중간 결과 검증 → 전략 조정
  • 초기 실수가 누적되므로 self-correction 능력 필수
  • 새 외부 정보가 들어오면 계획을 동적으로 재구성

두 가지 스케일링 루트

본 논문은 long-horizon agent 능력을 키우는 기존 시도를 두 갈래로 분류한다:

  • Route 1: Parameter Scaling: Kimi-K2.6, DeepSeek-V4-pro, GPT-5.5 같은 frontier 1T급 모델. 추론·도구사용·도메인 지식을 모두 파라미터에 내재화. 효과적이지만 데이터·연산이 거대한 자원을 요구하여 재현이 어렵다.
  • Route 2: Horizon Scaling: 파라미터를 키우지 않고 의사결정 과정을 명시화(trainable supervision)하여 trajectory 길이·품질을 확장. 이때 두 가지 병목이 발생: (a) 외부 지식·관찰·검증 신호를 통합한 인프라 부재, (b) 도메인 간 이질적 능력의 충돌.

왜 Knowledge-Action Graph가 필요한가

기존 corpus(web, books)는 텍스트만 담고 있어 ##action provenance, tool transcript, execution log, verifier outcome##이 없다. 모델이 grounded feedback에서 학습하려면 이 모든 신호가 typed object로 연결된 인프라가 필요하다. KAG는 단순 entity-relation 지식 그래프와 달리 "answer가 어떻게 획득되고 검증되었는가"의 process를 보존한다.

On-Policy Distillation (OPD)이란?

Teacher는 supervised 답안을 생성하고, student는 자신의 rollout을 teacher의 분포 하에서 평가받는 방식이다. Sampled-token OPD [Lu & Thinking Machines 2025]는 효율적이지만 ##realized rollout token만 점수화##하기 때문에 근처의 high-probability alternative들이 unconstrained 상태로 남는다. 최근 Fu et al. [2026]은 이로 인해 unstable·imbalanced guidance가 발생함을 지적했고, 본 논문은 SVA로 이 문제를 직접 다룬다.

방법론 1 — Long-Horizon Knowledge-Action Infrastructure (KAG)

Knowledge-Action Graph의 정의

각 도메인 $\mathcal{B}_d$에 대해, KAG는 typed 4-tuple로 정의된다 (식 1):

$$\mathcal{G}_d = (\mathcal{C}_d, \mathcal{A}_d, \mathcal{O}_d, \mathcal{V}_d)$$
  • $\mathcal{C}_d$: 도메인 corpus (evidence chunks, entities, facts, constraints)
  • $\mathcal{A}_d$: action space (tool call, retrieval query, code edit/execution, reasoning step)
  • $\mathcal{O}_d$: observation space (tool return, retrieved evidence, execution state, intermediate artifact)
  • $\mathcal{V}_d$: verifier set (correctness, evidence support, constraint satisfaction, goal completion에 대한 자동 체크)

그래프 노드는 $t$번째 action record $(s_t, a_t, o_t, v_t)$이며, 엣지는 support, dependency, production, verification, action-transition 관계를 인코딩한다.

직관적으로 말하면, 일반 지식 그래프가 "사실의 정적 집합"이라면 KAG는 "##문제 해결 과정의 동영상##"이다. 성공·실패 trajectory 모두 evidence-backed로 보존되어 cross-step credit assignment가 가능하다.

Self-Play Graph Search and Expansion

단일 패스로 생성한 reasoning 데이터는 품질이 보장되지 않는다. 따라서 proposer–solver–verifier 게임으로 그래프를 반복 확장:

  • $\pi_P$: 그래프 영역을 샘플링해 constrained task 제안
  • $\pi_S$: retrieval과 도구로 task 해결
  • $\pi_V$: answer, evidence, execution, trajectory, shortcut risk를 검증

각 task는 식 (2)로 표현된다:

$$x = (q, d, \tau, y^\star, \mathcal{E}_q, \mathcal{V}_q), \quad \mathcal{E}_q \subseteq \mathcal{C}_d, \quad \mathcal{V}_q \subseteq \mathcal{V}_d$$

trajectory $\tau$는:

$$\tau = [(s_1, a_1, o_1, v_1), \ldots, (s_T, a_T, o_T, v_T), y]$$

$\pi_V$가 다음 5가지를 모두 만족할 때만 task가 accepted된다:

  1. verifiable against some $v \in \mathcal{V}_q$
  2. valid: $\tau$가 $v$가 accept하는 답에 도달
  3. process-informative: 의미 있는 중간 결정을 exercise (one-shot lookup 아님)
  4. evidence-covering: $\mathcal{E}_q$의 evidence가 $\tau$에서 실제 consulted
  5. unambiguously specified, no shortcut solution

Atomic Abilities Decomposition

Long-horizon agentic competence는 다음 5개 atomic ability로 분해된다:

Ability역할
Information Acquisition검색·문서 읽기·증거 추출
Tool Calling스키마 기반 도구 호출, 인자 결합
Executable Iteration코드 작성·실행·디버깅의 반복
Evidence Verificationretrieved 증거로 주장 검증
Constraint Trackingformat·length·rule 등 제약 추적

방법론 2 — Domain-Routed On-Policy Distillation with SVA

왜 sampled-token OPD로 부족한가

기존 sampled-token OPD는 student가 sample한 $y_i \sim \pi_{\theta_s}(\cdot | x_i)$의 실제 token에만 supervision을 준다. 이는 token-level surrogate일 뿐이라 ##teacher 분포의 high-probability 대안 토큰##을 제약하지 못한다 → 분포의 일부만 정렬되고 나머지는 drift.

Salient Vocabulary Alignment (SVA)

position $t$에서 student와 routed teacher를 동일한 student-generated prefix $(x_i, y_{i, $$p_{s'}(u) = \pi_{\theta'_s}(u \mid x_i, y_{i,teacher의 top-k valid token 집합 $\mathcal{S}^{(k)}_{i,t}$ 위에서 두 분포를 renormalize:

$$\bar{p}_{s'}(u) = \frac{p_{s'}(u)}{\sum_{v \in \mathcal{S}^{(k)}_{i,t}} p_{s'}(v)}, \quad \bar{p}_{t,i}(u) = \frac{p_{t,i}(u)}{\sum_{v \in \mathcal{S}^{(k)}_{i,t}} p_{t,i}(v)}$$

per-sample SVA objective는 truncated reverse KL (식 4):

$$\ell^{(i)}_{\text{SVA}}(\theta'_s; \theta_{t,i}) = \frac{1}{|R_i|} \sum_{t \in R_i} \sum_{u \in \mathcal{S}^{(k)}_{i,t}} \bar{p}_{s'}(u) \log \frac{\bar{p}_{s'}(u)}{\bar{p}_{t,i}(u)}$$
직관적으로 말하면, "teacher가 가장 가능성 있다고 보는 top-k 후보 토큰들"에 한정해서 student를 정확히 그 ##teacher 분포 모양으로 맞추는## reverse KL이다. 한 토큰에만 점수를 주는 게 아니라 teacher가 인정한 small dictionary 전체를 정렬한다.

student-side coverage 모니터링 (식 5):

$$\rho(i, t) = \sum_{u \in \mathcal{S}^{(k)}_{i,t}} p_{s'}(u)$$

이 값이 클수록 student가 teacher가 선택한 vocabulary 안에 mass를 두고 있음을 의미.

Domain-Routed Normalized Objective

도메인마다 reasoning pattern·response style이 달라 gradient가 heterogeneous하다. 본 논문은 hard domain routing을 사용:

$$\theta_{t,i} = \theta^{d_i}_t$$

즉 각 sample은 자기 도메인 teacher에게만 supervised. soft mixture를 쓰지 않는 이유는 ##token-level에서 incompatible teacher signal을 섞으면 학습이 흔들리기 때문##.

mini-batch $\mathcal{B}$에서 active domain $\mathcal{D}_{\mathcal{B}}$에 대해 domain-normalized loss (식 6):

$$\mathcal{L}_{\text{MT-SVA}}(\theta'_s) = \frac{1}{|\mathcal{D}_{\mathcal{B}}|} \sum_{d \in \mathcal{D}_{\mathcal{B}}} \frac{1}{|\mathcal{B}_d|} \sum_{i \in \mathcal{B}_d} \ell^{(i)}_{\text{SVA}}(\theta'_s; \theta_{t,i})$$
직관적으로 말하면, "각 도메인 내부에서 loss를 평균낸 뒤 도메인끼리 다시 평균"하는 hierarchical averaging이다. instruction-following처럼 token이 짧은 도메인이 deep-research처럼 token이 긴 도메인에 묻히지 않도록 한다.

Stage 2: Domain-Level Teacher Training (도메인별 specialize)

(a) Search Teacher — SFT + RL (GRPO)

1단계: search trajectory만으로 SFT. 2단계: GRPO RL on ~2,000 multi-hop reasoning question.

도구: Web Search / Read Page / Code. Reward는 3가지 컴포넌트:

  1. Correctness reward: LLM judge로 final answer 평가 (semantic paraphrase 허용).
  2. Search behavior penalty:
    • Efficiency penalty: $K$ round 초과 시 linear penalty
    • Repetition penalty: 동일 query/URL 반복 시 small penalty (cap)
  3. Format calibration reward: parse 가능한 답변 형식.

GRPO 설정: rollout 8 / prompt, clip [0.2, 0.28], KL coef 0.001, entropy coef 0.0001, max response 4096, max sequence 131,072, 300 tool calls/rollout, temperature 1.0.

(b) Science Teacher — Two-stage SFT

Stage 1 (Reasoning-Enhanced SFT): tool 없이 non-tool reasoning trace로 ##intrinsic causal/mathematical chain##을 강화.
Stage 2 (Tool-Augmented SFT): search / visit / code / scholar 도구를 포함한 trajectory로 grounded interaction 학습.

(c) Instruction-Following Teacher — Two-stage RL (GRPO)

Stage 1: Nemotron instruction-following 데이터로 rule-based reward (length, keyword, language).
Stage 2: long-context learning 데이터로 answer matching reward. ##dynamic sampling##으로 모든 응답이 same reward인 group을 제거 (preference signal이 없는 데이터 필터링).

(d) Tool-Calling Teacher — SFT + PAPO-style RL

Outcome reward $r_i^{\text{out}} \in \{0,1\}$와 rubric-based process score (식 7):

$$r_i^{\text{proc}} = \frac{1}{|\mathcal{R}|} \sum_{j=1}^{|\mathcal{R}|} \mathbf{1}[\mathcal{R}_j \text{ is satisfied}]$$

Asymmetric advantage [PAPO, Tan et al. 2026]: process reward는 ##실패한 trajectory에만## 적용 (식 8):

$$A_i = A_i^{\text{out}} + \lambda_{\text{neg}} \mathbf{1}[r_i^{\text{out}} = 0] A_i^{\text{proc}}, \quad \lambda_{\text{neg}} = 0.5$$
직관적으로 말하면, 성공한 trajectory는 이미 모든 rubric을 충족하므로 process reward를 추가로 더하면 ##double-count##가 된다. 실패 케이스에 한해 "얼마나 가까웠는지" rubric 정도로 partial credit을 주어, near-miss 학습을 가속화.

Data reuse 공식 (식 9):

$$N_{\text{reuse}} \approx \frac{R \cdot B \cdot K}{|\mathcal{D}|}$$

Tool-specific RL은 단 ##64개의 hard task##로 PAPO-style advantage를 적용해 적은 데이터로 효율적 향상.

Stage 3: Multi-Teacher On-Policy Distillation

Stage 1의 full-domain SFT 모델을 student로 초기화. 각 prompt에 domain label을 부착하고 해당 도메인 teacher만 supervision signal을 줌. student가 자기 rollout을 생성하면, teacher는 그 prefix를 token-level로 채점 (offline imitation 아니라 student의 own trajectory를 점수).

Rollout bounding: turn budget $T_{\max}$, response budget $L^{\text{resp}}_{\max}$, context budget $L^{\text{ctx}}_{\max}$. 한계 도달 rollout은 TRUNCATED로 마킹·유효 prefix 보존. system 중단된 것은 ABORTED로 retry.

다중 도메인 데이터 파이프라인 (Section 3)

3.1 Long-Horizon Search

Wiki database 위에 corpus graph를 구축 — entity node + hyperlink edge. degree·text-length constraint로 노이즈 제거.

  • Relation-chain generation: corpus graph + domain KAG 위 controlled random walk. LLM selector가 local path coherence와 cross-domain shift의 균형을 잡음.
  • QA pair generation: chain의 마지막 entity를 정답으로 설정 → 자연어 질문으로 rewrite, proper noun 마스킹. verifier가 answer identity + evidence support를 검증.
  • Trajectory collection: search / read_page / code 도구로 deep-research task 수행. context 256K, tool call 무제한 per turn. wrong answer / 너무 짧은 history / guessing behavior 제거.

3.2 Machine Learning Engineering (MLE)

MLE-Dojo + ended Kaggle competition으로 task pool 구성. Agentic harness는 executable solution tree를 키움 — 새 script가 root, patch가 child node, execution이 observation을 attach.

Tool interface (Table 1):

  • Code authoring: write_full_code, patch_code, execute_code, execute_bash
  • Search-tree navigation: list_nodes, select_node, invalidate_node, update_answer, get_current_answer
  • Persistent memory: write_notes / read_notes (context compaction에도 살아남는 노트북)
  • Sub-agent: analyze (격리된 분석 sub-agent)

3.3 Scientific Reasoning

수학·물리 등 fundamental domain의 seed problem을 KAG로 expand. 두 방향:

  1. Harder reasoning variants: 도메인 지식 깊이 증가, symbolic 구조 추가, multi-step derivation 확장.
  2. Interaction-enriched variants: cross-domain 지식 통합, retrieval 필요, code-based 수치 계산 요구.

~15K 강화된 과학 문제 corpus 생성. trajectory는 (a) no-tool reasoning (pure derivation) (b) tool-augmented (search/visit/code/scholar) 두 종류.

3.4 Instruction Following

두 source:

  • NVIDIA Nemotron-RL의 13K multi-constraint instruction (WildChat + Open-Instruct 기반). length·format·keyword·language constraint를 condition node로 표현.
  • Self-built long-context pipeline: 10K verified long-context QA. document graph 구축 후 multi-hop QA 합성, local in-context rule이나 distractor 주입 → ##long-context evidence + in-context info 결합##.

3.5 Tool Calling

Tool extraction → tool-interaction graph → graph-compositional task synthesis → solvability assessment. Tool sandbox에서 simulated user 포함 multi-turn rollout 생성. arbitrary tool concatenation 회피 — later call이 earlier object·constraint·observation에 dependent.

결과 2: Domain Teacher Training (Tables 5–8)

Search teacher (Table 5): GAIA 59.8 → 95.1 (+35.3), Seal-0 41.4 → 54.1, HLE w/ tools 47.4 → 50.3, XBench-DS 77.0 → 86.0. ##search-specific RL이 가장 큰 향상을 가져옴##.

Science teacher (Table 6): FS-R 2.5 → 54.3 (가장 극적), HiPhO 37.0 → 46.9, FS-O 64.5 → 82.0. Two-stage SFT가 reasoning과 tool-use를 모두 강화함을 입증.

Long-instruction teacher (Table 7): IFBench 70.2 → 82.0, IFEval 91.9 → 93.4, LongBench V2 59.0 → 62.4. instruction-following에서도 RL이 효과적.

Tool teacher (Table 8): $\tau^2$-Bench Airline 16.0 → 72.0 (+56), Retail 30.7 → 82.5, Telecom 50.9 → 93.0, Avg 32.5 → 82.5. VitaBench Cross Domain 11.5 → 30.0, Delivery 39.3 → 56.0. ##단 64개의 hard task로도 PAPO-style RL이 극적 향상##.

이 실험이 뒷받침하는 주장: 도메인별로 specialized teacher가 베이스라인을 의미 있게 뛰어넘는다 → multi-teacher 풀이 의미가 있다.

OPD가 항상 teacher를 능가하지는 않음

저자는 명시적으로 인정: Table 6/7/9를 비교하면 ##OPD-trained 모델이 individual teacher를 항상 능가하지는 않는다##. 이는 expected — 각 teacher는 자기 도메인에 특화돼 있고, Agents-A1은 heterogeneous task 전반의 unified policy를 유지해야 함. OPD의 역할은 "teacher 강점을 single model로 transfer + full-domain SFT 대비 balance를 개선"하는 것이지 ##각 specialty를 능가하는 것이 아님##.

Long-Horizon Application Cases (Section 5.3)

(1) 12-Hour Optimization Run on Whale Call Detection (Figure 4)

ICML 2013 Whale Challenge에서 Agents-A1이 12시간 동안 end-to-end 최적화:

  • Initial CNN baseline: AUC 0.58
  • Audio augmentation (noise + gain): +0.13 → 0.71
  • Train on March 29 (temporal domain shift 식별): +0.10 → 0.81
  • Mel-CNN + deep ensemble: +0.08 → 0.93
  • Heavy augmentation (10x) + final tuning: +0.06 → ##AUC 0.9935 (gold medal)##

주목할 점: 단순 hyperparameter tuning이 아니라 ##데이터 진단·representation 설계·augmentation strategy의 multi-step 최적화##.

(2) Earth Science Analysis — Tropical Cyclone Nargis (Figure 5)

IBTrACS 데이터 자동 식별 → data extraction → cleaning → derived metric computation → visualization → result synthesis의 closed loop. Bay of Bengal 발생, NE recurvature, Myanmar landfall을 재구성. WMO/IMD와 JTWC/USA wind estimate를 둘 다 보존하여 ##operational convention conflation을 회피##.

비판적 분석

강점

  • 실증적 설득력: 35B 모델이 1T 모델을 ##구체적인 long-horizon benchmark에서 능가##한 것은 의미 있는 증명. 특히 FS-R(2.5 → 40.0)·HiPhO(37.0 → 46.4) 같이 절대값 차이가 큰 영역.
  • 방법론적 명료성: KAG 정의, SVA reverse KL formulation, domain-normalized loss, PAPO advantage 모두 식 단위로 명확. 재현 가능성 측면에서 우수.
  • 실패 분석의 정직성: SFT-only 단계에서 IFBench·$\tau^2$-Bench·HLE가 떨어졌음을 보고하고, 이것이 OPD 도입의 motivation이 되었다고 명시. OPD가 모든 teacher를 능가하지 않음도 인정.
  • 인프라 contribution: KAG는 단일 논문 결과를 넘어 ##재사용 가능한 long-horizon supervision 인프라##. 다른 35B 모델로도 적용 가능한 일반 framework.

약점·의문

  • $\tau^2$-Bench 재현 불일치: Qwen3.5-35B-A3B 공식 결과(81.2)와 저자 재현(32.5) 사이에 ##48점 차이##. 저자는 codebase version 차이라 했지만, 만약 32.5 baseline이 부정확하다면 Table 4의 $\tau^2$ 개선(32.5→82.5)이 과대평가일 수 있음.
  • Tool-augmented vs tool-free 비교의 공정성: HiPhO/FS-O/FS-R에서 Agents-A1은 ##tool-augmented##, 비교 모델(Kimi-K2.6 등)은 ##tool-free## 결과를 사용. 저자는 "tool 적용이 untrained 모델에 오히려 해가 될 수 있어 공정한 자체 평가 protocol을 따랐다"고 변호하지만, like-for-like 비교라고 보긴 어렵다.
  • Compute cost 미공개: ##학습에 사용된 GPU 시간·token 수가 명시되지 않음##. 35B 모델이지만 ~100K trajectories × 45K tokens × OPD/RL 단계까지 합치면 결코 작지 않을 가능성.
  • Code/data 부분 공개: 모델은 공개되지만 KAG construction pipeline·OPD 학습 코드의 공개 범위가 불확실.
  • 1T 모델 evaluation 의존성: 1T 모델 결과는 각자 technical report에서 인용 — ##통제된 evaluation environment가 아니라 self-report## 점수. SOTA 클레임의 strict apples-to-apples 비교는 어려움.
  • Generalization 검증 부족: 동일 KAG/OPD 레시피를 다른 base 모델(예: Llama·DeepSeek 계열 35B)에 적용한 결과 없음 → ##Qwen3.5-35B-A3B specific한지 일반화 가능한지 불명확##.
  • 도메인 충돌 해결의 추상도: hard routing이 "soft mixture가 incompatible signal을 섞는다"는 직관에 의존. 실제로 soft mixture와의 ablation 비교는 제시되지 않음.

한계의 자체 인정

저자도 Section 6에서 limitation을 명시:

  • 현재 능력은 ① Qwen3.5-35B-A3B baseline, ② KAG 인프라, ③ OPD 세 가지에 의존
  • Long-horizon에서 필요한 ##기본 atomic ability##(planning before reasoning, reflection before acting, key info summarization, important past info identification)가 아직 부족
  • 이들 기본 능력의 강화가 future work

Key Takeaways (이 논문에서 가져갈 것)

  1. Horizon scaling은 parameter scaling의 대안이 될 수 있다: ##verifiable·tool-rich·constraint-heavy 도메인##에서는 35B 모델이 trajectory 품질 + 도메인별 teacher + multi-teacher 증류만으로 1T를 능가 가능. 모든 도메인에서가 아니라 ##선택적으로 그렇다##는 점이 중요.
  2. KAG 4-tuple 정의 ($\mathcal{C}, \mathcal{A}, \mathcal{O}, \mathcal{V}$)는 long-horizon supervision의 일반 framework: agent training pipeline을 설계할 때 이 4축으로 데이터를 organize하면 process-level supervision이 자연스러워진다.
  3. SVA reverse KL은 OPD의 실용적 개선: sampled-token OPD가 unstable하다면, ##teacher top-k 토큰에 truncated reverse KL##이 간단하면서도 효과적. 학생이 teacher 분포의 모양 전체를 정렬하도록 함.
  4. Hard domain routing + domain-normalized loss: heterogeneous reasoning pattern을 가진 다도메인 데이터를 통합할 때, soft mixture보다 hard routing이 token-level conflict를 줄임. high-frequency 도메인의 dominance는 hierarchical averaging으로 해결.
  5. PAPO-style asymmetric advantage: 성공 trajectory에 process reward를 더하지 않고 ##실패에만 추가##하는 것이 double-counting을 피하고 near-miss learning을 가속.
  6. 실용적 도구 디자인: MLE 도메인의 write_notes / read_notes(context compaction에도 살아남는 노트)와 analyze sub-agent는 ##long-horizon agent 설계에서 일반적으로 유용한 패턴##.
  7. 오픈소스 35B 모델로 1T급에 도전하려면: (a) trajectory를 충분히 길게(45K+), (b) atomic ability별로 verifier를 갖춘 corpus, (c) 도메인 specialist teacher pool, (d) on-policy distillation으로 통합 — 이 네 가지가 실용적 레시피.
