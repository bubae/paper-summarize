TL;DR

  • 문제: 기존 unified multimodal model (이미지 이해 + 생성을 동시에 수행)은 파라미터가 수십억 개에 달하고 대규모 pre-training 데이터(50M~1B 샘플)가 필요하여, 모바일/에지 디바이스에서의 실시간 배포가 불가능하다.
  • 해결: ##1.6B 파라미터의 경량 vision-language-diffusion 모델 Mobile-O를 제안##한다. 핵심 모듈인 Mobile Conditioning Projector (MCP)가 VLM hidden state를 diffusion 모델에 직접 연결하며, quadruplet 형식의 unified post-training으로 수백만 샘플만으로 이해/생성 성능을 동시에 향상시킨다.
  • 왜 작동하는가: MCP가 VLM의 마지막 K개 layer의 feature를 learnable weight로 fusion하고 depthwise-separable convolution으로 정제하여, 기존 learnable query token 방식보다 적은 파라미터로 효과적인 cross-modal alignment을 달성한다. 또한 quadruplet 데이터 형식이 이해와 생성 task를 하나의 샘플에서 동시에 학습하게 하여 cross-modal transfer를 극대화한다.
  • 비유: 기존 모델이 "대형 공장에서 이해 라인과 생성 라인을 분리 운영"하는 방식이라면, Mobile-O는 "하나의 소형 작업대에서 이해와 생성을 양손으로 동시에 처리하는 숙련된 장인"과 같다. MCP는 두 손(이해·생성)을 자연스럽게 연결하는 신경계 역할이다.

핵심 기여 (Key Contributions)

  1. Mobile-O 아키텍처: 효율적인 unified vision-language-diffusion 모델로, 1.6B 파라미터만으로 SOTA급 multimodal 이해 및 이미지 생성 성능을 달성하며 모바일 디바이스에서 실시간 추론이 가능하다.
  2. Mobile Conditioning Projector (MCP): VLM의 multi-layer hidden state를 depthwise-separable convolution과 layerwise learnable fusion으로 diffusion 모델에 직접 연결하는 경량 cross-modal 모듈. 기존 learnable query token 접근법 대비 추가 토큰 예산 없이 파라미터를 절반으로 줄이면서 더 적은 pre-training 데이터로 동작한다.
  3. Quadruplet 기반 Unified Post-Training: (generation prompt, image, question, answer) 형태의 105K 샘플 unified dataset을 구성하여, 이해와 생성을 multi-task objective로 동시 최적화하는 post-training 기법. 기존 disjoint 또는 sequential training의 한계를 해결한다.
  4. 에지 디바이스 실시간 배포: iPhone에서 512×512 이미지 생성에 ~3초, 이해 TTFT 248ms를 달성. MacBook, Jetson Nano에서도 기존 모델 대비 6~11배 빠른 속도를 보인다.

1. Introduction — 문제 정의와 동기

Unified multimodal model은 하나의 아키텍처 내에서 이미지 이해(captioning, VQA)와 이미지 생성(text-to-image)을 모두 수행한다. BLIP-3o, Janus, Show-O 등이 이 방향을 추구하지만, 실제 모바일 배포에는 두 가지 근본적 장벽이 존재한다:

장벽 1: 과도한 모델 크기

대부분의 unified model은 무거운 visual encoder와 denoising module을 사용한다. 예를 들어 BLIP-3o는 2.6B UNet + 3B VLM + 1.5B DiT = 총 7.1B 파라미터가 필요하다. 최근 JanusFlow 같은 효율성 연구도 여전히 에지 디바이스 실시간 배포에는 부적합하다.

장벽 2: 데이터 의존성

효과적인 cross-modal alignment을 위해 50M~1B 규모의 pre-training 데이터가 필요하여, 학습 비용과 시간이 막대하다.

이러한 문제에서 출발한 핵심 연구 질문: "이해와 생성 모두에 효과적이면서, 모바일 폰 같은 소비자 디바이스에 배포 가능할 만큼 효율적인 unified multimodal model을 만들 수 있는가?"

2. Related Work — 배경 지식

2.1 Unified Multimodal Understanding & Generation

초기 unified model (MagVLT, Unified-IO 2)은 단일 transformer로 이해와 생성을 통합했다. 이후 Janus, BLIP-3o, JanusFlow 같은 hybrid 설계가 diffusion decoder를 추가하여 text-to-image 생성 품질을 개선했고, Emu3는 autoregression만으로도 text-to-image가 가능함을 보였다. 그러나 이들은 CLIP-ViT 같은 무거운 image encoder나 UNet-style denoiser에 의존하며, 이해와 생성을 disjoint하게 학습하거나 sequential하게 학습하는 한계가 있다.

2.2 Efficient Multimodal Understanding Models

FastVLM (Apple, CVPR 2025)은 본 논문의 이해 백본이다. FastViTHD라는 hybrid vision encoder를 도입하여, 고해상도 이미지에서 적은 수의 visual token을 생성함으로써 TTFT (Time-to-First-Token)를 획기적으로 줄였다. FastViT는 structural reparameterization을 활용한 빠른 hybrid vision transformer이며, FastVLM은 이를 확장하여 MobileCLIP 레시피로 pre-training한다. 0.5B 변형은 LLaVA-OneVision-0.5B 대비 85배 빠른 TTFT를 달성한다. SmolVLM은 aggressive tokenization과 architectural 최적화로 compact model에서도 경쟁력 있는 성능을 보여주었다.

2.3 Efficient Text-to-Image Generation Models

SANA (NVIDIA, 2024)는 본 논문의 생성 백본이다. SANA는 Deep Compression Autoencoder로 이미지를 32배 압축하여 latent token 수를 줄이고, 모든 vanilla attention을 linear attention으로 대체하여 복잡도를 $O(N^2)$에서 $O(N)$으로 낮춘 Linear DiT (Diffusion Transformer) 아키텍처를 사용한다. SANA-0.6B는 Flux-12B보다 20배 작으면서 100배 이상 빠른 throughput을 달성한다. 단, SANA는 text encoder로 Gemma-2B를 사용하므로 총 파라미터가 2.6B에 달한다. SnapGen은 체계적 architecture 최적화와 cross-architecture distillation으로 모바일 T2I 생성을 추구하지만, 이해 기능은 없다.

2.4 Data Efficiency and Training Stages in Unified Models

기존 unified model의 학습 전략은 크게 두 갈래이다:

  • Joint Training (Metamorph, Show-O): 이해와 생성 데이터를 섞어 multitask learning. 효과가 총 데이터 크기와 이해/생성 비율에 강하게 의존하며, 보통 각 task의 disjoint subset (예: LLaVA-665K + BLIP3o-60K)을 사용하여 cross-task alignment이 제한적이다.
  • Sequential Training (BLIP-3o, Janus): VLM을 먼저 학습한 후 backbone을 freeze하고 생성 모듈만 학습. 이해 능력은 보존되지만, 학습 중 cross-task interaction을 활용하지 못한다.

Mobile-O는 이 두 전략의 한계를 모두 해결하기 위해, quadruplet 형태의 unified post-training을 제안한다.

3. Method — Mobile-O 아키텍처

3.1 Baseline Mobile Unified Framework

Mobile-O의 기반은 기존 모바일 최적화 모델들을 결합하는 것에서 시작한다. 이해에는 FastVLM-0.5B, 생성에는 SANA-0.6B (DiT-style)를 사용한다.

VLM $f_\theta$ (FastVLM)와 diffusion decoder $g_\phi$ (SANA-0.6B)가 있을 때, text prompt $p$와 (이해를 위한) optional image $x$가 주어지면, VLM은 layer별 hidden state $\{H^{(1)}, \ldots, H^{(L)}\}$를 생성한다. 여기서 $H^{(\ell)} \in \mathbb{R}^{N \times d_{\text{vlm}}}$이며, $N$은 token 길이, $d_{\text{vlm}}$은 hidden size이다.

Diffusion model $g_\phi$는 cross-attention block을 가진 DiT-style decoder로, $d_{\text{cond}}$ 차원의 encoder feature를 받는다. ##핵심 설계 결정: SANA-0.6B가 원래 사용하던 Gemma-2B text encoder를 제거하고, 이해 모델의 LLM (Qwen2-0.5B)을 generation prompt 처리에도 공유##한다. 이를 통해 파라미터 효율이 크게 향상된다.

[Figure 2 설명]: Mobile-O 전체 아키텍처 개요. 왼쪽은 전체 framework — 효율적인 image encoder와 compact autoregressive LM이 visual understanding을 담당하고, lightweight linear DiT와 VAE encoder-decoder가 image generation을 수행한다. 오른쪽은 MCP 모듈의 상세 구조 — VLM의 마지막 K개 layer hidden state를 learnable weight로 융합하고, depthwise separable Conv1D, efficient channel attention, 그리고 output projection을 거쳐 diffusion model의 conditioning signal을 생성한다.

3.2 Mobile Conditioning Projector (MCP)

기존 unified framework는 VLM과 image decoder 사이에 learnable query token을 삽입하는 방식을 사용한다 (BLIP-3o, TBAC-UniImage 등). 이 방식은 대규모 모델에는 효과적이지만, ##대량의 pre-training 데이터 없이는 effective alignment이 어렵다##. Mobile-O는 이 대신 VLM hidden state를 diffusion decoder에 직접 연결하는 MCP를 설계한다.

Step 1: Layerwise Fusion

VLM의 마지막 $K$개 layer $S = \{L-K+1, \ldots, L\}$에서 hidden state를 추출하고, temperature-scaled softmax로 가중 결합한다:

$$\alpha_\ell = \frac{\exp(w_\ell / \tau)}{\sum_{j \in S} \exp(w_j / \tau)}, \quad H_{\text{fuse}} = \sum_{\ell \in S} \alpha_\ell H^{(\ell)} \in \mathbb{R}^{N \times d_{\text{vlm}}} \tag{1}$$

직관적으로, VLM의 서로 다른 깊이의 layer가 서로 다른 수준의 semantic 정보를 담고 있으므로, 학습 가능한 가중치 $\{w_\ell\}$이 각 layer의 기여도를 동적으로 조절한다. Temperature $\tau$는 학습 중 cosine annealing으로 조절된다.

Step 2: Compression and Refinement

융합된 표현을 compact space로 projection하고, depthwise-separable 1D convolution과 lightweight channel attention으로 정제한다:

$$\tilde{H} = \text{LN}(H_{\text{fuse}} W_c), \quad W_c \in \mathbb{R}^{d_{\text{vlm}} \times d_h} \tag{2}$$ $$\tilde{H} \leftarrow \text{SeqRefine}(\tilde{H}) \tag{3}$$

SeqRefine 블록은 depthwise-separable Conv1D → pointwise mixing → tiny MLP-based channel attention으로 구성된다. 핵심은 sequence length $N$ 방향(1D)으로 동작한다는 점이다. 즉, 2D spatial convolution이 아닌 token sequence 위의 1D convolution이므로, 비용이 크게 절감되면서도 language stream과의 token-level alignment이 유지된다.

Step 3: Output Projection

Diffusion model의 cross-attention이 기대하는 $d_{\text{cond}}$ 차원으로 최종 변환한다:

$$E = \text{LN}(\tilde{H} W_o), \quad W_o \in \mathbb{R}^{d_h \times d_{\text{cond}}}, \quad E \in \mathbb{R}^{N \times d_{\text{cond}}} \tag{4}$$

$g_\phi$의 모든 cross-attention layer가 동일한 sequence $E$를 encoder feature로 사용한다. 이는 latent diffusion에서 CLIP-conditioning과 유사하지만, VLM과 end-to-end로 학습된다는 점이 다르다.

복잡도 분석

Hidden size $d_h$와 kernel size $k$에 대해, refinement block의 비용은 token당 $O(k \cdot d_h)$ (depthwise) + $O(d_h^2)$ (pointwise)이다. 이는 full 2D convolution이나 새로운 query token에 대한 attention보다 훨씬 저렴하다. MLP connector의 3.3M 파라미터 대비 MCP는 2.4M으로 약 27% 감소하면서 더 높은 성능을 달성한다.

Mobile-O 아키텍처 흐름: [Image] → Image Encoder (FastViTHD) → Visual Tokens ↓ [Question] → Embedding Layer → Autoregressive LM (Qwen2-0.5B) → Answer (I2T Loss) ↓ (Hidden States: Layer N-3 ~ N) [MCP: Layerwise Fusion] ↓ (α-weighted sum) [Compress: Linear + LayerNorm] ↓ [SeqRefine: DW-Sep Conv1D + Channel Attn] ↓ [Output Projection → E] ↓ (Cross-Attention conditioning) [Gen. Prompt] → Embedding → LM → Gen. Tokens → Linear → DiT → VAE Decoder → [Generated Image] (T2I Loss)

3.3 Training Scheme — 3단계 학습

Mobile-O는 점진적으로 multimodal 능력을 강화하는 3단계 학습을 수행한다.

Stage 1: Cross-Modal Alignment

  • 목표: visual과 linguistic 표현 간 robust connection 수립
  • 데이터: JourneyDB 4M (다양한 visual concept의 고품질 text-image pair) + BLIP3o-Short-Caption 5M = 총 9M 샘플
  • Trainable: DiT + MCP만 학습. Visual encoder, LLM backbone, VAE는 freeze
  • 설정: LR 2e-4 (cosine → 2e-6), batch 512, DeepSpeed ZeRO-3, 50K steps (~3 epochs), ~3일 (8×A100)

Stage 2: Supervised Fine-tuning (SFT)

  • 목표: Stage 1 후 관찰된 특정 약점(복잡한 인체 동작, 일반 물체, 랜드마크)을 타겟팅하여 보완
  • 데이터: BLIP3o-60K + ShareGPT-4o-Image 45K = 105K curated prompt-image pair
  • Trainable: Stage 1과 동일 (DiT + MCP)
  • 설정: LR 2e-4 (cosine → 1e-6), batch 384, 20 epochs, ~15시간
  • 배경: Mobile-O의 Stage 1 pre-training corpus는 BLIP-3o의 20%에 불과하여, compact corpus의 coverage 부족을 보완하는 단계

Stage 3: Unified Multimodal Post-Training

  • 목표: ##이해와 생성 모두를 동시에 향상. 기존의 joint training이나 sequential training과 구별되는 핵심 기여##
  • 데이터: 105K quadruplet $\mathcal{S} = \{p, x_{\text{img}}, q, a\}$ (generation prompt, image, question, answer)
  • 데이터 구성 방법:
    1. GPT-4o로 각 이미지에 대한 highly detailed compositionally-aware caption 생성
    2. 다양한 visual understanding 측면을 다루는 diverse question-answer set 합성
  • Trainable: DiT + MCP + LLM + Visual Encoder (LoRA, r=16, α=32, dropout=0.1). VAE만 freeze
  • 설정: LR 1e-4 (cosine → 1e-6), batch 128, 7 epochs, ~5시간
[Figure 3 설명]: Unified Multimodal Post-Training 파이프라인 개요. 하나의 quadruplet 샘플에서 I2T loss (이해)와 T2I loss (생성)가 동시에 계산된다. Image encoder와 LLM은 LoRA로 학습하고, generation component (DiT, MCP, VAE Encoder)는 fully learnable이며, VAE Decoder만 freeze된다.

3.4 Training Objectives

전체 학습 목적함수는 이해와 생성 목적함수의 가중 결합이다:

$$\mathcal{L}_{\text{unified}} = \lambda_{\text{lang}} \mathcal{L}_{\text{lang}} + \lambda_{\text{diff}} \mathcal{L}_{\text{diff}} \tag{5}$$

Image-to-Text (I2T) Loss

표준 cross-entropy loss로 autoregressive LM의 출력 토큰을 학습한다:

$$\mathcal{L}_{\text{lang}} = -\sum_{t=1}^{|a|} \log P(a_t | x_{\text{img}}, q, a_{<t}) \tag{6}$$

직관적으로, 모델이 이미지와 질문을 보고 정답 토큰을 순차적으로 예측하도록 학습한다.

Text-to-Image (T2I) Loss

표준 noise prediction 대신 flow-matching objective를 사용한다. Flow matching은 noise 분포에서 data 분포로의 직선 경로를 학습하는 방법으로, 표준 diffusion 대비 더 빠르고 안정적인 학습을 제공한다.

Clean latent $x$ (VAE encoder 출력)과 noise $\epsilon \sim \mathcal{N}(0, I)$에서 noise level $\sigma \in [0, 1]$을 샘플링하여:

$$x_\sigma = (1 - \sigma)x + \sigma\epsilon, \quad v^\star(x_\sigma; \sigma) = \epsilon - x \tag{7}$$

DiT 모델이 MCP feature $c_p$로 conditioning된 velocity field $v_\phi(x_\sigma, \sigma, c_p)$를 예측한다:

$$\mathcal{L}_{\text{diff}} = \mathbb{E}_{x, p, \epsilon, \sigma} \left[ w(\sigma) \| v_\phi(x_\sigma, \sigma, c_p) - (\epsilon - x) \|_2^2 \right] \tag{8}$$

여기서 $w(\sigma)$는 scale-dependent weighting function이다. 직관적으로, 모델이 "현재 noisy 상태에서 noise를 빼고 원본 이미지를 더하는 방향(velocity)"을 예측하도록 학습하는 것이다. 이 formulation은 probability flow ODE를 직접 학습하므로, 표준 diffusion보다 빠르고 안정적이다.

5. Conclusion

Mobile-O는 unified vision-language-diffusion model로, quadruplet 형식의 unified post-training과 Mobile Conditioning Projector를 통해 에지 디바이스에서 고품질 이미지 이해와 text-to-image 생성을 달성한다. MacBook M2 Pro, Jetson Orin Nano, iPhone에서 기존 unified model 대비 latency와 memory efficiency 모두에서 우위를 보이면서 visual fidelity와 semantic accuracy를 유지한다. iPhone에서 2GB 이하 메모리 footprint로 ~3초 내 이미지 생성이 가능하여, 실시간 on-device 배포의 첫 실용적 사례를 제시한다.

Limitations

Mobile-O는 이해 모델의 경량 LLM을 text encoder로 재사용하며, 별도의 standalone language model을 사용하지 않는다. 이 설계는 메모리를 크게 줄이지만, text representation의 표현력을 제한할 수 있다. 예를 들어 SANA는 Gemma-2B-it을 전용 text encoder로 사용하여 더 강력한 linguistic backbone의 혜택을 받지만, Gemma-2B만으로 FP16에서 ~4GB 메모리가 필요하여 대부분의 모바일 디바이스에서는 비현실적이다.

비판적 분석 (Critical Analysis)

강점

  • 실용성 중심의 연구: "모바일에서 실시간 동작하는 unified model"이라는 명확한 목표와 실제 iPhone/Jetson 배포 결과는 학술적 기여를 넘어 산업적 의미가 크다.
  • 체계적인 ablation: MCP의 각 component, layer 수, post-training 형식까지 꼼꼼히 분리하여 분석했다.
  • 데이터 효율성: BLIP3o의 20%에 불과한 pre-training data로 경쟁력 있는 성능을 달성한 점은 실용적 가치가 높다.
  • 코드/모델/데이터 전체 공개: 재현 가능성이 높다.

약점 및 한계

  • 벤치마크 범위: 생성은 GenEval 단일 벤치마크로만 평가. FID, CLIP score 등 다양한 지표와 COCO 같은 추가 벤치마크가 없다. GenEval의 object-focused 특성상, 복잡한 compositional generation이나 photorealistic quality 평가에는 한계가 있다.
  • 표준편차/반복 실험 미보고: 메인 결과에 표준편차가 없어 통계적 유의성 확인이 어렵다. Edge deployment latency (Table 6)에만 ± 표기가 있다.
  • 공정한 비교 문제: JanusFlow (2.1B)와 비교 시 파라미터 수 차이(1.6B vs 2.1B)가 있다. 같은 1.6B 규모에서의 비교는 Show-O-Clip-ViT뿐이며, 이 모델은 상대적으로 약한 baseline이다.
  • 해상도 제한: 생성이 512×512로 제한된다. SANA-0.6B 자체는 4096×4096까지 지원하지만, Mobile-O에서는 모바일 제약으로 512로 축소했다. 현대 사용자 기대치(1024+)와 gap이 있다.
  • Quadruplet 데이터 구성의 확장성: GPT-4o를 사용하여 caption과 QA를 생성하는 과정의 비용과 품질 의존성에 대한 분석이 부족하다.
  • Counting 성능: GenEval counting (0.57)이 Show-O-Clip-ViT (0.67)보다 낮아, compositional understanding의 특정 측면에서 약점이 있다.

Key Takeaways

  1. Mobile deployment의 실현 가능성: Unified multimodal model이 2GB 이하 메모리로 iPhone에서 실시간 동작한다는 것은, on-device AI의 새로운 가능성을 열어준다. 프라이버시 보존, 오프라인 동작, 저지연이 필요한 실무 애플리케이션에 직접 적용 가능하다.
  2. Learnable query token의 대안: MCP는 VLM→diffusion 연결에서 learnable query token 없이 direct projection + layerwise fusion으로 더 적은 데이터와 파라미터로 효과적인 alignment을 달성한다. 다른 cross-modal 연결 설계에 참고할 수 있는 패턴이다.
  3. Unified post-training의 효과: Disjoint dataset이 아닌 quadruplet 형태로 하나의 샘플에서 이해+생성을 동시에 학습하는 것이 작은 데이터(105K)로도 양쪽 성능을 모두 올린다는 결과는, multi-task learning 전략 설계에 유용한 인사이트이다.
  4. 기존 효율적 모델의 조합: Mobile-O는 완전히 새로운 아키텍처가 아니라 FastVLM + SANA라는 기존 효율적 모델을 잘 결합한 것이다. 이는 "efficient component를 잘 연결하고 unified training하면 1+1 > 2"라는 실용적 교훈을 준다.
  5. Scaling 가능성: Mobile-O-1.5B 실험이 보여주듯 framework가 architecture-agnostic하므로, 향후 더 강력한 mobile backbone이 나오면 plug-in 식으로 교체하여 성능을 높일 수 있다.
