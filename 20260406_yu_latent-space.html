2. Foundation: What is Latent Space?

2.1 Concept: Explicit Space vs. Latent Space

4.1.1 Backbone ($\Phi^{\text{back}}$)

Latent computationмқҙ backbone мһҗмІҙм—җ intrinsically embeddedлҗңлӢӨ:

$$\mathbf{h}_{t+1} = \Phi^{\text{back}}(\mathbf{h}_{1:t}, \mathbf{x}, \mathbf{y}_{1:t}) \tag{3}$$

3к°Җм§Җ м„Өкі„ кҙҖм җ:

(a) Parameter-shared Backbone: к№ҠмқҖ кі мң  л Ҳмқҙм–ҙ мҠӨнғқмқ„ мҶҢк·ңлӘЁмқҳ мһ¬мӮ¬мҡ© к°ҖлҠҘ лӘЁл“ҲлЎң лҢҖмІҙн•ңлӢӨ.

  • Huginn: decoder-only architectureм—җм„ң shared recurrent blockлҘј multiple depth stepм—җ кұёміҗ мһ¬мӮ¬мҡ©. кі мң  parameter мҲҳлҘј нҒ¬кІҢ мӨ„мқҙл©ҙм„ң inference-timeм—җ м¶”к°Җ recurrent stepмңјлЎң test-time compute scaling к°ҖлҠҘ.
  • Looped Transformers: explicit layer-looping mechanism к°•м ң. Looping-based regularizationмқҙ л°ҳліө ліҖнҷҳ н•ҳм—җм„ң hidden-state dynamicsлҘј м•Ҳм •нҷ”.
  • PHD-Trans.: sliding-window attentionкіј cache managementлЎң long-context н•ҳм—җм„ң recurrent computationмқҳ мӢӨмҡ©м„ұ мһ…мҰқ.

(b) Iterative Backbone: variable лҳҗлҠ” learnable depth allocationмңјлЎң dynamic updating:

  • Ouro: recursive inference framework вҖ” iterationмқ„ architectural depth лҢҖмІҙ scaling axisлЎң нҷңмҡ©, parameter-shared transformationмқҳ л°ҳліө м Ғмҡ©мқҙ мқјкҙҖлҗң gains мӮ°м¶ң
  • LoopFormer: elastic-depth looped transformer вҖ” loop iteration мҲҳк°Җ кі м •лҗҳм§Җ м•Ҡкі  input/budgetм—җ л”°лқј к°ҖліҖ
  • PonderLM2: Jacobi-style parallel updateлЎң iterative refinement вҖ” к°Ғ forward passк°Җ н•ң tokenм”© м§„н–үн•ҳл©ҙм„ңлҸ„ multi-step hidden-state evolutionмқ„ лі‘л ¬ мҲҳн–ү

(c) Augmented Backbone:

  • Dreamer, MLRA: sequence-depth sparse attention mixtureмҷҖ low-rank attentionмңјлЎң multi-step latent transitionмқҳ кі„мӮ° tractability нҷ•ліҙ

4.2.1 Internal Representation

Backboneмқҳ standard forward passм—җм„ң мғқм„ұлҗҳлҠ” endogenous activationм—җм„ң м§Ғм ‘ мң лҸ„лҗңлӢӨ. 3к°Җм§Җ мЈјмҡ” нҳ•нғң:

(a) Hidden State: к°ҖмһҘ ліҙнҺём Ғ нҳ•нғң. л§Ҳм§Җл§ү hidden state лҳҗлҠ” mean-pooled representation:

$$\mathbf{z} = \mathbf{h}_L^T \quad \text{or} \quad \frac{1}{T}\sum_{t=1}^{T}\mathbf{h}_l^t \tag{8}$$

COCONUTмқҙ л§Ҳм§Җл§ү hidden stateлҘј лӢӨмқҢ input embeddingмңјлЎң м§Ғм ‘ н”јл“ңл°ұн•ҳлҠ” foundational patternмқ„ нҷ•лҰҪн–ҲлӢӨ.

(b) Weighted Embedding: hard token samplingмқ„ soft, probability-weighted combinationмңјлЎң лҢҖмІҙ:

$$\mathbf{z} = \mathbf{E}^\top \boldsymbol{\alpha}, \quad \boldsymbol{\alpha} = \text{softmax}(\mathbf{o}) \tag{9}$$

$\boldsymbol{\alpha}$к°Җ probability simplex мң„м—җ мһҲмңјлҜҖлЎң $\mathbf{z}$лҠ” vocabulary embedding vectorмқҳ convex hullм—җ м ңм•ҪлҗңлӢӨ. мқҙ differentiable relaxationмқҖ candidate token embeddingмқҳ superpositionмқ„ нҳ•м„ұн•ҳм—¬, discrete generation stepмқ„ нҶөн•ҙ gradient flowлҘј к°ҖлҠҘн•ҳкІҢ н•ҳкі  reasoning spaceмқҳ parallel explorationмқ„ мҙүм§„н•ңлӢӨ.

(c) Cache: м¶•м Ғлҗң key-value pairлҘј structured latent memoryлЎң нҷңмҡ©:

$$\mathbf{k}_l = \mathbf{H}_l\mathbf{W}_l^k, \quad \mathbf{v}_l = \mathbf{H}_l\mathbf{W}_l^v \tag{10}$$

SALS: KV cache мң„мқҳ sparse attention pattern нҷңмҡ©, LatentMAS: KV cacheлҘј shared working memoryлЎң мӮ¬мҡ©н•ҳм—¬ agent к°„ explicit text м—Ҷмқҙ coordination.

4.2.2 External Representation

Structurally independent auxiliary encoder $\Phi^{\text{aux}}$м—җм„ң мң лһҳ:

$$\mathbf{z} = \Phi^{\text{aux}}(\mathbf{x}_{\text{aux}}) \tag{11}$$

Backboneм—җ нҶөн•©лҗҳлҠ” 2к°Җм§Җ mode: (1) learned projection $\psi$лҘј нҶөн•ҙ backboneмқҳ native latent spaceлЎң м •л ¬ нӣ„ conditioning inputмңјлЎң мӮ¬мҡ©, (2) dense, non-verbalized supervision targetмңјлЎң мӮ¬мҡ© (backboneмқҳ internal state $\mathbf{h}$к°Җ projected external signalмқ„ matchн•ҳлҸ„лЎқ н•ҷмҠө).

кё°лҠҘм Ғ м—ӯн• м—җ л”°лқј 3к°Җм§Җ л°©н–Ҙ:

  • Reasoning Priors: expert reasoning modelм—җм„ң knowledge distillation (CODI: self-distillation loop, KaVa: compressed KV cache м§Ғм ‘ м „лӢ¬)
  • Perceptual Priors: pre-trained vision encoderк°Җ spatially/temporally rich feature map кіөкёү (3DThinker: 3D geometric prior, SkiLa: sketch token, Mull-Tokens: modality-agnostic latent token)
  • Embodied Priors: world model/3D encoderк°Җ environmental dynamics/geometryмқҳ structured latent representation мғқм„ұ (OccVLA: 3D occupancy token, LCDrive: action+scene token)

4.2.3 Learnable Representation

Backbone architecture лӮҙм—җ м§Ғм ‘ embeddedлҗң parameterized module $\Phi^{\text{comp}}$мқҙ end-to-end optimized:

$$\mathbf{z} = \Phi^{\text{comp}}(\mathbf{c}; \theta) \tag{13}$$

3к°Җм§Җ н•ҷмҠө нҳ•нғң:

  • Compression Learning: information-bottleneck principleлЎң explicit dataлҘј dense continuous vectorлЎң м••м¶• (CoLaR: variance-preserving scalingмңјлЎң reasoning token aggregation, LIVR: visual bottleneckмңјлЎң implicit spatial compression н•ҷмҠө)
  • Distribution Learning: deterministic point mapping лҢҖмӢ  stochastic distributionкіј structural manifoldлҘј capture (CTRLS: Dirichlet distributionмңјлЎң state transition modeling, MARCOS: conditional hidden Markov model)
  • Alignment Learning: cross-space projectionмңјлЎң rigid boundary м—°кІ° (KVCA: cross-attentionмңјлЎң shared latent manifold н•ҷмҠө, C2C: neural MLPлЎң heterogeneous лӘЁлҚё к°„ KV cache м§Ғм ‘ projection)

4.2.4 Hybrid Representation

Learnable moduleлЎң кө¬м„ұн•ҳлҗҳ, backboneкіј architecturally disjointн•ҳкІҢ мң м§Җ:

$$\mathbf{z} = \Phi^{\text{comp}}(\mathbf{c}; \theta) \tag{14}$$

$\Phi^{\text{comp}}$мқҙ External paradigmмІҳлҹј exogenous conditioning signal лҳҗлҠ” latent distillation targetмңјлЎң л°°м№ҳлҗңлӢӨ. 3к°Җм§Җ кё°лҠҘ:

  • Traces: discrete reasoning trajectoryлҘј compact continuous vectorлЎң distill (HCoT, Assorted, Latent-SFT, ThoughtComm)
  • Grounding: continuous sensory/control signalмқ„ structured latent tokenмңјлЎң ліҖнҷҳ (AURORA: VQ-VAEлЎң discrete visual latent code мғқм„ұ, UniVLA: heterogeneous videoм—җм„ң unified action spaceлЎң projection)
  • Augmentation: large contextual historyлҘј compressed soft prompt/augmented cache stateлЎң м••м¶• (DCA: offline coprocessorлЎң latent embedding мғқм„ұ нӣ„ KV cacheм—җ м§Ғм ‘ append, VisMem/CoMEM: dedicated visual memory token)

4.3 Computation: Latent Spaceм—җм„ң м–ҙл–Ө м—°мӮ°мқҙ мҲҳн–үлҗҳлҠ”к°Җ?

Fixed-depth, token-by-token generationмқҙлқјлҠ” conventional paradigmм—җм„ң лІ—м–ҙлӮҳ, latent spaceм—җм„ң лҚ” мң м—°н•ҳкі  нҡЁмңЁм Ғмқҙл©° нҷ•мһҘ к°ҖлҠҘн•ң computational mechanismмқ„ кө¬м¶•н•ҳлҠ” л°©лІ•мқ„ 4к°Җм§ҖлЎң л¶„лҘҳн•ңлӢӨ.

[Figure 7 м„ӨлӘ…]: Computation mechanismмқҳ 4 sub-type. Compressed: volume к°җмҶҢ, Expanded: depth/width нҷ•мһҘ, Adaptive: dynamic/selective updating, Interleaved: explicit-latent/text-vision/reasoning-memoryмқҳ heterogeneous mediums көҗм°Ё.

4.3.1 Compressed Computation

Explicit trajectory $\mathbf{r}$мқҙлӮҳ high-dimensional hidden state $\mathbf{h}$лҘј semantically dense latent representation $\mathbf{z}$лЎң нҲ¬мҳҒ:

$$\mathbf{z} = \Phi(\mathbf{h}), \quad |\mathbf{z}| \ll |\mathbf{h}| \tag{15}$$

3к°Җм§Җ м••м¶• лҢҖмғҒ:

  • Traces Compression: explicit reasoning trace кёёмқҙлҘј мӨ„мқҙл©ҙм„ң inferential fidelity ліҙмЎҙ (HCoT: hierarchical compression, SoftCoT: continuous soft-token spaceлЎң projectionн•ҳм—¬ reasoning depthмҷҖ discrete trace length л¶„лҰ¬, CCoT: variable-length latent allocation)
  • States Compression: internal cacheмқҳ linear м„ұмһҘ н•ҙкІ° (KaVa: knowledge distillationмңјлЎң KV cache м••м¶•, SALS: low-rank principal subspace projection, DeltaKV: successive cache к°„ semantically compressed residual м ҖмһҘ)
  • Features Compression: multimodal/embodied settingмқҳ high-dimensional visual/action representation м••м¶• (RoT: intermediate reasoning stateлҘј low-resolution image patchлЎң render, OneLatent: single latent visual tokenмңјлЎң competitive performance, LatentVLA: compact action latent for real-time planning)

4.3.2 Expanded Computation

Latent computationмқ„ depth лҳҗлҠ” width м°ЁмӣҗмңјлЎң нҷ•мһҘ:

$$\mathbf{h}_{t+1}^{(k)} = \Phi\left(\left\{\mathbf{h}_t^{(k)}\right\}_{k=1}^K\right), \quad t = 1,\ldots,T, \; k = 1,\ldots,K \tag{16}$$
  • Depth Expansion: лҸҷмқјн•ҳкұ°лӮҳ кө¬мЎ°м ҒмңјлЎң мң мӮ¬н•ң layerлҘј multiple recurrent passм—җ кұёміҗ мһ¬мӮ¬мҡ© (Huginn: recurrent depth, Ouro: recursive inferenceмқҳ scaling axis, LoopFormer: input-adaptive looping, ETD: encode-think-decodeм—җм„ң latent thought stateлҘј л°ҳліө refine)
  • Width Expansion: multiple parallel hypothesis/trajectoryм—җ кұёміҗ compute н• лӢ№ (SoftCoT++: continuous embedding spaceм—җм„ң multiple parallel reasoning pathлҘј instantiateн•ҳм—¬ zero-shot performance gainмқҙ path мҲҳм—җ monotonically scale, LatentTTS: learned value functionкіј concurrent latent tree search, PCCoT: synchronization pointм—җм„ң м •ліҙ көҗнҷҳн•ҳл©° parallel latent chain)
  • Structural Expansion: depth/widthлҘј лӢЁмҲң мҰқк°ҖмӢңнӮӨлҠ” лҢҖмӢ  мғҲлЎңмҡҙ topological primitive лҸ„мһ… (Latent-SFT: superposed latent chainмқ„ supervisionн•ҳм—¬ token-level realization м—Ҷмқҙ reasoning performance ліҙмЎҙ, ColaVLA: parallel but interactive high-level semantic reasoning + low-level motor planning streamмқ„ cross-attentionмңјлЎң м—°кІ°)

4.3.3 Adaptive Computation

Recurrence depth $T$мҷҖ trajectory width $K$лҘј input complexityм—җ мЎ°кұҙл¶ҖлЎң кІ°м •:

$$(T, K) = \mathcal{T}(\mathbf{h}_t; \mathbf{x}) \tag{18}$$
  • Depth/Width Adaptation: TaH (think-and-halt, MLP deciderк°Җ early exit), LWS (learnable halting policy), PLaT (latent-chain guidanceлЎң termination н•ҷмҠө), Dreamer/SpiralFormer (weight-tied recurrent transformerлЎң depth adaptivity)
  • Semantic Adaptation: к°ңлі„ token/semantic unitм—җ different budget н• лӢ№ (AL-CoT: token-level adaptation, PonderLM-3/AdaPonderLM: learned depth/haltingмңјлЎң token-wise heterogeneous computation, DLCM: tokenм—җм„ң semantically coherent conceptмңјлЎң adaptation лӢЁмң„ мқҙлҸҷ)
  • Control Adaptation: activation vector, routing policy, extraction mechanism л“ұмңјлЎң computation мЎ°м Ҳ (System-1.5: intermediate transformer block bypass, FR-Ponder: instance-adaptive activation steering, RISER: activation spaceмқҳ steering directionмңјлЎң composition)

4.3.4 Interleaved Computation

Discrete token embedding ($\mathcal{V}$)кіј continuous latent ($\mathcal{H}$)лҘј көҗм°Ён•ҳм—¬ heterogeneous generation sequence кө¬м„ұ:

$$\mathbf{r} = [r_1, z_1, r_2, z_2, \ldots, r_M, z_N], \quad r_i \in \mathcal{V}, \; z_j \in \mathcal{H} \tag{19}$$

3к°Җм§Җ interleaving мң нҳ•:

  • Explicit-latent Interleaving: natural-language tokenкіј latent internal stateлҘј single trajectory лӮҙм—җм„ң көҗм°Ё (Assorted: selected verbal CoT stepмқ„ latent activationмңјлЎң көҗмІҙн•ҙлҸ„ downstream accuracy ліҙмЎҙ, SpiralThinker: spiral curriculumмңјлЎң м җм§„м Ғ лӮҙмһ¬нҷ”, SwiReasoning: RLлЎң step-level switching policy н•ҷмҠө)
  • Modality Interleaving: continuous perceptual latentкіј linguistic tokenмқ„ көҗм°Ён•ҳм—¬ latent domainм—җм„ң intermediate reasoning мҲҳн–ү (AURORA: explicit perceptual reasoning step мӮҪмһ…, Mirage/LVR/VisMem/Monet: vision latentкіј textлҘј м§Ғм ‘ interleaveн•ҳм—¬ textual scene description мқҳмЎҙ м ңкұ°, IVT-LR: cross-attentionмңјлЎң textмҷҖ vision activation coupling)
  • Task Interleaving: memory store, retrieval component, generative head, agent sub-process л“ұ heterogeneous functional moduleмқ„ мЎ°мңЁ (LCR-SER: compressed history bufferмҷҖ ongoing inference interleave, MemGen/VisMem/FlashMem: persistent working memoryмҷҖ generation interleave, LatentMem: shared latent memoryлЎң multi-agent implicit communication, L2-VMAS: perceptual + planning streamмқ„ agent к°„ көҗм°Ё)

4.4 Optimization: Latent SpaceлҠ” м–ём ң, м–ҙл–»кІҢ мөңм Ғнҷ”лҗҳлҠ”к°Җ?

3к°Җм§Җ лӢЁкі„(pre-training, post-training, inference)м—җм„ң мөңм Ғнҷ”лҗҳл©°, к°Ғ лӢЁкі„лҠ” мөңм Ғнҷ” лҢҖмғҒ(л¬ҙм—Үмқҙ ліҖн•ҳлҠ”к°Җ)кіј supervision/objectiveк°Җ лӢӨлҘҙлӢӨ.

4.4.1 Pre-training

$$\theta^* = \arg\min_{\theta \in \Phi} \mathbb{E}_{(\mathbf{x},\mathbf{y})\sim\mathcal{D}} \left[\mathcal{L}(\mathbf{x}, \mathbf{y}, \mathbf{z}; \Phi_\theta)\right] \tag{20}$$
  • Autoregressive Supervision: next-token predictionмқҙлӮҳ recurrent loopingмқҳ мһҗм—°мҠӨлҹ¬мҡҙ л¶ҖмӮ°л¬јлЎң reasoning лӮҙмһ¬нҷ” (PonderLM-2: Jacobi-iteration кё°л°ҳ parallel training, Looped Trans.: continuous latent updateлЎң complex reasoning, Ouro: entropy-regularized objective)
  • Auxiliary Supervision: auxiliary semantics, contrastive learning, reconstruction objectiveлЎң latent space geometry лӘ…мӢңм Ғ мң лҸ„ (CoCoMix: cross-entropy + reconstruction loss, LARES: InfoNCE + cross-entropy, LAPA: VQ-VAEлЎң latent action pretraining, JALA: inverse dynamicsм—җ м •л ¬лҗң predictive action embedding)
  • Reinforcement: pre-training лӢЁкі„м—җм„ң reward signalмқ„ м§Ғм ‘ нҶөн•© (LoopRPT: noisy latent rolloutмңјлЎң next-token predictionмқ„ reasoning taskлЎң reframe)

4.4.2 Post-training

$$\theta^* = \arg\min_{\theta \in \mathcal{W}} \mathbb{E}_{(\mathbf{x},\mathbf{y})\sim\mathcal{D}} \left[\mathcal{L}(\mathbf{x}, \mathbf{y}, \mathbf{z}; \Phi_\theta)\right] - \beta\,\mathbb{E}_{\mathbf{r}\sim\Phi_\theta(\cdot|\mathbf{x})} \left[R(\mathbf{x}, \mathbf{r}, \mathbf{z})\right] \tag{21}$$
  • Explicit Supervision Fine-tuning: final human-readable outputм—җ loss function м Ғмҡ© (LATPC: continuous variable fine-tuningмңјлЎң safety к°•нҷ”, GeoSteer: learned latent manifold мң„лЎң hidden state steering)
  • Implicit Supervision Fine-tuning: knowledge distillation, contrastive alignment, step-wise reconstruction (SPOT: explicit reasoning trajectoryлҘј compact latent tokenмңјлЎң teacher spanм—җ anchoring, Latent-SFT: latent tokenмқ„ vocabulary-space superpositionмңјлЎң мһ¬м •мқҳн•ҳм—¬ KL divergence + cross-entropy н•ҷмҠө, LTA-Thinker/EPR-Latent: contrastive objectiveлЎң latent thought distribution мөңм Ғнҷ”)
  • Reinforcement Learning: policy gradient, reward, preference signalлЎң latent trajectory мһҗмңЁ л°ңкІ¬ (LaTRO: variational sampling process, I2B-LPO: iterative information bottleneckмңјлЎң latent policy exploration, SofT-GRPO: Gumbel reparameterizationмңјлЎң stable group relative policy optimization, ReLaX: strict reward regularizationмңјлЎң premature convergence л°©м§Җ)
[Table 6 мҡ”м•Ҫ]: 70+ л°©лІ•мқҳ optimization л¶„лҘҳ. Pre-training(14к°ң), Post-training(38к°ң), Inference(20к°ң). к°Ғк°Ғмқҳ modality, backbone, supervision мң нҳ•(CE, KL, Recon., InfoNCE, reward, self-reward л“ұ), objective, scenarioлҘј м •лҰ¬.

4.4.3 Inference

лӘЁлҚё к°ҖмӨ‘м№ҳ $\theta$лҠ” кі м •(лҳҗлҠ” trained)лҗҳкі , latent state $\mathbf{z}$ мһҗмІҙк°Җ мөңм Ғнҷ” ліҖмҲҳ:

$$\mathbf{z}^* = \arg\min_{\omega \in \Omega} \mathcal{J}(\mathbf{z}; \mathbf{x}, \Phi_\theta) \tag{22}$$
  • Inference Scaling: multiple candidate trajectory мғқм„ұ + reward-based heuristicмңјлЎң мөңм Ғ path м„ нғқ (LTO: continuous-space classifierлҘј latent reward modelлЎң нҷңмҡ©, LatentSeek: self-rewardлЎң catastrophic forgetting мҷ„нҷ”, SR: self-rewardлЎң soft reasoning exploration м •л ¬)
  • Inference Tuning: trial-and-errorм—җм„ң continuous, gradient-based optimizationмңјлЎң м „нҷҳ (LTPO: online policy gradient + intrinsic confidence-based reward, $\nabla$-Reasoner: decoding loop лӮҙм—җм„ң gradient descentлЎң continuous sample space м§Ғм ‘ refine, LatentEvolve: self-reward + reconstruction loss кІ°н•©, test-time scaling)
  • Inference Guidance: targeted structural constraint, contrastive logic, sparse interventionмңјлЎң latent representationмқ„ лҸҷм Ғ мң лҸ„ (REVIS: pure visual information vector м¶”м¶ңлЎң object hallucination мҷ„нҷ”, STIR: value-modulated trajectory intervention, SoftCoT++: contrastive learningмңјлЎң soft representation лӢӨм–‘м„ұ к°•м ң, VTI: contrastive learningмңјлЎң massive visual hallucination мҷ„нҷ”)
