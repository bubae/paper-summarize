TL;DR

  • 문제: 기존 speculative decoding(EAGLE-3 등)은 draft 모델이 여전히 자기회귀적(autoregressive)으로 토큰을 한 개씩 생성하기 때문에 drafting 자체가 직렬 병목이며, 모델 capacity 제약으로 acceptance length가 빠르게 포화되어 실용 속도 향상이 2~3배에 그친다.
  • 해결: drafter를 ##경량 block diffusion 모델##로 교체하여 한 번의 forward pass로 γ개의 draft 토큰을 병렬 생성하고, target 모델의 중간 hidden feature를 KV cache에 직접 inject하여 모든 draft layer가 풍부한 컨텍스트로 조건화되게 한다.
  • 왜 작동하는가: drafting cost가 토큰 수에 무관해지므로 draft 모델을 더 깊게(5-layer 등) 만들어 quality(acceptance length τ)와 latency를 동시에 잡을 수 있다. KV injection은 deeper layer까지 target context를 잃지 않게 보존한다.
  • 비유: EAGLE-3가 "글자를 한 개씩 받아 적는 비서"라면, DFlash는 "초안 전체 빈칸을 한 번에 채우는 비서". 그것도 상사(target)의 메모(hidden feature)를 직접 참조하면서.

핵심 기여 (Key Contributions)

  1. Diffusion drafter + speculative verification 결합: diffusion LM의 약점(생성 품질 저하, 많은 denoising step 필요)을 우회하면서 강점(parallel 생성)만 가져온다. 검증은 lossless이므로 품질은 target 모델로 보장.
  2. KV injection 조건화: target hidden feature를 모든 draft layer의 Key/Value에 영구 주입. EAGLE-3의 입력 단 fusion 대비 깊은 모델에서도 acceptance length가 계속 증가.
  3. Anchor 기반 random block sampling: 표준 block diffusion training과 달리, response에서 anchor 위치를 무작위 샘플링하고 그 다음 block_size−1 토큰을 마스킹. inference 시 bonus token에서 시작하는 패턴과 정확히 일치 + 데이터 증강 효과.
  4. Position-dependent loss decay: block 내 앞쪽 위치 오류가 뒤쪽 토큰을 모두 무효화하므로 $w_k = \exp(-(k-1)/\gamma)$로 가중치를 줘서 acceptance length를 균등 가중 대비 향상.
  5. 실증적 압도: ##Qwen3-8B에서 EAGLE-3 대비 평균 2.4×, baseline 대비 최대 6.1× speedup##. SGLang(B200) production setup에서 concurrency 1~32 전 구간에서 EAGLE-3 대비 우위.

1. 배경: 왜 또 speculative decoding인가?

1.1 Speculative decoding의 한계

Speculative decoding은 작은 draft model이 γ개의 후보 토큰을 제안하면, 큰 target model이 이를 병렬로 검증해 정확하면 받아들이고 틀리면 잘라내는 방식이다 (Leviathan et al., 2023). EAGLE 시리즈는 draft 모델이 target의 hidden feature를 활용하도록 발전해 왔으며, EAGLE-3는 SOTA로 약 2~3× speedup을 달성한다. 그러나 ##drafter 자체가 transformer 1-layer를 자기회귀적으로 γ번 돌려야 하기 때문에 drafting cost가 γ에 선형 증가##하고, 1-layer라는 얕은 capacity 때문에 acceptance length가 빠르게 포화된다.

1.2 Diffusion LM (dLLM)

Diffusion language model (LLaDA, Nie et al. 2025 등)은 마스킹된 토큰을 병렬로 denoise하는 방식으로 한 번에 여러 토큰을 생성한다. Fully parallel diffusion은 fixed-length 생성과 KV cache 부재 등의 문제로 AR 대비 quality가 낮고, Block diffusion (Arriola et al. 2025, Fast-dLLM v2, SDAR)이 그 절충점으로 등장했다. 하지만 standalone dLLM은 여전히 많은 denoising step과 품질 손실을 동반한다.

1.3 결합 동기

저자들은 핵심 질문을 다음과 같이 던진다: "Is there truly no free lunch? Can we build a diffusion drafter that is both lightweight and highly accurate?" 답은 두 가지 관찰에서 출발한다.

  • The target knows best: 큰 AR 모델의 hidden feature는 미래 토큰에 대한 정보를 implicitly 인코딩한다(Samragh et al. 2025). 따라서 drafter가 처음부터 추론할 필요 없이 ##target의 깊은 context feature를 그대로 빌려쓰면 된다##.
  • Drafting cost와 토큰 수의 디커플링: diffusion drafter의 draft cost는 γ에 거의 무관하므로 draft 모델을 더 깊게 만들 자유가 생긴다. quality와 latency를 동시에 잡는다.

2. Preliminaries: speedup 식과 AR vs Diffusion drafting

2.1 Speculative decoding의 평균 토큰당 latency

Target model $\mathcal{M}_t$, draft model $\mathcal{M}_d$. 각 cycle마다 drafter가 γ개 토큰을 제안하고, $\tau \in [1, \gamma+1]$개가 accept된다 (bonus token 1개 포함). 평균 토큰당 latency는

$$L = \frac{T_{\text{draft}} + T_{\text{verify}}}{\tau} \quad (1)$$

$L_{\text{target}}$을 target의 AR 토큰당 latency라 할 때 speedup은 $\eta = L_{\text{target}} / L$. 직관적으로 말하면, speedup을 키우려면 ##acceptance length τ를 늘리거나 drafting overhead $T_{\text{draft}}$를 줄이거나## 둘 중 하나여야 한다.

2.2 AR vs Diffusion drafting cost

Autoregressive drafter의 cost는 forward pass latency $t_{\text{step}}$을 γ번 반복하므로

$$T_{\text{draft}}^{\text{AR}} = \gamma \cdot t_{\text{step}} \quad (2)$$

이를 관리 가능 수준으로 유지하기 위해 EAGLE-3는 drafter를 단 1 transformer layer로 제한한다. 결과적으로 draft quality(acceptance length) 또한 capacity 부족으로 saturate.

Diffusion drafter는 γ개 토큰을 단일 forward pass로 생성:

$$T_{\text{draft}}^{\text{diff}} = t_{\text{parallel}} \quad (3)$$

현대 GPU에서 parallel op는 $t_{\text{parallel}} \ll \gamma \cdot t_{\text{step}}$. 따라서 ##moderate block size에서 drafting cost가 γ에 거의 둔감##. 결과적으로 5-layer DFlash가 EAGLE-3 1-layer보다 더 낮은 latency와 더 높은 acceptance length를 동시에 달성한다(Figure 3 참조).

3. 방법 (Method)

3.1 Inference 디자인 (Figure 2 요약)

[Figure 2 설명]: 시스템은 prompt가 들어오면 (a) Target model이 prefill을 돌려 hidden feature를 생성, (b) 선택된 5개 layer의 hidden state를 concat → projection → "fused target context feature"로 압축, (c) 이 feature를 모든 draft layer의 K/V로 inject, (d) Draft layer들이 mask token 위치를 bidirectional attention으로 한 번에 채움, (e) Target LM head로 sampling, (f) Target model이 병렬 verify.
[1] Prefill: Target model이 prompt 전체를 forward pass → hidden_state from layers {l_1, …, l_5} [2] Fusion: H_t = RMSNorm(W_c · [H^(l1); …; H^(l5)]) ∈ R^(D) (target hidden dim → draft hidden dim으로 한 번에 매핑) [3] KV injection: 각 draft layer i에서 Q_i = W_i^Q H_d (draft 토큰만 query) K_i = [W_i^K H_t ; W_i^K H_d] ← target feature를 K에 prepend V_i = [W_i^V H_t ; W_i^V H_d] ← V에도 동일 [4] Parallel draft: mask 위치 γ개를 bidirectional attention으로 한 번에 예측 [5] Sampling: Target LM head 공유 (parameter 절약) [6] Verify: Target model이 γ개 draft를 단일 forward pass로 검증, τ개 accept

3.2 KV injection이 acceptance length를 scale시키는 이유

EAGLE-3는 target hidden feature를 draft 모델의 token embedding과 입력 단에서 fusion한다. layer가 깊어질수록 이 정보가 dilute되어 acceptance length가 5-layer 이상에서 saturate된다.

DFlash는 fused context feature를 ##persistent contextual information##으로 취급해 모든 draft layer의 K/V에 직접 주입한다. cache되어 drafting iteration에 걸쳐 재사용. 결과적으로 draft 모델은 매 layer마다 강한 conditioning을 받으며, acceptance length가 layer 수에 비례해 증가한다(Table 9 ablation).

3.3 Parallel diffusion drafting

모든 mask 토큰이 단일 forward pass에서 bidirectional attention으로 결정된다. AR drafter의 다중 sequential pass 대비 drafting latency가 큰 폭으로 줄고, 더 깊은 draft 모델을 감당할 수 있다.

3.4 Training

3.4.1 KV injection (training-time)

Inference pipeline과 동일하게, clean prompt+response 전체를 target model에 통과시켜 hidden features를 추출·fuse → draft 모델의 K/V로 주입. clean과 masked 부분 모두에 대해 동일.

3.4.2 Anchor 기반 random block sampling (Figure 4 요약)

[Figure 4 설명]: 표준 block diffusion training은 response를 균등 block으로 나누고 각 block 내 임의 위치를 마스킹한다. DFlash는 (a) response에서 ##anchor 토큰을 random sampling##하고, (b) 각 anchor를 block의 첫 번째 위치로 두고, (c) 그 뒤 block_size−1 위치를 모두 mask. 따라서 draft 모델은 inference에서 직면할 패턴인 "직전 bonus token (clean) + 그 뒤 모두 mask"를 정확히 학습한다.

장점:

  • Inference-time 일치: bonus token 패턴과 정확히 매칭 (mismatch 제거)
  • 데이터 증강: 같은 sequence에서 매 epoch마다 다른 anchor 샘플링 → coverage 향상
  • Diverse target context: 다양한 anchor → 다양한 target feature에 노출

Table 13: ##standard sampling 4.13× → anchor sampling 4.69× (Math500)##, τ는 4.94→5.64.

3.4.3 Sparse attention mask (training 효율)

여러 mask block을 단일 sequence에 concat하고, block 간 attention을 차단(invisible token)하는 sparse mask를 Flex Attention(Dong et al. 2024)으로 구현. 한 forward/backward pass로 여러 block을 동시 학습.

3.4.4 Long-context training trick

모든 epoch에서 sequence당 mask block 수를 고정(512 anchor)하고 anchor 위치만 무작위 sampling. EAGLE-3는 training-time tree test 때문에 long-context training이 비싸지만, DFlash는 이 트릭으로 비용을 bound 한다.

3.4.5 Position-dependent loss decay

block 내 위치 $k$의 token에 대한 가중치:

$$w_k = \exp\left(-\frac{k-1}{\gamma}\right) \quad (4)$$

직관적으로 말하면, draft block의 첫 토큰이 틀리면 그 뒤 모든 토큰이 무용지물이므로 ##앞쪽 위치를 더 중점적으로 학습##시킨다. Figure 5: loss decay 적용 시 수렴 속도와 최종 acceptance length 모두 향상.

3.4.6 Shared embedding & LM head

draft 모델의 token embedding layer와 LM head를 target model의 것을 공유하고 freeze. trainable parameter는 draft transformer layer뿐. → 가벼운 "diffusion adapter".

4. 실험 (Experiments)

4.1 셋업

  • Target models: LLaMA-3.1-Instruct (8B), Qwen3 (4B, 8B, Coder-30B-A3B), Qwen3.5 (4B, 9B, 27B, 35B-A3B), Qwen3-Coder-Next, GPT-OSS (20B, 120B)
  • Training data: Nvidia Nemotron-Post-Training-Dataset-V2 + CodeAlpaca에서 ~800K sample. ##원본 응답이 아니라 target model이 생성한 응답을 사용 (target alignment)##.
  • Hyperparameters: AdamW lr=6e-4, gradient clip 1.0, cosine + 0.04 warmup, 6 epoch, max seq 3072 (Coder 4096), 512 anchor/sequence. γ for loss decay: block 16 → γ=7, block 10 → 5, block 8 → 4.
  • Draft 구조: 5 layer (Qwen3 Coder는 8 layer), block size 16 (LLaMA 3.1은 10).
  • 벤치마크: Math (GSM8K, MATH-500, AIME25), Code (HumanEval, MBPP, LiveCodeBench), Chat (MT-Bench, Alpaca). H200 GPU.
  • Baseline: Vanilla autoregressive decoding + EAGLE-3 (Li et al. 2025b).

5. 종합 평가

강점

  • 아이디어의 elegance: "diffusion model의 약점은 standalone generation quality, 강점은 parallel decoding"이라는 명확한 분해. speculative decoding의 lossless guarantee가 약점을 무력화. ##drafter의 역할을 generation이 아닌 prediction adapter로 재정의##.
  • 실증의 광범위함: 4B~120B 7개 이상 모델, math/code/chat/reasoning 4 카테고리, Transformers/SGLang/vLLM 3 framework, H200/B200 2 hardware, concurrency 1~32. SOTA 비교(EAGLE-3) 외에 baseline 6.1×까지 보고.
  • Production 의식: SGLang Spec-v2 scheduling, B200 FA4 backend 등 ##2025년 production stack에서 검증##. EAGLE-3가 high concurrency에서 baseline보다 느려지는 현상까지 보고하는 정직함.
  • Ablation 깊이: layer 수, hidden feature 수, block size train/test mismatch, KV vs input, random anchor, loss decay — 모든 design choice를 분해해 기여도를 보여줌. 특히 ##DFlash-AR (AR drafting + KV injection)을 따로 보고##해 KV injection의 universality를 입증.

한계 및 의문점

  • 훈련 비용 미보고: 800K sample × 6 epoch + 5-layer × 800M~3B target hidden feature offline cache. 절대적인 GPU-hour나 storage 수치가 빠져 있어 ##training cost는 비교가 어려움##.
  • Chat (MT-Bench) 효과 약화: Math/Code에서 5×+ 인 반면 MT-Bench는 2.5~3×. open-ended generation은 diffusion bidirectional의 장점이 작아지는 듯하나 명시적 분석은 없다.
  • EAGLE-3와의 비교 공정성: EAGLE-3는 official checkpoint를 사용했으나 ##동일 데이터로 retrained 비교는 LLaMA-3.1에서만 일부 (DFlash가 UltraChat+ShareGPT로 EAGLE-3와 동일 데이터로 학습된 케이스)##. Qwen3 비교는 다른 데이터+다른 모델 셋팅이므로 strict apples-to-apples는 아니다.
  • Block size 동적 스케줄링: large block은 verification cost 증가로 compute-bound batch에서 손해. 저자도 dynamic block-size scheduling을 future work로 남김.
  • Diffusion noise schedule 미언급: 표준 diffusion LM의 absorbing state 가정인지, 어떤 corrupting process를 쓰는지 본문에서 명시적 설명이 빈약. Arriola et al. 2025를 참조하는 것으로 보이나, 본 논문 단독 reproducibility에는 한계.
  • Acceptance length 통계 분포 부재: 평균 τ만 보고. greedy 환경에서는 분산도 중요한데(특히 long reasoning), histogram이 Math500 block 8 단 한 군데만 언급됨.
  • 코드 공개의 시점: GitHub link는 있지만 ICML 2026 채택 paper로서 model checkpoint 공개 여부와 라이센스가 abstract만으로는 불분명.

6. Key Takeaways

  1. "Diffusion model을 그대로 쓰지 말고, parallel decoding 성질만 빌려라": standalone dLLM의 quality 문제는 speculative verification으로 완전히 우회 가능. ##diffusion LM 연구의 실용 활용 path를 제시##.
  2. KV injection은 EAGLE 계열에도 즉시 적용 가능한 universal 개선: Table 9에서 DFlash-AR(AR drafter + KV injection)이 EAGLE-3-5L 대비 일관 우위. ##기존 EAGLE-3 코드베이스에 KV injection만 추가해도 +0.3~0.5× speedup 기대##.
  3. Drafter 디자인 자유도 증가: drafting cost가 토큰 수와 디커플되므로 drafter를 더 깊고 표현력 있게 만들 수 있다. ##speculative decoding 연구가 "더 좋은 drafter 아키텍처" 방향으로 본격적으로 전환되는 시점##.
  4. Production 적용 시 block size 16 + 5-layer 권장: layer 수와 block size의 trade-off를 ablation에서 깔끔히 정리. ##low concurrency엔 큰 block, high concurrency엔 작은 block의 dynamic 전략이 미래 방향##.
  5. Long reasoning 모델에서 특히 임팩트 큼: thinking mode에서 4×+ speedup 유지. ##reasoning model의 serving cost 절감에 직접 기여##할 수 있는 기술.
