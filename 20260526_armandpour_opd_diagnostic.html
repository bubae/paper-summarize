TL;DR

  • 문제: On-policy distillation (OPD)이 reasoning 모델 학습의 표준 도구가 되었지만, "어떤 teacher가 좋은가?", "어떤 context를 넣어야 하는가?", "self-distillation이 좋은가 external teacher가 좋은가?"에 대한 원칙적 답이 없다. 기존 평가는 비싼 학습 실행 후의 aggregate 성능에 의존해 token 수준 dynamics를 숨긴다.
  • 해결: ##학습 없이(training-free) per-token/per-question/per-teacher 해상도로 distillation 신호의 품질을 평가하는 진단 프레임워크.## 각 노드에서 student의 정답 확률을 최대로 끌어올리는 ideal per-node gradient를 정의하고, distillation gradient와의 cosine similarity(gradient alignment score)로 teacher 신호가 reward objective에 얼마나 정렬되는지 측정한다.
  • 왜 작동하는가: Dr. GRPO가 이 ideal gradient를 expectation에서 정확히 복원함을 보이고, GKD/MiniLLM/단일샘플 추정량 모두 동일한 local form $P_\theta^j(f_j - \bar f)$를 갖는다는 통합 관점을 제시해, reward 신호와 distillation 신호를 같은 좌표계에서 직접 비교할 수 있게 한다.
  • 비유: 등산 가이드(teacher)가 학생을 정상으로 이끄는지 평가하기 위해, 학생이 실제로 등반한 모든 갈림길에서 "이 가이드의 손짓이 정상으로 향하는 방향과 얼마나 일치하는가"를 cosine 각도로 측정. 학생이 이미 정상 근처일 때는 가이드 손짓이 잡음일 수 있고, 길을 잃었을 때야말로 진짜 도움이 된다.

핵심 기여 (Key Contributions)

  1. Ideal per-node gradient 유도: 각 노드에서 student의 정답 확률 $\mathbb{E}_k[P_\text{succ}^k]$을 최대화하는 logit-space gradient $P_\theta^j(P_\text{succ}^j - \bar P_\text{succ})$를 정의해, distillation의 oracle reference를 제공한다.
  2. Dr. GRPO와의 동치성 증명: ##Dr. GRPO objective가 KL 항을 marginalize하고 length normalization을 제거하면 이 ideal gradient를 expectation에서 정확히 복원##함을 보인다. 즉 reward 기반 학습이 수렴할 방향을 학습 없이 시뮬레이션할 수 있다.
  3. 주요 distillation 방법의 통합 분해: GKD, 단일샘플 GKD 추정량(Thinking Lab), MiniLLM이 모두 $\partial L/\partial z_j = P_\theta^j(f_j - \bar f)$의 공통 구조를 가지며, $f_k$만이 reward-driven($P_\text{succ}^k$)인지 distribution-driven($\pm \log(P_\theta^k/P_\text{te}^k)$)인지에서 다르다는 것을 보인다.
  4. Gradient Alignment Score (GAS): ideal gradient와 distillation gradient 간 cosine similarity로 token-level 진단 metric을 정의. $+$이면 teacher가 정답으로 밀어주고, $-$이면 오답으로 밀어주며, $\approx 0$이면 stylistic 잡음.
  5. Targeted-rollout + exponential depth window: long reasoning chain에서도 $P_\text{succ}^k$를 신뢰성 있게 추정하기 위해, 깊이별로 지수적으로 커지는 window를 두고 GKD gradient magnitude/probability difference로 우선순위를 매겨 sub-linear scaling을 달성. ##Teacher-independent generation tree sharing으로 8개 teacher를 동시에 평가하면서 compute를 ~7×## 절감.
  6. 경험적 발견 3가지: (a) ##incorrect rollouts에서 alignment가 일관되게 더 높다(distillation은 실패 trajectory에서 가장 유용).## (b) student capacity와 context design이 강하게 상호작용: 0.6B는 self-distillation, 1.7B BoolQ는 external Qwen3-8B가 승리. (c) 보편적 recipe는 없다 — wrong demonstration이 BoolQ/MMLU에서는 해롭지만 hard AIME에서는 최고.

1. 서론과 동기 (Background & Motivation)

1.1 On-Policy Distillation의 부상

On-policy distillation(OPD)은 student가 자기 자신의 rollout에서 매 token마다 teacher의 분포로 supervision을 받는 학습 패러다임이다. Qwen3, MiMo, GLM-5 등 최신 reasoning 모델들이 모두 이 기법을 post-training에 채택했으며, RL(GRPO 계열)의 sparse binary reward를 dense per-token gradient로 보완한다는 점에서 ##compute-efficient## 대안으로 주목받는다.

OPD에는 두 갈래가 있다:

  • Teacher distillation: 더 큰 외부 모델이 supervision 제공 (Hinton et al., 2015 계열)
  • Self-distillation: student가 자기 자신의 teacher가 되되, 추가 context(예: 정답 trace, 정답 요약)를 부여받아 "privileged conditioning" 상태에서 분포를 생성

1.2 미해결된 실용적 질문들

실무자는 OPD 파이프라인을 설계할 때 답이 없는 결정에 직면한다:

  • Teacher를 외부 큰 모델로 할지, student 자기 자신으로 할지?
  • Self-distillation context로 정답 trace를 통째로 줄지, concise summary로 줄지?
  • 이 선택이 task에 따라 달라지는가? Token에 따라 달라지는가?

지금까지는 이 질문들에 답하려면 비싼 학습 실행과 aggregate metric(예: BoolQ 정확도)을 비교해야 했고, 그 결과는 token 수준에서 무엇이 일어나고 있는지를 숨긴다. Figure 1의 책장 문제 예시처럼 — teacher의 token-level disagreement는 ##(a) reasoning correction##, ##(b) stylistic preference("therefore" vs "so")##, ##(c) noise##가 뒤섞여 있고, 표준 distillation은 이 셋을 같은 weight로 다룬다.

1.3 본 논문의 목표

저자들은 모든 학습을 우회하고 token 수준에서 teacher 신호의 품질을 직접 측정할 수 있는 진단 프레임워크를 제안한다. 핵심 통찰: ##reward objective와 distillation objective의 gradient를 같은 좌표계에서 비교할 수 있다면, 학습 없이도 "이 teacher 신호가 reward와 정렬되는가"를 확인할 수 있다.##

2. 방법론 (Methodology)

2.1 Generation Tree와 Success Probability

Student $\pi_\theta$에서 prompt $q$에 대해 $G$개의 trajectory를 sample하면, prefix를 공유하는 노드들이 generation tree를 형성한다. 각 노드 $u$에서 다음 token $k$를 선택한 후 정답에 도달할 경험적 확률 $\hat P_\text{succ}^k$를 측정한다 — 이것이 "이 갈림길에서 이 방향으로 가면 얼마나 자주 정답에 도달하는가"의 추정량이다.

2.2 Teacher Advantage

임의 학습 알고리즘과 무관하게, 노드 $u$에서 teacher가 student보다 더 나은 분포를 갖는지 측정:

$$\text{Adv}(u) = \sum_{k \in \mathcal{S}_u} P_\text{te}^k \hat P_\text{succ}^k - \sum_{k \in \mathcal{S}_u} P_\theta^k \hat P_\text{succ}^k \quad (2.1)$$

$\mathcal{S}_u$는 충분히 visit된 token 집합. 양수이면 teacher가 더 좋은 다음 token에 mass를 더 많이 배치하고 있다는 뜻. 그러나 teacher의 분포만으로는 부족하다 — 어떤 알고리즘이 그 지식을 useful gradient로 번역하느냐가 중요하다.

2.3 Ideal Reference Gradient

각 노드의 local objective는 student의 현재 분포에서 기대되는 정답 확률을 최대화하는 것:

$$L_\text{ideal}(u) = \sum_k P_\theta^k P_\text{succ}^k \quad (2.2)$$

Softmax Jacobian $\partial P_\theta^k/\partial z_j = P_\theta^k(\delta_{kj} - P_\theta^j)$을 이용해 logit $z_j$에 대한 gradient를 유도:

$$\frac{\partial L_\text{ideal}}{\partial z_j} = P_\theta^j (P_\text{succ}^j - \bar P_\text{succ}), \quad \bar P_\text{succ} = \sum_k P_\theta^k P_\text{succ}^k \quad (2.3)$$
직관: token $j$가 student의 현재 평균보다 더 자주 정답으로 이끈다면 $z_j$를 키우고, 그렇지 않다면 줄인다. 이 gradient는 "이 노드에서 student의 logit이 어디로 움직여야 정답률이 가장 빨리 오르는가"의 답이다.

2.4 Dr. GRPO가 Ideal Gradient를 복원한다

GRPO objective $J_\text{GRPO}$에서 KL 항($\beta = 0$), importance ratio($P_\theta^k/P_{\theta_\text{old}}^k$의 cancellation), length normalization($1/|o_i|$)을 차례로 제거하면(Dr. GRPO; Liu, 2025), node $u$에서의 reduced objective는 정확히 $\sum_k P_\theta^k P_\text{succ}^k$가 된다. 따라서:

$$\mathbb{E}\left[\nabla_{z_j} L_\text{DrGRPO}\big|_u\right] \propto P_\theta^j(P_\text{succ}^j - \bar P_\text{succ}) \quad (2.5)$$

즉 ##Dr. GRPO는 ideal gradient의 unbiased estimator이며, 실제 학습은 무수히 많은 rollout에서 이 gradient로 수렴한다.## 우리는 학습을 우회하고 $\hat P_\text{succ}^k$를 직접 추정해 ideal gradient를 노이즈 없이 얻는다.

2.5 주요 Distillation 방법의 통합 분해

저자들은 모든 주요 OPD gradient를 같은 form으로 환원한다:

$$\frac{\partial L}{\partial z_j} = P_\theta^j(f_j - \bar f), \quad \bar f = \sum_k P_\theta^k f_k \quad (2.11)$$
방법$f_k$의 형태의미
Dr. GRPO (ideal)$P_\text{succ}^k$Reward-driven, local
GKD (Agarwal et al., 2024)$-(\log P_\theta^k - \log P_\text{te}^k)$KL gradient, local, full-vocab
Single-sample GKD (Thinking Lab)$\log P_\text{te}^{r_i} - \log P_\theta^{r_i} - 1$Importance-weighted, sampled token only
MiniLLM (Gu et al., 2023)$\sum_{t' \geq t}(\log P_\text{te}^{o_{i,t'}} - \log P_\theta^{o_{i,t'}}) - 1$Trajectory-dependent reward-to-go

공통 $P_\theta^j$ factor의 함의: ##gradient magnitude는 student의 현재 confidence에 gating된다.## Teacher가 high-success token을 발견해도 student가 그 token에 거의 mass를 주지 않으면 update는 작다 → ##teacher는 student가 부분적으로 믿는 방향을 amplify할 수는 있어도 완전히 새로운 continuation을 1 step에 inject할 수는 없다.##

2.6 Gradient Alignment Score (GAS)

$\mathcal{S}_u$(충분한 visit count를 가진 token 집합)에 제한해 두 gradient 벡터의 cosine similarity:

$$\text{Align}(u) = \cos(\mathbf{g}_u^\text{ideal}, \mathbf{g}_u^\text{D}) = \frac{\sum_{j \in \mathcal{S}_u} g_j^\text{ideal} g_j^\text{D}}{\|\mathbf{g}^\text{ideal}\|_{\mathcal{S}_u} \|\mathbf{g}^\text{D}\|_{\mathcal{S}_u}} \quad (2.12)$$
  • $\text{Align}(u) > 0$: distillation이 정답 방향으로 push → ##helpful##
  • $\text{Align}(u) \approx 0$: orthogonal → ##gradient budget을 무관한 방향(예: stylistic)에 낭비##
  • $\text{Align}(u) < 0$: 오답 방향으로 push → ##actively harmful##

##이 score는 Section 2.1에서 제기한 질문 — reasoning-critical disagreement(±)와 stylistic disagreement($\approx 0$)를 token 수준에서 학습 없이 구분 — 에 대한 답이다.##

2.7 Targeted Rollout과 Exponential Depth Windows

Equation 2.12는 각 branching node에서 신뢰할 만한 $P_\text{succ}^k$ 추정량을 요구한다. Vocabulary 150K, sequence 수천 token에서 naive하게 모든 token-node 쌍에 rollout을 하면 천문학적 비용. 핵심 효율성 트릭:

  • Targeted rollout: 노드 $u$와 token $k$가 필요할 때 prefix(prompt + path to $u$ + $k$)를 구성해 student로 끝까지 sample. ##깊이 $d$에서 시작된 rollout은 path 위 모든 ancestor와 descendant의 $P_\text{succ}$ 추정을 동시에 enrich##해 총 budget이 sequence length에 sub-linear하게 증가.
  • Exponential depth windows: 토큰 1-50, 51-150, 151-350, … 처럼 깊이별로 지수적으로 커지는 window. 각 window 내에서 GKD gradient magnitude(상위 $k$개)와 student-teacher probability difference(상위 $r$개)로 token 우선순위. ##앞쪽 window는 작고 dense하게(branching이 많음), 뒤쪽은 크고 sparse하게(이미 path가 converge).##
  • Budget & significance: $\tau = 0.02$ threshold로 의미 있는 token만 enrich. 각 token에 $N_\text{min} = 100$ visit(AIME는 noise 때문에 $N_\text{sig} = 40$). 노드는 child당 $\geq 20$ visit이 있을 때 alignment 계산에 포함.
  • Teacher-independent tree sharing: ##generation tree와 $P_\text{succ}$ 추정량은 student의 rollout에만 의존하므로 모든 8개 teacher가 enriched tree를 공유한다.## Phase 1(rollout 생성)은 1회, Phase 2(teacher forward pass)는 teacher당 1회 → 독립 실행 대비 compute ~7× 절감.
3단계 파이프라인 (Figure 2): Phase 1 (Shared, student-only): prompt q → G initial rollouts → generation tree → exponential depth windows → targeted rollouts → enriched tree with reliable P_succ^k Phase 2 (Per-teacher): Each teacher → one forward pass on enriched tree → log P_te^k → distillation gradient g_D → Align(u) = cos(g_ideal, g_D) Phase 3 (Aggregate): Per-path / per-question / per-teacher statistics → identify where alignment is positive/negative

3. 실험 설계 (Experimental Setup)

3.1 모델과 데이터

  • Student 모델: Qwen3-0.6B, Qwen3-1.7B (Yang et al., 2025)
  • External teachers: Qwen3-4B, Qwen3-8B, Qwen3-14B (student와 같은 prompt 사용)
  • Self-distillation teachers (모두 student와 같은 model이지만 context가 다름):
    • Self-1C: 정답 solution 1개를 in-context
    • Self-1C (32B): 32B가 student-as-teacher에 정답 demo 생성
    • Self-Sum-1C: 정답 solution을 Qwen3-32B로 summarize한 후 in-context
    • Self-Sum-1C1W: 정답+오답 모두 summarize
    • Self-1C1W: 정답+오답 raw demonstration
  • 벤치마크:
    • BoolQ (Clark et al., 2019): True/False reading comprehension, short reasoning
    • MMLU (Hendrycks et al., 2021): multiple choice, medium-length chains
    • AIME 2025 (case study): ~5K-30K token traces, long mathematical reasoning

3.2 Compute Budget

각 question에는 다음이 필요:

  • $G = 200$ initial rollouts (4 대표 경로: 2 correct + 2 incorrect)
  • Branching point당 ~45K-200K targeted rollouts
  • Important token당 100 samples; node는 $\geq 20$ visit이 있을 때 통계 유의
  • 전체 실험: ~72 A100-days

3.3 측정 단위

##각 branching node(≥2 children, 각 child ≥20 visits, nonzero success-rate range)에서 GKD gradient의 cosine similarity, teacher advantage, success rate 통계를 측정.## Per-path(path 평균 cosine), per-question(correct/incorrect 분리), per-teacher(95% CI) 집계.

3.4 핵심 질문

이 framework로 답하는 4개의 핵심 질문:

  1. Distillation gradient는 reward와 어디서 정렬되는가? (4.1)
  2. 어떤 teacher가 가장 잘 정렬되는가? Student capacity에 따라 답이 달라지는가? (4.2)
  3. Path 내에서 alignment를 예측하는 feature가 있는가? (4.3)
  4. 이 패턴이 long mathematical reasoning에도 일반화되는가? (4.4)

저자의 해석 (Comprehensibility Hypothesis): ##Self-distillation은 student가 context를 이해할 수 있을 때만 도움이 된다.## 0.6B student는 큰 외부 모델의 reasoning style을 흡수하기에 capacity가 부족하다 → teacher의 분포가 incomprehensible. 같은 0.6B 모델에 정답 demo를 보여주면 student의 reasoning style 안에서 understandable nudge를 제공할 수 있다. 1.7B는 capacity gap이 작아 8B 모델의 다른 분포 정보를 exploit할 수 있다.

추가 발견 — Wrong demonstration이 hurts: BoolQ/MMLU에서 1C1W 변종이 1C 변종보다 일관되게 underperform. ##Contrastive signal보다 noise injection이 더 크다##. 단, Section 4.4에서 보듯 AIME에서는 반대.

Summarization 효과 (capacity-dependent): Self-Sum-1C가 1.7B BoolQ alignment를 거의 두 배(0.050 vs raw Self-1C 0.028), 하지만 0.6B에서는 BoolQ 약간 손해, MMLU 혼합. ##큰 student는 concise summary에서 key signal을 추출할 수 있지만 작은 student는 verbose trace가 필요하다.##

Self-1C (32B) 패턴: 32B가 생성한 정답 demo를 student-as-teacher에 보여주는 변종 — 0.6B에서는 잘 작동(0.047, 0.052)하지만 1.7B와 AIME에서는 약함. ##32B reasoning style이 hard problem일수록 student가 따라가기 어려움##(comprehensibility).

4.3 Path 내에서 무엇이 Alignment를 예측하는가?

실험 의도: Reasoning chain 안에서 alignment가 positive인 위치를 알 수 있다면, alignment-aware training의 cheap filter를 만들 수 있다.

결과 (Table 5, within-path Spearman correlation):

  • Teacher-student divergence (KL, JS, $L_2$) → ##weakly positive## with alignment ($\rho \approx 0.02$-$0.04$)
  • Distributional similarity (cosine of $\pi_\theta, \pi_\text{te}$) → ##weakly negative##
  • Magnitudes are uniformly weak ($|\rho| < 0.05$)

의미: 의미 있는 신호는 ##teacher가 student와 disagree하는 곳에 산다##. 하지만 어떤 disagreement가 helpful한지는 단일 forward pass로 알 수 없다 → ##divergence는 cheap necessary-condition filter일 뿐 reliable predictor가 아니다.##

Depth(normalized position)도 0.6B에서 약 +0.04로 positive — ##앞쪽 token은 templatic(헬퍼 토큰), 뒤쪽 token은 실제 reasoning step##. 이 효과는 1.7B에서 사라지고 AIME hard question에서는 weakly negative($\rho \approx -0.03$) — ##어려운 math에서는 teacher advantage가 depth와 함께 자라지 않는다##.

4.4 Case Study — AIME 2025 (Long Math Reasoning)

실험 의도: BoolQ(~200 tokens), MMLU 결과가 long-chain math reasoning(5K-30K tokens)에 일반화되는가?

설계: Qwen3-0.6B로 Q0(pass rate 87.5%), Q3(9.4%); Qwen3-1.7B로 Q7(34.4%), Q28(56.3%) — 8개 teacher 모두 평가, $N_\text{sig} = 40$.

결과:

  • Correct vs incorrect 패턴은 4개 모두에서 복제: incorrect $+0.097$ vs correct $-0.011$, $\Delta = -0.108$.
  • 최고 teacher가 BoolQ/MMLU와 다르다:
    • Q0(easy, 0.6B): Self-Sum-1C가 최고 ($+0.043$)
    • Q3(hard, 0.6B): ##Self-1C1W (wrong demo 포함)가 최고## ($+0.040$)
    • Q7(med, 1.7B): Self-Sum-1C 최고 ($+0.055$)
    • Q28(med-hard, 1.7B): ##Self-1C1W가 최고## ($+0.053$)
  • Summarization은 hard math에서 손해: Q3에서 Self-Sum-1C가 raw Self-1C에 패배 — ##0.6B는 hard math의 condensed summary를 decipher할 수 없다##.

저자의 해석: ##Wrong demonstration이 hard math에서 도움되는 이유는 contrastive signal("이 mistake를 피하라")이 학생이 parse할 수 있는 정보##인 반면, 단순한 task에서 wrong example은 distracting noise일 뿐이다. ##이 발견은 "no universal recipe" 명제의 강력한 증거##다.

4.6 Selective Distillation — Practical Path

저자가 제안하는 future direction:

  • ##Rollout-weighted distillation##: incorrect rollout에서 loss weight 증가
  • ##Divergence-gated distillation##: KL($\pi_\theta || \pi_\text{te}$)이 threshold 이상일 때만 gradient 적용
  • ##Multi-teacher distillation##: 여러 teacher의 complementary signal 결합
  • ##Adaptive context selection per domain##: task별로 optimal context를 선택

5. 본 논문의 통합 관점 — 왜 모든 OPD가 같은 구조인가

Section 2.5의 통합 분해는 OPD 분야에 대한 mechanistic 통찰을 준다. 어떤 distillation 알고리즘이든 logit space에서:

$$\frac{\partial L}{\partial z_j} = P_\theta^j \cdot (\text{어떤 신호}_j - \text{그 신호의 평균})$$

##단지 "어떤 신호"가 다를 뿐이다.## Dr. GRPO는 $P_\text{succ}^k$ (oracle); GKD는 $\log P_\theta^k - \log P_\text{te}^k$ (KL); single-sample은 sampled token의 importance weight; MiniLLM은 future trajectory의 reward-to-go.

이 공통 구조의 두 가지 함의:

  1. $P_\theta^j$ gating: ##학생이 거의 mass를 주지 않는 token은 어떤 teacher도 한 step에 부각시킬 수 없다##. Distillation은 "amplify what student partially believes", "inject new continuation"이 아니다.
  2. $\bar f$ baseline: 모든 방법이 average-subtraction을 사용 → low-magnitude difference에 sensitivity, distribution의 absolute level이 아니라 relative position에 반응.

이 framework로 reward learning과 distillation을 직접 비교할 수 있게 된 것이 본 논문의 핵심 이론적 기여다.

6. Related Work와 본 논문의 위치

6.1 OPD의 짧은 역사

Knowledge distillation (Hinton et al., 2015) → sequence-level distillation (Kim & Rush, 2016, exposure bias 문제) → OPD: MiniLLM (Gu et al., 2023, reverse KL via policy gradient), GKD (Agarwal et al., 2024, on/off-policy interpolation), Thinking Lab (2025, single-sample importance-weighted estimator), Yang et al. (2026, log-ratio을 implicit dense reward로). 이후 self-distillation으로 확장(Snell, Hübotter, Zhao, Ye, Shenfeld, Penaloza 등 2026의 동시작업).

6.2 본 논문의 차별점

저자들은 ##새로운 distillation 알고리즘을 제안하지 않는다##. 대신 ##모든 기존 알고리즘을 token 수준에서 reward objective와 align되는지 측정할 수 있는 진단 도구##를 제공한다.

Concurrent works:

  • Li et al. (2026): OPD가 thinking-pattern consistency와 high-probability overlap token(97-99%)의 progressive alignment를 필요로 함을 보임. 본 논문은 이 진보를 mechanistic으로 설명한다.
  • Kim et al. (2026): Self-distillation degradation을 epistemic verbalization suppression으로 추적. 본 논문은 보완적 — gradient-level alignment 관점.

비판적 분석 (Critical Analysis)

강점

  • 이론과 실험의 결합: Dr. GRPO ↔ ideal gradient 동치성 증명 + 4개 distillation 방법의 통합 분해라는 깔끔한 이론적 결과를, 8개 teacher × 2 model scale × 3 benchmark의 광범위한 실험으로 뒷받침.
  • Compute-efficient 진단: Targeted rollout + teacher-independent tree sharing으로 alignment 측정이 training보다 저렴 → 실무자가 학습 전에 teacher 선택을 평가 가능.
  • 일관된 sign + statistically significant: Correct vs incorrect 패턴이 4개 setting 모두에서 $p < 10^{-5}$로 robust, multiple metrics(mean cosine, weighted cosine, teacher advantage)에서 같은 방향.
  • 발견의 actionability: "Incorrect rollout에서 distillation 더 효과적" → rollout-weighted training. "Comprehensibility matters" → student capacity에 맞춰 teacher 선택. 단순 분석을 넘어 학습 알고리즘 설계 지침을 준다.

한계와 의문점

  • Alignment ↔ downstream gain의 연결 미증명: 본 논문은 ##gradient alignment를 측정하지만, 이 score가 높은 teacher로 실제 학습했을 때 정말 더 나은 모델이 나오는지는 직접 검증하지 않았다##. Selective distillation은 oracle 분석에 그친다.
  • 실험 규모: AIME 2025에서 4개 question만 분석. ~72 A100-days는 큰 규모지만 case study에 가깝고 일반화에 추가 검증 필요.
  • Targeted rollout의 student 의존: $\hat P_\text{succ}^k$를 student가 sample한 token에 대해서만 계산 → ##student가 결코 sample하지 않는 token은 evaluate되지 않는다##. 이는 framework의 본질적 한계며 저자도 인정.
  • 모델 다양성 제한: 모두 Qwen3 family. Llama/Mistral 등 다른 family의 모델에서도 같은 패턴이 나오는지 검증 안 됨.
  • Within-path correlation 약함 ($|\rho| < 0.05$): Cheap predictor가 약하다는 사실은 alignment-aware training의 실용성에 의문을 남긴다. 더 강력한 predictor(예: distillation gradient magnitude 자체)도 검토 필요.
  • Mean cosine 절댓값이 작다: 0.027-0.053 — alignment가 weakly positive라는 사실은 "OPD가 reward와 잘 정렬된다"보다는 ##"평균적으로는 거의 random direction을 가진다, 단 sign이 약간 positive로 기울었을 뿐"##에 가깝다. 이것이 distillation의 본질적 한계인지, 측정 방식의 noise인지는 불명확.
  • Self-1C(32B)와 self-distillation의 경계: 32B로 demo를 생성하지만 student를 teacher로 쓰는 setting은 self-distillation과 external distillation의 hybrid. Comprehensibility hypothesis로 깔끔히 설명되지 않는 case들이 있을 수 있음.

실용적 함의

  • Student capacity가 모든 것을 결정한다: 작은 student일수록 self-distillation, 큰 student일수록 external teacher가 유리. 단순 "큰 teacher가 좋다"는 직관에 반하는 강한 결과.
  • OPD pipeline의 default를 재고: 산업에서는 fixed teacher + uniform distillation loss가 표준이지만, 본 논문은 ##rollout outcome(성공/실패)에 따라 loss weight를 동적으로 조정##해야 함을 시사.
  • Hyperparameter search의 새 차원: Teacher 선택뿐 아니라 context format(raw vs summary, with vs without wrong demo)이 task difficulty와 함께 tuned되어야 함.
  • Per-task diagnostic의 필요성: BoolQ에서 잘 작동하는 setting이 AIME에서 안 통한다 → ##학습 전에 target task에서 alignment를 측정##하는 것이 비싸지만 가치 있다.

Key Takeaways (이 논문에서 가져갈 것)

  • Dr. GRPO = ideal distillation gradient: Reward-based RL과 distillation은 본질적으로 같은 local form ($P_\theta^j(f_j - \bar f)$)을 공유 → ##두 패러다임을 같은 좌표계에서 직접 비교 가능##. 새로운 OPD 알고리즘을 제안할 때 이 form을 baseline으로 두는 게 자연스럽다.
  • Distillation은 실패에서 가장 빛난다: Token-level alignment가 incorrect path에서 일관되게 더 높음 ($\Delta = -0.063$ ~ $-0.097$, $p < 10^{-9}$). 실무 적용: ##rollout-weighted distillation loss로 incorrect trajectory에 더 큰 weight##를 주는 것이 가장 직접적인 개선.
  • Student capacity-comprehensibility 트레이드오프: 작은 모델은 자기 자신을 가르치게 두라(Self-1C, Self-Sum-1C); 큰 모델은 더 큰 external teacher(Qwen3-8B)에서 배울 수 있다. 일반적인 "bigger teacher is better"는 잘못된 default.
  • Wrong demonstration은 task-dependent: 단순한 task에서는 distracting noise, hard math에서는 contrastive signal. ##Self-distillation context의 format을 task difficulty에 맞춰 선택##해야 한다.
  • Selective distillation의 잠재력 (10-15× signal): Oracle 분석은 token의 ~50%만 사용해도 모든 path가 full GKD를 이긴다. Divergence-based filter는 약하지만 cheap한 first-cut filter로 가치. ##Alignment-aware training은 OPD의 다음 frontier##.
  • $P_\theta^j$ gating의 실용적 함의: Teacher가 high-success token을 알아도 student가 거의 mass를 주지 않으면 update가 작다 → ##cold-start나 OOD task에서 OPD의 효과가 작은 이유##. 이런 경우 SFT나 prompt engineering으로 student의 분포를 먼저 reasonable한 영역으로 옮긴 후 OPD를 적용해야 한다.
