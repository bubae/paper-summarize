1. Background

1.1 Dense Spatial PerceptionмқҙлһҖ?

Pixel-levelмқҳ spatial structureлҘј ліөмӣҗн•ҳлҠ” task. Depth estimation (each pixelмқҳ м№ҙл©”лқјлЎңл¶Җн„° кұ°лҰ¬), semantic segmentation (each pixelмқҳ category), video object segmentation (temporal мқјкҙҖм„ұ мһҲлҠ” mask propagation), edge/boundary detection л“ұмқҙ н•ҙлӢ№. Robotics, embodied AI, world modelм—җм„ң н•„мҲҳ.

1.2 SSL кі„м—ҙмқҳ Limitations

Self-Distillation (DINO, iBOT, DINOv2, DINOv3)

Student ViT $f_\theta$мҷҖ EMA teacher $f_{\bar\theta}$к°Җ мһҲкі  (momentum $\lambda$ toward 1мңјлЎң annealing):

$$\bar\theta \leftarrow \lambda \bar\theta + (1-\lambda) \theta \quad (1)$$

Image-level distillation (DINO): Projection head $h$лЎң class tokenмқ„ C prototype нҷ•лҘ лЎң л§Өн•‘. TeacherлҠ” centering + sharpening.

$$\mathbf{p}^t = \text{softmax}\left(\frac{h_{\bar\theta}(\mathbf{z}^{\text{cls}}) - \mathbf{c}}{\tau_t}\right), \quad \mathbf{p}^s = \text{softmax}\left(\frac{h_\theta(\mathbf{z}^{\text{cls}})}{\tau_s}\right) \quad (2)$$ $$\mathcal{L}_{\text{DINO}} = -\mathbf{p}^t(\mathbf{x}^{(1)})^\top \log \mathbf{p}^s(\mathbf{x}^{(2)}) \quad (3)$$

Temperature $\tau_t < \tau_s$ (teacher sharpening) + center $\mathbf{c}$ (EMA of teacher outputs, Sinkhorn-Knopp normalization)мқҙ constant distribution collapse л°©м§Җ.

Patch-level distillation (iBOT): Student patch token subset $\mathcal{M} \subset \{1,\dots,N\}$мқ„ maskн•ҳкі  shared mask tokenмңјлЎң лҢҖмІҙ. Patch projection head $g$лЎң:

$$\mathcal{L}_{\text{iBOT}} = -\frac{1}{|\mathcal{M}|} \sum_{i \in \mathcal{M}} \mathbf{q}_i^{t\top} \log \mathbf{q}_i^s, \quad \mathbf{q}_i = \text{softmax}\left(\frac{g(\mathbf{z}_i^p) - \mathbf{c}_{\text{iBOT}}}{\tau}\right) \quad (4)$$

Random Maskingмқҳ м •нҳ•м Ғ кІ°н•Ё

Mask $\mathcal{M} \sim p_{\text{mask}}(\cdot; r)$к°Җ content-agnostic: tokenмқҳ depictionкіј л¬ҙкҙҖн•ҳкІҢ sample. к·ёлҹ°лҚ° мқҙлҜём§Җм—җм„ң tokenмқҳ recovery лӮңмқҙлҸ„лҠ” нҒ¬кІҢ лӢӨлҘҙлӢӨ: flat homogeneous regionмқҳ tokenмқҖ мқҙмӣғмңјлЎңл¶Җн„° predictable, boundaryлҘј straddleн•ҳлҠ” tokenмқҖ irreducible structural informationмқ„ лӢҙлҠ”лӢӨ.

SSL кі„м—ҙмқҳ мӢӨмҡ©м Ғ н•ңкі„

  • DINOv2: Dense feature qualityк°Җ scaleм—җм„ң fragile вҶ’ per-token speckle
  • DINOv3: Long training scheduleм—җм„ң dense feature mapмқҙ degrade вҶ’ Gram anchoringмқҙлқјлҠ” corrective mechanism лҸ„мһ…
  • SigLIP 2: Language supervisionмқҙ whole-image м •ліҙ мң„мЈј вҶ’ dense predictionм—җм„ң blocky noise
  • V-JEPA: JEPAлҠ” pixel-level nuisance modeling нҡҢн”јн•ҳм§Җл§Ң background bleed л¬ём ң

1.3 Boundary/Attraction Field Representation

Boundary detectionмқҳ classical л°©лІ•: Sobel, Canny operatorл¶Җн„° statistical detector (LSD)к№Ңм§Җ. ліё л…јл¬ёмқҖ л‘җ м•„мқҙл””м–ҙлҘј кі„мҠ№:

  1. A-contrario theory (Desolneux et al.): StructureлҠ” structureless null model н•ҳм—җм„ң unlikelyн•  л•Ңл§Ң meaningful вҶ’ мһ„мқҳ threshold м—Ҷмқҙ statistical validation.
  2. Attraction field / Boundary field: к°Ғ pixelмқҙ boundary segmentлЎңл¶Җн„°мқҳ vectorized geometry (distance + orientation)лҘј м ҖмһҘ вҶ’ dense supervision мӢ нҳё. Sparse boundary detection вҶ’ dense field representation.

1.4 Boundary Fieldмқҳ кө¬мЎ° (Section 3.3, Figure 4)

Image boundariesлҘј straight line segment set $L = \{\ell_j\}_{j=1}^M$мңјлЎң лӘЁлҚёл§Ғ (кіЎм„ мқҖ short segmentмқҳ chain). к°Ғ pixel $p$ near segmentлҠ” 4-D attribute vector:

$$\mathbf{a}(p) = (d_p, \theta_p, \phi_p^1, \phi_p^2) \quad (7)$$

м—¬кё°м„ң:

  • $d_p$: nearest segmentк№Ңм§Җ кұ°лҰ¬
  • $\theta_p$: segmentлЎңмқҳ direction
  • $\phi_p^1, \phi_p^2$: pixel $p$м—җм„ң ліё segment endpoint к°ҒлҸ„ (segment мң„м№ҳ нҠ№м •)

Support region $S_\ell = \{p: d_p \leq \tau_d\}$мқҳ м—¬лҹ¬ pixelмқҙ к°ҷмқҖ segmentм—җ лҢҖн•ң м •ліҙлҘј м ҖмһҘ вҶ’ deliberately redundant. Segment $L$мқҖ support pixelмқҳ vote aggregationмңјлЎң decode.

Why line segments over edge maps? Edge pixelмқҖ individual filter response вҶ’ arbitrary threshold. Line segmentлҠ” many support pixelлЎң л’·л°ӣм№ЁлҗҳлҠ” single hypothesis вҶ’ a-contrario null (uniform orientation)м—җ лҢҖн•ҙ statistically test к°ҖлҠҘ.

2. Boundary-centric Masked Representation Modeling

2.1 Boundary-Forcing Masked Modeling (Section 3.2)

Boundary token м •мқҳ

Teacherмқҳ online boundary prediction (label м—Ҷмқҙ, Section 3.4)мқ„ pixel mapмңјлЎң rasterize. Token $i$к°Җ boundary tokenмқј мЎ°кұҙ: patch $i$ (мҳҲ: $P \times P = 16 \times 16$) м•Ҳмқ„ boundary lineмқҙ нҶөкіј.

$$\mathcal{B} = \{i \in \{1,\dots,N\}: \text{predicted boundary intersects patch}(i)\} \quad (5)$$

Boundary mapмқ„ token gridм—җ max-poolн•ҳм—¬ нҡҚл“қ.

Boundary-forcing routing

$\mathcal{B}$лҘј н•ҷмғқмқҳ random mask $\mathcal{M}$м—җ к°•м ңлЎң м¶”к°Җ:

$$\mathcal{M}^+ = \mathcal{M} \cup \mathcal{B} \quad (6)$$

Every masked tokenмқҖ semantic iBOT objective $\mathcal{L}_{\text{iBOT}}$лҘј л°ӣкі , boundary tokenмқҖ additionally categorical boundary objective $\mathcal{L}_{\text{bnd}}$лҘј л°ӣлҠ”лӢӨ. Class tokenмқҖ м—¬м „нһҲ $\mathcal{L}_{\text{DINO}}$.

Why forcing works?

BoundaryлҠ” мқҙлҜём§Җмқҳ least redundant region: interior tokenмқҖ neighborлЎң copy к°ҖлҠҘн•ҳм§Җл§Ң boundary tokenмқҖ contextлЎңл§Ң recover. Forcingмқҙ н•ҷмғқм—җкІҢ "м •нҷ•нһҲ л¬ҙм—Үмқ„ freeлЎң м–»мқ„ мҲҳ м—ҶлҠ”м§Җ"лҘј hide вҶ’ surrounding contextм—җм„ң boundary geometry мһ¬кө¬м„ұ compelled.

Semantic targetмқҙ л‘җ regionмқҙ meetн•ҳлҠ” кіім—җм„ң inherently ambiguousн•ҳлҜҖлЎң мқҙ routingмқҙ conventional masked modelingмқҙ к°ҖмһҘ м·Ём•Ҫн•ң кіім—җ well-posed objective кіөкёү.

2.2 Categorical Reparameterization of Boundary Fields (Section 3.3)

Continuous regressionмқҳ мӢӨнҢЁ: Boundary field $\mathbf{a}(p)$лҘј $\ell_1$/$\ell_2$ lossлЎң м§Ғм ‘ regressionн•ҳл©ҙ teacher-student EMA loopм—җм„ң drift and collapse. Continuous targetмқҖ semantic branchмқҳ centeringВ·sharpening machineryлҘј exploitн•  мҲҳ м—Ҷкё° л•Ңл¬ё.

Per-pixel classificationмңјлЎң ліҖнҷҳ: к°Ғ field channelмқҳ rangeлҘј K binмңјлЎң discretize. Boundary position $p$мҷҖ channel $c \in \{d, \theta, \phi^1, \phi^2\}$мқҳ (validated) teacher value $a^c(p)$лҘј soft categorical label:

$$\bar y_k^c(p) \propto \exp\left(-\delta^c(k, a^c(p))^2 / \tau_\ell\right), \quad k = 1, \dots, K \quad (8)$$

м—¬кё°м„ң $\delta^c$лҠ” bin $k$мқҳ centerмҷҖ value $a^c(p)$ мӮ¬мқҙ кұ°лҰ¬, $\tau_\ell$мқҖ label temperature. Orientation channel $\theta$лҠ” $2\pi$лЎң wrapн•ҳлҜҖлЎң binsлҠ” circularмқҙкі  $\delta^\theta$лҠ” arc distance. LabelмқҖ narrow(bin лӘҮ к°ңл§Ң) вҖ” over-smoothingмқҖ targetмқ„ uniformмңјлЎң л°Җм–ҙ мӢ нҳёлҘј м§ҖмҡҙлӢӨ.

Studentмқҳ boundary headлҠ” мұ„л„җлі„ distribution $\hat y^c(p)$ мҳҲмёЎ:

$$\mathcal{L}_{\text{bnd}} = -\frac{1}{|\mathcal{B}|} \sum_{p \in \mathcal{B}} \sum_c \bar y^c(p)^\top \log \hat y^c(p) \quad (9)$$

Categoricalмқҳ мқҙм җ

  1. Self-distillationмқҳ stability мғҒмҶҚ: Teacher distributionмқ„ centering (Sinkhorn-Knopp normalized) + sharpening нӣ„ student supervision вҶ’ semantic self-distillationкіј лҸҷмқјн•ң collapse л°©м§Җ кё°м ң. Bespoke regularizer л¶Ҳн•„мҡ”.
  2. A-contrario nullкіјмқҳ native connection: "No boundary" null hypothesis = uniform bin distribution. Deviation from uniformity = free boundary strength measure (Number-of-False-Alarms, NFA).
  3. Non-boundary regionмқҳ мһҗм—°мҠӨлҹ¬мҡҙ мІҳлҰ¬: Non-boundary targetмқҖ simply uniform bin distribution вҶ’ лі„лҸ„ background class л¶Ҳн•„мҡ”.

2.3 Online Generation of Boundary Targets (Section 3.4)

Bootstrapping problem: ViTлҠ” scratch initialization вҶ’ мІҳмқҢм—” valid boundary target лӘ» л§Ңл“Ұ. Self-distillationмқҖ мІ« stepл¶Җн„° target н•„мҡ” вҶ’ chicken-and-egg.

Finding 1: Boundaries emerge from corner points (Section 3.2)

мЈјмҡ” кҙҖм°°: Sparse corner pointк°Җ мЈјм–ҙм§Җл©ҙ, boundary field к°’мқҙ uniformly randomмқҙм–ҙлҸ„ corner-anchored line fragmentлЎң decodeлҗңлӢӨ. Coherent line segmentлҠ” direction channelмқҙ image gradientмқҳ parameter-free level-line orientationмңјлЎң guidedлҗ  л•Ң emerge. Corner point м—ҶмқҙлҠ” decoding мӢӨнҢЁ.

мӢӨмҰқ: Figure 3м—җм„ң л‘җ мқҙлҜём§Җм—җ лҢҖн•ҙ corner pointлҘј fixн•ҳкі  boundary field value 5к°ң random draw. Bottom rowsм—җм„ң fieldк°Җ uniform randomмқҙм–ҙлҸ„ corner-anchored segmentк°Җ лӮҳмҳӨл©° short and fragmented. Top rowsм—җм„ң direction channelл§Ң gradient orientation guidedмӢң coherent, near-identical segment.

Voting redundancyлЎң мқён•ң emergence

SegmentлҠ” support regionмқҳ many pixelмқҙ same underlying segmentм—җ vote вҶ’ many weak vote aggregationмқҖ noiseм—җ robust. Fieldк°Җ pixel-accurateн•ҳкё° нӣЁм”¬ м „м—җ decoded geometry meaningful. TeacherлҠ” studentліҙлӢӨ мЎ°кёҲ лӮҳмқҖ boundary predictorл©ҙ usable signal мғқм„ұ (EMA updateк°Җ мқҙлҘј ліҙмһҘ).

Teacher-side pipeline (4лӢЁкі„)

  1. Global viewм—җм„ң dense boundary field мҳҲмёЎ
  2. Frozen single-block ViT (corner-point detector)к°Җ sparse corner point localize
  3. Corner + fieldлЎң candidate segment $L$ setмқ„ vote aggregationмңјлЎң decode (Section 3.3)
  4. A-contrario test (Appendix B)лЎң unsupported segment нҸҗкё° вҶ’ validated segmentлҘј clean target field $\mathbf{a}(\cdot)$лЎң re-render. мқҙ validated targetл§Ңмқҙ studentлҘј supervise.

Token-level head for a dense field

Boundary fieldлҠ” sub-token resolution ($P \times P$ patch лӮҙл¶Җмқҳ к°Ғ pixel)мқҙм§Җл§Ң backboneмқҖ $P \times P$ patchлӢ№ н•ҳлӮҳмқҳ feature м¶ңл Ҙ. Lightweight boundary head: к°Ғ patch tokenмқ„ small MLPлЎң мІҳлҰ¬, outputмқ„ $r \times r$ position tile (stride $s = P/r$, we use $s = 2$)лЎң expand & rearrange. к°Ғ position featureлҠ” $\ell_2$-normalized onto unit sphere, K learned bin prototypeм—җ лҢҖн•ҙ мұ„л„җлі„ score. Prototype: DINO projection headмҷҖ мң мӮ¬, bias-free linear layer with unit-normalized weight вҶ’ logitмқҖ cosine similarityлЎң by construction bounded. 2.4 Training Objective (Section 3.5)

м „мІҙ loss:

$$\mathcal{L} = \mathcal{L}_{\text{DINO}} + \lambda_i \mathcal{L}_{\text{iBOT}} + \lambda_b \mathcal{L}_{\text{bnd}} + \lambda_k \mathcal{L}_{\text{KoLeo}} \quad (10)$$
  • $\mathcal{L}_{\text{DINO}}$: view pair к°„ class token distillation
  • $\mathcal{L}_{\text{iBOT}}$: $\mathcal{M}^+$ м „мІҙ masked tokenм—җ лҢҖн•ң кі„мӮ° (boundary token нҸ¬н•Ё)
  • $\mathcal{L}_{\text{bnd}}$: $\mathcal{B} \subset \mathcal{M}^+$мқҳ boundary positionм—җ лҢҖн•ҙм„ңл§Ң кі„мӮ° вҶ’ boundary tokenмқҖ dual supervision
  • $\mathcal{L}_{\text{KoLeo}}$ (weight $\lambda_k = 0.1$): batch лӮҙ class token featureлҘј spreadн•ҳлҠ” regularizer

лӘЁл“  teacher quantity (semantic distribution + validated boundary target)мқҖ stop-gradient. Local viewлҠ” image-level distillationл§Ң supervise.

4.2 Distributed Training and Efficiency (Section 4.2)

Sparse and fused boundary computation

Boundary headлҠ” never densely run. Boundary token $\mathcal{B}$ (small fraction)л§Ң sub-token resolutionмңјлЎң unfold + bin prototype scoring. Head costлҠ” image size м•„лӢҢ boundary token мҲҳмҷҖ scale. Soft labelкіј cross-entropyлҘј K bin over 4 channelлЎң naiveн•ҳкІҢ materializeн•ҳл©ҙ memory м§Җл°° вҶ’ four label construction (circular arc-distance channel нҸ¬н•Ё)кіј categorical cross-entropyлҘј single custom kernelлЎң fuse.

Online target generation as batched GPU work

Teacher-side pipeline (Section 3.4)мқҳ field decoding, corner-line pairing, a-contrario test, validated segment re-renderingмқ„ лӘЁл‘җ batched CUDA kernelлЎң end-to-end кө¬нҳ„. л‘җ stage нҠ№лі„ мЈјмқҳ:

  • Corner-line pairing: Distance between corner pointмҷҖ candidate segment endpointлҘј pairwise кі„мӮ°. NaГҜve materializationмқҖ memory intensive вҶ’ distance computationкіј argmin reductionмқ„ fuse, memory linear.
  • A-contrario test of LSD: Sequential CPU design вҶ’ reformulate as per-candidate reduction over support pixel вҶ’ batch лӮҙ лӘЁл“  candidate segment лі‘л ¬ validate.

Segment de-duplicationмқ„ CPUм—җм„ң н•ҳл©ҙ stall вҶ’ host round-trip мҷ„м „ м ңкұ°н•ҙм•ј target generationмқҙ лӮҳлЁём§Җ stepкіј cleanн•ҳкІҢ overlap. Final recipeм—җм„ң boundary target мғқм„ұмқҙ total step timeмқҳ small fractionл§Ң м¶”к°Җ.

4.3 Implementation Details (Section 4.3)

DINOv3 stagging follow: long self-distillation pretraining вҶ’ Gram-anchoring phase (late trainingм—җм„ң dense feature quality нҡҢліө) вҶ’ short high-resolution adaptation.

Budget

  • Pretraining: 300k iterations, global batch size 3072
  • Gram anchoring: 100k iterations
  • High-resolution adaptation: 100k iterations at 512px
  • Total: 500k iterations
  • DINOv3мҷҖ л№„көҗ: 1M + 100k + 30k iterations at batch size 4096 вҶ’ LingBot-VisionмқҖ DINOv3 sample мҲҳмқҳ 1/3 лҜёл§Ң

Backbone and heads

  • Boundary head (Section 3.4): 3-layer per-token MLP, head dim 512, output stride $s=2$, learnable tile positional embedding, $K=32$ bin per field channel
  • DINO & iBOT projection heads: DINOv2 мҠӨнғҖмқј, separate iBOT head
  • Frozen corner-point detector: single-block ViT, backboneліҙлӢӨ orders of magnitude мһ‘мқҢ

Pretraining hyperparams

  • AdamW, base LR scales as $\sqrt{\text{bs}/1024}$, linear warmup, cosine decay
  • Weight decay: 0.04 вҶ’ 0.2 (cosine)
  • Teacher temperature: 0.04 вҶ’ 0.07 in first 30k iterations
  • EMA teacher momentum: 0.994 вҶ’ 1.0
  • $\lambda_i = \lambda_b = 1$, $\lambda_k = 0.1$
  • Global crop 256px, local crop 112px
  • Stability: teacher target at non-boundary positionмқҖ uniform random labelлЎң мұ„мӣҖ (constant background value лҢҖмӢ ) вҶ’ trivial all-background solution м ңкұ°

Model distillation for efficiency

Flagship ViT-g нӣҲл Ё нӣ„ smaller backboneмңјлЎң distill: 300M ViT-L, 86M ViT-B, 21M ViT-S. DINOv2/v3мқҳ distillation practice follow: frozen ViT-gк°Җ EMA teacherлҘј replace, smaller studentлҠ” semantic + boundary-aware representation мғҒмҶҚ. Same iteration budget as pretraining stage per student. Gram-anchoring phase skip (frozen teacherм—җм„ң drift м•Ҳ мқјм–ҙлӮЁ), high-resolution adaptation at 512px for 100k iterations. Global batch size 3072. ScheduleмқҖ DINOv3мқҳ 1M + 250k iterations at batch 4096ліҙлӢӨ нӣЁм”¬ м§§мқҢ.
