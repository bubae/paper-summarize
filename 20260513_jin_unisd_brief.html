한 줄 요약

"외부 선생이 없을 때, 자기 자신의 여러 모습(다른 시점, 다른 맥락, 다른 토큰)을 교차검증해서 신뢰할 만한 신호만 골라 학습하라 — 그게 UniSD다."

왜 이 문제가 중요한가

LLM 후처리 학습에는 강한 외부 선생이 필요하다. 그러나 점점 더 많은 상황에서 그 선생을 구할 수 없다:

🔒 비싸거나 막혀있다

GPT-5 API는 비용이 크고 access가 제한적. 대규모 distillation에는 부담.

🏥 도메인 특화

의료/법률처럼 좁은 도메인에선 일반 teacher가 오히려 지식을 흐리는 경우도 있다.

🔄 Continual learning

새 skill 습득 시 기존 능력을 유지하려면 자기 자신의 past version이 가장 좋은 reference.

그래서 self-distillation (SD) — 외부 선생 없이 자기 자신을 선생으로 삼는 방법이 주목받았다. 하지만 SDFT, OPSD, GKD 같은 개별 레시피만 있고 "무엇이 진짜 효과적인지" 비교할 framework가 없었다.

배경 지식: 알아두면 좋은 6가지 개념

1) Knowledge Distillation (KD)

강한 teacher 모델의 지식을 약한 student 모델로 옮기는 기법. 1세대(Hinton, 2015)는 teacher의 출력 분포를 student가 흉내내게 했다.

2) Self-Distillation (SD)

외부 teacher 없이 자기 자신(또는 자기의 변종)을 teacher로 사용하는 SD. 외부 선생을 구할 수 없는 환경의 대안. 단, 자기 자신이 선생이므로 자기가 틀린 곳에서 자신감 있게 틀린다는 위험이 있다.

3) On-policy 학습

학생이 자기 분포에서 직접 sample한 sequence 위에서 학습. SFT (off-policy: 고정 데이터셋)와 대비. 학습-추론 mismatch (exposure bias)를 줄인다.

4) EMA (Exponential Moving Average)

최근 weight들의 가중 평균. 일반적 사용 예: training loss를 매 step의 raw 값이 아니라 EMA로 smoothing. LLM에선 학습 weight 자체를 EMA로 smoothing해서 "안정된 자기"를 만든다.

5) JSD (Jensen-Shannon Divergence)

두 확률분포의 거리. KL divergence보다 대칭적이고 bounded해서 학습이 안정적. UniSD는 JSD를 self-teacher와의 거리 측정에 사용.

6) Token-level vs Sequence-level

학습 신호를 토큰 하나하나에 줄 것인지(token-level), 문장 전체에 한 번 줄 것인지(sequence-level). UniSD는 둘 다 옵션으로 제공하고, task별 trade-off를 분석한다.

UniSD의 통찰: Self-Distillation을 3개 축으로 분해

저자들은 "왜 self-distillation이 어려운가?"를 3가지 독립된 어려움으로 정리했다:

▲ UniSD의 3-axis taxonomy: self-distillation의 어려움은 3가지 독립된 문제로 분해된다.

축 1: Supervision Reliability (공급 신뢰성)

자기 자신이 teacher라는 사실 자체가 문제다. 자기가 모르는 곳에서는 자신감 있게 잘못된 답을 가르친다. → 해결: 여러 시점/맥락의 자기 의견을 교차검증해서 일치하는 부분만 믿는다.

축 2: Representation Alignment (표현 정렬)

토큰 분포는 맞춰도 내부 hidden state는 어긋날 수 있다. → 해결: hidden representation 자체를 정렬하는 부가 loss.

축 3: Training Stability (학습 안정성)

Self-teacher가 학습 중에 자기랑 같이 변하면 target이 움직이는 moving-target 문제. → 해결: 자기 자신의 smoothed 과거 버전을 고정 reference로.

① Multi-Teacher Agreement

같은 prompt에 여러 보조 context를 더해 self-teacher에 K번 통과시킨다. K개 답이 모두 일치하는 토큰만 "신뢰할 만한 신호"로 인정. 일치하지 않는 토큰은 학습에서 제외(mask).

🗳️ 비유: 자기 자신에게 "5가지 다른 방식으로 같은 문제를 풀어봐"라고 시킨 뒤, 5번 다 같은 답이 나온 부분만 진짜 자기 답으로 받아들인다. 들쭉날쭉한 부분은 자기도 확신이 없다는 뜻이니 학습에 쓰지 않는다.

② EMA Teacher

학습 step n에서 weight를 $\bar\theta_n = \beta \bar\theta_{n-1} + (1-\beta)\theta_n$로 smoothing. 이 $\bar\theta$를 teacher로 사용. 가장 강한 단일 컴포넌트(+4.5점 base 대비).

👤 비유: 며칠 전의 안정된 자기를 멘토로 삼는 것. 어제 자기는 너무 가까워서 의미 없고, 1주일 전 자기는 너무 멀어서 옛 자기. EMA는 "최근 며칠의 평균 자기" — 부드럽고 안정된 reference.

③ Token-Level Contrastive Learning

마진 기반 contrastive loss: $\max(0, \gamma + d^+ - d^-)$. 잘된 토큰 위치는 self-teacher에 가깝게, 잘못된 토큰은 멀게 만든다.

🔍 비유: 모범답안과 자기 답안을 토큰 단위로 비교하면서, 일치하는 토큰엔 +, 틀린 토큰엔 -를 표시하는 셀프 첨삭.

④ Feature Matching

Hidden representation을 직접 정렬: $\|f^\theta - f^*\|^2$. 토큰 분포 외에 모델 내부의 의미 구조까지 self-teacher에 맞춘다.

🧠 비유: 표면(토큰)만 맞추는 게 아니라, "왜 그렇게 생각했는지"의 내부 사고 흐름까지 비슷하게 만든다.

⑤ Divergence Clipping

JSD가 threshold $\kappa$를 넘는 토큰에서는 weight를 $\min(1, \kappa/JSD)$로 cap. 한 토큰의 큰 불일치가 학습 전체를 흔드는 것을 방지.

🛡️ 비유: 한 시험 문제에서 너무 큰 점수 차가 나면 보정해서 전체 성적이 한 문제로 좌우되지 않게 한다.

▲ 각 컴포넌트의 단독 효과 vs 전체 통합. UniSD*가 가장 강한 단일 컴포넌트(EMA)를 능가 → 5개 컴포넌트가 서로 보완적.

분포 보존 (Distribution Retention)

SFT의 숨은 비용: base 분포를 망가뜨린다. 학습 후 모델은 정답은 잘 맞히지만 base 모델이 "이건 plausible하다"고 보던 답이 더 이상 안 나온다 → downstream/OOD에서 회귀.

▲ SFT는 base 분포를 1.14 → 1.68로 크게 망가뜨리지만, UniSD 변형들은 1.09–1.13으로 base 모델을 거의 보존한다.

중요한 발견들

① 더 많은 보조 context (K↑)가 항상 좋은 건 아니다

K=3 ~ K=7 사이에서 task별로 non-monotonic 곡선. 너무 적으면 신뢰성 약하고, 너무 많으면 context 간 모순으로 인해 reliability 추정이 더 noisy해진다.

② Agreement strength (γ)의 trade-off

γ가 작으면(weak filter) peak performance 높지만 K 선택에 민감(±2.20 변동). γ 크면(strong filter) 평탄하지만 낮음(±0.85). task가 잘 정의되면 약한 γ, 노이즈 많으면 강한 γ.

③ 보조 context 유형도 task-specific

Retrieval (nearest neighbor) → ScienceQA/GPQA/HumanEval 같은 reasoning task에 강함. Induced (task description 생성) → ToolAlpaca 같은 format-sensitive task에 더 적합. Random → 안정적이고 평균적.

④ Token-level vs Sequence-level granularity

Token-level: completion의 일부만 일치해도 그 부분 keep → peak↑. Sequence-level: 전체 sequence 단위 reliability → 안정성↑ peak↓.

⑤ 3 family 모두에 일반화

Qwen2.5(+5.4), Llama-3.1(+3.1), Gemma-3(+2.2) 모두 개선. single architecture에 overfit한 trick이 아님. CoS-E에서 gain 작은 이유는 instruction-tuned 모델이 이미 commonsense를 잘 인코딩해 추가 여지가 적기 때문 — 솔직한 negative result도 같이 보고.

활용 분야

▲ UniSD의 활용 가능 시나리오.

한계점

① 하이퍼파라미터 폭증

5개 컴포넌트 결합으로 6+개의 hyper-parameter (β, γ_c, κ, λ_c, λ_f, K, γ). UniSD*가 진짜 unified인지, 아니면 task별로 다른 sweet spot을 fine-tune한 결과인지 불명확.

② Agreement의 cost

K-fold forward pass는 SFT 대비 ~5.4× wall-clock (Qwen2.5-7B 기준 100분 vs 18.6분). Production 비용 측면에서 가성비 의문.

③ Feature matching의 layer 선택

어느 layer의 hidden state를 매칭할지에 따라 효과가 달라질 수 있는데, 논문은 default choice의 ablation 미흡.

④ OOD에서의 회귀 사례

18개 model-dataset pair 중 1개에서 raw보다 나빠짐. 어떤 조건에서 OOD harm이 발생하는지의 분석이 더 필요.

⑤ 3-axis taxonomy가 conceptual

reliability/alignment/stability가 직교 axis라는 주장은 흥미롭지만 수학적으로 증명되진 않음. 다른 분류(예: signal vs noise)도 가능.

한 줄 결론

외부 teacher가 없을 때 self-distillation은 가능하지만,
한 가지 기법이 아닌 3개 축을 동시에 다뤄야 진짜 효과가 나온다.
UniSD는 self-supervised post-training의 reusable backbone.

가져갈 인사이트 3가지

  • "Self-distillation의 적은 자기 자신이다": 외부 선생이 없는 게 문제가 아니라, 자기가 자신감 있게 틀리는 곳을 거르는 것이 본질이다.
  • "Accuracy만 보지 말고 retention도 봐라": SFT는 정확도는 올려도 base 분포(1.14 → 1.68)를 망가뜨린다. OOD에서 어떻게 행동하는지를 함께 측정해야 자기훈련의 진짜 가치가 보인다.
  • "EMA teacher는 5줄 짜리 가성비 갑": 5개 컴포넌트 중 단일 최강 효과(+4.5). 구현은 가장 쉽다. 외부 teacher 없는 환경에서 시작할 때 EMA부터 도입하라.
