1. Background: The Native Multimodal Landscape

현재 multimodal AI의 핵심 논쟁: 비언어 modality를 어떤 representation으로 모델링할 것인가? Continuous embedding vs. Discrete token, 그리고 그 사이의 hybrid들. LongCat-Next가 왜 "fully discrete"를 선택했는지 이해하려면, 주요 접근들의 landscape를 먼저 파악해야 한다.

Native multimodal 접근의 landscape. LongCat-Next는 fully discrete + pure NTP이면서 RVQ로 정보 손실 문제를 해결한다.

각 접근의 핵심 차이

언어 통일 비유

Chameleon은 모든 나라 사람에게 "500개 단어짜리 기초 영어"만 가르친 것과 같다 --- 소통은 되지만 깊은 표현이 불가능하다. Transfusion은 "영어(text)로 말하되, 그림 그릴 때만 별도의 회화 수업(diffusion)"을 받게 한 것이다. LongCat-Next는 "8개 레벨의 정교한 어휘 체계"를 만들어, 기초 단어부터 전문 용어까지 모두 표현할 수 있는 풍부한 discrete 언어를 설계한 접근이다.

왜 Fully Discrete + Pure NTP를 선택하는가?

LongCat-Next가 discrete + NTP를 고집하는 실용적 이유를 정리하면:

  1. Architectural Synergy: LLM의 성숙한 optimization/scaling infrastructure (FlashAttention, MoE, pipeline parallelism 등)를 modification 없이 재사용 가능. Continuous embedding + diffusion loss는 별도의 training infrastructure가 필요하다.
  2. Unification of Understanding and Generation: 하나의 NTP objective로 이해와 생성을 통합. Dual-objective (NTP + diffusion)는 training dynamics가 복잡하고, loss weight tuning이 critical하다.
  3. Seamless Cross-Modal Interaction: 모든 modality가 같은 discrete vocabulary를 공유하므로, cross-modal reasoning이 별도의 alignment module 없이 자연스럽게 발생한다.
  4. Native Data Scaling: Uncurated in-the-wild 데이터에서 self-supervised learning이 가능. Image-text pair가 아닌 raw image도 next-token prediction으로 학습 가능.
  5. RL Compatibility: Discrete action space는 finite MDP를 형성하여, RLHF/GRPO 같은 기존 LLM RL 방법을 직접 적용 가능. Continuous latent에서의 RL은 continuous action space 문제로 인해 훨씬 복잡하다.
DiNA의 핵심 이점: 기존 LLM infrastructure를 그대로 재사용하고, modality-specific tokenizer만 추가하면 omni-modal 시스템이 된다

2. The Dual Bottleneck Problem

Discrete visual modeling의 근본적 문제: 이미지 같은 고차원 연속 신호를 유한한 codebook으로 압축하면 두 가지 병목이 동시에 발생한다. LongCat-Next는 이를 명시적으로 정의하고 각각에 대한 원칙적 해결책을 제시한다.

Dual bottleneck: Semantic capacity (SAE로 해결) + Information loss (RVQ로 해결) = dNaViT

Semantic Completeness 원칙 (Eq.1)

Discrete representation z가 semantically complete하다는 것은, 임의의 image-centric query Q에 대해:

P(A | z, Q) ~ P(A | I, Q)

이것이 함의하는 두 가지:

  • Discriminative Invariance: z가 원본 I의 핵심 semantic attribute를 보존 (fine-grained recognition ~ semantic reasoning)
  • Generative Sufficiency: z가 faithful image reconstruction에 필요한 essential visual semantic을 capture (structural + textural content)
MP3 압축 vs FLAC 비유

Bottleneck 1은 "어떤 주파수 대역을 남길 것인가"의 문제다 --- 인간 청각에 중요한 대역을 버리면 안 된다 (semantic completeness). Bottleneck 2는 "남은 대역을 몇 비트로 양자화할 것인가"의 문제다 --- 비트 수가 적으면 양자화 노이즈가 커진다 (RVQ가 multi-level로 해결). SAE는 "어떤 정보를 남길지" 결정하는 psychoacoustic model이고, RVQ는 "남은 정보를 정밀하게 인코딩"하는 multi-pass encoder다.

3. dNaViT Architecture Deep Dive

3.1 SAE (Semantic-and-Aligned Encoder)

Language-aligned semantic encoder가 pre-quantization space로 특히 적합한 이유:

  • Semantic richness: High-level concept + fine-grained visual detail 모두 capture
  • Language affinity: Unified discrete token space로의 seamless 통합 가능

실제로 Qwen2.5-ViT (28x spatial compression)를 SAE로 직접 채택한다. 별도의 SAE training 없이 large-scale vision-language training으로 암묵적 semantic completeness가 달성: (1) Global Alignment (CLIP-style contrastive), (2) Detailed Learning (multi-aspect language-conditioned supervision).

우리 시스템에의 시사점: 이미 학습된 vision-language encoder (QwenViT, MoonViT, AIMv2 등)를 SAE로 바로 활용 가능하다. 별도의 tokenizer encoder training이 필요 없다는 것이 engineering 관점에서 큰 이점이다.
SAE는 semantic richness (무엇이 있는가)와 language affinity (LLM과의 호환성)를 동시에 제공한다

3.2 RVQ (Residual Vector Quantization): 8-Level Residual-of-Residual

RVQ는 8단계의 cascaded quantization으로 residual을 반복 포착하여 information loss를 최소화한다

3.3 De-tokenizer: Pixel Decoder + Flow-Matching Refiner

De-tokenizer는 pixel decoder (semantic structure)와 flow-matching refiner (fine detail)의 2-stage 구조

3.4 Intrinsic Information Recovery: Random ViT > Pretrained ViT (!)

Counter-intuitive 발견: Randomly initialized ViT-B/16이 pretrained ViT-B/16보다 image reconstruction에서 우수하다.

EncoderPSNR (higher=better)SSIM (higher=better)설명
ResNet5020.880.509No residual connection at ViT level
ViT-B/16 (Pretrained)21.860.581Semantic 풍부, low-level 약함
ViT-B/16 (Random Init)30.520.887Residual pathway가 pixel info 보존
QwenViT (w/o merger)18.160.460Aggressive downsampling 영향

메커니즘: ViT의 residual connection 구조상, 각 layer의 출력은 z_l = z_{l-1} + F_l(z_{l-1}). Identity mapping이 earlier layer의 fine-grained visual signal이 higher-level abstraction에 의해 덮어쓰이지 않도록 보장한다. Random ViT의 경우 F_l이 noise-like signal을 생성하여 decoder가 "denoise"하기 쉽고, 동시에 residual pathway가 pixel-level information을 보존한다.

필사본 비유

Pretrained encoder는 "전문 번역가가 원본을 의역한 것" --- 의미는 정확하지만 원문의 글자 하나하나는 사라진다. Random encoder는 "원본을 복사기로 복사하면서 약간의 노이즈가 추가된 것" --- 의미 해석은 못하지만 원본의 모든 글자가 (약간 흐릿하게) 그대로 남아있다. De-tokenizer는 이 "약간 흐릿한 복사본"에서 원본을 복원하는 것이 "의역된 번역"에서 원문을 복원하는 것보다 훨씬 쉽다.

Tokenizer 설계 시사점: Encoder architecture 선택 시 residual connection의 정보 보존 특성을 고려해야 한다. Semantic richness (SAE)와 pixel-level recoverability (residual pathway)는 다른 source에서 온다. dNaViT는 SAE로 semantic을, residual connection으로 pixel info를 동시에 확보하는 설계이다.

3.5 dNaViT 통합 구조: Native Resolution + Additive Encoding

dNaViT의 end-to-end 구조: SAE -> RVQ -> Additive Encoding -> LLM -> DepthTransformer -> De-tokenizer

dNaViT의 추가 핵심 설계 결정:

  • Native Resolution: Fixed-resolution 대신 임의 해상도에서 직접 처리 (최대 1,736x1,736). Resizing/cropping/padding에 의한 information loss 방지. Variable-length FlashAttention 사용. Max training sequence length: 8,192 tokens.
  • Additive Encoding: Multi-level RVQ token에 대해 level별 embedding을 sum으로 결합. Exponential representation space를 확보하면서 single autoregressive step의 efficiency 유지. Level 간 embedding은 공유되지 않아 complementary information capture.
  • Randomly Initialized Codebook: Visual codebook embedding이 LLM과 jointly co-evolve. Semantic content가 tokenizer 내부에 고정되지 않고, LLM의 visual embedding space에 internalize. Pre-quantization feature는 RVQ의 discrete clustering assignment에만 사용.
  • DepthTransformer: Multi-level token의 efficient decoding을 위한 task-aware decoder. LLM이 single-step autoregressive prediction을 수행하면, DepthTransformer가 multi-level output을 parallel로 생성.
주파수 분해 비유

Additive Encoding은 오디오의 푸리에 변환과 유사하다. 기본 주파수(Level 1)가 대략적인 멜로디를 결정하고, 고차 harmonic (Level 2~8)이 음색과 디테일을 추가한다. 이 harmonic들을 합산(additive synthesis)하면 원래 소리가 복원된다. dNaViT에서도 Level 1의 "시각적 기본 주파수"에 Level 2~8의 "시각적 하모닉"을 더해 원본 이미지에 가까운 표현을 만든다. 각 level의 embedding이 독립적으로 학습되어(비공유), 서로 다른 "주파수 대역"의 정보를 complementary하게 캡처한다.

Random Delay Training + Internal Linguistic Guidance

Speech generation에서 두 가지 전략을 unified framework으로 통합:

전략방식장점단점
ParallelText + Audio 동시 생성, audio에 delay 적용Zero response latency, full-duplexModality repr. conflict 가능
SerialText 먼저 생성 -> Audio 후속높은 linguistic qualityLatency 존재

Random delay training: 각 aligned text-audio segment의 delay를 1 ~ text segment length까지 random 선택. Serial (delay = full length)과 Parallel (delay = 1)이 이 spectrum의 two extremes가 된다. 단일 학습으로 둘 다 지원.

Internal Linguistic Guidance (Moshi 영감): AS/AE/TE special token으로 audio segment 경계 표시. Text가 audio의 semantic planning을 guide하여, speech가 text-level linguistic capability를 상속.

동시통역 vs 순차통역 비유

Parallel generation은 동시통역이다 --- 화자가 말하는 동안 실시간으로 번역을 시작한다. Latency가 거의 없지만, 문장의 끝을 모르는 상태에서 번역하므로 때때로 어색한 표현이 나올 수 있다. Serial generation은 순차통역이다 --- 화자가 한 문장을 끝낸 후에 번역을 시작하므로, 정확도는 높지만 대기 시간이 발생한다. Random delay training은 "동시통역과 순차통역 사이의 모든 중간 단계"를 연습하게 하여, inference 시에 상황에 맞는 최적의 strategy를 선택할 수 있게 한다.

Random delay training은 parallel-serial의 전체 spectrum을 커버하여 inference 시 유연한 선택을 가능하게 한다
우리 시스템에의 시사점: Audio tokenizer의 dual-branch 설계가 우리 audio 파이프라인에 직접 적용 가능하다. 특히 decreasing codebook size (8K -> 1K)는 hierarchical에서 semantic(상위) -> acoustic detail(하위)의 자연스러운 분리를 유도한다. Random delay training은 latency-quality trade-off를 inference time에 조절할 수 있게 해준다. Internal Linguistic Guidance (AS/AE/TE tokens)는 speech generation에서 text-level reasoning을 상속받는 핵심 메커니즘이다.

읽기-쓰기 비유

많이 읽는 사람이 더 잘 쓴다 --- LongCat-Next에서도 마찬가지다. Image understanding (읽기)으로 학습한 semantic representation이 image generation (쓰기)의 quality를 높여준다. 핵심은 둘이 같은 discrete token space를 공유하기 때문에, understanding에서 학습한 "semantic plan"이 generation에 직접 전달된다는 것이다.

6.6 전체 성능 Overview: Unified Model 중 최강

LongCat-Next는 understanding과 generation 모두에서 Pareto optimal에 가장 가깝다 (우상단이 최적)
Seminar Point: 기존 unified model들은 understanding이 좋으면 generation이 약하거나 (NEO-unify), generation이 좋으면 understanding이 약한 (OneCat) trade-off를 보인다. LongCat-Next가 유일하게 understanding MMMU 70.6 (최고)이면서 generation GenEval 84.4 (상위)를 동시 달성한다. 이것이 DiNA의 synergy 효과를 가장 잘 보여주는 증거이다.

7. Critical Ablation Results

7.1 Discrete vs Continuous Gap: 300B Tokens에서 수렴

충분한 data scaling (300B tokens)으로 discrete와 continuous의 성능 gap이 ~1%까지 수렴한다
Key Takeaway: "Discrete = inferior"라는 통념은 data scaling 부족에서 비롯된 것이다. 충분한 token으로 학습하면 discrete representation에 inherent performance ceiling이 없다. 이는 우리 시스템에서 discrete tokenization을 채택할 때 가장 중요한 근거가 된다.

  • Pre-Buffer가 gap을 줄인다 (Exp IV vs V): OCRBench 720 vs 705 (+15). 단순 FFN이지만 codebook embedding의 expressiveness를 개선.
  • 300B tokens에서 수렴 (Exp VI vs VII): OCRBench 858 vs 844 (-14, 1.6%), DocVQA 96.0 vs 96.0 (동일!), MMMU 58.0 vs 60.0 (discrete가 오히려 +2). 이는 discrete modeling에 inherent ceiling이 없음을 강력히 증명.
  • MMMU에서 discrete가 역전 (Exp VI vs VII): 60.0 vs 58.0. Semantic reasoning task에서 discrete token이 더 효과적일 수 있음을 시사.
외국어 학습 비유

Continuous representation은 "모국어로 번역해서 이해하는 것"이고, Discrete representation은 "외국어를 직접 배우는 것"이다. 처음에는 번역이 훨씬 빠르고 정확하지만 (초기 gap), 충분한 학습 시간이 주어지면 외국어를 직접 배운 사람이 "원어민 수준"에 도달한다 (300B tokens에서 수렴). 그리고 원어민 수준이 되면, 번역을 거치지 않기 때문에 미묘한 뉘앙스를 더 잘 포착할 수 있다 (MMMU에서의 역전).

Modality-agnostic MoE가 training 과정에서 명시적 지정 없이 자연스럽게 modality-specialized role을 발전시킨다

핵심 발견: Router가 점차 distinct하고 stable한 selection pattern을 발전시킨다. Expert capacity 활용도가 507.1 -> 584.6 (+15.3%)으로 향상. 이는 modality-specific branching 없이도 효과적인 specialization이 가능하다는 것을 의미한다.

8. RL for Discrete Multimodal

Discrete representation은 RL에서 inherent advantage를 제공한다: finite Markov Decision Process (MDP) 구조를 자연스럽게 유지하여, flow model의 ODE-to-SDE 변환 같은 복잡한 과정이 필요 없다.

8.1 RL for Text-to-Image Generation

Modified GRPO: RVQ의 multi-level 특성을 반영하여 per-level probability ratio를 사용:

r_t^(l) = pi_theta^(l)(a_t^(l)|s_t) / pi_old^(l)(a_t^(l)|s_t)

L_GRPO = E[ sum_l w_l * min(r_t^(l) * A_hat, clip(r_t^(l), 1-eps, 1+eps) * A_hat) ]

4가지 Reward Dimension:

8.2 RL for Understanding: Entropy Explosion Problem

Critical Problem: Entropy Explosion

Training이 진행될수록 policy entropy와 training-inference mismatch가 동시에 증가. 소수의 anomalous token에서 sampling probability가 0.4+이지만 training probability는 10^-2 수준 --- 이러한 token이 나머지 generation을 오염시키는 positive feedback loop을 형성한다.

기존 해결 시도 (Truncated IS, Multiple IS, sequence-level correction) 모두 근본적 해결 실패 --- anomalous token의 extreme divergence가 정상 token에 의해 평균화되기 때문.

Solution: Dual Sequence-Level Filter

  1. Entropy-based Filter: 각 minibatch에서 sequence-wise entropy의 mean + n*sigma를 threshold로, 이를 초과하는 sequence를 discard
  2. Training-Inference Probability Difference Filter: 임의 token에서 |pi_sampler - pi_actor| > delta이면 해당 sequence 전체를 discard

두 filter를 모두 통과한 sequence만 gradient update에 기여하는 hard binary mask. Token-level이 아닌 sequence-level filtering으로 소수의 anomalous token이 전체 sequence를 오염시키는 문제를 근본 차단.

9. Infrastructure: VHalf Pipeline Parallelism

V-Half pipeline은 embedding과 modality loss를 동일 device에 배치하여 heterogeneous workload의 bubble을 제거한다

공장 생산 라인 비유

일반적인 pipeline parallelism은 공장의 직선 생산 라인과 같다 --- 각 공정이 순서대로 진행된다. 그런데 첫 공정(재료 투입)과 마지막 공정(품질 검수)이 모두 가벼운 작업이라면, 중간의 무거운 공정들 사이에 idle time이 발생한다. V-Half은 "재료 투입과 품질 검수를 같은 작업자에게 맡기고, 무거운 공정들만 나머지 작업자들이 균등하게 나누는" 전략이다. 재료 투입 후 바로 같은 자리에서 품질 검수까지 할 수 있으니, 재료를 멀리 보냈다가 다시 가져올 필요가 없다 (zero-copy).

9.5. LLM Backbone: LongCat-Flash-Lite A3B

10. Seminar Discussion Points

Discussion 1: "Discrete vs Continuous: is the debate over?"

  • LongCat-Next는 300B token 학습 후 discrete가 continuous와 ~1% margin으로 수렴함을 보인다. 하지만 이것이 모든 task에서 일반화되는가?
  • Pixel-level fidelity에서는 여전히 continuous가 유리할 수 있다 (de-tokenizer가 flow-matching refiner에 의존하는 점에서)
  • Video generation처럼 더 높은 information density를 가진 modality에서도 discrete가 scale할 수 있는가?
  • 우리의 결정: 현재 우리 시스템에서 discrete tokenization을 주축으로 채택해야 하는가, 아니면 hybrid approach가 더 안전한가?

Discussion 2: "Modality-specific branching은 불필요한가?"

  • LongCat-Next는 modality-aware MoE, 3D RoPE, bidirectional attention 등을 일절 사용하지 않는다
  • MoE가 자연스럽게 modality-specialized role을 발전시킨다는 발견 (Figure 11)
  • 반론: Qwen2-VL의 3D RoPE, InternVL의 modality-specific routing이 각 modality에서 더 높은 ceiling을 제공할 수 있다
  • Trade-off: Simplicity + infrastructure 호환성 vs per-modality 최적화. 우리 시스템의 engineering 복잡도 관점에서 어떤 선택이 적절한가?

Discussion 3: "Understanding-Generation synergy의 실용적 함의"

  • Loss 차이 (0.006 und, -0.02 gen)가 downstream task 성능에서도 유의미한 차이를 만드는가?
  • 이 synergy가 audio modality에서도 동일하게 적용되는가? (논문에서 audio에 대해서는 명시적 분석 없음)
  • 우리 파이프라인: 현재 understanding과 generation을 separate stage로 학습하고 있다면, unified loss로 전환하는 것이 이점이 있는가?

Discussion 4: "Training pipeline에서의 practical lessons"

  • 4-stage training (Pre-Align -> Pre-train -> Mid-train -> SFT)의 각 stage가 필수적인가? Stage skipping의 영향은?
  • Pre-Align에서 backbone frozen + embedding/DepthT만 학습하는 것이 critical --- 이 warm-up 없이 바로 full training하면?
  • Data cleaning의 decontamination + quality filtering + diversity preservation이 우리 데이터에도 적용되어야 하는가?
  • Compute budget: 2T tokens x 68.5B model --- 우리가 사용 가능한 compute에서 어떤 scale-down이 가능한가?

Discussion 5: "Random ViT > Pretrained ViT --- Tokenizer 설계의 재고"

  • 이 발견이 의미하는 바: Encoder architecture의 residual connection이 정보 보존의 핵심이지, 학습된 weight가 아니다
  • 그렇다면 SAE (pretrained semantic encoder)는 왜 필요한가? --- Semantic completeness (질문 답변 능력)는 learned feature에서, pixel recovery는 residual pathway에서 각각 온다
  • 우리 tokenizer: 현재 사용 중인 encoder의 residual connection 설계를 재검토해야 하는가? Merger/pooling layer가 pixel info를 과도하게 파괴하고 있지는 않은가?

  • 4가지 reward dimension (comprehensive, OCR, alignment, quality)의 weight 설계가 결과에 얼마나 sensitive한가?
  • Discrete token의 RL 친화성이 continuous latent 대비 실질적으로 얼마나 큰 이점을 제공하는가?

Discussion 7: "Data Scaling Laws for Discrete Multimodal"

11. Limitations and Future Work

De-tokenizer Pixel Fidelity

Pixel decoder가 semantic structure는 잘 복원하지만, high-frequency detail이 부족하다. Flow-matching refiner로 보완하지만, 전문 generative model (FLUX, Seedream) 대비 pixel fidelity에서 열세. FID 등 정량적 pixel-level 분석이 논문에 부재.

Any-to-Any Generation 미지원

현재 image-to-text, text-to-image, audio-to-text 등 pair 단위 변환만 지원. Image+audio 동시 생성, multi-turn grounded dialogue, interleaved multimodal reasoning은 future work. 진정한 omni-modal system으로의 확장이 필요.

재현성과 Compute Cost

68.5B model + 2T tokens은 대부분의 연구 그룹에서 재현 불가. Ablation이 Qwen-7B에서 수행되어 full-scale 결과와의 직접 연관성에 의문. Training FLOPs, inference latency, throughput 분석이 부재하여 practical efficiency 평가가 어렵다.

비교의 공정성

Table에서 *로 표시된 "public report" 수치가 다수이며, 동일 evaluation protocol 여부가 불명확. 주요 결과에 표준편차 미보고. SAE (Qwen2.5-ViT)의 학습 비용이 LongCat-Next의 total compute cost에 미포함.

Vision Tokenizer 미성숙

논문이 인정: 현재 버전은 "fully optimized되지 않았다." De-tokenizer가 semantic decoding consistency에 초점을 두어 pixel fidelity보다 semantic 정확성을 우선시. Video, 3D 등 다른 visual modality로의 확장은 미탐구.

Audio에서 Qwen3-Omni에 일관되게 뒤처짐

ASR, TTS, Audio Understanding 모든 카테고리에서 Qwen3-Omni보다 낮은 성능. 이는 LongCat-Next의 audio가 vision에 비해 상대적으로 less mature하거나, Qwen3-Omni가 audio에 더 많은 resource를 투입했을 가능성을 시사한다. Audio modality의 further optimization이 필요하다.

RVQ의 Computational Overhead

8-level RVQ는 single-level VQ 대비 token 수를 8배로 증가시킨다. 각 spatial position에서 8개의 token을 처리해야 하므로 sequence length가 길어지고, 이는 attention의 quadratic cost를 증가시킨다. 논문이 additive encoding으로 이를 완화하지만, de-tokenization 시의 DepthTransformer overhead에 대한 정량적 분석이 부재하다.

Ablation Scale의 제한

핵심 ablation 실험들 (discrete vs continuous gap, understanding-generation conflict, MoE specialization)이 Qwen-7B backbone에서 수행되었다. Full-scale 68.5B 모델에서 동일한 패턴이 유지되는지의 직접적 검증이 없다. 특히 MoE의 specialization dynamics는 모델 규모에 따라 크게 달라질 수 있다.

Video와 3D 미탐구

현재 vision tokenizer가 image에만 집중하며, video (temporal dynamics)와 3D (depth, spatial structure)는 탐구되지 않았다. RVQ 기반 discrete tokenization이 이러한 더 높은 information density의 modality에서도 scale하는지는 미검증이다. 이는 omni-modal 시스템을 목표로 할 때 중요한 gap이다.

First-generation iPhone 비유

LongCat-Next는 "모든 modality를 discrete token으로 통합"이라는 패러다임의 첫 번째 대규모 실증이다. First-generation iPhone이 완벽하지 않았지만 (카메라 품질, 앱스토어 부재) 스마트폰 패러다임의 viability를 증명한 것처럼, LongCat-Next도 DiNA 패러다임의 가능성을 증명했다. 실제 production deployment까지는 de-tokenizer 품질 개선, any-to-any generation, compute 효율화 등의 iteration이 필요하다.

Key Takeaways for Our Team

  1. Discrete = viable, no inherent ceiling: 충분한 data scaling (300B+)과 적절한 tokenizer 설계 (SAE + RVQ)로, discrete가 continuous와 ~1% margin까지 수렴한다. "Discrete = inferior" 통념을 버려야 한다. 핵심 enabler는 multi-level RVQ (8-stage)와 Pre-Buffer module이다.
  2. Unified training의 이점: Understanding이 generation을 enhance한다 (generation loss -0.02, understanding loss +0.006). 별도 학습보다 unified loss가 유리하다. 이는 understanding이 풍부한 semantic representation을 제공하여 generation의 semantic planning을 강화하기 때문.
  3. Modality-agnostic의 가능성: Explicit modality-specific branching (3D RoPE, bidirectional attention, modality-aware routing) 없이도 MoE expert가 자연스럽게 specialization을 발전시킨다. 이는 engineering simplicity와 LLM infra 호환성의 큰 이점이다.
  4. Tokenizer 설계의 핵심 원칙: (a) Semantic completeness = SAE (language-aligned encoder), (b) Information preservation = RVQ (multi-level residual), (c) Pixel recovery = residual connection (architectural property). 이 세 가지를 분리하여 설계하는 것이 핵심.
  5. RL의 discrete-friendly 특성: Discrete token space에서 기존 LLM RL 방법 (GRPO)을 직접 적용 가능. Continuous latent의 ODE-to-SDE 변환 불필요. Entropy explosion에 대한 dual sequence-level filter 해법이 실용적이며 다른 discrete RL 시나리오에도 적용 가능.
  6. Audio tokenizer의 practical 설계: Whisper encoder + decreasing codebook RVQ + dual branch (understanding + reconstruction) + random delay training. 이 설계가 ASR/TTS/understanding을 unified framework에서 동시에 지원한다.
  7. Platonic representation convergence: 충분한 native multimodal training 하에서 서로 다른 modality의 embedding이 같은 conceptual space로 수렴한다 (interwoven pattern). 이는 cross-modal reasoning과 transfer의 강력한 기반이 된다.

Recommended Action Items

  1. Tokenizer audit: 현재 우리 vision/audio tokenizer의 residual connection 설계와 information preservation 특성을 점검. Merger/pooling layer가 pixel info를 과도하게 파괴하고 있지 않은지 확인.
  2. RVQ 도입 평가: 현재 single-level VQ를 사용하고 있다면, 8-level RVQ로의 전환이 generation quality에 미치는 영향을 작은 scale에서 실험.
  3. Unified loss 실험: Understanding-only와 unified (understanding + generation) training의 A/B 테스트를 소규모로 진행.
  4. RL pipeline 점검: Entropy explosion 발생 여부 모니터링, dual filter 도입 검토.
  5. V-Half pipeline 평가: 현재 training infra에서 embedding/loss module의 workload 불균형 정도를 profile하고, V-Half 적용 가능성 검토.
