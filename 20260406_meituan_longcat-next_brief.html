먼저 알아야 할 것: 멀티모달 AI란?

인간은 눈(시각), 귀(청각), 말(언어)을 하나의 뇌로 처리합니다. AI도 이처럼 여러 감각을 동시에 다루는 것이 목표인데, 이를 멀티모달 AI라고 합니다.

기존 방식: 부품을 따로 만들어 붙이기

CLIP + Adapter + LLM — 부품을 조립한 방식은 이해(understanding)만 가능

새로운 방식: 처음부터 하나로 (Native Multimodal)

이미지, 텍스트, 오디오를 모두 이산 토큰으로 변환 → 하나의 모델이 이해+생성 동시 수행
비유: 다국어 통합 사전

영어·한국어·일본어를 각각 다른 사전으로 번역하는 것이 기존 방식. LongCat-Next는 세 언어를 하나의 통합 사전에 넣어 모든 언어를 같은 체계로 다루는 것입니다.

핵심 문제: 이미지를 "단어"로 바꿀 때의 두 가지 문제

텍스트는 원래 이산적(단어 단위)이지만, 이미지는 연속적(픽셀 값)입니다. 이미지를 "단어"로 바꾸려면 두 문제를 풀어야 합니다.

SAE (Semantic-and-Aligned Encoder)로 이미지의 의미를 완전히 보존
RVQ (Residual Vector Quantization)로 8단계에 걸쳐 점진적으로 변환

SAE: 이미지의 "의미"를 뽑아내는 인코더

SAE의 핵심 원칙은 Semantic Completeness — 변환된 토큰만 보고도 원본 이미지에 대한 모든 질문에 답할 수 있어야 한다는 것입니다.

SAE가 추출한 토큰만으로 원본 이미지에 대한 어떤 질문이든 답할 수 있어야 한다

SAE는 새로 학습하지 않고, 이미 대규모 vision-language 학습을 거친 QwenViT, MoonViT 같은 기존 encoder를 직접 활용합니다.

RVQ: 8단계로 정보 손실 줄이기

한 번에 변환하면 정보가 많이 손실됩니다. RVQ는 "대략적 스케치 → 보정 → 보정 → ..."을 8번 반복해서 원본에 가까워집니다.

8개 level의 codebook이 계층적으로 정보를 보정 — 대략적인 것부터 미세한 질감까지
비유: 화가의 그리기 과정

Level 1은 "대충 윤곽", Level 2는 "형태 잡기", Level 3는 "색칠"... 마치 화가가 스케치부터 시작해 디테일을 점점 추가하듯, RVQ는 각 단계에서 이전이 놓친 부분을 보정합니다.

De-tokenizer: 토큰을 다시 이미지로

Pixel Decoder가 구조를, Flow-Matching Refiner가 디테일을 담당

놀라운 발견: 학습 안 한 ViT가 더 잘 복원한다?!

이미지 복원 능력이 "학습된 지식" 때문인지, "네트워크 구조" 자체 덕분인지 실험한 결과:

학습하지 않은 Random ViT가 복원을 더 잘한다! — Residual Connection 구조 자체가 정보를 보존

시사점: 이미지 복원 능력은 encoder가 배운 "지식"이 아니라, residual connection이라는 구조 자체가 low-level 신호를 보존하기 때문입니다. Tokenizer 설계 시 "residual 구조를 가진 encoder를 쓰는 것"이 핵심!

Audio Tokenizer: 소리도 같은 방식으로

Audio도 Vision과 동일한 RVQ 기반 이산화 — 이해(ASR)와 생성(TTS) 모두 지원

말하기: 동시통역 vs 순차통역

Random Delay Training으로 두 전략을 하나의 프레임워크로 통합 학습
비유: 동시통역 vs 순차통역

Parallel은 동시통역 — 빠르지만 가끔 부정확. Serial은 순차통역 — 정확하지만 지연. LongCat-Next는 학습 시 delay를 랜덤하게 바꿔가며 두 가지 모두를 익힙니다.

핵심 결과: 보고, 그리고, 말하기 모두 잘한다

1. 시각 이해: 수학·문서·차트에서 최강

이산(discrete) 모델링도 연속(continuous) 방식 대비 성능 ceiling이 없음을 증명

2. 이해+생성 시너지

"보는 법을 아는 AI가 더 잘 그린다" — 이해 학습이 생성을 enhance
비유: 독서와 글쓰기

많이 읽는 사람이 글도 잘 쓰듯, AI도 이미지를 잘 이해하면 이미지 생성 품질이 올라갑니다. 이 시너지가 수치로 증명되었습니다.

3. MoE Expert의 자연스러운 전문화

명시적 분기 없이도 MoE expert가 학습 중 자연스럽게 modality별로 전문화

RL: 이산 토큰이면 강화학습도 쉽다

이산 토큰의 큰 장점: 기존 텍스트 RL 기법(GRPO)을 이미지 생성에도 그대로 적용할 수 있습니다.

한계점

재현성

68.5B 모델 + 2T 토큰 학습은 대부분의 연구 그룹에서 재현 불가능합니다.

SAE 비용 미포함

QwenViT를 직접 채택하지만, 이 encoder의 학습 비용은 별도입니다.

Any-to-Any 미지원

현재 image↔text, audio↔text만 가능. 이미지+소리 동시 생성은 미래 과제입니다.

비교 공정성

표에서 *표시 외부 수치가 다수. 동일 evaluation protocol 여부가 불명확합니다.
