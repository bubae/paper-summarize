먼저 알아야 할 것: AI는 어떻게 말하나?

Voxtral TTS를 이해하려면, "텍스트를 음성으로 바꾸는 AI(TTS)"가 어떤 과정을 거치는지를 먼저 알아야 합니다. 세 가지 핵심 개념을 설명합니다.

개념 1: TTS — 글자를 소리로 바꾸는 기술

TTS(Text-to-Speech)는 텍스트를 입력받아 사람처럼 자연스러운 음성을 출력하는 기술입니다. Siri, Google Assistant, 네이버 클로바 등에서 이미 사용되고 있습니다.

TTS는 글자를 넣으면 자연스러운 음성이 나오는 AI 기술

개념 2: "음성 토큰" — AI가 소리를 이해하는 방법

ChatGPT 같은 AI가 텍스트를 "토큰(단어 조각)"으로 쪼개서 이해하듯, 음성 AI도 소리를 "음성 토큰"으로 쪼개서 처리합니다. 이 음성 토큰에는 두 종류의 정보가 담깁니다.

하나의 음성 프레임 = 1개 semantic token (무슨 말?) + 36개 acoustic token (어떤 목소리?)
영화 제작의 비유

Semantic token은 시나리오(대본)입니다 — "어떤 대사를 어떤 순서로 말할지"를 결정합니다.


Acoustic token은 배우의 연기입니다 — 같은 대사라도 "누가, 어떤 감정으로, 어떤 톤으로 말하느냐"에 따라 완전히 달라집니다.


Voxtral TTS의 핵심 혁신: 시나리오 작성과 연기 디렉팅을 완전히 분리하여, 각각에 최적화된 다른 AI를 사용합니다.

기존 TTS의 문제: 왜 느리고, 왜 부자연스러운가?

기존 최신 TTS(예: Moshi)는 음성 토큰을 하나씩 순서대로(autoregressive) 생성합니다. 37개 토큰(1 semantic + 36 acoustic)을 모두 하나씩 생성하면, 아주 느립니다.

기존 방식은 37개를 하나씩 생성(462 steps), Voxtral은 분업하여 9 steps로 해결

Module 1: Voxtral Codec — "소리를 토큰으로 변환하는 번역기"

음성 파형(소리의 진동)을 AI가 처리할 수 있는 토큰(숫자 코드)으로 변환합니다. 1초에 12.5번 "스냅샷"을 찍어, 각 스냅샷을 37개 토큰으로 표현합니다.

Voxtral Codec은 소리를 "의미"와 "음색"으로 분리하여 각각 최적의 방식으로 압축
MP3 압축의 비유

MP3가 음악 파일을 압축하듯, Voxtral Codec은 음성을 압축합니다. 차이점은, 의미(가사)와 음색(악기 소리)을 따로 압축한다는 것입니다. 덕분에 나중에 "같은 가사를 다른 악기로 연주"하는 것(=같은 텍스트를 다른 목소리로 읽기)이 자연스럽게 가능합니다.

Module 2: Ministral 3B — "대본을 쓰는 AI 작가"

Mistral AI의 30억 파라미터 언어 모델이 텍스트를 읽고, semantic token을 순서대로 생성합니다. 이 과정에서 "어떤 발음을, 어떤 순서로, 어떤 억양으로 말할지"를 결정합니다.


입력: [참조 음성 3초] + [읽을 텍스트] → 출력: [semantic token 시퀀스]

참조 음성의 스타일을 학습하여, 텍스트에 맞는 semantic token을 자동 생성

Module 3: Flow-Matching — "연기를 다듬는 AI 성우 감독"

각 semantic token이 결정될 때마다, 3-layer flow-matching transformer가 해당 프레임의 acoustic token 36개를 동시에 생성합니다. 노이즈에서 시작하여 8번의 정제를 거쳐 자연스러운 음질을 만듭니다.

Flow-Matching은 이미지 생성 AI와 같은 원리로 음질을 생성하되, 매우 가볍고 빠름

3초만 들으면 목소리를 복제한다

Voxtral TTS의 가장 놀라운 기능은 Zero-Shot Voice Cloning입니다. 단 3초의 음성 샘플만으로, 그 사람의 목소리로 어떤 텍스트든 읽어줍니다. 별도의 학습이나 fine-tuning 없이, 즉시 가능합니다.

3초 참조 음성만으로, 학습 없이 즉시 9개 언어로 음성 생성 가능

비밀 무기: AI가 스스로 품질을 개선하다 (DPO)

Voxtral TTS는 학습 후 DPO (Direct Preference Optimization)라는 기법으로 스스로 품질을 향상시킵니다. ChatGPT가 "인간 피드백으로 개선"되는 것과 같은 원리입니다.

DPO를 semantic(발음)과 acoustic(음질) 양쪽에 동시 적용한 최초의 TTS 시스템

비교 대상의 한계

Human evaluation에서 비교한 ElevenLabs Flash v2.5는 ElevenLabs의 "빠른 버전(Flash)"이지 최상위 모델이 아닙니다. 또한 GPT-4o-mini-tts 등 최신 대형 모델과의 비교는 포함되지 않았습니다.

학습 데이터 비공개

학습에 사용된 음성 데이터의 규모, 출처, 언어별 비율이 공개되지 않았습니다. 모델 성능의 재현 가능성과 학습 데이터의 저작권/윤리적 문제를 판단하기 어렵습니다.

안전성 우려

3초 음성만으로 목소리를 복제하는 기술은 딥페이크 음성 사기, 사칭 등에 악용될 위험이 있습니다. 논문에서 안전성 가드레일이나 워터마킹에 대한 논의가 부족합니다.

자동 평가 지표의 한계

WER, UTMOS, Speaker Similarity 같은 자동 지표는 실제 사용자가 느끼는 "자연스러움"과 완전히 일치하지 않습니다. 특히 감정 표현, 문맥에 맞는 억양 변화 등은 자동 지표로 측정하기 어렵습니다.

균형 잡힌 시각

Voxtral TTS는 오픈소스 TTS로서는 확실히 새로운 기준을 세웠습니다. Hybrid AR+FM 아키텍처는 이론적으로 우아하고, 실제 성능도 상용 제품과 경쟁할 수준입니다. 다만, 학습 데이터 비공개와 안전성 논의 부족은 실제 도입 시 면밀한 검토가 필요한 부분입니다.
