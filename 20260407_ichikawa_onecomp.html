TL;DR

  • 문제: Post-Training Quantization (PTQ)의 실무 적용이 어렵다 — 알고리즘 선택, bit-width 계획, calibration 전략, hardware 의존적 실행 등 파편화된 도구들을 조합해야 한다
  • 해결: OneComp는 모델 식별자와 GPU 예산만 주면 자동으로 mixed-precision 할당, preprocessing, layer-wise → block-wise → global PTQ의 progressive refinement pipeline을 실행하는 오픈소스 프레임워크
  • 왜 작동하는가: Layer-wise PTQ의 첫 양자화 모델을 pivot으로 삼아, 이후 모든 refinement stage가 동일 모델을 점진적으로 개선하는 구조를 채택. QEP가 error propagation을 보정하고, JointQ가 scale과 integer weight를 공동 최적화하며, MDBF가 1-2 bit 극한 압축을 지원
  • 비유: 전문 요리사가 하나의 요리를 여러 단계(밑간 → 조리 → 소스 → 플레이팅)로 점점 완성해가는 것처럼, OneComp는 양자화를 layer-wise 초벌 → block-wise 중간조리 → global 마무리의 progressive refinement으로 수행하되, 사용자에게는 "한 줄 명령"만 요구한다

핵심 기여 (Key Contributions)

  1. End-to-end automated PTQ pipeline: 모델 식별자만으로 investigation → preprocessing → layer-wise → block-wise → global PTQ → fine-tuning을 자동 실행하는 resource-adaptive 워크플로우
  2. AutoBit: ILP 기반 mixed-precision planning으로, activation-aware Hessian error proxy를 사용하여 layer/module별 최적 bit-width를 자동 할당
  3. QEP + LPCD: ##Layer-wise PTQ에서 error propagation을 closed-form으로 보정(QEP)하고, 기능적으로 결합된 submodule 단위로 coordinate descent 최적화(LPCD)하여, single-GPU에서도 block-wise PTQ에 근접하는 품질 달성##
  4. JointQ: Scale과 integer weight를 local search로 공동 최적화하여 GPTQ 포맷 호환성을 유지하면서 정확도 개선
  5. DBF/MDBF: 1-2 bit 극한 양자화를 위한 structured binary-factor 표현으로, MDBF는 rank-$l$ multi-envelope로 DBF의 magnitude 표현력 한계를 극복

1. Introduction

Foundation model은 수십~수백 billion parameter를 가지며, full precision 배포 시 메모리와 연산 비용이 과도하다. PTQ는 학습 후 weight precision을 줄여 메모리 footprint과 inference latency를 직접 개선하는 핵심 전략이지만, 실무 적용은 fragmented ecosystem이 큰 장벽이다: Hessian-aware rounding (GPTQ), activation-aware scaling (AWQ), rotation-based outlier suppression (SpinQuant), block-wise optimization (BRECQ), binary factorization (DBF) 등이 각기 다른 전제, hyperparameter, failure mode를 가지며, 최적 조합은 모델, task, hardware에 따라 달라진다.

OneComp는 이 expert workflow를 자동화된 reproducible pipeline으로 변환한다. 핵심 설계 원칙 세 가지:

  1. Path-to-plan API: 모델 architecture와 GPU budget에서 workflow를 자동 도출
  2. Resource-adaptive engine: Layer-wise → block-wise → global PTQ로 품질이 단조 증가하며, 사용자는 hardware 예산에 맞는 지점에서 중단 가능
  3. Extensible refiner architecture: 새 알고리즘이 pipeline 재설계 없이 refiner로 통합 가능

2. Background

2.1-2.3 Quantization Fundamentals

Uniform quantization: 실수 값 $w$를 bit-width $b$, scale $s > 0$, zero-point $z$로 정의되는 유한 격자에 매핑:

$$q = \text{clamp}\left(\left\lfloor \frac{w}{s} \right\rceil + z,\; q_{\min},\; q_{\max}\right), \qquad \widehat{w} = s \cdot (q - z) \tag{1}$$

Asymmetric min-max calibration:

$$s = \frac{w_{\max} - w_{\min}}{q_{\max} - q_{\min}}, \qquad z = \text{clamp}\left(\text{round}\left(q_{\min} - \frac{w_{\min}}{s}\right),\; q_{\min},\; q_{\max}\right) \tag{2}$$

Symmetric quantization ($z = 0$):

$$s = \frac{\max(|w_{\min}|, |w_{\max}|)}{2^{b-1} - 1}, \qquad \widehat{w} = s \cdot q \tag{3}$$

Quantization granularity: Per-tensor (전체 행렬에 하나의 scale), per-channel (행마다), per-group (group size $G$의 연속 원소마다). Per-group quantized matrix의 memory footprint:

$$M_{\text{group}} \approx \frac{NM \cdot b}{8} + \frac{NM}{G}(m_s + m_z) \tag{5}$$

2.4 Quantization Targets in Transformer Models

LLaMA-style Transformer block의 구조: RMSNorm → attention ($W_q, W_k, W_v, W_o$) → RMSNorm → SwiGLU FFN ($W_{\text{gate}}, W_{\text{up}}, W_{\text{down}}$). $L$-block 모델은 $7L$개의 large projection matrix + embedding + output head를 가지며, 이들이 parameter 대부분을 차지한다.

2.5 PTQ vs QAT

PTQ는 학습 파이프라인 접근 없이 소규모 calibration set으로 수분~수시간 내에 적용 가능하다. QAT는 STE (Straight-Through Estimator)로 quantization noise 하에서 학습하여 very low bit에서 우수한 정확도를 달성하지만, 수백 GPU-hour의 비용이 필요하고 100B+ 모델로의 scaling이 어렵다.

2.6 PTQ Regimes: Layer-wise, Block-wise, Global

Layer-wise PTQ: 한 번에 하나의 linear layer를 양자화. Peak GPU memory = weight matrix 하나 + Hessian approximation. Single consumer GPU에서 가능하지만, error propagation 문제 (upstream 양자화 오류가 downstream에 누적)가 있다.

Block-wise PTQ: Transformer block 단위로 모든 linear layer를 공동 최적화. Intra-block dependency를 포착하지만 더 많은 GPU memory가 필요하다.

Global PTQ: 전체 모델을 GPU에 올려 KL divergence 등 end-to-end objective로 공동 최적화. 가장 정확하지만 teacher+student 모두 올려야 하므로 매우 비싸다.

##OneComp의 핵심 아이디어: 이 세 regime을 isolated alternative가 아닌 coherent refinement pipeline으로 조직한다. Layer-wise PTQ가 초기 양자화 모델(pivot)을 생성하고, 이후 stage가 동일 모델을 점진적으로 개선한다.##

3. OneComp Overview

Foundation Model → [Investigation + AutoBit] → [Preprocessing (Smooth/Rotation)] → [Layer-wise PTQ (QEP, LPCD, JointQ/MDBF)] → Quantized Model (deployable pivot) → [Block-wise PTQ (Comp + Block-Q)] → improved model → [Global PTQ + Fine-tune (KL distill + QLoRA)] → best model

3.1 End-to-End Workflow

Pipeline은 자동으로 모델 architecture를 inspecting하고 모든 quantizable linear layer를 감지하여 sequential processing order를 결정한다. Calibration data 구성도 자동화 (concat_chunk strategy 등).

3.2 Quantized Model as Pivot

Layer-wise PTQ의 결과를 deployable pivot으로 처리하는 것이 핵심 architectural choice이다. 모든 후속 stage (block-wise, global, fine-tuning)는 이 pivot을 입력으로 받아 개선된 모델을 출력하는 post-processing step으로 구현된다. 장점: (1) 단순한 진입점, (2) 점진적 품질 향상, (3) 자연스러운 extensibility.

3.3 User Interface

최소한의 인터페이스: uv run onecomp <model_path> 한 줄로 전체 파이프라인 실행. Advanced user를 위한 optional argument로 개별 단계 제어 가능.

4. Quantization Engines

4.1 AutoBit: Automatic Mixed-Precision Planning

Transformer layer는 양자화 sensitivity가 현저히 다르다: 초기 layer와 attention output projection은 fragile하고, 중간 MLP layer는 aggressive compression에 내성이 있다. ##AutoBit은 ILP (Integer Linear Programming)로 global memory budget 하에서 layer/module별 bit-width를 자동 할당한다:##

$$\min_{\{c_l \in \mathcal{C}_l\}} \sum_{l=1}^{L} \text{err}(l, c_l) \quad \text{s.t.} \quad \sum_{l=1}^{L} \text{cost}(l, c_l) \leq C^* \tag{16}$$

SCIP solver로 ILP를 풀며, module 수준에서도 서로 다른 bit-width를 허용한다. Error metric은 activation-aware second-order Taylor expansion:

$$\text{err}(l, c_l) \approx \frac{1}{2}\text{tr}\left(B^{(l)}\Delta W^{(l,c)} A^{(l)} \Delta W^{(l,c)\top}\right) \tag{17}$$

여기서 $A^{(l)}$, $B^{(l)}$는 input Gram matrix와 output curvature이다. Fast RTN proxy로 $\Delta W^{(l,c)}$를 계산하여 효율적으로 error를 추정한다.

4.2 Preprocessing via Equivalent Transformations

SmoothQuant: Per-channel scaling vector $\boldsymbol{s} \in \mathbb{R}^N$으로 activation outlier를 weight로 전이:

$$Y = XW = \underbrace{(X\text{diag}(\boldsymbol{s}))}_{\widetilde{X}} \cdot \underbrace{(\text{diag}(\boldsymbol{s})^{-1}W)}_{\widetilde{W}} \tag{18}$$

Rotation-based transforms: Orthogonal transformation $R \in \mathbb{R}^{N \times N}$으로 에너지를 재분배:

$$Y = XW = \underbrace{(XR)}_{\widetilde{X}} \cdot \underbrace{(R^\top W)}_{\widetilde{W}} \tag{19}$$

OneComp는 SpinQuant의 Cayley SGD로 task-adaptive orthogonal rotation을 학습하고, Sinkhorn weight balancing으로 행/열 magnitude를 균일화한다.

4.3 Layer-wise PTQ

4.3.1 Optimization: QEP and LPCD

Standard activation-aware layer-wise PTQ objective:

$$\widehat{W}_l = \underset{\widehat{W} \in \mathcal{Q}}{\text{argmin}} \left\|\widehat{X}_l(\widehat{W} - W_l)\right\|_F^2 \tag{22}$$

여기서 $\widehat{X}_l$은 이전 양자화된 layer들을 거친 perturbed activation이다. 문제: 표준 objective는 원본 weight $W_l$을 target으로 사용하지만, upstream 양자화로 인해 activation이 변했으므로 이는 suboptimal이다.

QEP (Quantization Error Propagation): Error propagation을 명시적으로 보정한 target weight를 사용:

$$\widehat{W}^{\text{QEP}} = \underset{\widehat{W}}{\text{argmin}} \left\|\widehat{X}_l\widehat{W} - X_l W_l\right\|_F^2 \tag{23}$$

Error $\boldsymbol{\delta}_l = X_l - \widehat{X}_l$로부터 error-propagation matrix $\boldsymbol{C}_l = \widehat{X}_l^\top \boldsymbol{\delta}_l$를 계산하고, corrected target:

$$W_l^*(\alpha_l) = \left(I_N + \alpha_l(\widehat{H}_l + \lambda I_N)^{-1}\boldsymbol{C}_l\right)W_l \tag{24}$$

여기서 $\alpha_l \in [0, 1]$이 propagation strength를 interpolate한다. 직관적으로, QEP는 "원래 weight를 맞추는 것"이 아니라 "perturbed input에서도 원래 output을 최대한 재현하는 weight"를 찾는다.

LPCD (Submodule-Aware Coordinate Descent): Transformer 내의 기능적으로 결합된 layer 그룹 (QK module: $\{W_q, W_k\}$, VO module: $\{W_v, W_o\}$, gate-up-down module: $\{W_{\text{gate}}, W_{\text{up}}, W_{\text{down}}\}$)을 cyclic block coordinate descent로 공동 최적화:

$$\min_{\widehat{M}_1 \in \mathcal{Q}, \ldots, \widehat{M}_R \in \mathcal{Q}} \mathcal{J}(\widehat{M}_1, \ldots, \widehat{M}_R) \tag{25}$$

각 iteration에서 하나의 block $M_r$의 양자화 제약을 일시 해제하여 continuous optimal을 구한 뒤, 다시 양자화 domain으로 projection:

$$\widehat{M}_r^{(t)} \leftarrow \Pi_\mathcal{Q}(\overline{M}_r^{(t)}) \tag{27}$$

수렴은 보통 2-4 outer iteration에서 달성되며, 추가 비용은 base layer-wise PTQ 대비 modest하다.

4.3.2 Representation: JointQ and DBF/MDBF

JointQ: GPTQ와 동일한 packed-weight format을 사용하되, scale $s_t$, zero-point $z_t$, integer weight $q_{ij}$를 joint local search로 공동 최적화:

$$\min \|XW - X\widehat{W}\|_F^2 \quad \text{s.t.} \quad \widehat{W}_{ij} = s_{g(i,j)}(q_{ij} - z_{g(i,j)}) \tag{28}$$

각 행에서 integer weight 하나를 변경 제안 → 해당 group의 scale 업데이트 → objective 개선 시 accept. 출력은 standard GPTQ-format checkpoint으로 기존 inference kernel과 호환된다.

DBF (Double Binary Factorization): 1-2 bit 극한 양자화를 위해 weight matrix를 두 binary sign matrix와 diagonal scaling의 곱으로 factorize:

$$\widehat{W}_l = D_{\boldsymbol{a}^{(l)}} S_a^{(l)} D_{\boldsymbol{m}^{(l)}} (S_b^{(l)})^\top D_{\boldsymbol{b}^{(l)}} \tag{30}$$

Inference 시 single high-precision GEMM → two binary GEMMs + diagonal rescaling으로 대체된다.

MDBF (Multi-envelope DBF): DBF의 rank-one magnitude envelope을 rank-$l$ multi-envelope로 확장하여 magnitude 다양성을 표현:

$$\widehat{W}_l = \widehat{U}^{(l)}(\widehat{V}^{(l)})^\top, \quad \widehat{U}^{(l)} = S_a^{(l)} \odot (A^{(l)}(Q^{(l)})^\top), \quad \widehat{V}^{(l)} = S_b^{(l)} \odot (B^{(l)}(G^{(l)})^\top) \tag{31}$$

MDBF는 Multi-Envelope SVID initialization + ADMM-based refinement의 two-stage optimization을 사용한다.

4.4 Block-wise PTQ

4.4.1 Block-wise Compensation

Layer-wise PTQ loop 중 interleaved로 실행되는 optional step. Transformer block 내에서 이미 양자화된 sublayer의 오류를 남은 FP16 parameter가 흡수하도록 최적화:

$$\mathcal{L}_{b,r}^{\text{comp}} = \frac{1}{n}\left\|F_{b,r}^{(\text{mix})}(X_b; \boldsymbol{\phi}_{b,r}) - F_b^{(\text{fp})}(X_b)\right\|_F^2 \tag{32}$$

Adam optimizer, LR $10^{-4}$, 5 epochs. Single-GPU workflow 내에서 자연스럽게 통합된다.

4.4.2 Block-wise Quantization

Layer-wise PTQ 완료 후의 dedicated refinement stage. Full-precision teacher block과 양자화된 student block 사이의 distillation:

$$\mathcal{L}_b = \frac{1}{n}\sum_{j=1}^{n}\left\|Y_{b,j}^{(\text{fp})} - Y_{b,j}^{(\text{q})}(\boldsymbol{\theta}_b)\right\|_F^2 + \lambda_{\cos}\frac{1}{n}\sum_{j=1}^{n}\left(1 - \text{CosSim}(Y_{b,j}^{(\text{fp})}, Y_{b,j}^{(\text{q})})\right) \tag{35}$$

Cross-block quantization: Sliding window ($K=2$)로 인접 block pair를 공동 최적화하여 inter-block error propagation을 감소시킨다.

Smooth Straight-Through Estimator (STE): Discrete rounding의 미분 불가능성을 sigmoid surrogate로 우회:

$$\bar{q}(x) = \lfloor x \rfloor + \sigma(k(\{x\} - 0.5)) \tag{37}$$

Progressive STE Temperature Annealing (PSTA): 학습 중 temperature $k$를 선형 증가시켜 smooth rounding → hard rounding으로 점진적 전환:

$$k_e = k_{\min} + (k_{\max} - k_{\min}) \cdot \frac{e}{E-1} \tag{38}$$

4.5 Global PTQ

전체 모델을 GPU에 올려 end-to-end로 최적화. Loss function:

$$\mathcal{L} = \mathcal{L}_{\text{KL}} + \lambda_{\text{inter}}\mathcal{L}_{\text{inter}} + \lambda_{\text{ent}}\mathcal{L}_{\text{ent}} \tag{39}$$

Token-level KL distillation:

$$\mathcal{L}_{\text{KL}} = \frac{1}{n}\sum_{j=1}^{n}\sum_{v \in \mathcal{V}} p_{v,j}(\log p_{v,j} - \log q_{v,j}), \quad p_{v,j} = \text{softmax}(z_j^{(\text{fp})}/\tau), \quad q_{v,j} = \text{softmax}(z_j^{(\text{q})}/\tau) \tag{40}$$

Intermediate hidden-state alignment (cosine distance):

$$\mathcal{L}_{\text{inter}} = \frac{1}{L}\sum_{l=1}^{L}\left(1 - \frac{\boldsymbol{h}_l^{(\text{fp})\top}\boldsymbol{h}_l^{(\text{q})}}{\|\boldsymbol{h}_l^{(\text{fp})}\|_2 \cdot \|\boldsymbol{h}_l^{(\text{q})}\|_2}\right) \tag{41}$$

Entropy regularization:

$$\mathcal{L}_{\text{ent}} = \frac{1}{n}\sum_{j=1}^{n}\sum_{v \in \mathcal{V}} q_{v,j} \log q_{v,j} \tag{42}$$

Optimization techniques: SAM (Sharpness-Aware Minimization)으로 flat minima 유도, Progressive layer unfreezing으로 출력 가까운 layer부터 점진적으로 학습 범위 확대.

4.6 Quantization-Aware Fine-tuning via QLoRA

최종 optional stage. Quantized backbone을 freeze하고 low-rank adapter ($B \in \mathbb{R}^{N \times r}$, $A \in \mathbb{R}^{r \times M}$)만 학습:

$$W_{\text{eff}} = \widehat{W} + BA \tag{44}$$

Domain adaptation과 양자화 잔여 오류 회복의 이중 목적을 수행한다.

5.6 Progressive Quality Improvement Across Execution Regimes

Llama-3.2-1B에서 DBF 양자화를 적용하고 layer-wise → block-wise → global PTQ의 각 stage별 품질 변화를 측정 (Figure 6):

  • Higher bit-width에서 개선 폭이 크다 (scaling parameter에 더 많은 flexibility)
  • 1 bit에서도 global PTQ가 여전히 beneficial하며, end-to-end KL objective가 block-level MSE로 포착하기 어려운 cross-layer dependencies를 교정한다

6. Related Work

Post-training quantization은 효율적 deep learning의 가장 활발한 연구 분야 중 하나이다. Surveys (Gong et al. 2024, Gholami et al. 2022, Zhao et al. 2025), mixed-precision (APTQ, SqueezeLLM), preprocessing (SmoothQuant, QuIP, QuIP#, QuaRot, SpinQuant, OmniQuant), layer-wise PTQ (GPTQ, AWQ, ZeroQuant, QEP, LPCD, LoaQ), block-wise PTQ (BRECQ, CBQ), global PTQ (LLM-QAT, PV-Tuning, AQLM), extreme low-bit (OneBit, DBF, LittleBit, MDBF, VPTQ, BitNet), fine-tuning (QLoRA) 등 방대한 관련 연구가 있다. OneComp는 이들을 통합된 pipeline으로 조직한다는 점에서 차별화된다.

7. Conclusions

OneComp는 모델 식별자만으로 완전 자동화된 resource-adaptive PTQ toolkit을 제공한다. JointQ는 3-4 bit에서 GPTQ를 일관되게 능가하고, DBF/MDBF는 1-2 bit 극한에서 의미있는 정확도를 보존하며, AutoBit은 memory budget 하의 heterogeneous bit-width 할당을 자동화한다. QEP는 layer-wise와 global PTQ 사이의 품질 gap을 significant하게 축소한다. 향후 activation quantization, KV-cache compression, knowledge distillation, structured pruning을 통합한 comprehensive compression platform으로 확장할 계획이다.

비판적 분석 (Critical Analysis)

  • 실험 규모: 8B/14B 모델에서만 평가. 70B+ 모델이나 MoE 모델에서의 검증이 필요하다
  • Benchmark 한계: WikiText-2 PPL + 4개 zero-shot task만 사용. MMLU, HumanEval 등 현대적 벤치마크에서의 평가가 부재하다
  • Global PTQ 미완성: Global PTQ stage가 "현재 개발 중이며 향후 릴리스 예정"이라고 명시. 핵심 stage의 실제 통합 검증이 아직 완료되지 않았다
  • MDBF 미지원: MDBF도 "향후 릴리스 예정"으로, 가장 혁신적인 extreme low-bit 기능이 아직 사용 불가하다
  • Wall-clock time 미보고: 각 stage의 실행 시간과 GPU memory 사용량에 대한 정보가 없어, "resource-adaptive"의 실질적 의미를 평가하기 어렵다
  • Downstream task 부재: Generation quality (perplexity)와 분류 정확도만 측정하며, 실제 사용 시나리오 (chatbot, code generation 등)에서의 평가가 없다
  • 코드 공개: GitHub에 공개 (https://github.com/FujitsuResearch/OneCompression)되어 재현 가능성이 높다
  • 경쟁 프레임워크 비교 부재: AutoGPTQ, llama.cpp, vLLM 등 기존 양자화 도구와의 직접 비교가 없다

Key Takeaways

  1. Progressive refinement이 핵심 원칙: Layer-wise PTQ의 pivot checkpoint을 기반으로 block-wise → global로 단조 개선하는 구조는, 양자화를 "one-shot" 작업이 아닌 "progressive polishing"으로 재정의한다. 이는 특히 resource-constrained 환경에서 anytime algorithm으로서 가치가 있다
  2. Error propagation correction의 중요성: QEP가 base GPTQ 대비 보여주는 드라마틱한 개선 (PPL 665.94 → 7.67)은, layer-wise PTQ의 핵심 bottleneck이 알고리즘 자체가 아니라 error propagation임을 보여준다
  3. Mixed-precision의 실용성: AutoBit의 결과는 uniform bit-width가 특히 sensitive module이 있는 모델에서 매우 비효율적이며, module-aware allocation이 필수적임을 시사한다
  4. Extreme low-bit의 가능성: MDBF가 1 BPW에서도 의미있는 accuracy를 유지함은, edge deployment를 위한 극한 압축의 실현 가능성을 보여준다
  5. 오픈소스 통합 프레임워크의 가치: Fragmented PTQ 생태계를 하나의 cohesive pipeline으로 통합하는 것 자체가 실무적으로 큰 기여이며, 연구자가 개별 component를 개선하고 비교하기 쉬운 실험 플랫폼을 제공한다
