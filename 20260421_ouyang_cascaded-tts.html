TL;DR

  • 문제: 표현력 있는 conversational TTS는 fine-grained 음성 스타일·감정 제어를 위해 대규모로 감정 라벨링된 음성 데이터가 필요하지만, 이런 데이터는 확보·주석이 매우 비싸다.
  • 해결: LLM이 만드는 textual style token과 사람이 엄선한 audio prompt를 짝지어 cascaded AR-prosody + Diffusion-acoustic 파이프라인에 공급하는 In-Context Learning(ICL) 기반 프롬프트 프레임워크를 제안. 추가로, 추론 시점의 posterior sampling을 학습 시점으로 옮긴 ICL-based online RL에서 AES-CE(주관 미적 품질) 리워드와 CTC (Connectionist Temporal Classification) 정렬 loss를 결합해 AR prosody 모델을 직접 최적화한다.
  • 왜 작동하는가: 각 style token에 대응되는 한 개의 고품질 오디오 프롬프트가 ICL 컨텍스트 역할을 해서, 가중치 업데이트 없이 단일 샘플만으로도 세밀한 스타일이 TTS에 전이된다. AR 모델은 prosody를, Diffusion 모델은 timbre를 담당하도록 **prosody-timbre decoupling**을 도입해 speaker drift를 억제하고, CTC loss가 "reward hacking"(리워드만 높이려고 텍스트를 왜곡하는 현상)을 억제한다.
  • 비유: 외국어 성우 더빙을 한 마디씩 녹음해 두지 않고, "이 문장은 이런 뉘앙스로"라는 텍스트 지시서(textual style token)와 한 줄짜리 견본 녹음(audio prompt) 한 개씩만 주면 성우가 나머지를 즉석에서 해내는 것. 품질 평가는 AES-CE라는 "관객 만족도 계기판"이 실시간으로 체크하고, CTC는 "대본을 빼먹지 말 것"이라는 감독의 주의이다.

핵심 기여 (Key Contributions)

  1. Cascaded Prompting Framework: Textual style token과 audio prompt를 함께 사용하는 2단계(AR prosody + Diffusion acoustic) cascaded TTS 파이프라인. 각 스타일 토큰마다 하나의 고품질 오디오 프롬프트만 있으면 fine-grained 스타일 적응이 가능해, 감정 라벨링 대규모 데이터 없이 ##single-shot 스타일 전이##를 실현한다.
  2. Prosody-Timbre Decoupling: AR prosody 모델은 fine-grained style prompt를 사용해 운율을 제어하고, Diffusion acoustic 모델은 coarse-grained style prompt만 사용해 음색을 제어한다. 두 단계의 audio prompt 화자가 서로 달라도 됨을 실증하며, speaker drift를 구조적으로 완화한다.
  3. ICL-based Online RL: 기존의 "여러 샘플을 뽑아 best 후보를 posterior로 고르기" 대신, 학습 중 AR prosody 모델 자체를 subjective aesthetic reward(AES-CE)로 직접 업데이트. 추론 시 동일한 audio prompt를 사용하므로 policy가 ICL 조건에 맞춰 정렬된다.
  4. CTC Regularizer로 Reward Hacking 억제: AES-CE만 보상하면 텍스트 환각이 유발되는 reward hacking이 발생. 이를 막기 위해 CTC alignment loss를 reward에 결합해 문자열 일치 제약을 유지한다.
  5. 실증적 검증: Human CMOS 평가에서 ICL 모델이 Zero-shot 대비 Naturalness +7.5, CVAD +79.6, GPT-4o 대비 CVAD +5.6 net win rate. RL-AES-CTC가 SFT-only 대비 CMOS +7.1% winrate, 95% CI (3.97%, 10.23%)로 통계적 유의성 확보.

1. 배경과 Motivation

1.1 Expressive TTS의 두 패러다임

Expressive conversational TTS 연구는 크게 두 흐름으로 나뉜다.

  1. 2단계(Cascaded) 파이프라인: AR 모델이 텍스트로부터 semantic/prosody token을 예측하고, NAR(non-autoregressive) 모델이 이 token으로부터 acoustic feature를 생성. 대표적으로 MoonCast[6]와 CoVoMix[7]가 있다. Prosody와 timbre를 별도 단계에서 다루기에 각 축을 독립적으로 제어하기 쉽다.
  2. End-to-End Generative: Flow matching/diffusion 기반 단일 모델로 대화 waveform을 한번에 합성. ZipVoice-Dialog[8], CoVoMix2[9]가 이에 해당. 또 다른 방향으로 NotebookLM[10]·Sesame-CSM[11]은 hierarchical transformer가 token을 autoregressive하게 생성한 뒤 디코더가 waveform으로 변환.

1.2 Emotion 제어 방식의 진화

감정 제어는 크게 3세대로 진화해왔다.

  • 1세대 - Discrete 카테고리: happiness, sadness, anger 같은 mainstream emotion label로 코스한 감정 제어[1, 2].
  • 2세대 - Dimensional 모델: Valence(쾌-불쾌), Arousal(각성), 선택적 Dominance(지배성) 연속 축을 조절해 fine-grained 감정 제어[3, 4].
  • 3세대 - Natural Language Description: 자연어로 감정과 스타일을 자유롭게 기술. EmoVoice[5]가 LLM을 활용해 자유 텍스트 감정 prompting 가능.

본 논문은 이 3세대 흐름에서 출발하지만, 자연어 description만으로는 aesthetic quality와 일관성 제어가 어렵다는 점을 지적하며, 텍스트 토큰 + 오디오 프롬프트의 하이브리드 접근을 취한다.

1.3 강화학습 기반 TTS 개선

TTS 품질 향상을 위한 RL은 크게 두 갈래다.

  • Online RL: 모델이 생성 → reward 평가 → 업데이트 루프를 실시간으로 수행. Emotion detector[12], UTMOS predictor, diffusion-based perceptual loss 등으로 reward 설계[13].
  • Offline RL / Preference Optimization: Gao et al. Emo-DPO[14] 같은 DPO 기반 접근으로 감정 선호도 정렬.

기존 online RL은 감정 ##discriminability나 perceptual speech quality에 초점##을 맞추지만, aesthetic/naturalness에 대한 subjective human preference를 직접 reward로 쓰는 사례는 제한적이었다. 본 논문은 AES-CE(Aesthetic Quality Score - Content Enjoyment)를 reward로 삼고, 대표 prompting 파이프라인과 자연스럽게 결합되도록 설계했다.

1.4 왜 ICL인가

LLM의 ICL(In-Context Learning)은 가중치 업데이트 없이도 예시만으로 새 태스크에 적응한다. TTS 맥락에서도 audio prompt를 "예시"로 제공하면 모델이 paralinguistic 특성(prosody, valence, arousal, dominance)을 추출해 새로운 발화에 적용할 수 있다. 핵심 관찰: fine-grained 스타일마다 단 하나의 high-quality audio prompt만 큐레이션하면 수천 개의 감정 speech sample을 학습 데이터로 수집하는 것보다 훨씬 확장성이 좋다.

2. Proposed Approach

2.1 전체 구조

[User 발화] → ASR → LLM(conversational context 관리) │ ├─ textual style token 생성 │ (예: [excited], [cheerful], [thrilled]) │ └─ 문맥 텍스트 응답(text) │ Audio Prompts Library (style↔audio 매핑) │ │ (fine-grained prompt) │ ▼ ▼ [AR Prosody Model] ← textual style token + 응답 text │ (prosody tokens) │ │ (coarse-grained prompt) ▼ [Acoustic Model (Diffusion)] ← prosody tokens │ ▼ [Vocoder(BigVGAN)] → waveform ▲ │ Online RL (AES-CE + CTC) │ AR prosody만 업데이트

Figure 1에 따르면 전체 파이프라인은 ##ASR → LLM → AR prosody decoder → Acoustic model → Vocoder## 순서이다. LLM은 text 응답과 함께 textual style token을 생성해 TTS에 전달한다. 주목할 것은 cascaded 구조 안의 두 지점(AR, Acoustic)에서 audio prompt가 각각 ##granularity 다르게## 투입된다는 점이다.

2.2 Cascaded Prompting

2.2.1 Textual Style Token

LLM이 expressive text prompt와 surrounding conversational context를 바탕으로 contextually 적합한 style token을 생성한다. 예시: [delightful], [excited], [cheerful], [thrilled]. 이 토큰이 TTS의 1차 expressivity control signal이다. 중요한 설계 특징: fully compatible with real-time AI systems. 즉, LLM 생성 토큰을 그대로 downstream TTS가 소비하므로, 추가 intermediate 모듈 없이 실시간 음성 응답 시스템에 꽂을 수 있다.

2.2.2 Audio Prompt의 역할

Textual style token은 "어떤 스타일인지"를 나타내는 ##discrete 라벨##이지만, 모델이 그 라벨의 실제 paralinguistic 정취(prosody, valence, arousal, dominance)를 구체화하려면 참조 음성이 필요하다. Audio prompt가 이 역할을 한다. 각 스타일 토큰마다 10개 이상의 후보 오디오를 생성해 사람이 선별하며, 최종 한 개만 사용한다.

2.2.3 AR Prosody Prompting (Fine-grained)

AR prosody 모델은 대규모 인간 음성 코퍼스로 사전학습되어, 임의 voice style의 prosody를 capture하는 discrete acoustic token을 생성하도록 pretrained 되어 있다. Fine-grained audio prompt가 AR 모델에 투입된다. 선별 절차:

  1. 각 candidate audio prompt 당 ##AES-CE##(Aesthetic Quality Score on Content Enjoyment)를 Monte Carlo로 평가하고 lower-bound score를 계산.
  2. Lower-bound가 threshold 이상인 후보만 살리고 최고 점수 후보를 선택.
  3. Human-in-the-loop validation으로 artifact(hallucination, 부자연스러운 호흡 등) 필터링.

2.2.4 Diffusion-based Acoustic Prompting (Coarse-grained)

Acoustic 단계에서 동일한 fine-grained prompt를 쓰면 speaker drift가 발생한다. 같은 speaker라도 녹음 세션마다 volume, spatial characteristic, timbre가 달라져 multi-turn 대화 중에 화자가 "변한 것처럼" 들리는 문제다. 해결책:

  • Fine-grained style을 몇 개의 coarse category로 그룹화하고, 각 category의 prompt를 acoustic prompting에 사용.
  • Figure 2에서 세 가지 prompting 설정(no prompt / fine-grained / coarse-grained)을 비교한 결과, ##coarse-grained가 모든 스타일 토큰에 대해 speaker similarity를 일관되게 더 높게 유지##한다.
[Figure 2 설명]: x축은 textual style token(delightful, thrilled, cheerful ... apologetic 등 20여 개), y축은 평균 speaker similarity(0.20 ~ 0.80). 세 곡선은 각각 (1) default only prompt, (2) fine-grained style prompt, (3) coarse-grained style prompt. Coarse-grained 곡선이 대부분 0.60 ~ 0.80 구간에 분포해 다른 두 조건을 지속적으로 상회. 이는 acoustic 단계에서 style granularity를 낮춘 prompt가 timbre 일관성에 유리함을 직관적으로 보여준다.

2.2.5 Decoupled Control의 핵심 발견

실험적으로 매우 흥미로운 관찰: AR prosody용 audio prompt 화자 ≠ Acoustic용 audio prompt 화자여도 품질이 유지된다. 이는 다음을 시사한다.

  • Voice timbre는 ##주로 acoustic 단계의 prompt##에서 결정된다.
  • AR 모델은 주로 prosody(운율)를 제어한다.
  • 따라서 prosody style library와 timbre(speaker) library를 독립적으로 구성·혼합할 수 있다 → 소수 speaker × 다수 스타일 조합만으로 방대한 variation 가능.

2.3 Generation Quality Evaluation Protocol

평가 프로토콜은 subjective human listening의 체계화를 목표로 한다.

  • CVAD: Clarity, Valence, Arousal, Dominance 4개 paralinguistic dimension을 5점 Likert로 평가. Rater는 각 dimension의 appropriateness와 overall expressivity를 판단.
  • CMOS (Comparative MOS): 동일 conversational context에서 서로 다른 시스템의 AI voice를 비교해 net win rate를 산출.
  • Speaker Consistency: Multi-turn 대화에서는 ECAPA-TDNN speaker verification 모델로 각 샘플 embedding과 reference target embedding(큐레이션한 target speaker embedding pool의 평균)의 cosine similarity를 측정. threshold 0.7 이하이면 speaker drift로 판단.

2.4 ICL-based Online Reinforcement Learning

2.4.1 설계 동기

전통적 posterior sampling 방식은 추론 시 여러 candidate를 뽑고 best를 후보 중에서 re-rank해 선택한다. 계산 비용이 크다. 대신 저자들은 ##reward를 학습 중에 직접 사용##해 "한 번에 best를 낳도록" AR prosody 모델을 업데이트한다. RL policy는 inference와 동일한 audio prompts에 조건부이므로 ICL-based online RL이다. 즉, 모델은 예시(prompt) 조건에서 더 나은 음성을 만드는 법을 학습한다.

2.4.2 Reward 함수

AES-CE는 human preference와 잘 correlate하는 perceptual metric이다. 그러나 AES-CE만 최적화하면 reward hacking이 발생: 모델이 aesthetic score를 올리려고 text content와 무관한 음성을 만들어내거나(text hallucination) 문자열을 왜곡한다. 해결로 CTC loss를 reward에 결합한다.

결합 reward:

$$ R(\tau) = \alpha_{\mathrm{AES}} \cdot \mathrm{AES}(\mathcal{F}(\tau)) - \alpha_{\mathrm{CTC}} \cdot \mathcal{L}_{\mathrm{CTC}}(\tau, w_0) \qquad (1) $$

여기서

  • $\tau$: AR prosody model이 transcript $w_0$에 대해 생성한 discrete audio token 시퀀스
  • $\mathcal{F}(\cdot)$: frozen acoustic model + vocoder pipeline. $\tau$를 실제 waveform으로 변환.
  • $\mathrm{AES}(\cdot)$: 합성된 waveform에 대한 AES-CE score
  • $\mathcal{L}_{\mathrm{CTC}}(\tau, w_0)$: 생성된 audio token과 ground-truth transcript의 CTC alignment loss
  • $\alpha_{\mathrm{AES}} = 1.0$, $\alpha_{\mathrm{CTC}} = 3.0$: 두 항의 balance

직관적으로 말하면: "aesthetic은 올리되, 대본과 발음이 정렬되는 정도를 강하게 penalize"하는 정책을 학습한다. $\alpha_{\mathrm{CTC}}$가 $\alpha_{\mathrm{AES}}$보다 3배 크다는 점이 중요 — 저자들은 AES만 올리려는 drift를 실제로 강하게 억제해야 함을 보여준다.

2.4.3 RL Objective

$$ J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta} [R(\tau)] - \beta \cdot \mathrm{KL}(\pi_\theta \| \pi^0) \qquad (2) $$
  • $\pi_\theta$: AR prosody 모델의 online(trainable) policy
  • $\pi^0$: reference policy, ##SFT(Supervised Fine-Tuned) baseline##
  • $\beta = 50.0$: KL regularization 계수. 정책이 reference에서 너무 멀어지지 않도록 제한.

직관적으로: reward 최대화 + SFT baseline과의 KL 거리 최소화. SFT만으로 학습된 안정적 policy를 anchor로 삼아 그 근방에서 reward를 개선하는 보수적 RL.

2.4.4 Training 세팅

  • Monte Carlo estimation: $\mathbb{E}[R(\tau)]$ 추정을 위해 transcript 당 6 samples 사용.
  • Audio prompt는 학습과 추론 양쪽에서 동일한 것을 사용 → training efficiency 및 ICL 조건 재현.
  • Optimizer: Adam, learning rate $5.0 \times 10^{-7}$.
  • Acoustic model과 vocoder는 완전히 freeze, AR prosody model만 업데이트.

Audio-conditioned transcript dataset: (audio prompt, target transcript) pair로 구성하되, audio prompt의 내용과 target transcript는 독립. 즉, "스타일만 audio prompt에서, 내용만 transcript에서"라는 ICL 구도를 정확히 학습 시그널로 구현.

3. 실험 (Experiments)

3.1 실험 세팅

  • Baseline 모델: Meta LLaMA 3 70B[16] + 인하우스 TTS 시스템. TTS 구조는 Tortoise-TTS[17] 유사, vocoder는 BigVGAN.
  • 데이터셋: 인하우스 voice actor 데이터. Vendor가 coarse-grained voice style 제공. 그 후 청취 판단을 통해 fine-grained style audio로 re-map.
  • 평가 대상: 10 speakers (7명 미국 영어, 3명 영국 영어), 스타일당 400 샘플 생성.
  • CVAD 평가: 스타일당 50 샘플, 5명의 전문 rater.

3.2 ICL Prompting 효과 (Cascaded Prompting vs Zero-shot)

실험 의도: Cascaded prompting(ICL)이 zero-shot baseline(decoder에 prompt 없음, acoustic model에 default prompt만 있음) 대비 실제로 naturalness와 expressivity를 개선하는가? 그리고 GPT-4o external TTS API[18]와 비교해 어떠한가?

이 실험이 뒷받침하는 주장:

  • Naturalness +7.5: ICL은 이미 강한 zero-shot TTS baseline 대비 지각 naturalness에서 ##명확한 향상##을 보여준다.
  • CVAD +79.6: Expressivity 차원에서는 차이가 지배적. Prompt가 없는 baseline은 emotion/valence/arousal을 거의 생성하지 못하는 반면, ICL은 fine-grained paralinguistic variation을 잘 살려낸다.
  • CVAD +5.6 vs GPT-4o: 상용 강력 baseline 대비해서도 cascaded prompting이 우위를 보임. 특히 human-in-the-loop audio prompt 큐레이션이 기여한다는 해석.

(c) CTC training loss without CTC backpropagation: 초반 감소 후 1.6 부근에서 진동하며 더 높은 수준에 머묾. 즉, CTC를 reward에 결합해 실제로 역전파해야만 hallucination이 억제된다는 증거.

정량 분석: CTC loss를 backprop하지 않은 설정에서는 AES-CE 리워드는 오르지만 CTC loss가 크게 유지되거나 오히려 나빠진다. 이는 reward hacking의 직접적 증거 — 모델이 텍스트 대본을 정확히 발음하지 않으면서 AES-CE만 올리는 생성을 학습한다. CTC regularization이 있을 때만 두 가지(품질↑, 정렬 유지) 목표가 동시에 달성된다.

4. 비판적 분석 (Critical Analysis)

4.1 실증 규모의 한계

  • CVAD 평가는 스타일당 50샘플, rater 5명으로 수행. Variance가 적지 않을 수 있으며, 서로 다른 스타일 간 효과 차이에 대한 per-style 분해는 제공되지 않는다.
  • Subjective metric에 크게 의존: AES-CE 자체가 subjective 판단을 모방한 모델이고, 최종 평가도 CMOS 기반. 객관적 지표(예: WER, F0 RMSE, speaker similarity score)를 추가로 공개했다면 reproducibility가 개선됐을 것이다.
  • 비교 대상 GPT-4o는 TTS 전용 모델이 아닌 multimodal system. "같은 조건 비교"는 아니지만 외부 참조로서 의미.

4.2 데이터·모델 공개 여부

  • TTS 시스템, 데이터, LLaMA 3 70B 기반 인프라 모두 인하우스이며 코드/모델 공개 X. 저자들도 "유사한 구조 + 공개 expressive speech data로 대체 재현 가능할 것"이라고 언급하지만 이는 검증 불가능한 주장이다.
  • Audio prompt library 구성에 사람이 개입한 human-in-the-loop validation은 scale이 커질수록 병목이다. 스타일 수가 수백 개로 확장될 때의 경제성 검토가 필요.

4.3 방법론적 고려사항

  • KL 계수 $\beta = 50$은 매우 큰 값으로, SFT 근방에서 작은 반경만 탐색하는 보수적 policy가 된다. 더 공격적 탐색(작은 $\beta$)에서의 행동(예: 지나친 감정 과장)에 대한 ablation이 없다.
  • Prosody-timbre decoupling은 ##매력적이지만 조건부 가정##이다. 특정 스타일(예: whisper)에서 prosody와 timbre를 완전히 분리할 수 있을지, 그리고 낯선 speaker/스타일 조합에서 fallback이 어떤지에 대한 논의 부재.
  • AES-CE가 "Content Enjoyment"를 모델링한다고는 하나, 음성 분야에서의 human preference는 다양한 축(감정 적합도, clarity, pacing 등)을 포함한다. 단일 reward로 축약된 것이 expressivity의 특정 축을 강조·억제했을 가능성.

4.4 Ethics와 Safety

  • Voice cloning 관련 ethics statement가 본문에 명시되지 않았다. 인하우스 voice actor data 사용이라는 점은 긍정적이나, 실제 배포 시 consent mechanism과 misuse 방지(deepfake 음성) 가이드가 필요.
  • Audio prompt 큐레이션 시 rater들의 bias(연령, 문화권 등)가 "aesthetic" 선호에 반영되므로, 다양한 인구 집단에 대한 공정성 검증이 이상적이다.

5. Key Takeaways

  • Cascaded + ICL prompting: "Textual style token(LLM 생성) + audio prompt(사람 큐레이션)" 짝이 대규모 emotion speech 라벨링 없이 fine-grained expressive TTS를 single-shot으로 구현하는 강력한 레시피. Conversational AI 시스템에 바로 꽂기 쉬운 구조.
  • Prosody-timbre decoupling: AR(prosody) 단계에는 fine-grained prompt, Acoustic(timbre) 단계에는 coarse-grained prompt를 쓰면 speaker drift를 억제하면서 expressivity를 살릴 수 있다. 각 단계의 prompt 화자가 달라도 된다는 점은 voice·style library의 독립적 확장을 허용한다.
  • ICL-based Online RL: Posterior re-ranking 대신 policy를 직접 학습하면 inference는 단일 샘플로 저렴해지고 품질은 CMOS +7%로 향상. Reward를 AES-CE 같은 perceptual model로 설계하면 subjective naturalness와 정렬.
  • CTC regularization의 중요성: Perceptual reward 단독 최적화는 ##text hallucination 형태의 reward hacking##을 유발한다. CTC loss를 $\alpha_{\mathrm{CTC}} = 3\alpha_{\mathrm{AES}}$ 비중으로 reward에 결합하면 이 drift가 실험적으로 제어된다.
  • 실무 적용: 실시간 음성 어시스턴트, 팟캐스트 생성, multi-character 대화 합성 등에 바로 응용 가능. Audio prompt library를 조직 내 성우 데이터로 구축하면 low-labeling-cost 고품질 expressive TTS를 얻을 수 있다는 point가 특히 실용적이다.
