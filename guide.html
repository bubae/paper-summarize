Paper Summarize 사용 가이드

이 프로젝트는 무엇인가?

AI/ML 논문 PDF 또는 URL을 입력하면 Claude Code가 연구자 수준의 섹션별 요약을 HTML로 자동 생성하는 워크플로우입니다. 수식은 KaTeX로 렌더링되고, 비판적 분석과 검증 과정이 포함됩니다.

프로젝트 구조

  • CLAUDE.md — 워크플로우 지침 및 템플릿 (Claude Code가 참조)
  • index.html — 요약 목록 및 네비게이션 (자동 생성/업데이트)
  • guide.html — 이 사용 가이드 (CLAUDE.md 기반 자동 생성)
  • *.html — 개별 논문 요약 파일 ({YYYYMMDD}_{첫저자성}_{키워드}.html 형식)
  • pdf_directory/ — 논문 PDF 저장 디렉토리 (git에서 제외)

입력 방법

Claude Code에서 아래 방법 중 하나로 논문을 전달하면 자동으로 요약이 생성됩니다.

1. PDF 디렉토리

pdf_directory에 새 논문 있으면 요약해줘

pdf_directory/에 PDF를 넣어두면, 기존 요약 HTML과 대조하여 아직 처리되지 않은 PDF를 자동 감지하고 순차 처리합니다.

2. URL 직접 입력

https://arxiv.org/abs/2602.21371

arXiv URL을 주면 자동으로 PDF를 다운로드하고 요약합니다. 여러 URL을 한번에 줄 수도 있습니다:

https://arxiv.org/abs/2602.12345
https://arxiv.org/abs/2603.67890
https://arxiv.org/pdf/2604.11111

3. 파일 목록 (txt / csv)

URL이나 파일 경로가 담긴 텍스트 파일을 전달할 수 있습니다.

txt 예시 (papers.txt):

# 이번 주 읽을 논문
https://arxiv.org/abs/2602.12345
https://arxiv.org/abs/2603.67890
/path/to/local/paper.pdf

csv 예시 (papers.csv):

url,memo
https://arxiv.org/abs/2602.12345,attention 관련
https://arxiv.org/abs/2603.67890,RL 관련

요약 워크플로우 (3단계)

Phase 1 — 빠른 개요 (Quick Overview)

  1. 메타데이터 추출: 제목, 저자, 학회/저널, 연도
  2. 논문 구조(섹션 목록) 파악
  3. TL;DR 생성 (문제 → 해결 → 왜 작동하는가 → 직관적 비유)
  4. 핵심 기여(Key Contributions) 3~5개 bullet 정리

Phase 2 — 섹션별 정밀 요약 (Deep Dive)

각 섹션을 순서대로 깊이 있게 순차 처리합니다. 수식에는 직관적 설명을 병행하고, 실험 결과에는 패턴 분석을 추가합니다.

Phase 3 — 검증 (Verification)

별도의 검증 에이전트가 원문 대비 요약을 교차 검증합니다.

#검증 항목확인 내용
1섹션 완전성원문의 모든 섹션이 요약에 포함되었는가?
2수치 정확성정확도, 파라미터 수 등 핵심 수치가 원문과 일치하는가?
3수식 검증주요 수식이 원문과 동일한가?
4누락된 핵심 내용중요한 실험 결과, ablation이 빠지지 않았는가?
5환각 검출원문에 없는 내용이 추가되지 않았는가?
6템플릿 준수모든 필수 섹션이 채워졌는가?
7용어 일관성약어 풀어쓰기, 기술 용어 영문 유지

검증 판정:

  • PASS: 수정 사항 없음 → 완료
  • MINOR: 경미한 수정 → 자동 수정 후 완료
  • MAJOR: 중대한 누락/오류 → 재작성 후 재검증

요약 구조

모든 논문 요약 HTML은 다음 섹션들을 포함합니다:

  1. 메타데이터: 저자, 소속, 학회/저널, 연도, 링크
  2. TL;DR: 문제, 해결, 왜 작동하는가, 비유
  3. 핵심 기여 (Key Contributions): 3~5개 ordered list
  4. 섹션별 정밀 요약: 원문 구조를 따라 깊이 있는 요약
  5. 비판적 분석: 강점, 한계, 질문
  6. Key Takeaways: 이 논문에서 가져갈 것 3~5개
  7. 개인 메모: Q&A 및 추가 코멘트 누적 영역

AI 논문 특화 가이드라인

반드시 포착하는 핵심 요소

  • 모델 아키텍처: 층(layer) 단위 설명, 입력→출력 흐름
  • 손실 함수 (Loss Function): 전체 목적함수와 각 항(term)의 역할
  • 학습 세부사항: optimizer, learning rate, batch size, epoch, hardware
  • 데이터셋: 이름, 규모, 전처리, train/val/test 분할
  • 베이스라인 비교: 비교 모델, 공정성 여부
  • Ablation Study: 컴포넌트 제거/변경 시 성능 변화
  • SOTA 달성 여부: 벤치마크, 개선 폭

분야별 중점 확인 사항

분야중점 확인 사항
NLP / LLMtokenizer, context length, pre-training corpus, fine-tuning 전략, 평가 벤치마크
Computer Visionbackbone, resolution, augmentation, FLOPs/파라미터 수
Reinforcement Learningenvironment, reward design, exploration 전략, sample efficiency
Generative Model생성 품질 지표 (FID, IS, CLIP score), sampling 방법, inference 속도
Graph Neural Network그래프 구성 방법, message passing 방식, scalability
Multimodalmodality 간 alignment 방법, fusion 전략, 각 modality의 encoder

서술 품질 원칙

  • 스토리라인 우선: "왜 → 어떻게 → 효과" 흐름
  • Background 충분히: 해당 분야에 익숙하지 않은 연구자도 이해할 수 있는 수준
  • 수식 + 직관: 모든 핵심 수식 뒤에 직관적 한 줄 설명
  • 실험은 "왜 → 결과 → 뒷받침": 실험 의도, 수치, 주장 연결을 명확히
  • 패턴 분석: 테이블 나열 후 인사이트 서술

수식 / 그림 / 테이블 처리

요소처리 방식
수식KaTeX 렌더링. 인라인 $...$, 블록 $$...$$. 원문 notation 유지
그림blockquote로 내용과 의미를 상세히 기술
테이블HTML 테이블로 재현. 큰 테이블은 핵심 행 발췌 + 트렌드 설명
알고리즘pseudo-code + 텍스트 흐름도
하이라이트##텍스트##로 감싸면 노란 배경 강조 (핵심 기여, SOTA 수치, 핵심 인사이트 등)

Q&A 처리

요약된 논문에 대해 질문하면, 답변과 함께 해당 내용이 요약 HTML에 자동 반영됩니다:

  • 특정 섹션과 관련되면 해당 섹션 끝에 Q&A 블록으로 추가
  • 횡단적인 질문이면 문서 끝 "개인 메모" 섹션에 추가

정확성 기준

  • 원문에 없는 내용 추가 금지 (비판적 분석, Key Takeaways 제외)
  • 수식 번호는 원문과 일치
  • 약어는 첫 등장 시 풀어쓰기
  • 섹션 누락 없이 전체 커버
  • 하이퍼파라미터, 데이터셋 규모 등 구체적 수치 반드시 포함

언어 규칙

  • 기본 언어: 한국어
  • 기술 용어: 영어 원문 유지 (예: "attention mechanism", "gradient descent")
  • 형식: 한국어 설명 (English term)

파일 저장 규칙

  • 요약 파일: {YYYYMMDD}_{첫저자성}_{키워드}.html
  • 날짜는 요약을 생성하는 당일 기준
  • index.html에 자동 등록 (역시간순 정렬)
  • git commit & push로 GitHub Pages에 자동 배포
