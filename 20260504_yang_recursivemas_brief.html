"AI들이 서로 말로 설명하지 않고,
각자의 생각 자체를 직접 전달하면 어떻게 될까?"

왜 이 논문이 중요한가

최근 AI는 한 명의 거대 모델에서 여러 전문가 AI가 협업하는 멀티에이전트(multi-agent) 구조로 진화하고 있습니다. 코드를 짜는 AI, 검토하는 AI, 통합하는 AI가 팀을 이루는 식이죠.

그런데 이 협업에는 보이지 않는 비효율이 있습니다. 에이전트들이 서로 "말(텍스트)"로 통신하기 때문에 매번 자기 생각을 단어로 옮기고, 다른 AI는 그걸 다시 읽고 이해해야 합니다. 사람의 회의처럼 들리지만 — AI 입장에서는 매번 같은 정보를 압축·해제하는 낭비입니다.

RecursiveMAS는 이 비효율을 정면으로 다룹니다. 에이전트들이 아예 자기 생각의 "원본"인 잠재 표현을 직접 전달하도록 만들어, 협업 속도를 최대 2.4배 높이고 토큰 사용을 75.6%까지 줄이면서 정확도까지 평균 +8.3% 끌어올렸습니다.

배경 지식 — 4개 개념을 차근차근

① 멀티에이전트 시스템(MAS)이란?

여러 AI가 역할을 나눠 맡고 협업하는 시스템입니다. 예를 들어 수학 문제를 풀 때:

  • Planner: 문제를 단계로 쪼갬
  • Solver: 각 단계를 실제로 계산
  • Critic: 답이 맞는지 검토

② 텍스트 기반 통신의 문제

지금까지 AI들은 "사람이 읽을 수 있는 글"로 통신했습니다. Planner가 문장을 만들고 → Solver가 그 문장을 읽어 이해 → 또 문장을 만들고 → Critic이 읽고 ... 그런데 AI가 글을 만드는 것은 매번 수만 단어 후보 중 하나를 선택하는 큰 연산이에요. 이걸 매 협업 단계마다 해야 하니 누적 비용이 커지죠.

또한 글로 옮기는 순간 정보가 단순화됩니다. AI 머릿속의 풍부한 의미(수천 차원 벡터)가 단어 하나라는 좁은 통로로 흘러나가니까요.

화상 회의에서 매번 자기 생각을 영어로 적어 채팅창에 올리고, 상대는 그걸 다시 자기 머릿속 개념으로 변환하는 협업을 상상해보세요. 의미는 전달되지만 매번 "한국어 → 영어 → 한국어" 변환을 거치는 셈입니다.

③ Recursive Language Model이란?

최근 AI 연구의 새 화두 중 하나는 "더 큰 모델 대신, 같은 모델을 여러 번 돌리자"입니다. 이를 recursive language model이라고 합니다. 모델 안의 신경망 층(layer)을 여러 번 반복 적용해서 reasoning 깊이를 키우는 거죠.

RecursiveMAS의 핵심 아이디어: 각 에이전트 = 거대한 가상 모델의 한 층으로 보고, 시스템 전체를 한 개의 recursive model처럼 다루자!

④ Latent Thought (잠재 사고)

AI가 글을 만들기 직전, 머릿속에는 단어 하나가 아니라 수천 차원의 벡터(예: 4096차원)가 있습니다. 이게 진짜 "의미"이고, 단어는 그걸 압축한 결과예요. RecursiveMAS는 이 압축을 안 하고 벡터 자체를 다음 단계로 넘깁니다.

그림을 설명할 때 보통은 "노을이 아름다운 풍경"이라고 글로 옮기죠. 그런데 그림 파일 자체(원본 픽셀)를 그냥 넘겨주면 정보가 훨씬 풍부합니다. RecursiveMAS는 "그림 파일 자체"를 넘기는 셈.

핵심 아이디어 — 3가지 모듈

STEP 1
Inner Link (자기 자신과의 재귀)

같은 에이전트가 여러 step을 통과할 때, 마지막 hidden 벡터를 다음 step의 입력 벡터로 직접 매핑합니다. 글로 디코딩 안 하고 latent thought 시퀀스를 누적.

STEP 2
Outer Link (이질 에이전트 간 다리)

크기가 다른 두 LLM은 hidden 차원이 다릅니다. Outer link가 한 에이전트의 latent를 다음 에이전트의 좌표계로 변환해 호환성을 만듭니다.

STEP 3
Inner-Outer Loop 학습

1단계: 각 에이전트의 inner link를 따로 워밍업. 2단계: 시스템을 n round 펼쳐 마지막 출력 정답과 비교한 손실로 모든 outer link를 한 번에 학습.

RecursiveLink의 구조

residual 구조 덕분에, 모듈은 "처음부터 새 좌표를 만들지" 않아도 되고 "원본과의 미세한 차이"만 학습합니다. 이게 학습 안정성과 효율의 핵심.

이미 작동하는 자전거를 받았다고 해봅시다. 새 자전거를 0부터 만드는 대신, "이 자전거에 보조 안장 하나만 더 달자"라고 하면 훨씬 쉽죠. residual은 "원본 그대로 + 보정만"이라는 뜻입니다.

왜 이렇게 절감되는가? (시각화)

다양한 협업 패턴에서 전부 효과

협업 패턴별 결과:

  • Sequential (Planner→Critic→Solver): +8.3% 평균 정확도, ×2.4 속도
  • Mixture (Math/Code/Sci 전문가 병렬): +6.2% (가장 강한 specialist 대비)
  • Distillation (큰 Expert → 작은 Learner): Learner +8.0% & Expert 대비 1.5× 빠른 추론
  • Deliberation (Reflector + Tool-Caller): tool-calling 성능 +4.8%

즉, 특정 architecture에 종속되지 않고 모든 협업 형태에서 일관된 이득.

한계 및 주의점

① 해석 가능성의 trade-off

잠재 통신은 사람이 읽을 수 없습니다. 디버깅, 안전성 검사, 사용자 신뢰 영역에서 "왜 이런 답을 냈는지" 추적이 어려워집니다.

② 새 조합마다 outer link 재학습 필요

에이전트가 바뀌면 차원이 다른 link를 다시 학습해야 합니다. plug-and-play modularity와는 거리가 있어요.

③ 사용자가 중간에 끼어들기 어려움

마지막 라운드에서만 텍스트가 나오므로, multi-turn dialogue나 사람-인-더-루프 시나리오에는 부적합합니다.

④ 그래디언트 안정성의 가정

이론적 보장은 "AI가 confident할 때(token entropy 작을 때)" 가정합니다. 실제로 어려운 reasoning에서는 uncertainty가 큰 영역이 많아, 가정이 깨질 수 있습니다.

가져갈 인사이트

  1. "여러 AI = 가상의 큰 한 모델"이라는 관점 전환. 협업 시스템을 단일 학습 대상으로 다룰 수 있음.
  2. residual + latent 통신은 학습 안정성과 효율을 동시에 가져다주는 매우 적은 design.
  3. Recursion depth = 새 scaling axis. 더 큰 모델이 아니라 "같은 모델 여러 번"으로 reasoning 깊이를 늘릴 수 있음.
  4. 이질 모델도 묶을 수 있다. 작은 specialist + 큰 일반 모델을 한 시스템으로.
