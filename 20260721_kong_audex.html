TL;DR

  • 문제: Qwen3-Omni, Qwen3.5-Omni 같은 unified audio-text LLM은 audio 능력을 추가하는 대신 text reasoning 능력이 크게 저하되는 문제(regression). Reasoning, knowledge, agentic 능력 등 텍스트 기반 지능이 심각하게 손상됨.
  • 해결: ##Audex는 Nemotron-Cascade-2-30B-A3B (MoE, Mamba2-Transformer hybrid)를 backbone으로 하는 unified audio-text 모델. Audio input은 encoder+MLP로 text embedding space에 project되고, output side는 vocabulary를 확장해서 audio codec token과 text token을 uniformly 취급##.
  • 왜 작동하는가: (1) 강력한 text-only pretrained backbone에서 시작, (2) multi-stage SFT curriculum (text → audio warmup → audio gen → audio gen+understanding)으로 catastrophic forgetting 완화, (3) text-only Cascade RL + MOPD 적용 후에도 audio 태스크 성능이 유지되는 특성 활용.
  • 비유: 국어 특기생을 음악 특기생으로도 훈련시킬 때, 국어 실력을 그대로 유지하면서 점진적으로 음악 능력을 layer 별로 쌓아 올리는 커리큘럼. 훈련이 끝나도 국어 시험 성적은 그대로.

핵심 기여 (Key Contributions)

  1. Text intelligence regression 없는 unified audio LLM: ##Audex-30B-A3B는 강력한 backbone Nemotron-Cascade-2 대비 reasoning/knowledge/alignment/long-context/agentic 모든 텍스트 벤치마크에서 marginal-or-zero regression##으로 SOTA audio 성능 달성.
  2. 단일 통합 아키텍처: cascaded thinker-talker 없이 하나의 MoE Transformer decoder가 audio input(encoder + MLP projection)과 text/audio output token을 uniformly 처리. Megatron-LM, vLLM 같은 기존 LLM 인프라와 호환.
  3. 포괄적 능력: audio understanding, ASR, speech translation, TTS, audio generation, S2S generation을 하나의 모델로. Open 모델 중 유일하게 speech를 넘어서는 general audio generation 지원.
  4. Multi-stage SFT curriculum과 empirical 분석: text SFT → audio warmup → audio gen SFT → audio gen + understanding SFT의 stage-wise 학습 vs single-stage consolidated SFT 비교. ##Multi-stage가 stability와 long-context 유지 측면에서 우월##.
  5. Cascade RL + MOPD의 audio 태스크 non-regression: text-only RL을 audio 모델에 적용해도 audio 성능이 유지 또는 marginal 개선. Cascade RL이 catastrophic forgetting에 강한 이유가 audio 태스크에도 확장됨을 보임.

1. Introduction — Text Intelligence Regression 문제

1.1 배경: Audio LLM의 현주소

Audio는 음성, 음악, 환경 소리를 포함하는 AGI의 필수 modality. 최근 audio understanding 중심 모델(Audio Flamingo, Qwen2-Audio, Kimi-Audio, Voxtral, Step-Audio 2)과 unified 이해+생성 모델(Qwen-Omni, MiMo-Audio, UALM)이 등장.

1.2 문제: Text Intelligence의 감퇴

Multimodal input/output을 지원하는 LLM들은 특히 multimodal generation을 지원할 때 text 벤치마크에서 눈에 띄는 regression을 보임. 예시: Qwen3-Omni는 text-only counterpart Qwen3 대비 reasoning 벤치마크에서 눈에 띄는 감퇴. Qwen3.5-Omni도 유사. Qwen3-Omni technical report Table 5는 이를 자체 문서화.

##이는 심각한 문제##: reasoning, knowledge, tool use 같은 agentic 능력의 대부분이 text에 encoded 되어 있으므로, text intelligence regression은 곧 전반적 지능 감퇴를 의미. Frontier proprietary LLM (GPT-4o, Gemini 2.5)는 이를 해결했지만 open 모델(DeepSeek-V4, GLM-5.1)은 여전히 text-only. Qwen3.5, Kimi-2.5는 vision input은 있지만 audio output은 미지원.

1.3 Audex의 위치

Audex는 text-only Nemotron-Cascade-2-30B-A3B (30B MoE Mamba2-Transformer hybrid, 3B activated) 위에 audio 능력을 추가하되, ##backbone의 text intelligence를 marginal-or-zero regression으로 preserve##. Cascaded 시스템도 아니고 thinker-talker도 아닌 single unified MoE Transformer decoder. Audio input은 encoder + MLP projection으로 text embedding space에 fusion, audio output은 vocabulary 확장으로 text와 동일하게 취급.

2. Main Results (Overview)

Audex-30B-A3B를 두 축으로 평가: (i) text-only 능력 (reasoning, knowledge, alignment, long-context, agentic) 대비 LLM backbone, (ii) audio 능력 (audio understanding, ASR, speech translation, TTS, audio generation, S2S).

3. Model Architecture

3.1 LLM Backbone

Nemotron-Cascade-2-30B-A3B: hybrid Mamba2-Transformer MoE. 52 layers, model dimension 2688, 128 routable experts, 6 activated. Audex는 Cascade-2 SFT 후 체크포인트에서 초기화. Vocabulary는 131,072 → 205,312로 확장 (audio token 수용). 추가 토큰 임베딩은 N(0, 0.02²) Gaussian으로 초기화.

Audex-2B는 dense 2B Transformer backbone (28 layers, dim 2048, MLP intermediate 9216), Cascade-2-30B-A3B와 동일 recipe로 학습.

3.2 Audio Encoder

Audio Flamingo 3의 AF-Whisper (Whisper Large-v3 파생, 일반 audio로 확장 fine-tune) 사용. 16kHz input, 30초당 25Hz output (30초 = 750개 특성), 차원 1280. 30초 window로 split, 마지막 partial window는 zero-pad. 2-layer MLP adapter가 audio feature를 LLM model dimension으로 변환.

3.3 Audio Codec

Audex는 speech와 non-speech에 서로 다른 codec을 dispatch. 저자들의 핵심 통찰: speech는 단일 layer quantizer로 낮은 tokens-per-second가 효율적, non-speech는 multi-layer quantizer (RVQ 등)로 더 높은 tokens-per-second target이 필요 (복잡한 auditory structure).

3.3.1 Speech Codec — X-Codec2

  • 50Hz frame rate
  • Single-layer finite scalar quantization (FSQ), 65,536 codebook size
  • 즉, 50 tokens/s × 65,536 vocab

3.3.2 Non-Speech Codec — X-Codec

  • General-purpose audio codec, 50Hz
  • 8-layer RVQ (Residual Vector Quantization), layer당 1,024 codebook
  • UALM 방식 따라 첫 4개 RVQ layer만 사용 (token 수 감소)
  • Copet 등의 flattening pattern으로 2D RVQ token을 text처럼 1D flatten
  • 50 × 4 = 200 tokens/s, 1024 × 4 = 4096 vocab size

3.3.3 Vocabulary 구성

Speech/audio token은 <speechcodec_{id}>, <audiocodec_{id}>로 vocabulary에 append:

  • 65,536 (speech) + 8,192 (non-speech, 전체 8×1024 예약) = 73,728 audio-related
  • + <speechgen_start>, <speechgen_end>, <audiogen_start>, <audiogen_end>, <audiogen> 특수 token
  • Total = 131,072 + 65,536 + 8,192 + 5 = 204,805
  • Megatron-LM tensor parallelism 위해 512로 나뉘도록 205,312로 padding

3.3.4 Audio Decoding

  • Non-speech (X-Codec): enhancement VAE를 별도 학습 (BigVGAN-v2 기반), 16kHz → 48kHz upsample + codec artifact 감소
  • Speech (X-Codec2): X-Codec2의 non-causal decoder를 Vocos 기반 causal ConvNeXt streaming decoder로 대체. Small look-ahead window로 high semantic accuracy와 fast streaming inference 확보

4.3 Training Loss와 Classifier-Free Guidance

4.3.1 Loss 함수

Standard LLM loss:

$$\max_\theta \left\{ \frac{1}{\text{\#Trainable tokens}} \sum_{i: x_i \text{ is trainable}} \log p_\theta(x_i \mid x_{Packed sequence의 경우, 각 sample j의 loss vector를 L_j, 길이를 |L_j|라 하면 square averaging loss 사용:

$$\text{Scaled-Loss}(\{L_1, \dots, L_k\}) = \left( \sum_{j=1}^{k} \frac{\text{sum}(L_j)}{\sqrt{|L_j|}} \right) \cdot \left( \sum_{j=1}^{k} \sqrt{|L_j|} \right)^{-1} \tag{2}$$

##이는 short와 long sample 간 loss 기여를 균형화##. 짧은 샘플이 loss에 과도한 영향을 미치는 것을 방지 (일반적인 mean averaging의 단점을 완화).

4.3.2 MoE Load Balancing

Audex-30B-A3B의 MoE layer는 auxiliary-loss-free load balancing: update rate 10^-3, top-K routing scale 2.5, standard load balancing loss는 계수 10^-6로 minimal한 역할.

4.3.3 Classifier-Free Guidance (CFG)

CFG 활성화를 위해 unconditional distribution p_θ(·|∅)도 학습. Inference 시 logit 수정:

$$p^{\text{CFG}}_\theta(\cdot | x_{UALM 방식 따라 ∅는 i-1 consecutive un-trainable padding token으로 설정 (x_{와 동일 길이). CFG 학습을 위한 pre-processing 접근: speech/audio generation data 10%를 randomly 선정, transcription/caption을 random 수의 padding token으로 변환하여 training dataset에 blend. On-the-fly conditional flipping을 피해 parallelism 구현 단순화.

4.4 Training Stages

4.4.1 Supervised Fine-Tuning

Audio- 또는 speech-related pretraining 없이 best-in-class audio LLM 구축이 가능하다는 점을 실증. Text-only pretrained LLM에서 시작해 두 SFT 전략 비교.

Multi-stage SFT strategy

  1. Text SFT: Nemotron-Cascade 2 SFT와 동일. 이미 SFT된 체크포인트를 직접 재사용
  2. Audio Warmup: vocabulary 확장 (audio + speech token), audio encoder 및 initialized MLP adapter 추가. ##이 stage에서는 MLP adapter와 audio token embedding만 학습##. Text token embedding freeze (학습 시 text quality 저하 관찰). Trainable modules: MLP adapters + Audio Token Embeddings.
  3. Audio Generation SFT: 전체 LLM unfreeze, audio/speech generation data + text-only data. Audio understanding, ASR, AST는 이 stage에 미포함 (audio/speech generation이 완전히 새로운 task라 더 많은 학습 필요).
  4. Audio Generation + Understanding SFT: MLP adapters와 LLM unfreeze, audio understanding + ASR + AST + audio generation + text-only. Audio encoder는 frozen 유지.

Single-stage consolidated SFT strategy

Multimodal 데이터를 early training stage(예: pretraining)에 포함하는 최근 연구들에 영감. Pretrained base LLM에 audio warmup 직접 적용 후, 모든 SFT 데이터 (audio generation + audio understanding + text)를 single SFT stage에서 결합.

4.4.3 Cascade RL + Multi-Domain On-policy Distillation

Multi-stage SFT 체크포인트를 Audex SFT로 삼고 RL 진행. Text-only RL만 실시. Cascade 2와 유사한 RL:

  1. Instruction-Following RL
  2. Multi-domain RL
  3. Multi-domain On-policy Distillation (MOPD)
  4. Reinforcement Learning from Human Feedback (RLHF)
  5. Long-context RL

##흥미로운 발견: audio 및 speech-related task는 text-only RL 이후에도 marginal 또는 zero regression을 보이며, text task에서는 significant 개선##. Nemotron-Cascade 4.1.1절 "Why Cascade RL for LLMs Is Resistant to Catastrophic Forgetting"이 이유를 설명. 이는 audio-text RL 없이도 text-only RL로 unified model의 text intelligence를 회복/강화할 수 있는 promising direction. 저자는 audio-text RL을 future work로 남김.

4.5 Infrastructure

  • Training: Megatron framework + Transformer Engine. 4-way TP, 32-way EP (expert parallelism), 8-way CP (context parallelism), sequence parallelism. Context length 262,144. 512× NVIDIA H100 GPU, BF16, AdamW, gradient clipping 1.0.
  • Data Loading: Megatron-Energon multimodal dataloader, balanced greedy knapsack packing. 8,192 packing buffer size, num_workers=4 per DP rank. TTS/TTA는 pretokenized (base64 encoded) — audio codec forward pass overhead 방지.
  • CFG: unconditional sample을 미리 선정 (padding token으로 text 대체), original dataset과 blend. Inference 시 vLLM에서 conditional/unconditional pair를 same batch로 submit — custom logits processor가 pair를 track하고 Eq. (3)로 next-token logit 결합.

6. Related Work (요약)

6.1 Unified Understanding + Generation

Vision domain에서는 활발히 연구됨(NVLM 등). 근본적 challenge: multimodal generation과 text intelligence를 동시에 최적화하기 어려움. Multimodal data를 early stage(pretraining)에서 fuse하면 도움되고, MoE 아키텍처도 도움. Diffusion head를 붙이는 접근도 있으나 Audex와는 orthogonal.

General audio domain에서는 unification이 덜 연구됨. UALM이 첫 걸음: proper data blending과 recipe로 SOTA text-to-audio generation, strong understanding, small text degradation 달성. UALM 한계: (i) 세 개 text 벤치마크만 평가, saturate됨, (ii) text RL 미연구.

Audex는 UALM 대비 text-to-audio, audio understanding 개선, TTS와 ASR 지원 추가, ##text SFT+RL의 효과 comprehensive study##.

6.2 Speech LLMs

ASR, AST, TTS, S2S를 다루는 speech-focused LLM들: Chen et al., Défossez et al., Huang et al., Kharitonov et al. 등. Speech output은 codec model (VQ-VAE style)의 discrete token으로 표현. RVQ, FSQ, 다양한 codebook design과 parallelized token pattern 사용. Duplex model은 별도 line (speak-while-listen).

6.3 Audio Understanding Models

Speech LLM을 non-speech sound (music, environmental sound)로 확장. Continuous representation input을 audio encoder에서 받는 것이 일반적. Discrete audio codec을 input으로 하는 시도도 있으나 quality gap 존재.

6.4 Audio Generation Models

Text-to-image와 유사, general audio에 확장. Early: LLM으로 discrete audio token 예측. Recent leading approach: diffusion 기반 (sample quality, controllability, duration, inference speed에서 dominance). UALM이 LLM-based로 gap을 좁힘. Audex는 UALM 기법을 30B MoE로 scale up.

비판적 분석 (Critical Analysis)

강점

  • Text intelligence 보존의 방법론적 명료함: Multi-stage SFT curriculum과 text-only Cascade RL의 결합으로 catastrophic forgetting을 comprehensive하게 다룸. Qwen3-Omni 등 경쟁 모델과의 격차가 특히 reasoning/agentic 벤치마크에서 명확 (예: NIAH 99.4% vs 0.0%).
  • 단일 통합 아키텍처의 실용성: Thinker-Talker 없이 하나의 MoE Transformer가 모든 task를 수행. Megatron-LM/vLLM 기존 infrastructure 사용 가능한 점은 배포 관점에서 큰 장점.
  • Multi-stage vs Single-stage의 clean ablation: NIAH를 통해 single-stage가 long-context attention을 파괴한다는 것을 직접 보여준 것이 매우 informative. "총 훈련 budget 절감 vs stability" trade-off를 데이터로 제시.
  • Text RL의 non-regression 발견: audio-text RL 없이 text-only RL만으로도 unified model의 성능을 강화할 수 있다는 것은 실용적 가치가 큼. Cascade RL의 catastrophic forgetting 저항성이 audio에도 확장됨을 실증.
  • Public 체크포인트 공개: Audex-30B-A3B와 Audex-2B 모두 공개 — reproducibility에 기여.
  • Speech/non-speech codec의 분리 dispatch: 각 domain의 특성에 맞는 codec (single-layer FSQ vs 8-layer RVQ) 선택이 clean한 설계 결정.

약점 및 의문점

  • Speaker similarity (SIM) 저조: Table 6에서 Audex 30B-A3B의 SIM은 45.3 — Seed-TTS 76.2, F5 TTS 64.7 대비 현저히 낮음. 저자도 codec 선택과 training recipe가 이를 optimize하지 않는다고 인정. 실사용에서 voice cloning이 필요한 경우 큰 단점.
  • AST에서 Voxtral 대비 약함: Voxtral-Small-24B가 BLEU 41.0 vs Audex 30B-A3B 34.0. Multilingual speech translation에서는 여전히 speech-specific 모델이 우세.
  • Audio codec 크기의 임의성: X-Codec2 (65536 vocab)와 X-Codec (4096 vocab × 4 layers)의 선택 근거는 UALM 관행에 기댐. 다른 codec design에서 성능 차이가 얼마나 될지 불명확.
  • TTA output length 고정 (10초): variable length가 "학습 불안정"이라고만 명시. Duration control이 자유롭지 않아 실용 시나리오에서 제약.
  • MMAR, MMSU에서 Qwen3-Omni-Thinking에 뒤짐: audio-language reasoning의 특정 subset (music, speech understanding)에서 unified 접근이 specialized 모델보다 열등. Audex의 general-purpose 강점이 모든 audio task에 확장되지는 않음.
  • Audio-text RL 미실행: 저자가 future work으로 남김. Text-only RL만으로 얻은 gain의 한계가 어디까지인지 미검증. Audio 태스크에서 RL이 진짜 필요한 시나리오 (예: dialogue generation)를 다루지 않음.
  • Prompting TTS의 SIM 저조와 fixed-voice의 실용성 제약: fixed-voice TTS 1.70 WER는 인상적이지만 하나의 voice로만 speech 생성. Multi-voice 시스템에는 부적합.
  • Data licensing이 detail하지 않음: "permissively licensed"라는 표현이 반복되나 구체적 dataset 리스트가 completeness 측면에서 부족. 재현 시 정확한 데이터 확보의 어려움.
  • Text SFT weight의 임의성: 각 SFT stage에서 text 데이터 blend ratio (44%, 59%, 69%, 75%)의 결정 근거가 설명되지 않음. Hyperparameter search 과정 공개가 필요.

재현성

  • Positive: 30B-A3B와 2B 모두 공개, Megatron-LM/Megatron-Energon 오픈소스 사용.
  • Negative: 학습에 512× H100 필요 (compute barrier), in-house data curation의 세부 recipe (필터링 기준, noise 합성 파이프라인)이 부재.
  • Base backbone Nemotron-Cascade-2-30B-A3B는 별도 공개 여부에 의존.

Key Takeaways — 이 논문에서 가져갈 것

  • "Multimodal ≠ regression on text": 강력한 text LLM에서 시작하여 (1) MLP adapter + token embedding warmup, (2) LLM unfreeze로 gradual한 audio task 확장, (3) text-only RL로 마무리 — 이 조합으로 text intelligence를 marginal loss로 유지 가능. Design principle로 유용.
  • Single-stage consolidated SFT의 함정: multimodal SFT를 한꺼번에 하면 total training budget은 줄어들지만 long-context attention이 파괴됨 (NIAH 99% → 0%). Total budget 최소화가 항상 우월한 것은 아님.
  • Speech와 non-speech codec의 분리 dispatch: 두 도메인의 특성 차이 (speech = 낮은 tokens/s, non-speech = 높은 tokens/s가 필요)를 고려한 codec 선택이 unified model의 quality-efficiency 균형을 개선.
  • Text RL이 audio task에 regression을 일으키지 않는다: Cascade RL의 catastrophic forgetting resistance가 audio-text unified model에도 확장됨. Audio-text 공동 RL 없이도 unified 모델 강화 가능.
  • CFG는 TTA에 필수, TTS는 optional: text-to-audio는 λ ∈ [3, 5]에서 quality 최적, text-to-speech는 CFG 없이도 좋음. Inference 비용 대비 필요한 task에만 적용.
  • Audio understanding에서 unified 접근의 우세 영역: deductive reasoning (Audio Entailment)과 hallucination detection (CMM)에서 unified LLM이 audio-specific 모델보다 강함 — 텍스트 reasoning 능력이 audio에 transfer.
